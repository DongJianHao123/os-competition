#### 3.1.1 详细模块介绍

**openharmony 板级适配模块**：该项目最核心的模块，为 Ascend 310B 开发板适配 OpenHarmony 5.0.3.135 操作系统。board_310B 包含内核构建脚本（build_kernel.sh、make-boot.sh、make-ohos.sh）、内核补丁（linux-5.10 的 43 个补丁，涵盖 GPU panfrost、mesa3d、设备树、音频、摄像头、HDF 等）、启动配置（init.cfg、init.310B.cfg）、音频 ALSA 适配、摄像头 v4l2 驱动适配、分布式硬件配置等。soc_310B 提供 SoC 级显示、GPU、RGA、x264 硬件支持。vendor_310B 包含产品参数、HDF 驱动框架配置（khdf/uhdf）、音频策略、图像分区、预安装应用列表、资源调度、安全配置、窗口管理等完整的产品级配置。

**CANN 部署模块**：提供在 OpenHarmony 板端部署 CANN 8.2.RC1 推理运行时的自动化脚本。install_cann_openharmony.sh 自动检测 USB 挂载的离线包、解压安装 CANN 运行时库（libascendcl.so、libruntime.so 等）、部署 Python 3.10 运行时。set_cann_env.sh 设置 PATH、LD_LIBRARY_PATH、ASCEND_HOME 等环境变量，并配置日志级别以减少推理开销。

**YOLOv5 推理模块**：基于 Ascend OM 模型格式的 YOLOv5 目标检测推理程序。main.py 实现完整的推理流水线：图像加载 → letterbox 预处理（640×640）→ NPU 推理（InferSession）→ NMS 后处理 → 坐标变换 → 绘制边界框和标签 → 保存结果。det_utils.py 提供 letterbox 缩放填充、xyxy/xywh 坐标转换、non_max_suppression、scale_coords 等标准 YOLOv5 工具函数。测试输入 world_cup.jpg 可成功检测出人物和足球。

**OpenHarmony 图形应用模块**：基于 ArkTS 和 ArkUI 框架开发的图形界面应用，用于在 OpenHarmony 桌面展示 YOLOv5 检测结果。主界面 Index.ets 显示标题「YOLOv5 Detection Result」、检测结果图片和「OpenHarmony + Ascend 310B NPU」平台说明。已签名 HAP 可安装到开发板并在显示器上展示。
