#### 3.1.1 详细模块介绍
项目实现了一个**证据驱动（evidence-driven）** 的小型操作系统内核分析与比对智能体系统。核心流程为：analyzer 对目标仓库进行确定性扫描，提取文件、函数、能力信号等结构化事实（ProjectFacts）；indexer + search 构建并检索历史仓库索引，找出 Top-K 相似候选；comparator 计算多维相似度；agents.py 编排 8 个 Agent 按序执行（源分析→历史检索→相似度分析→证据审查→RAG 上下文→LLM 报告→报告规划），每一步生成可审计的 AgentStep 记录；reporter 将结果输出为 Markdown 报告。系统设计强调**可审计性**和**幻觉控制**，所有结论均基于代码中提取的确定性事实，LLM 仅作为可选增强。
