#### 3.1.1 详细模块介绍
系统"灵枢"（LingShu）是一个面向 Linux 内核宕机诊断与上游补丁匹配的端到端系统。支持三种输入模态（dmesg 日志、vmcore 转储文件、自然语言描述），通过四个核心模块实现从问题特征提取到补丁推荐的全流程。

**模块一（异构问题特征提取）**：核心代码位于 `src/lingshu/extractors/`，包含 dmesg_parser.py（1813行，正则解析内核崩溃特征）、vmcore_analyzer.py（38798字节，分析 vmcore 转储）、llm_enhancer.py（15362字节，LLM 增强特征）、problem_parser.py（24495字节，问题描述解析）、syzkaller.py（12473字节，syzkaller 支持）、text_signals.py（8083字节，文本信号提取）、source_context.py（11626字节，源码上下文）、benchmark.py（46317字节，基准测试）。该模块输出标准化的 crash_feature JSON，包含 labels、entities、summary、payload.call_trace、payload.runtime_context 等字段。

**模块二（补丁数据湖与索引）**：核心代码位于 `src/lingshu/patch_ingest/`，包含 commit_parser.py、filtering.py、enrich.py、git_source.py、models.py、normalize_pipeline.py、patch_feature_export.py、possible_patch_filter.py、raw_export.py、serialize.py、vector_index.py、checkpoint.py 等。从 Linux v3.10+ 上游导出 commit，生成 patch_feature JSONL，经 Qwen3-Embedding-8B 向量化后存入 Milvus（HNSW 索引 + 标量元数据）。

**模块三（混合语义检索）**：核心代码位于 `src/lingshu/retrieval/`，包含 retriever.py、milvus_store.py、vector_db.py、vector_store.py。使用 strict/relaxed/none 三级自动降级策略，结合向量查询与 metadata filter 召回 Top-50 候选。

**模块四（重排与可解释输出）**：核心代码位于 `src/lingshu/rerank/` 和 `src/lingshu/advisor/`，包含 cross_encoder.py、local_cross_encoder.py、model_cross_encoder.py、evidence.py、diagnostic_advisor.py、llm_review_provider.py。通过 Cross-Encoder、规则证据打分和 Advisor 生成 Top-3 推荐，包含 commit id、分数、根因解释、证据列表和风险标记。
