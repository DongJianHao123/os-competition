#### 3.1.1 详细模块介绍
**task1_transplantation（系统移植）**：团队将 OpenHarmony 5.0.3 标准版适配到启智01开发板（昇腾 310B）。通过补丁方式修改了设备板级配置（BUILD.gn、init.cfg、网络设置、环境变量）、YOLOv5 服务端和推理主程序、显示驱动（强制 1080p 分辨率）、以及 EGL 函数动态加载修复。补丁按顺序应用，并提供一键编译脚本和常见问题解决方案。

**task2_cann（CANN 推理引擎移植）**：团队在 OpenHarmony 上部署了华为 CANN AI 计算框架。提供了从 Windows 主机通过 HDC 部署脚本到设备的完整流程，包括 CANN 安装包推送、安装、环境变量设置、Python 依赖安装（torch、torch_npu、ais_bench 等）以及 NPU 验证。验证脚本使用 pyACL API 确认运行时正常。

**task3_yolov5（YOLOv5 目标检测应用）**：团队开发了完整的 YOLOv5 推理应用，包含三个部分：1）CLI 推理程序（main.py）支持单张/批量推理，输出检测结果图像和 JSON；2）HTTP 服务器（server.py）提供 RESTful API 接口；3）OpenHarmony 图形应用（ArkTS）提供可视化界面。模型使用 YOLOv5s 权重，通过 ONNX 导出并转换为昇腾 OM 格式。提供 135 张测试图片和检测结果示例。

**task4 性能优化**：团队对系统进行了内存和启动时间两方面的优化。内存优化包括：MMZ 内存从 3072M 缩减至 2048M、模块按需加载（蓝牙、无障碍、memmgr 等）、ZRAM 调优（lz4 算法、512MB）、VM 参数调整（min_free_kbytes=64MB），整体节省约 1.1GB 物理内存，启动内存占用从 1.7GB 降至约 0.9GB。启动时间优化通过延迟启动非关键服务（slogd、sklogd 等），将系统启动时间从 50s 缩短至约 42s（优化 16%）。提供 bootchart 启动分析图。
