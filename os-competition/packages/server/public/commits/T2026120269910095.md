#### 3.1.1 详细模块介绍
项目采用内核态 eBPF 探针 + 用户态 Go 分析引擎的经典架构。内核态包含 Stage1（8 个 tracepoint 探针，覆盖 execve/exit/openat/unlink/connect/clone/write 系统调用及心跳监控）和 Stage2（4 个 uprobe 探针，同时支持 OpenSSL 和 Go crypto/tls 的 HTTPS 明文截获）。用户态采用 6 级 Processor 流水线链（Decoder → HeartbeatProc → AgentClassifierProc → StaticRuleProc → WindowProc → AlertEmitterProc），结合四层递进规则引擎（Pattern/Threshold/Baseline/Composite）和动态基线学习（Median+MAD），最终输出 JSONL 告警（含 causal_chain 归因链路）。PID 身份验证系统设计了五层递进验证机制，从 Profile 配置到二进制指纹再到运行时完整性巡检，有效防伪装攻击。跨 Agent 检测通过全局 CrossAgentState 跟踪文件访问竞争。测试套件覆盖 6 种攻击场景和 4 种性能基准。
