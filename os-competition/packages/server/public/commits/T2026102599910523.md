#### 3.1.1 详细模块介绍

本项目ebpfagent是一个基于eBPF的AI Agent行为安全监控系统，分为Rust和Go两大部分。

**Rust端**：zhy-ebpf模块使用Aya框架编写eBPF内核程序，通过tracepoint挂载点监控execve、openat、connect、unlinkat、clone、fork、vfork等关键系统调用，并通过uprobe监控SSL_write和SSL_read以捕获AI Agent与LLM通信的明文数据。仅采集进程名以"agent"开头的进程，确保监控目标聚焦。zhy-collector作为用户态采集器，通过RingBuf异步读取eBPF事件，按(pid, event_type, path)复合键在1秒窗口内聚合，输出JSON行到stdout。xtask提供便捷的构建和运行命令。

**Go端**：main.go加载config/config.yml配置，初始化内存存储（可选Redis），注册11个安全检测器（safety_handle）和6个逻辑/RL检测器（rl_handle），组装Pipeline事件处理管线。事件流经隔离注入层后，所有检测器并发执行，结果通过告警输出模块打印。安全检测器覆盖shell执行、特权提升、敏感文件访问、无文件执行、进程注入、ptrace调试、preload劫持、进程内存读取、sudo越权、工作区违规等场景。逻辑检测器覆盖高频API调用、重复prompt/死循环、I/O浪费、目标锚定偏离、信息隐瞒、提前终止等AI Agent特有行为异常。
