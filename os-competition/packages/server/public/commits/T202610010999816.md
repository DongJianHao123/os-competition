#### 3.1.1 详细模块介绍
**API层**：VectorStore是L0-L5分层写入/召回/精排的总入口，内部包含WriteBuffer、WorkspaceCache、HotVectorIndex、SegmentManager、PersistentStorage、DualWAL、CompactionManager等子系统。关键API包括insert、batch_insert、search、batch_search、build_index、save/load、evict_cold_segments。L5磁盘后备机制使SIFT1B查询内存仅占数据总量6.6%。AgentMemory是面向Agent对话场景的高层封装，管理对话生命周期，包含G-LRU混合淘汰策略和Token Budget管理。ContextCompressor负责对话历史摘要压缩。

**索引模块**：QuantizedVamanaIndex实现了Buffered Vamana冷段构建，包含Phase0-D五阶段（Find Medoid→Parallel Search+Prune→Merge Reverse Edges→Light Repair→Quality Refinement）。核心算法包括RobustPrune、FastRobustPrune（有界候选集剪枝，降低构建时间16%-29%）、Degree Guard三级保护、ImmediateBufferedPrune（反向边缓冲剪枝，降低Zero Indegree 99.8%）、Light Repair（定向修复）、Quality Refinement（渐进式质量精修，Recall提升2.2pp）。图搜索使用GreedyBeamSearch三种变体，epoch-based visited marker实现O(1)访问标记。量化器支持ScalarQuantizer (SQ8, 4×压缩)和PQQuantizer（含OPQ旋转矩阵）。

**存储层**：WriteBuffer为L0高并发写缓冲，16分片CAS原子大小追踪、背压控制、批量Flush。SegmentManager管理L3不可变段生命周期（Create→Build→Load→Search→Evict→Compact→Delete），支持多段并行搜索、跨段桥接（anchor采样）、分层存储主动驱逐。VamanaSegment封装单个不可变Vamana段，包含量化图索引、全精度向量文件、RerankCache、段元数据。DualWAL双重预写日志（Intent Log + Commit Log）自适应Group Commit，WALEngine记录格式CRC32+length+type+payload。

**I/O层**：IOEngine统一封装同步/异步读取，支持Linux io_uring、Windows OVERLAPPED、macOS工作线程池/同步回退，batch_readv批量读取。BlockCache用户态块缓存池，4KB块大小，Clock淘汰，O(1)淘汰，预取标记。GraphAwareCacheManager采用TinyLFU风格准入+SLRU淘汰，图拓扑感知准入（hub节点权重）。PipelineController段级预取与平台I/O控制，流水线模式降低冷段阻塞。SegmentLayout基于BFS的节点重排序优化I/O局部性。
