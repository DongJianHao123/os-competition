#### 3.1.1 详细模块介绍
LIM-HNSW系统采用五层架构：Agent层→访问层→VectorEngine引擎层→HNSW/PQ/rerank索引与计算层→数据文件与日志存储层。核心设计思路是将HNSW图和PQ码保留在内存中用于导航和近似距离估计，全量向量存储在SSD上，仅对有限的重排序集读取全向量。查询路径：query embedding → PQ查找表 → HNSW多层导航 → 合并memtable/最近候选 → 去重过滤 → 缓存或SSD全向量读取 → 精确L2重排序 → Top-K ID → 恢复原始文本。写入路径：记住文本 → 生成embedding → 追加memory_texts.jsonl → 追加WAL → 更新memtable → 通过fresh overlay暴露 → flush → 追加data.bin → 更新偏移量和PQ → 增量HNSW插入 → 持久化hnsw.bin → 重置WAL。
