#### 3.1.1 详细模块介绍

**baec-xpu-tee-research 模块**：该模块是一个OP-TEE安全研究测试工具集，包含一个CA（host/baec_xpu_tee_research.c，516行）和多个TA。CA提供5个测试用例：test_ta_fs（测试TA持久化文件系统）、test_ta_data_io（从1MB到1GB逐步增加共享内存大小进行压力测试）、test_ta_data_io_replay（模拟中间人重放攻击，验证TA能否检测重放）、test_ta_isolation（测试TA内存隔离，让TA_B读取TA_A的私有数据地址）、test_ns_access_sec（测试非安全世界访问安全内存区域）。TA部分包含主TA（baec_xpu_tee_research_ta.c，325行）实现inc_value、test_fs、test_io三个命令，以及两个隔离测试TA（ta_isolation_a/b）。开发过程从2026年6月25日首次提交框架，到6月27日完成核心功能，再到6月30日完善README，呈现逐步演进。

**demo 模块**：最简单的OP-TEE入门示例，CA调用TA执行1+2加法运算。CA代码49行，TA代码50行，结构清晰完整，适合作为OP-TEE开发的起点模板。

**trusted-ai-platform-optee 模块**：项目的核心模块，实现了一个基于Docker容器化的可信AI平台。包含4个CA角色和4个TA角色：
- CA Server：证书签发服务，通过文件IPC接收证书请求，调用外部trusted_lib库签发X.509证书
- TEE Worker：远程证明服务端，处理challenge-sync/ack握手、digest验证、本地度量请求
- Code Provider：远程证明客户端—代码提供方，包含可选的MITM重放攻击模拟
- Data Provider：远程证明客户端—数据提供方，发送数据分片并验证签名

4个容器通过共享volume `/tmp/optee_e2e_ipc/`下的`.req`/`.rsp`文件进行IPC通信，端到端流程分为四个阶段：证书签发→TaskId协商→远程证明→本地度量。Docker部署脚本（docker-deploy.sh，336行）功能完善，支持自动/手动编排、前置条件检查、native模式回退。

### 4.2 遭遇的代码问题

| --- | --- |
| 飞腾派Docker运行需要cgroup v1，但默认内核使用cgroup v2且缺少BPF支持 | 在U-Boot bootargs中添加`systemd.unified_cgroup_hierarchy=0`回退cgroup v1 |
