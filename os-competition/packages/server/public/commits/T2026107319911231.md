#### 3.1.1 详细模块介绍

**基础命令模块**：实现了 TCM2 平台的基础生命周期管理命令，包括 Startup（启动平台）、Shutdown（关闭平台）、GetCapability（查询能力）、GetRandom（获取随机数）、Hash（SM3 摘要计算）。所有命令均通过 CLI、TCP JSON、HTTP REST 三种接口提供，统一错误码处理。开发从 2026-06-26 初始提交开始，逐步完善参数校验和错误处理。

**PCR 度量模块**：实现了 PCR_Read 和 PCR_Extend 命令，支持 24 个 PCR（索引 0-23），每个 PCR 32 字节，初始值为全 0。PCR_Extend 使用 SM3(old_pcr || input_data) 进行链式扩展，形成完整性度量链。支持正常启动度量和篡改检测两种场景，篡改后 PCR 值变化可被检测。

**事件日志模块**：实现了 EventLog（查询 PCR 扩展事件）和 VerifyEventLog（事件日志重放验证），通过重放事件日志重新计算 PCR 值并与当前 PCR 比较，验证完整性度量链的一致性。

**远程证明模块**：实现了 Quote（生成证明摘要）和 VerifyQuote（验证证明），Quote 计算方式为 SM3(pcr_index_list || selected_pcr_values || nonce)，将 PCR 值和 nonce 绑定，防止重放攻击。验证时重新计算期望 Quote 并与输入比较，nonce 错误或 PCR 变化时验证失败。

**策略封存模块**：实现了 Seal（封存）和 Unseal（解封），将秘密绑定到 PCR 策略摘要。封存时读取指定 PCR 计算策略摘要，打包为密封 blob；解封时重新计算策略摘要并验证 blob 完整性。PCR 匹配时解封成功（输出 UNSEAL_SUCCESS），PCR 被篡改时解封失败（输出 PCR_POLICY_MISMATCH_DETECTED）。

**密钥对象管理模块**：实现了完整的密钥生命周期管理，支持创建（CreateKey）、列出（ListKeys）、查询详情（GetKeyInfo）、导出公钥（ExportPublic）、删除（DeleteKey）操作。支持 SM2-SIM（签名密钥）和 SM4-SIM（对称加密密钥）两种密钥类型，私钥和 key_secret 不可导出（返回 PRIVATE_KEY_NOT_EXPORTABLE）。

**国密算法接口模块**：SM3 为纯 Python 实现，用于 PCR、Hash、Quote、Seal、EventLog 和密钥派生。SM2-SIM 和 SM4-SIM 为模拟接口，聚焦于 TCM/TPM 命令流程和密钥对象边界，计划后续对接 GmSSL 或 OpenSSL 国密支持。

**TCP JSON 服务模块**：基于 Python socketserver 的 ThreadingTCPServer 实现，默认监听 127.0.0.1:2321。支持 24 个命令，客户端发送一行 JSON 命令，服务端返回一行 JSON 响应。多线程处理并发连接，错误处理完善。

**HTTP REST 服务模块**：基于 Python http.server 实现，默认监听 127.0.0.1:8080。提供 13 个 REST 端点（GET /capability、GET /pcr/{index}、POST /command 等），REST 路径自动转换为内部 JSON 命令，与 TCP JSON 服务共用命令分发逻辑。

**Web Dashboard 模块**：位于 web/tcm2_dashboard/，包含 index.html、styles.css、app.js。三栏大屏布局，展示平台状态、PCR 值、事件日志、密钥对象等信息。只读展示，不展示私钥等敏感材料。

**自动化测试模块**：基于 Python unittest，共 53 个测试用例，覆盖 CLI 命令、TCP 服务、HTTP 服务、并发客户端、密钥对象、度量启动、远程证明、策略封存、事件日志、错误码等。一键运行通过（输出 OK）。

**演示脚本模块**：提供 11 个演示脚本，包括一键验证（run_all_demos.sh）、项目展示（demo_for_judges.sh）、分模块演示等。每个脚本输出明确的成功标记（如 ALL_TCM2EMU_DEMOS_PASSED、TCM2EMU_JUDGE_DEMO_PASSED 等）。
