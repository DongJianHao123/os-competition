#### 3.1.1 详细模块介绍

**Collector模块**：离线数据治理的核心入口，使用PyDriller流式遍历Linux Kernel Git仓库（145万+ commits），以O(1)内存消耗完成全量commit采集。通过parser子模块进行commit message和diff的结构化解析，subsystem子模块基于文件路径和主题识别28个内核子系统，bugtype子模块通过关键词匹配识别21种Bug类型，analysis子模块进行锁/RCU/refcount等高级语义分析。模块输出CommitInfo数据模型，包含commit_id、subject、body、subsystem、bug_type、diff语义分析结果等丰富字段，为后续向量化索引提供高质量输入。

**Indexer模块**：向量化索引的核心引擎，采用BGE-M3模型将commit文本编码为1024维向量。embedding子模块支持GPU加速、批量编码（batch_size=64）、多模型切换和自动降级（真实编码→Mock随机向量）。milvus子模块实现Milvus Lite和FAISS双后端架构，支持混合检索（向量相似度+标量过滤），自动降级保障可用性。pipeline子模块编排离线全量/增量索引和在线查询流程，核心创新"对称Root Cause Embedding"确保离线索引和在线查询的语义空间一致。

**Retriever模块**：在线检索的核心流水线，实现四阶段漏斗式精准匹配。recall阶段从Milvus/FAISS召回Top-100候选，filter阶段通过子系统、版本、Bug类型等规则过滤，rerank阶段使用BGE-Reranker-v2进行深度语义重排，LLM Judge阶段进行最终因果关联评分。支持三种检索模式：fast（<100ms，中精度）、standard（<1s，高精度，推荐）、deep（2-10s，最高精度）。评分融合权重可配置（Vector 0.2, Reranker 0.4, LLM Judge 0.4）。

**Analyzer模块**：宕机分析的核心组件，支持dmesg文本日志和vmcore内存镜像两种输入。dmesg子模块通过20+正则模式匹配Panic信息、Call Trace等关键特征，vmcore子模块通过drgn调试器提取内核对象和寄存器状态。rootcause子模块实现4层分层根因推断：Layer1（28条专家规则，置信度0.60-0.95）、Layer2（调用栈结构分析，4类×122函数，0.50-0.65）、Layer3（Bug类型泛化，0.40-0.55）、Layer4（Panic关键词兜底，0.40-0.50），可选LLM协同推理增强。

**Generator模块**：报告生成的最终环节，将诊断结果和检索结果整合为可读的诊断报告。llm子模块提供DeepSeek、Qwen、OpenAI和Ollama的统一接口，支持自动降级和重试。prompt子模块包含场景化Prompt模板和Few-shot示例（UAF和Deadlock）。report子模块支持Markdown、JSON、HTML三种输出格式，包含补丁对比表、因果推理和根因分析等丰富内容。

**Knowledge模块**：内核领域知识库，包含10种Bug模式知识图谱（use_after_free、deadlock、null_pointer等），6种锁类型+5种死锁模式+8条锁排序规则，12个子系统交互关系（层级、耦合、调用）。为LLM提供上下文增强，支持基于症状的Bug匹配、锁分析和子系统检测。

**Models模块**：全局数据模型定义，包括BugSeverity、AnalysisMode、RetrievalStrategy、BackendType、IndexType等枚举，AppConfig、DatabaseConfig、ModelConfig等配置模型，以及DiagnosisRequest/Response等API模型。采用dataclass设计，可选Pydantic集成用于FastAPI。

**Common模块**：公共基础设施，提供7大类21种异常类型、loguru统一日志（结构化+性能计时）、30+工具函数、单例配置中心（环境变量+YAML覆盖）、25种标准Bug类型分类体系。

**API模块**：FastAPI接口层，包含analyze、search、stats三个路由模块，Pydantic请求/响应模型，CORS/计时/限流/日志中间件，依赖注入和Redis+内存双任务存储。支持mock/real双路径和查询重试机制。

**Frontend模块**：Vue 3前端应用，包含Dashboard（仪表盘）、CrashAnalysis（宕机分析，63KB核心页面）、KnowledgeBase（知识库）、History（历史记录）、LlmExplain（LLM解释）五个页面。采用Element Plus组件库、ECharts图表、Pinia状态管理，暗色专业主题，支持实时轮询和Diff语法高亮。
