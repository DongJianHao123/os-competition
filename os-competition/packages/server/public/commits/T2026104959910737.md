#### 3.1.1 详细模块介绍
本项目采用五层架构设计：任务层（YAML任务定义）、统一语义层（6类核心动作抽象）、执行与验证层（前置条件检查+动作执行+目标验证）、模拟器适配层（AI2-THOR/TDW/VirtualHome适配器）、证据层（状态截图、差异对比、回放GIF、任务报告）。核心代码位于embodied_semantic_api/目录下，包含core（核心逻辑）、adapters（模拟器适配）、planners（规划器）、scripts（工具脚本）、tasks（任务定义）五个子模块。项目已在AI2-THOR、TDW、VirtualHome三个模拟器上完成验证，覆盖12类任务、313个任务产物、993次高层动作调用，全部通过自动化验证。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| Final Goal Was Not Checked：早期任务成功仅从动作返回值推断 | TaskRunResult.success现在同时要求action_success和goal_success |
| YAML `on` / `off` Parsing：PyYAML将未引用的on/off键解析为布尔值 | 任务加载和目标评估将布尔键归一化为语义字符串 |
| VirtualHome Small-Object Navigation：执行器可能走到父表面而非小物体 | close接受与目标、其父表面/容器的接近关系 |
| Moving Containers Dropped Contained Objects：移动容器时丢失INSIDE边 | 高层语义保留已持有容器的包含对象边 |
| AI2-THOR Scene Mismatch：任务使用vh_kitchen场景，对AI2-THOR无效 | 任务YAML支持scenes映射和命令行--scene/--scenes覆盖 |
| Weak Visual Evidence：图后端证据渲染为纯状态表 | 图证据现在渲染房间框、对象节点、语义关系边和审计表 |
| Windows AI2-THOR Build：AI2-THOR 5.0.0在Windows上无Unity构建 | 下载固定到D:/cscc_restored/.downloads/ai2thor_home |
