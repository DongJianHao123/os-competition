#### 3.1.1 详细模块介绍
AgentSentinel 是一个基于 Linux eBPF 的 AI Agent 运行时异常监测系统。项目从内核视角捕获系统调用（exec/open/unlink/connect/fork/exit 等），在用户态完成 PID/Session 归因、规则检测和轻量语义评分，输出因果证据链。系统分为以下核心模块：

1. **eBPF 内核探针层**：使用 eBPF tracepoint 技术，在内核态捕获 Agent 进程的系统调用行为。探针程序（sslsniff.bpf.c）监控 exec/open/unlink/connect/fork/exit 等关键事件，并支持可选的 OpenSSL uprobes 用于捕获 SSL 数据。该层基于 libbpf 库开发，代码成熟稳定。

2. **用户态处理器 (processor)**：这是系统的核心处理模块，约 18 万行 C 代码。负责从 eBPF 环形缓冲区读取事件，进行 PID 血缘关系追踪、session 归因、语义规则匹配、路径策略检查、因果评分计算、摘要聚合，最终输出 JSONL 格式的结构化证据。支持 8 种异常类型检测：SENSITIVE_FILE_ACCESS、UNEXPECTED_SHELL_EXEC、OUT_OF_BOUNDS_WORKSPACE、OUT_OF_BOUNDS_DELETION、CRITICAL_API_ABUSE、CROSS_LAYER_LOGIC_LOOP、CROSS_AGENT_INDUCEMENT、ATTACK_CHAIN。

3. **Web 后端 (FastAPI)**：基于 FastAPI 框架构建，提供 RESTful API 接口。包含 SQLite 数据仓库、JSONL 证据导入、异常查询与检索、多 Agent 分析器、准确率/性能运行器等模块。实现了基于 JWT 的角色权限控制（viewer/analyst/ingestor/admin）。

4. **Web 前端 (React/Vite)**：使用 React + TypeScript + Vite 构建的交互式仪表盘，展示 sessions 列表、异常详情、证据链、因果置信度等信息。前端通过 API 与后端通信，提供可视化的安全监测界面。

5. **多 Agent 异常分析**：支持跨 Agent 会话级的异常分析，能够检测跨 Agent 诱导（CROSS_AGENT_INDUCEMENT）和攻击链（ATTACK_CHAIN）等复杂攻击模式。该模块通过关联多个 Agent 的会话上下文，识别跨 Agent 的恶意行为链。

6. **实验与验证框架**：包含完整的实验脚本和测试框架，支持准确率实验（P9/P11 数据集）、性能 Benchmark（idle/normal/high_event/multi_agent 四种 workload）、检测等价性测试、Monitor Replay 等价性测试等。所有实验均生成可复现的 JSON/JSONL 结果和 Markdown 报告。
