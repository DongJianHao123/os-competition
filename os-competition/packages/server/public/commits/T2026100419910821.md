#### 3.1.1 详细模块介绍
项目采用分层 Multi-Agent 协作架构，由 CoordinatorAgent 实现 10 状态状态机（INIT → CVE_ANALYZING → PATCH_FETCHING → PATCH_RANKING → PREFLIGHT → BUILDING → VERIFYING → REPORTING → LEARNING → DONE），协调 9 个 Specialist Agent 完成端到端热补丁生成。核心管线为 7 步：CVE 检索 → Patch 获取/解析 → 预检过滤（7 项检查）→ kpatch-build 编译 → LLM 改写（最多 5 轮）→ L1/L2/L3 验证 → 报告生成。系统支持离线优先的 54 条 CVE 知识库、ChromaDB 向量检索增强、7 策略回退链（direct_apply → 3way_apply → fuzz_apply → function_level_adaptation → dependency_analysis → LLM_rewrite → manual_review），以及 MCP 服务接入阿里云百炼智能体平台。
