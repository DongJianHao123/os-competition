#### 3.1.1 详细模块介绍
项目采用多智能体架构，以 pipeline.py 为核心编排引擎，协调 9 个 Agent（coordinator, patch_locator, semantic_analyzer, kpatch_constraint, rewriter, build_log_classifier, analysis_synthesizer, validator, reporter）完成 CVE 热补丁的自动生成全流程。系统输入 CVE-ID 后，自动查询公开 CVE 与补丁来源（NVD、linux-stable、lore），下载上游修复 patch，在目标源码树上执行 apply 预检，调用 kpatch-build 生成 Linux livepatch .ko 模块，并输出可复现的 JSON 报告。支持最多 5 轮改写重试，失败时综合语义约束、kpatch 约束和日志证据进行改写适配。长时记忆（LTM）系统存储了 33 条结构化知识，涵盖 kpatch 约束、失败模式、改写策略、语义模式、子系统知识、验证方案和环境兼容性。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| kpatch 构建失败（compile_failed） | 通过 build_log_classifier 识别失败类别，analysis_synthesizer 生成改写指导，rewriter 进行 LLM 改写或保守改写 |
| patch 应用失败（patch_apply_failed） | 通过 rewriter 调整上下文行或回退保守改写 |
| kpatch 硬阻塞（如 init 函数、全局状态、结构体布局变更） | kpatch_constraint 识别硬阻塞，标记为 unsupported_semantics |
| 环境依赖缺失 | check_environment.py 预检，FAILURE_ANALYSIS.md 分类处理 |
| LLM 输出非标准 diff | rewriter 使用 _looks_like_unified_diff 校验，失败回退保守改写 |
| 重复无效重试 | coordinator 检测相同 patch 和 evidence 的重复尝试，提前终止 |
