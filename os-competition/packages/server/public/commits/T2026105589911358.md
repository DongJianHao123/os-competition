#### 3.1.1 详细模块介绍
SLIM-ARC采用三层协同架构设计。底层为内核协同层，通过mmap将GGUF模型文件映射到虚拟地址空间（45GB VSZ），利用MADV_RANDOM禁用内核顺序预读，仅page fault时按需加载，使RSS降至2GB，节省98%物理内存。中层为运行时调度层，包含层感知预取调度器（prefetch_scheduler），在graph_compute中异步预取后续层权重；MoE Router Hook提取ffn_moe_topk张量跨层预测激活专家，仅预取10/512活跃专家节省约98% I/O；Unified I/O Scheduler协调权重/KV/专家三路I/O带宽；StreamingLLM KV Eviction采用sink(4)+sliding window(1024)策略管理KV Cache。顶层为量化优化层，集成IQ4_XS权重量化（4.25 bpw）、KV q4_0量化、FlashAttention等算法级优化。各模块通过环境变量控制运行时行为，通过scripts/apply-slim-arc.py脚本集成到llama.cpp中。

### 3.2 开发目标
- 明确程度：明确
- 依据：README和架构文档清晰定义了目标：使80B参数MoE大模型（Qwen3-Next-80B-A3B，~45GB）在仅8GB RAM的边缘设备上流畅运行，通过协同OS虚拟内存机制与MoE稀疏性实现。优化链目标明确：Baseline(0.08 t/s) → +MADV_RANDOM(0.42) → +KV q4_0(0.76) → +IQ4_XS(2.45) → +FlashAttention(5.16 t/s)，累计64.5倍加速。
