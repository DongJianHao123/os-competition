#### 3.1.1 详细模块介绍
**VFS层**：在MIT xv6-riscv rev5基础上新增约800行代码，实现了完整的虚拟文件系统抽象。核心数据结构包括`struct super_block`（设备号、文件系统类型、操作函数、私有数据）、`struct inode`（统一inode包装，含引用计数、类型、大小、块指针、私有数据）、`struct vfsmount`（挂载点，最多16个，自旋锁保护）。核心接口包括`vfs_mount()`（挂载文件系统）、`path_at_mount()`（最长前缀匹配查找挂载点）、`vfs_lookup()`（路径解析返回inode）。锁顺序设计遵循Level1 bcache.lock → Level2 inode cache lock → Level3 文件系统内部锁的层次，禁止逆序获取。

**HAL层**：将RISC-V和LoongArch的架构差异隔离在`kernel/hal/`目录下，每个架构子目录包含entry.S、main.c、trap.c、vm.c、proc.c、uart.c等文件。统一接口包括main()、trap()、kinit()、kvminit()、procinit()、userinit()等。RISC-V使用PLIC中断控制器，LoongArch使用EXTIOI+APIC，两者差异在HAL层完全封装。

**ext2文件系统**：约800行代码实现ext2只读（含部分写）支持。磁盘布局包括Boot Block、Super Block、Group Descriptors、Data Blocks。关键数据结构包括ext2_super_block（魔数0xEF53）、ext2_inode（15个块指针）、ext2_dir_entry。块映射算法支持直接/一级间接/二级间接/三级间接。端序使用le16/le32宏处理小端序。

**FAT32文件系统**：约666行代码实现FAT32文件系统。支持BPB解析、FAT簇链管理（遍历/分配/回收）、短文件名和长文件名（LFN，UCS-2编码）目录项解析、文件读写和目录操作。关键计算公式包括每簇字节数、数据区起始扇区、簇N扇区号等。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| RISC-V scheduler panic: sched locks（vfs_mount持自旋锁时调用ext2_init导致睡眠） | 重构vfs_mount，将文件系统初始化移到锁外执行 |
| inode引用计数泄漏（VFS迁移后） | 统一引用计数管理，添加断言检查 |

| ext2端序问题（小端序在LoongArch上需转换） | 实现read_le16/read_le32/read_le64宏 |
| ext2整数溢出（乘法block*BS可能溢出） | 使用checked_mul风格或断言 |
| FAT32 BPB解析偏移错误 | 对照规范计算，使用DEBUG_FAT32调试 |
| FAT32 LFN支持（UCS-2小端编码解析复杂） | 实现UCS-2解析和校验和验证 |
| LoongArch启动失败（无串口输出） | 检查栈指针、链接脚本，实现早期putc |
| LoongArch中断控制器差异（EXTIOI vs PLIC） | 阅读手册，实现extioi.c驱动 |
| 合并冲突（VFS和HAL分支） | 约定每日merge时间窗口，小步提交 |
