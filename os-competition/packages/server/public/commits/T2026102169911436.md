#### 3.1.1 详细模块介绍

**VFS核心层**：实现了完整的虚拟文件系统调度层，包含8个挂载槽位的挂载表、路径规范化（clean_absolute_path处理.和..并拒绝隐藏后端）、引用计数（openref数组配合vfs_open_mount/vfs_close_mount）、能力位（7种能力位标识不同文件系统特性）、隐藏后端保护（拒绝访问.ext2/.fat32路径）、busy umount语义（umount时检查openref>0则返回EBUSY）。代码质量高，锁保护完善，边界检查全面。

**文件系统适配器**：三个后端适配器（xv6/ext2/fat32）均通过静态vfs_ops操作表实现，将VFS操作委托给底层函数。xv6和ext2支持HARDLINK和DEVICE能力，FAT32设置CASEFOLD能力位。适配器本身较薄（31~107行），核心逻辑在VFS层和底层文件系统实现中。

**HAL硬件抽象层**：通过编译时#ifdef LOONGARCH选择架构，提供hal_arch()和hal_machine()查询接口。设计简洁，支持RISC-V和LoongArch双平台。

**系统调用层**：新增mount/umount/mountinfo/mountcaps/mountrefs/statfs/rename/getcwd/halinfo等系统调用。sys_rename实现尤其全面（约155行），处理了同文件重命名、已存在目标替换、目录保护等边界情况。

**进程管理VFS集成**：每个进程维护cwdpath字符串路径和cwdvfsmount引用计数。fork时复制并增加引用，exit时释放引用，确保挂载点引用生命周期正确管理。

**文件描述符层**：文件读写通过vfs_readi_type/vfs_writei_type分发到对应文件系统。根目录读取时过滤隐藏后端目录（.ext2/.fat32），防止用户态访问。

**LoongArch移植**：完整复制了RISC-V内核和用户态代码到ports/loongarch/，适配了LoongArch汇编指令、寄存器、中断处理等。包含QEMU运行环境和启动脚本。

**自动化测试**：run-vfstests.sh支持riscv/loongarch/both三种模式，自动构建、启动QEMU、运行vfstests、检查成功标记。check-sync.sh通过规范化比较确保双平台代码同步。

**用户态测试程序**：test1~test5覆盖文件复制、挂载边界、命名空间隔离、隐藏后端保护、busy umount、重复挂载/卸载、FAT32大小写不敏感等场景。vfstests为综合回归测试，串联所有测试。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| 路径规范化中需拒绝访问隐藏后端（/.ext2、/.fat32）和通过..从挂载点逃逸 | 在clean_absolute_path和vfs_resolve_ref中增加reserved_backend_path检查，path_stays_under_mount确保路径不逃逸 |
| 挂载引用生命周期管理：open、chdir、exec等操作期间需持有引用，失败时显式释放 | 新增vfs_hold_path()在路径解析阶段获得mount引用，各调用点正确管理引用释放 |
| busy umount语义：打开文件、dup fd、子进程继承fd、cwd位于挂载点内时umount必须失败 | vfs_umount检查openref>0返回EBUSY，test3/test4持续补强回归测试 |
| 内核栈压力问题：新增大尺寸路径缓冲导致RISC-V回归触发kerneltrap | 将规则下沉到VFS层，移除syscall层重复大缓冲 |
| FAT32大小写不敏感查找 | 在VFS层通过VFS_CAP_CASEFOLD能力位标识，查找时进行大小写转换 |
| 跨文件系统重命名拒绝 | sys_rename中检查源和目标是否在同一VFS类型和挂载点 |
