#### 3.1.1 详细模块介绍

**GPIO 驱动模块**：从 Phytium 5.10 BSP 移植到 RROS 5.13 内核树。驱动分为两层：gpio-phytium-core.c 实现底层 GPIO 寄存器操作（get/set/direction/irq 处理），gpio-phytium-platform.c 实现平台驱动探测（支持 OF 和 ACPI）、中断控制器注册、电源管理挂起/恢复。Kconfig 提供 GPIO_PHYTIUM_CORE 和 GPIO_PHYTIUM_PLAT 两个选项，PLAT 依赖 ARM64 并自动选择 CORE、IRQ_DOMAIN、GENERIC_IRQ_CHIP、GPIOLIB_IRQCHIP。提交 8cf422e 导入驱动，56381f2 在 rros_defconfig 中使能。

**MMC 驱动模块**：从 Phytium 5.10 BSP 移植的 MCI 主机控制器驱动，支持 eMMC/SD/SDIO。代码量最大（2205 行），包含 phytium-mci.c（核心驱动）、phytium-mci.h（头文件）、phytium-mci-plat.c（平台探测）。提交 bc3521f 导入驱动，fed4698 修复了 !PM_SLEEP 条件下的 PM ops 指针编译问题并在 defconfig 中使能。

**以太网 MACB/PHY 驱动模块**：为 Cadence MACB GEM 控制器添加 Phytium 定制支持。核心是 phytium_gem1p0_sel_clk() 函数，根据 PHY 接口模式（10GBASER、SGMII、RGMII、RMII 等）和链路速率配置 SerDes/PMA 时钟多路复用寄存器。同时从 BSP 导入 Motorcomm YT8521 PHY 驱动。提交 4cf52d9 实现 MACB 增强，f8cb393 导入 PHY 驱动，15f5336 在 defconfig 中使能。

**RROS 实时内核模块**：基于 EVL 内核的 Rust 实时框架。包含多个 Rust 模块提交：c364808 修复线程退出时因过期定时器导致的 teardown oops；3952461 移除 RT 热路径上的 debug printk 以降低延迟；bda5834 使用 FileDescriptorReservation::commit 安装元素 fd；4ee18c9 回溯填充元素 fundle。这些提交展示了真实的 bug 修复和性能优化过程。

**基准测试模块**：设计严谨的实时性能对比套件。rros_rt.c 是 RROS 实时探针，支持周期调度抖动测试（-m cyclic，与 cyclictest 等价）和上下文切换测试（-m pp，两 RT 线程 sem ping-pong）。xeno_pp.c 是 Xenomai Cobalt 同口径探针，pt_lat.c/pt_pp.c 是 PREEMPT_RT 对应探针，方法学对齐确保公平对比。membw.c 是内存带宽压力器用于隔离实验。所有数据在飞腾派 V2.1（E2000Q/pe2204）上实测，RROS cell 独占 CPU0/1（FTC310 小核），root cell 用 CPU2/3（FTC664）。
