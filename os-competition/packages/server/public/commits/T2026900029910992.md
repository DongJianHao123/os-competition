#### 3.1.1 详细模块介绍

**crash_parser模块**：项目核心入口，负责将原始崩溃日志（dmesg文本、vmcore二进制、问题描述文本）解析为结构化特征。支持12种崩溃类型识别，提取调用栈帧（函数名、文件路径、行号）、RIP寄存器值、崩溃符号、内核模块信息等。代码量约12万行，包含完整的单元测试（tests/test_parser.py + tests/fixtures/下26个dmesg和10个vmcore测试用例）。开发过程从P1（统一dmesg/vmcore解析器）开始，P9适配syzbot报告格式，P17增加vmcore二进制多模态支持，P21增加自然语言描述解析第三路，持续演进至比赛结束。

**ingestion模块**：数据基础设施，从Linux内核Git仓库（约144万commit）中提取fix-like commit（约70万条），构建BM25稀疏索引（528MB）、符号倒排索引（863,936个符号）、文件路径倒排索引（215,740条）、子系统倒排索引（47个）、目录树索引，以及增强commit语义（含crash_pattern、fix_action、root_cause_hint等字段）。还构建了修复关系图（136,156对）和稳定回溯索引。开发过程从P1开始，P4完善证据索引体系，持续优化至P18。

**retrieval模块**：多路检索引擎，包含BM25搜索、符号匹配、文件路径匹配、子系统匹配、向量搜索（已禁用）五路召回，通过RRF融合算法合并结果。BM25是最关键的检索通道（消融实验显示移除BM25导致MRR下降0.2011），符号索引通过精确函数名匹配提供重要补充。向量检索（bge-small-en-v1.5）被证伪引入噪声，已在生产配置中禁用。开发过程从P1-P2构建基础检索管线，P7进行C1-C11调优，P16恢复文件路径信号使Hit@3从0.177提升至0.229。

**rerank模块**：7维加权评分系统，对召回结果进行精细排序。评分维度包括：BM25得分（权重0.30）、符号匹配（0.15）、文件路径重叠、子系统匹配、Fix信号（0.15）、Bugtype信号（0.12）、Blame-fixer信号（0.40）。还集成了Cross-Encoder重排序（BAAI/bge-reranker-v2-m3），但计算开销较大。LTR（Learning to Rank）已禁用。开发过程从P5-6实现Top-3分层精排，P18增加bugtype症状↔描述信号，P20增加Route A因果链信号（blame→Fixes），使held-out Hit@3从0.292提升至0.3125。

**explanation模块**：生成可解释的推荐理由，展示因果证据链（如"该commit的Fixes标签引用了崩溃栈中的函数X"）。功能基本可用，但解释深度和多样性有提升空间。

**api模块**：基于FastAPI的RESTful接口，提供`POST /api/search`（接受input_type/content/top_n参数，返回排序后的候选patch列表）和`GET /api/health`（健康检查）两个端点。启动时加载配置和索引，支持多输入类型（dmesg/vmcore/description）。

**eval模块**：完整的评测体系，包含消融实验（评估各检索/重排序组件的贡献）、held-out评测（100个syzbot真实崩溃报告，严格防泄漏）、网格搜索（调优权重参数）、per-type breakdown（按崩溃类型分析性能）、诊断脚本（分析召回失败原因）。方法学严谨，诚实标注了数据泄漏问题和语义鸿沟定位。

**demo模块**：基于Streamlit的交互式演示应用，支持dmesg文本输入、vmcore文件上传、问题描述输入三种方式，内置kGym基准测试用例。展示崩溃特征提取结果、候选patch列表、评分详情、因果证据链，以及预期commit命中检测。

**scripts模块**：包含索引构建脚本、LTR训练脚本、证据包生成脚本、部署脚本（Docker/ES批量索引/更新索引）等辅助工具。
