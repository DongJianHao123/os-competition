#### 3.1.1 详细模块介绍

**probes/（eBPF探针层）**：基于 libbpf CO-RE 技术，通过 tracepoint 捕获系统调用事件。syscall_monitor.bpf.c 实现了四项性能优化：LRU Hash 自动淘汰、路径预过滤（先读1字节判断首字符，再读16字节做敏感路径检测）、采样控制（普通OPEN每4条提交1条）、PERCPU_ARRAY 零锁计数器。process_tree.bpf.c 通过共享 watched_pids Map 实现进程树追踪，消除 fork→execve 竞争窗口。tls_intercept.bpf.c 通过 SSL_write/SSL_read uprobe 捕获 TLS 明文，为 HTTPS 场景预留能力。

**loader/（用户态加载与持久化层）**：AgentMonitorLoader 类负责完整的 BPF 生命周期管理。采用 Map Pinning 方案二：先加载 syscall_monitor 创建共享 Map 并 pin 到 /sys/fs/bpf/ebpf_agent/，再让 process_tree 通过 bpf_map__reuse_fd 复用 Map fd，实现内核态直接共享。支持 --wait 等待模式、--pids 直接指定、PID 文件读取三种启动方式。Ring Buffer 轮询主循环每 30 秒输出统计和 drop 计数器，支持优雅退出。

**analyzer/（分析与归因层）**：异常检测规则引擎包含 7 条规则：非预期 Shell 启动（CRITICAL）、敏感文件越权访问（CRITICAL）、工作区逃逸（CRITICAL）、工作区外异常文件删除（WARNING）、逻辑死循环（WARNING，120秒内API≥8次+重复Prompt+反复tmp读写）、API调用频率异常（WARNING，120秒内≥20次）、多Agent资源竞争（WARNING）。SemanticBridge 通过时序窗口（120秒）+ 意图关键词正则匹配 + 进程归属三重关联实现因果归因，支持工具调用增强置信度。

**agent_env/（Agent测试环境）**：支持 normal（正常工作流）、loop（逻辑循环）、malicious（恶意行为，含 llm_decision 和 forced_regression 两种模式）、realistic（真实场景验证，仅使用外观正常的漏洞工具）、quick（快速测试）等多种场景。malicious_agent.py 分三个阶段（敏感文件访问→Shell启动→工作区逃逸），forced_regression 模式在 Python 层直接触发 syscall 保证 eBPF 捕获。

**dashboard/（输出与展示层）**：纯 Python 标准库 HTTP 服务，无外部依赖。提供 /api/dashboard 端点返回聚合 JSON 数据，包含时间线（最多180条）、告警统计（按类型和级别）、热力图（按Agent统计事件数）、关系图（Agent-工具-告警关系）、进程树（最多140节点）、性能指标（syscall写入率、丢弃计数等）、因果链详情。静态文件服务有路径穿越防护。

**tools/（工具脚本）**：perf_measure.sh 是大型性能基准测试脚本，覆盖无监控/有监控任务耗时、API 延迟、syscall 捕获速率、Ring Buffer 丢失、采样丢弃等指标。run_dashboard.sh 用于单独启动 Dashboard。
