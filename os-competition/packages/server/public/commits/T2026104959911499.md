#### 3.1.1 详细模块介绍
项目基于 llama.cpp 上游，构建了一套完整的"观测→决策→执行→验证"闭环预取系统。核心模块 ElasticPrefetcher 实现了自适应预取窗口：当内存充足时预取 4-6 层，内存压力升高时收缩至 1-2 层，并动态调整释放保留层数。PosixPrefetcher 和 Win32Prefetcher 分别封装了 Linux 和 Windows 平台的预取原语（madvise/mlock 和 ReadFile+OVERLAPPED/PrefetchVirtualMemory）。LayerGuide 通过解析 GGUF tensor 名称中的 blk.N. 前缀实现层边界检测，并集成 cgroup v2 内存压力读取（workingset_refault、pgscan）以动态调整策略。SystemMonitor 采集缺页率、工作集、磁盘 I/O 等指标，支持 30 样本滑动窗口和 CSV 日志。整个系统通过环境变量 ENABLE_PREFETCH 控制开关，支持 baseline/prefetch/adaptive 三种模式同入口对比。
