#### 3.1.1 详细模块介绍
项目基于xv6-riscv第5版，初始提交包含了完整的xv6操作系统源代码。代码结构清晰，按模块组织在kernel/、user/、mkfs/等目录中。然而，赛题要求实现的VFS（虚拟文件系统）和HAL（硬件抽象层）功能——包括同时支持xv6、ext2、fat32文件系统以及RISC-V和LoongArch平台——在代码中并未体现。fs.c中仍然是直接操作具体inode和磁盘的xv6原生文件系统实现，没有VFS抽象层（如vnode、通用文件操作接口、挂载/卸载逻辑）。Makefile中仅配置了RISC-V目标，未发现LoongArch相关编译配置。新增的tests/目录包含测试用例（test1.c文件拷贝测试、test2.c跨文件系统复制测试），但test2.c中引用的ext2文件系统操作并未在代码中实现。
