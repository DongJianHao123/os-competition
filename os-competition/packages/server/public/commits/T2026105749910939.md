#### 3.1.1 详细模块介绍
**DNS双端缓存加速**：项目核心模块。eBPF侧通过XDP程序拦截DNS查询（UDP/53），在XDP_TX层面直接返回缓存的DNS响应，实现零拷贝加速。TC egress/ingress程序分别处理客户端和服务端的缓存插入与命中。支持TTL（超时淘汰）、LRU（最近最少使用）和LFU（最不经常使用）三种淘汰策略，用户态策略引擎（auto_policy.go）可根据命中率、填充率等指标自动切换最优策略。客户端和服务端可独立配置缓存策略。

**gRPC缓存注入（实验性）**：基于sk_msg/sk_skb钩子，在socket层拦截gRPC请求。用户态程序解析HTTP/2帧，提取方法名（如/helloworld.Greeter/SayHello），将响应序列化后写入eBPF map。eBPF侧在sk_msg verdict中检查缓存命中，命中时直接修改msg payload返回缓存响应。采用白名单机制避免缓存带副作用的请求。支持TTL过期。README中声明该模块代码主要由LLM生成。

**LDAP监控**：轻量级监控模块。eBPF侧在XDP层拦截TCP/389端口的LDAP流量，将原始载荷通过ringbuf推送到用户态。Go侧解析BER编码的LDAP消息，统计操作类型、结果码和延迟。功能较为基础，主要用于监控而非加速。
