#### 3.1.1 详细模块介绍
项目基于 llama.cpp 开源框架，针对内存受限场景（嵌入式设备、边缘计算节点）进行系统级优化。核心创新点包括：(1) Lazy KV Cache：通过 mmap + MAP_NORESERVE 延迟分配物理内存，配合 madvise(MADV_DONTNEED) 主动回收，在不影响吞吐的前提下节省约 125 MiB 物理内存；(2) AHKVM 自适应分层 KV 缓存管理器：将 KV Block 统一建模为 HOT（DRAM-F16/Q8）、WARM（DRAM-Q4/Q8）、COLD（SSD-Q4）三层，通过 Budget Manager 实时监控 RSS 并执行闭环控制，实现模型加载时间从 356s 降至 8.23s（降幅 97.69%）；(3) FlexInfer Layer Streaming：使用异步双缓冲预取下一层权重，重叠计算与 I/O，配合 Balanced Memory Locking 跨层均衡权重固定策略，在 2800 MiB 下解码速度从 0.698 tokens/s 提升至 1.250 tokens/s（+79.1%）；(4) 完整的 cgroup 鲁棒性测试体系，在 1800-2800 MiB 物理内存限额下进行梯度压测。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| KV Cache 提前分配物理内存导致内存浪费 | 改用 mmap + MAP_NORESERVE 延迟分配物理页 |
| clear() 不释放物理页，上下文重置后内存未回收 | 增加 madvise(MADV_DONTNEED) 主动回收物理页 |
| mmap MAP_NORESERVE 跨平台兼容性问题 | 通过条件编译处理差异，提供回退路径 |
| 缺页中断对性能的影响 | 理论分析+实测验证，缺页开销可忽略 |
| 异步预取 IO 阻塞与 Cache 竞争 | 使用 posix_fadvise(POSIX_FADV_DONTNEED) 丢弃页缓存 |
| 多线程层边界同步开销 | 采用粗粒度层边界同步策略 |
