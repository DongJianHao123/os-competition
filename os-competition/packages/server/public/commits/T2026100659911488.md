#### 3.1.1 详细模块介绍
KernelSage 是一个面向小型操作系统仓库的分析比对智能体，由天津师范大学"一定要以人类的身份赢啊"团队开发。项目采用分层架构：入口与任务调度层 → 数据采集层 → 画像分析层 → 历史对比层 → 报告生成层 → 展示交付层。

核心工作流程为：collector.py 扫描仓库文件结构 → parser.py 抽取符号 → analyzer.py 生成 8 维 OS 机制画像（KernelProfile）→ selector.py 从 141 个历史样本中选择最相似的 Top5 → agent.py 分析相似线索 → reporter.py 生成描述/比较报告 → html_reporter.py 生成 HTML 版本 → 部署到网站。

LLM 增强为可选模块，受显式调用、dry-run、缓存、审计与回退四道约束，默认不触发 API 调用。项目核心特征包括源码证据链优先、七维 OS 机制画像、分层历史样本库、可控 LLM 增强、HTML 证据报告和 query-evidence 检索、本地优先和可复现交付。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| LLM API 不可用/调用失败 | 失败回退到规则版报告，支持 dry-run 和缓存 |
| 历史仓库来源复杂无法 clone | fetch_repos.py 对失败做结构化记录，只纳入成功 clone 样本 |
| 样本来源分级不清 | manifest_audit.py 检查 verified_award 样本的奖项和来源链接 |
| 关键字误判 OS 机制 | analyzer.py 削弱短前缀，按维度过滤 evidence，排除无关代码 |
| C++/unikernel/RTOS 覆盖不足 | collector.py 扩展 C++ 扩展名，parser.py 复用 C/C++ 符号解析 |
| 功能重合和代码相似边界不清 | reporter.py 声明功能重合不是抄袭裁定；similarity.py 输出 review leads |
| LLM 幻觉和越界判断 | cli.py 只有显式指定时才进入 LLM 分支；llm_audit.py 检查引用范围 |
| 大样本库重复分析耗时 | profile_cache.py 使用 fingerprint 缓存画像，命中后耗时从 75 秒降至 3.4 秒 |
| 本地网站无法被评委访问 | 部署到阿里云 ECS，使用 Nginx 提供公网访问 |
| 缓存短路导致 dry-run 失效 | 将 dry_run_path 分支移动到缓存读取之前 |
| API 异常未统一处理 | 将 URLError、JSON 解析失败等统一包装为 RuntimeError |
| 路径逃逸安全风险 | 新增 validate_repo_id() 和 resolve_repo_dir() 防止路径逃逸 |
| 空文件行号校验误判 | 修复空 root_path 和空文件行号校验的边界条件 |
