#### 3.1.1 详细模块介绍
项目采用经典的编译器前端架构：词法分析（lexer）→ 语法分析（parser）→ 解释执行（vm）。lexer模块使用logos库高效地将源码字符串转换为Token流，支持JS关键字、字面量、运算符等30种Token类型。parser模块实现递归下降解析器，按照运算符优先级（赋值→逻辑或→逻辑与→比较→加减→乘除→基本表达式）构建AST，支持变量声明、if/else条件分支、while循环等语句。vm模块采用树遍历方式直接解释执行AST，维护变量环境HashMap，支持基本数据类型（数字、字符串、布尔、Null）的运算和流程控制。main模块提供CLI和REPL两种交互方式，REPL支持多行输入自动检测括号匹配。
