#### 3.1.1 详细模块介绍
本项目将国产机器人软件栈 AGIROS（ROS 2 兼容发行版）及其自研实时通信中间件 TravoDDS 移植到国产嵌入式实时操作系统 openEuler Embedded / UniProton 上。项目采用自底向上、Linux 基线对照、补丁/Shim 集中收敛、每里程碑可复现脚本断言的工程方法。软件栈自下而上为：UniProton (musl libc + libc++ + lwIP) → OSAL Shim (pthread/线程/同步/网络/内存/日志适配) → TravoDDS (DCPS + RTPS, UDP-only 裁剪, asio over lwIP) → ROS2 层 (rcl → rmw-travodds-cpp → rosidl typesupport) → 应用/Demo。核心移植范式是主任务忙轮询同步驱动 I/O 以规避异步线程饿死问题。项目共20次提交，时间跨度2026-06-02至2026-06-30，涵盖从环境基线到rclcpp完整pub/sub、服务、定时器、图内省等全部里程碑，所有硬指标均2×PASS通过。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| UniProton裸机线程模型下pthread默认优先级导致后台线程饿死 | 同步直驱策略：忙轮询TravoDDS_PollSharedIo、同步重宣告、QoS降级BEST_EFFORT/VOLATILE |
| libc挂死点（musl vs newlib兼容性） | 通过libc-compat shim头文件适配 |
| C++运行期裸机适配（全局构造、异常、RTTI） | 线程shim + 裁库 + init_array补丁 |
| memorypool realloc在裸机不可用 | 改为malloc+memcpy+free |
| SPDP发现挂死 | 动态participantId + 单播发现 + 同步重宣告 |
| 跨节点virtio-net驱动 | 自写轮询式virtio-net驱动 |
| rcl栈溢出导致跨节点挂死 | SmokeInit栈16KB→256KB |
| Node创建死锁（rclcpp） | skip-graph + Node-only构造器 |
| 整库链接时间过长（12.3s） | -Wl,--gc-sections提速至~4s（3×） |
| asio interrupter在lwIP挂死（aarch64双participant） | 未完全解决，已知限制 |
| FP/SIMD陷阱（aarch64） | CPACR_EL1.FPEN使能修复 |
