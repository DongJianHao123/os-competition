#### 3.1.1 详细模块介绍
**监控层（monitor）**：通过Python Monkey Patch非侵入式接入vLLM推理流程。patch.py拦截executor.execute_model追踪请求级块分配与引用，拦截CacheEngine.swap_out/swap_in记录物理迁移事件，通过模型层pre_hook/post_hook记录每层访问。stats.py维护全局KVBlockStats单例，管理canonical ID映射、引用计数更新、读写统计、抖动惩罚计算。block_page.py定义KVBlockPage数据结构，记录每个block的完整元数据。block_history_logger.py将事件流水导出为JSON。

**调度层（scheduler）**：policy.py实现基于多维度加权评分的热度模型（W_REF=800, W_RECENCY=260, W_RECENT_RATE=80, W_TOTAL_READ=4, W_LAYER=3, W_MISS=900），将block分为HOT/WARM/COLD。patch_scheduler.py替换vLLM Scheduler._schedule，在GPU空闲块低于水位线时触发自定义WARM块swap，含冷却时间（0.2s）和防抖动熔断（5s窗口内swap_out>32次或miss_ratio>0.75时回退原生策略2s）。engine_bridge.py将block级候选转换为allocator.swap()调用。disk_tier.py仅实现元数据spill记录。

**实验入口（final_working.py）**：使用facebook/opt-125m模型，max_model_len=512，gpu_memory_utilization=0.12，60个请求分两阶段并发测试，输出cProfile性能分析和JSON日志。
