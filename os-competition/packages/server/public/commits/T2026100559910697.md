#### 3.1.1 详细模块介绍
项目基于ArceOS/StarryOS宏内核架构，针对RK3588平台（Orange Pi 5 Plus）进行了深度适配。核心工作包括：将StarryOS迁移到TF卡启动环境，新增RK3588S SDMMC动态驱动以识别TF卡，修复GPT rootfs分区选择逻辑，修复串口终端换行和空闲后无法输入问题，刷入SPI U-Boot实现TFTP网络启动，打通/dev/dri/card1到RKNN Runtime的NPU推理路径并运行YOLO模型。当前系统可进入StarryOS shell，识别TF卡和rootfs，支持网络加载内核，NPU推理链路已基本可用。

驱动方面，项目独立开发了多个RK3588专用Rust驱动：rknpu驱动实现了完整的NPU硬件控制（CNA/DPU/DMA等单元），rk3588-clk驱动实现了时钟树管理（PLL、门控、时钟选择），sdmmc驱动支持eMMC 4.x/5.x和SD标准，rockchip-pm驱动控制NPU电源域，arm-scmi驱动实现SCMI协议通信，some-serial提供统一串口抽象。此外，rdrive框架提供了动态驱动管理能力，支持FDT设备树探测。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| TF卡无法识别（SDMMC驱动缺失） | 新增RK3588S SDMMC动态驱动，匹配rockchip,rk3588-dw-mshc |

| YOLO模型推理结果饱和（FP16归一化问题） | 恢复为Rockchip标准UINT8 NHWC路径 |
| 网络启动失败 | 刷入SPI U-Boot，设计boot script实现TFTP网络启动 |
