#### 3.1.1 详细模块介绍
项目基于 rCore-Tutorial-v3 扩展，在 RISC-V 64 + QEMU 平台上实现面向 AI 智能体的操作系统内核扩展。任务一扩展了进程控制块（PCB），新增 AgentInfo 结构体，包含 agent_type、heartbeat_interval、resource_quota、loop_state、context_path_meta 等字段，并在用户地址空间中分配 32 KiB 的 Agent Context 区。任务二设计了固定 256 字节的结构化工具调用协议帧（ToolRequest/ToolResponse），实现了 query_process、get_system_status、context_snapshot 三个内置工具，并支持动态扩展。任务三实现了基于 FIFO 环形缓冲区的上下文路径管理，每个节点 128 字节，最多 256 个节点，用户态可直接内存读取实现零系统调用。任务四在内核侧实现了文件属性查询工具 query_file，支持按 type/owner/tag/keyword 过滤，并通过 index_steps 和 scan_steps 字段量化性能对比。任务五实现了完整的 Agent Loop 内核运行机制，包括心跳机制（10ms 时钟中断检查）、事件驱动（32 位掩码订阅/投递）、LoopState 生命周期管理（Idle→Running→WaitingTool→Done）以及三桶优先级调度（high/normal/low）。任务六整合所有模块，实现了 Agent 系统管理员综合演示场景，并交付了性能基准测试（用户态直读比系统调用快 5307×）、多 Agent NPC 生态系统和优先级调度演示等进阶成果。
