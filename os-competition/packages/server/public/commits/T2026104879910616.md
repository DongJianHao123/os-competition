#### 3.1.1 详细模块介绍

**llm-tz 模块**：项目的核心模块，基于 OP-TEE 可信执行环境。TA 端（llm-tz/ta/）实现了私有参数的安全存储（ffn_store.c、act_perm_store.c、act_group_perm_store.c）、策略绑定与校验（policy_store.c）、权重加载（weight_loader.c）、关键计算可信校正（layer_ta.c、matmul.c）等功能。Host 端（llm-tz/host/main.c）实现了与 TA 的通信、provision 工具和推理入口。项目从 v2.2（protected_linear schema for FFN LoRO/FM）开始，经过 v2.5（activation channel permutation）、v2.6（activation group permutation）、v2.7（unify policy）、v2.9（bind TEE operations to sessions）、v2.10（authenticated policy session）、v2.11（RKNPU integrity verification & submit gate）、v2.12（FFN restore optimization）等多个版本的持续演进，功能不断完善。

**driver-npu 模块**：在 RKNPU 内核驱动中增加 LLM-TZ session 状态机，包括 PREPARE_SESSION、GET_MEASUREMENT、ACTIVATE_SESSION、CLOSE_SESSION 四个 ioctl，以及 owner tgid submit gate。驱动启动时测量自身 text section 的 SHA-256 哈希值，TA 验证通过后才允许 ACTIVATE_SESSION。只有执行 ACTIVATE_SESSION 的进程（tgid）才能提交 NPU job，其他进程提交被拒绝（-EACCES）。

**AttackValidation 模块**：提供自动化攻击验证脚本，验证三道防线（Policy Session 认证、RKNPU 驱动安全状态机、运行时策略校验）的有效性。包含恶意 CA 会话劫持、未测量 RKNPU 路径、策略篡改等攻击场景的测试。

**demo 模块**：提供 Web 可视化演示，包含 FastAPI 后端和 Vite/React 前端。支持 Protected 单聊模式、Public-only vs Protected 一键对比、Trust Rail 面板（显示 TEE/RKNPU/TA restore 状态）和 Evidence Log。

**ggml/src/ggml-rknpu2 模块**：在 RKNPU2 ggml 后端中集成 LLM-TZ bridge，通过 matmul_client.c 与 OP-TEE TA 通信，在 protected MatMul 节点调用 TA 进行可信校正。

**tools/gguf_split 模块**：提供模型保护工具集，包括 GGUF 模型拆分（split_layer.py）、关键线性层保护（obfuscate_protected_linear.py）、激活置换保护（obfuscate_activation_permutation.py、generate_activation_perm_bundle.py）、策略生成（obfuscate_protection_policy.py）等。

### 3.2 开发目标
- 明确程度：明确
- 依据：项目 README 明确说明目标为"基于 OP-TEE 与 RKNPU 的边缘大模型可信推理系统"，在保留 RKNPU 硬件加速效率的同时，将策略校验、私有参数、关键计算可信校正和激活保护逻辑放入 Secure World，降低公开模型文件被离线复用、运行路径被绕过以及关键中间数据被篡改的风险。性能目标为代表性模型推理性能损耗控制在30%以内，PPL恢复偏差在±4%以内。
