#### 3.1.1 详细模块介绍
xPipeANN 基于 PipeANN（OSDI'25）进行二次开发，针对 ARM64/Docker 受限内存环境优化外存向量检索。项目保留 PipeANN 的外存 Vamana 图 + PQ 压缩 + 流水线搜索骨架，新增五大改进：(1) ARM64/NEON 适配，替换 x86 SSE/AVX 为 NEON SIMD；(2) io_uring 安全启用，通过编译选项控制，失败时自动回退同步 pread；(3) 用户态 4KiB 扇区缓存（FIFO，默认 16384 页 ≈ 64 MiB），减少 MeanIO；(4) PQ64 + 参数联合调优（R64/PQ64, beam=32, mem_L=8, L=10）；(5) WAL/Delta 动态写入原型，支持增量插入与后台合并。Python 绑定通过 pybind11 实现，提供 faiss 风格 API 及 LangChain/Qdrant 集成。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| ARM64 平台 x86 SSE/AVX 指令不可用 | 替换为 NEON SIMD 实现 L2 距离和 PQ 查找 |
| CMake 在 ARM64 上构建失败 | 修复 CMakeLists.txt，添加 ARM64 检测和编译选项 |
| io_uring 在受限运行时环境（Docker）中不可用 | 添加编译选项控制，失败时自动回退同步 pread |
| x86 原子操作在 ARM64 上不兼容 | 替换为 C++11 标准原子操作 |
| 原始 PipeANN 在 ARM64/AIO 后端性能不佳 | 启用 io_uring + SQPOLL + 用户态读缓存，QPS 提升 3.39× |
| PQ16 在低搜索深度下召回率不稳定 | 切换至 PQ64，联合调优 R64/PQ64/beam/mem_L/L 参数 |
| 动态写入时并发搜索的一致性 | 实现 WAL 顺序追加 + Delta 内存段，搜索时合并主索引和 Delta 结果 |
