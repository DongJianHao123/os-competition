### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| vLLM 0.22 和 0.23 版本 API 差异导致 connector 兼容性问题 | 在 vllm_connector.py 中添加版本判断，handle_preemptions 兼容 list/dict 两种参数格式，bind_connector_metadata 同时处理 dict 和 object 两种 metadata 格式 |
| save_kv_layer 全 layer 搬迁效率低 | 改为 per-block 选择性搬迁，添加 pinned buffer 池复用机制防止 hostAlloc 内存泄漏 |
| 迁移抖动导致频繁升降级 | 在 migration_plan.py 和 tier_tracker.py 中引入 3 周期冷却期（cooling_off）机制 |
| 碎片率过高影响分配效率 | 在 allocator.py 中实现 defragment 压缩功能和 cleanup_empty_slabs |
| Prometheus 指标可能为百分比或小数 | 在 metrics_collector.py 中添加 kv_cache_usage > 1 时除以 100 的归一化逻辑 |
