#### 3.1.1 详细模块介绍

**数据接入层 (ingest)**：实现了 SchemaRegistry 统一 schema 管理，6 类数据源适配器（CSV/Parquet/JSON/NumPy/PSM/SMD），将原始数据标准化为 MetricPoint/EventPoint 记录。支持宽表和长表布局，时间戳单位自适应。

**预处理层 (preprocess)**：实现了 MetricSeries/WindowedFeature 类型系统，WindowEngine 的 pivot→align→slice 流水线，CounterTransform 处理 rate/difference/reset 三种模式，RobustScaler/log1p/winsorize 三种归一化策略。

**异常检测层 (detection)**：5 种检测器（MAD、EWMA、Quantile、PCA Residual、Isolation Forest）通过统一 BaseDetector 接口集成，DetectorEnsemble 加权融合（权重 0.25/0.20/0.15/0.20/0.20），支持域先验加成和连续窗口防抖。PCA 检测器创新性地采用历史拟合+末行外样本重建+比值评分，避免单尖峰被吸收为主成分。EWMA 残差归一化使用历史残差 std 避免最新样本污染。

**知识图谱层 (graph)**：构建了 Linux 内核显式知识图谱，三层结构（子系统/机制/症状），初始 16 节点 9 边，v1.0 扩展至 28 节点 18 边。支持 YAML 配置加载、metric→node 映射、异常种子 k-hop 子图裁剪。

**排序层 (ranking)**：在反向子图上运行 Personalized PageRank（damping=0.85），结合 -log(confidence) 边权重建最短传播路径，五信号融合评分（PPR 0.45、路径 0.25、先验 0.10、证据 0.15、种子命中 0.05）输出 Top-K 候选。

**因果复核层 (causal)**：对图拓扑排出的 Top-K 候选，用滞后互相关 + 可选 Granger 检验做经验证据复核，按 boost=0.4/penalty=0.2/threshold=0.6 重排。Granger 检验 statsmodels 不可用时安全降级。

**证据层 (evidence)**：按需 eBPF 证据编排，将"节点→探针"映射形式化为预算约束下最大化根因后验信息增益的决策问题。ProbePlan/Runner 分离，TTL 缓存避免重复，max_probes 上限控制，异常安全降级。

**解释层 (explain)**：RcaResult 数据结构 + 五段式 Markdown 报告（异常摘要、Top-K 根因表、分数分解、传播路径、证据清单）。受约束 LLM 解释器通过 IEL 验证器对输出进行 evidence_id/node_id 白名单正则校验，编造则回退确定性模板。定义了中间解释语言 IEL 与独立验证器 IELVerifier，形成有可证明性质的受约束生成框架。

**检索层 (retrieval)**：三种可验证接地检索算法——VGR（接地谓词将无编造保证前移到检索输入端）、CHAR（因果假设驱动检索）、KPAGR（传播路径级相似度超越节点级 GraphRAG），分层组合形成双重无编造防线。

**CLI 层 (cli)**：Click 框架实现，analyze 命令串通全管线（ingest→preprocess→detection→graph→ranking→causal→evidence→explain），支持 --causal/--evidence/--llm/--stream 等选项，流式模式实时输出各阶段事件。

**后端服务器 (server)**：SSE 和 WebSocket 双协议支持，demo_fault_trigger 现场故障触发，全链路真实化（真实 eBPF 证据 + 真实大模型解释 + 增量流式）。

**前端 UI (ui)**：Vue 3 + Element Plus + Vite 构建，包含 DashboardView（仪表盘）、EventsView（事件视图）、IncidentView（根因分析视图），WebSocket 连接真实后端实时更新。

**eBPF 探针 (ebpf)**：libbpf CO-RE runqlat 探针在 VM 上真实采集 run-queue 等待直方图（425 样本，p99≈256 μs），经 LibbpfProbeRunner 闭环喂回证据链。

**评测框架 (benchmark)**：13 类内核故障场景模板（FS-01~FS-13），5 个真实注入 case（FS-02 cgroup CPU 节流、FS-04 直接内存回收、FS-08 脏页回写、FS-10 网络丢包、FS-12 futex 锁争用），含 inject/recover/validate 脚本和 spec.yaml 定义。离线评测器支持 Hit@K/MRR/传播路径匹配度计算。
