#### 3.1.1 详细模块介绍

**SSD存储路径模块**：采用固定4KB记录布局，每个节点占用4096字节，支持直接偏移计算和SSD页级随机读。索引文件格式为4096字节文件头后跟N个4096字节节点记录。支持Linux O_DIRECT直接I/O，不可用时自动回退到pread。DiskIndexWriter负责将内存图写入SSD索引文件，DiskIndexReader负责从SSD索引文件读取节点记录。该模块为整个系统的磁盘访问基础，所有上层优化（异步I/O、预取、缓存）均依赖于此。

**缓存与零拷贝模块**：实现Graph-Aware 2Q缓存策略，在图遍历的in-degree信息基础上扩展传统2Q算法，高入度hub节点优先保留在Am队列。评分公式为score(page) = access_score + alpha * log(1 + in_degree) + beta * recent_hit - gamma * cold_age。pin/unpin机制通过引用计数防止关键页面被驱逐，RAII守卫确保异常安全。compute_distance_direct直接从缓存页内部指针计算距离，消除一次内存拷贝。

**异步预取模块**：封装AsyncPageReader，支持同步读、异步读、批量提交和io_uring回退。维护pending read队列，达到batch size后批量提交，控制in-flight I/O深度。PrefetchPlanner做page-level聚合、XGBoost收益打分、top-K/threshold/io_depth限流后提交。QueryPageSession管理pending/ready page和统计。保留同步读取fallback路径，保证io_uring不可用时系统仍能运行。

**beam_width + io_uring批读模块**：将beam_width参数（每轮最大扩展数）与io_uring批量I/O结合。每轮收集最多effective_beam_width个未扩展候选，去重其页面后批量加载。io_uring批量提交使用单次io_uring_submit()调用，不可用时回退到同步读。beam_width=0表示跟随search_width。

**动态写入模块**：采用"base graph + delta layer"架构，写入先落WAL再入MemTable，达到阈值后flush为SSTable。查询时合并base graph、MemTable和SSTable结果，经全精度rerank后返回top-k。WAL保证崩溃恢复，SSTable不可变简化并发控制，Level-0 Compaction优先减少读放大。Manifest原子更新避免文件状态不一致。

**流式ANN层模块**：在静态DiskANN/Vamana SSD索引上增加轻量级增量更新层，支持流式插入、延迟删除、混合查询和周期性批量合并。采用三层架构（BaseIndex on SSD, DeltaIndex in memory, DeleteList for tombstones），HybridSearcher合并两个索引的结果。

**XGBoost预取提前触发模块**：将预取触发点提前到搜索流水线更早阶段。设计四个触发点（T0入口预热、T1预波前窗口、T2扩展后前瞻、T3重排前瞻），其中T1已实现。所有触发点共享统一调度器和限流逻辑。新增CLI参数--prefetch-early-trigger和--prefetch-io-depth-guard-ratio。

**预取时机设计模块**：记录异步预取的触发点设计。邻居预取在当前节点弹出后，读取其图邻居，经PQ ADC粗过滤后提交。批滚动窗口预取在批处理循环内，当前批I/O等待完成后立即提交下一批页面，实现CPU/I/O重叠。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |

| 传统OS文件I/O机制无法处理基于图的向量索引遍历的随机读模式 | 设计用户态高效缓存池，放弃OS Page Cache，实现Graph-Aware 2Q缓存策略 |

| BufferPool预算溢出 | 实现内存预算控制，resident ratio控制在0.199992（约20%） |
| Compaction尾延迟影响查询 | 后台执行Compaction，使用不可变读视图避免阻塞前台查询 |
| io_uring线程安全问题 | 实现io_uring回退机制，不可用时自动切换到pread同步读 |
| 增量插入连接性问题 | 增量数据不立即并入主图，通过增量层查询合并，后续周期性merge |
