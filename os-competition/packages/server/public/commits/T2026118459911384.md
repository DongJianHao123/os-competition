#### 3.1.1 详细模块介绍
Agent-OS 基于 rCore-Tutorial-v3 ch6 固定提交 c17b725 构建，扩展了面向 AI Agent 的内核机制。项目采用机制与策略分离的设计原则，内核态负责 Agent 身份、调度、能力检查、资源配额、工具分发、事件唤醒和上下文元信息等机制，用户态负责上下文记录、工具结果缓存和 Agent 决策逻辑等策略性数据。

Agent 进程抽象模块实现了 Agent ID 原子分配器（从 1 开始，0 表示无 Agent）、Agent 类型枚举（NormalProcess/UserAgent/SystemAgent/DemoAgent）、能力位图（ToolCall/SendMessage/QueryFile/ShareContext/AdminInspect）和资源配额管理。Agent 创建时通过 sys_agent_create 系统调用 fork 调用者并为子进程分配 Agent ID、元数据和独立 Context 区。

上下文与内存管理器在用户地址空间中映射 Agent Context 区（4 KiB 到 1 MiB，页对齐），位于 Trap Context 下方并保留 guard page。Context Path 采用固定 40 字节 ContextNodeHeader 环形缓冲区，支持 push（追加）、query（查询）、rollback（回滚到历史节点）和 clear（清空）操作，FIFO 淘汰策略。

工具管理器维护静态工具注册表，包含 5 个内核工具：query_process（查询进程/Agent 信息）、get_system_status（查询系统状态）、get_context_status（查询 Context Path 状态）、send_message（向子 Agent 发送消息）、query_file（按属性查询文件）。工具调用采用固定 16 字节请求头 + 定长参数描述符 + 尾部值区的结构化协议，支持 TLV 参数校验。

Agent 调度器实现心跳触发（sys_agent_heartbeat_set/stop）、事件关注（sys_agent_watch/unwatch）、阻塞等待（sys_agent_wait）和 Loop 状态管理（sys_agent_loop_set_state）。支持的事件包括 Heartbeat、Message、FileChanged、ChildExited。

存储与文件管理器在内核侧实现 FileAttrStore 原型，不修改 easy-fs 磁盘 inode 布局。支持文件属性的设置、读取、删除，以及基于 Owner/FileType/Tag/Priority/Summary 的精确匹配 BTree 索引和 Summary 关键词倒排索引，多条件查询选择候选最少的索引。

IPC 与协作管理器实现固定容量收件箱（最多 8 条消息，每条 64 字节），通过 send_message 工具向直接子 Agent 发送消息，触发事件唤醒。

审计与安全管理器实现能力位图检查（创建 Agent 时校验能力、工具调用时检查所需能力），工具调用自动记录 Context 节点摘要。
