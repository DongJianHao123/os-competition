#### 3.1.1 详细模块介绍
**ascend-open-harmony（系统移植模块）**：基于OpenHarmony 5.0.3 release和Ascend HDK 24.1.RC2，完成昇腾310B芯片的板级适配。包含device_board_ascend（板级代码，含310BFormatTest制卡工具、ascend/310B硬件配置、patch补丁集、tests测试）、device_soc_ascend（SoC代码）、vendor_ascend（厂商定制配置）。补丁集覆盖构建白名单添加、内核驱动修复、系统用户添加、GPU/EGL图形支持、音频ALSA适配、摄像头V4L2适配、HDC调试工具修复、多屏显示支持、服务裁剪优化等。提供一键编译脚本build.sh和自动化合patch脚本ascend_env.sh。

**cann-install（CANN推理引擎移植模块）**：在OpenHarmony系统上安装CANN 8.2.RC1推理环境。包含HDC工具配置、环境初始化、Python 3.10安装、toolkit/runtime/opp/ncs/compiler/aoe/pyACL/test-ops/hccl等子包安装、NNAL(atb)加速库安装、kernels算子包安装、PyTorch/torch_npu等推理依赖安装、MindIE-LLM/Qwen3-0.6B模型运行。提供YOLOv5目标检测demo和Resnet50分类demo验证。

**yolo-detect（YOLOv5推理应用模块）**：板端Python命令行推理程序，使用ais_bench库通过CANN/ACL API调用昇腾NPU进行YOLOv5推理。支持图片预处理（letterbox缩放、BGR2RGB、CHW转换）、模型推理（InferSession）、后处理（NMS非极大值抑制）、结果可视化（OpenCV绘制检测框）和JSON结果输出。同时提供HTTP推理服务（yolo_service.py）供图形应用调用。

**yoloapp（OpenHarmony图形应用模块）**：ArkTS图形界面工程，通过HTTP调用yolo-detect服务。提供AppScope和entry模块框架，但entry/src/main/ets/目录为空，核心UI代码未提交。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| 内核大小超过30M限制（因打开CONFIG_HILOG、CONFIG_HISYSEVENT、CONFIG_BLACKBOX等开关） | 更新hboot2将内核大小限制从30M升为40M |
| 鸿蒙系统根文件系统默认只读，无法安装CANN包 | 手动mount -o remount,rw / |
| 鸿蒙自带ln不支持-r参数，CANN包shell脚本无法执行 | 手动解压并分步骤安装 |
| initmodel.sh执行失败 | 屏蔽第一行的/var/dmp_daemon命令 |
| pip依赖解析器报错 | 可忽略，不影响安装 |
| libhccl.so/libtorch_npu.so找不到 | 重新设置LD_LIBRARY_PATH环境变量 |
| Qwen3-0.6B目录权限错误 | 执行chmod 755 /cann-install/Qwen3-0.6B |
| 构建系统不识别Ascend 310B硬件模块 | 添加Ascend-specific build targets到compile_standard_whitelist.json |
| 内核HDF驱动编译失败（缺少SEC_FUNCTION库） | 注释掉SEC_LIB_SOURCES和SEC_FUNCTION_OBJECTS行 |
| 缺少HwHiAiUser等系统用户 | 添加新用户和组到group/passwd |
| GPU渲染和EGL图像支持默认关闭 | 启用graphic_2d_feature_rs_enable_eglimage标志 |
| EGL函数符号缺失（EglSetCacheDir） | 添加OHOS_MESA3D条件编译，提供自定义SetCacheDir实现 |
| 大数据结构初始化导致栈溢出崩溃 | 将大initializer list替换为push_back()调用 |
| 鸿蒙自带toybox不支持awk、chattr、tr等命令 | 移植bash和busybox |
