#### 3.1.1 详细模块介绍
项目采用分层架构设计，从下到上依次为：硬件感知层（驱动占位）→ Linux HAL层（GPIO/UART/PWM/sysfs抽象）→ sensor_daemon后台采集服务（核心业务逻辑）→ 数据持久化层（JSON/CSV文件）→ Qt本地交互层 + remote_server远程Web查看层。此外还有rootfs部署与自启动层、测试验证层。

**sensor_daemon**：核心后台服务，每秒循环执行：生成仿真传感器数据（使用正态分布随机数，周期注入异常值）→ 读取config/thresholds.json阈值配置（每秒动态读取，无需重启）→ 调用check_alarms()判断是否越界 → 调用decide_device_status()和apply_device_status()控制设备 → 写入data/latest_sensor.json、data/device_status.json、data/alarm.log。支持SIGINT/SIGTERM优雅退出。

**remote_server**：基于POSIX sockets的轻量HTTP服务器，单线程阻塞accept循环，监听8080端口。提供4个只读REST API和嵌入式HTML/CSS/JS仪表盘（每秒自动刷新）。JSON解析和序列化均为手动实现，无外部依赖。

**qt_monitor**：Qt Widgets桌面应用，主窗口1040x720，包含4个传感器卡片、阈值配置表单（QDoubleSpinBox/QSpinBox）、报警开关按钮、报警日志表格（QTableWidget，显示最近20条）。使用QTimer每秒刷新数据，QSaveFile原子写入配置。

**Linux HAL层**：通过sysfs接口操作GPIO和PWM，通过POSIX termios操作UART。所有硬件操作均封装为独立类/函数，与上层业务逻辑解耦。当前为仿真阶段，HAL层已实现但未连接真实硬件。
