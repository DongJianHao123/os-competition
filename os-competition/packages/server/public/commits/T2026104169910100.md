#### 3.1.1 详细模块介绍
项目是一个基于 RISC-V QEMU virt 平台的 AMP（非对称多处理）双核异构通信框架。Hart 0 运行 StarryOS（Linux 兼容内核），Hart 1 运行 rt-async 裸机异步实时运行时。两核通过共享内存（SHM）和核间中断（IPI）实现低延迟通信。

**ov-rpc 模块**：核心 RPC 框架，基于 ov_channels 共享内存无锁 SPSC 环形缓冲。定义了 CH0（请求）、CH1（响应）、CH2（急停）三个通道。支持四种调用模式：call（请求-响应+IPI 回复）、call_poll（请求-响应+客户端轮询）、send（单向通知）、urgent（高优先级急停）。提供 define_service_client! 宏生成类型安全客户端。包含 30+ 单元测试覆盖所有功能。

**rt-async-app 模块**：Hart 1 上的固件应用。demo.rs 是主入口，使用优先级抢占调度器。intercom.rs 实现弹性忙等消息处理：设置 BUSY 标志→处理所有消息→弹性自旋等待更多工作→清除 BUSY→最终竞争检查→休眠等待 IPI。实现了 ECHO、ADD、DELAY 三个 RPC 服务。

**chip-qemu-virt-rt 模块**：硬件抽象层，封装了 UART1 初始化与轮询、PLIC 中断控制器配置、CLINT 定时器与 IPI 操作、以及向 Hart 0 发送 IPI 的功能。

**xtask 模块**：构建系统，提供 cargo xtask 子命令。setup 克隆并打补丁 OpenSBI/QEMU，build 构建所有组件，run 启动双核 QEMU 实例，install 将用户程序安装到 StarryOS rootfs。

**user-test-sched 模块**：最完善的测试程序，使用 sandwich 计时法测量 DELAY 操作的延迟抖动。配置 SCHED_FIFO 实时调度和 CPU 亲和性，对 0/50/100/200/500/1000/5000 µs 延迟各测 300 轮，输出 min/max/mean/stddev/p50/p95/p99 统计和直方图。
