#### 3.1.1 详细模块介绍
OSEval项目采用极简单Agent架构设计。main.py作为CLI入口，通过argparse接收target、--compare、--config等参数，加载oseval.yaml配置后调用pipeline。pipeline.py编排完整的分析流水线：Phase0克隆/复制仓库，Phase1进行仓库普查生成survey_report.md，Phase2深入分析生成full_report.md，Phase2b生成评估报告eval_report.md，Phase3a生成项目简述brief.md，Phase3b提取结构化特征features.json，Phase3c注册到past_projects.json。Compare模式额外执行Phase4a/4b进行历史项目对比。agent.py实现核心Agent循环，通过Jinja2模板构建系统提示，支持bash命令执行、持久化PTY子进程管理等5种工具调用。config.py基于Pydantic实现配置验证，支持多角色模型配置。context.py实现滑动窗口上下文管理，使用启发式token估算。llm_client.py封装OpenAI客户端，支持流式响应和工具调用。sandbox.py提供沙箱化bash执行环境。process_manager.py基于pty.fork实现跨平台持久交互子进程管理。ui.py基于rich库提供终端UI。registry.py维护past_projects.json项目注册表。build_tools_detector.py通过YAML配置驱动构建工具检测。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| Git克隆时Windows NTFS文件名限制 | 默认启用core.protectNTFS=false |
| 上下文窗口溢出 | 滑动窗口截断策略，保留system message，以完整turn为单位丢弃 |
| LLM输出JSON解析不稳定 | 三级降级策略：直接解析→代码块提取→首字符定位 |
| 跨平台终端兼容性 | Windows强制UTF-8，POSIX使用pty.fork，Windows使用pywinpty |
| 工具链检测失败导致Agent反复尝试 | 预先通过build_tools_detector检测，结果注入system prompt |
