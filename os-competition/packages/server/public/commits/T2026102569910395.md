#### 3.1.1 详细模块介绍

**VFS 核心层**：项目在 MIT xv6-riscv rev5 基础上增加了 VFS 层，核心设计是在文件描述符层增加 `FD_VFS` 类型，通过挂载表（最长前缀匹配）分派到不同文件系统后端。未命中挂载点的路径继续走原版 xv6fs，保证向后兼容。VFS 维护自己的打开文件表，通过 `vfsresolve()` 完成路径分派。新增 `mount(source, target, fstype)` 和 `vfsinfo()` 两个系统调用。代码量约 3360 行，是项目最大的单个文件。

**ext2 后端**：直接解析 ext2 磁盘格式，支持块大小 1024 字节、单个 block group、inode 12 直接块+1 一级间接块。实现了路径遍历、文件创建/删除、目录创建/删除、读目录、文件读写、truncate、硬链接等功能。镜像作为 xv6fs 里的普通文件承载，通过 `readi()/writei()` 读写。

**FAT32 后端**：解析 FAT32 BPB，支持 512 字节扇区、簇链管理、8.3 短文件名（统一转大写）。实现了目录项查找/创建/删除、簇链扩展、文件读写时分配新 cluster、mkdir 写入 `.`/`..`、unlink 释放簇链等功能。FAT32 不支持硬链接（返回失败）。

**HAL 抽象层**：将启动、trap、页表、中断、磁盘、串口等平台相关代码收敛到 `hal.h` 统一接口。RISC-V 后端调 PLIC、virtio_disk、`w_stvec`、`w_satp` 等；LoongArch 后端使用 DMW 窗口、TLB refill、LS7A 中断控制器、PCI virtio-blk。内核主路径只包含 `hal.h`，不直接依赖架构细节。

**LoongArch 移植**：从零实现 LoongArch 架构支持，包括：entry 启动（配置 DMW 窗口、初始化 TLB）、TLB refill 处理（`tlbrefill_loongarch.S`）、trampoline 用户态切换、上下文切换（`swtch_loongarch.S`）、中断处理（LS7A 寄存器）、PCI virtio-blk 磁盘驱动、UART 轮询串口。编译带 `-msoft-float -msimd=none` 避免向量指令。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| VFS 路径规范化：挂载点路径带尾斜杠（如 `/mnt/`）与不带斜杠（如 `/mnt`）被视为不同路径 | 在 `vfsmount()` 中规范化目标路径，去除尾斜杠 |
| ext2 目录扩容：往目录塞入多个文件时目录块空间不足 | 实现 `ext2_dirlink_inode()` 中的目录块扩展逻辑，分配新块并链接 |
| FAT32 簇链复用：反复创建删除大文件后 FAT 链可能损坏 | 实现 `fat32_free_chain()` 正确释放簇链，并在分配时复用空闲簇 |
| LoongArch 用户态 TLB 缺页：用户态访问未映射页面时无法正确转发 | 实现 `tlbrefill_loongarch.S` 将用户态缺页转回通用 trap 处理 |
| LoongArch 中断路由：UART 和 virtio 中断无法正确分发 | 使用 LS7A 寄存器做 IRQ mask/status/clear，通过 `UART0_IRQ`/`VIRTIO0_IRQ` 判定 |
| 双架构编译：切换架构时 .o 文件混用导致链接错误 | 在 Makefile 中要求 `make clean` 后再切换架构 |
| FAT32 硬链接：FAT32 不支持硬链接但未明确拒绝 | 在 `vfslink()` 中管控，只允许 ext2 同挂载点内链接，FAT32 返回失败 |
| ext2 单间接块：超过 12 个直接块的文件无法写入 | 实现 `ext2_inode_block()` 中的一级间接块映射 |
