#### 3.1.1 详细模块介绍
**task1 - OpenHarmony移植补丁集**：该项目核心工作是将OpenHarmony 5.0.3 Release移植到昇腾310B开发板（Ascend HDK 24.1.RC2）。包含43个补丁文件，按子系统分类：构建系统（添加昇腾白名单、相机白名单、exec_script支持）、HDF驱动（删除SEC_FUNCTION、V4L2相机适配、音频ALSA配置、编解码器）、图形GPU（EGL Image支持、FrameBuffer BGRA8888修复、多屏显示、bootanimation等待时间）、多媒体（相机状态枚举、音频策略解析、音效禁用、视频像素格式）、输入（多屏鼠标隐藏修复）、窗口管理（多屏Launcher修复）、启动（添加HwUser用户、sandbox配置、video设备权限、selinux适配）、开发工具（HDC小流修复）、安全（关闭selinux）。补丁命名规范（编号+子系统+功能描述），便于追溯。提供一键构建脚本build.sh和烧录指南。

**task2 - CANN推理环境搭建**：基于启智01开发者套件（昇腾310B平台），在OpenHarmony系统上搭建CANN 8.2.RC1全量推理环境。涵盖hdc调试工具使用、环境初始化、Python 3.10安装、CANN toolkit各子包（toolkit/runtime/opp/ncs/compiler/aoe/pyACL/test-ops/hccl）安装、加速库nnal安装、kernels包安装、推理依赖（torch 2.1.0、torch_npu等）安装、环境变量配置。验证了两个推理demo：YOLOv5目标检测（world_cup.jpg）和beagle分类（概率0.905956），以及MindIE Qwen3-0.6B大模型对话推理。文档含截图证据和常见错误处理。

**task3 - YOLO目标检测**：基于Ascend NPU的YOLOv5目标检测命令行工具（detect.py 503行 + det_utils.py 229行）。支持通过ais_bench进行NPU推理，包含letterbox预处理、NMS后处理、坐标缩放、结果可视化（OpenCV绘制边界框）、多种输出格式（标注图片、YOLO格式txt、JSON）。支持批量图片处理、置信度/IoU阈值调节、类别过滤、每类颜色区分、详细参数配置。模型文件yolo.om约15MB，提供测试图片world_cup.jpg和运行截图Screenshot.png。

**task4 - 310B板级配置与性能优化**：昇腾310B开发板的完整板级配置目录。config.json定义产品名310B、设备厂商ascend、arm64架构、API 8，包含安全、HDF、图形、AI、多媒体、通信、USB等子系统组件配置。product.gni委托device.gni进行实际配置。HDF配置（device_info.hcs）定义platform、display_composer、allocator、camera、audio等7个host的驱动设备。音频HAL配置5个adapter（primary×2、primary_ext、usb、a2dp）。窗口显示配置（display_manager_config.xml）设置dpi=240、无曲面屏、无裁切。资源调度配置（res_sched_config.xml）为空。安全配置含关键进程列表和sanitizer检查列表。预装应用配置含install_list等。性能测试与优化报告PDF存在但无法解码文本内容。
