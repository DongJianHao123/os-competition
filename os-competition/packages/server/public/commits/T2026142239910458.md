#### 3.1.1 详细模块介绍
项目crashvisor是一个基于AArch64 Type-1虚拟化架构的Rust no_std虚拟机监视器，运行在QEMU virt平台。当Guest内核崩溃时，EL2监控层在异常入口处冻结Guest执行，保存完整现场（通用寄存器、系统寄存器、Stage-2故障信息、故障地址附近4KB内存快照和时间戳），数据以二进制blob写入预留内存区域，同时通过串口输出格式化dump。

**main.rs**：项目入口，使用`#![no_std]`和`#![no_main]`，通过`include_bytes!`嵌入Guest镜像。创建VM（vmid=0），映射PL011 UART MMIO区域，加载Guest内核到0x40080000，激活Stage-2转换，初始化VcpuContext（包括ELR_EL2、SPSR_EL2、SP_EL1、SCTLR_EL1、TTBR0/1_EL1、TCR_EL1、MAIR_EL1、VBAR_EL1），通过内联汇编`enter_guest`进入Guest。自定义panic_handler打印恐慌位置并进入WFI循环。

**boot.rs**：从QEMU入口0x80000开始，读MPIDR判断CPU0，EL3→EL2降级，创建EL2 1:1页表（2MB块，1GB覆盖），配置TCR/MAIR/VTCR，使能MMU，跳转`rust_main`。其他核心进入WFE自旋等待。页表存放在`.data.pgtables`段，EL2栈16KB在`.bss`段。

**vm.rs**：Stage-2三级页表（PGD→PMD→PTE），4KB粒度，40-bit IPA。提供`stage2_map_block`（2MB块映射）、`stage2_map_page`（4KB页映射）、`stage2_walk`（三级walk翻译IPA→PA）。使用Bump allocator从0x200000线性分配。v0.2修复了`stage2_map_block`中pmd_pa使用陈旧L1条目的bug，增加了`stage2_walk()`和TLB无效化。

**vcpu.rs**：定义`VcpuContext`结构体（repr(C)），包含31个通用寄存器、SP_EL1、ELR_EL2、SPSR_EL2、7个EL1系统寄存器、fault_ipa和esr。实现Clone和Copy trait。

**exception.rs**：向量表（2KB对齐） + 汇编trampoline（~150条指令） + Rust trap分发。trampoline保存全部GPR到EL2栈，读取系统寄存器，构造VcpuContext，调用Rust handler，恢复并eret返回。按ESR_EL2.EC分类处理：WFI/SVC/HVC/SMC/系统寄存器/数据中止/指令中止等。数据中止时先尝试按需分页（分配物理页、建立Stage-2映射、TLB失效），内存耗尽或权限故障视为真正崩溃。

**crash.rs**：`CrashReport`结构体（repr(C)）约4.5KB，包含magic（0xC7A5B0FF）、自增序号、时间戳、31个GPR、SP_EL1、ELR_EL2、SPSR_EL2、7个EL1系统寄存器、2个Stage-2错误寄存器、故障地址附近4KB内存快照及基地址。环形缓冲区存储最多4份报告。`snapshot_guest_memory`通过Stage-2 walk将Guest IPA翻译为物理地址后memcpy。UART驱动轮询PL011状态寄存器，支持`uart_putc`、`uart_puts`、`uart_put_hex`、`dump_hex`。

**dump.rs**：将崩溃报告写入物理地址0x48000000，格式为[u32 count][CrashReport × N]，可通过QEMU pmemsave提取。

**regs.rs**：定义HCR_EL2、SCTLR_EL1/EL2、VTCR_EL2、SPSR_EL2、ESR_EL2、MAIR_EL2、TCR_EL2等位掩码和默认值，以及Stage-2页表标志、页表常量、Guest物理内存布局。

**guest/test_guest.c**：裸机aarch64测试Guest内核，初始化UART，安装EL1异常向量表，设置x8~x12为标记值（0xDEAD0000等），故意触发数据中止（解引用0x1000），若未被捕获则执行未定义指令，最后进入WFI循环。

**build.rs**：检查目标是否为aarch64-unknown-none，指定链接脚本link.ld。
