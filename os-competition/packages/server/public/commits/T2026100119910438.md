#### 3.1.1 详细模块介绍
项目由两个核心子系统组成：osgdb（操作系统跨特权级调试）和async-debug（异步执行流调试）。osgdb实现了边界断点定位与方向识别技术，能够在内核态与用户态切换时自动切换断点组，支持动态进程组注入和跨Rust版本变量读取。async-debug提供了异步函数调用树的追踪能力，通过Async Inspector面板以图形化方式展示异步执行流。项目整体以VS Code插件形式交付，通过DAP协议与GDB/MI2后端通信，实现了统一的调试体验。测试验证覆盖了37个状态机单元测试、StarryOS全流程验证和embassy异步框架验证。
