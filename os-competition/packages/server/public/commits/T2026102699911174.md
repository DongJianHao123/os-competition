#### 3.1.1 详细模块介绍
Agent-OS 是基于 rCore-Tutorial-v3 (v3.6.0) 扩展的操作系统内核，面向AI智能体场景。项目采用增量式开发，从任务一到任务六逐步扩展内核功能。

**任务一**：在原有PCB基础上扩展Agent元数据（AgentQuota、AgentInfoStruct），在用户地址空间中为Agent进程分配128KB专属Context区。实现了sys_agent_create（创建Agent进程，支持ELF加载和配额设置）和sys_agent_info（查询Agent进程元数据，包括心跳状态、上下文路径统计、工具调用计数等）。

**任务二**：实现了结构化交互接口，包含4个内核工具：query_process（按类型查询Agent进程）、get_system_status（获取系统状态）、send_message（进程间消息传递）、query_file（按名称查询文件）。通过sys_tool_call和sys_tool_list系统调用提供统一的工具调用协议。

**任务三**：实现了环形缓冲上下文路径管理，节点存储在用户态Context区。支持上下文节点推送（sys_context_push）、查询（sys_context_query）、回滚（sys_context_rollback）、清除（sys_context_clear）和元数据查询（sys_context_meta）。采用FIFO淘汰策略，受配额限制。

**任务四**：扩展文件系统，实现文件属性存储（键值对）、反向属性索引（O(1)查询）和内容摘要索引。提供6个系统调用：属性设置/获取/删除/查询、摘要设置/查询。属性索引支持多条件查询。

**任务五**：实现Agent Loop机制，包括心跳定时器（可配置间隔）、事件监听（支持IPC消息、文件修改、定时器、系统告警4种事件类型）、阻塞等待（返回触发事件位掩码）。非Agent进程调用相关系统调用返回-1。

**任务六**：覆盖5/5创新加分方向。D1事件补全：文件修改事件、定时器事件、系统告警事件；D2智能预测：N-gram预测引擎，支持查询和反馈；D3零拷贝共享：跨Agent进程页表级共享内存；D6沙箱探索：文件系统快照/提交/回滚，支持事务性操作；D8动态工具注册：运行时工具注册和注销。
