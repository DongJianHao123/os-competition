#### 3.1.1 详细模块介绍
项目采用四层架构设计：内核探针层（eBPF 程序采集进程、文件、网络、SSL 事件）、事件采集层（collector/main.c 加载 BPF 骨架并共享 RingBuffer）、分析引擎层（analyzer/*.c 实现规则匹配与告警生成）、输出层（JSON 格式告警日志）。eBPF 探针使用 tracepoint 钩子（execve、fork、openat、unlinkat、connect）和 uprobe（SSL_write/SSL_read），用户态 Collector 通过 bpf_map__reuse_fd 实现多探针共享 RingBuffer。Analyzer 实现了 7 条安全规则：SHELL_SPAWN（低危）、SHADOW_ACCESS（高危）、SSH_KEY_ACCESS（中危）、WORKSPACE_ESCAPE（高危）、SENSITIVE_DELETE（严重）、SUSPICIOUS_OUTBOUND（高危）、CONNECTION_BURST（高危），以及 AGENT_LOOP（高危）循环检测。Agent Registry 实现 PID→Agent 身份归因，支持静态配置和动态注册，通过进程树溯源实现跨层关联。Prompt Registry 通过 Unix DGRAM Socket 接收 Agent 上报的 prompt，支持重复检测和 SSL 明文提取。
