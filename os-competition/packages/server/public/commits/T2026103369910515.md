#### 3.1.1 详细模块介绍
项目基于 uCore C 教学操作系统，在 RISC-V 64 QEMU 平台上扩展了 Agent 能力。核心设计理念是"内核强制机制、用户空间策略"——内核提供安全的系统调用接口和数据结构，Agent 逻辑在用户态实现。项目共完成 6 个任务：(1) Agent 进程注册与基础信息查询；(2) 结构化工具调用（4 个内置工具）；(3) Context Path 上下文路径管理（环形缓冲区 FIFO）；(4) 文件属性索引查询（含性能对比）；(5) Agent Loop 心跳驱动循环（含 mailbox 事件唤醒）；(6) 综合管理 Agent 演示。代码结构清晰，os/ 目录为内核源码，user/ 目录为用户态库和测试程序，docs/ 目录包含设计和测试文档。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| syscall 分发中 SYS_unlinkat 缺少 break 导致 fall-through 到 SYS_spawn | 代码中存在该 bug，需添加 break 修复 |
| sys_pipe 中 f0/f1 未初始化即用于错误判断 | 需在声明时初始化为 -1 |
| sys_exec 使用静态 strpool 数组，非线程安全 | 当前单核环境下问题不大，多核需改为动态分配 |
| sys_wait 未验证用户空间地址可写性 | 需添加 copyout 前的地址验证 |
| sys_spawn, sys_set_priority, sys_fstat, sys_linkat, sys_unlinkat 为 stub（返回 -1） | 这些功能未实现，需后续完善 |
| 文件标签持久化未实现 | 当前标签仅内存中有效，重启后丢失 |
| 复杂事件源（如定时器、信号）未实现 | 当前仅支持 mail 事件 |
