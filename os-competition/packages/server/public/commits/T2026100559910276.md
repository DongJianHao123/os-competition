#### 3.1.1 详细模块介绍

**Trace事件写入模块**：实现了异步JSONL格式的trace事件写入器，支持tensor、KV cache、expert routing和OS内存事件的记录。采用生产者-消费者模式，通过后台线程异步写入文件，避免对推理主路径的性能影响。支持批量写入和缓冲区管理，确保trace数据完整性。

**Tensor/OS hint跟踪模块**：核心优化模块，实现了从tensor访问追踪到OS hint下发、expert prefetch的完整链路。包含ExpertSliceCache类实现基于策略的expert缓存替换，支持route/lru/lfu/window_lfu/least_stale五种策略。实现了异步expert prefetch队列，支持deadline-aware优先级调度（score/deadline/deadline_score三种模式）。通过madvise/posix_fadvise系统调用向内核发送内存访问提示，降低major page faults。

**分析脚本模块**：Python分析引擎，处理trace输出的JSONL文件，生成tensor访问热力图、KV复用统计、expert路由分布、OS内存指标等分析结果。支持多维度指标计算和可视化输出。

**替换策略模拟模块**：trace-driven离线模拟器，在真实trace数据上回放LRU/LFU/window-LFU/least-stale等缓存替换策略，评估不同预算下的miss率和eviction次数，生成Pareto前沿图辅助策略选择。
