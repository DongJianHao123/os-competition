#### 3.1.1 详细模块介绍
项目H-KVMS针对大语言模型推理中KV缓存显存瓶颈，设计并实现了GPU/CPU/SSD三层异构存储管理系统。系统通过vLLM原生OffloadingSpec扩展点透明集成，无需修改vLLM核心源码。核心创新包括：(1)多因子热度感知淘汰策略，综合访问频率、驻留时间、block年龄等多维度因子；(2)Eager D2S Slot提前释放优化，在解码阶段提前释放GPU slot；(3)init_patch显存预算突破，绕过vLLM初始化时的显存预算限制。系统在RTX 5090 + Qwen2.5-7B平台上达成KV缓存命中率99.98%、平均访问延迟−0.02%（优于原生）、有效上下文扩展倍数1.3333×、吞吐率提升+84.2%的硬指标。开发过程遵循三阶段路线（概念验证→最小可行闭环→指标攻坚），建立了六级证据链体系确保结果可复现。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| vLLM初始化时显存预算限制导致无法分配SSD offload空间 | 通过init_patch绕过显存预算检查，在初始化阶段预留offload空间 |
| SSD写入操作拖慢生成过程 | 采用双写缓冲(Dual-Write Buffer)机制，异步写入SSD，避免阻塞解码 |
| 原生vLLM的D2S(Device-to-SSD)调度过于保守 | 实现Eager D2S策略，在解码阶段提前释放GPU slot，提高GPU利用率 |
| 冷数据识别不准确导致频繁迁移 | 设计多因子热度感知淘汰策略，综合访问频率、驻留时间、block年龄等多维度因子 |
