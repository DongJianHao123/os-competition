#### 3.1.1 详细模块介绍

**injector.c**：用户态探针控制面的总入口。它解析命令行参数和探针规格，在运行进程中解析目标函数地址，利用运行时元数据和补丁规划器生成安装计划，通过session_ctl驱动两阶段远程安装，并将会话状态持久化到磁盘。支持install、update、cleanup、status、enable、disable、dump-log、clear-log等全部命令。代码约2100行，结构清晰，注释完整。

**session_ctl.c**：底层远程操作后端，是项目中最核心的执行引擎。它通过ptrace与目标进程交互，执行远程mmap/mprotect/munmap、字节级补丁写入、线程管理。支持install-stage、install-apply、install-session、enable、disable、dump-log、clear-log、cleanup、status、debug-retctx、debug-regs等命令。代码约2900行，是项目中最大的源文件。

**patch_plan.c**：指令级重写模块，是项目中最接近指令级重写的模块。它将目标函数入口附近的原始机器码转换为安全的入口补丁（跳转到trampoline）和gateway（重定位的原始指令）。支持x86_64和ARM64双架构，包含内置x86解码器和ARM64指令重定位逻辑。代码约2100行。

**hook_runtime.c**：注入目标进程的运行时逻辑，与trampoline汇编共同构成探针热路径。它实现函数参数捕获（前6个寄存器参数）、返回值捕获、条件过滤、callout（调用目标进程函数）、日志写入（512KB环形缓冲区）和SIGSEGV恢复。无libc依赖，仅使用原始系统调用，支持多线程安全。代码约1140行。

**log_parser.c**：二进制日志快照解码器。它读取目标进程导出的环形缓冲区快照，按时间顺序重排，解析单个日志记录（函数进入/返回/参数/错误事件），并以JSON（自动化回归测试）、纯文本（人工阅读）或perf-script风格（与perf/ftrace时间线合并）三种格式输出。

**elf_utils.c**：提供两个核心能力：(1) 读取/proc/pid/maps构建目标进程已加载模块视图；(2) 解析ELF文件和符号表，将符号/提供者名称解析为运行时地址。

**json_min.c**：轻量级JSON解析器，支持对象、数组、字符串、非负整数、布尔值和null，无外部依赖，用于读取配置文件。

**trampoline.S / trampoline_arm64.S**：汇编级跳板代码，分别支持x86_64和ARM64架构。负责保存/恢复CPU上下文，分派到C运行时（func_hook/func_ret_hook），并处理返回路径。

**injector.py**：Python版CLI入口和编排模块（早期版本）。提供install、enable、disable、dump-log、clear-log、cleanup、update、status等命令。后续核心功能（install、cleanup、enable、disable等）已迁移到C实现以显著降低延迟。

**run_tests.py**：自动化回归测试框架。启动目标进程，发送命令，收集结果，运行16个测试类别（基本功能、多探针、切换、多线程、条件过滤、高级条件、多子句条件、字符串过滤、callout、热更新、perf导出、perf合并、getpid基准、demo_sum6基准、安装/清理延迟、内核uprobe对比），并生成Markdown报告。
