#### 3.1.1 详细模块介绍

**ACT模型定义模块**：项目核心模块，实现了完整的Action Chunking Transformer架构。视觉编码器使用ResNet18作为骨干网络，将RGB图像编码为特征序列；状态编码器通过两层MLP将机器人状态映射到隐藏维度；CVAE模块支持条件变分自编码器的训练和推理，包含重参数化采样和KL散度损失计算；Transformer Encoder/Decoder采用pre-norm结构，支持自注意力、交叉注意力和残差连接；动作预测头从Decoder输出预测动作序列；时间集成模块支持推理时的动作平滑。该模块由组织方提供，选手在此基础上进行导出和部署。

**ONNX导出工具**：选手自行开发的工具，将PyTorch模型封装为适合ONNX导出的形式。定义了ACTONNXWrapper类，依次执行图像编码、状态编码、latent编码、拼接、位置编码、Transformer Encoder/Decoder和动作头输出。导出参数包括input_names=['image','state','latent'], output_names=['action'], opset_version=17, do_constant_folding=True。

**C++推理程序**：选手自行开发的核心成果，在StarryOS/QEMU riscv64环境中使用ONNX Runtime C++ API实现推理。程序读取预处理后的二进制张量（image, state, latent），创建ONNX Runtime会话，执行模型推理，对输出的8步action chunk进行反归一化（使用预计算的q01/q99分位数），并通过比较左右速度判断方向（LEFT/RIGHT/STRAIGHT）。推理耗时约18.4-18.7秒，峰值内存占用454MB。

**文档与结果展示**：选手提供了详细的开发文档Report_Task3.md和结果展示docs/task3_results.md，完整记录了从PyTorch参考实现到StarryOS/QEMU推理的全过程，包括环境搭建（Docker, make ARCH=riscv64）、依赖安装（apk add onnxruntime onnxruntime-dev g++ make）、文件传输（HTTP server + wget）、编译运行等步骤，并对两个代表性样例（frame_000000和frame_000227）进行了验证，方向判断结果分别为LEFT和RIGHT，与预期一致。
