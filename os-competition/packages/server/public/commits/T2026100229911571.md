#### 3.1.1 详细模块介绍
**CacheManager**：采用LRU淘汰策略，使用Python OrderedDict实现O(1)访问。在10000向量×200查询的基准测试中，缓存命中率达到98.7%，显著减少了磁盘I/O。支持配置缓存容量上限，适配内存受限场景。

**PrefetchEngine**：基于线程池实现异步预取，在HNSW图遍历过程中，根据当前节点的邻居信息预测下一步需要访问的节点，提前发起I/O请求。与HNSWIndex深度集成，有效隐藏I/O延迟。

**LSMManager**：借鉴LSM-Tree思想，将随机写转换为顺序追加写。包含MemTable（内存表）、SSTable（排序字符串表）和CompactionManager（合并管理器）。支持WAL（预写日志）保证数据持久性，异步合并策略减少写放大。

**HNSWIndex**：实现分层可导航小世界图索引，使用numpy加速向量距离计算。支持ef_construction=200、ef_search=150等参数配置。在纯读场景下Recall@10=93.4%，混合读写场景下Recall@10=92.8%，均超过赛题要求的85%。

**MmapVectorStore**：利用Python内置mmap模块实现大向量数据集的内存映射访问，避免传统文件I/O的系统调用开销。支持按需加载，减少内存占用。

**AdaptiveIOController**：根据当前I/O负载和缓存状态动态调整预取策略和缓存管理策略，在读写混合场景下优化整体性能。

**VectorIOEngine**：作为统一入口，整合CacheManager、PrefetchEngine、LSMManager、HNSWIndex等模块，提供向量插入、检索、删除等标准API，兼容LangChain接口。
