#### 3.1.1 详细模块介绍
AstraKV-W 项目围绕 LLM 推理中的 KV Cache 优化展开，系统按六层组织：入口与编排层（cli.py, scripts/entrypoints/, configs/）、对象抽象层（astrakv/kv_cache/）、证据观测层（astrakv/runtime/）、策略决策层（astrakv/prefetch/, astrakv/scheduler/）、执行验证层（vLLM/LMCache endpoint, astrakv/vm/）、评估展示层（scripts/reporting/, scripts/plotting/, results/）。各模块以骨架和适配器模式实现，核心贡献在于建立可观测、可分析、可调度的 KV 缓存管理框架，而非直接修改 vLLM/LMCache 等第三方运行时内部。

- 明确程度：明确
