#### 3.1.1 详细模块介绍
OxideJS 是一个用 Rust 编写的轻量级 JavaScript 执行引擎，采用经典的 parse → compile → execute 流水线架构。项目核心创新在于"单 Kernel 多 VM"运行时结构：KernelCore 进程级永久共享 key 驻留、Shape、编译缓存和 IC 模板；KernelSession 按会话管理 BuiltinWorld 与 global object；多个 oxide_vm 实例独立运行，共享同一份 KernelCore。

编译器模块（oxide_compiler）将 oxc 解析生成的 AST 编译为自研的 32-bit 固定指令寄存器式字节码，支持闭包捕获分析、函数声明提升、符号表管理等。虚拟机模块（oxide_vm）执行字节码，采用 NaN-boxing 值表示、Shape 对象布局（隐藏类思想）、Inline Cache 缓存属性偏移，并实现了 Epoch Arena + JsString GC 的分层内存管理。内置对象模块（oxide_builtins）用 Rust 原生实现了 Array、String、Object、Map、Set、Date、JSON、RegExp、Error 等 ECMAScript 标准内置对象。

团队在开发过程中重点进行了大量性能优化：IC 命中/未命中路径 depth=0 快速路径内联、Shape 加 depth 字段和 positions 反向索引实现 O(1) 属性查找、复合赋值运算符 int 快速路径跳过 coercion 开销、字符串拼接消除冗余拷贝、热路径冗余检查消除、内置函数跳过冗余哈希校验等。同时修复了多个关键 Bug：嵌套函数调用闭包捕获链路、闭包值传递链路、继承作用域分析误判等。

| 复合赋值运算符每次执行都进行 coercion | 增加 int 快速路径，当两个操作数均为 int 时直接计算 |
