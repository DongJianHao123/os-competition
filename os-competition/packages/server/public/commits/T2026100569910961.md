#### 3.1.1 详细模块介绍
**HAL 层**：定义在 hal.h/hal.c 中，提供 hal_is_riscv/hal_is_loongarch 架构检测、hal_irq_enable/hal_irq_disable 中断控制、hal_irq_init PLIC 初始化、hal_timer_init/hal_timer_read 定时器、hal_mmu_init MMU 存根。使用 #ifdef 编译时隔离，Makefile 条件编译，上层代码不直接操作寄存器。

**VFS 层**：定义在 vfs.h/vfs.c 中，提供五个核心接口。vfs_open 根据路径前缀 /ext2/ 路由到 ext2 操作，否则使用 xv6 原生 namei。vfs_read/vfs_write 根据 file.type 分发到对应文件系统。vfs_close 释放资源。vfs_fstat 获取文件状态。sysfile.c 中系统调用全部改为调用 VFS 接口。

**ext2 层**：定义在 ext2.h/ext2.c 中，实现 ext2_mount、ext2_read_super、ext2_read_bgdt、ext2_read_inode、ext2_write_inode、ext2_readdir、ext2_read、ext2_lookup、ext2_alloc_block、ext2_alloc_inode、ext2_open_inode。使用硬编码磁盘布局（超级块块 2001，块组描述符块 2002，根 inode 2，测试文件 inode 12），块号转换因子 4。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| HAL 层头文件包含顺序导致编译错误 | 调整 include 顺序，补充缺失的 defs.h |
| VFS 中 create 函数在 sysfile.c 和 fs.c 重复定义 | 将 create 从 sysfile.c 移至 fs.c |
| 设备文件空指针 panic | 在 vfs_read/write 中增加 FD_DEVICE 的空指针检查 |
| 目录打开失败（vfs_open 对目录类型处理不当） | 修正 vfs_open 中目录文件的打开逻辑 |
| fstat 返回 type=0 | 修复 fstat 中文件类型赋值 |
| O_CREATE 导致日志事务冲突 | 调整 O_CREATE 处理流程，避免在日志事务中嵌套操作 |
| virtio 设备 2 不稳定导致 ext2 读取失败 | 将 ext2 数据嵌入 virtio 设备 1（与 xv6 文件系统共用） |
| printf 延迟导致 ext2 读取前数据未就绪 | 在 ext2_mount 前增加延迟等待 |
| ext2_read_super 空指针 | 修复 ext2 超级块读取中的指针初始化 |
| fs.img 重建后 inode 位置偏移 | 调整 ext2 中 inode 块号计算 |
| /ext2 路径匹配问题 | 修正 vfs_open 中路径前缀比较逻辑 |
| ext2_read 中内核/用户空间缓冲区拷贝错误 | 使用 copyout 替代直接内存访问 |
| O_CREATE 在 ext2 中导致 panic | 将 ext2 写操作标记为预留接口，暂不启用 |
