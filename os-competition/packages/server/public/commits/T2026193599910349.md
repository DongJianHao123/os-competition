#### 3.1.1 详细模块介绍
**进程管理模块**：在 uCore ch8 基础上扩展了进程生命周期管理，实现了 ZOMBIE 状态（exit 时保留进程结构，wait 时完全清理）和 ORPHAN 收养机制（孤儿进程被 init 进程收养）。实现了进程组管理（setpgid/getpgid），基于 nice 值的静态优先级调度（优先级 = 19 - nice，范围 -20~19），以及资源限制（rlimit，支持 RLIMIT_NOFILE 和 RLIMIT_NPROC）。系统调用数约 8 个。

**内存管理模块**：在原有 SV39 页表管理基础上，新增内存保护（mprotect 修改页权限）、内存同步（msync，当前为 no-op）、System V 风格共享内存（shmget/shmat/shmdt，支持最多 16 个段，每段最多 16 个附件）、内存锁定（mlock/munlock，进程级锁定范围管理）、内存回收（mem_reclaim_pages 扫描 ZOMBIE 进程回收页面）和内存统计（get_mem_stats/get_mem_reclaim_stats）。系统调用数约 9 个。

**文件系统模块**：在 xv6 风格 5 层文件系统（buf/inode/log/file/dir）基础上，新增文件属性查询（stat/fstat/fstatat，支持 AT_SYMLINK_NOFOLLOW）、符号链接（symlinkat/readlinkat）、硬链接（linkat/unlinkat）、文件权限修改（fchmod/fchmodat）、目录操作（mkdirat/unlinkat）、文件重命名（renameat2）和文件预读缓存（readahead，bmap_readahead 不分配新块）。系统调用数约 13 个。

**并发同步模块**：新增读写锁（rwlock，写优先策略）、屏障（barrier，N 线程同步）、原子操作库（基于 GCC 内置原子操作）、无锁队列（Michael & Scott 算法，CAS 实现）和无锁栈（Treiber 栈），以及死锁检测框架（DFS 环检测）。系统调用数约 15+ 个。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| ZOMBIE 进程资源释放时机不当导致内存泄漏 | 分离资源清理和状态管理，exit() 时清理资源但保留结构，wait() 时完全清理 |
| uCore 未实现 clone() 系统调用，优先级调度无法使用线程 | 改用 fork() 创建进程 |
| 文件系统预读可能分配新块，不符合预读语义 | 实现 bmap_readahead() 只获取已存在块号，用 bread()/brelse() 填充缓存 |
| 无锁数据结构直接返回内核指针导致用户空间访问错误 | 改用进程池返回索引 ID |
| 无锁数据结构销毁函数错误释放池条目 | 只释放节点重置池状态 |
| 原子操作类型不一致 | 统一使用 volatile uint64_t* 类型 |
| 测试程序常量名/类型名错误（O_CREAT→O_CREATE，struct stat→Stat） | 修正常量名和类型名 |
| 测试程序数组过大导致栈溢出（4096→1024） | 减小数组大小 |
| 测试程序 ID 0 误判为错误 | 改用 (unsigned long long)-1 表示错误 |
