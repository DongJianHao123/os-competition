#### 3.1.1 详细模块介绍
项目采用模块化设计，各模块职责清晰。main模块作为入口，解析用户输入的目标程序路径和PID后调用inject_probe。injector模块负责反汇编分析目标函数指令边界，并生成FF 25绝对跳转指令覆盖原函数入口。probe_lifecycle模块是核心，实现了探针从注入、监控、撤销到清理的完整生命周期，支持最多16个并发探针，每个探针有独立监控线程。process_control模块通过ptrace的PTRACE_SEIZE/INTERRUPT机制冻结目标进程所有线程，采用"快捕慢等"策略提高效率。remote_mm模块通过远程系统调用注入在目标进程内分配RWX内存。trampoline_manager模块构建跳板代码，包含参数采集和返回值采集两套汇编固件，支持动态地址修补。utils模块提供跨进程内存读写（处理8字节对齐）、ELF符号解析（支持C++名字还原）、参数类型解析等基础工具。target模块是测试目标，包含多个不同参数数量和类型的函数用于验证。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| 函数入口指令长度不足14字节无法安全Hook | 使用Capstone反汇编计算指令边界，不足MIN_HOOK_LEN则跳过该函数 |
| 函数末尾无法安全劫持获取返回值 | 通过替换返回地址跳转到跳板2（返回值跳板），安全采集返回值后再跳回 |
| 返回值寄存器被覆盖导致返回错误值 | 在跳板中完整保存和恢复寄存器现场，确保返回值不被破坏 |
| 多线程目标进程冻结时线程逃逸 | 采用"快捕慢等"异步并行策略：先快速向所有线程发送打断信号，再统一轮询等待 |
| 跨进程内存读写8字节对齐问题 | 读取时先8字节对齐后逐字读取再裁剪，写入时组合数据+NOP填充按8字节块写入 |
| 远程内存分配地址随机化（ASLR/PIE） | 通过/proc/pid/maps获取模块基地址，结合ELF符号偏移计算函数绝对地址 |
| 探针清理时需感知当前状态 | 清理逻辑自带状态感知，自动判断探针处于开启还是关闭状态，先关闭再清理 |
