#### 3.1.1 详细模块介绍
**QEMU CPU 推理模块**：基于 ONNX Runtime 的 C++ 推理程序，加载静态 QDQ 量化后的 ONNX 模型（INT8+FP16 混合精度），在 StarryOS QEMU 环境中运行。支持图像预处理（双线性插值 resize 至 224×224 + ImageNet 归一化）、状态归一化（基于分位数的 min-max 映射）、动作反归一化。提供单帧推理和 CSV manifest 批量评测两种模式，计算 MAE、RMSE、多阈值转向准确率等指标。包含自定义 ORT 分配器跟踪、分阶段计时和内存快照功能。

**SG2002 TPU 推理模块**：基于 CVI Runtime 的 C++ 推理程序，在 SG2002 真机上运行。支持三种模型加载方式（路径/fd/内存映射），提供 CPU 标量和 RVV 向量化两种预处理后端。RVV 后端使用 vlxseg3bu 向量化 gather-load 实现 RGB 横向插值，将 resize 从 26.236 ms 降至 13.972 ms（加速约 46.7%）。支持 INT8+BF16 混合量化模型，包含完整的评测框架（开环/闭环模式、转向决策、JSON 结果输出）。

**QEMU /proc 补丁**：为 StarryOS 内核添加 /proc 伪文件系统支持，使用 SimpleFs 抽象构建。提供 /proc/[pid]/stat（52 字段 Linux 兼容格式）、statm、status（含 VmRSS/VmHWM）、cmdline、comm、exe、fd、maps、mounts、meminfo 等条目。使用 Weak 引用避免循环引用，支持 oom_score_adj 读写。部分字段（如 utime/stime/minflt）为默认值。

**SG2002 DMA/SDMMC 补丁**：为 SG2002 平台实现 CV1811 SD/MMC 主机控制器驱动。支持完整的 SD 卡初始化序列（CMD0-CMD58, ACMD6/41/51）、4-bit 总线宽度、1.8V/3.0V/3.3V 电压切换、UHS-I SDR50 模式。读取路径支持 ADMA2 → SDMA → PIO 三级回退，ADMA2 在初始化时通过 PIO 对比验证。写路径仅 PIO。包含 T-Head 缓存一致性操作。

**SG2002 ION 补丁**：为 SG2002 实现 ION 内存管理子系统，提供 System、DmaCoherent、Carveout 三种堆类型。Carveout 堆使用 first-fit 分配器（BTreeMap 管理空闲区间），分配对齐到 4K 页。ION 设备支持 HEAP_QUERY、ALLOC、FREE、IMPORT 四个 ioctl 命令，mmap 将缓冲区映射到用户空间。为 CVI/NPU 的 DMA 模型加载提供连续物理内存。
