#### 3.1.1 详细模块介绍
本项目基于llama.cpp二次开发，新增src/flexinfer/目录实现FlexInfer内存管理机制。核心模块包括：

**平衡内存锁定（balanced_lock）**：实现论文§3.3-3.4的平衡锁算法。通过flexinfer_compute_lock_plan函数，在给定内存预算下，先锁定全局张量（token_embd/output），再按Algorithm 1批量锁定FFN张量，最后用balanced_lock_candidates均衡分配剩余预算。支持四种消融模式：balanced（默认）、unbalanced（按层顺序）、attn-first（注意力优先）、ffn-first（FFN优先）。代码位于src/flexinfer/balanced_lock.h/cpp，约25KB。

**异步预取（runtime）**：实现论文§3.5的异步预取机制。在decode前通过std::async异步touch未锁定张量的内存页，触发内核缺页加载。支持通用decode前预取和Qwen2层级预取。代码位于src/flexinfer/runtime.h/cpp，约5KB。

**张量收集与分类**：在模型加载阶段，Collector遍历GGUF权重，记录每个张量的名称、层号、类别、偏移、大小等信息。Classifier按规则将张量分为GLOBAL、ATTN、FFN、NORM、OTHER五类，避免误分类。

**锁页与回退**：使用mlock锁定选中张量的mmap区间，失败时记录失败张量、移除计划、加入预取、重算actual plan。未锁定页使用MADV_DONTNEED提示内核回收。

**实验与消融**：提供完整的实验框架，包括单元测试（验证balanced方差≤unbalanced、Algorithm 1全FFN阶段、消融差异）、真实模型加载测试（5种配置）、llama-bench消融benchmark、Docker/cgroup内存压力测试脚本。
