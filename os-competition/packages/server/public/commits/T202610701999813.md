#### 3.1.1 详细模块介绍
Kmon 项目基于 AxVisor Type-1 Hypervisor（Rust 编写），实现了完整的虚拟机崩溃监控与转储系统。项目采用模块化设计，核心模块包括：

**监控模块**：通过 pvpanic-mmio doorbell 机制实现协作式 guest panic 检测，配合 VmHealth 状态机对崩溃类型进行分类（ConfirmedCrash/RunError/EntryFailure/GuestShutdown/SuspectHang）。该模块已通过 QEMU e2e 验证，源码位于 infra/tgoskits/components/axdevice/src/pvpanic.rs。

**信息采集模块**：在 guest 崩溃时进行有界采集（约 20-40 KiB），覆盖 KernelText/KernelData/KernelLog/Stack 等内存段，同时捕获 AArch64 寄存器快照。采集数据封装为 ACDA（Automatic Crash Dump Artifact）二进制格式，包含 40B 信封和 FNV-1a 校验。该模块通过 shell-chunk export 方式导出，Host 侧可完整重建。

**语义重构模块**：Host 侧离线 analyzer 对 ACDA artifact 进行解码，提取 KernelLog、执行保守栈字节扫描、进行 root-cause 分类，并生成 ELF64 AArch64 ET_CORE mini-vmcore（含 PT_NOTE 和 PT_LOAD 段），可用 readelf/gdb-multiarch 进行离线分析。

**raw-block 存储模块**：实现了 RB0-RB4 原始块设备写入与解码管线，支持全量物理内存 dump（单次 256MB payload），通过 QEMU virtio-blk 作为 dev stand-in 验证。RB5 真实裸机落地列为决赛工作。

**开发过程**：项目采用 AI Agent 编排开发模式，从 2026-05-08 至 2026-06-30 历经多轮迭代。开发过程中经历了方向性返工（x-kernel 弃用）、roadmap drift 修正、环境阻塞（OOM、下载失败）等挑战，均诚实记录在 AI 使用时间线和审计日志中。项目采用 test-first/failing-first 回归、tmux dev-container 验证、多 worker 并行 sprint 等工程实践。初赛收尾阶段（2026-06-30）完成了两个遗留缺陷修复（axvmconfig 模板字段补全、gdb-multiarch 容器安装）并通过了绿色验收门 7 步全流程。
