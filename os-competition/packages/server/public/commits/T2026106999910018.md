#### 3.1.1 详细模块介绍

**项目概述**：本项目基于 OSDI'25 发表的 XSched 开源框架（XpuOS/xsched，Apache-2.0 协议），将其适配到龙芯 2K3000 开发板的集成 LG200 GPU 上。XSched 是一个跨平台 GPU 调度框架，支持 CUDA/HIP/LevelZero/OpenCL/Ascend 等多种后端，通过统一的 XQueue 抽象和三级抢占能力模型实现多任务调度。本团队新增了 LG200 平台支持，包括平台注册、Level-1 HAL（资源发现、队列封装、命令封装）、设备发现工具、XQueue 提交链路验证和基准测试工具。

**平台注册模块**：在 `include/xsched/types.h` 中新增 `kPlatformLG200` 枚举值，在 `protocol/include/xsched/protocol/def.h` 中注册平台定义，在 `protocol/src/names.cpp` 中添加 "LG200" 名称映射。这是将 LG200 纳入 XSched 统一框架的第一步。

**LG200 HAL 层**：遵循 XSched 的 HAL 分层设计，实现了 Level-1（待提交命令抢占级）的队列和命令封装。资源发现通过 Linux DRM/sysfs 扫描候选设备，读取 `gpu_busy_percent`、VRAM 使用量等字段。由于 LG200 驱动 ABI 尚未完全确认，第一阶段使用 callback command 作为执行体，避免绑定未稳定的 SDK 接口。

**工具链**：`lg200_info` 用于设备信息读取，`lg200_callback_demo` 验证 XQueue 提交路径，`lg200_bench` 提供吞吐量、平均延迟、P95、P99 等统计指标，支持 CSV 输出。Windows 本地模拟 Demo 可在无板端硬件的情况下展示完整流程。
