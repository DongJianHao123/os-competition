#### 3.1.1 详细模块介绍

**probe模块**：提供底层探测工具集，包括通过dlsym解析函数地址、通过解析/proc/self/maps获取已加载共享库信息、通过mprotect修改内存页权限为RWX、安全的内存读写操作以及字节数组转十六进制字符串工具。该模块是其他所有模块的基础依赖。

**utils模块**：提供内存页对齐相关的工具函数，包括获取系统页大小、地址向下页对齐、计算覆盖指定内存范围所需的页数，以及打印当前进程内存映射信息。这些函数被memory模块用于mprotect调用前的地址对齐计算。

**handler模块**：定义探针框架的核心数据类型。regs_t结构体按栈顺序保存x86_64调用约定的寄存器参数(rdi, rsi, rdx, rcx, r8, r9, rax)，probe_handler是用户注册的回调函数类型，probe_filter是条件过滤函数类型，HookFlags枚举控制探针行为（如是否捕获返回值）。

**hook模块**：项目的核心模块，实现了一个完整的、线程安全的Inline Hook框架。支持在x86_64架构上动态生成跳板代码，通过覆盖目标函数入口指令跳转到跳板，跳板保存寄存器上下文、调用用户注册的回调函数、执行原始指令、最后跳回原函数。支持多探针共存（同一函数可注册多个探针，按注册顺序依次触发）、条件探针（通过过滤器函数控制是否触发）、动态开关（临时禁用/启用探针而不卸载）、多线程安全（通过暂停/恢复机制和互斥锁保护）、返回值捕获（通过替换返回地址实现）。ARM64支持部分实现（仅入口跳转）。

**remote模块**：实现跨进程内存操作，支持通过ptrace附加/分离远程进程，使用高效的process_vm_readv/writev系统调用进行远程内存读写（回退到ptrace PEEKDATA/POKEDATA），解析/proc/pid/maps获取远程进程内存映射，以及通过比较本地和远程共享库映射偏移来解析远程符号地址。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| 初始Hook实现中跳板代码使用pushfq/popfq保存/恢复标志寄存器，导致栈对齐问题 | 在第八次提交中移除pushfq/popfq，优化寄存器保存序列，保持16字节栈对齐 |
| 多线程环境下安装/卸载Hook时可能导致正在执行的线程访问部分修改的指令 | 引入暂停/恢复机制（pause_all/resume_all），通过原子标志和自旋等待确保修改期间无线程执行跳板代码 |
| 单探针设计无法满足同一函数多个探针的需求 | 第五次提交重构为FuncGroup+RegisteredHandler架构，支持同一函数注册多个探针并按序触发 |
| 跨进程内存读取时ptrace方式效率低 | 优先使用process_vm_readv/writev系统调用（无需ptrace停止目标进程），失败时回退到ptrace方式 |
| 远程符号解析需要跨进程计算地址 | 通过比较本地和远程进程的共享库映射偏移量来计算远程符号地址 |
