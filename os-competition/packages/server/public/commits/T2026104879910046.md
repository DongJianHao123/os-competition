#### 3.1.1 详细模块介绍
本项目基于 Phytium D3000M ARM64 平台，使用 Rust 语言和 Type-1 虚拟化技术，构建了一套双 VM 解耦架构的虚拟化内核崩溃监控与语义恢复系统。Monitor Guest（ArceOS Unikernel, Rust no_std）独立于 Target Guest（Linux）运行，通过 HVC 超调用通信，无共享内存。

**HVC 通信层**：定义了 7 条 hypercall（#7-#13），涵盖崩溃检测、寄存器读取、内存读取、文件导出等功能。设计原则是"最小权限"——Monitor Guest 只能通过 HVC 获取必要信息，不能直接操作 Target Guest 内存。

**崩溃监控模块**：采用三条互补检测路径确保全覆盖。路径 A 通过 HVC #10 轮询 + #13 主动通知，响应时间 ≤200ms；路径 B 通过跨 VM 读取 jiffies 实现死锁探测，响应时间 ≤30s；路径 C 通过 SPSR/ESR 事后识别 Double Fault。特别实现了 jiffies_live 假阳性防护——必须观察到 jiffies 变化才启用 hang 计时。

**Capture 信息采集模块**：完整的采集流水线包括：冻结 VM → 读寄存器（ESR/FAR 双保险锁存）→ 内存转储（128MB 部分转储）→ dmesg 采集（prb ringbuffer 两阶段解析）→ kallsyms 符号表解码 → 模块采集（三策略：ELF 扫描 + dmesg 提取 + 链表遍历）→ 持久化（JSON/文本/二进制）→ 导出。

**Recovery 语义恢复模块**：提供交互式 crash 控制台，支持 bt（栈回溯）、regs（寄存器）、info（系统信息）、dmesg（内核日志）、modules（模块信息）、memory（内存查看）等命令。栈回溯采用 FP 链 + dmesg fallback 双重策略。
