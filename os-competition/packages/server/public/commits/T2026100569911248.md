#### 3.1.1 详细模块介绍

**EDF 调度器模块**：在 NuttX 内核的 sched/sched/ 目录下新增 sched_edf.c 文件，实现了完整的 EDF 调度算法。核心数据结构为全局 EDF 就绪队列 `g_edf_readytorun`（按截止时间升序排列的双向链表）。主要函数包括：`nxsched_edf_queue_insert`（插入任务）、`nxsched_edf_queue_remove`（移除任务）、`nxsched_edf_select`（选择下一个运行任务）、`nxsched_process_edf`（每个 Tick 检查预算，预算耗尽时重新插入队列并可能触发切换）、`nxsched_start_edf`（设置周期、运行时间、截止时间偏移等参数）、`nxsched_resume_edf`（周期过期时刷新截止时间）。设计文档 edf-design.md 详细描述了背景动机、整体架构（EDF 与 FIFO/RR 隔离机制）、数据结构、关键算法、设计权衡（链表 vs 最小堆、SMP 支持、预算执行精度）及后续优化方向。

**EDF 基准测试模块**：在 apps/benchmarks/rt_bench/ 目录下新增 edf_test.c，是一个 EDF vs FIFO/RR 截止时间错过率比较测试程序。支持同构任务集测试（run_policy_test）、异构任务集测试（run_mixed_policy_test）、混合场景测试（run_mixed_scenario，5个任务周期20ms，截止期限从18ms到6ms）、高负载异构场景测试（run_mixed_highload_scenario，6个任务总利用率0.85）、利用率扫描（run_sweep，从1到10个任务）。通过命令行参数 --sweep/-s、--mixed-high、--mixed/-m 选择测试模式。

**RT-Bench 基准测试模块**：在 apps/benchmarks/rt_bench/ 目录下的 rt_bench.c，是一个实时调度基准测试工具。包含三个测试：Test 1 中断响应延迟（使用 DWT CYCCNT 和 POSIX 定时器测量绝对延迟）、Test 2 上下文切换时间（N=2/4/8/16 个任务的环形信号量传递）、Test 3 调度延迟（在不同后台负载 0/2/4 个自旋任务下测量）。使用高精度 DWT 计数器计时，支持 CSV 格式输出。

**数据采集与可视化工具**：tools/collect_data.py 通过串口自动连接 STM32F411 目标板，执行预定义测试命令并保存结果；tools/plot_results.py 读取测试数据生成图表（延迟分布图、EDF 对比图、调度延迟 vs 负载折线图等）和 Markdown 报告。
