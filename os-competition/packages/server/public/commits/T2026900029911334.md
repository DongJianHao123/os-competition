#### 3.1.1 详细模块介绍

本项目是一个基于 LLM（大语言模型）的自动化内核 CVE 热补丁生成系统，目标是为 Anolis OS ANCK 6.6.102-5.2.an23.x86_64 内核自动生成符合 kpatch 约束的热补丁模块。

**核心工作流（四阶段）：**
1. **信息收集阶段**：通过 search_cve 查询 NVD 获取 CVE 描述和上游 commit，通过 fetch_patch 下载补丁并检查是否已修复
2. **分析阶段**：通过 analyze_patch 解析补丁结构，检测 kpatch 约束违规（如结构体变更、asm 文件、__init 函数等）
3. **构建阶段**：通过 write_patch 写入对齐上下文的补丁，通过 build_livepatch 在 Docker 中调用 kpatch-build 构建，支持最多 10 次重试
4. **验证阶段**：通过 verify_livepatch 加载、验证并卸载热补丁模块

**关键特性：**
- 自动上下文对齐：write_patch 自动从真实内核源码获取上下文行，确保补丁可应用
- 智能错误分类：build_livepatch 将构建失败分为 7 种类型（patch_apply_failed, compile_failed, crc_mismatch, fentry_missing, kpatch_constraint, no_changes, unknown）
- 已修复 CVE 预检：fetch_patch 阶段通过 patch --dry-run 检测补丁是否已应用
- 断点续跑：run_testset.py 支持进度追踪，可从中断处继续
- 安全加固：所有文件路径经过正则验证，防止路径穿越
