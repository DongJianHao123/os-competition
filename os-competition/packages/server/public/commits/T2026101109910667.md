#### 3.1.1 详细模块介绍
**ArceOS 内核**：项目核心组件，采用模块化架构设计，支持 x86_64、aarch64、riscv64、loongarch64 四种架构。提供异步运行时（axtask）、网络栈（axnet/axnet-ng）、文件系统（axfs/axfs-ng）、内存管理（axmm/axalloc）、设备管理（axdriver）、同步原语（axsync）等模块。每个模块独立维护，有 CHANGELOG 和版本管理。

**StarryOS**：基于 ArceOS 构建的类 Linux 操作系统，实现了进程管理（starry-process）、信号处理（starry-signal）、虚拟内存（starry-vm）等 POSIX 核心功能。支持多架构运行，有完整的测试套件。

**AxVisor**：Type-1 虚拟机监控器，支持 x86_64（VMX/SVM）、aarch64、riscv64 架构。包含 VCPU 管理、中断虚拟化（VGIC/VPLIC/VLAPIC）、内存虚拟化（EPT/NPT）、设备模拟等完整虚拟化功能。

**设备驱动框架**：采用 rdrive 驱动框架，支持 FDT 和 PCI 设备枚举。包含丰富的真实硬件驱动：NVMe、SDHCI、DWMMC、FXMAC、Realtek RTL8125、Intel E1000、Rockchip NPU、ARM PL011/PL031、DW APB UART、ARM GIC、RISC-V PLIC 等。

**TPU/ML 推理优化**：针对 AFAC2026 赛题，在 256MB 内存限制的 RISC-V 平台上优化 ACT 模型推理。通过交叉对比 ncnn、ONNX Runtime、tract 三种框架，最终采用 ORT INT8+内存优化方案，峰值 RSS 仅 220.4MB，平均推理 9.9s/帧。

**ROS2/DDS 调优**：操作系统设计赛（OS 功能挑战赛道）项目，优化 ROS2 DDS 实时性。包含四个实验阶段：环境搭建与基准测试、中间件核心调优（QoS/SHM）、系统级极限优化（多 DDS 对比/RT 内核/网络抗干扰）、自动化验证与报告产出。

**KernelInsight**：面向操作系统内核赛道作品的仓库智能分析与历史对比系统。自动导入 Git 仓库，分析源码结构，从原创性、新颖性、可实践性、技术难度、完成度五个维度评分，生成 Markdown 分析报告。
