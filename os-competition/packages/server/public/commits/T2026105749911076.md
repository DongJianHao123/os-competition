#### 3.1.1 详细模块介绍

**probe 核心模块**：ALIS 的核心探针引擎，实现了基于 ptrace 的纯用户态函数级动态插桩。采用宿主-目标（Host-Target）分离架构，通过 ptrace 系统调用控制目标进程执行和内存，通过 POSIX 共享内存（shm）与目标进程内的探针 shellcode 交换数据。核心数据流为：目标进程执行到被 hook 的函数入口 → 执行 jmp 跳转至 shellcode 区域 → shellcode 保存寄存器上下文，将函数参数写入 ring buffer → 可选地在栈上压入 ret_probe 地址 → 跳回原函数继续执行 → 原函数返回时自动跳转至 ret_probe shellcode → 将返回值写入 ring buffer → 宿主侧监控线程轮询 ring buffer 并调用用户回调。探针注入流程包括：地址解析 → 入口指令分析 → 内存分配（mmap RWX shellcode 区域 + shm heap 区域）→ Ring Buffer 建立 → Shellcode 构造 → 写入与激活。支持相对跳转（5 字节）和绝对跳转（12 字节，使用 r11 寄存器），包含 Damaged Zone 跨越机制确保线程安全。

**ELF 符号解析器**：实现了三级查找策略——GNU hash（O(1) 期望）→ SYSV hash（O(k)）→ .dynsym 线性扫描（O(n)），支持 .symtab 回退和 .gnu_debuglink 调试文件搜索，兼容 PIE/非 PIE 可执行文件。代码结构清晰，包含 bloom filter 优化和安全边界检查。

**环形缓冲区**：无锁通信通道，基于共享内存 + `lock xadd` 原子操作。三指针协议：head（分配槽位）、committed（数据就绪）、tail（消费完毕）。条目对齐到 64 字节缓存行避免 false sharing，type 字段作为就绪标志最后写入确保数据完整性。arg/ret 匹配使用 rsp 作为 key，通过 pending_args_ 哈希表暂存。

**探针 Shellcode**：手写 x86_64 汇编，分为 arg_probe（函数入口）和 ret_probe（函数返回）。通过栈篡改技术捕获返回值：arg_probe 在栈上压入 ret_probe 地址覆盖原返回地址。使用 r10 保存原始 rsp 作为匹配 key，r12/r13 保存类型标志，支持浮点返回值（xmm0）。
