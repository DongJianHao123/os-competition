#### 3.1.1 详细模块介绍
**任务1：系统移植**：搭建Ubuntu 20.04编译环境，下载OpenHarmony 5.0.3源码和310B适配仓，替换板级支持文件，一键编译生成镜像。关键问题包括交叉工具链路径错误（需指定aarch64-target-linux-gnu）、JDK版本不匹配（需同时安装JDK 8和JDK 11）、制卡程序架构不匹配（x86 vs ARM，使用qemu-user-static解决）。最终成功启动图形桌面环境，支持HDMI 1080P显示和USB输入。

**任务2：CANN移植**：下载CANN 8.2.RC1 ARM64软件包，安装NNRT运行时和开发工具包，加载NPU驱动，配置环境变量。通过npu-smi info和Python ACL API验证NPU可用性。提供了完整的一键安装脚本（install_cann.sh）和环境配置脚本（cann_config.sh），以及安装验证脚本（check_cann.sh）。

**任务3：YOLOv5应用开发**：使用ATC工具将YOLOv5 ONNX模型转换为OM格式。开发Python程序main.py（ACL Runtime API推理+NMS后处理+WebSocket/HTTP服务）。开发OpenHarmony App（ArkTS/ArkUI，Stage模型），核心挑战是沙箱隔离，尝试7种文件共享方案均失败，最终采用WebSocket（端口8080）+HTTP（端口8081）网络回路（127.0.0.1）绕过沙箱。App通过WebSocket接收检测结果推送，通过HTTP加载图片展示。

**任务4：性能优化**：通过内核配置裁剪、服务启动顺序调整、init脚本优化、日志等级调整优化启动时间（基准50s）；通过内核内存精简、后台服务裁剪、图形内存优化、文件系统缓存参数调整优化内存占用（基准used 1.7 GB）。
