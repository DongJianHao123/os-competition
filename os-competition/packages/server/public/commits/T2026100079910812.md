#### 3.1.1 详细模块介绍

**RK3588 平台开发**：团队在 Orange Pi 5 Plus 开发板上完成了 StarryOS 的完整启动流程。环境搭建阶段解决了串口线波特率（1500000）、TFTP 服务器 Docker 部署、eMMC Maskrom 烧录等技术问题。启动时间分析发现总启动约 11.4 秒，其中 Shell 进程启动中的 init.sh 脚本（含大量调试命令）占 5.6 秒，精简后可降至约 5 秒。工作流优化将内核加载从 TFTP 网络加载改为 SD 卡 ext4load 本地加载，消除了对网线的依赖。SD 卡驱动问题排查是团队最核心的技术贡献：系统在 filesystem syncing 处卡死，经系统性分析定位为 DwMmc 中断风暴问题，临时修复为在 handle_irq 中添加约 1ms spin_loop 延迟，最终由上游周睿老师通过 PR#1446 重构 SDMMC IRQ 流程完成根本修复。

**应用层测试**：团队在 StarryOS 上集成了大量第三方应用测试。Redis、Nginx、MySQL 等数据库/Web 服务器应用提供了完整的 QEMU 测试流程。Go 语言测试（2018 个断言）和 Python 语言测试（21 个模块）实现了工业级语言层地毯式覆盖，输出确定化可逐字节比对。Apache 和 FFmpeg 部分功能因上游问题暂未完全启用。
