#### 3.1.1 详细模块介绍
KPatch-MAS 采用四智能体协作架构：Analyst（分析师）分析上游 CVE 补丁，输出结构化 Repair Plan JSON；Navigator（寻路者）根据 Repair Plan 定位并读取目标函数源码；Coder（编码者）将上游补丁改写为 kpatch 兼容的热补丁，输出 JSON 格式修改指令；Critic（评审者）诊断编译失败原因并给出反馈，可触发 Re-analyst 重分析。此外还有 Test-Agent 生成动态对撞测试脚本。系统配合四阶段对撞测试（PoC 触发→编译加载→回归验证→安全回滚），实现从上游补丁到可加载 .ko 模块的端到端自动化。技术栈为 Python + FastAPI + Pydantic + kpatch，目标内核为 Anolis OS 23 (ANCK 6.6.102-5.2.an23.x86_64)。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| replace_lines 不支持多行模式匹配 | 引入滑动窗口算法，支持多行替换 |
| 快速语法检查假阳性过多 | 简化检查逻辑，减少假阳性 |

| 增量恢复变量名错误 | 修复变量名 |
| Coder 生成的补丁语义不等价 | 通过 Critic 反馈循环迭代修正 |
| klp_shadow 误用（API、include、GFP flags） | Prompt 优化，添加 klp_shadow 模板 |
| 环境问题导致 vmlinux 损坏 | 增加环境健康检查 |
| 模型切换导致成功率暴跌（70.5%→18.2%） | 建议切换回 qwen-max 模型 |
