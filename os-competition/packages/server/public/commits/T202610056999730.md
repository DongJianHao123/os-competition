#### 3.1.1 详细模块介绍
项目采用分层模块化设计，共28个核心模块（src/目录下），分为八个层次：流水线调度层、仓库读取层、代码解析层、AI理解层、调用图与画像层、历史知识库层、RAG检索层、评估报告层。代码解析模块使用双引擎策略（优先Tree-sitter AST解析，失败时回退到正则切分），支持Rust、C、C++、Python、Bash、汇编等多种语言。AI函数理解模块利用大语言模型进行六维分析，支持并发处理、断点续跑和异常降级。调用图构建模块采用双轨提取（AI语义+正则规则），区分内部/外部调用，有置信度机制。混合检索模块结合结构化检索（权重0.65）和RAG语义检索（权重0.35），从历史项目库中检索相似项目。五维评分模块从原创性、新颖性、可实践性、技术难度、完成度五个维度评分，每维满分20分，总分100分，采用双轨评分机制（规则参考分+AI评分），偏差约束±3分。

此外，项目还包含ROS2/DDS调优与验证开发内容（README.md中描述），涉及QoS策略定制、传输层优化（SHM）、多DDS横向对比（CycloneDDS、Iceoryx）、RT内核改造（PREEMPT_RT）、网络抗干扰（tc netem）、动态QoS等，但该部分为实验记录和开发文档，核心提交代码为KernelInsight分析系统。
