#### 3.1.1 详细模块介绍
本项目围绕赛题"基于 eBPF 的网络服务加速与双端缓存系统"，以 eBPF 为核心，在 Linux 内核网络路径中完成 DNS 与 gRPC 两类网络服务的性能监控和快路径处理。DNS 模块从 tc 监控 MVP 起步，逐步扩展 XDP 缓存加速，实现了完整的 DNS 查询/响应匹配、延迟统计、缓存命中/未命中/过期统计，以及缓存命中时直接 XDP_TX 回包的能力。gRPC 模块实现了 transport 层监控和 h2c unary fast-cache 原型，支持通过 pinned BPF map 在线更新策略。统一策略工具 cachectl 提供了独立的控制面路径，支持 DNS 缓存、gRPC 方法白名单和 gRPC 响应缓存策略的统一校验与加载。虚拟化路径实验和云原生接入验证展示了项目在 OpenStack/KVM 和 Kubernetes 环境中的可部署性。
