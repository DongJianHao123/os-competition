#### 3.1.1 详细模块介绍

**axvisor-master (VMM/Hypervisor)**：基于 Rust 的 Type-1 Hypervisor，运行于 ARM64 EL2。核心 VMM 模块位于 src/vmm/，包含 crash_monitor.rs（实现 9 个 hypercall 子功能：ReadGuestMem、PauseVM、ResumeVM、GetVCpuStats、GetVCpuRegs、CaptureCrash、DumpRegion、HostLog、FullMemoryDump）、vcpus.rs（vCPU 运行循环，每个 vCPU 对应一个 axtask）、hvc.rs（hypercall 分发）、ivc.rs（跨 VM 共享内存通信）、fdt/（FDT 生成与解析）。HAL 层支持 aarch64 和 x86_64 架构。开发过程中修复了多个关键 bug：UB 指针逃逸（改用 Arc 消除 unsafe）、VM 状态守卫（防止在 Running 状态下读取不一致数据）、错误传播改进等。

**StarryOS (Guest OS)**：Linux 兼容宏内核，基于 ArceOS 生态。内核模块完整实现了内存管理（mm/）、任务管理（task/，含 futex、信号、定时器）、系统调用（syscall/，100+ Linux 系统调用）、文件系统（file/，含 epoll、pipe、signalfd、网络）、伪文件系统（pseudofs/，含 procfs、devfs、tmpfs）。monitor.rs 实现了崩溃检测与采集逻辑，通过 hypercall 与 Host 通信。

**Monitor 应用**：用户态 C 程序，运行在 StarryOS 上。通过 hypercall 轮询检测 Target VM 的崩溃状态，状态机实现 RUNNING→SUSPECTED→CRASHED 三级去抖判定。检测参数：POLL_MS=200ms, FAIL_ENTRY_THRESHOLD=3, NO_PROGRESS_POLLS=5。检测到崩溃后触发 PauseVM 冻结现场，然后执行 CaptureCrash 采集。

**kylin_boot**：麒麟系统引导集成。包含 crash_init（静态 init 程序）和 crash_trigger（支持 6 种崩溃类型的触发程序），通过 dracut pre-pivot hook 自动集成到 KylinOS rootfs。支持 panic、null 指针解引用、除零、栈溢出、死循环、OOM 六种崩溃场景。

**crash_analyzer**：离线分析工具（analyze.py，约 32KB）。支持 AArch64 FP Unwind 栈回溯、ELF 符号解析、内核版本识别、模块布局分析。输出格式支持 Text/JSON/Markdown 三种。crash 包结构包含 meta.json、modules.json、regs.json、code.bin、stack.bin、log.txt、data.bin 及 SHA256 校验和。

**测试框架**：完整的四层测试体系。单元测试（56 Rust + 21 Python，<5s 完成）、集成测试（单 VM 烟雾测试 5 检查点、Crash-Demo E2E 6 检查点）、E2E 测试（综合 20 检查点）、KylinOS 崩溃矩阵（6 种类型 × 4 区域 = 30 检查点）。总计 138 个用例/检查点，全部通过。

### 4.2 遭遇的代码问题

| --- | --- |
| UB 指针逃逸：get_vm_stats() 使用 unsafe 裸指针从 BTreeMap 获取引用，BTreeMap 重平衡时指针失效 | 改用 Arc<VmStats>，返回 cloned Arc，消除所有 unsafe 代码 |
