#### 3.1.1 详细模块介绍
本项目将 openvela 完整移植到 Renesas RA 系列开发板，主要验收对象为 RT-Thread Vision-Board / Renesas RA8D1（Cortex-M85），同时保留 DshanMCU RA6M5（Cortex-M33）作为最小 NSH bring-up 基线。开发周期约8天（2026-06-10至2026-06-29），共51个commit。

**阶段一（系统启动与基础驱动）**：完成芯片层和板级框架搭建，`__start()` → `nx_start()` 链路打通，SCI9 UART控制台稳定工作，GPIO LED/按键、SysTick定时器、NVIC/ICU中断控制器全部板端通过。reset后约0.585s进入nsh>。

**阶段二（核心子系统验证）**：多任务调度（FIFO/RR）、文件系统（Data Flash、QSPI、RAM-MTD挂载LittleFS并读写）、软件定时器精度、IPC（semaphore和POSIX message queue）全部板端通过。

**阶段三（扩展能力适配）**：RW007 Wi-Fi STA连接/DHCP/ping、SPI/I2C（camera SCCB/I2C读写OV5640）、PWM呼吸灯、DMA memcpy、RTC日历设置/读回/复位保持、PM Sleep/WFI（RTC周期唤醒和按键唤醒）全部板端通过。BLE scan基础通路可用。

**额外工作**：Cortex-M85 bring-up、Renesas FSP v5.7.0 HAL集成、SDRAM初始化、MIPI DSI LCD/framebuffer、Camera/OV5640 video框架、片上Data Flash MTD、外部W25Q QSPI Flash MTD、RW007 BLE原生命令、Camera到LCD联动示例、分阶段验收应用、同步+延迟初始化策略。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| BSS/DATA初始化错误 | `SystemInit()`后清除MSP limit，调整调用顺序 |
| PLL配置寄存器写入无效 | 确认调试器访问权限限制（S-TYPE-3安全属性），改用SRAM snapshot验证 |
| UART发送完成检测 | SCI_B模式需等待`SSR.TDRE`和`SSR.TEND`，添加超时保护 |
| UART接收乱码 | 同时检查`FRSR.R`和`CSR.RDRF`，启用`CCR3.RXDESEL`，使用FSP生成的baud配置 |
| MIPI DSI无显示 | 重新排序DSI init序列（PHY→HS clock→LP data→command），帧缓冲定位到SDRAM |
| Camera SCCB读写失败 | 修正上电序列（PWDN/RESET时序），封装分步寄存器操作 |
| RW007 Wi-Fi SPI通信和DHCP | CS时序改为每次交互独立toggle；修正以太网帧封装偏移 |
| DMAC Memory-to-Memory不生效 | open后缺少`enable()`调用，添加后正常 |
| LCD显示异常 | 添加VSYNC计数器，帧缓冲64-byte对齐，复位时序调整 |
