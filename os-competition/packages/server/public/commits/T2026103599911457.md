#### 3.1.1 详细模块介绍

**Task1: 系统移植**：项目基于OpenHarmony 5.0.3标准系统，针对启智01开发板（基于Ascend 310B处理器）进行移植。内核配置（ascend310B_oh_defconfig）针对ARM64架构，启用了Ascend AI特性（DVPP、OOM、共享内存池等）、DRM显示驱动（HISI Ascend 310B、Panfrost Mali GPU）、HDF硬件驱动框架、USB 3.0、音频、摄像头ISP等外设支持。设备树补丁共6个，分别修改了内核启动命令行以支持OHOS、修改GPU中断命名并启用DRM、启用hiusbc0 USB控制器并添加引脚复用、支持8T非 removable存储设备、添加音频设备节点、添加ISP摄像头（IMX219）支持。内核补丁共4个，包括添加Panfrost GPU驱动支持、支持Mesa3D pitch对齐、HDF框架集成、以及一个大型综合补丁（添加accesstokenid、hyperhold、blackbox、hievent、hilog、hisysevent、hungtask、zerohung等驱动和epfs、hmdfs文件系统）。编译构建使用build.sh脚本，成功编译后烧录至开发板并验证系统启动。

**Task2: CANN部署**：在已移植的OpenHarmony系统上，通过HDC工具连接开发板，确认NPU设备（310B1）状态正常（Health OK，功耗8.4W，温度48℃），验证Python 3.10.2环境可用，并成功导入ACL Runtime Python接口（aclruntime），确认CANN推理环境就绪。

**Task3: YOLOv5推理**：基于Ascend ACL Runtime（通过ais_bench库）实现YOLOv5目标检测推理。代码结构清晰，包含三个核心文件：det_utils.py提供预处理（letterbox缩放填充）、后处理（NMS、坐标缩放、裁剪）工具函数；main.py实现完整推理管线（图像预处理→模型推理→后处理→可视化绘制）；yolo_cli.py提供命令行接口，支持--model、--labels、--image、--output、--conf、--iou、--device等参数。此外，app_server.py实现基于HTTP的多线程图片检测服务，通过子进程调用yolo_cli.py进行推理，提供POST /detect和GET /result_image/两个端点。模型文件yolo.om（约15MB）和COCO类别标签文件已提供。推理结果展示了桌面和世界杯场景的检测效果。

**Task4: 性能优化**：针对OpenHarmony系统进行启动时间和内存占用优化。启动优化包括内核/服务裁剪、文件系统/镜像优化、内核启动参数调优、SD卡I/O调优；内存优化包括系统进程/服务精简、ZRAM/swap调整、内核内存调优、用户空间内存优化。优化后启动时间降至45秒，内存占用约1.5GB，均达到≥10%的提升目标。
