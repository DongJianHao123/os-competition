#### 3.1.1 详细模块介绍
项目基于 xv6-riscv rev5（MIT 6.1810 教学内核）进行扩展。README.md 声称实现了虚拟文件系统（VFS）和硬件抽象层（HAL），以及轻量级函数指针动态注册路由架构和高效教学级 Dentry Cache。然而，实际代码分析显示：
- **未发现 VFS 层实现**：kernel/fs.c 仍使用 xv6 原始的五层文件系统（块-日志-文件-目录-命名），没有抽象 VFS 接口。
- **未发现 HAL 层实现**：没有设备抽象层或硬件抽象接口。
- **未发现 Dentry Cache 实现**：没有目录项缓存结构。
- **未发现函数指针动态注册路由架构**：唯一的函数指针表是标准 xv6 的 syscalls[] 数组。
实际修改包括：进程管理函数重命名（kfork/kexit/kwait/kkill）、自定义 forkret 逻辑、惰性内存分配（vmfault/ismapped）、孤儿 inode 回收（ireclaim）、自定义 sbrk 惰性分配、pause 系统调用，以及三个测试程序（dorphan/forphan/logstress）。
