#### 3.1.1 详细模块介绍

**aka-rk3588（应用层）**：基于开源网球拾取机器人C++源码，团队将其从Linux环境迁移至StarryOS。核心为tennis.cpp中的追球状态机（CHASE追逐→ALIGN对齐→GRAB抓取→SEARCH搜索），通过帧协议与下位机（ESP32）通信。硬件抽象层（HAL）将摄像头、NPU推理、电机控制解耦为cam_hal、npu_hal、motor_hal三个接口，支持TCP、QEMU串口、真实板卡串口三种通信模式。源码级硬件替换层（tennis_mock）在不修改tennis.cpp的前提下，为各硬件模块提供mock替换，实现无外设运行验证。

**os-src（操作系统层）**：基于StarryOS（Rust编写，兼容Linux syscall ABI），团队完成了SD卡驱动（时钟频率从6.25MHz优化至约100kHz）、ext4文件系统适配、UART串口驱动、TCP/IP网络栈、NPU驱动骨架（/dev/rknpu，支持ioctl/mmap/DMA-BUF）等关键内核模块。启动时间从49秒优化至33秒，shell交互实现即时响应。

**工具链**：包括mock_tennis（纯软件模拟、串口模拟、TCP模拟三种验证方式）、create_img（SD卡镜像构建）、yolov8_rknn（AI模型转换）等。
