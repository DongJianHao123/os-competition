#### 3.1.1 详细模块介绍
K-RCA项目是一个面向Linux操作系统内核故障的根因分析系统，采用多阶段流水线架构。数据采集模块通过读取/proc文件系统（无需root权限）收集76维内核指标，包括CPU、内存、磁盘I/O、网络、软中断、负载、PSI压力等。故障注入模块支持CPU压力、内存压力、磁盘I/O、网络重载、THP压缩、进程炸弹等6类真实内核故障的注入。预处理模块对时序数据进行对齐、缺失值填充和鲁棒归一化。异常检测模块集成三种算法（Robust Z-Score、EWMA、Isolation Forest），采用加权融合投票机制，并包含回退策略。因果发现模块融合OS先验知识图、滞后相关性分析和Granger因果检验，可选PC算法增强因果方向。根因分析模块使用KCRank算法（加权评分公式：异常分×0.30 + 时间领先×0.25 + 出度×0.20 + 传播×0.20 - 症状惩罚×0.10）进行排序，可选随机游走重排和专家池路由。可视化模块导出因果图、时间线和传播路径，Web API提供FastAPI服务和深色科技风前端界面。
