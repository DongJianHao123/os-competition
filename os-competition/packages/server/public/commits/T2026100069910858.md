#### 3.1.1 详细模块介绍
JanOS 系统采用分层流水线架构，从语料层到服务层共 7 层。语料采集模块自动从 GitHub 克隆 150+ 个内核仓库并按文件切片；索引层构建 BM25 词法索引与 1024 维向量嵌入索引，支持混合检索；指纹与相似度引擎提取 OS 专有静态指纹（系统调用、数据结构、调度策略等 20+ 维度），通过三通道加权融合（词法 0.25 / 语义 0.20 / 指纹 0.55）计算相似度，并引入基线减除算法扣除同源模板共性；调用图模块使用 tree-sitter 抽取函数调用关系并染色分类；报告生成模块自动产出描述报告、top-5 比较报告、开发历史报告与 AI 使用报告；反幻觉机制通过 claim-evidence 硬约束（生成时绑定证据行号）、生成后证据回扫（分级判定：精确命中/行号近似/引文不匹配）、异源 critic 二审三道防线确保可信；谱系重建模块按代码传承关系构建家族树；Web 服务层提供交互式浏览与 AI 助手功能。所有模块均经过充分测试与迭代，代码质量高，文档完整。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| 同源红海导致相似度普遍虚高：大量内核基于相同教学模板（如 rCore），直接比较相似度虚高 | 引入基线减除算法，通过集合运算扣除同源模板的共性，计算去模板后的真实重合行数 |
| 大模型生成描述出现幻觉与证据误引：LLM 生成的结论可能引用不存在的代码行 | 建立 claim-evidence 硬约束（生成时绑定证据行号）+ 生成后证据回扫（分级判定：精确命中/行号近似/引文不匹配），不通过则披露 |
| 指纹字段串台：不同维度的指纹特征被错误归类 | 引入异源 critic 二审机制，由不同模型交叉验证，通过 merge_critics.py 合并结果 |
