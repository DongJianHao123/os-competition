#### 3.1.1 详细模块介绍
项目采用Clean Architecture分层架构，核心模块位于src/unified_api/core/，包含动作执行器（ActionExecutor）、执行上下文（ExecutionContext）、统一动作接口（UnifiedActionInterface）、配置管理（Config）和异常/日志工具。适配器层（adapters/）提供模拟器无关的抽象基类和工厂，当前实现了MockNavigateAdapter（用于测试）和AI2THORAdapter（基础实现）。领域层（domain/）定义了动作模型、枚举和值对象。验证模块（verification/）实现了导航结果验证器。开发过程遵循TDD方法，按Sprint 2.1→2.2→2.3→2.4顺序迭代，每个Sprint均有完整的DoD检查清单、测试报告和技术债务文档。
