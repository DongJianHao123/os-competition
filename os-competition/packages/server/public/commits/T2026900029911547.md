#### 3.1.1 详细模块介绍
AgentGuard采用"内核态采集 + 用户态分析"的端侧架构，核心为"规则—统计—机器学习"三层协同异常检测架构。系统通过eBPF技术在内核态动态加载监测程序，贯通"应用层交互语义—底层系统调用"的数据链路，实现对多个并发智能体的实时观测、异常识别与跨层因果归因。

eBPF探针层提供20+类探针，覆盖进程创建/退出、文件操作、TCP连接、系统调用、文件打开、信号、bind、capable、内存泄漏、OOM、挂载、TCP RTT、块设备I/O延迟、SSL/TLS等维度。探针使用CO-RE（Compile Once - Run Everywhere）技术，兼容内核≥5.15。

Agent核心负责探针的编排管理、智能体标识与生命周期管理。支持server模式（HTTP服务器，端口8527）、run模式（运行单个探针）、safe模式（安全规则检测）、seccomp模式四种运行模式。

异常检测引擎分为三层：规则层实时拦截已知威胁（Shell启动、敏感文件访问等）；统计层基于时序特征初筛（逻辑死循环、资源滥用等）；机器学习层使用改进的Isolation Forest集成模型进行深度异常检测。

跨层因果链模块构建从Prompt→Response→Syscall→File/Network的完整因果链路，基于风险评分（0-10）实现精确因果匹配，支持告警触发和可视化展示。
