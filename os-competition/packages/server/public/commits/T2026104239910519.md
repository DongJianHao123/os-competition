#### 3.1.1 详细模块介绍
项目采用经典编译前端四阶段设计：词法分析（lexer.rs）将源码转换为 Token 流，语法分析（parser.rs）将 Token 流解析为 AST，解释器（interpreter.rs）直接树遍历执行 AST 并产生运行时值。值系统（value.rs）支持六种类型和原型链，错误处理（error.rs）提供源码位置追踪。Agent 模块（agent.rs）封装了面向 AI Agent 的结构化执行接口，支持资源限制和 JSON 输出。CLI 入口（main.rs）提供 REPL、内联执行和文件执行三种模式。整个引擎零外部依赖，纯 Rust 标准库实现。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| Rust 所有权与 JS 引用语义冲突 | 采用 Rc<RefCell<Object>> 模式实现共享可变对象 |
| 闭包环境捕获 | remember_closure 机制，以函数指针地址为键存储环境 Rc 指针 |
| 数组空洞与 undefined 区分 | Vec<Option<Value>>，None 表示空洞 |
| Object.defineProperty 自引用描述符 | 先读取描述符到局部变量再写入 |
| Array.prototype.forEach 实时迭代 | 先记录初始长度，每次迭代读取当前值 |
| 自动分号插入（ASI）简化 | optional_semicolon 可选消费 + 换行检测 |
| 字符串原型方法自动装箱 | get_property_on_value 中查找对应原型对象 |
| 调用深度限制递归计数 | enter_call/leave_call 自增自减，超限时先恢复再返回错误 |
