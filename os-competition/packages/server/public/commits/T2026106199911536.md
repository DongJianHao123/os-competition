#### 3.1.1 详细模块介绍
**VFS层**：项目在xv6内核中实现了虚拟文件系统层，通过固定大小的mount table管理多个文件系统挂载。核心机制包括：最长挂载点匹配（支持根目录"/"匹配所有路径）、相对路径转换、cwd（当前工作目录）相对路径支持（init初始化为/，fork继承，exit/free清空）。VFS层负责将open/read/write/close/fstat/mkdir/unlink/link等系统调用分发到对应的文件系统驱动。同时实现了busy umount检查（拒绝卸载有打开文件的挂载点）和未支持系统调用（如mknod、exec）的显式失败守卫。

**ext2文件系统驱动**：完整实现了ext2文件系统的核心操作。支持1K/2K/4K block size，通过超级块魔数0xEF53识别格式。文件读写支持direct block（12个）、single-indirect、double-indirect、triple-indirect四级寻址，支持跨block group分配释放和稀疏读零填充。目录操作支持创建、删除、direct/indirect扩展（最多820个文件）。实现了硬链接（nlink维护）和open-after-unlink（删除已打开文件时延迟释放资源，最后一个引用关闭时释放）。fstat返回真实inode号、文件类型、设备号、链接计数、大小。

**FAT32文件系统驱动**：完整实现了FAT32文件系统的核心操作。通过BPB字段和0x55AA签名识别格式。支持8.3短名、ASCII LFN、UTF-8 BMP LFN、非BMP UTF-8 LFN（四字节UTF-8转UTF-16 surrogate pair）。实现了Unicode NFD/casefold等价匹配（查表比较，支持分解重音/预组合字符、ß/ss casefold）。目录操作支持创建、删除、cluster链扩展。LFN链管理支持创建（生成唯一短别名）、删除（从LFN链起点到短目录项逐项标记）、复用。open-after-unlink生命周期管理与ext2类似。fstat使用目录项偏移派生的伪inode号。硬链接明确拒绝（FAT32无POSIX hard link语义）。

**HAL层**：将架构相关代码抽象为hal.h接口，包括hal_cpuid()、hal_interrupt_init()、hal_interrupt_init_hart()、hal_virtio_mmio(dev)、hal_virtio_dev_from_irq(irq)。RISC-V后端（hal_riscv.c）封装了r_tp()、PLIC、UART/virtio IRQ、双virtio-mmio地址。LoongArch后端（hal_loongarch.c）提供交叉编译烟测通过的骨架，完整启动链路未完成。

**块设备层**：在原有xv6单virtio磁盘基础上，新增第二块virtio-mmio磁盘（EXTRADEV=2）作为测试盘，避免干扰根文件系统。提供blkdevtest验证读写通路。

**用户态测试工具**：项目包含大量专用测试程序，覆盖文件系统各功能维度：挂载测试（mnttest）、块设备测试（blkdevtest）、文件系统类型识别（fstypetest）、文件读写（fsreadtest、vfswritetest、vfscopytest、vfsbigtest）、文件删除（vfsdeltest）、indirect块（vfsindtest、vfstripletest）、目录操作（vfsdirtest、vfsdirtst、vfsdirind、vfsdirread）、长文件名（vfsutfstest、vfslfnstress、vfsunicodecmp）、边界测试（vfsedgetest）、状态查询（vfsstattest）、unlink-open（vfsunlinkopen）、cwd（vfscwdtest）、硬链接（vfslinktest）等。
