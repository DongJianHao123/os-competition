#### 3.1.1 详细模块介绍
HeteroKV 是一个将 LLM 推理中的 KV Cache 从单一 GPU 驻留扩展为 GPU/CPU/SSD 三层管理的系统。核心设计包括：APC-first 策略与 vLLM Automatic Prefix Caching 协同工作，避免冗余管理；OS-style 三层 KV Cache 控制面（GlobalKVBlockTable 作为单点真相源）；可解释冷热决策（LRFU + Hotness 双策略）；靠前 block/prefix 保护机制；SSD 乐观/悲观双模式；以及证据驱动的能力边界声明。系统提供完整的单元测试（~88 个测试文件）、集成测试、真实 vLLM runner 和 P3 原型 runner。对比系统包括 Pure vLLM、LMCache 及 HeteroKV 多种变体。

| --- | --- |
| APC-first 策略与 vLLM APC 的协同问题 | 设计 APC-first 策略，避免对 vLLM 已管理的 prefix block 进行冗余管理 |

| 预取优雅期管理 | 实现 prefetch grace period 和 completion tracking |
