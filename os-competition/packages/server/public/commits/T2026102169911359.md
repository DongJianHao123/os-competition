#### 3.1.1 详细模块介绍

**Agent 进程管理模块**：在 rCore 的 ProcessControlBlock 基础上扩展了 Agent 相关字段（agent_type、heartbeat_interval、loop_state、watched_events、pending_events、context_path_meta），新增 `sys_agent_create` 和 `sys_agent_info` 系统调用。Agent 进程拥有独立的用户态 Agent Context 区域，通过 PCB 元数据管理。该模块是 Agent-OS 的核心创新，实现了 Agent 与普通进程的共存。

**AI 调度器模块**：基于 3 级 MLFQ 调度器，新增 SchedFeatures（等待/运行比、抢占/自愿切换/阻塞/唤醒比、交互性评分、平均 CPU 突发）采集，通过规则树分类器（RuleTree）将任务分为 Interactive/Balanced/CpuBound 三类，动态调整队列级别和时间片。支持 Baseline 和 TrainedV1 两种配置，可通过系统调用动态切换。实验数据覆盖 4 轮，从基础 MLFQ 到 AI 调度逐步演进。

**Agent 循环与事件模块**：实现了 Agent 的生命周期管理，包括心跳设置/停止、事件监听/取消、阻塞等待（支持心跳和事件两种唤醒方式）。Agent 可以通过 `sys_agent_watch` 注册感兴趣的事件，通过 `sys_agent_wait` 阻塞等待事件或心跳超时，通过 `sys_agent_notify` 向指定 Agent 发送事件。

**LLM 查询模块**：实现了 Agent 与 LLM 的交互，通过 UART（QEMU 环境）进行 RPC 通信。支持缓存（FNV-1a 哈希）、限速（令牌桶）、种子文件回退（`llm_seed.txt`）。多 worker 并发查询时，缓存命中可显著降低延迟。

**工具调用模块**：Agent 通过共享内存请求/响应结构调用 7 种工具：GetSystemStatus、QueryProcess、ReadContext、QueryFile、QueryLlm、ReadFile、PatchFile。工具调用结果通过上下文路径记录，支持 Agent 的自主推理和决策。

**GUI 图形界面模块**：基于 `embedded_graphics` 实现了桌面、终端、按钮、图标、图片等组件。通过 GPU 设备驱动进行帧缓冲渲染，支持 BMP 图片解码。该模块在 OS 内核中直接集成 GUI 框架，具有较高的创新性。

**用户态 Agent 程序**：采用 demo/worker 模式，每个 Agent 功能对应一个 demo（负责环境准备和结果验证）和一个 worker（负责实际 Agent 逻辑）。覆盖了工具调用、上下文路径、文件查询、LLM 查询、循环事件、QRE（查询-推理-执行）工作流、自主 Agent、管理 Agent、DevStudio 等场景。
