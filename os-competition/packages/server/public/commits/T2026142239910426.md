#### 3.1.1 详细模块介绍

**Lab0 - 环境搭建**：基于MIT xv6-rev11（x86架构），在Ubuntu 20.04/22.04/WSL2环境下配置QEMU模拟器、GCC+Make工具链。提供setup.sh自动化脚本和详细的Step-by-Step教程，确保环境可复现。

**Lab1 - 系统调用**：实现两个自定义系统调用——hello（无参数，输出"Hello, this is a custom system call."，系统调用号22）和add2（两个整数参数，返回两数之和，系统调用号23）。完整覆盖从用户态程序到内核态处理的全流程：修改syscall.h、syscall.c、sysproc.c、usys.S、user.h，创建用户态测试程序hello.c和add2.c，更新Makefile。

**Lab2 - 进程管理与调度**：实现pstate系统调用（系统调用号24），遍历所有进程并输出其状态（UNUSED/EMBRYO/SLEEPING/RUNNABLE/RUNNING/ZOMBIE）。在fork()和scheduler()中添加调试输出以观察进程创建和调度过程。可选任务：修改调度策略（时间片从1改为2）。

**Lab3 - 内存与页表**：实现pgcount系统调用（系统调用号25），统计已分配的物理内存页数。高级任务：实现memstat系统调用，通过copyout将内存统计信息（总页数/已用页数/空闲页数）传递到用户空间。在kalloc/kfree中添加调试输出观察内存分配释放过程。

**Lab4 - 文件系统**：实现fdcount系统调用（系统调用号26），统计当前进程打开的文件描述符数量；实现fdinfo系统调用（系统调用号27），输出每个文件描述符的类型（PIPE/INODE）、读写属性和inode编号。在fdalloc()和fileclose()中添加调试输出观察文件描述符的分配和释放。

**Lab5 - 综合集成**：编写综合测试程序，依次调用hello、add2、fork3个子进程、每个子进程创建文件并写入数据、父进程读取验证、输出pgcount和fdcount统计。记录fork子进程的总耗时，对比文件读写前后的pgcount变化。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| 编译警告：kalloc和kfree隐式声明 | 在vm.c中添加`#include "kernel/kalloc.h"` |

| make clean后make失败（undefined reference to 'main'） | 修复Makefile依赖链，确保vectors.S等生成文件正确重建 |
