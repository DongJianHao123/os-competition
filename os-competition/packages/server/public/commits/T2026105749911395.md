#### 3.1.1 详细模块介绍

**dm crate（核心库）**：项目核心为平台无关的 Rust 库，定义统一的 `BlockDevice` trait，包含 `read_block` 和 `write_block` 方法。`DmCryptDevice` 实现 AES-256-XTS 透明加解密，支持 Linux dm-crypt 表格式（aes-xts-plain64），含 IV offset 和 data offset 参数。`DmVerityDevice` 实现只读完整性校验，使用 SHA-256 Merkle 树验证每个读取块。`HashTree` 实现标准二叉树 Merkle 树构建与验证。`sha256` 为纯 Rust 实现，无外部依赖。`crypto` 模块包含 DemoStreamCipher（用于最小演示）和 Aes256XtsCipher（Linux 兼容实现，含 GF(2^128) 乘法）。

**dmctl（命令行工具）**：提供 12 个子命令，包括 init-image、crypt-write/crypt-read、raw-read-hex、crypt-table-fill、verity-root/verity-verify/verity-format-dm、tamper、demo-crypt、demo-verity、benchmark。demo-crypt 和 demo-verity 为完整的端到端演示。

**asterinas_port（内核适配层）**：包含 VirtioBlkAdapter 骨架（需填充实际 Asterinas 块 I/O 调用）、cmdline_skeleton（解析 dm_crypt.* 和 rootfs_verity.* 启动参数）、以及三个内核集成补丁。补丁 0001 将 aster-dm 组件引入 Asterinas 构建系统；补丁 0002 修复 BIO DMA 路径并添加 ext2 魔数自检；补丁 0003 实现 dm-root 模式，通过 pivot_root 切换到受保护根文件系统。

**文档体系**：包含 12 个 Markdown 文档（设计、集成、需求映射、性能、测试、合规、根文件系统保护、cryptsetup 兼容性、答辩大纲、环境状态、AI 工具披露、最终报告）和完整的 LaTeX 技术文档（9 个章节 + 2 个附录 + 29 张示意图）。

### 4.2 遭遇的代码问题

| --- | --- |
| Rust 所有权与 I/O 队列管理冲突 | 使用 Arc<Mutex<Vec<u8>>> 共享缓冲区，利用 try_borrow_mut 避免死锁，热路径使用 Cell/RefCell 实现内部可变性 |
