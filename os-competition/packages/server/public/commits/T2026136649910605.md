#### 3.1.1 详细模块介绍
**QEMU ACT 推理模块**：该模块实现了在 QEMU 模拟的 StarryOS 环境下，使用 ONNX Runtime 的 CPU 后端运行 ACT 模型推理。核心代码 `act_infer.c` 读取图像、状态和潜在变量三个输入，输出 8 个时间步的动作（左右轮速度 + 夹爪值），并通过后处理将归一化输出映射到物理速度范围。配套 Python 脚本用于导出 ONNX 模型、冻结参考权重、准备输入数据。该模块验证了 ACT 模型在 StarryOS 上的基本推理可行性。

**SG2002 BF16 TPU 推理模块**：该模块将 ACT 模型部署到 SG2002 芯片的 TPU 上。通过 TPU-MLIR 工具链将 ONNX 模型转换为 BF16 cvimodel，并在 StarryOS 上利用 TPU 硬件加速推理。关键创新包括：使用 ION carveout 内存分配器管理 TPU 所需的大块连续物理内存（105 MiB），通过内核补丁实现 TPU 设备驱动、ION 设备、mmap 支持等。优化后单次推理约 84ms。

**SG2002 混合 INT8/BF16 量化模块**：在 BF16 推理基础上，通过混合量化搜索找到最优的 INT8/BF16 层分配方案，将部分层量化为 INT8 以减少模型大小和推理延迟。混合量化后模型大小从 ~96 MB 降至 ~85 MB，推理速度从 ~84ms 提升至 ~68ms（约 14.7 FPS），且输出余弦相似度接近 1.0，精度损失极小。

**StarryOS TPU/ION 内核适配模块**：这是项目的核心技术贡献。补丁 `sg2002-starryos-tpu-ion-carveout.patch`（678 行）对 StarryOS 内核进行了多处修改：实现 ION carveout 堆分配器（物理地址 0x8950_0000，大小 105 MiB）；添加 TPU 和 ION 设备注册；修改内存映射以支持大页填充；调整文件读取以支持大块数据；预留 carveout 内存区域等。这些修改使 StarryOS 能够支持 TPU 推理所需的内存管理和设备交互。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| TPU 推理需要大块连续物理内存 | 实现 ION carveout 堆分配器，预留 105 MiB 连续内存区域 |

| BF16 推理延迟较高（~84ms） | 混合量化优化后降至 ~68ms |
