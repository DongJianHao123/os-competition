#### 3.1.1 详细模块介绍
**VFS虚拟文件系统层**：项目在xv6原有文件系统基础上，新增了薄VFS分发层。通过定义`struct inode_ops`操作函数表（包含read、write、stat、trunc、lookup、create、unlink、link等函数指针）和`struct fs_ops`文件系统操作表，实现了对不同文件系统后端的统一调用。VFS维护一个挂载表（最多8个挂载点），支持路径解析时跨挂载点边界（如".."穿越）。默认后端为xv6原生文件系统，ext2和FAT32通过`ext2_get_fs_ops`和`fat32_get_fs_ops`注册。代码约6165行（含注释），结构清晰，但xv6后端的create和unlink操作未实现（设为0指针）。

**ext2文件系统**：实现了完整的ext2最小读写闭环。定义了ext2磁盘数据结构（超级块、块组描述符、inode、目录项），实现了inode读写、块/inode分配释放、块映射（12直接块+1一级间接块）、目录查找、目录项增删、文件/目录创建、删除、硬链接、读写、stat、截断和挂载等操作。采用单block group设计，仅支持常规文件和目录。代码约23095行，功能完整但缺少多block group、符号链接、日志等高级特性。

**FAT32文件系统**：实现了FAT32基础功能，包括BPB解析、簇链管理、目录项操作、8.3文件名转换、inode管理、挂载和核心VFS操作。支持文件读写创建删除和目录操作，硬链接返回-1（不支持）。代码约20505行，基础功能完整但缺少长文件名（VFAT/LFN）支持。

**HAL硬件抽象层**：提供6个抽象接口：hal_intr_init、hal_intr_inithart、hal_irq_claim、hal_irq_complete、hal_disk_init、hal_disk_rw。RISC-V平台调用PLIC和virtio-mmio，LoongArch64平台调用APIC/EXTIOI和virtio-pci。代码仅510行，接口简洁但功能覆盖基本需求。

**LoongArch64架构移植**：在arch/loongarch/目录下包含18个文件，涵盖启动代码（entry.S、start.c）、内存管理（vm.c）、中断处理（trap.c、apic.c、extioi.c）、TLB重填（tlbrefill.S）、上下文切换（swtch.S）、virtio-pci驱动（virtio_pci.c）等。参考了xv6-loongarch-exp项目并进行裁剪适配。
