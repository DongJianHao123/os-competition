#### 3.1.1 详细模块介绍
KVHC 项目采用分层架构设计，从上到下依次为：API 层（统一入口 KVCacheManager）→ 框架适配器层（vLLM/HuggingFace/TRT-LLM）→ 核心逻辑层（Predictor/Scheduler/Runtime Coordinator）→ 存储层（GPU/CPU/SSD 三层）→ HAL 硬件抽象层（6 种后端）。各模块通过标准接口解耦，支持优雅降级和错误容错。

API 模块提供了完整的公共接口和数据类型，包括 BlockID、Tier 枚举、BlockFeatures（22 维特征）、MigrationDecision 等。HAL 模块通过工厂模式创建对应硬件后端，NVIDIA 后端实现最完整（11229 字节），国产硬件后端（昇腾、寒武纪）为 Alpha 质量。Predictor 模块实现了四种预测策略：MLP（PyTorch 3 层网络，含在线学习）、RuleBased（5 状态有限状态机）、Reactive（无状态）和 Composite（加权集成）。Scheduler 模块使用 Utility 驱动调度公式，支持带宽预算、防抖动、传输合并等生产级特性。Storage 模块实现了 GPU→CPU(NUMA)→SSD 三层存储，含迁移账本和优雅降级机制。
