#### 3.1.1 详细模块介绍
FrameVM 项目基于 Asterinas 开源项目（https://github.com/asterinas/asterinas）的 framevm 分支进行二次开发。核心组件 FrameVisor 运行在 OSTD 可信底座之上，向下统一管理 CPU、内存、I/O、中断和 DMA 等底层资源，向上为多个 non-root kernel 实例提供类型安全的虚拟化接口。与传统 Hypervisor 不同，FrameVisor 不模拟完整虚拟硬件，也不依赖频繁的 Trap-and-Emulate 路径，而是将资源抽象为更高级、更安全的软件接口。每个 FrameVM 实例是一个受限的客户内核，只能通过 FrameVisor 的受控 API 访问硬件。隔离边界从"硬件边界"转变为"由语言和 FrameKernel 定义的软件边界"。FrameVsock 利用单一地址空间和 RRef 所有权转移，将物理数据拷贝转换为对象所有权转移，实现更高效的跨域通信。
