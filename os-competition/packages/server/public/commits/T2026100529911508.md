#### 3.1.1 详细模块介绍
**PDDL时序规划建模**：项目从基础PDDL场景开始（v1.0），构建了双机器人搬运任务模型，包含move、pickup-box、putdown-box三个durative action，使用location和package类型。随后演进到v1.1，引入corridor类型实现走廊资源互斥，增加robot-free和box-free谓词实现锁定机制，新增move-through-corridor和move-with-box-corridor动作，使模型支持多机器人时序规划和并发动作执行。v1.1的改进解决了v1.0中缺少资源互斥的问题，但仍有部分动作可能被规划器重复执行、缺少机器人间位置冲突检测、箱子状态建模不够严格等不足。

**Python并行调度求解器**：实现了基于Dijkstra最短路径的并行机器人任务调度。核心功能包括：build_graph将JSON地图转为邻接表，dijkstra以travel_time为权重计算最短路径，generate_plan为每个机器人维护独立时间线生成move→pickup→move→drop动作序列，最终输出含时间戳的JSON计划。代码结构清晰，但存在拼写错误，且assign_tasks函数未被generate_plan实际调用（后者内联了类似逻辑），路径中的resource字段未用于调度决策，缺少多机器人冲突避免机制。
