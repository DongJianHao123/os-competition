#### 3.1.1 详细模块介绍

**st-core** 是整个系统的数据模型基石，定义了从原始事件（RawEvent）到上下文事件（ContextualEvent）再到语义事件（SemanticEvent）的完整事件层级。EventStore 提供因果链查询（CausalChainQuery）和事件导出能力。OSProfile trait 定义了 OS 语义恢复的接口，ProbeBackend trait 定义了探针后端的统一抽象。BreakpointGroup 支持断点分组管理，ExecutionDomain 处理地址空间切换。所有设计均遵循文档中定义的架构边界。

**st-dapd** 是系统的调试适配器核心，实现了 DAP（Debug Adapter Protocol）的 stdio 服务器。它包含完整的请求处理框架（requests.rs，23099 字节）、自定义命令系统（custom_commands.rs，20658 字节）、probe 管理（probe_manager.rs，6016 字节；probe_commands.rs，29755 字节）、trace 状态管理（trace_state.rs，39776 字节）、事件摄取（event_ingestor.rs）、函数调用追踪（function_call_tracker.rs）、域管理（domain_manager.rs）和渲染（render.rs）。GDB/MI 后端（backend.rs，23649 字节）实现了 GDB/MI 协议的解析和交互。guest_sidecar（19577 字节）处理内核侧主动上报的流式事件。linux_ebpf（15935 字节）实现了 eBPF 后端的集成。

**st-profile** 实现了 OS 语义解释层，支持 rCore（mod.rs 38937 字节，guest_trace.rs 8511 字节，object_decoder.rs 10721 字节）、CaelestisOS（mod.rs 23107 字节）、ArceOS（mod.rs 3181 字节）和 StarryOS（mod.rs 2682 字节）。profile.rs（42943 字节）是核心的 BuiltInProfile 实现。builders 模块提供 interrupt、IPC、scheduler 等语义透镜的构建函数。
