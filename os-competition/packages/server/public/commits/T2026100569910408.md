#### 3.1.1 详细模块介绍

**HAL（硬件抽象层）**：项目核心创新点，为 xv6-riscv 引入统一的硬件抽象层，支持 RISC-V 和 LoongArch 两种架构。HAL 接口设计包括 hal/hal.h 统一入口，以及 hal_arch.h（架构相关）、hal_intr.h（中断）、hal_timer.h（定时器）、hal_vm.h（虚拟内存）、hal_memlayout.h（内存布局）、hal_console.h（控制台）、hal_ctx.h（上下文）等子系统头文件。关键技术对照表详细列出了 RISC-V 与 LoongArch 在关中断、获取核心ID、设置异常向量、页表基址、TLB刷新、定时器、中断控制器、特权级、页表格式等方面的差异。LoongArch 采用软件 TLB 重填机制（TLB miss 时触发 TLB 重填异常，由内核软件完成 walk 和 TLB 填入），与 RISC-V 硬件自动 walk 不同。设计决策包括 PTE 格式、CSR 访问封装、多核启动、scause 兼容、上下文结构体条件编译、TLB 策略（先用 DMW）、磁盘策略（先用 ramdisk）、Docker 隔离等。

**VFS（虚拟文件系统层）**：实现统一的 VFS 接口，定义了 vnode（虚拟节点）、vnode_ops（节点操作表）、vfs_ops（文件系统操作表）、mount（挂载实例）等核心数据结构。vfs_init 初始化 vnode 表并注册 xv6fs、ext2、fat32 三种文件系统。vfs_mount 实现挂载流程（检查重复、调用 mountfn、解析挂载点、加入 mounttable），vfs_namei 实现路径解析（支持挂载点穿越和 '..' 从挂载根返回）。所有文件操作（open/read/write/close/readdir/create/unlink/mkdir/link/mknod/truncate）均委托给 vnode 操作函数，支持跨文件系统操作（硬链接禁止跨 FS）。

**ext2 文件系统**：完整的 ext2 VFS 后端实现，约 30101 行代码。实现了 ext2_mount_priv（挂载私有数据，含超级块、块组描述符）和 ext2_vnode_priv（vnode 私有数据，含磁盘 inode）数据结构。关键函数包括：ext2_bmap（逻辑块到物理块映射，支持 12 个直接块 + 256 个单级间接块）、ext2_ialloc/ext2_ifree（inode 分配/释放）、ext2_balloc/ext2_bfree（块分配/释放）、ext2_dir_add/ext2_dir_lookup（目录项管理）。VFS 操作表 ext2_vnops 包含 lookup/read/write/stat/readdir/create/unlink/mkdir/link/mknod/read_kernel/truncate/destroy 共 13 个操作。

**FAT32 文件系统**：FAT32 的 VFS 后端实现（约 13027 行），支持基本的文件系统操作（mount, read, write, create, mkdir, ls）。有独立的 fat32test 测试程序。由于 LoongArch 的 ROM 大小限制（kernel.bin 4.26MB > ~4MB），FAT32 仅支持 RISC-V 架构。

**LoongArch 移植**：将 xv6-riscv 移植到 LoongArch 架构，涉及架构核心差异处理：页表格式（PPN 分两段存储）、软件 TLB 重填机制、中断控制器（EIOINTC）适配、CSR 映射（CRMD↔sstatus, ERA↔sepc 等）。移植分周进行：第4周环境搭建，第5周CPU+内存+串口，第6周中断+定时器+调度，第7周磁盘+文件系统+测例。最终状态：RISC-V 支持 xv6fs+ext2+fat32，LoongArch 支持 xv6fs+ext2。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| LoongArch 下 ext2 的 L1 两个测试失败 | 修复 VFS→HAL 合并后 bio.c bread() 函数缺少设备号验证的问题 |
| FAT32 在 LoongArch 上无法运行 | 因 kernel.bin 4.26MB 超过 ROM 大小限制（~4MB），保持 RISC-V only |
| LoongArch 软件 TLB 重填机制 | 与 RISC-V 硬件自动 walk 不同，需实现软件 TLB refill handler |
| 中断控制器差异（PLIC vs EIOINTC） | 通过 HAL 接口抽象，分别实现 hal_plic.c 和 LoongArch 中断控制器适配 |
| 页表格式差异（PPN 分两段存储） | 重写 PA2PTE/PTE2PA 宏 |
| PCI virtio 替代 MMIO | LoongArch QEMU 使用 PCI virtio，需实现 PCI 枚举 |
| QEMU -kernel 可能不工作 | 需使用 -bios 或其他启动方式 |
| -mcmodel 差异 | LoongArch 需使用 -mcmodel=medium |
| csrxchg 语义差异 | 需理解 LoongArch csrxchg 指令的原子读写语义 |
