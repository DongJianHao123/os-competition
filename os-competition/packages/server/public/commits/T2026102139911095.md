#### 3.1.1 详细模块介绍

**mobile_use核心模块**：项目基于Mobile-Agent-v3.5架构，团队进行了大量二次开发。主循环`run_gui_owl_1_5_for_mobile.py`实现了完整的agent闭环：接收自然语言指令→任务规划→UI感知→动作决策→设备执行→异常自愈。关键创新包括：Hybrid Loop（VLM+UI Tree Fast Path双通路）、模型职责拆分（图像模型qwen3.6-plus负责截图理解，文本模型deepseek-v4-pro负责历史压缩和任务解释）、中文场景增强（ADB Keyboard整体输入、App直启机制）、保守执行策略（高风险操作禁止快速路径）。

**assistant_apk桥接模块**：团队自研的Android端侧桥接APK，通过AccessibilityService在手机上运行HTTP服务（127.0.0.1:8765），提供`/tree`（获取UI树）、`/action`（执行点击/滑动/输入等动作）等接口。相比传统`uiautomator dump`方案，UI树获取速度提升约52.7倍（41.9ms vs 2206.1ms），大幅降低操作时延。

**benchmarks评测模块**：团队构建了完整的评测体系，包含smoke集（快速验证）、公开30任务集、competition 30任务集（覆盖应用内操作13题、跨应用协作8题、干扰项测试9题）。批量运行脚本支持任务过滤、流式输出、自动生成JSON/Markdown/CSV格式报告。
