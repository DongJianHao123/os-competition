#### 3.1.1 详细模块介绍

**act-starryos-deploy 模块**：该模块实现了 ACT (Action Chunking Transformer) 模型在 StarryOS/QEMU RISC-V 环境中的完整推理部署。开发过程包括：(1) 使用 PyTorch 训练 ACT 模型并导出为 ONNX 格式（约 193.26 MiB）；(2) 编写 Python 预处理脚本将输入图像和状态转换为 float32 二进制文件；(3) 交叉编译 ONNX Runtime 为 RISC-V 64 动态库（耗时较长，WSL2 跑崩一次）；(4) 编写 C 语言推理程序 `act_onnx_c.c`，使用 ONNX Runtime C API 加载模型、执行推理、反归一化输出；(5) 为 StarryOS 增加 `/proc` 内存统计补丁；(6) 将可执行文件和运行时库写入 StarryOS rootfs；(7) 在 QEMU/StarryOS 中执行推理并统计时间、内存、可执行文件大小。两组实验（frame_000227.jpg 和 frame_000001.jpg）均成功输出动作预测，推理耗时约 16-18 秒，VmHWM 约 218 MB。

**act-starryos-rk3588 模块**：该模块实现了 ACT 模型在 RK3588 + StarryOS 实际板端环境的 NPU 推理部署。开发过程包括：(1) 使用 RKNN-Toolkit2 将 ONNX 模型转换为 RKNN 格式，生成 FP16 基线模型（96.95 MiB）和混合量化模型（50.23 MiB）；(2) 编写 C++ RKNN Runtime 推理程序 `main.cpp`，支持加载 RKNN 模型、预处理输入、NPU 推理、反归一化输出；(3) 编写板端 benchmark 脚本 `benchmark_final_10_images.sh`，对 10 张测试图片分别运行两种模型，收集 NPU 延迟、峰值 RSS 等指标；(4) 实验结果量化对比：混合量化模型大小减少 48.19%，平均 NPU 推理时间基本一致（33.758 ms vs 33.607 ms），模型加载时间快 10.37%，方向输出完全一致。
