#### 3.1.1 详细模块介绍
项目采用前后端分离架构。后端基于FastAPI构建REST API，核心RAG引擎（rag_kernel_patch.py）实现了从宕机日志中提取crash特征（类型、调用栈、子系统、BUG位置），通过DashScope text-embedding-v3进行向量化，在ChromaDB中检索相似补丁，再使用qwen-max大模型生成诊断报告。索引构建脚本（index_commits.py）支持从多种格式（git show/format-patch, git log, LKML email等）解析补丁元数据并批量入库。前端使用Streamlit框架，提供双栏交互界面：左侧输入宕机日志（支持demo选择、文件上传、文本粘贴），右侧展示诊断结果（crash特征、AI报告、匹配补丁列表及评分）。前端还包含本地兜底诊断模块（rag_core.py），在后端不可用时基于关键词匹配提供同构结果。知识库数据经过格式转换脚本（convert_to_oso_format.py）将前端采集的758条数据与后端20条合并为449条最终知识库。
