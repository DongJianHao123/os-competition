#### 3.1.1 详细模块介绍
本项目基于 MIT xv6-riscv rev5，目标是增加虚拟文件系统（VFS）和硬件抽象层（HAL），同时支持 xv6fs、ext2、FAT32 三种文件系统以及 RISC-V 和 LoongArch 两个平台。

**VFS 核心**：采用最小调度层设计，通过 `vfs_filesystem` 注册文件系统实现，`vfs_inode_ops` 操作表分发 inode 操作，`vfs_mount` 记录挂载点关系。路径解析在挂载点处切换到目标文件系统根 inode，`..` 在挂载点根目录返回宿主侧父目录。设计原则是保持 xv6fs 稳定，不支持的功能稳定返回失败。

**ext2 后端**：支持 1024 字节块大小、128 字节 inode 记录、直接块和单间接块、目录项操作、文件创建/删除/读写、块/inode bitmap 回收。不支持硬链接、非空目录删除、日志。

**FAT32 后端**：支持 512 字节扇区、每簇一扇区、短 8.3 文件名、根目录和子目录、文件创建/覆盖/读写、目录创建、删除、簇链回收、FSInfo 空闲计数维护。不支持长文件名、硬链接、非空目录删除。

**HAL 抽象层**：将平台差异收敛到 `kernel/hal/` 和 `kernel/loongarch/` 中。HAL 接口涵盖启动初始化、定时器、中断、上下文切换、CSR 访问、设备地址查询等。RISC-V 和 LoongArch 通过 `ARCH=riscv` 或 `ARCH=loongarch` 编译选择。

**LoongArch 平台**：使用 `ertn` 替代 `sret`，`invtlb` 替代 `sfence.vma`，APIC/EXTIOI 替代 PLIC。由于 LoongArch TLB 缺页要求页表层级预先存在，实现了 `uvmreserve()` 预分配页表层级。块设备使用编译时嵌入的磁盘镜像（ramdisk），而非 VirtIO。
