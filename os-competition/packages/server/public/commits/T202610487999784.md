#### 3.1.1 详细模块介绍

**wasi_sophon 插件**：团队自主开发的 Sophon BM1688 平台 WasmEdge 插件。核心文件 `yolov5_module.cpp` 注册为 `wasi_nexus_module` 模块名，提供 `load`（接收 JSON 配置，含模型路径、设备ID、阈值）和 `detect`（接收图像字节流，返回 JSON 格式检测结果）两个宿主函数。插件封装了 sophon-mw SDK 的 BMCV/BMRT 接口，通过 `YoloV8_det` 类完成图像解码、预处理、硬件推理和后处理。构建使用 CMake 的 `wasmedge_add_library` 宏。该插件在 Git 历史中由 Neoidea 在 6月22日-28日期间通过多次提交和合并完成。

**wasi_rknn 插件**：由 ss-Zhong 在 6月9日首次提交（初始提交 cadaa06），是项目最早的代码。核心文件 `rknn_yolo_module.cpp` 同样注册为 `wasi_nexus_module`，提供 load/detect 接口。detect 函数支持三种输入模式：JSON 字符串（含 image 字段）、文件路径字符串、原始图像字节（自动写入临时文件）。后处理使用 rknn_model_zoo 的 post_process 函数。插件还包含 rknpu2/ 目录下的三个 YOLO 实现变体（标准 yolov8.cc、RV1106/1103 适配、零拷贝版本），体现了一定的代码演进。

**wasi_rdk 插件**：由 Neoidea 在 6月27日-28日期间开发。核心文件 `rdk_yolo_module.cpp`（519行）是三个插件中代码量最大的，实现了完整的 hbDNN 推理流程：模型初始化、输入输出张量管理、letterbox/resize 预处理、异步推理、动态输出头解析、NMS 后处理。支持可配置的预处理模式、最大检测数和类别数。

**Nexus 上层调用示例**：`nexus/wasm/sophon_detect.cpp` 展示了 WASM guest 如何通过 `import_module("wasi_nexus_module")` 调用宿主插件的 load/detect 函数。`nexus/host/cpp/WasmEdge_Sophon.cpp` 展示了 C++ host 如何加载插件并运行 WASM 模块。`nexus/host/rust/` 提供了 Rust 版本的 host 示例。

**前端演示控制台**：`frontend/` 目录包含完整的单页应用。`server.py`（354行）作为代理服务器，通过 SSH 连接到远程边缘设备，提供 REST API 管理 WasmEdge 推理服务。`app.js`（279行）实现平台选择、运行时管理、图像预览、模型加载、检测和结果可视化（Canvas 绘制边界框）。支持 Sophon、RK3588、RDK 三个平台切换。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| WASM 线性内存指针段错误 | 使用 WasmEdge 的 getSpan 安全访问内存，确保指针有效性 |
| 交叉编译链接失败 | 调整 CMakeLists.txt 链接顺序和库依赖 |
| OpenCV 解码空 Mat | 检查图像数据完整性，添加错误处理 |
| tokio 多线程不兼容 | 使用单线程 tokio runtime 运行 Axum HTTP 服务 |
| RKNN 不接受内存字节 | 将图像字节写入临时文件，传递文件路径给 RKNN API |
| RDK 输出头数量不固定 | 动态枚举输出张量，根据实际输出头数量解析 |
| 插件模块名冲突 | 统一使用 wasi_nexus_module 模块名，避免与现有插件冲突 |
