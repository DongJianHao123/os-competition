#### 3.1.1 详细模块介绍
HotMigrate 是一个面向长上下文大模型推理的异构 KV Cache 管理系统，将 vLLM 的 GPU 本地前缀缓存扩展为 GPU→CPU→SSD 三层可恢复缓存。系统核心设计理念是"转移感知"（transition-aware）：通过建模块级热度的转移趋势（而非静态热集成员），在层级间主动驱动 KV Cache 迁移。

**存储层（storage/）**：实现了完整的 PagedKV 层级存储架构。GPU 层由 vLLM PagedAttention 管理，CPU 层使用固定容量 pinned byte pool 按 KV block 对齐分配，SSD 层使用 segment log + sidecar index 实现持久化。包含异步 DMA 引擎（支持双缓冲近似，提升优先级高于降级）、块表管理、密封块迁移语义、层级分配器等核心组件。

**运行时（runtime/）**：包含异步缓存协调器（async_cache），负责缓存策略、热度追踪、驱逐决策、SSD 策略、传输管理等核心运行时逻辑。还包含控制器（controller）、意图缓存（intent_cache）、意图规划器（intent_planner）、KV 目录（kv_catalog）、层策略（layer_policy）、迁移价值（migration_value）、查询价值（query_value）、服务（serving）等模块。

**预测器（predictor/）**：实现了转移感知的预测流水线。主预测器为线性或 2 层 MLP（隐藏层 32/64 单元），输出 4 类 softmax（PROMOTE/KEEP_HOT/DEMOTE/KEEP_COLD）+ 回归分数 + 置信度。回退预测器为基于规则的状态机。包含完整的训练流水线、离线评估、泛化分析、运行时快照和诊断工具。

**调度器（scheduler/）**：效用驱动迁移调度器，效用函数 U(b) = benefit(b) - λ·cost(b) - μ·risk(b)。包含防抖动守卫（最小驻留时间、双阈值滞后、提升冷却、迁移惩罚）和迁移合并器（相邻同向块合并为单个 DMA 操作）。

**集成层（integration/）**：与 vLLM V1 引擎深度集成，包含异步缓存连接器（async_cache_connector）、vLLM 适配器（vllm_adapter）、桥接层（vllm_bridge）、协调器（vllm_hma_coordinator）、请求门控（vllm_request_gating）、物理状态管理（vllm_physical_state）等。集成点覆盖 vLLM V1 scheduler、worker、model runner 和 benchmark metadata。

**评估框架（evaluation/）**：完整的实验评估框架，支持性能比较（perf_compare）、上下文扫描（context_sweep）、执行管理（execution）、日志分析（logs）、SSD 压力测试（ssd_pressure_gate）、结果聚合（results）等。
