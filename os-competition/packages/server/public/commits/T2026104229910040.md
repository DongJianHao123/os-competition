#### 3.1.1 详细模块介绍
项目基于 Microsoft DiskANN 的 Vamana 图算法和 SSD 索引，使用 Rust 语言实现了多策略读写优化的近似最近邻向量检索系统。核心创新包括：Page Shuffle（基于 Starling 论文的 BNP/BNF/BNS 算法）优化磁盘布局、Page Search 实现块级顶点扩展、LSM-Tree 风格的动态增量更新（DeltaStore + DeleteSet + 后台合并线程）、两级缓存（BFS 邻域缓存 + K-Means 预热缓存）。项目还实现了与 Milvus/Knowhere 的完整集成（C FFI + C++ 适配器 + CMake 构建脚本），以及 Garnet 远程缓存服务的向量集支持。量化模块支持球形量化（RaBitQ）和标量量化，覆盖 L2、内积、余弦三种距离度量。

### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| INRIA texmex 数据集 ground truth ID 为 1-based，DiskANN 内部使用 0-based，导致召回率从 ~0.99 降至 ~0.003 | 在 convert_data.py 中添加 zero_based 参数，将 ID 减 1 转换；在 benchmarks.rs 中添加缓存过期检查 |
| BNF/BNS 迭代进度输出使用 \r 覆盖导致日志混乱 | 改用 eprintln 每行独立输出，并移除 flush 调用 |
| tracing subscriber 输出干扰 eprint 进度条 | 将 tracing 输出重定向到 stdout |
| BNS 进度输出被 progress_enabled() 守卫限制 | 移除 progress_enabled() 守卫，始终显示 OR(G) 输出 |
