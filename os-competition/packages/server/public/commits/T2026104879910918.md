#### 3.1.1 详细模块介绍
项目基于GVirtuS框架，采用前端-后端分离架构。前端运行在无GPU的客户机中，通过提供与CUDA Runtime ABI兼容的替代动态库透明截获CUDA调用，将参数编码后通过RPC协议转发到后端；后端运行在有NVIDIA GPU的宿主机中，接收RPC请求后调用真实CUDA API执行，并将结果返回前端。系统支持多种通信方式（TCP/RDMA/共享内存/VMCI/Virtio/ZMQ等），默认使用TCP。并发模型采用fork-per-client，每个前端连接由独立子进程处理，实现进程级CUDA会话隔离。项目还集成了llamacpp补丁，使llama.cpp能够在无GPU环境下通过GVirtuS远程调用GPU进行大模型推理。
