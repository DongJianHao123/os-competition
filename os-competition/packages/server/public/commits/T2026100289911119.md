### 4.2 遭遇的代码问题
| 问题描述 | 解决方法 |
| --- | --- |
| AI 对加锁顺序不适用计数型 semaphore 产生幻觉 | 用 RISC-V Reader 和原仓库代码纠正 |
| AI 认为 Work=sem_total 导致假阳性 | 修正为 sem_available = total - hold_sum |
| Condvar 单返设计不符合组件契约 | 改为双返 (bool, Option<ThreadId>) |
| 死锁检测从弱一次遍历 stub 强化 | 实现真 DFS 等待图，pending_tid 入 tids |
| tg-sync 单体 crate 不利于独立测试 | 拆分为 tg-mutex/tg-sem/tg-condvar 三个独立 crate |
