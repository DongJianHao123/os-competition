#### 3.1.1 详细模块介绍
项目基于 NexOS 教学内核（RISC-V + QEMU virt），在其上扩展了 Agent-OS 赛题全部六个任务的核心功能。Agent 进程模型将 AI Agent 视为操作系统一等公民，扩展了 PCB 结构（agent.h），支持 Agent 类型标识、心跳间隔、资源配额、循环状态和 Context 区管理。结构化工具调用（agent_tool.h）定义了定长协议，内核提供 7 种内置工具，支持参数传递和结果写入 Context。Context Path 在用户地址空间维护环形缓冲区，内核仅管理元数据，避免频繁陷入内核。Agent 友好文件系统在 inode 上叠加扩展属性和摘要，通过哈希索引加速查询。Agent Loop 采用事件驱动模型，支持心跳、消息、文件三种唤醒源，调度器按优先级选择进程运行。多 Agent 投票演示展示了长期记忆、消息通信、Context 复用和文件属性查询等综合能力。
