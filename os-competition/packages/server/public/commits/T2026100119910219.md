#### 3.1.1 详细模块介绍

**任务1：OpenHarmony 系统移植**：团队在 Ubuntu 24.04 x86_64 服务器上搭建编译环境（Intel Xeon 6核12线程，22GB内存，295GB硬盘），下载 OpenHarmony 5.0.3 Release 源码并进行交叉编译。首次全量编译约3-4小时，编译高峰内存使用12-16GB，链接阶段峰值16-18GB，磁盘总占用约156GB。成功烧录镜像后，实现了图形桌面显示、HDMI输出、USB鼠标键盘输入、HDC调试连接、GPU加速及ALSA音频功能。

**任务2：CANN 推理引擎移植**：通过 HDC 工具连接启智01开发板，依次完成环境初始化（initenv.tar.gz）、Python 3.10 安装、CANN Toolkit 9个子包安装（解决 compiler 子包 pip3 报错和 y/n 交互卡住问题）、NNAL 加速库安装、CANN Kernels 安装、推理依赖 Python 包离线安装（torch-2.1.0, torch_npu, torchvision 等），最终验证 torch.npu.is_available() 返回 True。

**任务3：YOLOv5 目标检测推理**：传送推理样例（yolo.om 模型、world_cup.jpg 测试图片、main.py 推理脚本），修改 main.py 删除 Jupyter 依赖，使用 InferSession 加载 OM 模型进行 NPU 推理。安装 opencv-python-headless 和 requests 等依赖，最终成功检测到3个对象（person, person, sports_ball）。后续开发了 OpenHarmony APP（ArkTS）和 HTTP 推理服务（yolo_infer_server.py），实现实时摄像头检测，端到端性能约90ms/帧（约11 FPS）。

**任务4：系统性能优化**：通过禁用10个非必要 SA 服务（蓝牙、电话、投屏、分布式音频/摄像头/屏幕、智能语音、云文件、备份），启动时间从50s降至43s（改善14%），内存占用从1700MB降至1529MB（改善10.1%），用户进程从91个减至86个，优化措施可恢复。

**拓展任务**：完成 USB 摄像头适配（解决镜像问题）、大蒜三合一检测应用、舵机分拣联动控制。
