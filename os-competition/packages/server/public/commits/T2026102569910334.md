#### 3.1.1 详细模块介绍
**虚拟文件系统 (VFS)**：项目核心创新点。采用分层设计：`struct file → struct vnode → vnode_operations（函数指针表）`。每种文件系统实现自己的 vnode 操作函数表（xv6fs_vnodeops、ext2_vnodeops、fat32_vnodeops），底层对应各自的 inode 结构。VFS 层管理全局 vnode 池（100项）和挂载表（NMOUNT项），支持路径解析时自动跨越挂载点边界。代码约378行，使用自旋锁保护全局表，睡眠锁保护 vnode。

**ext2 驱动**：实现了 ext2 文件系统的基本操作，包括超级块读取、块组描述符加载、inode/block 位图管理、目录项线性查找、文件读写（通过 ext2_bmap 映射逻辑块到物理块）。支持直接块（12个）和一级间接块，二级/三级间接块仅存桩。目录创建时支持块分裂（但代码中部分逻辑被注释）。unlink 和 truncate 操作未释放底层块资源。

**FAT32 驱动**：实现了 FAT32 文件系统的挂载初始化、FAT 链管理（查找空闲簇、扩展链）、目录项解析（仅支持短文件名 8.3 格式）、文件读写。使用 512 字节扇区与 1024 字节 BSIZE 之间的映射。不支持长文件名（LFN），不支持 mkdir/rmdir。

**LoongArch HAL**：为龙芯架构新增了完整的引导、启动、上下文切换、陷阱处理代码。包括：boot/la/entry.S（DMW 窗口设置、TLB 重填、栈初始化）、kernel/la/start.c（UART 初始化、定时器初始化、陷阱初始化）、kernel/la/swtch.S（寄存器保存/恢复）、kernel/trap/la/kernelvec.S（内核陷阱向量）、kernel/trap/la/trap.c（陷阱分发：中断、系统调用、缺页、非法指令、断点）、kernel/loongarch.h（CSR 定义、PTE 宏、陷阱帧结构）。

**多磁盘 virtio 驱动**：将原有单磁盘 virtio 驱动重构为支持多磁盘（NUM_DISKS=3），通过 disk_id_for_dev 函数将设备号映射到磁盘 ID。支持 VIRTIO0（xv6fs）、VIRTIO1（ext2）、VIRTIO2（FAT32）三个设备。
