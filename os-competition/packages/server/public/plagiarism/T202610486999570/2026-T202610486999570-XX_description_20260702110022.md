## 第四部分：分模块架构说明

### MM（内存管理）

底层证据较强。关键锚点：

- `arch/src/loongarch64/page_table.rs`：LoongArch64 页表操作。
- `arch/src/loongarch64/trap.rs#loongarch64_trap_handler`：处理 `LoadPageFault`、`StorePageFault`、`IllegalInstruction` 等，并调用 `emulate_load_store_insn`。
- `os/src/mm/memory_set.rs`：地址空间管理，`MemorySet::from_elf` / `from_elf_with_interp` 用于 ELF 加载。
- `os/src/syscall/process.rs#sys_sbrk`：通过 `memory_set.append_heap_to` / `shrink_heap_to` 调整堆。
- `os/src/mm/heap_allocator.rs#handle_alloc_error`：内核堆分配失败时收集帧分配器、任务数、fd 表等状态。

未体现：swap、NUMA、huge page、页回收算法等底层证据未体现。

### PROC（进程管理/任务调度）

底层证据强。关键锚点：

- `os/src/task/process.rs`：PCB、fork、exec、线程管理。
- `os/src/syscall/process.rs#sys_fork`、`sys_clone`、`sys_exec_internal`、`sys_waitpid`、`sys_waitid`、`sys_exit`。
- `os/src/syscall/thread.rs#sys_thread_create`：创建线程并 `add_task`。
- `os/src/task/mod.rs#exit_current_and_run_next`、`handle_signals`。
- `os/src/task/processor.rs#run_tasks`：调度循环、`fetch_task`、`arch::switch_to_task`。
- `os/src/task/futex.rs`、`os/src/sync/condvar.rs`：阻塞/同步原语。
- 信号：`sys_sigaction`、`sys_sigreturn`、`sys_kill`、`sys_tkill`。
- 资源/凭证：`sys_prlimit64`、`sys_capget`、`sys_getgroups`、`sys_set_priority` 等。
- procfs：`os/src/fs/vfs/procfs.rs` 提供 `/proc/[pid]/stat|status|maps|task` 等。

未体现：完整多核 SMP 调度、cgroups、namespaces、LSM 等底层证据未体现。

### FS（文件系统）

底层证据强。关键锚点：

- `os/src/syscall/fs.rs`：open/read/write/mount/link/unlink/mkdir/name_to_handle_at 等。
- `os/src/fs/vfs/procfs.rs`：procfs 目录/文件/读写。
- `os/src/fs/vfs/ext4/inode.rs`：ext4 包装，调用 `lwext4_dir_entries`、`ext4_cache_*`、`ext4_errno` 等。
- `easy-fs/`、`easy-fs-fuse/`：自定义简易文件系统及其打包工具。
- 特殊文件：`DevNull`、`DevZero`、`DevUrandom`、`MemFdFile`、pipe、`eventfd.rs`、`timerfd.rs`。

未体现：独立的 ext4 原生实现（当前为 lwext4 封装）、Btrfs/ZFS/XFS 等底层证据未体现。

### DEV（设备驱动）

底层证据较弱。关键锚点：

- `os/src/drivers/block/virtio_blk.rs`：virtio-blk 读写、重试。
- `os/src/drivers/block/cached_block_device.rs`：块缓存。
- `os/src/drivers/net/mod.rs`：virtio-net 探测、收发。
- `os/src/boards/qemu.rs#dispatch_irq`：UART/键盘/鼠标/块设备 IRQ 分发。

未体现：独立的 TCP/IP 协议栈、路由表、防火墙、netfilter 等底层证据未体现。

## 第六部分：边界与未体现内容

1. **网络协议栈**：原创网络代码仅 28 个函数，核心 TCP/IP 由第三方 smoltcp 实现；本项目应视为“smoltcp 集成 + Unix domain socket 自定义实现”，而非独立网络协议栈。
2. **ext4 文件系统**：`os/src/fs/vfs/ext4/` 为封装层，底层调用 lwext4 接口，不能视为完全原生的 ext4 实现。
3. **设备驱动**：仅见 virtio-blk、virtio-net、UART/键盘/鼠标 IRQ 分发；PCI 枚举、DMA、GPU、声卡、SATA/NVMe 等底层证据未体现。
4. **调度/异步运行时**：存在任务切换、阻塞/唤醒，但未见基于 Rust `async/await` 的完整异步运行时（waker、reactor、epoll 与 async 桥接等）底层证据。
5. **多核/SMP**：`run_tasks` 呈现单核调度循环，未见多核调度、CPU 亲和性完整实现等底层证据。
6. **高级内核机制**：cgroups、namespaces、LSM、seccomp、swap、NUMA、huge page、内核调试器、电源管理等底层证据未体现。
7. **第三方/公共依赖**：lwext4、smoltcp、xmas-elf 等应单独说明，不应夸大为主体原创内核代码。
8. **方向性判断**：本报告中“可能的系统方向”仅基于目录、函数名和调用链的保守推断，不是最终身份或原创性结论；不做抄袭/违规判断。
