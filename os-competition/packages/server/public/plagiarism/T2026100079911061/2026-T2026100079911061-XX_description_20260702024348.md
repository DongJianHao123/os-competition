## 第四部分：分模块架构说明

### MM（内存管理）

底层证据可见：
- 页表/地址空间：`kernel/src/main.rs#rust_main` 中 `AddressSpace::new`、`PTEFlags::new`、`PhysAddr`。
- 页异常处理：`kernel/src/trap.rs#page_fault_handler` 调用 `handle_current_mmap_fault`、`handle_current_stack_growth_fault`、`deliver_current_sigsegv`；`kernel/src/trap.rs#handle_cow_fault` 处理写时复制。
- 内存映射：`kernel/src/syscall/memory_ops/mapping.rs#sys_mmap` 支持 `MmapFileBacking::Ext4/Rootfs`、匿名/文件映射、映射区域替换。
- 物理页分配：`kernel/src/mm/frame.rs#pop_free_block` / `remove_free_block` 使用 `FREE_AREA_HEADS`、`FREE_NEXT`、`FREE_ORDER`，呈现 buddy-like 空闲块管理。

未体现：swap、slab 分配器、DMA 一致性管理、SMP 页表同步。

### PROC（进程管理）

底层证据强，**不是**仅轻量级任务：
- PCB/进程表：`kernel/src/task/process.rs`、`kernel/src/task/scheduler.rs` 维护 `PROCESS_LIST`。
- 创建：`kernel/src/syscall/process_ops.rs#sys_clone_common`、`sys_fork_like_clone`、`sys_shared_vm_clone`、`read_clone3_args`。
- 执行：`kernel/src/exec.rs#build_ext4_elf_staged_exec_from_image`、`map_main_elf_image`、`commit`。
- 等待/僵尸：`kernel/src/syscall/process_ops.rs#wake_wait_child_if_blocked`、`notify_thread_group_exit_if_ready`、`release_zombie_thread_group_members`、`reap_background_process_skeletons`。
- 线程组与 pidfd：`pidfd::PidFd::new`、`prepare_current_thread_group_for_exec`。
- procfs：`kernel/src/procfs.rs#render_file`、`resolve_pid_child`、`fdinfo_ino`。

### 任务调度/异步运行时

- 调度原语：`kernel/src/task/scheduler.rs#request_reschedule`、`make_ready`、`wake_futex_waiters`、`wake_signalfd_readers_if_ready`。
- 阻塞模型：大量 syscall（`sys_read`、`sys_write`、`sys_accept4`、`sys_ppoll`、`sys_futex`）使用 `BlockReason` 进入阻塞，再由事件或 timer 唤醒。
- 定时器驱动：`kernel/src/trap.rs#trap_handler` 调用 `deliver_due_legacy_itimers`、`deliver_due_posix_timers`。

未体现：完整抢占式 CFS、多核 SMP 调度、时间片轮转硬件 timer 驱动细节。

### FS（文件系统）

- 自研内存文件系统：`kernel/src/task/scratchfs.rs#rename_path`、`direct_child_entry`。
- procfs：`kernel/src/procfs.rs`。
- ext4 集成：`kernel/src/ext4.rs#lookup_path`、`path_kind_from_entry`、`walk_dir_entries`，被 exec、`getdents64` 使用。
- 挂载：`kernel/src/syscall/mount_ops.rs#sys_mount` / `sys_umount2` 支持 tmpfs、vfat、scratchfs。
- 路径/FD 抽象：`kernel/src/syscall/fs_path.rs`、`kernel/src/task/fd.rs`、`kernel/src/syscall/fd_ops.rs`、`proc_fd.rs`。

注意：`kernel/src/ext4.rs` 被单独归入“文件系统/第三方库”，应视为集成/移植代码，而非原创文件系统核心。

### DEV（设备驱动）

- VirtIO 块设备：`kernel/src/virtio_blk.rs#probe_and_init`、`read_sector_inner`（MMIO、descriptor ring、`BlkReqHeader`）。
- VirtIO 网卡：`kernel/src/virtio_net.rs#probe`、`read_info`（仅探测与 MAC 读取）。
- Console/SBI：`kernel/src/console.rs#putchar`、`kernel/src/sbi.rs#shutdown`。

设备驱动层非常薄（仅 27 函数），未见中断注册、DMA 映射、完整网卡收发路径。

### SYSCALL（系统调用）

- 统一入口：`kernel/src/syscall/dispatcher.rs#dispatch` 调用 212 个函数，覆盖 process/fs/io/socket/signal/time/poll/futex/memory/eventfd/inotify/pidfd/ioctl 等。
- 分类实现：`process_ops.rs`、`fs_ops.rs`、`io_ops.rs`、`socket_ops.rs`（含 `sockopt.rs`、`msg.rs`）、`signal.rs`、`time_ops.rs`、`poll_ops.rs`、`futex_ops.rs`、`memory_ops/mapping.rs`、`fd_ops.rs`、`proc_fd.rs`、`fs_mutation.rs`、`event_ops.rs`、`identity_ops.rs`、`cwd_ops.rs`、`device_ops.rs`。

### TRAP/IRQ（中断异常）

- 用户态陷入：`kernel/src/trap.rs#trap_handler` 统一处理 syscall 返回、timer 到期、fd wait 清理、僵尸回收、trap 分发。
- 同步异常：`kernel/src/trap.rs#user_sync_exception_handler`、`do_segfault`、`deliver_or_finish_sigsegv/sigbus/sigill/sigtrap`。
- 内核态异常：`kernel/src/trap.rs#kernel_trap_handler` 输出信息后 `sbi::shutdown`。

未体现：IRQ 控制器驱动、硬件 timer 驱动、中断向量表注册、设备中断下半部。

### NET（网络协议栈）

- Socket API：`kernel/src/syscall/socket_ops.rs#sys_connect`、`sys_accept4`、`sys_sendto`；`kernel/src/syscall/socket_ops/msg.rs#recvmsg_inet_socket`；`kernel/src/syscall/socket_ops/sockopt.rs#sys_getsockopt`。
- Inet socket 抽象：`kernel/src/task/inet_socket.rs#connect_stream`。

未体现：完整 TCP/IP 协议栈、ARP/IP/路由、网卡中断收发驱动。`virtio_net.rs` 仅探测级别。
