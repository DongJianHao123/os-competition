### 4. 核心代码调用流对比剖析

* **【进程管理/`sys_exec` 系统调用】**
* **底层技术析理**: 两者均实现了 Linux `execve` 的语义，但**文件解析与寻址模型完全不同**。目标项目采用基于 VFS（虚拟文件系统）的 `app_inode.get_dentry().path` 机制，通过 Dentry/Inode 对象进行路径解析，体现了更完整的 VFS 抽象层。参考项目则使用简单的 `abs_path.as_str()` 字符串路径，并硬编码了 `LD_LIBRARY_PATH` 等环境变量。这表明两者在内核的文件系统抽象层次和进程启动参数构建策略上存在根本性架构差异。
* **代码溯源链接**:
  - 目标项目: `os/src/syscall/process.rs#sys_exec`
  - 参考项目: `os/src/syscall/process.rs#sys_exec`

* **【进程管理/`current_task` 与同步原语】**
* **底层技术析理**: 这是控方指控的核心，但恰恰暴露了**同步策略的底层异构性**。目标项目使用 `PROCESSOR.lock()`，表明其 `Processor` 结构体被包裹在如 `Mutex` 的标准同步锁中，遵循 Rust 标准库的锁模式。参考项目使用 `PROCESSOR.exclusive_access()`，这通常是基于 `UPSafeCell` 或类似机制的自定义独占访问封装，是一种更早期或教学化的内核同步模式。两者虽然都返回当前任务，但实现线程安全的方式在抽象层次和具体机制上截然不同。
* **代码溯源链接**:
  - 目标项目: `os/src/task/processor.rs#current_task`
  - 参考项目: `os/src/task/processor.rs#current_task`

* **【内存管理/`exit_current_and_run_next` 资源回收】**
* **底层技术析理**: 两者在进程退出时的资源清理流程上存在相似性，如均涉及 `Arc::downgrade` 处理父进程引用、`INITPROC` 托管孤儿进程、`KContext::blank` 切换上下文。然而，目标项目的调用链显著更长且复杂（+68 vs +15），包含了 `FutexKey::new` 唤醒等待者、`Arc::strong_count` 检查等额外步骤。这表明目标项目在处理 futex、进程组关系等更精细的 Linux 语义时，进行了独立的扩展开发。
* **代码溯源链接**:
  - 目标项目: `os/src/task/mod.rs#exit_current_and_run_next`
  - 参考项目: `os/src/task/mod.rs#exit_current_and_run_next`

### 5. 单方模块与降权项
*   **网络协议栈**: 仅存在于目标项目，是独立开发的**强证据**。
*   **第三方库 (`virtio-drivers`, `lwext4`)**: 双方设备驱动和部分文件系统代码高度相似，均源于对这些公共上游 crate 的直接引用或 patch。**此部分相似度不计入原创性抄袭判定。**
*   **ABI 强制同名函数**: `sys_getpid`, `sys_clone`, `sys_exit` 等函数名由 Linux 系统调用规范强制定义，其存在本身不构成抄袭证据。
*   **Rust 语言惯用法**: `Option::take`, `Option::as_ref().map()`, `Vec::new`, `String::from` 等是 Rust 编程的标准实践，不具备项目级的独创性。
*   **跨板块弱信号**: 控方提出的 `init_procfs` 与 `create_proc_dir` 的相似性，因分属 `vfs` 和 `os/fs` 不同模块，且调用链差异巨大，**不予采纳为同源证据**。

### 6. 保守结论
综合全模块 AST 证据，目标项目与参考项目的客观相似度仅为 9.67%，远未达到可判定为“同源性”或“违规复用”的阈值。双方在进程管理、内存管理等核心板块的少量重合，经技术析理，可明确归因于 Linux ABI 强制接口、Rust 语言标准惯用法以及操作系统内核的通用设计范式。在同步原语、文件系统路径解析、系统调用实现广度等关键架构决策点上，双方表现出显著且根本的差异。特别是目标项目独立实现了网络协议栈、futex、SysV IPC 等复杂子系统，进一步强化了其独立开发的结论。设备驱动模块的高相似度源于共同依赖的上游开源库，不应作为抄袭证据。因此，本庭裁定，指控方提出的“抄袭与违规复用”指控**不成立**。
