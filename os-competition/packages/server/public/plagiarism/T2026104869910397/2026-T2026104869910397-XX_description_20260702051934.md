## 第四部分：分模块架构说明

### MM（内存管理）

**有底层证据支持**：
- `os/src/syscall/memory.rs#sys_mmap`、`sys_mlock2`、`sys_shmget`、`sys_shmctl` 提供用户态内存映射、共享内存、锁页等系统调用。
- `os/src/mm/memory_set.rs#sync_file_writeback` 实现文件映射页的回写；`load_interp` 负责 ELF interpreter 加载。
- `arch/src/addr.rs` 与 `arch/src/loongarch64/trap.rs` 配合处理地址转换与缺页异常。

**未体现/证据不足**：底层物理页分配器（buddy/slab）实现、swap 换出、页表遍历细节在提供的 AST 证据中未完整呈现。

### PROC（进程管理/任务调度）

**有底层证据支持**：
- `os/src/syscall/process.rs#sys_clone`、`sys_exec`、`sys_waitid`、`sys_exit_group` 覆盖进程/线程创建、程序执行、等待、退出。
- `os/src/task/task.rs#fork`、`#exec` 使用 `MemorySet::from_elf`、`FdTable::from_existed_table`、`KernelStack::new` 等构建新任务上下文。
- `os/src/task/processor.rs#run_tasks` 实现任务切换、信号检查、futex 定时器检查。
- 支持 futex（`sys_futex`）、消息队列（`sys_msg*`）、POSIX 消息队列（`sys_mq_*`）、调度参数/亲和性（`sys_sched_*`）、pidfd、namespace（`sys_setns`/`sys_unshare`）、capabilities（`sys_capget`/`sys_capset`）。
- `vfs/src/procfs` 提供 `/proc` 伪文件系统，暴露进程状态、内存信息、CPU 信息、挂载信息等。

**异步运行时**：证据中未见显式的 async/await、waker、reactor 等异步运行时结构；当前证据更偏向传统任务调度器。

### FS（文件系统）

**有底层证据支持**：
- VFS 层：`vfs/src/tmpfs`、`vfs/src/devfs`、`vfs/src/procfs` 实现内存文件系统、设备文件系统、proc 文件系统。
- 持久化 FS：`lwext4/src/fs.rs#mount`、`lwext4/src/file.rs#write_at`/`read_at`、`lwext4/src/dentry.rs#concrete_*` 对 C lwext4 进行 Rust 封装。
- 系统调用层：`os/src/syscall/fs.rs` 包含 `sys_openat`、`sys_read`、`sys_write`、`sys_pread64`、`sys_pwritev`、`sys_mount`、`sys_fcntl`、`sys_ioctl`、`sys_getdents`、`sys_epoll_pwait`、`sys_renameat` 等。
- 文件抽象：`os/src/fs/inode.rs#open_file`、`os/src/fs/pipe.rs`、`os/src/fs/stdio.rs` 提供 inode、管道、标准 IO。

### DEV（设备驱动）

**有底层证据支持**：
- `virtio-drivers/src/device/blk.rs`：VirtIO 块设备读/写/刷新。
- `virtio-drivers/src/device/console.rs`、`input.rs`、`socket/connectionmanager.rs`：VirtIO 控制台、输入、socket 设备。
- `isomorphic_drivers/src/net/ethernet/intel/ixgbe.rs`、`e1000.rs`：Intel 10GbE/1GbE 网卡驱动及中断处理。
- `visionfive2-sd/src/lib.rs`：VisionFive2 板载 SDHCI 控制器初始化与 SD 卡读写。

**说明**：`virtio-drivers`、`isomorphic_drivers` 等目录大概率是第三方 crate 或公共依赖，不应视为该作品原创内核主体。

### SYSCALL（系统调用）

**有底层证据支持**：
- `os/src/syscall/mod.rs#syscall` 是总入口，调用链覆盖 fs、process、memory、signal、socket 等子系统，规模达 39599，引用 233 个调用目标。
- 子模块：`os/src/syscall/fs.rs`（187 函数）、`os/src/syscall/process.rs`（107 函数）、`os/src/syscall/memory.rs`、`os/src/syscall/signal.rs`。

### TRAP/IRQ（中断异常）

**有底层证据支持，但函数数量少（30）**：
- `arch/src/loongarch64/trap.rs#loongarch64_trap_handler` 处理非法指令、浮点不可用、Load/Store 页错误等异常。
- `arch/src/loongarch64/trap.rs#enable_external_irq`、`init_interrupt` 控制外部中断与 TLB 初始化。
- `arch/src/loongarch64/unaligned.rs#emulate_load_store_insn` 通过 `read_fpr`/`write_fpr` 软件模拟非对齐浮点/通用访存。
- `arch/src/lib.rs#add_irq` 提供 IRQ 注册记录。

**未体现/证据不足**：完整中断控制器驱动（如 PLIC/APLIC/IOAPIC）、多核 IPI、时钟中断注入细节在提供的证据中未完整呈现。

### NET（网络协议栈）

**底层证据未体现完整网络协议栈**：
- 仅有 `os/src/socket/crypto.rs`（MD5/SHA 系列哈希算法封装、`validate_authenc_key`、`make_hash_op` 等）与 `os/src/socket/mod.rs`（字段解码、状态快照）等辅助代码。
- 系统调用分发器中引用了 `sys_accept`、`sys_bind`、`sys_sendmsg`、`sys_setsockopt` 等，但实现层未在提供的证据中展示。
- 网络设备驱动（ixgbe/e1000/VirtIO socket）存在，但 **TCP/IP 协议栈、路由、ARP、netfilter 等核心网络协议栈实现无底层证据**。
