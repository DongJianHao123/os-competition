## 第四部分：分模块架构说明

### MM（内存管理）

**底层证据较强**。内核侧实现了：

- **堆分配器**：`src/kernel/alloc.c#kmalloc` / `kfree` / `krealloc`，基于块头（`BLOCK_TO_DATA`、`GET_SIZE`、`IS_FREE`、`insert_free`、`try_merge`）的链表式分配器；
- **页式内存管理**：`src/kernel/proc.c#map_page`、`walk_page`、`unmap_page` 使用 `VPN0/VPN1/VPN2`、`alloc_page_table`、`make_pte`、`pte_to_ppn`，并引用 RISC-V 的 `satp`，说明采用三级页表；
- **页分配**：`alloc_page` / `free_page` 被 ELF 加载器调用；
- **VirtIO 块设备 DMA 内存**：`src/kernel/virtio_blk.c#virtio_blk_init_one` 使用 `alloc_pages`、`ALIGN_UP`、`mmio_read32/mmio_write32`。

**未体现**：交换（swap）、请求分页（demand paging）、写时复制（COW）、内存去重、大页支持。

### PROC（进程管理/任务调度）

**传统进程管理底层证据未体现**。现有证据仅包括：

- `src/kernel/proc.c#proc_init`、`proc_create`、`proc_set_current`、`proc_get`、`proc_get_free_pid`、`proc_destroy`：PCB 雏形与 PID 分配；
- `src/kernel/proc.c#enter_user_process`：通过 `_enter_user_trampoline` 进入用户态；
- `src/kernel/syscall.c#sys_exit`：进程退出并切换页表。

**未见**：`fork`/`exec`/`wait`/`exit` 完整生命周期、父子进程关系、僵尸进程处理、调度队列、时间片轮转/CFS 等调度算法。`任务调度/异步运行时` 模块的 34 个函数大多属于 ARM 中断/SVC 路由或 Windows 主机事件循环，不能等同于内核调度器。

### FS（文件系统）

**底层证据较强**。`src/kernel/ext4.c` 实现了 ext4 的只读/有限功能：

- `ext4_mount`：读取超级块、块组描述符，建立挂载上下文；
- `ext4_open`、`ext4_read`、`ext4_readdir`：按 inode、extent 映射读取文件与目录；
- `extent_map_logical_to_physical`、`file_map_block`、`read_inode`、`get_inode_table_block`：extent 与 inode 表解析；
- 底层块读写通过 `virtio_blk_read` 完成。

**未体现**：VFS 抽象层、文件描述符表、写操作、日志（journaling）、权限检查、硬链接/符号链接、缓存层。

### DEV（设备驱动）

**证据分为两类**：

1. **内核原创驱动**：`src/kernel/virtio_blk.c#virtio_blk_init_one`，通过 MMIO 配置 VirtIO 块设备，初始化描述符环。
2. **第三方/板级 HAL**：`Libary/` 下大量 STM32F10x 标准外设库函数（`GPIO_Init`、`USART_Init`、`TIM_ICInit`、`SPI_Init`、`I2C_Init`、`FSMC_NORSRAMInit`、`SDIO_SendCommand`、`ADC_Init` 等），以及 `Syscall/` 中的 CH340、USB HID、RGB LED、ADC 随机数等封装。

**未体现**：内核级中断驱动框架、设备树/总线枚举、DMA 驱动抽象、定时器驱动。

### SYSCALL（系统调用）

**存在两套系统调用语境**：

1. **RISC-V 内核系统调用**：`src/kernel/syscall.c#trap_handler` 分发 `sys_read/write/openat/close/brk/fstat/exit` 等；`sys_openat` 直接调用 ext4 与页表遍历。
2. **ARM SVC/安全服务**：`Syscall/stm32f10x_it.c#SVC_Router` 处理 `Kernel_ReadData/WriteData/ReadTask/WriteTask`，并调用 `SecCore_AuditSVC` 进行审计。

**未体现**：系统调用表动态注册、参数校验框架、用户态/内核态完整隔离机制。

### TRAP/IRQ（中断异常）

**底层证据较弱**。统计仅 3 个函数归入该模块：

- `Libary/misc.c#NVIC_Init`
- `Libary/stm32f10x_exti.c#EXTI_GenerateSWInterrupt`
- `Syscall/ch340_comm.c#CH340_Comm_Init` 中配置 USART 中断与 `USART1_IRQHandler`

`src/kernel/syscall.c#trap_handler` 是 RISC-V 侧异常/系统调用入口，但未见完整 IDT/异常向量表、时钟中断、上下文保存恢复细节。

**未体现**：完整中断向量表、异常分级处理、嵌套中断控制、定时器 tick 中断。

### NET（网络协议栈）

**内核网络协议栈底层证据未体现**。`网络协议栈` 模块的 45 个函数全部位于 `exe/` 的 Windows 主机工具中：

- `do_handshake`、`Proto_BuildEncryptedCmd`、`Proto_ParseAck`、`Serial_ReadFrame`、`Serial_WriteFrame`；
- `Crypto_AES_GCM_Encrypt` / `Crypto_HMAC_SHA256` 调用 Windows `BCrypt*` API。

这些属于**应用层安全通信协议**（基于串口/HID），并非 TCP/IP 协议栈。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块/特性

1. **传统进程管理**：无 `fork`/`exec`/`wait`/`zombie`、无父子进程关系、无调度队列与调度算法；
2. **完整任务调度/异步运行时**：无内核线程、无协程/waker/异步运行时、无时间片管理；
3. **内核级网络协议栈**：无 TCP/IP、无 socket 抽象、无网卡驱动；现有“网络”证据均为 `exe/` 主机应用层协议；
4. **完整中断/异常子系统**：无完整 IDT/向量表、无定时器 tick、无上下文切换保存恢复细节；
5. **文件系统高级特性**：无写操作、无日志、无 VFS、无文件描述符表、无缓存层；
6. **虚拟内存高级特性**：无 swap、无 demand paging、无 COW、无大页；
7. **SMP/多核**：无多核启动、核间中断、自旋锁等证据；
8. **用户态隔离与权限**：除页表与 trap 入口外，无完整权限模型、 capability、系统调用参数校验框架。

### 需要单独说明的分类问题

- 模块画像中大量函数来自 `Libary/` 的 STM32F10x 标准外设库，这些属于**第三方/硬件协议强约束代码**，不应被夸大为本项目的原创内核主体；
- `进程管理`、`任务调度/异步运行时`、`网络协议栈` 等模块的函数数量受分类算法影响，存在把外设等待函数、HAL 初始化函数、主机工具通信函数归入其中的现象；
- `Syscall/` 目录名称容易与 `src/kernel/syscall.c` 混淆：前者是 ARM 固件的安全核心/SVC 例程，后者是 RISC-V 内核的系统调用分发器。
