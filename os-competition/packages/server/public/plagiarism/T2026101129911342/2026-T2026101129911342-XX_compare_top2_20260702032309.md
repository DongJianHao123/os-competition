## 4. 核心代码调用流对比剖析

### 【进程管理】信号掩码系统调用

**底层技术析理**:
目标项目 `sys_rt_sigprocmask` 采用**直接用户态内存裸指针访问**模型，通过 `(oldset as *mut u64).write_volatile` 和 `(set as *const u64).read_volatile` 直接操作用户态传入的 64 位信号掩码字。该实现不依赖任何进程抽象层，无 `current_task`、无 `Process` 结构体、无 `MemorySet` 安全转换。

参考项目 `sys_rt_sigprocmask` 采用**进程抽象 + 安全内存访问**模型，通过 `current_task().unwrap()` 获取当前任务上下文，使用 `SignalMaskFlag::from_bits(how).ok_or` 解析操作类型，通过 `memory_set.get_data` 进行用户态内存安全读写。

**异构性判定**: 两者在**抽象层次**（无进程抽象 vs 完整 Task/Process 模型）、**内存模型**（裸指针 volatile vs MemorySet 安全转换）、**同步策略**（无锁 vs 隐含在 current_task 的 Mutex 中）三个维度均存在根本性差异。目标项目的裸指针架构不可能是从参考项目的安全抽象“降级重写”而来——若真抄袭，`current_task`、`memory_set` 等核心抽象必然以某种变形保留。

**代码溯源链接**:
- 目标项目: `kernel/src/syscall.rs#sys_rt_sigprocmask`
- 参考项目: `os/src/syscall/signal.rs#sys_rt_sigprocmask`

### 【内存管理】堆分配器

**底层技术析理**:
目标项目 `alloc`/`dealloc` 实现了一个**基于空闲链表的自旋锁堆分配器**。核心数据结构为 `bnext`（下一空闲块指针）和 `bsize`（块大小），同步机制使用 `self.lock.compare_exchange(false, true, Ordering::Acquire, Ordering::Relaxed)` 实现 CAS 自旋锁，分配策略为 `align_up` + 链表遍历 + `self.insert` 回收。

参考项目 `alloc`/`dealloc` 实现了一个**基于全局堆分配器的页缓存分配器**。核心调用为 `HEAP_ALLOCATOR.try_lock().unwrap().alloc`，使用 `Layout::from_size_align(PAGE_SIZE, PAGE_SIZE)` 固定页大小分配，返回 `PhysAddr::from` 物理地址。

**异构性判定**: 两者在**抽象层次**（裸链表堆分配器 vs 页缓存分配器）、**内存模型**（可变块大小 vs 固定 PAGE_SIZE）、**同步策略**（CAS 自旋锁 vs try_lock Mutex）三个维度完全不同。目标实现的是通用 `GlobalAlloc` trait，参考实现的是特定页帧分配器，领域边界不同。

**代码溯源链接**:
- 目标项目: `kernel/src/heap.rs#alloc`
- 参考项目: `os/src/mm/frame_alloc/page_cache.rs#alloc`

### 【任务调度/异步运行时】用户态进入

**底层技术析理**:
目标项目 `run_user`（riscv64/loongarch64 双架构）是一个**架构相关 HAL 层函数**，负责单次用户态进入与退出。调用流为：`KernelContext::default` 初始化上下文 → `arm_timeout` 设置定时器 → `init_trap` 配置 trap → `__run_user`/`__la_run_user` 汇编进入用户态 → `disarm_timeout` 清除定时器。该函数不涉及任务调度队列，仅处理单次上下文切换。

参考项目 `run_tasks` 是一个**多任务调度主循环**。调用流为：`processor.get_idle_task_cx_ptr` 获取 idle 上下文 → `cur_task.inner_lock` 锁定当前任务 → `next_task.inner_lock` 锁定下一任务 → `check_futex_timer` 检查 futex 超时 → 上下文切换。该函数是完整的调度器核心。

**异构性判定**: 两者在**抽象层次**（单函数 HAL vs 调度器主循环）、**对象管理模型**（无 Task 抽象 vs Arc<Mutex<Task>>）、**同步策略**（无锁汇编切换 vs inner_lock 互斥）三个维度完全不同。目标 `run_user` 是参考 `run_tasks` 中上下文切换部分的底层实现，但两者处于完全不同的抽象层级，调用图谱无交集。

**底层技术析理**:
