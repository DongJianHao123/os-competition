## 第四部分：分模块架构说明

### MM（内存管理）

底层证据较强。核心包括内核堆分配器、进程地址空间 `MemorySet`、页表操作、ELF 加载时的内存映射、线程退出时的资源回收，以及异常入口中的缺页处理。

- `os/src/mm/heap_allocator.rs#handle_alloc_error`：堆分配失败处理，记录实际/用户分配字节数，并可能触发当前任务退出。
- `os/src/mm/memory_set.rs#dealloc_user_res`：回收用户栈、trap context 等用户资源，调用 `munmap` 与 `remove_area_with_start_vpn`。
- `os/src/task/task.rs#load_elf` / `os/src/task/task.rs#sys_clone`：ELF 加载与 clone 时通过 `MemorySet::from_elf` / `MemorySet::from_existing_user` 构建或复刻地址空间。
- `os/src/hal/arch/loongarch64/trap/mod.rs#trap_handler`：读取 `BadV`/`ERA`，解析指令，使用 `LAFlexPageTable` 处理缺页。

### PROC（进程管理/任务调度）

传统进程管理证据充分，不是轻量级任务模型。包含 clone、execve、wait4/waitid、exit、信号、 futex、POSIX timer、capability、UID/GID、rlimit、sched affinity/nice/policy 等。

- `os/src/syscall/process.rs#sys_clone_with_options`：解析 `CloneFlags`，创建子任务/线程。
- `os/src/task/task.rs#sys_clone`：构造新 PCB，复制/共享 `MemorySet`、文件表、信号、定时器、futex 等。
- `os/src/syscall/process.rs#do_execve_from_base` / `os/src/syscall/process.rs#sys_execve`：执行新程序，处理 shebang、参数与环境变量。
- `os/src/syscall/process.rs#sys_wait4` / `sys_waitid`：阻塞等待子进程事件，收集 rusage。
- `os/src/task/signal.rs#do_signal`：信号投递，构造 `MachineContext`，设置用户态信号处理函数。
- `os/src/task/threads.rs#do_futex_wait_until`：futex 等待队列与超时阻塞。
- `os/src/task/mod.rs#exit_current_and_run_next` / `exit_group_and_run_next`：任务退出与调度切换。

**调度/异步运行时**：有 `sched_entity.set_affinity/set_policy/set_nice`、CFS weight 转换等证据，但**未体现独立的 async/await executor、waker、reactor**。现有证据偏向传统线程调度。

### FS（文件系统）

文件系统是代码量最大的模块，证据强。包含 VFS 目录树、ext4、procfs、devfs、anon fd、pipe、mount 等。

- `os/src/fs/directory_tree.rs#open` / `rename` / `exchange` / `link_file` / `delete`：VFS 路径解析与目录操作，含 `PATH_CACHE` 与 busybox 重定向逻辑。
- `os/src/fs/dev/proc.rs`（`entries`、`content`、`get_dirent`、`apply_write`）：procfs 实现，暴露 `/proc/[pid]/cmdline、comm、fd、stat、status、task` 等。
- `os/src/fs/dev/anon.rs#new_pid`：eventfd/timerfd/memfd 等匿名文件描述符。
- `os/src/fs/ext4/ext4_inode.rs#allocate_new_block`：ext4 块分配、位图更新、checksum 同步。
- `os/src/fs/ext4/extent.rs#ext_remove_leaf` / `extent_remove_space` / `ext_grow_indepth`：ext4 extent 树操作。
- `os/src/fs/ext4/layout.rs#create` / `unlink` / `get_dirent`：ext4 inode 创建、删除、目录项读取。
- `os/src/syscall/fs.rs#sys_mount` / `sys_openat_path` / `sys_copy_file_range` / `sys_fcntl`：文件系统相关系统调用。

**注意**：`文件系统/第三方库` 分类包含 404 个函数，ext4 相关代码可能为第三方适配/移植，不宜直接视为完全原创。

### DEV（设备驱动）

原创设备驱动证据较弱（仅 79 个函数），大量驱动代码位于 `os/path-deps/` 与 `dependency/` 中，属于第三方或适配驱动。

- `os/src/drivers/block/virtio_blk.rs#new`：VirtIO block 设备封装。
- `os/src/drivers/block/sata_blk.rs#pci_init`：AHCI/SATA 块设备初始化。
- `os/path-deps/isomorphic_drivers/src/net/ethernet/intel/ixgbe.rs#new` / `msend`：Intel 10GbE 网卡初始化与发送。
- `os/path-deps/isomorphic_drivers/src/net/ethernet/intel/e1000.rs#send` / `receive`：Intel e1000 网卡收发。
- `os/path-deps/pci/src/lib.rs#probe_function`：PCI 设备枚举。

### SYSCALL（系统调用）

系统调用层证据强，分为文件、进程、网络三类，入口为 dispatch。

- `os/src/syscall/dispatch.rs`：系统调用号分发（235 函数）。
- `os/src/syscall/fs.rs#sys_read` / `sys_write` / `sys_openat_path` / `sys_copy_file_range` / `sys_fcntl`：文件类调用。
- `os/src/syscall/process.rs#sys_clone_with_options` / `sys_wait4` / `sys_execve` / `sys_prctl` / `sys_prlimit`：进程类调用。
- `os/src/syscall/net.rs#sys_sendto` / `sys_getsockopt` / `unix_sockaddr_path`：网络类调用。

### TRAP/IRQ（中断异常）

LoongArch64 侧证据较完整，RISC-V 侧有入口但细节较少。

- `os/src/hal/arch/loongarch64/trap/mod.rs#trap_handler`：用户态/内核态异常入口，处理缺页、系统调用等。
- `os/src/hal/arch/loongarch64/trap/mod.rs#trap_from_kernel`：内核态异常处理。
- `os/src/hal/arch/loongarch64/register/base/estat.rs#cause`：异常/中断原因解析。
- `os/src/hal/arch/loongarch64/trap/mod.rs#enable_timer_interrupt`：定时器中断使能。
- `os/src/hal/arch/loongarch64/trap/mem_access.rs#get_addr`：访存指令地址解析。
- `os/src/hal/arch/riscv/trap/mod.rs#trap_handler`：RISC-V 异常入口。

### NET（网络协议栈）

网络协议栈证据中等。存在 TCP/UDP/Unix socket 抽象、端点绑定、loopback 队列，但底层协议栈可能依赖外部实现（如 smoltcp）。

- `os/src/net/tcp.rs#connect` / `_accept`：TCP 连接与接受，调用 `NET_INTERFACE.poll`。
- `os/src/net/udp.rs#new` / `drop` / `unregister_udp_endpoint` / `dequeue_loopback_packet`：UDP socket 生命周期与 loopback 包处理。
- `os/src/net/mod.rs#can_bind` / `find_unix_by_path`：端口绑定检查与 Unix socket 查找。
- `os/src/syscall/net.rs#sys_sendto` / `sys_getsockopt`：socket 系统调用。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块/能力

- **独立的 async/await 运行时**：仅有 `block_current_and_run_next`、futex 等待队列、调度参数设置等传统线程调度证据，未体现 reactor、waker、executor、async task 状态机等。
- **原创设备驱动主体**：`os/src/drivers/` 下原创封装层仅 79 个函数，网卡/块设备核心逻辑大量位于 `os/path-deps/isomorphic_drivers` 与 `dependency/virtio-drivers` 等第三方代码中。
- **ext4 原创性边界**：`文件系统/第三方库` 分类含 404 个函数，ext4 相关代码（`os/src/fs/ext4/`）被归为此类，说明其可能为第三方移植/适配，不宜直接视为完全原创文件系统。
- **底层网络协议栈实现**：socket 抽象层证据充分，但 TCP/IP 协议状态机可能依赖外部 crate（如 smoltcp），本仓库未直接体现自研协议栈核心。
- **RISC-V 中断/异常细节**：仅有 `os/src/hal/arch/riscv/trap/mod.rs#trap_handler` 入口证据，寄存器解析、上下文保存等细节未在提供的证据中展开。
- **以下子系统底层证据未体现**：USB、HID、GPU/显示、音频、ACPI/设备树解析、虚拟化/Hypervisor、内核调试/跟踪、电源管理、swap/页缓存写回、加密子系统、动态可加载内核模块等。

### 判断边界说明

- 本作品“可能的系统方向”仅基于目录结构、函数分布与调用链推断，不是对作品最终定位或原创性的结论。
- 第三方库/依赖（`dependency/`、`os/path-deps/`）的函数数量已单独统计，不应与 `os/src/` 原创内核主体混为一谈。
- 本描述性文档仅用于帮助评审快速定位技术实现范围，不做抄袭、违规或身份判定。
