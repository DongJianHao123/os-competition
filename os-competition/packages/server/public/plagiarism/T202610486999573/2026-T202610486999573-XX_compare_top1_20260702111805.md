* **底层技术析理**: 双方 `proc_sched` 函数的调用图谱完全一致，均遵循 xv6 教学内核的上下文切换范式：先通过 `assert` 和 `spinlock_holding` 确保已持有当前进程锁，再通过 `intr_get` 验证中断已关闭，最后调用 `swtch` 进行上下文切换。此调用流是保证多核调度正确性的最小安全契约，并非某一项目的独创表达。差异点在于目标项目调度器名为 `proc_scheduler` 且额外调用 `mycpuid`，表明其在调度循环中可能引入了 CPU 亲和性等独立设计。

  - 目标项目: `kernel/proc/proc.c#proc_sched`
  - 参考项目: `OSakura-rv/kernel/proc/proc.c#proc_sched`
