## 第四部分：分模块架构说明

### MM（内存管理）
底层证据较强。包含物理页分配（`pmem_alloc`/`pmem_free`）、页表建立与拆除（`vm_mappages`、`vm_unmappages`、`vm_getpte`）、用户虚拟内存管理（`uvm_copy_pgtbl`、`uvm_mmap`、`uvm_munmap`、`uvm_heap_grow`、`uvm_copyin`/`uvm_copyout`）、ELF 加载（`elf_load`）以及 `sys_mmap`/`sys_execve` 中的映射操作。  
证据锚点：`kernel/mem/kvm.c#vm_mappages`、`kernel/mem/uvm.c#uvm_copy_pgtbl`、`kernel/syscall/sysfunc.c#sys_execve`、`kernel/fs/file.c#file_write`。

### PROC（进程管理/任务调度）
底层证据体现为传统 Unix 式进程管理：PCB 分配（`proc_alloc`）、fork/clone（`proc_fork_inner`、`sys_clone`）、execve、exit（`proc_exit`）、wait4（`proc_wait4`）、调度器循环（`proc_scheduler`）、yield/sleep/wakeup（`proc_yield`、`proc_sleep`、`proc_wakeup`、`timer_update`）。  
证据锚点：`kernel/proc/proc.c#proc_scheduler`、`kernel/proc/proc.c#proc_exit`、`kernel/proc/proc.c#proc_wait4`、`kernel/proc/proc.c#proc_fork_inner`。

**注意**：虽然模块画像将本类别标为“任务调度/异步运行时”，但 AST 调用链中未出现 `async/await`、`future`、`waker`、`task_queue`、`ring_fifo` 等异步运行时原语，现有证据只能支撑“传统进程调度”，不能支撑“完整异步运行时”。

### FS（文件系统）
底层证据强且分层清晰：
- 自定义文件系统层：`fs_init`、`inode`、`dir`、`file`、`buf` 构成的索引节点与目录树系统。
- 块缓存层：`buf_read`/`buf_write` 通过自旋锁/睡眠锁管理，并下接 VirtIO 磁盘。
- ext4 兼容层：`ext4_init`、`ext4_read_file`、`read_inode`、`read_extent_tree` 等，使用 `pmem_alloc` 做缓存。
- 内存文件/ramfs：`ramdir_init`、`ramdir_add`、`ramdir_exists`、`ramfs_get_or_create`。  
证据锚点：`kernel/fs/fs.c#fs_init`、`kernel/fs/file.c#file_read`、`kernel/fs/inode.c#inode_read_data`、`kernel/fs/ext4.c#ext4_init`、`kernel/fs/buf.c#buf_read`。

### DEV（设备驱动）
底层证据较弱，种类有限。仅发现：
- VirtIO 块设备：`virtio_disk_rw`、`virtio_disk_intr`。
- UART 控制台：`uart_putc_sync`、`console_read`、`console_write`。
- PLIC 中断控制器：`plic_init`、`plic_inithart`、`plic_claim`、`plic_complete`。
- RISC-V 定时器：`timer_init`、`timer_update`。  
证据锚点：`kernel/dev/virtio.c#virtio_disk_rw`、`kernel/dev/uart.c#console_read`、`kernel/dev/plic.c#plic_inithart`、`kernel/dev/timer.c#timer_init`。

### SYSCALL（系统调用）
底层证据强。`kernel/syscall/syscall.c#syscall` 作为分发器，调用 `sysfunc.c` 中大量实现；`sysfunc.c` 是项目最密集文件（40 个函数），覆盖 `sys_openat`、`sys_execve`、`sys_mmap`、`sys_pipe2`、`sys_clone`、`sys_wait4`、`sys_getdents64` 等。用户态通过 `user/user_syscall.c` 发起。  
证据锚点：`kernel/syscall/syscall.c#syscall`、`kernel/syscall/sysfunc.c#sys_openat`、`kernel/syscall/sysfunc.c#sys_execve`、`kernel/syscall/sysfunc.c#sys_mmap`。

### TRAP/IRQ（中断异常）
底层证据体现 RISC-V 陷阱处理：`trap_kernel_handler` 处理内核态陷阱，`trap_user_handler` 处理用户态陷阱，`external_interrupt_handler` 处理 PLIC 外设中断，`timer_interrupt_handler`/`timer_init` 处理时钟中断。涉及 CSR 读写（`r_scause`、`r_sepc`、`w_sepc`、`w_sstatus`、`w_satp` 等）。  
证据锚点：`kernel/trap/trap_kernel.c#trap_kernel_handler`、`kernel/trap/trap_user.c#trap_user_handler`、`kernel/trap/trap_kernel.c#external_interrupt_handler`、`kernel/dev/timer.c#timer_init`。

### NET（网络协议栈）
**底层证据未体现**。AST Call Map 与模块画像中均未发现网络协议栈相关函数或目录。
