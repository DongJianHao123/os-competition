  - **目标项目独有**: `proc_wait4`/`proc_wait_copy_status` (完整的 wait4 语义)、`fork_return`/`proc_fork_set_usp` (分层 fork 实现)、`mmap_init`/`pmem_init` (独立内存管理框架)。这些独有抽象是独立架构设计的铁证。
  - **参考项目独有**: `ext4_file_close`、`fat32_file_close`、`fat32_dir_addInode` 等深度绑定 FAT32/Ext4 的函数，目标项目均未实现。
- **降权项说明**:
  - **跨板块硬凑**: 控方将 `kernel/fs/inode.c#inode_read_data` (文件系统) 与 `kernel/fs/fat32/fat32_dir.c#lookup_path` (文件系统) 强行归入【任务调度/异步运行时】板块进行对比，属于板块归类错误，其证据效力已被降级为弱信号。
  - **公共接口与泛型函数**: `printf`, `main`, `syscall` 等函数名和基础调用模式（如 `va_arg`）是 C 语言标准库和操作系统启动流程的通用范式，不能作为同源证据。
  - **同名标准函数**: `sys_getpid`、`mycpu` 等函数名直接来源于 xv6 公开教材，其实现逻辑由功能语义唯一确定，同名且同调用图是必然结果。

### 6. 保守结论
综合全模块证据，本案客观 AST 相似度仅 3.57%，双向覆盖率均不足 10%。控方提出的“铁证”与“强指控”，经深度析理，均为 RISC-V 硬件架构、xv6 教学范式和 POSIX 标准接口所导致的**领域强制同构**，而非独创性代码的复用。双方在进程调度策略（协作式 vs 异步终止）、内存管理模型（mmap/pmem vs pmem_alloc_pages）、文件系统架构（VFS 紧耦合 vs 多文件系统松耦合）及系统调用分发机制（静态分发 vs 函数指针表）等核心子系统上，展现出根本性的**架构异构性**。目标项目 91.22% 的独有 AST 指纹进一步夯实了其独立开发的定性。因此，整体抄袭风险极低，应判定为独立开发。
