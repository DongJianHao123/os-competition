## 第四部分：分模块架构说明

### MM（内存管理）
- **物理内存**：`kernel/src/pmm.c` 实现基于位图（bitmap）的物理页分配器，`pmm_init` 初始化可用内存区域，`pmm_alloc_page` / `pmm_free_page` 进行分配与回收。
- **虚拟内存/页表**：`kernel/src/vmm.c` 为架构无关入口，调用 `kernel/arch/riscv64/vmm.c` 与 `kernel/arch/loongarch64/vmm.c` 中的 `vmm_arch_init`、`vmm_arch_enable`、`vmm_arch_map_page`、`vmm_arch_protect_page`、`vmm_arch_set_user_bit` 完成页表遍历、映射、权限设置与 TLB 刷新。
- **用户空间映射**：`proc_create_user`、`elf_load`、`sys_pipe` 等均通过 `pmm_alloc_page` + `vmm_arch_map_page` 建立用户页映射。
- **证据强度**：强。

### PROC（进程管理）
- **PCB 与 PID**：`kernel/src/proc.c` 维护进程表，`alloc_pid`、`find_unused`、`find_by_pid` 为进程查找/分配原语。
- **生命周期**：`proc_create`、`proc_create_user`、`proc_fork` 创建进程；`sys_execve` 加载 ELF；`sys_exit` / `proc_exit_code` 退出；`sys_wait` / `sys_waitpid` / `proc_waitpid` 等待子进程。
- **父子关系**：`proc_set_ppid`、`sys_getppid`、`proc_waitpid` 体现父子进程关系。
- **资源**：每个进程维护 `cwd`、`brk`、`fd table`（通过 `proc_get_cwd`、`proc_set_cwd`、`proc_get_brk`、`proc_set_brk` 访问）。
- **证据强度**：较强；传统进程管理有明确证据。

### SCHED（任务调度/异步运行时）
- **调度器**：`kernel/src/sched.c#schedule` 完成上下文切换（`switch_to`）；`sched_tick` 由定时器触发，调用 `schedule`；`sched_init` 初始化并启用定时器。
- **定时器**：`kernel/arch/riscv64/sched.c` 使用 `rdtime` + `sbi_set_timer`；`kernel/arch/loongarch64/sched.c` 使用 CSR 写周期/compare 寄存器。
- **自愿切换**：`sys_yield`、`sys_sleep`、`proc_yield` 主动让出 CPU；LoongArch 侧还存在 `sched_cooperative_tick` 配合 `delay_loop` 的协作式路径。
- **异步运行时**：代码中未见 `async/await`、`task`、`channel`、`waker`、`ring_fifo` 等异步运行时原语，**“异步运行时”部分底层证据未体现**。
- **证据强度**：调度器较强，异步运行时无证据。

### FS（文件系统）
- **VFS 层**：`kernel/src/fs.c` 提供统一抽象，包括 `vfs_lookup`、`vfs_open`、`vfs_create`、`vfs_mkdir`、`vfs_unlink`、`vfs_getdents`、`vfs_stat`、`vfs_get_path`、`vfs_add_dentry`、`vfs_alloc_inode`、`vfs_alloc_file`。
- **具体文件系统**：
  - `ramfs`：`kernel/src/ramfs.c` 内存文件系统，`ramfs_init`、`ramfs_write` 等。
  - `devfs`：`kernel/src/devfs.c` 设备文件系统，挂载 UART、null 等设备节点。
- **挂载**：`sys_mount`、`sys_umount`、`mount_find`、`mount_alloc_slot` 实现挂载表管理。
- **证据强度**：强。

### DEV（设备驱动）
- **可见设备层**：仅通过 `devfs` 暴露的字符设备，如 `uart_dev_read/write`、`null_dev_read/write`。
- **缺失**：未见块设备驱动、磁盘控制器、DMA、显示/键盘/鼠标等外设驱动；UART 也仅作为控制台输出（`uart_putc`、`uart_puts`、`uart_put_hex`、`uart_put_dec`）。
- **证据强度**：仅字符控制台设备，整体设备驱动证据弱。

### SYSCALL（系统调用）
- **分发入口**：`kernel/src/syscall.c#syscall_handler` 为通用系统调用分发器；`syscall_handler6` / `ksyscall6` 处理 6 参数调用（如 `sys_mmap6`）。
- **覆盖范围**：文件、进程、内存、时间、目录、挂载、管道、信号（stub）等。
- **证据强度**：中枢强。

### TRAP/IRQ（中断异常）
- **初始化**：`kernel/arch/riscv64/trap.c#trap_init` 与 `kernel/arch/loongarch64/trap.c#trap_init` 设置 CSR（`csr_write`）。
- **入口**：`kernel/src/trap.c#trap_handler` 读取异常原因并分发至 `syscall_handler` 或 `sched_tick`。
- **局限**：仅体现定时器中断与系统调用异常路径；**外部中断控制器（如 PLIC、LoongArch PIC/IOAPIC）及设备 IRQ 处理底层证据未体现**。
- **证据强度**：入口级弱。

### NET（网络协议栈）
- **底层证据未体现**：源码中未见网络协议栈、套接字、网卡驱动、TCP/IP 等相关函数或目录。

## 第六部分：边界与未体现内容

### 明确无底层证据支撑的模块/能力
1. **网络协议栈（NET）**：无套接字、协议层、网卡驱动相关函数。
2. **块设备/磁盘 I/O**：无 block layer、DMA、磁盘控制器驱动；`ramfs` 为纯内存文件系统。
3. **完整异步运行时**：无 `async/await`、`task`、`channel`、`waker`、`ring_fifo` 等原语；“任务调度/异步运行时”模块中的异步运行时部分底层证据未体现。
4. **外部中断控制器与设备 IRQ**：仅有定时器中断路径，未见 PLIC、LoongArch PIC/IOAPIC 等中断控制器及多设备 IRQ 处理。
5. **信号实际投递**：`sys_sigaction`、`sys_sigreturn`、`sys_kill` 仅调用 `proc_current` / `proc_find_by_pid`，未见信号帧投递、挂起位图、处理函数切换等完整实现。
6. **SMP/多核**：无自旋锁、CPU 编号、核间中断、per-CPU 结构等证据。
7. **虚拟内存高级机制**：无 page fault handler、swap、页面置换、COW（写时复制）等证据。
8. **安全/权限模型**：仅通过 `vmm_arch_set_user_bit` 区分用户/内核页，未见完整 capability、ACL、权限检查框架。

### 需要单独说明的第三方/硬件约束代码
- **RISC-V SBI 与 CSR 操作**：`sbi_set_timer`、`rdtime`、`csr_read`/`csr_write` 等属于 RISC-V 硬件接口/固件约定，不应视为原创内核主体。
- **LoongArch CSR 与 TLB 指令**：`csr_write`、`tlb_flush_all` 等为 LoongArch 架构强约束代码。
- 这些代码是跨架构移植所必需的硬件胶水层。
