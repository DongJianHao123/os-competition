## 第四部分：分模块架构说明

### MM（内存管理）
底层证据较强。核心函数包括：
- `kernel/mem.c#mem_init`：初始化内存子系统，并通过 UART 输出内存信息
- `kernel/mem.c#page_alloc` / `kernel/mem.c#page_free`：物理页分配与释放
- `kernel/mem.c#kmalloc`：内部闭环逻辑，疑似内核堆分配
- `kernel/mem.c#get_free_pages` / `kernel/mem.c#get_total_pages`：内存统计接口
- `kernel/main.c#kernel_main` 直接调用上述接口，说明内存管理是启动流程的核心环节

目前仅体现物理页级分配，**虚拟内存、页表、地址映射等底层证据未体现**。

### PROC（进程管理/任务调度）
证据为轻量级任务模型，而非完整进程管理：
- `kernel/sched.c#proc_create`：创建任务/进程，分配页面并初始化上下文
- `kernel/sched.c#proc_exit`：退出时释放页面并触发调度
- `kernel/sched.c#proc_yield` / `kernel/sched.c#schedule`：任务让出与调度器核心
- `kernel/sched.c#timer_triggered` / `kernel/sched.c#set_next_timer` / `kernel/sched.c#read_time`：定时器中断驱动的抢占式调度

**传统进程管理（fork/exec/wait/zombie、PCB、父子进程关系、地址空间隔离）底层证据未体现**。

### FS（文件系统）
**底层证据未体现**。模块画像中“文件系统”仅统计到 `kernel/uart.c#uart_read_reg` 与 `kernel/uart.c#uart_write_reg`，实际为 UART 寄存器读写，不属于文件系统。未见 VFS、inode、文件描述符、块设备、路径解析等证据。

### DEV（设备驱动）
证据较强，主要为 UART 串口驱动：
- `kernel/uart.c#uart_init`：初始化 UART，读写寄存器
- `kernel/uart.c#uart_read_reg` / `kernel/uart.c#uart_write_reg`：寄存器访问原语
- `kernel/uart.c#uart_putc` / `kernel/uart.c#uart_getc`：字符收发
- `kernel/uart.c#uart_puts` / `kernel/uart.c#uart_int` / `kernel/uart.c#uart_hex`：格式化输出

该层依赖硬件 UART 寄存器协议，属于典型的 16550-like 串口驱动封装。

### SYSCALL（系统调用）
**底层证据未体现**。未找到系统调用分发、syscall 表、用户态/内核态切换封装等函数。

### TRAP/IRQ（中断异常）
证据中等，为顶层分发框架：
- `kernel/trap.c#trap_init`：陷阱/中断初始化
- `kernel/trap.c#handle_trap`：总入口，分发异常与中断
- `kernel/trap.c#handle_exception`：异常处理，主要输出调试信息
- `kernel/trap.c#handle_interrupt`：中断处理，定时器中断触发调度

### NET（网络协议栈）
