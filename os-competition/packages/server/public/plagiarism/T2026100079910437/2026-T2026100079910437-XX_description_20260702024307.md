## 第四部分：分模块架构说明

### MM（内存管理）

**底层证据较强**。核心实现位于 `arceos/modules/axalloc`、`arceos/modules/axmm`、`arceos/modules/axdma` 以及 `api/src/syscall/mm/`。

- 页分配器/字节分配器：`arceos/modules/axalloc/src/lib.rs#alloc_pages`、`dealloc_pages` 使用 `PAGE_SIZE` 布局，并通过 `balloc`/`palloc` 两级锁保护。
- 运行时初始化：`arceos/modules/axruntime/src/lib.rs#init_allocator` 从内核镜像后的空闲物理内存初始化全局分配器。
- 物理内存区域：`arceos/modules/axhal/src/mem.rs#init` 登记 RAM/MMIO/保留区。
- 地址空间后端：`arceos/modules/axmm/src/backend/cow.rs#alloc_new_at` 实现 COW backend，从文件读取并映射；`api/src/syscall/mm/mmap.rs#sys_mmap` 支持 `alloc/cow/file/linear/shared` 多种后端。
- DMA：`arceos/modules/axdma/src/dma.rs#dealloc_coherent` 提供一致性 DMA 内存分配/释放。

### PROC（进程管理/任务调度）

**传统进程管理有底层证据支撑，且与异步运行时深度耦合**。

- 进程抽象：`core/src/executor/src/executor.rs#new` 构造包含 `main_task`、`ProcessSignalManager`、`Scope`、`exe_path` 的执行器结构，相当于 PCB。
- 进程表/进程组：`core/src/task.rs#get_process_data`、`get_process_group`、`PROCESS_TABLE`、`PROCESS_GROUP_TABLE`。
- 生命周期：`api/src/syscall/task/clone.rs#sys_clone`、`api/src/syscall/task/execve.rs#sys_execve`、`api/src/syscall/task/wait.rs#sys_waitpid`、`api/src/task.rs#do_exit`、`api/src/syscall/task/exit.rs#sys_exit_group`。
- procfs：`api/src/vfs/proc.rs#lookup_child` 暴露 `stat/status/maps/mounts/cmdline/comm/exe/fd` 等节点。
- 信号：`core/src/executor/src/signal.rs#send_signal_to_process`、`send_signal_to_thread` 通过 `PID2PC`、`TID2TASK` 投递信号。

### 任务调度/异步运行时

**底层证据强**。包含 `asynctask`、`axtask`、`core/src/executor` 与 `vsched2` 扩展。

- 任务控制块：`arceos/modules/asynctask/src/task.rs#new` / `new_user` 维护 `TaskId`、原子状态、时间统计、`wait_wakers`。
- 运行队列/等待队列：`arceos/modules/axtask/src/run_queue.rs#exit_current`、`arceos/modules/axtask/src/wait_queue.rs#wait_timeout_until`。
- 调度器：`core/src/executor/src/executor.rs#pick_next_task` 调用 `KERNEL_SCHEDULER.lock().pick_next_task`。
- vDSO 虚拟调度：`core/src/vsched/trap.rs#new_handler`、`core/src/vsched/trap_vector.rs#vsched2_trap_entry_stub` 通过 `libvsched2::VDSO_VTABLE` 拦截/转发 trap。

### FS（文件系统）

**底层证据强**。

- 高层文件接口：`arceos/modules/axfs/src/highlevel/file.rs`（79 个函数）。
- ext4：`arceos/modules/axfs/src/fs/ext4/inode.rs#create`、`read_dir`、`metadata`、`append`、`link` 等。
- 虚拟文件系统：`api/src/vfs/tmp.rs`、`api/src/vfs/proc.rs`、设备节点 `api/src/vfs/dev/tty.rs`、`event.rs`、`loop.rs`。
- 系统调用：`api/src/syscall/fs/mount.rs#sys_mount`、`api/src/syscall/fs/fd_ops.rs#sys_openat`/`add_to_fd`、`api/src/syscall/fs/io.rs#sys_fallocate`/`sys_splice`、`api/src/syscall/fs/ctl.rs#sys_readlinkat`。
- fd 表按进程作用域隔离：`FD_TABLE.scope(&task.as_thread().proc_data.scope.read())`。

### DEV（设备驱动）

**底层证据中等，以框架枚举和关键控制器/网络设备为主**。

- 中断控制器：`arceos/modules/axdriver/src/dyn_drivers/intc/gicv2.rs#probe_gic`、`gicv3.rs#probe_gic`。
- 设备枚举：`arceos/modules/axdriver/src/lib.rs#init_drivers` 遍历 `block/display/input/net` 设备。
- 网络设备：`arceos/modules/axnet/src/device/ethernet.rs#process_arp`、`arceos/modules/axnet/src/device/vsock.rs#handle_vsock_event`。
- 块设备、显示设备、输入设备的具体驱动实现**底层证据未充分展开**，仅见注册与枚举。

### SYSCALL（系统调用）

**底层证据强**。

- 统一入口：`api/src/syscall/mod.rs#handle_syscall`（规模 23917，调用 217 处），根据 `Sysno` 分发。
- 覆盖：task、fs、mm、net、sync（futex）、signal、io_mpx（epoll/select/poll）、time。
- 用户指针访问：`vm_read`/`vm_write`/`vm_load_string` 等贯穿各 syscall。

### TRAP/IRQ（中断异常）

**底层证据较弱**（仅 14 个函数）。

- 信号返回/陷阱帧：`core/src/executor/src/signal.rs#load_trap_for_signal`、`signal_return`。
- 完整的通用中断分发表、PLIC/APIC、时钟中断驱动等**底层证据未体现**。

### NET（网络协议栈）

**底层证据强**。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块/能力

- **完整的中断控制器驱动与通用 IRQ 分发**：仅有 GICv2/v3 probe、`register_irq_hook`、RISC-V trap vector 等少量证据，缺少 PLIC/IO-APIC、中断亲和性、完整中断向量表。
- **时钟/定时器中断驱动底层实现**：`sys_setitimer`、`itimer` 有证据，但硬件 timer 初始化与 tick 中断处理未在提供的函数中体现。
- **块设备驱动具体实现**：`init_drivers` 枚举了 block 设备，但无具体块设备驱动（如 NVMe、SDHCI、AHCI）的实现证据。
- **显示/输入设备驱动具体实现**：仅见枚举与 devfs 接口（`event.rs`、`tty.rs`），缺少 GPU、键盘、鼠标等硬件驱动细节。
- **虚拟内存页错误完整处理链**：`vsched_trap_dispatcher` 中调用了 `handle_page_fault`、`page_table().query`，但页表遍历、缺页置换、swap 等细节未体现。
- **文件系统缓存、日志与崩溃恢复**：ext4 元数据操作有证据，但 buffer cache、 journaling、fsck 等未体现。
- **安全/权限子系统**：仅见 `mode/uid/gid` 设置，缺少 capability、SELinux/LSM、ACL 等证据。
- **电源管理**：仅见 `system_off`，缺少 suspend、cpufreq、idle 调度等。

### 需要单独说明的依赖与第三方代码

- **`arceos/` 目录**：大量文件属于 ArceOS 基础框架（模块、HAL、ulib），是本作品的**基础平台**，不是本次新增内核主体的全部。
- **`deps/` 目录**：`elf_parser`、`vqueue_vdso` 等为公共依赖/第三方/自研依赖库，用于 ELF 加载、vDSO 队列等，不应直接等同于原创内核主体。
- **`vdso_output`、`vdso_vsched2_output`、`vdso_vqueue_output`**：属于 vDSO 生成产物或绑定层，是用户态/内核态交互接口，而非内核核心实现。

### 判断边界

- 本文档所有方向判断（“基于 ArceOS”、“Linux ABI 兼容”、“vDSO 虚拟调度”等）均来自目录结构、AST Call Map 与函数名，**仅为可能的技术方向，不代表最终身份或原创性结论**。
- 不做抄袭、违规或原创性判定。
