## 第四部分：分模块架构说明

### MM（内存管理）

- **物理内存分配**：`arceos/modules/axalloc/src/lib.rs#alloc` 提供全局分配器，启动时由 `arceos/modules/axruntime/src/lib.rs#init_allocator` 调用 `axalloc::global_init` 与 `global_add_memory` 初始化。
- **虚拟地址空间**：`arceos/modules/axmm/src/aspace.rs#try_clone` 维护 `MemoryArea`、`Backend`、`PageIterWrapper`，支持地址空间复制；`arceos/modules/axmm/src/backend/alloc.rs#handle_page_fault_alloc` 实现按需分页，调用 `alloc_frame` 与 `pt.map`。
- **页错误处理**：四架构 trap handler（`x86_trap_handler`、`riscv_trap_handler`、`loongarch64_trap_handler`、`handle_sync_exception`）均调用 `handle_page_fault`。
- **系统调用**：`api/src/imp/mm/mmap.rs#sys_mmap` 处理 `mmap`/`munmap`/`mprotect`；`api/src/interface/mm/shm.rs#sys_shmget` 提供 SysV 共享内存接口。

### PROC（进程管理/任务调度）

- **传统进程抽象**：`process/src/process.rs`、`process/src/thread.rs`、`process/src/process_group.rs`、`process/src/session.rs` 维护 `PROCESS_TABLE`、`THREAD_TABLE`、`PROCESS_GROUP_TABLE`、`SESSION_TABLE`，支持 PID/TID、父子关系、进程组、会话、zombie、reaper。
- **生命周期系统调用**：
  - `api/src/imp/task/clone.rs#sys_clone_impl_from_trap_frame` 实现 `clone`/`fork`，复制/共享 FD_TABLE、FS_CONTEXT、CURRENT_DIR、地址空间。
  - `api/src/imp/task/execve.rs#sys_execve_impl` 实现 `execve`，关闭 `close-on-exec` 文件、卸载用户态地址空间。
  - `api/src/imp/task/exit.rs#sys_exit_impl` 处理线程退出、`clear_child_tid`、信号。
  - `api/src/imp/task/wait.rs#sys_wait4` 处理 `wait4`、zombie 回收。
- **调度与同步**：`arceos/modules/axtask/src/run_queue.rs#init` 初始化运行队列与 idle 任务；`arceos/modules/axtask/src/wait_queue.rs` 提供 `wait`/`wait_timeout`/`notify_one`/`requeue`；`api/src/imp/task/schedule.rs#sys_sched_getaffinity` / `sys_sched_setaffinity` 提供 CPU 亲和性。
- **信号与 futex**：`api/src/imp/task/signal.rs` 实现 `rt_sigprocmask`、`rt_sigtimedwait`、进程/进程组信号发送；`api/src/imp/task/futex.rs#sys_futex` 实现 futex 等待/唤醒。

### FS（文件系统）

- **高层文件接口**：`arceos/modules/axfs-ng/src/api/open.rs#open`、`arceos/modules/axfs-ng/src/highlevel/file.rs#_open`、`arceos/modules/axfs-ng/src/api/path.rs#resolve_path_existed` 负责路径解析、打开、权限检查。
- **VFS 层**：`modules/vfs/src/node/dir.rs#rename` 等实现通用 inode/directory 操作。
- **具体文件系统**：
  - `api/src/core/fs/imp/tmp.rs#new` / `src/fs/imp/tmp.rs#new`：tmpfs。
  - `src/fs/imp/proc.rs#builder`、`api/src/core/fs/imp/proc1/mod.rs#builder`：procfs，暴露 `/proc/PID`、`stat`、`status` 等。
  - `modules/lwext4_rust/c/lwext4/src/ext4.c` 等：第三方 ext4 实现（C），通过 Rust 封装 `modules/lwext4_rust/src/inode/file.rs#read_at` 接入。
- **文件描述符与 I/O**：`api/src/imp/fs/fd_ops.rs#sys_fcntl`、`api/src/imp/fs/ctl.rs#sys_ioctl`、`api/src/interface/fs/io.rs#sys_readv/writev/preadv/pwritev`、`api/src/interface/fs/poll.rs#sys_select_`、`api/src/interface/fs/epoll.rs#sys_epoll_wait` 等实现 fd 管理、读写、轮询。

### DEV（设备驱动）

- **驱动枚举**：`arceos/modules/axdriver/src/lib.rs#init_drivers` 探测并枚举 block、net、display 设备。
- **平台启动**：`arceos/tools/raspi4/chainloader/src/bsp/device_driver/bcm/bcm2xxx_pl011_uart.rs#init` 初始化树莓派 PL011 UART；`arceos/tools/raspi4/chainloader/src/main.rs#kernel_main` 负责链式加载。
- **网络原始设备**：`arceos/tools/bwbench_client/src/device.rs#new` 使用 Linux raw socket 做测试，属于工具代码而非内核驱动。
- **内核内部驱动证据有限**：未看到自定义 NVMe/SD/eMMC/AHCI/DMA 控制器、GPU 驱动等底层实现。

### SYSCALL（系统调用）

- **分发入口**：`src/syscall.rs#handle_syscall` 规模较大，依据系统调用号分发。
- **实现层**：`api/src/imp/*` 与 `api/src/interface/*` 提供 Linux 兼容实现，覆盖 fs、net、mm、task、signal、futex、epoll/select/poll。
- **架构入口**：各架构 trap handler 通过 `crate::trap::handle_syscall` 进入分发。

### TRAP/IRQ（中断异常）

- **异常处理**：x86_64、aarch64、riscv64、loongarch64 均有 trap handler，处理同步异常、页错误、断点、系统调用。
- **中断控制**：
  - x86_64：`arceos/modules/axhal/src/arch/x86_64/idt.rs#new` 初始化 IDT。
  - riscv64：`arceos/modules/axhal/src/arch/riscv/mod.rs#set_trap_vector_base` 设置 stvec；`arceos/modules/axhal/src/platform/riscv64_qemu_virt/irq.rs#init_percpu` 开启外部/软件/定时器中断。
  - loongarch64：`arceos/modules/axhal/src/platform/loongarch64_qemu_virt/irq.rs#set_enable` 配置 ecfg.lie。
- **IRQ 注册**：`arceos/modules/axhal/src/irq.rs#register_handler_common` 维护 `IRQ_HANDLER_TABLE`。

### NET（网络协议栈）

- **基于 smoltcp**：`arceos/modules/axnet/src/smoltcp_impl/mod.rs#init` 初始化接口、IP、网关；`tcp.rs#connect/bind/send/recv`、`udp.rs#bind`、`dns.rs#query` 均通过 `SOCKET_SET` 操作 smoltcp socket。
- **Socket 系统调用**：`api/src/imp/net/socket.rs` 实现 `accept`、`recvfrom`、`sendto`、`getaddrinfo`、`setsockopt`/`getsockopt` 等。
- **地址转换**：`api/src/imp/net/socketaddr.rs` 处理 `sockaddr_in`/`sockaddr_in6` 与 Rust `SocketAddr` 互转。

## 第六部分：边界与未体现内容

- **未体现虚拟化/ hypervisor**：无 KVM、VMM、vCPU、EPT/VMID 等相关证据。
- **未体现安全子系统**：无 LSM、capabilities、seccomp、ACL、审计等安全框架证据。
- **未体现高级电源/ACPI/热管理**：仅有基本中断使能，无 ACPI、CPU 频率、休眠等代码。
- **设备驱动深度有限**：`axdriver` 主要做设备枚举；除 PL011 UART 初始化外，未看到自定义 NVMe、SD/eMMC、AHCI、USB、DMA、GPU 等控制器驱动实现。
- **未体现实时调度**：仅有 `sched_getaffinity`/`sched_setaffinity`，无 `SCHED_FIFO`/`SCHED_RR` 等实时调度策略证据。
- **未体现内核跟踪/性能分析子系统**：`syscall_trace` 目录仅 1 个文件，证据不足；无 ftrace、perf、kprobe 等。
- **网络栈为 smoltcp 封装**：TCP/IP 协议核心来自第三方 smoltcp，非自研完整协议栈。
- **ext4 来自第三方 C 库**：`modules/lwext4_rust/c/lwext4` 为 lwext4 第三方实现，不应视为原创文件系统核心。
- **“可能的系统方向”仅为基于目录与调用链的推断**，不代表最终作品身份、功能完整度或原创性结论。
- 本文档不做抄袭/违规判断。
