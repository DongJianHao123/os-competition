## 第四部分：分模块架构说明

### MM（内存管理）

底层证据显示实现了基于页表的虚拟内存管理。关键锚点：
- `os/src/mm/memory_set.rs#MemorySet::from_elf` / `#map_elf`：ELF 加载与段映射。
- `os/src/mm/memory_set.rs#mmap` / `#munmap`：用户态 `mmap/munmap` 的 VMA 增删。
- `os/src/mm/memory_set.rs#MemorySet::from_existed_user`：`fork` 时复制/共享地址空间。
- `os/src/mm/memory_set.rs#load_dl_interp_if_needed`：动态链接器 `.interp` 段处理。
- `os/src/task/task.rs#dealloc_user_res`：回收用户栈、trap 上下文等。
- `os/src/syscall/mem.rs#sys_shmget` / `#sys_shmat`：System V 共享内存接口。

此外，`os/src/hal/trap/mod.rs#trap_handler` 中读取 `badv` 并处理页错误/地址异常，说明具备运行时缺页处理路径。TLB 相关处理由 `os/src/hal/trap/mod.rs#tlb_refill_handler` 提供。

### PROC（进程管理/任务调度）

底层证据支持**传统进程管理**，而非仅轻量级任务。关键锚点：
- `os/src/task/process.rs#fork` / `#exec` / `#new`：PCB 创建、复制、ELF 替换。
- `os/src/task/mod.rs#exit_current_and_run_next`：进程/线程组退出与清理。
- `os/src/syscall/process.rs#sys_fork` / `#sys_exec` / `#sys_exit` / `#sys_exit_group` / `#sys_futex` / `#sys_kill`。
- `user/src/bin/initproc.rs#main`：用户态 init 使用 `fork` / `waitpid`。
- `os/src/task/manager.rs#insert_into_pid2process` / `#remove_from_pid2process` / `#pid2process`：全局 PID 映射。
- `os/src/task/processor.rs#run_tasks` / `#schedule` / `#__switch`：调度循环与上下文切换。

**任务调度**为基于 `PROCESSOR` / `TASK_MANAGER` 的协作/抢占式调度；`os/src/hal/trap/mod.rs#timer_handler` 通过 `suspend_current_and_run_next` 实现定时器抢占。**异步运行时底层证据未体现**（未见 `waker/async task/executor` 等结构）。

### FS（文件系统）

文件系统呈现分层结构：内核 VFS/封装层 + 第三方 ext4 实现。关键锚点：
- `os/src/fs/mod.rs#open` / `#create_file`：文件打开/创建，涉及 `FileClass`、`OSInode`、`MNT_TABLE`。
- `os/src/fs/ext4_lw/inode.rs#read_all`：ext4 inode 读接口。
- `os/src/fs/devfs.rs#open_device_file`：`/dev/null`、`/dev/zero`、`/dev/random`、`/dev/tty`、`/dev/rtc` 等字符设备。
- `os/src/fs/pipe.rs#make_pipe`：管道实现。
- `os/src/fs/mod.rs#create_proc_dir_and_file` / `#remove_proc_dir_and_file`：procfs 进程目录。
- `os/src/syscall/fs.rs#sys_mount`：挂载表操作。

ext4 主体实现位于 `os/libs/lwext4_rsut/`，属于第三方库。

### DEV（设备驱动）

设备驱动证据集中在 VirtIO MMIO 与字符设备。关键锚点：
- `os/src/drivers/virtio/blk.rs#new` / `#read_block`：VirtIO 块设备。
- `os/src/drivers/virtio/net.rs#new`：VirtIO 网卡设备。
- `os/src/net/socket/mod.rs#init` / `#new`：网络接口初始化，封装 VirtIO-net 为 smoltcp 设备。
- `os/src/fs/devfs.rs#open_device_file`：各类伪设备文件。

### SYSCALL（系统调用）

存在中央系统调用分发器。关键锚点：
- `os/src/syscall/mod.rs#syscall`：114 分支，分发到 `sys_*`。
- `os/src/syscall/fs.rs#sys_read` / `#sys_write` / `#sys_open` / `#sys_pipe` / `#sys_pselect6`。
- `os/src/syscall/net.rs#sys_sendto` / `#sys_recvfrom`。
- `os/src/syscall/process.rs#sys_exec` / `#sys_fork` / `#sys_futex`。
- `os/src/syscall/sync.rs#sys_mutex_create` / `#sys_condvar_create` / `#sys_semaphore_create`。
- `os/src/syscall/mem.rs#sys_shmget` / `#sys_shmat`。

### TRAP/IRQ（中断异常）

中断/异常处理代码量小但锚点明确，且大量出现 LoongArch CSR（`crmd`、`estat`、`era`、`badv`、`eentry`、`stvec`、`tcfg`、`ticlr`、`ecfg`、`sie` 等），说明目标架构为 LoongArch64。关键锚点：
- `os/src/hal/trap/mod.rs#trap_handler`：用户态陷阱总入口，处理系统调用、定时器、异常、信号。
- `os/src/hal/trap/mod.rs#timer_handler`：定时器中断，触发调度。
- `os/src/hal/trap/mod.rs#tlb_refill_handler`：TLB 重填。
- `os/src/hal/trap/mod.rs#trap_handler_kernel` / `#trap_from_kernel`：内核态异常。
- `os/src/hal/trap/mod.rs#set_kernel_trap_entry` / `#set_user_trap_entry` / `#enable_timer_interrupt`。

### NET（网络协议栈）

网络协议栈为**内核 socket 封装层 + 第三方 smoltcp**。关键锚点：
- `os/src/net/socket/tcp.rs#connect` / `#recv` / `#send` / `#shutdown`。
- `os/src/net/socket/udp.rs#bind`。
- `os/src/net/socket/listen_table.rs#accept`。
- `os/src/net/socket/loopback.rs#snoop_tcp_from_ip`：回环 TCP 包处理。
- `os/src/syscall/net.rs#sys_sendto` / `#sys_recvfrom`。

`smoltcp` 位于 `os/libs/smoltcp/`，提供 TCP/UDP/IPv4/IPv6/ICMP 等协议实现；内核代码主要完成 socket 抽象、地址转换、阻塞/非阻塞调度、与文件描述符表集成。

## 第六部分：边界与未体现内容

1. **第三方库边界**  
   - 网络协议栈核心实现为 `os/libs/smoltcp/`，内核仅提供 socket 抽象与调度集成。  
   - ext4 文件系统核心实现为 `os/libs/lwext4_rsut/`，内核仅提供 Rust 绑定、VFS 封装与缓存层。  
   - `smoltcp/examples/` 下的 ping/server/httpclient 等示例程序不属于内核运行时。

2. **没有底层证据支撑的模块/特性**  
   - **SMP/多核调度**：证据中仅见单处理器 `PROCESSOR` 与 `TASK_MANAGER`，未见 CPU 局部结构、核间中断、自旋锁适配多核等。  
   - **异步运行时**：未见 `waker`、`async fn`、`executor`、`future` 等结构，实际为传统任务调度。  
   - **内核级 IPv6 socket API**：`smoltcp` 包含 IPv6 协议解析，但内核 socket 封装层锚点主要使用 IPv4（`Ipv4Addr`、`SocketAddrV4`）。  
   - **原创 TCP/IP 与 ext4 实现**：协议与文件系统主体来自第三方库，不能判定为项目原创实现。  
   - **安全/权限模块**：仅见 `uid/gid/euid` 等基础字段，未见 MAC、ACL、Capabilities、SELinux 等。  
   - **图形/显示/音频/USB/NVMe/PCI 等驱动**：底层证据未体现。  
   - **虚拟化/ hypervisor / 容器隔离**：底层证据未体现。  
   - **实时调度策略**：仅见基础优先级与定时器，未见 RT-scheduler、EDF、优先级继承等。  
   - **内核级调试/追踪子系统**：仅见 `backtrace` 工具函数，未见 Kprobe、Tracepoint、perf 等。

3. **判断性质说明**  
   - 本报告中“可能的系统方向”“面向 LoongArch64”等判断均基于 CSR 名称、目录结构、调用链等底层证据的保守推断，**不是最终身份或原创性结论**。  
   - 本报告不做抄袭/违规判断，也不对代码质量、性能、完整性进行评分。
