## 第四部分：分模块架构说明

### MM（内存管理）

**已体现内容**：
- 用户态地址空间（`aspace`）与多种后端：`alloc`、`cow`、`file`、`linear`、`shared`。
- COW 后端文件回填：`kernel/src/mm/aspace/backend/cow.rs#alloc_new_at` 调用 `file.read_at`、`phys_to_virt`。
- ELF 加载器：`kernel/src/mm/loader.rs#load_user_app`、`load` 调用 `ELF_LOADER.lock().load`。
- `mmap`/`mprotect`/`shmat`：`kernel/src/syscall/mm/mmap.rs#sys_mmap`、`sys_mprotect`；`kernel/src/syscall/ipc/shm.rs#sys_shmat`。
- 用户指针安全访问：`kernel/src/mm/access.rs` 提供 `vm_read`/`vm_write`。

**未直接体现**：页故障（page fault）处理例程、TLB 刷新、物理页分配器（buddy/slab）底层实现、架构相关 MMU 初始化。

### PROC（进程管理/任务调度）

**已体现内容**：
- 传统进程管理证据充分：进程表 `PROCESS_TABLE`、任务表 `TASK_TABLE`、进程组/会话表 `PROCESS_GROUP_TABLE`/`SESSION_TABLE`。
- 生命周期：`do_clone`、`sys_fork`、`sys_execve`、`sys_waitpid`、`do_exit`、`sys_exit`/`sys_exit_group`。
- 进程关系：`sys_getpid`、`sys_getppid`、进程组信号 `send_signal_to_process_group`。
- `pidfd`：`kernel/src/syscall/fs/pidfd.rs#sys_pidfd_open`、`sys_pidfd_getfd`、`sys_pidfd_send_signal`。
- 信号：`sys_rt_sigprocmask`、`sys_rt_sigaction`、`sys_kill`、`send_signal_to_process`。
- 调度相关：`axtask::yield_now`、`sys_sched_setaffinity`、`set_current_affinity`、定时器任务 `alarm_task`。

**未直接体现**：具体调度算法（CFS/Round-Robin 等）的底层实现、上下文切换汇编、CPU 本地存储细节。

### FS（文件系统）

**已体现内容**：
- VFS 高层：`axfs-ng-patched/src/highlevel/file.rs#open` 路径解析、符号链接处理、权限检查。
- ext4 实现：`axfs-ng-patched/src/fs/ext4/inode.rs#create`、`read_dir`、`metadata`、`append`、`link` 等。
- 伪文件系统：`kernel/src/pseudofs/proc.rs`（procfs）、`dev/`（devfs，含 tty/pty/event/loop/random/display）、`tmp.rs`（tmpfs）。
- 挂载：`sys_mount` 支持内存文件系统、块设备文件系统。
- 文件描述符表：`FD_TABLE`、`add_to_fd`、`sys_openat`、`sys_close_range`、`sys_fcntl`。
- 高级 I/O：`sys_splice`（pipe/直接发送）、`sys_readlinkat`、`sys_truncate`、`sys_fallocate`、`sys_memfd_create`。

**未直接体现**：块设备驱动本身（仅通过 `axfs::get_blkdev_fs` 抽象使用）、日志文件系统（journaling）底层、磁盘缓存写回策略。

### DEV（设备驱动）

**已体现内容**：
- 字符设备抽象：`kernel/src/pseudofs/dev/mod.rs#builder` 创建 random、display、event、tty 等。
- TTY/PTY：`kernel/src/pseudofs/dev/tty/mod.rs#ioctl`、`terminal/ldisc.rs`、`pty.rs#create_pty_pair`。
- 输入事件设备：`kernel/src/pseudofs/dev/event.rs#ioctl`。
- loop 设备：`kernel/src/pseudofs/dev/loop.rs#ioctl`。
- 网络设备：`axnet-ng-patched/src/device/ethernet.rs#handle_frame`/`send`、`axnet-ng-patched/src/device/vsock.rs`。
- 块设备通过 `axfs` 的 `blkdev_fs` 封装使用。

**未直接体现**：具体硬件寄存器驱动（如 PCIe、DMA、中断控制器）、真实磁盘控制器驱动、显示/输入硬件的底层 HAL。

### SYSCALL（系统调用）

**已体现内容**：
- 统一入口：`kernel/src/syscall/mod.rs#handle_syscall`（217 条调用，规模 23834）。
- 子系统覆盖：文件（`fs/`）、网络（`net/`）、进程/线程（`task/`）、内存（`mm/`）、IPC（`ipc/`）、信号（`signal.rs`）、时间（`time.rs`）、资源（`resources.rs`）、I/O 多路复用（`io_mpx/`）、同步（`sync/futex.rs`）。
- Linux 错误码兼容：`LinuxError::ENOSYS`、`LinuxError::from(err).code()`。

**未直接体现**：系统调用 ABI 的汇编入口/退出路径、用户态/内核态切换的架构细节。

### TRAP/IRQ（中断异常）

**已体现内容**：在提供的证据中，该模块仅有 3 个函数且多为信号/终端相关辅助函数，如 `kernel/src/file/signalfd.rs#from_signal_info`、`kernel/src/pseudofs/dev/tty/terminal/termios.rs#signo_for`/`default`。

**结论**：**底层证据未体现** 架构相关的中断描述符表（IDT）、异常向量表、IRQ 分发、时钟中断处理、页故障处理例程。

### NET（网络协议栈）

**已体现内容**：
- 协议覆盖：TCP、UDP、Unix Domain Socket、Vsock。
- 核心文件：`axnet-ng-patched/src/tcp.rs`、`udp.rs`、`unix/mod.rs`、`vsock/`。
- 网络设备：`axnet-ng-patched/src/device/ethernet.rs`（ARP、帧收发）、`device/vsock.rs`。
- 路由：`axnet-ng-patched/src/router.rs#dispatch`、`snoop_tcp_packet`。
- 监听表：`axnet-ng-patched/src/listen_table.rs#accept`。
- 异步模型：`axnet-ng-patched/src/service.rs#register_waker` 使用 waker 轮询。

**未直接体现**：完整 DHCP/ICMP、硬件网卡驱动、TCP 拥塞控制算法细节、网络缓冲区与 DMA 交互。
