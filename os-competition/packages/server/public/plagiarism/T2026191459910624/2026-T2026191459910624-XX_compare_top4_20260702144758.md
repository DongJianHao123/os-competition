## 4. 核心代码调用流对比剖析

### 【进程管理 / 进程退出路径】

- **底层技术析理**: 
  目标项目进程退出路径为 `sys_exit` → `do_exit`，`do_exit` 内部执行 `FutexKey::new_current`（futex 唤醒）、`SignalInfo::new_kernel`（信号发送）、`clear_child_tid.vm_write(0)`（清除子线程 ID）、`SHM_MANAGER.lock().clear_proc_shm`（共享内存清理）、`axtask::yield_now`（调度让出）。参考项目退出路径为 `sys_exit` → `sys_exit_impl`，内部执行 `SignalInfo::new`（信号发送）、`addr_clear_child_tid.get`（读取清除地址）、`close_all_file_like`（关闭文件）、`axtask::exit` + `axtask::yield_now`（任务退出）。两者在**清理顺序、资源回收范围、同步原语选择**上均不同：目标清理共享内存（`SHM_MANAGER`），参考关闭所有文件描述符（`close_all_file_like`）；目标使用 `vm_write` 主动写入，参考使用 `get` 读取地址。

- **代码溯源链接**:

  - 参考项目: `api/src/imp/task/exit.rs#sys_exit_impl`

### 【内存管理 / mmap 系统调用】

- **底层技术析理**:
  目标项目 `sys_mmap` 使用 `Backend` 枚举多态架构，根据映射类型分发到 `Backend::new_alloc`（匿名映射）、`Backend::new_cow`（写时复制）、`Backend::new_file`（文件映射）、`Backend::new_linear`（线性映射）、`Backend::new_shared`（共享内存映射），地址对齐使用 `(addr + length).align_up`。参考项目 `sys_mmap` 使用 `MmapFlags::from_bits_truncate` + `MmapProt::from_bits_truncate` 位标志解析，通过 `PhysAddr::from` 进行物理地址转换，后端统一处理而非多态分发。两者在**内存后端抽象层次**（多态枚举 vs 位标志解析）和**地址空间管理模型**上存在根本差异。
