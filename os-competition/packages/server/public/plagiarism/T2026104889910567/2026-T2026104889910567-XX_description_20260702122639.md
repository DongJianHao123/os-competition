## 第四部分：分模块架构说明

### MM（内存管理）

底层证据较强。核心实现包括：

- **物理页分配**：`src/mm/buddy.c` 提供伙伴系统，`buddy_alloc_nolock`、`buddy_free`、`split_blocks`、`merge_blocks` 构成主要路径。
- **对象缓存**：`src/mm/slab.c` 提供 slab 分配器，`kmem_cache_alloc_nolock`、`kmem_cache_init`、`slab_init`、`slab_free` 依赖 `buddy_alloc`。
- **内核页表**：`src/mm/vm.c#kvminit` 创建内核页表并映射设备内存。
- **进程地址空间**：`sys_mmap`、`sys_brk`、`load_elf_segments`、`uvmcopy`、`pagetable_destroy` 等涉及 `mappages`、`kzalloc_page`、`kfree_page`。
- **用户态拷贝**：`copyin`、`copyout`、`copyinstr` 在系统调用中广泛使用。

未体现：swap、页替换算法、COW（除 `uvmcopy` 的显式页拷贝外未见 COW 缺页处理）等高级特性。

### PROC（进程管理/任务调度）

底层证据支持**传统进程模型**，而非轻量级协程/异步运行时：

- **PCB 生命周期**：`src/task/proc.c#alloc_proc`、`free_proc`、`src/task/kthread.c#init_user_process`、`kthread_create`、`kthread_exit`。
- **fork/exec/wait/exit**：`src/syscall/proc.c#sys_clone`、`sys_execve`、`sys_wait4`、`sys_exit`。
- **地址空间/文件表继承**：`fd_table_dup`、`uvmcopy`。
- **调度**：`src/task/sched.c#enqueue_proc`、`dequeue_proc`、`context_switch_yield`、`sched_yield`、`wakeup`、`sleep`，基于链表运行队列和自旋锁。

未体现：完整 SMP 调度、调度类/CFS、优先级调度、信号完整实现、僵尸进程处理细节。

### FS（文件系统）

证据强，呈现类 Unix VFS 结构：

- **VFS 层**：`src/fs/vfs.c#vfs_open`、`vfs_mount_root`、`vfs_mount`、`vfs_unlink`、`vfs_create`、`vfs_mkdir`、`vfs_rename`。
- **路径解析**：`src/fs/namei.c#vfs_path_lookup`、`vfs_path_walk`、`vfs_path_parent`、`vfs_lookup_single`。
- **dentry/inode 缓存**：`src/fs/dentry.c#dentry_alloc`、`dentry_free`；`src/fs/inode.c#inode_alloc`、`inode_get`。
- **具体文件系统**：`src/fs/ramfs.c`（内存文件系统）、`src/fs/ext4_ops.c` / `ext4_dir.c` / `ext4_file.c` / `ext4_inode.c` / `ext4_sb.c`（Ext4）。
- **块缓存与管道**：`src/mm/bcache.c`、`src/ipc/pipe.c`。

注意：`src/misc/lz4.c`、`src/misc/sha2.c`、`src/misc/printf.c`、`src/misc/string.c` 等属于通用工具或第三方/公共域代码；`ext4` 系列代码需单独评估其原创性，不宜直接视为内核原创主体。

### DEV（设备驱动）
