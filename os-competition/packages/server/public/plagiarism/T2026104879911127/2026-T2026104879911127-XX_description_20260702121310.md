## 第四部分：分模块架构说明

### MM（内存管理）
底层证据较强。核心实现位于 `arceos/modules/axalloc/`、`arceos/modules/axmm/`、`arceos/modules/axdma/` 以及 `core/src/mm.rs`。
- 页帧分配：`arceos/modules/axalloc/src/lib.rs#alloc_pages` / `#dealloc_pages` 使用 `palloc`/`balloc` 两级锁保护。
- 启动初始化：`arceos/modules/axruntime/src/lib.rs#init_allocator` 调用 `axalloc::global_init` 与 `global_add_memory`。
- 用户空间内存：`api/src/syscall/mm/mmap.rs#sys_mmap` 支持 `alloc/cow/file/linear/shared` 多种后端。
- COW 后端：`arceos/modules/axmm/src/backend/cow.rs#alloc_new_at` 按需分配页帧并从文件读取数据。
- DMA：`arceos/modules/axdma/src/dma.rs#dealloc_coherent` 提供一致性 DMA 内存分配/释放。
- ELF 加载：`core/src/mm.rs#load_user_app` / `#load` 负责用户程序加载。

### PROC（进程管理/任务调度）
进程管理有底层证据但函数量偏少，任务调度/异步运行时证据较强。
- 进程表/线程表：`core/src/task.rs#add_task_to_table` 操作 `PROCESS_TABLE`、`PROCESS_GROUP_TABLE`、`TASK_TABLE`。
- 进程生命周期：`api/src/syscall/task/clone.rs#sys_clone`、`api/src/syscall/task/execve.rs#sys_execve`、`api/src/task.rs#do_exit`、`api/src/syscall/task/wait.rs#_try_reap_child`。
- 进程组/会话/信号：`core/src/task.rs#send_signal_to_process`、`#send_signal_to_process_group`、`#kill_all_non_init_processes`。
- procfs：`api/src/vfs/proc.rs#lookup_child` / `#child_names` / `#new_procfs` 暴露 `/proc/[pid]/stat|status|maps|fd|...`。
- 任务调度：`arceos/modules/axtask/src/run_queue.rs#exit_current`、`arceos/modules/axtask/src/task.rs`（54 函数）、`arceos/modules/axtask/src/wait_queue.rs`。
- 注意：`arceos/ulib/axlibc/c/` 中的 `fork`、`execve`、`waitpid`、`wait3`、`pthread_cond_wait` 等 C 包装当前为 `unimplemented`，说明用户态 C 库尚未完全对接。

### FS（文件系统）
证据最强。实现集中在 `arceos/modules/axfs/` 与 `api/src/vfs/`、`api/src/syscall/fs/`。
- 高层文件接口：`arceos/modules/axfs/src/highlevel/file.rs`（80 函数）。
- ext4：`arceos/modules/axfs/src/fs/ext4/inode.rs#create` / `#read_dir` / `#metadata` / `#update_metadata` / `#append` / `#link`；`arceos/modules/axfs/src/fs/ext4/fs.rs#new`。
- 虚拟文件系统：`api/src/vfs/tmp.rs`、`api/src/vfs/proc.rs`、`api/src/vfs/dev/`（tty、event、loop）。
- 系统调用：`sys_openat`、`sys_mount`、`sys_readlinkat`、`sys_fcntl`、`sys_splice`、`sys_fallocate` 等。
- Page Cache：`arceos/modules/axfs/src/highlevel/sync.rs#sync_page_caches` 操作 `PAGE_CACHE_REGISTRY`。

### DEV（设备驱动）
证据中等，主要覆盖中断控制器与网络设备初始化。
- 中断控制器：`arceos/modules/axdriver/src/dyn_drivers/intc/gicv2.rs#probe_gic`、`gicv3.rs#probe_gic`。
- 驱动枚举：`arceos/modules/axdriver/src/lib.rs#init_drivers` 遍历 block/display/input/net 设备。
- 网络设备：`arceos/modules/axnet/src/device/ethernet.rs#send` / `#process_arp`。
- 块设备/显示/输入设备仅有枚举调用链，具体驱动实现底层证据未充分展开。

### SYSCALL（系统调用）
证据强。统一入口为 `api/src/syscall/mod.rs#handle_syscall`（规模 23436，调用 210）。
- 覆盖 fs、net、mm、task、signal、sync（futex）、ipc（shm）、io_mpx（select/poll/epoll）、time。
- 关键函数：`api/src/syscall/fs/mount.rs#sys_mount`、`api/src/syscall/fs/fd_ops.rs#sys_openat`、`api/src/syscall/mm/mmap.rs#sys_mmap`、`api/src/syscall/task/clone.rs#sys_clone`、`api/src/syscall/net/socket.rs#sys_socket`、`api/src/syscall/sync/futex.rs#sys_futex`。

### TRAP/IRQ（中断异常）
**底层证据较弱**。仅有：
- `arceos/modules/axhal/src/irq.rs#register_irq_hook`：注册单一 IRQ hook。
- `api/src/time.rs#inc_irq_cnt` / `#irq_cnt`：IRQ 计数。
- `api/src/file/signalfd.rs#from_signal_info`、`api/src/terminal/termios.rs#signo_for` 等信号/终端相关。
- 未见完整 trap 入口、异常向量表、页 fault 处理、系统调用陷入/返回的底层汇编证据。因此“完整中断/异常处理架构”底层证据未体现。

### NET（网络协议栈）
证据强。位于 `arceos/modules/axnet/`。
- TCP：`arceos/modules/axnet/src/tcp.rs#connect` / `#bind` / `#recv`。
- UDP：`arceos/modules/axnet/src/udp.rs#send` / `#recv`。
- Unix Domain Socket：`arceos/modules/axnet/src/unix/`、`arceos/modules/axnet/src/unix/dgram.rs#recv`。
- Vsock：`arceos/modules/axnet/src/vsock/connection_manager.rs#on_connection_request`、`arceos/modules/axnet/src/vsock/stream.rs#accept`。
- 路由：`arceos/modules/axnet/src/router.rs#dispatch` / `#snoop_tcp_packet`。
- 监听表：`arceos/modules/axnet/src/listen_table.rs#accept`。

- **安全/加密/SELinux/Capability**：无相关函数或调用链证据。
- **实时调度策略**：仅见普通运行队列与 `sys_sched_setaffinity`，无 SCHED_FIFO/RR 等实时类证据。
- **完整 C 库对接**：`axlibc/c/` 中 `fork`、`execve`、`waitpid`、`wait3`、`pthread_cond_wait` 等标记为 `unimplemented`，说明用户态 C 程序兼容尚未完成。

### 2. 第三方库/公共依赖边界
- 项目大量代码位于 `arceos/` 目录，属于 ArceOS 模块化内核框架；`axfs/src/fs/ext4/` 可能基于 `ext4-rs` 等第三方 crate 实现。
- `axnet` 的 TCP/IP 协议状态机、包解析风格与 `smoltcp` 类似，可能在其之上封装。
- `arceos/ulib/axlibc/c/` 中的 `math.c`、`stdio.c`、`pow.c` 等属于 C 运行时/数学库移植，不应视为原创内核主体。
- `arceos/tools/` 下的 `deptool`、`raspi4/chainloader` 为构建工具与树莓派链式加载器，不属于内核核心。
