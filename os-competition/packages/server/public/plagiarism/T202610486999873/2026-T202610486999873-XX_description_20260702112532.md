## 第四部分：分模块架构说明

### MM（内存管理）
**证据较强**。核心代码位于 `os/src/mm/`：
- `os/src/mm/memory_set.rs#from_elf` / `from_elf_file`：按 ELF program header 建立用户地址空间映射；
- `os/src/mm/memory_set.rs#mprotect`：修改已有映射权限，涉及 `pte_permission_for_frame`、`VAddrRange`；
- 多处系统调用使用 `crate::mm::translated_byte_buffer`、`valid_user_buf` 进行用户态地址校验与拷贝；
- `os/src/syscall/process.rs#sys_shmat` 与 `sync_thread_group_vm_frames` 体现共享内存/线程组帧同步能力。

### PROC（进程管理/任务调度）
**证据较强**。核心结构为 `PID2TCB` 全局任务表与 `TaskControlBlock`：
- 进程创建：`os/src/syscall/process.rs#sys_clone` 调用 `current_task.fork`、`add_task`；
- 程序执行：`os/src/syscall/process.rs#sys_exec` -> `os/src/task/task.rs#exec_loaded` 重建地址空间与陷阱帧；
- 进程等待/回收：`sys_waitpid`、`sys_waitid`、`reap_exited_task`、`WAITPID_QUEUE`；
- 信号：`signal_task`、`sys_rt_sigtimedwait`、`sys_sigaction`、`sys_sigprocmask`；
- 线程组同步：`sync_thread_group_vm_frames`、`sync_thread_group_vm_area_metadata`；
- Futex/robust list：`futex_wake_key`、`futex_requeue_key`、`process_task_robust_list`；
- 凭证与调度：`sys_setreuid`、`sys_setresgid`、`sys_setpriority`、`sys_sched_getaffinity`、`sys_setpgid`、`sys_setsid`。

调度器可见：
- `os/src/task/processor.rs#run_tasks`：取任务、上下文切换 `context_switch_pt`、回收已退出任务；
- `os/src/task/mod.rs#wake_signal_wait_task`、`wake_poll_wait_tasks`、`wake_parent_waiter`：多类等待队列唤醒。

**异步运行时**：仅有 `block_current_and_run_next`、`block_current_until`、`net_poll` 等阻塞-让出原语，**完整 async/await runtime 的底层证据未体现**。

### FS（文件系统）
**证据较强，但核心实现为第三方库**：
- 自研 VFS 层：`os/src/fs/vfs.rs`（`VfsSuperBlock`、`ROOT_DENTRY`、`truncate_file`、`read_at`）、`os/src/fs/inode.rs`（`OSInode`、`open_file_new`、`write`、`write_at`、`stat_inner`、`getdents64`）、`os/src/fs/proc.rs`、`os/src/fs/stdio.rs`；
- 挂载层：`MOUNT_TABLE`、`sys_mount`、`sys_umount2`；
- 特殊文件：`DevBlock`、`DevZero`；
- 底层 ext4：`lwext4_rust/c/lwext4/src/` 下 600+ 函数，包括块分配、extent、目录索引、journal 等，**属于 lwext4 第三方库，非本项目原创文件系统主体**。

### DEV（设备驱动）
**证据较弱（仅 23 个函数）**：
- 块设备：`os/src/drivers/block/virtio_blk.rs#probe_all`、`new_at`，基于 `MmioTransport` 与 `VirtIOBlk`；
- 网络设备：`os/src/drivers/net/virtio_net.rs#receive`、`consume`，基于 VirtIO net 收发缓冲区；
- 抽象：`BLOCK_DEVICE`、`ROOT_BLOCK_DEVICE`。
- **非 VirtIO 设备、中断控制器、时钟、GPIO、显示、音频等驱动底层证据未体现**。

### SYSCALL（系统调用）
**证据较强**：
- 入口：`os/src/syscall/mod.rs#syscall`（164 条核心调用），记录最近系统调用参数并分发；
- 进程类：`os/src/syscall/process.rs`（166 函数）；
- 文件类：`os/src/syscall/fs.rs`（103 函数）；
- 网络类：`os/src/net/syscall.rs`（71 函数）。

### TRAP/IRQ（中断异常）
**底层证据未体现**。提供的 AST Call Map 中未出现 trap vector、exception handler、page fault handler、PLIC/CLINT/AIA 等中断控制器代码。仅见 `context_switch_pt`、`IrqRestoreGuard::disable` 等调度/关中断原语，以及 `polyhal` 抽象层调用。

### NET（网络协议栈）
**证据中等（36 个函数）**：
- Socket 抽象：`os/src/net/socket.rs`；
- 系统调用：`os/src/net/syscall.rs#sys_bind`、`sys_connect`、`sys_accept`、`sys_sendto`、`sys_recvfrom`、`sys_getpeername`、`sys_getsockname`；
- 本地队列：`LOCAL_TCP_CONNS`、`LOCAL_UDP_QUEUE`；
- 核心协议栈使用 smoltcp 类型（`TcpSocket`、`UdpSocket`、`IpEndpoint`、`Ipv4Address`），**原创 TCP/IP 协议实现底层证据未体现**。

## 第六部分：边界与未体现内容

- **TRAP/IRQ**：未出现 trap vector、异常分发、页故障处理、中断控制器（PLIC/CLINT/AIA/IOAPIC 等）代码；

- **多样化设备驱动**：仅见 VirtIO block/net 包装，未见非 VirtIO 块设备、网卡、字符设备、显示、音频、USB、NVMe 等驱动；
- **SMP/多核与底层锁**：未出现自旋锁、RCU、per-CPU、核间中断等证据；
- **安全/加密/审计/容器**：无相关底层证据。
