## 第四部分：分模块架构说明

### MM（内存管理）
- 内核级内存管理证据来自进程加载、页表切换与地址空间扩展：
  - `xv6-k210/kernel/exec.c#exec` 使用 `MAKE_SATP`、`PGROUNDUP`、`copyout` 构建用户页表并加载 ELF；
  - `xv6-k210/kernel/trap.c#usertrapret` 负责从内核态返回用户态时写 `satp`、`sepc`、`sstatus`；
  - `xv6-k210/kernel/sysproc.c#sys_sbrk` 通过 `growproc` 调整进程堆大小；
  - `xv6-k210/kernel/vm.c` 函数密集（32 个函数），但本次未提供详细调用链，具体页表遍历/映射实现需进一步审阅源码。
- `bootloader/SBI/rustsbi-*/src/main.rs#main` 中 `ALLOCATOR.lock().init` 属于 M-mode 固件堆初始化，不等同于内核虚拟内存管理。

### PROC（进程管理/任务调度）
- 具备完整的传统进程管理证据：
  - `xv6-k210/kernel/proc.c#fork` / `fork_at` / `wait` / `sched` / `sleep` / `wakeup` 覆盖 fork/exec/wait/PCB/阻塞唤醒；
  - `xv6-k210/kernel/proc.c#run_next_init_test` 负责 init 测试序列，调用 `exec`、`ext4_active`、`try_exec_busybox_command`；
  - `xv6-k210/kernel/proc.c#proc_on_timer_tick` / `pick_next_proc` / `mlfq_maybe_update_locked` / `proc_get_sched_algo` 表明实现了 MLFQ、优先级、RR、时间片等调度策略扩展；
  - `xv6-k210/kernel/sysproc.c` 暴露 `sys_set_priority`、`sys_set_timeslice`、`sys_set_sched_algo` 等接口。

### 任务调度/异步运行时
- **完整异步运行时（waker / task / executor / future）底层证据未体现**。
- 该模块统计中的函数主要是：
  - 传统 sleep/wakeup 阻塞调度（`console.c#consoleread`、`sleeplock.c#acquiresleep`、`file.c#fileread`）；
  - 系统调用分发（`syscall.c#linux_dispatch`）；
  - 自旋锁/中断开关（`spinlock.c#acquire`、`intr.c#pop_off`）。
- 因此应判定为“传统调度 + 阻塞 I/O 原语”，而非独立异步运行时。

### FS（文件系统）
- 采用 FAT32 + ext4 双文件系统结构，并带有 Linux ABI 兼容层：
  - `xv6-k210/kernel/fat32.c#fat32_init` 初始化时调用 `ext4_init`、`ext4_probe`；
  - `xv6-k210/kernel/fat32.c#dirlookup`、`lookup_path` 与 `xv6-k210/kernel/ext4.c#ext4_dirlookup`、`lookup_path` 交互；
  - `xv6-k210/kernel/ext4.c` 包含 extent、inode 读取、虚拟目录项（`virtual_create`、`virtual_lookup`）等实现；
  - `xv6-k210/kernel/sysfile.c#sys_open`、`sys_rename`、`sys_chdir` 提供原生 xv6 文件调用；
  - `xv6-k210/kernel/syscall.c#linux_sys_openat`、`linux_sys_newfstatat`、`linux_sys_getdents64`、`linux_sys_readv/writev`、`linux_sys_readlinkat` 等实现 Linux ABI。
- `ext4.c` 代码量大、逻辑与标准 ext2/4 解析高度相似，应作为第三方/移植代码单独评估。

### DEV（设备驱动）
- 存储与外设驱动证据较完整：
  - SD 卡：`xv6-k210/kernel/sdcard.c#sdcard_read_sector`、`sdcard_write_sector`、`sd_init`；
  - SPI 控制器：`xv6-k210/kernel/spi.c#spi_receive_data_normal_dma`、`spi_send_data_normal`；
  - DMA 控制器：`xv6-k210/kernel/dmac.c#dmac_set_single_mode`、`dmac_wait_idle`；
  - UART 控制台：`xv6-k210/kernel/uart.c#uartintr`、`uartstart`、`uartputc`；
  - GPIO：`xv6-k210/kernel/gpiohs.c#gpiohs_set_drive_mode`；
  - PLIC 中断路由：`xv6-k210/kernel/plic.c#plicinithart`；
  - Virtio 块设备：`xv6-k210/kernel/virtio_disk.c#virtio_disk_intr`。

### SYSCALL（系统调用）
- 双 ABI 设计：
  - 原生 xv6 系统调用通过 `xv6-k210/kernel/syscall.c#syscall` 查表 `syscalls[num]` 分发；
  - Linux ABI 通过 `is_linux_abi_proc` 判断后进入 `linux_dispatch`，再分发到 `linux_sys_*` 系列函数（如 `linux_sys_execve`、`linux_sys_wait4`、`linux_sys_brk`、`linux_sys_clone` 等）。

### TRAP/IRQ（中断异常）
- RISC-V 路径：
  - M-mode：`bootloader/SBI/rustsbi-*/src/main.rs#start_trap_rust` 处理机器态异常、读取 `mcause`/`mepc`、设置 CLINT 定时器；
  - S-mode：`xv6-k210/kernel/trap.c#trapinithart`、`usertrap`、`kerneltrap`、`devintr` 处理监管态异常、设备中断、PLIC 认领/完成；
- LoongArch 独立路径：
  - `la-minimal/main.c#trap_install`、`la_trap_handler` 仅演示 CSR 读写与 UART 输出，**与 `xv6-k210` 主内核的调用集成关系底层证据未体现**。

### NET（网络协议栈）
- **底层证据未体现**：未出现 socket、TCP/IP、网卡驱动、网络缓冲区等相关函数或文件。
