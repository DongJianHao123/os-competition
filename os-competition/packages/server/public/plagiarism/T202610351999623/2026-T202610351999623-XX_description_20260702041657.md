## 第四部分：分模块架构说明

### MM（内存管理）
- **证据强**。采用 RISC-V 分页：`kernel/src/arch/riscv64/mmu/paging.rs#init` 构建内核页表，`kernel/src/mm/pgtable.rs` 提供页表遍历。
- 进程地址空间由 `kernel/src/mm/address_space.rs` 管理，支持 `fork_cow`、`handle_cow_fault`、`mprotect_range`、`setup_tls`、`map_user_page`。
- 缺页异常处理在 `kernel/src/mm/fault.rs#handle_page_fault`，可处理 COW 缺页与 ELF 按需加载。
- 物理页帧分配在 `kernel/src/mm/frame`（`procfs::generate_meminfo` 调用 `available_frames`）。
- ELF 装载由 `kernel/src/loader/mod.rs#load_elf` 与 `kernel/src/loader/from_pages.rs#load_elf_from_pages` 完成。

### PROC（进程管理/任务调度）
- **传统进程管理证据充分**。`kernel/src/sched/exec.rs#fork_current`、`kernel/src/sched/lifecycle.rs#exit_current` / `waitpid_current`、`kernel/src/sched/manager.rs#spawn_user` 构成 fork/exec/exit/wait 生命周期。
- 进程控制块通过 `MANAGER.tasks[pid]` 数组管理，调度器维护可运行队列：`MANAGER.push_runnable`、`MANAGER.pop_next_runnable`、`MANAGER.schedule`。
- 进程组/会话：`sys_setpgid`、`sys_setsid`、`sys_getpgid`、`sys_getsid`、`current_process_group`。
- 信号：`send_signal`、`setup_signal_frame`、`sys_rt_sigprocmask`、`sys_rt_sigsuspend`。
- 定时器：`sys_setitimer`、`sys_timer_create/settime/gettime/delete`、`timerfd::process_timerfds`。
- 线程同步：`futex_wake_addr`、`sys_set_tid_address`、`clear_child_tid_and_wake`。
- **关于“异步运行时”**：代码中未出现 futures/executor/waker 等典型异步运行时结构，实际为“阻塞-唤醒”式调度器；`timerfd` 与 `ppoll` 等 waiter 机制属于同步阻塞 I/O 的等待原语。

### FS（文件系统）
- **证据强**。VFS 层：`kernel/src/fs/vfs.rs#init`、`kernel/src/fs/dir_ops.rs#vfs_open` / `vfs_getcwd`。
- 具体文件系统：
  - `ramfs`：根文件系统初始化。
  - `procfs`：`kernel/src/fs/procfs.rs` 提供 `/proc/[pid]/stat`、`/proc/meminfo`、`/proc/cpuinfo`、`/proc/mounts` 等。
  - `devfs`：`kernel/src/fs/devfs.rs` 提供设备节点与 `ioctl`。
  - `ext4`：`kernel/src/fs/ext4/` 实现 inode 操作、目录项、extent 管理、块分配。
- 管道 IPC：`kernel/src/ipc/pipe/blocking.rs#pipe_read_blocking` / `pipe_write_blocking`。
- **注意**：`kernel/src/fs/ext4/` 属于 ext4 磁盘格式协议强约束代码，部分函数可能为遵循标准布局的参考实现，应单独评估其原创性。

### DEV（设备驱动）
- **证据中等**。块设备：`kernel/src/drivers/virtio_blk/` 实现 MMIO virtio-blk 探测、读扇区、写扇区、描述符环 I/O。
- 中断控制器：`kernel/src/arch/riscv64/plic.rs#init` 初始化 PLIC。
- 字符设备：`kernel/src/arch/riscv64/uart.rs` 提供 UART 初始化、IRQ 处理、`try_getc`；`kernel/src/tty/driver.rs#raw_getc` 为 TTY 后端。
- 时钟：`kernel/src/arch/riscv64/trap/timer.rs` 处理 SBI timer 扩展、tick、定时器触发。

### SYSCALL（系统调用）
- **证据强**。统一入口：`kernel/src/syscall/dispatch.rs#dispatch` 根据 a7 分发到各子模块。
- 进程类：`kernel/src/syscall/process.rs`（fork/exec/exit/waitpid/kill/setpgid/itimer/timer/tid/futex 等）。
- 文件类：`kernel/src/syscall/file.rs`（openat/getdents64/fstat/fchmod/fchown/chdir/getcwd/lseek 等）。
- IO 类：`kernel/src/syscall/io.rs`（readv/writev/sendfile/ppoll/getrandom/sysinfo/readlinkat/statfs 等）。
- 信号类：`kernel/src/syscall/sig.rs`（rt_sigaction/rt_sigprocmask/rt_sigsuspend）。
- 用户态拷贝校验：`kernel/src/syscall/check.rs`（`read_bytes_from_user`、`write_bytes_to_user`、`validate_user_buffer` 等）。

### TRAP/IRQ（中断异常）
- **证据强**。`kernel/src/arch/riscv64/trap/handler.rs#__trap_handler` 为总入口，负责用户态上下文保存/恢复。
- `handle_exception` 分发系统调用与异常（含页fault）。
- `handle_interrupt` 通过 PLIC 分发外部中断（UART），并调用 `timer::on_tick`。
- `kernel/src/arch/riscv64/trap/timer.rs` 实现 SBI timer v0.2 探测、`on_tick`、`set_next_trigger`。

### NET（网络协议栈）
- **底层证据未体现**。未发现网络设备驱动、socket 系统调用、TCP/IP 协议栈相关函数。
