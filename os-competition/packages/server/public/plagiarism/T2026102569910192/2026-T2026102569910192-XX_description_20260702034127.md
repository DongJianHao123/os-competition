## 第四部分：分模块架构说明

### MM（内存管理）
**证据较强**。核心缺页处理函数 `SWTC/kernel/src/mm/memory_space/page_fault_handler.rs#handle_page_fault_async` 使用 `PageBuilder` 按权限与文件信息构造页映射，并设置 `PTEFlags`。ELF 加载通过 `SWTC/kernel/src/mm/memory_space/mod.rs#map_elf` 按 program header 建立 `VmArea`，并支持动态链接器加载 `load_dl_interp_if_needed`。用户态 `mmap` 由 `SWTC-la/xapi/src/mm/mmap.rs#sys_mmap` 实现。初始进程地址空间在 `SWTC/kernel/src/process/mod.rs#new_initproc` 中通过 `MemorySpace::from_elf` 创建。

### PROC（进程管理/任务调度）
**证据较强，具备传统进程管理要素**。存在 PCB 式的 `Process`/`Thread` 结构、进程组 `ProcessGroup`、会话 `Session`（见 `SWTC-la/xmodules/xprocess/src/process.rs`、`SWTC-la/xcore/src/task/proc.rs`）。支持 `clone`/`fork`（`SWTC-la/xapi/src/task/clone.rs#do_clone`、`SWTC/kernel/src/process/mod.rs#clone_thread`）、ELF 执行（`MemorySpace::from_elf`）、退出与僵尸回收（`SWTC-la/xapi/src/task/exit.rs#do_exit`、`SWTC/kernel/src/process/thread/exit.rs#handle_exit`、`SWTC-la/xapi/src/task/wait.rs#sys_wait4`）。信号、futex、IPC（shm/sem）均有对应系统调用。调度方面采用异步运行时：`SWTC/kernel/src/process/thread/thread_loop.rs#threadloop` 与 `SWTC/kernel/src/process/thread/schedule.rs#poll` 中出现 `UserTaskFuture`、`executor::spawn`、waker、per-hart 任务队列。

### FS（文件系统）
**证据强**。存在 VFS 层（`SWTC-la/arceos/modules/axfs-ng/src/highlevel/fs.rs#open`）、挂载管理器（`SWTC/kernel/src/fs/file_system.rs#mount/unmount`）、inode 缓存（`INODE_CACHE`）。支持多种文件系统：`TmpFS`、`VFAT`、通过第三方 `lwext4_rust` 支持的 `Ext4`、`procfs`（`/proc/meminfo`、`/proc/mounts`、进程目录）。fd 表、文件描述符操作（`sys_openat/read/write/preadv2/pwritev2`）、高级特性如 `fanotify`（`SWTC-la/xapi/src/fs/fd/fanotify.rs`）、`epoll`（`SWTC-la/xapi/src/iomux/epoll.rs#sys_epoll_wait`）、`renameat2` 均有实现。

### DEV（设备驱动）
**证据中等偏弱**。驱动框架通过 `SWTC-la/arceos/modules/axdriver/src/lib.rs#init_drivers` 枚举 block/net/display 设备。具体驱动仅见 `fu740` 平台的 UART（`SWTC/kernel/src/driver/fu740/uart.rs`）和 SD 卡 SPI（`SWTC/kernel/src/driver/fu740/sdcard.rs`），以及字符设备 TTY 初始化（`SWTC/kernel/src/driver/mod.rs#init_char_device`）。PCI、USB、GPU、音频、NVMe 等驱动**底层证据未体现**。

### SYSCALL（系统调用）
**证据较强**。存在大型统一分发器 `SWTC-la/src/syscall.rs#handle_syscall_impl`（205 条核心调用），覆盖文件、进程、内存、网络、信号、时间、资源、调度、IPC 等。具体实现分散在 `SWTC/kernel/src/syscall/fs.rs`、`process.rs`、`time.rs`、`resource.rs`、`signal.rs`，以及 `SWTC-la/xapi/src/task/`、`SWTC-la/xapi/src/ipc/`、`SWTC-la/xapi/src/iomux/` 等目录。

### TRAP/IRQ（中断异常）
**证据中等**。架构相关 trap 入口完整：`loongarch64_trap_handler`、`riscv_trap_handler` 负责分发系统调用、缺页、断点；`SWTC/kernel/src/trap/kernel_trap.rs#kernel_trap_handler` 处理内核态中断与超时。定时器超时处理见 `SWTC/kernel/src/timer/mod.rs#handle_timeout_events`。IRQ 开关与统计见 `SWTC-la/arceos/modules/axhal/src/irq.rs`。LoongArch 异常入口基址设置见 `SWTC-la/arceos/modules/axhal/src/arch/loongarch64/mod.rs#cpu_init`。

### NET（网络协议栈）
**原创实现证据弱**。协议实现主要依赖第三方 `smoltcp`（`SWTC-la/arceos/crates/smoltcp`）。内核层仅提供 socket 抽象与系统调用适配：`SWTC/kernel/src/net/tcp.rs`、`udp.rs`、`config.rs`，以及 `SWTC-la/xcore/src/net/socket.rs`。因此可判定为 **基于 smoltcp 的网络协议栈封装**，而非自研 TCP/IP 协议栈。

## 第六部分：边界与未体现内容

### 明确没有底层证据支撑的模块/特性
1. **原生 TCP/IP 协议栈实现**：网络协议实现基本来自第三方 `smoltcp`，内核仅提供 socket 封装与系统调用适配。
2. **完整外设驱动生态**：仅见 UART、SD 卡 SPI、字符设备 TTY 与驱动枚举；PCI、USB、NVMe、SATA、GPU、音频、显示控制器等驱动**底层证据未体现**。
3. **安全机制**：_capabilities、MAC、SELinux/AppArmor、seccomp 等**底层证据未体现**。
4. **虚拟化/容器支持**：KVM、hypervisor、cgroups、namespace 完整实现**底层证据未体现**（仅见 `CloneFlags` 与部分命名空间相关代码）。
5. **电源管理与热插拔**：CPU 频率、休眠、设备热插拔**底层证据未体现**。
6. **内核调试/性能分析**：除日志与 `stack_trace!` 外，perf、tracepoint、kprobe 等**底层证据未体现**。
7. **swap/页面换出**：缺页处理中未见 swap 分区或页换出逻辑**底层证据未体现**。

### 需要单独说明的第三方/公共依赖
- **`smoltcp`**：网络协议栈主体，函数数量占网络相关代码绝大多数。
- **`lwext4_rust` / `lwext4` C 库**：Ext4 文件系统实现，属于第三方库。
- **`heapless`**：通用无分配集合容器，属于第三方依赖。
- **ArceOS 模块（`axhal/axtask/axdriver/axfs-ng` 等）**：底层硬件抽象、任务调度、驱动框架、文件系统框架，可能为复用/改造组件，不等同于全部原创内核主体。

### 判断边界
- 本作品中 `SWTC-la` 与 `SWTC` 的划分、LoongArch64 与 RISC-V 双架构支持、基于 ArceOS 组件构建等判断，均来自目录路径与函数调用链，属于**可能方向**，不是最终身份或原创性结论。
- 本文档仅基于 AST Call Map、函数名、模块路径与调用链描述技术事实，不做抄袭/违规判断，也不对代码原创性比例下结论。
