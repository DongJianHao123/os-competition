## 第四部分：分模块架构说明

### MM（内存管理）

底层证据较强。包含：
- **物理页分配器**：`kernel/src/l0/30_page/page_alloc/body.rs#init` 从 boot memory map 初始化空闲页框，`dealloc_page` 支持回收与区间合并。
- **内核堆**：`kernel/src/l0/40_heap/kernel_heap/api.rs#dealloc_global` 维护 used/free bytes 与空闲块链表。
- **用户地址空间**：`kernel/src/l1/20_task/task_mm_signal/api/memory.rs` 提供 `clone_mapping`、`ensure_page_present`、`map_data` 等，用于 ELF 加载、COW/懒加载映射、栈/brk 映射。
- **缺页处理**：`arch/imp/{riscv64,loongarch64}.rs` 中的 `handle_page_fault` 调用链可见。

未体现：swap、NUMA、hugepage、完整 COW（`clone_mapping` 存在但 COW 语义未在函数名中明确）。

### PROC（进程管理/任务调度）

底层证据为传统进程管理，而非轻量任务：
- PCB 抽象：`TaskId`、父/子进程链表、信号掩码与待处理信号队列。
- `kernel/src/l1/50_bridge/fork/api.rs#clone_current` 实现 fork/clone。
- `kernel/src/l1/50_bridge/exec/exec.rs#exec_with_env` 实现 ELF exec，含 interpreter、用户栈、TLS、地址空间替换。
- `kernel/src/l1/50_bridge/exit_wait/api.rs#wait_current` / `exit_current` 实现 wait/exit/zombie 语义。
- 调度器：`kernel/src/l0/70_run/scheduler/api.rs#schedule` 维护就绪队列、当前任务指针、时钟中断。

### FS（文件系统）

- **VFS 层**：`kernel/src/l1/30_vfs/vfs_fd/api.rs` 提供 fd 表、路径解析、符号链接缓存、procfs、mount 点管理。
- **后端**：主要依赖第三方库 `crate/ext4_rs`，实现块位图、inode、extent、目录项、读写。
- **系统调用接口**：`kernel/src/l2/30_syscall_handlers/sys_fs/api.rs` 实现 `mount`、`readv`、`writev`、`readlinkat`、`getdents64` 等。

需单独说明：ext4 文件系统主体为 `ext4_rs` 第三方 crate，内核 VFS 与 syscall 封装为原创层。

### DEV（设备驱动）

- 仅发现 **VirtIO 块设备** 驱动：`virtio_mmio_blk` 与 `virtio_pci_blk`。
- `kernel/src/l0/80_device/device_core/body.rs#init` 探测并注册设备。
- `kernel/src/l0/80_device/device_core/virtio_pci_blk.rs#submit` 实现请求队列提交。

未体现：网络设备、显示、键盘、声卡、SATA/NVMe 等。

### SYSCALL（系统调用）

- `kernel/src/l2/40_table/syscall_table/api.rs#dispatch` 是大表分发中心。
- `kernel/src/l2/50_syscall_entry/syscall_entry/api.rs#handle_trap` 为统一入口。
- `kernel/src/l2/30_syscall_handlers/` 下分 `sys_fs`、`sys_signal`、`syscall_handlers` 实现具体 handler。
- 参数封送：`syscall_args/api.rs#read_value`、`read_path`、`read_buffer` 等。

### TRAP/IRQ（中断异常）

- 架构层：`kernel/src/arch/imp/riscv64.rs` 与 `loongarch64.rs` 的 `__namenotfound_*_handle_trap` 分发 ecall、页错误、时钟中断。
- `kernel/src/arch/syscall.rs#handle_trap` 将 trap 转交给上层 syscall_entry。
- 用户态拷贝：`kernel/src/l0/50_trap/user_copy_raw/api.rs#read_bytes / write_bytes / read_cstr`。
- 空闲等待：`kernel/src/arch/imp/loongarch64.rs#wait_for_interrupt`。

### NET（网络协议栈）

**底层证据未体现**。仅发现 `kernel/src/l1/40_ipc_net_device/ipc_net_device/api.rs` 中的 `with` / `recv` 等接口，疑似 IPC/loopback 占位，无 TCP/IP、UDP、socket、NIC 驱动等完整网络栈证据。
