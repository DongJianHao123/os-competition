## 第四部分：分模块架构说明

### MM（内存管理）
底层证据较强。核心模块为 `axmm`，用户态内存管理集中在 `src/mm.rs`。
- 物理页分配：`axruntime/src/lib.rs#init_allocator` 调用 `axalloc::global_init` 与 `global_add_memory`。
- 地址空间：`axmm/src/aspace.rs#clone_or_err` 支持 `new_alloc`、`new_cow`、`new_shared`、`new_segment_shared`，说明实现了写时复制（COW）与共享内存后端。
- 页错误：`axmm/src/backend/alloc.rs#handle_page_fault_alloc` 处理按需分配、引用计数（`inc_frame_ref`/`dec_frame_ref`）、拷贝内核字节等；`src/mm.rs#handle_page_fault` 处理用户态页错误并做诊断。
- ELF 加载：`src/mm.rs#load_user_app_inner_with_image`、`map_elf`、`read_user_image` 完成 ELF 解析、段映射、解释器加载。
- TLS：`src/mm.rs#map_initial_thread_tls`、`map_initial_thread_tls_musl` 映射初始线程本地存储。

### PROC（进程管理/任务调度）
底层证据强，呈现传统 Unix 进程模型。
- PCB 与关系：`src/task.rs` 中 `task_ext` 包含 `children`、`parent_task`、`zombie_children`、`process_group`、`leader_tid`、`robust_list_head` 等；`process_leaders`、`zombie_processes` 维护全局表。
- 生命周期：`clone_task`（184 条调用）、`exec_with_args_env_loader`、`exit_current_task`、`sys_exit`、`sys_exit_group`、`wait_child`、`wait4_child_state_event`、`waitid_exited_event`。
- 孤儿进程：`reparent_orphaned_children_on_exit`。
- 调度：`axtask/src/run_queue.rs#exit_current`、`axtask/src/wait_queue.rs#wait_timeout_until`/`wait_until`/`notify_task`；`main.rs` 设置 `set_task_switch_hook`、`set_wait_interrupt_hook`。
- 注意：虽然模块名包含“异步运行时”，但底层证据主要表现为任务调度、等待队列、定时器原语，独立的 Future/async 执行器实现细节未在提供的证据中单独量化。

### FS（文件系统）
底层证据最强，覆盖 VFS、挂载、procfs、ramfs、ext4 包装。
- VFS 与根文件系统：`axfs/src/root.rs#init_rootfs`、`mount`、`remove_dir`；`arceos_posix_api/src/imp/fs.rs#open_path`、`add_file_or_directory_fd`。
- 挂载：`syscall_imp/fs/ctl.rs#sys_mount` 支持块设备镜像挂载，调用 `open_mount_source_blockdev`、`read_mount_source_image`。
- 具体文件系统：`axfs/src/fs/lwext4_rust.rs` 是 ext4 的 Rust 包装；`patches/axfs_ramfs/src/dir.rs#rename` 为 ramfs；`axfs/src/mounts.rs#procfs` 为 procfs。
- 命名空间：`CURRENT_DIR`、`CURRENT_DIR_PATH`、`CURRENT_FS_CRED` 按 `ns` 区分，说明有按进程命名空间隔离的文件系统状态。

### DEV（设备驱动）
底层证据中等，框架清晰但具体驱动细节有限。
- 驱动框架：`axdriver/src/lib.rs#init_drivers` 探测 block/net/display 设备。
- UART：`arceos/tools/raspi4/chainloader/src/bsp/device_driver/bcm/bcm2xxx_pl011_uart.rs#init` 初始化 PL011。
- 原始网络设备：`tools/bwbench_client/src/device.rs#new` 使用 `libc::socket`、`ETH_P_ALL`。
- 未体现：具体 NVMe/SATA/SD/eMMC 块控制器驱动、GPU 驱动、声卡驱动等底层证据未体现。

### SYSCALL（系统调用）
底层证据强，入口为 `src/syscall_imp/mod.rs#handle_syscall`。
- 大分发器：`handle_syscall` 规模 33973，245 条调用，按 `Sysno` 分发。
- 进程/线程：`syscall_imp/task/thread.rs#sys_exit`、`sys_exit_group`、`read_clone3_args_from_user`、`read_exec_image_from_fd`、`read_exec_c_string_array`。
- 文件：`syscall_imp/fs/io.rs#sys_openat`、`syscall_imp/fs/ctl.rs#sys_mount`。
- 网络/IPC/同步：`syscall_imp/utils/misc.rs#futex_wait`、`sys_recvfrom`、`sys_getsockname`、`sys_socketpair`；`syscall_imp/utils/sysv_ipc.rs`；`syscall_imp/utils/timerfd.rs`。
- 注意：大量系统调用通过 `arceos_posix_api` 再封装到底层模块。

### TRAP/IRQ（中断异常）
底层证据中等，入口清晰但函数数量较少。
- 多架构入口：`axhal/src/arch/loongarch64/trap.rs#loongarch64_trap_handler`、`axhal/src/arch/riscv/trap.rs#riscv_trap_handler`、`axhal/src/arch/aarch64/trap.rs#handle_sync_exception`、`axhal/src/arch/x86_64/idt.rs#new`。
- 分发：`crate::trap::handle_syscall`、`handle_user_enter`/`handle_user_return`、`handle_page_fault`、`handle_breakpoint`。
- 中断控制：`axhal/src/platform/loongarch64_qemu_virt/irq.rs#set_enable`、`axhal/src/irq.rs#register_handler_common`。
- 诊断：`src/signal.rs#handle_user_trap_diagnostic`、`axhal/src/trap.rs#handle_user_trap_diagnostic`。

### NET（网络协议栈）
底层证据中等，核心为 POSIX socket 封装 + smoltcp 第三方库包装。
- Socket API：`arceos_posix_api/src/imp/net.rs#create_socket`、`accept`、`send`、`recv`、`recvfrom`、`from_sockaddr`/`into_sockaddr`，支持 IPv4/IPv6、Unix domain socket。
- 底层协议栈：`axnet/src/smoltcp_impl/tcp.rs#begin_connect`/`send`/`recv`/`shutdown`；`udp.rs#send_impl`/`recv`/`bind`；`dns.rs#query`。
- 说明：TCP/UDP/IP 协议实现本身来自第三方 `smoltcp`，本项目主要提供 socket 抽象、地址转换、等待队列与错误码映射。

### 需要单独说明的第三方/公共依赖
- **`lwext4_rust`**：ext4 文件系统能力来自该第三方 Rust 包装，本项目提供 VFS 集成与缓存失效逻辑，不应视为完全原创的 ext4 实现。
- **`smoltcp`**：TCP/UDP/DNS 协议实现来自 smoltcp 库，本项目提供 socket 抽象、地址转换与等待机制。
- **`arceos/ulib/axlibc/c/`**：`.c` 文件多为用户态 C 库例程（如 `pow.c`），与内核主体无关。
