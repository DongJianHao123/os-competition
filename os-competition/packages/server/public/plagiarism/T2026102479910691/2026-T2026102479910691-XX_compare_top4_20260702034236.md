## 4. 核心代码调用流对比剖析

### 【内存管理】页表映射调用流

- **底层技术析理**: 
  - **目标项目**采用 `kalloc_zero → walk_pgtbl → uvm_map` 调用链。`kalloc_zero` 基于 `kalloc_page` 实现零化分配，`walk_pgtbl` 返回最后一级 PTE 指针，`uvm_map` 将物理页插入该位置。整体模型为“分配器 + 零化 + 用户虚拟内存映射（uvm_*）”。
  - **参考项目**采用 `page_alloc → page_walk → page_insert` 调用链。`page_alloc` 基于空闲链表（`list_add/list_del`）分配页框，`page_walk` 返回 PTE 地址，`page_insert` 调用 `remove_page` 处理已存在映射。整体模型为“页框管理 + 宏转换（PA2PTE/PTE2PA）+ 页表插入”。
  - **异构性**: 两者内存对象模型不同（`kalloc_*` vs `page_*`），辅助设施不同（`uvm_create/uvm_alloc` vs `page2PTE/PTE2PA/remove_page`），外部调用图谱无交集。多级页表遍历的“分配→逐级索引→写入 PTE”流程是 RISC-V Sv39 架构规范决定的通用模式，非独创表达。

- **代码溯源链接**:
  - 目标项目: `kernel/mm.c#walk_pgtbl` → `kernel/mm.c#uvm_map`
  - 参考项目: `kernel/memory/page.c#page_walk` → `kernel/memory/page.c#page_insert`

### 【中断/异常处理】Trap 分发调用流

- **底层技术析理**:
  - **目标项目** `trap_handler` 读取 `r_scause`（异常/系统调用原因寄存器），根据 cause 值分发给 `syscall_handle` 进行系统调用处理，异常时调用 `uart_putc/uart_puts` 输出调试信息。处理对象为**同步异常 + 系统调用**。
  - **参考项目** `trap_handler` 读取 `r_sip`（软件中断挂起寄存器），分发给 `timer_handler` 处理定时器中断，最后写回 `w_sip` 清除中断标志。处理对象为**定时器/软件中断**。
  - **异构性**: 两者中断模型完全不同——目标处理同步异常，参考处理异步中断。读取的 CSR 寄存器不同（`scause` vs `sip`），分发目标不同（`syscall_handle` vs `timer_handler`），写回操作不同（无 vs `w_sip`）。唯一共享的外部调用 `uart_puts` 是通用调试输出，不构成同源证据。
