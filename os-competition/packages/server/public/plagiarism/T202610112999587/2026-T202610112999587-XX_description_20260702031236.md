## 第四部分：分模块架构说明

### MM（内存管理）

底层证据较强。核心实现位于 `libs/allocator`，包含带元数据的物理分配器：`libs/allocator/src/lib.rs#deallocate` 通过 `registry.remove` 与 `physical_allocation_from_record` 回收并校验分配记录；`libs/allocator/src/managed.rs#read_allocation_from_header` 与 `audit` 进行堆边界检查与分配审计。用户态地址空间方面，`kernel/src/user.rs#load_user_image_from_path_inner` 使用 `VmFlags` 建立只读/可执行/可写映射，并调用 `ops.activate` 切换地址空间；`load_exec_image_from_file` 负责 ELF phdr 解析。启动阶段 `kernel/src/acpi.rs#kernel_start_init` 与 `kernel/src/dtb/mod.rs#kernel_start_init` 均调用 `alloc_ops.init_kernel_page_table` 初始化内核页表。

**未直接体现**：完整的缺页异常（page fault）处理路径、swap/页面回收、NUMA 感知、内存 cgroup。

### PROC（进程管理 / 任务调度）

底层证据强。`libs/sched` 实现了任务控制块、调度器与生命周期：`libs/sched/src/scheduler.rs#schedule_once` 完成 CPU 当前任务切换；`libs/sched/src/spawn.rs#clone_task` 创建任务与线程组，`exit_task` 处理退出、SIGCHLD 与僵尸回收；`libs/sched/src/operation.rs` 提供 `wait_common`、`execve_with_context`、`exit_group` 等原语。`kernel/src/sched.rs` 包含信号帧安装/返回、execve、clone 用户上下文处理。系统调用层 `kernel/src/syscalls/process.rs` 覆盖 `clone`/`clone3`/`execveat`/`futex`/`waitid`/`prlimit64`/`sched_setaffinity`/`pidfd_open` 等。

**未直接体现**：完整的 cgroup/namespace 实现、ptrace 完整细节、实时调度器完整策略。

### FS（文件系统）

底层证据最强。VFS 与伪文件系统集中在 `general/src/vfs/`：`procfs.rs` 导出 `/proc` 信息，`sysfs.rs` 导出 `/sys`，`devtmpfs.rs` 处理终端输入信号，`tmpfs.rs` 提供内存文件系统。磁盘文件系统以库形式存在：`libs/extfs`（ext2/3/4）和 `libs/fatfs`。`libs/vfs` 还实现了 Unix socket与 netlink socket。系统调用入口在 `kernel/src/syscalls/fs.rs`，覆盖 `mount`/`openat2`/`ppoll`/`readlinkat`/`pwritev2` 等。

**未直接体现**：NFS/网络文件系统、文件系统日志崩溃恢复、quota、块层 I/O 调度器。

### DEV（设备驱动）

底层证据较强。块层入口在 `general/src/dev/block.rs`（`validate_bio`、`submit_bio_wait`）。virtio 系列驱动包括 `general/src/dev/drivers/virtio_blk.rs`、`virtio_pci.rs`、`virtio_block_common.rs`，完成请求构造、DMA 描述符提交与完成轮询。中断控制器驱动有 `general/src/dev/drivers/loongson_irq.rs`（probe/translate_source/program_source）。平台设备由 `kernel/src/dtb/mod.rs#platform_device_info_from_dtb` 与 `general/src/dev/platform.rs#register_firmware_irq_handler` 对接 DTB 资源。`general/src/dev/function.rs` 与 `general/src/dev/net.rs` 提供 PnP 设备枚举与网络设备控制接口。

**未直接体现**：USB host/device 控制器驱动（仅 `general/src/dev/usb.rs` 有 vendor_product 存根）、GPU/DRM、NVMe/AHCI、DMA 引擎、音频设备。

### SYSCALL（系统调用）

底层证据存在但分类较散。`kernel/src/syscalls/` 下 `process.rs`、`fs.rs`、`signal.rs`、`ipc.rs`、`syslog.rs` 实现了大量 Linux ABI 包装。`hal/src/user.rs#patch_loongarch64_enosys_stub` 用于修补未实现系统调用的 ENOSYS 桩。统计中“系统调用”模块仅 37 个函数，说明系统调用分发层较薄，具体逻辑多下沉到 process/fs/signal 等模块。

**未直接体现**：seccomp、审计系统调用、系统调用跟踪/perf 事件入口。

### TRAP/IRQ（中断异常）

底层证据中等。`general/src/dev/irq.rs#dispatch_line` 按 IRQ line 分发注册的处理程序并执行 bottom-half；`unregister` 支持卸载。`general/src/dev/drivers/loongson_irq.rs` 实现具体中断控制器编程。`arch/src/riscv64/trap/exception.rs#looks_like_fpu_instruction` 从用户态拷贝指令以判断是否为 FPU 指令。`kernel/src/dtb/pci.rs#install_irq_routing` 处理 DTB PCI 中断路由。

**未直接体现**：完整的异常向量表/系统调用入口汇编、NMI、性能监控中断、MCE。

### NET（网络协议栈）

底层证据强，但需区分原创与第三方。`libs/mygo-smoltcp` 是 smoltcp 协议栈的分支（2456 个函数），TCP 核心在 `libs/mygo-smoltcp/src/socket/tcp.rs#dispatch`/`process`。自有集成层在 `libs/net/src/stack.rs`：`raw_send`、`udp_send_to`、`parse_raw_packet_meta`、`socket_recv_next_len` 等将 smoltcp 端点、接口锁与 socket 句柄桥接；`libs/net/src/interface.rs` 同步静态网关路由。`libs/vfs/src/netlink_socket.rs#handle_newaddr`/`handle_newroute` 处理 RTM_NEWADDR/NEWROUTE。`libs/socket/src/state.rs` 管理 socket 生命周期。

**未直接体现**：自有 TCP 拥塞控制完整实现（依赖 smoltcp）、IPsec、BPF/XDP、netfilter。

## 第六部分：边界与未体现内容

### 1. 第三方 / 强约束代码需单独说明
- `libs/mygo-smoltcp`（2456 个函数）是 smoltcp 协议栈的分支，TCP/UDP/ICMP/IP 核心协议状态机属于该第三方/衍生代码，不应视为本项目原创网络协议主体。
- `libs/acpi` 包含 AML 解释器（如 `do_execute_method`），属于 ACPI 规范强约束实现。
- `libs/extfs`、`libs/fatfs` 为文件系统规范实现，具有强格式约束。
- `libs/efi` 与 `arch/*/loader.rs` 为固件/架构启动约定代码。

### 2. 没有底层证据支撑的模块/能力
- **完整缺页异常与 swap 子系统**：仅见分配器与用户镜像加载，未见 page fault handler、swapout/reclaim。
- **内核模块动态加载**：未见 `init_module`/`finit_module` 或模块重定位相关路径。
- **USB 控制器驱动**：仅见 `general/src/dev/usb.rs#vendor_product` 等存根，无 EHCI/xHCI/ohci 实现。
- **GPU/DRM、音频、视频、输入子系统**：无底层证据。
- **虚拟化 / Hypervisor**：无 KVM/VMX/SVM 相关代码。
- **安全模块 / LSM / seccomp / audit**：未见相关策略框架。
- **NFS / 网络文件系统**：未见。
- **块设备 I/O 调度器**：仅见 bio 提交与 virtio 完成，无 elevator/调度策略证据。
- **文件系统日志恢复与 quota**：extfs 有元数据写入，但日志恢复路径未直接体现。
- **完整实时调度器实现**：有 `sched_attr` 与策略编码，但具体 EDF/RT 调度细节未充分暴露。
- **系统调用入口 / 异常向量表底层汇编**：仅见部分 arch trap 文件，未见完整 syscall entry。
- **多核同步原语底层实现**（如自旋锁、RCU 核心）：调度器使用锁，但原语实现未在证据中突出。
