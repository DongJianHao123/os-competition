#### 4. 核心代码调用流对比剖析
* **【任务调度/异步运行时】**
* **底层技术析理**: 目标项目采用基于 `task_index` 的数组/位图调度模型，通过 `task_pick_next` 选择任务，并集成 `epoll`、`futex`、`timerfd` 等现代Linux异步机制。参考项目采用基于 `LIST_*` 宏的链表调度模型，通过 `sched_yield` 操作 `env` 结构体链表。两者在数据结构、调度算法和异步I/O模型上完全不同。
* **代码溯源链接**:
  - 目标项目: `src/kernel/task.c#task_pick_next`
  - 参考项目: `lib/sched.c#sched_yield`

* **【内存管理/异常处理】**
* **底层技术析理**: 目标项目的 `trap_handler` 直接读取RISC-V CSR寄存器 `scause`/`stval` 进行异常分发，调用 `syscall_dispatch` 和 `task_handle_page_fault`。参考项目的 `trap_handler` 则调用 `breakpoint_handler`、`page_fault_handler`、`syscall_handler` 等独立处理函数。两者在异常分发机制、处理函数命名和调用链上无重合。
* **代码溯源链接**:
  - 目标项目: `src/arch/riscv64/trap.c#trap_handler`
  - 参考项目: `lib/traps.c#trap_handler`

* **【文件系统】**
* **底层技术析理**: 目标项目实现了完整的 ext4 文件系统支持，包含 `extent_lookup_rec`、`preload_ext4_root_files` 等特有函数，并与 memfs 结合。参考项目使用 FAT32 文件系统，包含 `emake`、`enext` 等FAT32特有函数。两者在文件系统类型、磁盘布局、目录项结构和操作逻辑上完全不同。
* **代码溯源链接**:
  - 目标项目: `src/kernel/ext4.c#ext4_getdents`
  - 参考项目: `lib/fat32.c#enext`
