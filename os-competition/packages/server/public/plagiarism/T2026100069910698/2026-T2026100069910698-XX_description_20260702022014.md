## 第四部分：分模块架构说明

### MM（内存管理）

- **页分配**：`vm_page_allocator_init` 初始化页列表，`vm_alloc_page` / `vm_free_page` 用于分配与释放。
- **地址空间**：`sys_mmap` 管理用户态映射，`sys_brk` 管理堆边界（`task_group_brk_current`）。
- **写时复制（COW）**：`vm_handle_cow_fault` 处理页表项复制与 `sfence_vma` 刷新。
- **页故障入口**：RISC-V 异常处理 `trap_handler` 调用 `task_handle_page_fault`。
- **用户栈**：`user_stack_prepare_args_aux_on` 为 exec 准备参数、环境变量与 AUXV。

> 证据锚点：`src/kernel/vm.c#vm_handle_cow_fault`、`src/kernel/vm.c#vm_page_allocator_init`、`src/kernel/user_stack.c#user_stack_prepare_args_aux_on`、`src/kernel/syscall.c#sys_mmap`、`src/arch/riscv64/trap.c#trap_handler`。

### PROC（进程管理/任务调度）

- **任务结构**：通过 `current_task` 与 `task_group_*` 系列操作推断存在 PCB/任务组结构。
- **生命周期**：`sys_clone3_from_tf`、`sys_vfork_from_tf` 创建任务；`sys_exit` / `sys_exit_group` → `task_set_exit` / `task_exit_group` → `task_finish` 退出；`sys_wait4` → `task_wait` 等待子进程。
- **调度**：`task_pick_next` 选择下一个任务，`task_request_resched` 请求重调度，`task_sleep_until` 实现阻塞睡眠，`task_on_trap_return` 在异常返回时处理调度与信号。
- **同步与事件**：futex（`sys_futex`、`task_wait_futex_until`）、信号（`sys_rt_sigaction`、`sys_rt_sigtimedwait`、`sys_rt_sigprocmask`）、epoll/timerfd/eventfd/pipe。

> 证据锚点：`src/kernel/task.c#task_wait`、`src/kernel/task.c#task_exit_group`、`src/kernel/task.c#task_finish`、`src/kernel/task.c#task_on_trap_return`、`src/kernel/syscall.c#sys_clone3_from_tf`、`src/kernel/syscall.c#sys_wait4`。

> 说明：现有证据体现的是“任务 + 阻塞/睡眠/唤醒”式调度，**未体现**显式的 `async/await`、`waker`、`Future` 等异步运行时原语，不宜直接判定为完整异步运行时。

### FS（文件系统）

- **VFS 层**：`fs.c` 提供路径解析、FD 管理、`fs_open`、`fs_getdents`、`fs_link_path`、`fs_rename_path`、`fs_unlink_path` 等。
- **后端**：
  - `ext4.c`：superblock、inode、extent、目录项解析（`inode_read_at`、`lookup_child`、`extent_lookup_rec`）。
  - `memfs`：内存文件系统，用于根目录/预加载文件（`preload_ext4_root_files`、`memfs_create_path_raw`）。
- **系统调用**：`sys_openat`、`sys_readlinkat`、`sys_newfstatat`、`sys_statx`、`sys_utimensat`、`sys_mount`、`sys_umount2`、`sys_unlinkat`、`sys_mkdirat`、`sys_linkat`、`sys_renameat`、`sys_chdir`、`sys_getcwd` 等。

> 证据锚点：`src/kernel/fs.c#fs_open`、`src/kernel/fs.c#fs_init`、`src/kernel/ext4.c#ext4_init`、`src/kernel/ext4.c#inode_read_at`、`src/kernel/syscall.c#sys_openat`。

### DEV（设备驱动）

- **VirtIO 块设备**：
  - `src/drivers/virtio_blk.c#block_read_sector`（MMIO 风格）
  - `src/arch/loongarch64/virtio_pci_blk.c#block_init`（PCI 探测 + IO 端口访问）
- **VirtIO 网络设备**：`src/drivers/virtio_net.c#net_init`、`net_send_packet`、`net_recv_packet`。
- **字符输出**：`src/kernel/printk.c#printk` → `uart_putc`。

> 证据锚点：`src/drivers/virtio_net.c#net_init`、`src/drivers/virtio_blk.c#block_read_sector`、`src/arch/loongarch64/virtio_pci_blk.c#block_init`、`src/kernel/printk.c#printk`。

### SYSCALL（系统调用）

- `src/kernel/syscall.c` 为系统调用中枢，包含 `syscall_dispatch` 分发表。
- 覆盖 IO（read/write/pread/pwrite/sendfile）、文件（openat/close/dup3/fcntl）、进程（clone/exit/wait4）、内存（mmap/brk）、网络（socket 系列）、信号、poll（epoll/poll/select/ppoll）、时间（timerfd/setitimer）等。
- 大量调用使用 `user_range_ok`、`copy_user_cstr`、`copy_user_vec` 进行用户态地址校验与拷贝。

> 证据锚点：`src/kernel/syscall.c#syscall_dispatch`、`src/kernel/syscall.c#sys_read`、`src/kernel/syscall.c#sys_write`、`src/kernel/syscall.c#sys_openat`、`src/kernel/syscall.c#sys_execve_from_tf`。

### TRAP/IRQ（中断异常）

- 仅有 RISC-V64 侧证据：`src/arch/riscv64/trap.c` 中 `trap_init` 设置 `stvec`，`trap_handler` 统一处理异常、系统调用与定时器中断，`trap_enable_timer` / `timer_set_next` 配置时钟。
- 定时器中断驱动调度：`task_pick_next`、`task_request_resched`。

> 证据锚点：`src/arch/riscv64/trap.c#trap_handler`、`src/arch/riscv64/trap.c#trap_init`、`src/arch/riscv64/trap.c#trap_enable_timer`。

> 说明：该模块函数数仅 3，证据较弱；LoongArch64 侧的中断/异常框架在提供的 Call Map 中**未体现**。

### NET（网络协议栈）

- **Socket API**：`sys_bind`、`sys_listen`、`sys_accept`、`sys_accept4`、`sys_sendto`、`sys_recvfrom`、`sys_shutdown`、`sys_getsockname`、`sys_getpeername`、`sys_getsockopt`。
- **内部缓冲**：`socket_send_buffer`、`socket_recv_buffer`、`task_wake_socket`。
- **底层驱动**：`virtio_net.c` 完成实际收发。

> 证据锚点：`src/drivers/virtio_net.c#net_init`、`src/kernel/syscall.c#sys_sendto`、`src/kernel/syscall.c#sys_recvfrom`、`src/kernel/syscall.c#sys_accept`。

> 说明：仅有 Socket 层与 VirtIO 驱动交互的证据，**未体现**完整 TCP 状态机、IP 路由、ARP/ICMP 等协议栈细节。
