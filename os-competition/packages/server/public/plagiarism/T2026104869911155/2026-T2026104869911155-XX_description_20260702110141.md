## 第四部分：分模块架构说明

### MM（内存管理）
底层证据显示存在用户地址空间管理、ELF 加载、栈映射与堆分配器：
- `src/mm/user.rs#load_elf_with_args` / `map_initial_stack` / `apply_static_pie_relocations` 负责 ELF 解析后的加载、重定位与初始栈构造。
- `src/mm/heap.rs#add_free_block` / `remove_free_block` 表明存在内核堆分配器。
- `src/mm/paging.rs` 存在，但 AST 证据中未展开其页表操作细节。
- 用户态内存访问通过 `aspace.read` / `aspace.write` 完成；`riscv_handle_trap` 会调用 `handle_page_fault`，但缺页处理的具体策略底层证据未体现。

### PROC（进程管理/任务调度）
**不是单纯的轻量级任务/协程调度**，底层证据包含传统 Unix 进程管理要素：
- 创建：`src/task/mod.rs#fork_from` 深拷贝地址空间并复制 cgroup、bpf、fd_table 等状态。
- 执行：`src/task/mod.rs#load_exec_image` / `sys_execve` / `sys_execveat` 完成 ELF 解析与镜像替换。
- 退出/等待：`src/task/riscv64.rs#exit_group_current` / `handle_wait` / `reap_zombie` / `queue_child_exit_signal` 体现父子关系、僵尸进程回收。
- 调度：`src/task/riscv64.rs#schedule_next_ready` 与多种 `TaskState`（`PipeReadWaiting`、`PipeWriteWaiting`、`EpollWaiting`、`SignalWaiting`、`FutexWaiting` 等）构成阻塞-唤醒状态机。
- 信号：`sys_rt_sigtimedwait` / `sys_rt_sigprocmask` / `handle_signal_timedwait` 提供部分信号机制。

### FS（文件系统）
文件系统是本项目代码量最大的模块，呈现多层结构：
- **内存文件系统**：`src/fs/file.rs` 中 `OpenFile`、`FileData`、`directories` 等结构实现目录、常规文件、重命名、创建、锁、lease、record lock。
- **挂载与 overlay**：`mount_overlay_lower`、`rename_memory`、`mkdir_memory`、`sys_mount` 支持 overlay lowerdir 与内存层组合。
- **事件通知**：`fanotify`（`sys_fanotify_init`、`queue_fanotify_permission_event`）与 `inotify`（`queue_inotify_child_event`）证据存在。
- **procfs / cgroup 伪文件**：`synthetic_proc_pid`、`cgroup_procs_open_context`、`process_accounting_path` 等。
- **ext4 集成**：`src/fs/ext4.rs` 可解析 ext4 inode、目录项，并通过 `from_ext4` 系列函数将 ext4 内容导入内存文件系统。该部分在模块统计中被归入“文件系统/第三方库”，应视为集成/适配代码，不宜直接等同于原创内核主体。

### DEV（设备驱动）
设备驱动证据**非常有限**：
- 仅见 `src/drivers/block/virtio_mmio.rs` 的 virtio-mmio 块设备驱动（`probe`、`read_sector`、`wait_used`、`read_device_features`）。
- 控制台串口：`src/arch/riscv64.rs#putchar`、`src/arch/loongarch64.rs#console_init`。
- 无网络设备、无显示/输入/时钟等其他设备类驱动证据。

### SYSCALL（系统调用）
系统调用层证据强：
- `src/task/mod.rs#handle_syscall_return` 作为大分发器，覆盖 146+ 个 `sys_*` 调用。
- `src/task/riscv64.rs#handle_syscall` 处理架构相关入口（clone、fork、exec 等）。
- 覆盖文件、进程、信号、挂载、epoll、fanotify、bpf、模块加载等 Linux 兼容接口。

### TRAP/IRQ（中断异常）
- RISC-V 异常入口：`src/task/riscv64.rs#riscv_handle_trap` 分发系统调用、缺页、时钟中断、关机。
- 时钟中断：`src/arch/riscv64.rs#set_timer_after_us` → `handle_timer` → 调度。
- LoongArch64 仅见 `console_init` / `shutdown` 桩代码，完整异常/中断处理底层证据未体现。

### NET（网络协议栈）
**底层证据未体现完整网络协议栈**：
- 仅见 socket 文件抽象（`socket_sendto`、`alloc_ephemeral_port`）、netlink 响应构造（`netlink_response_for_request`）、合成 IPv4 ICMP socket 等局部代码。
- 无 TCP/IP 协议状态机、无 NIC 驱动、无完整 BSD socket 层证据。

- **完整内核模块加载器**：`sys_init_module` / `sys_finit_module` / `loaded_modules` 仅体现接口与列表，无模块重定位、符号解析、依赖加载等证据。
