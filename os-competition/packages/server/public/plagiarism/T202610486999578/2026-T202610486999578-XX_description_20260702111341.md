## 第四部分：分模块架构说明

### MM（内存管理）

有较为明确的用户内存管理证据：

- `kernel/src/syscall/mm/mmap.rs#sys_mmap` 实现了 `mmap` 系统调用，包含地址对齐、随机化偏移计算、文件映射等逻辑。
- `kernel/src/mm/aspace/backend/cow.rs#alloc_new_at` 提供 Copy-On-Write 后端，从文件读取内容并映射到物理页。
- 用户空间指针访问通过 `VmBytes`、`VmBytesMut`、`UserConstPtr`、`UserPtr` 等抽象进行校验与拷贝（如 `kernel/src/syscall/fs/fanotify.rs#sys_name_to_handle_at`）。
- 共享内存由 `kernel/src/syscall/ipc/shm.rs` 的 `attach_process` / `detach_process` 管理。
- 页故障处理由 `patches/axcpu/src/{riscv,aarch64,loongarch64}/trap.rs#*_trap_handler` 调用 `handle_page_fault` 完成。

底层页表分配与 TLB 刷新代码多位于 `axcpu`/`axhal` 补丁中，本数据集仅见间接调用。

### PROC（进程管理/任务调度）

具备传统进程管理的基础证据，但调度核心依赖外部 `axtask`：

- `kernel/src/syscall/task/clone.rs#do_clone` 创建新任务/进程，处理 `parent_tid`、`pidfd`、线程结构。
- `kernel/src/syscall/task/execve.rs#sys_execve` / `sys_execveat` 实现程序加载，涉及路径解析与 fd 表重置。
- `kernel/src/syscall/task/wait.rs#apply` 与 `kernel/src/task/ops.rs#get_process_data` 维护进程表、进程组、父子关系。
- 信号机制：`kernel/src/task/signal.rs#send_signal_to_process`、`kernel/src/syscall/signal.rs#sys_rt_sigprocmask`。
- `pidfd`：`kernel/src/syscall/fs/pidfd.rs#sys_pidfd_open` / `sys_pidfd_getfd` / `sys_pidfd_send_signal`。
- procfs：`kernel/src/pseudofs/proc.rs#lookup_child` / `builder` 动态生成 `/proc/<pid>/stat|status|fd|fdinfo|exe|maps|mounts|ns` 等条目。
- futex：`kernel/src/syscall/sync/futex.rs#sys_futex_waitv`、`kernel/src/task/futex.rs#wait_if`。

调度器本体（`axtask`）在 `patches/` 中，内核侧主要暴露为 syscall 与同步原语封装。

### FS（文件系统）

证据最强：

- 伪文件系统：`kernel/src/pseudofs/proc.rs`（procfs）、`kernel/src/pseudofs/tmp.rs`（tmpfs）、`kernel/src/pseudofs/journal.rs`。
- 挂载系统：`kernel/src/syscall/fs/mount.rs#sys_mount` / `sys_umount2`、`kernel/src/syscall/fs/new_mount.rs#sys_fsopen` / `sys_fsmount` / `sys_fsconfig` / `sys_move_mount`，支持块设备镜像挂载、bind mount、挂载传播图（`MOUNT_GRAPH`）。
- fd 与路径：`kernel/src/syscall/fs/fd_ops.rs#sys_openat`、`kernel/src/syscall/fs/io.rs#sys_read` / `sys_write` / `sys_pread64` / `sys_fallocate` / `sys_writev`。
- 高级特性：`fanotify`（`kernel/src/syscall/fs/fanotify.rs`）、`io_uring`（`kernel/src/syscall/fs/io_uring.rs#process_sqe`）、`name_to_handle_at`。
- 第三方 ext4：`patches/axfs-ng/src/fs/ext4/inode.rs#create` / `read_dir` / `metadata` / `append` / `link`，`patches/axfs-ng/src/fs/ext4/fs.rs#new`。

### DEV（设备驱动）

设备驱动证据相对有限，主要集中在网络与块设备镜像：

- 块设备：`kernel/src/syscall/fs/mount.rs#sys_mount` 使用 `DiskImageBlockDevice::open` 打开磁盘镜像；`kernel/src/syscall/module.rs#ltp_block_dev_exit` 为测试用块设备模块。
- TTY：`kernel/src/pseudofs/dev/tty/mod.rs#ioctl` 处理 `Termios` / `WindowSize` 等终端 ioctl。
- 网络设备：`patches/axnet-ng/src/device/ethernet.rs#send` / `process_arp`、`patches/axnet-ng/src/device/vsock.rs#handle_vsock_event`。
- 平台 IRQ：`patches/axplat-loongarch64-qemu-virt/src/irq.rs#handle` / `set_enable`。

没有显卡、声卡、USB、NVMe 等完整驱动证据。

### SYSCALL（系统调用）

证据充分：

- 统一入口：`kernel/src/syscall/mod.rs#handle_syscall`，按 `Sysno` id 分派，规模 34613，调用数 291。
- 覆盖 fs、net、task、mm、signal、futex、io_mpx（epoll/select/poll）、bpf、`io_uring`、module、ipc（shm）等。
- 错误码采用 `LinuxError`，返回值按 Linux 约定处理。

### TRAP/IRQ（中断异常）

证据存在但较薄：

- 架构 trap handler：`patches/axcpu/src/riscv/trap.rs#riscv_trap_handler`、`patches/axcpu/src/aarch64/trap.rs#aarch64_trap_handler`、`patches/axcpu/src/loongarch64/trap.rs#loongarch_trap_handler`，处理页故障、断点、异常分发。
- x86_64 中断描述符表：`patches/axcpu/src/x86_64/idt.rs#init`、`patches/axcpu/src/x86_64/init.rs#init_trap`。
- LoongArch 平台 IRQ：`patches/axplat-loongarch64-qemu-virt/src/irq.rs#handle` / `set_enable`。

详细的向量表、中断控制器驱动、时钟 tick 驱动等多在 `axcpu`/`axplat` 补丁中，本数据集未完整体现。

### NET（网络协议栈）

证据较强：

- socket 系统调用：`kernel/src/syscall/net/socket.rs#sys_socket` / `sys_bind` / `sys_connect`，支持 `AlgSocket`、`CanSocket`、`Ipv6RawSocket`、`NetlinkSocket`、`PacketSocket` 等。
- IO 路径：`kernel/src/syscall/net/io.rs#recv_impl` / `sys_recvmsg`。
- Netlink / rtnetlink：`kernel/src/syscall/net/netlink.rs#handle_newlink` / `handle_newaddr` / `handle_sendmsg` / `handle_getaddr`；`kernel/src/net/rtnetlink.rs#parse_rtm_newlink` / `parse_rtm_setlink` / `build_ifinfo_reply`。
- 网络命名空间与 veth：`kernel/src/net/namespace.rs#bootstrap_ltp_veth`、`kernel/src/net/ioctl.rs#socket_if_ioctl`。
- 传输层补丁：`patches/axnet-ng/src/udp.rs#bind` / `send` / `recv`、`patches/axnet-ng/src/tcp.rs#recv`、`patches/axnet-ng/src/router.rs#dispatch` / `snoop_tcp_packet`。
- vsock：`patches/axnet-ng/src/vsock/stream.rs#accept`、`patches/axnet-ng/src/vsock/connection_manager.rs#on_connection_request`。
