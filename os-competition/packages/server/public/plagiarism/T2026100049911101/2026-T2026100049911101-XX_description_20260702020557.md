## 第四部分：分模块架构说明

### MM（内存管理）

- **已体现证据**：
  - `kernel/src/syscall/mm.rs#sys_mmap` 提供 `mmap` 系统调用，涉及 `MapFlags`、`MmapProt`、`VirtAddr`、`UserRef`。
  - `kernel/src/tasks/exec.rs#exec_with_process` 在 ELF 加载时建立内存映射区域、初始化用户栈 `kernel/src/tasks/stack.rs#init_task_stack`。
  - `kernel/src/tasks/task.rs#cow_fork` 与 `kernel/src/user/mod.rs#user_cow_int` 体现写时复制（COW）机制。
  - `crates/runtime/src/frame.rs#alloc_much` 为物理页帧分配器。
  - `crates/executor/src/executor.rs#init` 中出现 `PageTable::current`、`BOOT_PAGE`，说明存在页表切换相关封装。
- **未体现/证据弱**：页表遍历、TLB 刷新、缺页异常入口的完整底层代码未在提供的 Call Map 中直接出现。

### PROC（进程管理/任务调度）

- **已体现证据**：
  - `kernel/src/syscall/task.rs#sys_clone` 支持 `CloneFlags`、`clear_child_tid`，可创建线程/进程。
  - `kernel/src/syscall/task.rs#sys_wait4` 实现 `wait4`，包含 `WaitPid`、读取 `child.exit_code`、调用 `child.release` 回收。
  - `kernel/src/syscall/task.rs#sys_exit_group` 遍历 PCB 中的 `threads` 并调用 `thread_exit`。
  - `kernel/src/tasks/mod.rs#add_user_task` 打开标准 I/O 并构造 `UserTask`。
  - `kernel/src/tasks/initproc.rs#initproc` 为初始化进程入口，负责 `run_tests`、`start_shell`。
  - `kernel/src/tasks/task.rs` 中出现 `pcb.lock`、`fd_table`、`parent`、`children`、`exit_code` 等 PCB/父子关系字段。
- **调度/异步运行时**：`crates/executor/src/executor.rs#run_ready_task` 与 `crates/executor/src/thread.rs#spawn` 提供基于 `TASK_QUEUE`、`Waker`、`Future` 的异步调度。

### FS（文件系统）

- **已体现证据**：
  - `filesystem/fs/src/lib.rs#init` 初始化 `ProcFS`、`RamFs`、`DevFS`、`ext4fs`、`ext4rsfs`、`fatfs_shim`，并执行 `mount_fs`。
  - `filesystem/fs/src/file.rs#open` / `#getdents` 提供统一的文件打开与目录读取抽象。
  - `filesystem/ramfs/src/lib.rs#mkdir` 实现内存文件系统。
  - `filesystem/fs/src/fatfs_shim.rs#open/readat/writeat` 封装 FAT32。
  - `filesystem/ext4fs/src/lib.rs#create/seek/readat/symlink/resolve_link/read_dir` 封装 ext4（疑似 C 库 wrapper）。
  - `filesystem/ext4rsfs/src/lib.rs#lookup` 为 Rust 版 ext4 封装。
  - `filesystem/procfs/src/lib.rs#new/read_dir` 与 `interrupts.rs/meminfo.rs/mounts.rs` 实现 procfs。
  - `filesystem/devfs/src/tty.rs#ioctl` 实现设备文件控制。
  - `kernel/src/syscall/fd.rs#sys_mount` / `#sys_umount2` 提供挂载/卸载接口。
- **说明**：文件系统栈是本项目证据最强的模块，但部分实现为第三方库/外部 crate 的 shim。

### DEV（设备驱动）

- **已体现证据**：
  - `driver/general-plic/src/lib.rs#register_irq` / `#try_handle_interrupt` / `plic.rs#set_irq_enable` 提供 RISC-V PLIC 中断控制。
  - `driver/kvirtio/src/lib.rs#enumerate_pci` / `#init_mmio` / `#virtio_device_probe` 实现 VirtIO PCI/MMIO 枚举，支持 `virtio_blk`、`virtio_net`、`virtio_input`。
  - `driver/kvirtio/src/virtio_net.rs#recv` 为 VirtIO-Net 接收路径。
  - `driver/ns16550a/src/lib.rs#init_driver` 初始化 UART。
  - `driver/kgoldfish-rtc/src/lib.rs#init_rtc` 初始化 RTC。
  - `crates/devices/src/device.rs#add_device` 将设备按 blk/net/input/rtc/uart 分类注册。
- **未体现/证据弱**：中断向量表、异常处理上下文保存/恢复、DMA 映射细节未直接出现。

### SYSCALL（系统调用）

- **已体现证据**：
  - `kernel/src/syscall/mod.rs#syscall` 为统一入口，使用 `Sysno::new` 分派。
  - `kernel/src/syscall/fd.rs` 包含 44 个函数，覆盖 `openat`、`readlinkat`、`mount`、`umount2`、`writev`、`readv`、`poll`、`ppoll`、`pselect`、`epoll_wait/ctl`、`fcntl`、`sendfile`、`utimensat`、`renameat2`、`copy_file_range` 等。
  - `kernel/src/syscall/task.rs` 覆盖 `clone`、`wait4`、`exit_group`、`futex`、`fork`、`kill`、`tkill`、`setsid` 等。
  - `kernel/src/syscall/mm.rs#sys_mmap`、`kernel/src/syscall/time.rs`、`kernel/src/syscall/signal.rs`、`kernel/src/syscall/socket.rs`、`kernel/src/syscall/sys.rs`、`kernel/src/syscall/shm.rs` 分别覆盖内存、时间、信号、网络、系统信息、共享内存。
- **说明**：系统调用接口与 Linux 语义高度对应，但具体 ABI/TrapFrame 保存细节证据有限。

### TRAP/IRQ（中断异常）

- **已体现证据**：
  - `driver/general-plic/src/plic.rs#get_irq_claim` / `#complete_irq_claim` / `lib.rs#try_handle_interrupt` 提供 PLIC 中断响应。
  - `kernel/src/user/mod.rs#handle_syscall` 为 syscall 陷入处理入口。
  - `kernel/src/user/entry.rs#check_signal` / `#check_thread_exit` 处理信号与线程退出检查。
  - `kernel/src/tasks/stack.rs#init_task_stack` 使用 `TrapFrame` 构造初始上下文。
- **未体现/证据弱**：除 PLIC 与 syscall 入口外，完整的异常向量表、时钟中断调度、上下文切换汇编代码未在提供的证据中体现。

### NET（网络协议栈）

- **已体现证据**：
  - `kernel/src/socket.rs#new` / `#reuse` / `#drop` / `#new_with_inner` 封装 socket，依赖 `NET_SERVER.blank_tcp` / `blank_udp` / `get_tcp` / `get_udp`。
  - `kernel/src/syscall/socket.rs#sys_bind` / `#sys_connect` / `#sys_listen` / `#sys_accept4` / `#sys_sendto` / `#sys_recvfrom` / `#sys_shutdown` 提供 socket 系统调用。
- **未体现/证据弱**：**完整网络协议栈底层证据未体现**。当前仅见 socket 抽象层与对外部网络服务（`NET_SERVER`）的转发，无独立 TCP/IP 协议栈实现证据。

  - **SMP 调度细节**：`crates/executor` 支持多核初始化，但具体 per-CPU 调度、核间中断证据不足。
  - **内核模块/动态加载**：无证据。
  - **日志/文件系统 journaling**：无证据。

- **第三方库/公共依赖说明**：
  - `crates/executor`、`crates/runtime`、`crates/devices` 为公共运行时/设备抽象，不应视为原创内核主体。

  - 这些封装层是文件系统能力的重要组成部分，但原创性需结合具体实现进一步判断。

- **方向性判断**：
  - 本项目硬件线索（PLIC、VirtIO、NS16550a、Goldfish RTC）指向 **RISC-V/QEMU 风格平台**，但这仅为技术方向推断，不是最终平台身份结论。
  - “实现了什么”以文件系统、系统调用接口、进程/线程生命周期、异步运行时、设备驱动为主；“完整网络协议栈”与“底层中断异常框架”仅为可能方向，不是已确认实现。
