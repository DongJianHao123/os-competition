## 第四部分：分模块架构说明

### MM（内存管理）
- **物理内存**：`os/src/mm/frame_allocator.rs#init_frame_allocator` 通过 `PhysAddr` 计算可用物理区间并初始化页框分配器。
- **内核堆**：`os/src/mm/heap_allocator.rs#handle_alloc_error` 在堆分配失败时输出诊断信息。
- **虚拟地址空间**：`os/src/mm/memory_set.rs`（149 个函数）是核心，提供 `unmap_user_vma_range`、`remove_area_with_start_vpn`、`dealloc_trap_context_slot`、以及 `resolve_cow_fault` / `resolve_lazy_fault`。
- **进程资源回收**：`os/src/task/id.rs#dealloc_user_res` 在线程退出时回收 trap context 槽与 VMA。
- **证据强度：强**。

### PROC（进程管理/任务调度）
- **PCB/线程控制块**：`os/src/task/process_block.rs`、`os/src/task/task_block.rs`。
- **生命周期**：`clone_from_parts` / `fork_impl`（COW/deep copy）、`syscall_execve` / `exec_interpreter`、`syscall_wait4` / `reap_zombie_child`、`exit_group_and_run_next`。
- **调度**：`os/src/task/processor.rs#idle_task`、`block_current_and_run_next`、`suspend_current_and_run_next`、`should_preempt_current_on_tick`；`os/src/task/manager/run_queue.rs#resolve_sync_wakeup_hart` 处理同步唤醒与 CPU 选择；`os/src/task/manager/fair.rs` 出现 fair 调度相关计算。
- **同步与信号**：futex、condvar、epoll、signal、robust list、ptrace。
- **cgroup 集成**：`os/src/fs/cgroupfs/`、`cgroup_attach_process_to_target`、`refresh_process_legacy_cpu_fair_group_cache`。
- **证据强度：强**。

### FS（文件系统）
- **内核文件抽象**：`OSInode`、`PseudoFile`、`PseudoDir`、`PseudoShmFile`、`Pipe`、`ProcPseudoFile`、`MagicLink`、`NetSocketFile`。
- **系统调用层**：`syscall_openat`、`syscall_read`/`write`、`syscall_pwrite64`、`syscall_splice`、`syscall_copy_file_range`、`syscall_fcntl`、`mount_utils`。
- **procfs / cgroupfs / sysfs 风格文件**：`os/src/fs/procfs/`、`os/src/fs/cgroupfs/`。
- **ext4 后端**：`ext4-fs` crate（`create_dir`、`ensure_blocks`、`create_symlink`、`create_special`）与 `ext4-fs-packer` 工具；应视为外部/移植组件。
- **证据强度：强**，但需注意 ext4 实现主体不在 `os/` 目录内。

### DEV（设备驱动）
- **块设备**：`os/src/drivers/block/virtio_blk.rs` 支持 MMIO/PCI range 解析、`share`/`unshare` DMA buffer、`try_new_with_base`。
- **控制台**：`os/src/console/mod.rs#write_str` 输出到 UART。
- **证据强度：弱**；仅见 virtio-blk 与 console，未见通用设备模型、中断控制器驱动、USB/GPU/声卡等。

### SYSCALL（系统调用）
- **入口**：`os/src/syscall/mod.rs#syscall` 保存参数并分发。
- **覆盖**：进程/线程、文件系统、网络、信号、futex、epoll、timerfd、module、ptrace、mount、ioctl。
- **密集文件**：`os/src/syscall/net/socket.rs`（302 函数）、`os/src/syscall/net/netlink.rs`、`os/src/syscall/net/netdev.rs`、`os/src/syscall/net/sendrecv.rs`。
- **证据强度：强**。

### TRAP/IRQ（中断异常）
- **用户态陷入**：RISC-V 与 LoongArch 双架构 `trap_handler`、`handle_user_exception`、`trap_return`。
- **定时器/IPI**：LoongArch 侧 `enable_timer_interrupt`、`enable_ipi_interrupt`、`clear_ipi_interrupt`。
- **内核态陷入**：`os/src/arch/riscv64/trap/handler.rs#trap_from_kernel`。
- **证据强度：弱**；仅见 trap handler 骨架，未见完整中断控制器（PLIC/APLIC/IOAPIC）驱动与通用 IRQ 子系统。

### NET（网络协议栈）
- **底层协议栈**：`os/src/net/mod.rs#new_stack` 使用 smoltcp 的 `Interface`、`IpAddress`、`Ipv4Address`；`poll_busy_with_sockets_mut_in` 轮询接口；`parse_ipv4_src_dst` / `parse_ipv6_src_dst` 解析报文头。
- **Socket 文件抽象**：`os/src/fs/net_socket.rs`（231 函数）实现 TCP/UDP/Netlink/Packet/WireGuard socket 的文件接口；`new_tcp_with_domain`、`udp_recv_from`。
- **系统调用层**：`os/src/syscall/net/socket.rs`、`sendrecv.rs`、`netlink.rs`、`netdev.rs`、`wireguard.rs`。
- **证据强度：syscall/socket 层较强；原创 TCP/IP 协议栈底层证据未体现**，主要依赖 smoltcp。

1. **设备驱动证据薄弱**：仅见 `virtio_blk` 与 `console`；未见通用 PCI 枚举、USB、GPU、声卡、NVMe、SATA、键盘/鼠标等驱动，也未见完整中断控制器（PLIC/APLIC/IOAPIC）驱动。

3. **ext4 文件系统主体为外部 crate**：`ext4-fs/` 与 `ext4-fs-packer/` 共 194 个函数，应视为移植/第三方组件，不能等同于 `os/` 目录内的原创内核文件系统。
4. **中断/异常子系统不完整**：仅有架构相关 trap handler 与定时器使能，未见通用 IRQ 子系统、中断路由、设备中断注册框架。
5. **“异步运行时”证据未明确**：虽然模块画像包含“任务调度/异步运行时”，但函数名与调用链以传统抢占式调度、阻塞、睡眠、futex、epoll 为主，**未见 async/await、Future、Waker、Reactor 等异步运行时核心证据**。
6. **架构支持范围**：代码同时出现 RISC-V64 与 LoongArch64 路径，但未见 x86_64/ARM/AArch64 证据。
7. **本报告仅基于 AST Call Map、函数名、路径与调用链进行技术描述**；所有“可能方向”均为保守推断，不构成对作品身份、原创性或合规性的最终结论。
