## 第四部分：分模块架构说明

### MM（内存管理）

- **证据锚点**：
  - `os/src/mm/memory_set.rs#from_elf_lazily`
  - `os/src/mm/memory_set.rs#from_elf`
  - `os/src/mm/memory_set.rs#check_valid_user_vpn_range`
  - `os/src/arch/la64/trap/mod.rs#trap_handler`
  - `os/src/arch/riscv64/trap/mod.rs#trap_handler`
- **说明**：支持按 ELF 程序头懒加载/直接构建地址空间，包含 `MapArea`、权限检查、用户态虚拟地址范围校验；页故障处理在 `la64` 与 `riscv64` 两个架构目录中均有体现，调用链中出现 `PageFaultCause`、`SigInfo::new`、`VirtAddr`、`PageTable::from_token` 等，说明具备用户态缺页异常分发与 SIGSEGV 生成能力。但大页、swap、COW 等高级机制在提供的证据中未明确体现。

### PROC（进程管理/任务调度）

- **证据锚点**：
  - `os/src/task/task.rs#kernel_clone`
  - `os/src/syscall/task.rs#sys_execveat`
  - `os/src/syscall/task.rs#sys_exit_group`
  - `os/src/syscall/task.rs#deal_zombie_task`
  - `os/src/task/processor.rs#run_tasks`
  - `os/src/syscall/stub_fd.rs#sys_pidfd_send_signal`
  - `os/src/futex/futex.rs#futex_wait`
- **说明**：存在 `kernel_clone` 构建任务、`execveat` 加载新程序、`exit_group` 终止线程组、`deal_zombie_task` 回收僵尸并发送 `SIGCHLD`，以及 `run_tasks` 的调度循环。`task.rs` 中出现 `tgid`、`tid`、`thread_group`、`parent` 等字段，`sys_getppid` 体现父子关系。因此传统进程管理（fork/exec/wait/zombie/PCB/父子关系）有底层证据支撑。但调度算法细节（CFS/RR/优先级）在证据中未充分展开。

### FS（文件系统）

- **证据锚点**：
  - `os/src/fs/namei.rs#filename_lookup_inner`
  - `os/src/fs/namei.rs#open_last_lookups`
  - `os/src/fs/fanotify.rs#queue_event`
  - `os/src/fs/fdtable.rs#fcntl`
  - `os/src/fs/mount.rs#read_proc_mounts`
  - `os/src/fs/proc/mod.rs#init_procfs`
  - `os/src/fs/dev/mod.rs#init_devfs`
  - `os/src/ext4/mod.rs#mknod`
  - `os/src/ext4/inode.rs#write_direct`
- **说明**：文件系统是本项目最强模块。包含 VFS 层（dentry、inode、file、fdtable、mount）、路径解析、fanotify 事件队列、procfs/devfs 伪文件系统，以及一个可读写 ext4 实现（extent、truncate、mkdir、rename、mknod）。`proc_la2000` 与 `proc` 双目录暗示对 LoongArch 平台的 procfs 有定制。但日志（journal）、软恢复、配额等 ext4 高级特性在证据中未体现。

### DEV（设备驱动）

- **证据锚点**：
  - `os/src/drivers/net/la2000/drv_eth.rs#eth_init`
  - `os/src/drivers/net/starfive/drv_eth.rs#eth_init`
  - `os/src/drivers/net/starfive/drv_eth.rs#eth_rx`
  - `os/src/drivers/net/starfive/drv_eth.rs#eth_tx`
  - `os/src/arch/la64/drivers/pci.rs#init`
- **说明**：设备驱动证据主要集中在网络设备：StarFive GMAC、LoongArch la2000 以太网、VirtIO-net（`MmioTransport`、`VirtIOHeader`），以及 PCI 枚举。字符设备通过 devfs 注册（null/zero/tty/rtc/loop）。但块设备驱动（NVMe/SATA/SD）、显示、音频、USB、定时器之外的中断控制器等**底层证据未体现**。

### SYSCALL（系统调用）

- **证据锚点**：
  - `os/src/syscall/mod.rs#syscall`
  - `os/src/syscall/fs.rs#sys_read`
  - `os/src/syscall/fs.rs#sys_write`
  - `os/src/syscall/fs.rs#sys_pread`
  - `os/src/syscall/fs.rs#sys_pwrite`
  - `os/src/syscall/sched.rs#sys_sched_getaffinity`
  - `os/src/syscall/net.rs#syscall_sendmsg`
- **说明**：`syscall` 函数规模 19007、调用数 245，是统一系统调用分发入口，覆盖 fs、task、sched、net、epoll、signal、mm 等类别。`copy_from_user`/`copy_to_user` 在大量调用链中出现，说明具备用户态/内核态数据拷贝校验。但具体系统调用号表、兼容层（32/64 兼容）证据未给出。

### TRAP/IRQ（中断异常）

- **证据锚点**：
  - `os/src/arch/la64/trap/mod.rs#trap_handler`
  - `os/src/arch/la64/trap/mod.rs#kernel_trap_handler`
  - `os/src/arch/la64/trap/mod.rs#init`
  - `os/src/arch/la64/trap/timer.rs#set_next_trigger`
  - `os/src/arch/riscv64/trap/mod.rs#trap_handler`
  - `os/src/arch/riscv64/trap/mod.rs#kernel_trap_handler`
  - `os/src/arch/la64/register/base/estat.rs#cause`
- **说明**：存在 LoongArch64 与 RISC-V64 两套异常入口，包含异常原因解析（`EStat::cause`）、定时器中断设置、页故障/非法指令处理。但模块函数总量仅 42，说明中断控制器（PLIC/APLIC/IOINTC）、多核 IPI、时钟中断之外的设备中断框架**证据较弱或底层证据未体现**。

### NET（网络协议栈）

- **证据锚点**：
  - `os/src/net/socket.rs#read`
  - `os/src/net/socket.rs#recv_from`
  - `os/src/net/tcp.rs#connect`
  - `os/src/net/tcp.rs#accept`
  - `os/src/net/udp.rs#bind`
  - `os/src/net/loopback.rs#snoop_tcp_from_ip`
  - `os/src/net/mod.rs#snoop_tcp_packet`
- **说明**：网络栈采用 `SOCKET_SET` + `LISTEN_TABLE` + `TcpSocket`/`UdpSocket` 的抽象，风格接近 smoltcp。支持 TCP/UDP、loopback、Unix 域套接字（`os/src/net/unix.rs#passwd_lookup` 等）。但协议栈是否完全自研或基于第三方库，需要与源码中的 `smoltcp` 依赖声明核对，不能仅凭函数名判定为原创内核主体。
