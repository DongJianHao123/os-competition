## 第四部分：分模块架构说明

### MM（内存管理）

底层证据显示内核采用**页式内存管理 + Buddy 页分配器 + Slab 对象缓存**的组合：
- `src/mm/slab.c#kmem_cache_alloc_nolock` 调用 `buddy_alloc`、`slab_alloc`，说明 Slab 建立在 Buddy 之上。
- 提供 `kmalloc`/`kfree`/`kzalloc` 等内核堆接口，广泛用于 VFS、dentry、syscall 路径。
- 用户态内存管理证据：`sys_mmap`、`sys_mremap`、`sys_brk` 使用 `PAGE_ALIGN_UP`/`PAGE_ALIGN_DOWN`、`find_free_user_range`；进程创建时使用 `MAKE_SATP` 构造 SATP。
- 用户态/内核态数据拷贝：`copyin`/`copyout`/`copyinstr` 在几乎所有系统调用中出现。

### PROC（进程管理/任务调度）

证据支持**传统 Unix 风格进程模型**：
- PCB 分配：`alloc_proc`、`alloc_pid`、`free_proc`。
- 进程创建：`sys_fork`、`sys_clone`（调用 `fd_table_dup`、`dentry_get`、`enqueue_proc`）、`sys_execve`（调用 `copyin_str_array`、`file_lseek`、加载镜像）。
- 进程退出与等待：`sys_exit`、`reap_zombie_threads`、`find_child_by_pid`、`wait_queue_wakeup_addr`。
- 同步原语：等待队列 `wait_queue_sleep`/`wait_queue_wakeup_*`、futex `sys_futex`/`futex_requeue_addr`。
- 调度器：`enqueue_proc`、`dequeue_proc`、`sched_yield`、`context_switch_yield`，配合 `r_sstatus`/`w_sstatus` 开关中断，属于基于运行队列的调度器。

> 注：虽然模块画像标签包含“异步运行时”，但底层证据中**未见 `async`/`await`、`waker`、`ring_fifo`、任务 Future 等异步运行时原语**，实际体现的是传统任务调度器。

### FS（文件系统）

文件系统证据较强，分为 VFS 与具体文件系统两层：
- **VFS**：`vfs_mount_root`、`vfs_mount_at`、`vfs_umount_at`、`vfs_path_walk`、`vfs_open_cwd`、`dentry_alloc`/`dentry_insert`/`dentry_free`。
- **ext4**：`ext4_mount`、`ext4_lookup`、`ext4_create`、`ext4_unlink`、`ext4_rename`、`ext4_mkdir`、`ext4_write_file`、`ext4_add_dir_entry`。
- 文件抽象：`file_alloc`、`file_read`、`file_write`、`file_lseek`、`fd_get`/`fd_install`/`fd_free`。
- 特殊文件：`open_dev_rtc_file`、`open_dev_special_file`、`open_proc_mounts_file`，说明实现了部分 `/dev`、`/proc` 伪文件。

### DEV（设备驱动）

设备驱动集中在块设备、VirtIO 与中断控制器：
- 块设备层：`blk_register`、`blk_read`、`blk_write`、`blk_flush`，通过主/次设备号 `MAJOR`/`MINOR` 查找设备。
- VirtIO-MMIO：`virtio_mmio_probe`、`virtio_mmio_init`、`virtio_negotiate_features`、`virtio_irq_handler`。
- VirtIO-blk：`virtio_blk_init_one`、`virtio_blk_isr`。
- VirtIO-net：`virtio_net_init_one`、`virtio_net_init`（仅初始化，未见完整协议栈）。
- PLIC：`plic_claim`、`plic_complete`、`plic_disable`、`plic_set_threshold`。
- Timer：`handle_timer` 调用 `sbi_set_timer`。

### SYSCALL（系统调用）

系统调用是代码量最大的模块：
- 分发：`syscall` 调用 `syscall_lookup` 查表；`syscall_register`/`syscall_init` 注册。
- 文件类：`sys_openat`、`sys_read`、`sys_write`、`sys_pread64`、`sys_close`、`sys_lseek`、`sys_dup3`、`sys_fcntl`、`sys_mount`。
- 进程类：`sys_fork`、`sys_clone`、`sys_execve`、`sys_exit`、`sys_wait4`、`sys_kill`、`sys_tgkill`、`sys_getpid`、`sys_getppid`、`sys_gettid`。
- 内存类：`sys_mmap`、`sys_mremap`、`sys_brk`。
- 同步类：`sys_futex`、`sys_set_tid_address`、`sys_set_robust_list`。
- 网络类 socket 接口：`sys_bind`、`sys_listen`、`sys_connect`、`sys_setsockopt`（仅 fd 层分发，无协议栈证据）。

### TRAP/IRQ（中断异常）

RISC-V 异常/中断处理证据完整：
- 初始化：`trap_init` 设置 `w_sie`/`w_sstatus`。
- 用户态陷入：`usertrap` 读取 `r_scause`/`r_sepc`/`r_stval`，分发系统调用、定时器、外部中断。
- 返回用户态：`usertrapret` 设置 `w_sepc`/`w_sstatus`/`MAKE_SATP`。
- 外部中断：`handle_external` 经 PLIC 分发到 `virtio_blk_isr`/`virtio_net_isr`。
- 定时器：`handle_timer` 调用 SBI 设置下一次时钟。

### NET（网络协议栈）

**底层证据未体现完整的网络协议栈**。现有证据仅包括：
- 系统调用层对 socket fd 的校验与分发：`sys_bind`、`sys_listen`、`sys_connect`、`sys_setsockopt` 调用 `S_ISSOCK`、`fd_get`、`file_put`。
- VirtIO-net 设备初始化：`virtio_net_init_one`、`virtio_net_init`。

没有 TCP/IP、ARP、路由表、sk_buff、网络设备发送/接收队列等底层实现证据。

5. **高级文件系统特性**：ext4 已实现基本读写/目录操作，但日志（journal）、extent、快照、配额等高级特性无证据。
