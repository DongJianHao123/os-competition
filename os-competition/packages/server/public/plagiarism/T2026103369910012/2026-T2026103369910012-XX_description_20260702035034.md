## 第四部分：分模块架构说明

### MM（内存管理）
底层证据体现为：用户地址空间区域管理、多种 mmap 后端（匿名分配、文件映射、COW、线性、共享）、ELF 加载器、用户栈 auxv 构造、`msync`/`mprotect` 类语义、SysV 共享内存、用户指针访问封装（`kernel/src/mm/access.rs`）。  
关键锚点：`kernel/src/syscall/mm/mmap.rs#sys_mmap`、`kernel/src/mm/loader.rs#load_user_app`、`kernel/src/mm/aspace/backend/cow.rs#alloc_new_at`、`kernel/src/syscall/ipc/shm.rs#sys_shmat`。  
**未体现**：物理页分配器（伙伴系统）、页错误/缺页异常处理器的架构相关代码、TLB/Cache 管理。

### PROC（进程管理/任务调度）
底层证据体现为：进程表 `PROCESS_TABLE`、进程组 `PROCESS_GROUP_TABLE`、会话 `SESSION_TABLE`、任务表 `TASK_TABLE`；支持 `clone/fork`、`execve`、`exit/exit_group`、`waitpid`、信号掩码/动作、`pidfd`、`procfs` 暴露 `/proc/[pid]/...`。  
关键锚点：`kernel/src/syscall/task/clone.rs#do_clone`、`kernel/src/syscall/task/execve.rs#sys_execve`、`kernel/src/task/ops.rs#do_exit`、`kernel/src/syscall/task/wait.rs#sys_waitpid`、`kernel/src/pseudofs/proc.rs#lookup_child`。  
**说明**：任务调度核心（runqueue、上下文切换汇编、时钟 tick 调度）在当前仓库中多为 `axtask` 外部 crate 的封装，底层调度器实现证据不足。

### FS（文件系统）
证据最强：包含 `axfs-ng-local` 提供的高层 VFS 与 ext4 实现，以及内核内的 `procfs`、`devfs`、`tmpfs`、mount/umount、fd 操作、路径解析、管道/splice/fallocate、文件锁fcntl 等。  
关键锚点：`axfs-ng-local/src/highlevel/file.rs#open`、`axfs-ng-local/src/fs/ext4/inode.rs#create`/`read_dir`/`metadata`、`kernel/src/pseudofs/proc.rs#lookup_child`、`kernel/src/pseudofs/dev/mod.rs#builder`、`kernel/src/syscall/fs/mount.rs#sys_mount`、`kernel/src/syscall/fs/fd_ops.rs#sys_openat`、`kernel/src/syscall/fs/io.rs#sys_splice`。  
**说明**：`axfs-ng-local` 属于“文件系统/第三方库”类别，应单独标注，不宜直接等同于 kernel 主体原创文件系统。

### DEV（设备驱动）
证据体现为伪设备层：`/dev/tty`、`/dev/pts`、`/dev/input/event*`、`/dev/loop*`、`/dev/random`、`/dev/log`、framebuffer 相关路径等，均通过 `kernel/src/pseudofs/dev/` 下的文件系统节点暴露。  
关键锚点：`kernel/src/pseudofs/dev/tty/mod.rs#ioctl`、`kernel/src/pseudofs/dev/tty/terminal/ldisc.rs#read`/`new`、`kernel/src/pseudofs/dev/tty/pty.rs#create_pty_pair`、`kernel/src/pseudofs/dev/event.rs#ioctl`、`kernel/src/pseudofs/dev/loop.rs#ioctl`。  
**未体现**：真实硬件总线驱动（PCI/USB）、块设备控制器、DMA、中断驱动的设备模型。

### SYSCALL（系统调用）
存在集中式分发器 `kernel/src/syscall/mod.rs#handle_syscall`，覆盖 fs、net、ipc、signal、task、mm、time、keyring、io_mpx 等 150 个函数，使用 `Sysno` 与 `LinuxError` 进行 Linux 兼容映射。  
关键锚点：`kernel/src/syscall/mod.rs#handle_syscall`、`kernel/src/syscall/fs/ctl.rs#sys_readlinkat`、`kernel/src/syscall/sync/futex.rs#sys_futex`、`kernel/src/syscall/task/clone.rs#do_clone`。

### TRAP/IRQ（中断异常）
**底层证据未体现**。当前仓库仅见 3 个与 signal/termios 相关的辅助函数（`kernel/src/file/signalfd.rs#from_signal_info`、`kernel/src/pseudofs/dev/tty/terminal/termios.rs#signo_for`/`default`），无 trap vector、异常分发、IRQ 控制器、页错误入口等代码。

### NET（网络协议栈）
仅见 socket 系统调用层：`sys_socket`、`sys_socketpair`、`sys_accept4`、`sys_sendmsg`、`recv_impl`、地址解析、cmsg 处理，以及 `SocketInner::Tcp/Udp/Unix`、`DgramTransport`、`StreamTransport`、`UnixSocket` 等抽象。  
关键锚点：`kernel/src/syscall/net/socket.rs#sys_socket`、`kernel/src/syscall/net/io.rs#sys_sendmsg`、`kernel/src/syscall/net/addr.rs#read_from_user`。  
**底层证据未体现**：TCP/IP 状态机、ARP/ICMP/IP 分片、NIC 驱动、数据包收发路径。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块/能力
1. **中断/异常/TRAP 入口**：无 trap vector、异常分发、页错误处理、系统调用入口汇编、IRQ 控制器（APIC/PLIC/GIC）代码。
2. **CPU 调度器核心**：`axtask` 外部 crate 的 runqueue、上下文切换、时钟 tick 调度在当前仓库未展开。
3. **网络协议栈底层**：无 TCP/IP 状态机、ARP/ICMP/IP 分片、NIC 驱动、skb/数据包收发路径。
4. **真实硬件设备驱动**：无 PCI/USB/SATA/NVMe/块设备控制器/DMA 驱动。
5. **物理内存管理**：无伙伴系统、slab 分配器、页表遍历/TLB  shootdown 的架构相关代码。
6. **文件系统底层缓存与日志**：无通用页缓存、ext4 journal、写回策略证据。
7. **安全/LSM/Capabilities**：仅有 `setuid/setgid/prctl` 等简单系统调用，未见完整 capability、seccomp、LSM 钩子。
8. **多架构/SMP 启动**：仅见 `kernel/src/entry.rs#init` 与 `src/main.rs#main`，无 boot 汇编、per-CPU、SMP 初始化证据。
9. **时钟硬件驱动**：`timer.rs` 使用 `ALARM_LIST`，但硬件 timer/clocksource 驱动未体现。

### 需要单独说明的内容
- **`axfs-ng-local/` 属于“文件系统/第三方库”**：其 ext4 与高层 VFS 实现虽证据充分，但应视为独立 crate/本地库，不宜直接等同于 `kernel/` 主体原创文件系统。
- **“任务调度/异步运行时”函数数量多**：主要体现为 futex、epoll、poll、TTY 等异步阻塞封装，而非完整的自研调度器。
- **本报告仅基于 AST Call Map、函数名、路径锚点与调用链**，所有“可能的系统方向”均为保守推测，不构成最终身份或原创性结论，也不做抄袭/违规判断。
