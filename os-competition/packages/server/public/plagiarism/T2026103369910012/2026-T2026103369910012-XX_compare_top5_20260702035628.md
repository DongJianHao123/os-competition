#### 4. 核心代码调用流对比剖析
* **【进程管理/进程退出路径】**
* **底层技术析理**: 目标项目 `do_exit` 使用 `FutexKey::new_current` 处理 robust futex、`SignalInfo::new_kernel` 构造内核信号、`clear_child_tid.vm_write(0)` 直接写入用户态内存、`SHM_MANAGER.lock().clear_proc_shm` 清理共享内存。参考项目 `do_exit` 则使用 `FD_TABLE.clear` 清空文件描述符表、`UserPtr::<Pid>::from` 操作用户态指针、`clear_child_tid.get_as_mut` 获取可变引用、`axtask::exit` 触发调度器退出。目标项目以 `vm_write` 为核心的用户态内存访问模型与参考项目以 `UserPtr` 为核心的模型形成根本性架构差异。
