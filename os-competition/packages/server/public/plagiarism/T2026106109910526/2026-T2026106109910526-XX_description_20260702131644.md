## 第四部分：分模块架构说明

### MM（内存管理）
底层证据较强。内核堆使用 slab 分配器（`os/src/mm/allocator/slab_allocator.rs#dealloc`、`dealloc_by_layout`），按 16/128/1024 等尺寸分级缓存。用户虚拟地址空间通过 `UserPtrRaw` / `UserSliceRaw` 进行用户态指针校验与拷贝（`os/src/mm/user.rs`、`os/src/mm/vm/uvm.rs`）。`sys_process_vm_readv/writev` 可直接跨进程访问目标 `VmSpace`。EXT4 inode 使用页缓存（`Page::new`、`cache_read_at`、`cache_write_at`）。ELF 加载路径支持动态解释器 `/lib/libc.so`（`uvm.rs#load_dl_interp_if_needed`）。

### PROC（进程管理/任务调度）
底层证据较强。`os/src/task/task.rs` 中的 `TaskControlBlock` 维护 fork、do_exit、handle_zombie、reparent_children、thread_group、process_group 等状态。`os/src/syscall/process.rs` 实现 `sys_clone`、`sys_fork`、`sys_execve`、`sys_waitpid`、`sys_waitid`、`sys_prctl`、capabilities（`sys_capget`/`capset`）、seccomp（`load_seccomp_filter`、`set_seccomp_mode`）、credentials（`setreuid`/`setregid`/`getresuid`/`getresgid`/`setgroups`/`getgroups`）。`os/src/task/task.rs#exit_robust_list` 与 `os/src/syscall/futex.rs` 实现 futex/robust list。信号管理见 `os/src/signal/manager.rs` 与 `sys_rt_sigprocmask`/`sys_rt_sigtimedwait`。

### 任务调度/异步运行时
底层证据较强。`os/src/executor/mod.rs` 提供 `spawn`、`kernel_spawn`、`run_until_idle`、`enqueue_on_processor`，使用 `TASK_QUEUE` 与 `async_task::spawn`。`os/src/processor/schedule.rs` 提供 `load_balance`、`migrate_tasks`、`update_load_avg`，`os/src/processor/processor.rs` 提供 `switch_out_current_task`、`update_current_timeline`。`sys_sched_getaffinity`/`setaffinity` 支持 CPU 亲和性。

### FS（文件系统）
底层证据较强。VFS 层在 `os/src/fs/vfs/`（`dentry.rs#find`、`file.rs#open_file`、`fs.rs#open_or_create_file`），全局 `DCACHE` 缓存 dentry。具体文件系统包括 EXT4（`os/src/fs/ext4/`）、procfs（`os/src/fs/procfs/`）、devfs（`os/src/fs/devfs/`）以及 tmpfs（`TmpInode`/`TmpDentry`/`TmpSysInode`）。挂载由 `sys_mount`/`sys_umount2` 经 `FS_MANAGER` 处理。系统调用覆盖 `openat`、`open_by_handle_at`、`name_to_handle_at`、`renameat2`、`getdents64`、`utimensat`、`pread`/`pwrite`、`readv`/`writev`、`preadv2`/`pwritev2`、`fallocate`、`fcntl` 等。

### DEV（设备驱动）
底层证据较强。串口：`os/src/devices/serial.rs#get_serial`、`scan_char_device`。网络设备：`os/src/devices/net.rs#receive/preprocess`、`os/src/devices/manager.rs#init_net`、`os/src/drivers/net/virtio_net.rs#receive/recycle_rx_buffer`。块设备：`os/src/drivers/block/pci_blk.rs#try_new`（VirtIO PCI blk）、`os/src/drivers/block/mmc/mod.rs#card_init`（MMC/SD）。设备统一通过 `DEVICE_MANAGER` 管理。

### SYSCALL（系统调用）
底层证据强。统一入口 `os/src/syscall/mod.rs#syscall`（254 条内部调用），按模块分发到 `fs.rs`（111 函数）、`process.rs`（78）、`net.rs`（52）、`mm.rs`、`io.rs`、`sche.rs`、`futex.rs`、`fd.rs` 等，呈现 Linux-compatible ABI 特征。

### TRAP/IRQ（中断异常）
底层证据中等。`os/src/trap/mod.rs#user_trap_handler` / `kernel_trap_handler` 处理用户态/内核态陷入，调用 `TIMER_MANAGER.check()` 与 `DEVICE_MANAGER`。HAL 层对 RISC-V PLIC（`hal/src/component/irq/riscv64.rs`）和 LoongArch EIOINTC/PLATIC（`hal/src/component/irq/loongarch64/`）进行抽象；`hal/src/component/trap/loongarch64/mod.rs#save/restore` 处理上下文保存。

### NET（网络协议栈）
底层证据较强，但需区分第三方库。套接字层在 `os/src/net/socket.rs` 与 `os/src/syscall/net.rs`。TCP/UDP/RAW 分别由 `os/src/net/tcp.rs`、`os/src/net/udp.rs`、`os/src/net/raw.rs` 实现，底层调用 `smoltcp` 的 `SOCKET_SET.with_socket_mut`、`poll_interfaces`。`os/src/net/mod.rs#modify_packet` 进行以太网/IPv4/TCP 包解析与监听表处理。`os/src/net/crypto.rs` 提供类似 `AF_ALG` 的加密套接字接口。**smoltcp 为第三方网络栈，不应视为本项目完全原创的网络协议实现。**
