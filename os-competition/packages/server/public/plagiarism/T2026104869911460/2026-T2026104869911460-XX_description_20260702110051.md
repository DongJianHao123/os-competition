## 第四部分：分模块架构说明

### MM（内存管理）
底层证据较强。关键锚点包括：
- `NoAxiom/kernel/src/syscall/mm.rs#sys_mremap`：处理虚拟地址重映射、TLB 刷新。
- `NoAxiom/kernel/src/mm/validate.rs#validate`：基于当前页表根 `Arch::current_root_ppn()` 遍历页表项。
- `NoAxiom/kernel/src/task/execve.rs#execve` 与 `NoAxiom/kernel/src/task/fork.rs#inner_fork` 中均调用 `MemorySet::load_elf` 与地址空间克隆逻辑。
- 共享内存：`NoAxiom/kernel/src/syscall/mm.rs#sys_shmctl` 操作 `SHM_MANAGER`。
- 页表与地址转换由 `NoAxiom/lib/arch/src/{rv64,la64}` 提供硬件抽象。

### PROC（进程管理/任务调度）
底层证据强，覆盖传统 Unix 进程模型：
- 创建：`NoAxiom/kernel/src/task/fork.rs#inner_fork` 克隆 PCB、命名空间、futex 队列。
- 执行：`NoAxiom/kernel/src/task/execve.rs#execve` 加载 ELF 并初始化陷阱上下文。
- 退出：`NoAxiom/kernel/src/task/exit.rs#exit_handler` 清理任务、唤醒 futex 等待者、向父进程发 `SIGCHLD`。
- 等待：`NoAxiom/kernel/src/task/wait.rs#take_ready_child` 处理 `wait4/waitid`。
- 同步：`NoAxiom/kernel/src/syscall/process.rs#sys_futex` 操作 `FUTEX_SHARED_QUEUE`。
- 信号：`NoAxiom/kernel/src/syscall/signal.rs#__sys_sigtimedwait`、`sys_sigprocmask`。
- 命名空间与权限：`sys_unshare`、`sys_setns`、`sys_capget/capset`、uid/gid 系列调用。
- 异步等待：`NoAxiom/kernel/src/io/async_bridge.rs#poll_wait` 体现任务阻塞与 waker 机制。

**调度核心**：`sched_getaffinity/setaffinity` 与 `ksync` 异步原语有体现，但具体 CFS/优先级调度器实现细节在提供的 Call Map 中**底层证据未充分体现**。

### FS（文件系统）
底层证据最强：
- VFS 抽象：`NoAxiom/kernel/src/fs/vfs/basic/dentry.rs#mknodat_son`、`NoAxiom/kernel/src/fs/vfs/basic/file.rs#read_dir`。
- 初始化：`NoAxiom/kernel/src/fs/vfs/mod.rs#vfs_init` 挂载 RamFs、AsyncSmpExt4、AsyncSmpFat32、DevFs。
- 具体实现：ramfs、rust_fat32、ext4、procfs、devfs 均有对应 `dentry.rs`/`file.rs`。
- 系统调用：`sys_openat`、`sys_renameat2`、`sys_splice`、`sys_ioctl`、`sys_mount` 等覆盖打开、重命名、管道拼接、挂载。
- procfs：`NoAxiom/kernel/src/fs/vfs/impls/proc/mod.rs#init` 生成 `/proc` 下大量伪文件（stat、status、fd、maps、meminfo 等）。

**第三方库**：`NoAxiom/lib/fatfs` 与 `NoAxiom/lib/ext4_rs` 是文件系统核心解析库，属于外部/移植代码，不应视为原创内核主体。

### DEV（设备驱动）
底层证据中等：
- 块设备：`NoAxiom/lib/driver/src/impls/block/vf2_sdcard/mod.rs#init_sdcard` 及寄存器操作；`NoAxiom/lib/driver_ahci/src/drv_ahci.rs#ahci_exec_ata_cmd`。
- 网络设备：`NoAxiom/lib/driver/src/impls/net/loopback.rs#new` 提供回环网卡。
- 设备树：`NoAxiom/lib/driver/src/probe/dtb.rs#dtb_init` 解析 DTB。
- devfs：`NoAxiom/kernel/src/fs/vfs/impls/devfs/mod.rs#init` 创建 null/zero/full/tty/rtc/loop 等字符设备节点。

更广泛的 PCI、USB、GPU、音频、ACPI 等驱动在提供的证据中**底层证据未体现**。

### SYSCALL（系统调用）
底层证据强：
- 入口：`NoAxiom/kernel/src/syscall/syscall.rs#syscall_inner` 是统一分发器，调用链覆盖 200+ 具体系统调用。
- 子模块：`fs.rs`、`process.rs`、`mm.rs`、`net.rs`、`ipc.rs`、`signal.rs`、`sched.rs`、`time.rs`、`io.rs` 分别实现文件、进程、内存、网络、IPC、信号、调度、时间、IO 多路复用（epoll/pselect/ppoll）。

### TRAP/IRQ（中断异常）
底层证据中等：
- 用户态陷阱：`NoAxiom/kernel/src/trap/utrap_handler.rs#user_trap_handler` 处理异常、外部中断、信号。
- 内核态陷阱：`NoAxiom/kernel/src/trap/ktrap_handler.rs#kernel_trap_handler` 分发中断/异常/软中断。
- IPI：`NoAxiom/kernel/src/trap/soft_int.rs#ipi_handler` / `send_ipi`。
- 架构层：`NoAxiom/lib/arch/src/rv64/trap.rs#rv_kernel_trap_handler` 与 `trap_init`，以及 `la64/context.rs` 陷阱上下文打印，说明支持 RISC-V 64 与 LoongArch 64。

### NET（网络协议栈）
底层证据较弱：
- socket 适配层：`NoAxiom/kernel/src/net/tcpsocket.rs#accept`/`poll_inner`、`NoAxiom/kernel/src/net/udpsocket.rs#bind`/`connect`。
- 使用 `SOCKET_SET`、`TCP_PORT_MANAGER`、`UDP_PORT_MANAGER` 等结构，类型名（`TcpSocket`、`Interface`、`IpAddress`）与 smoltcp 高度相似，说明协议核心很可能依赖外部网络栈，而非自研完整 TCP/IP。
- 仅 `NoAxiom/kernel/src/net/mod.rs#net_udp_test` 等测试函数体现基本回环测试。
