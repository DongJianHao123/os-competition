## 第四部分：分模块架构说明

### MM（内存管理）

底层证据较强。核心实现分布在 `modules/axalloc`、`modules/axmem`、`modules/axruntime` 及用户库 `ulib/axlibc` 中。

- 物理内存分配：`modules/axalloc/src/lib.rs#alloc` 使用 `balloc.alloc` / `balloc.add_memory`，按 2 的幂对齐分配。
- 内核启动初始化：`modules/axruntime/src/lib.rs#init_allocator` 调用 `axalloc::global_init`，基于 `memory_regions` 建立可用内存区域。
- 虚拟地址空间：`modules/axmem/src/memory_set.rs#from_elf`、`load_elf`、`apply_relocations` 构建 `MemorySet`、`MapArea`、`VirtAddr`、`VirtPageNum`，完成 ELF 段映射与重定位。
- 缺页处理：`modules/axruntime/src/trap.rs#handle_page_fault` 访问 `KERNEL_SPACE`、构造 `MapPermission`。
- 用户堆：`ulib/axlibc/src/malloc.rs#realloc` 实现带控制块的堆分配器。
- 资源回收：`modules/axprocess/src/id.rs#dealloc_user_res` 释放 `memory_set`、trap context、用户栈等。

### PROC（进程管理/任务调度）

底层证据充分，具备传统 Unix 式进程管理。

- 进程控制块：`modules/axprocess/src/process.rs#fork`、`exec_with_interpreter`、`wait_pid` 中出现 `MemorySet::from_existed_user`、`RecycleAllocator`、`SignalFlags`、`BTreeMap`、`Mutex`、父子进程关系、退出码等。
- 调度切换：`modules/axprocess/src/processor.rs#run_tasks_once`、`schedule` 调用 `__switch`、`set_current_task_ptr`、`drop_task_ptr`。
- 任务状态机：`modules/axprocess/src/lib.rs#block_current_and_run_next`、`suspend_current_and_run_next`、`yield_current_and_run_next`、`exit_current_and_run_next`。
- 等待/通知：`modules/axtask/src/wait_queue.rs#wait_timeout_until`、`notify_task`、`notify_one`、`notify_all`。
- 同步原语：`modules/axprocess/src/semaphore.rs`、`modules/axprocess/src/condvar.rs`。
- 信号：`modules/axsyscall/src/signal.rs#syscall_rt_sigprocmask`、`syscall_rt_sigreturn`。
- 入口：`examples/init/src/main.rs#spawn_user_process` 加载 ELF 并创建用户进程。

### FS（文件系统）

文件系统是代码量最大的模块，证据强。

- VFS 层：`modules/axfs/src/init.rs#init_fs` 创建根 `Dentry`、`MountNamespace`、`PageCache`。
- 挂载与路径解析：`modules/axfs/src/vfs/mount/namespace.rs#mount`、`read_mounts`；`modules/axfs/src/vfs/namespace/mount.rs#mount` 操作 `BLOCK_DEVICES`、`FS.lock`、`MOUNT_NS.mount`；`modules/axfs/src/vfs/namespace/open.rs#lookup_open_path`、`build_file`。
- EXT4：`modules/axfs/src/fs/ext4fs.rs#read_dir`、`read_blocks`、`write_blocks`、`create`、`lookup`、`link`、`rename`、`unlink`，使用缓存回写机制。
- procfs：`modules/axfs/src/fs/builtin.rs#procfs` 提供 `/proc` 风格节点。
- 用户态验证：`user/apps/shell/src/main.rs#execute_program`、`user/apps/ls/src/main.rs#ls_path`、`user/apps/fs_bench/src/main.rs` 系列测试。

### DEV（设备驱动）

证据以驱动抽象和工具链为主，具体硬件驱动实现细节在提供的片段中偏少。

- 驱动初始化：`modules/axdriver/src/lib.rs#init_drivers` 通过 `AllDevices::probe` 枚举 block/display/net 设备。
- 内存区域：`modules/axhal/src/mem.rs#common_memory_region_at` 描述 text/rodata/data/bss/percpu/mmio 区域。
- 平台启动：`tools/raspi4/chainloader/src/main.rs#kernel_main` 及 `tools/raspi4/chainloader/src/bsp/device_driver/bcm/bcm2xxx_pl011_uart.rs#init` 为 Raspberry Pi 4 链式加载器与 UART 初始化。
- 原始网络设备：`tools/bwbench_client/src/device.rs#new`、`bind_interface`、`recv` 使用主机 `libc::socket` 做性能测试。
- 中断注册：`modules/axhal/src/irq.rs#register_handler_common`、平台相关 `register_handler`。

### SYSCALL（系统调用）

证据强，呈现 Linux 兼容系统调用层。

- 总入口：`modules/axsyscall/src/lib.rs#syscall` 进行 `SyscallId` 解析、信号检查、抢占检查。
- 进程：`modules/axsyscall/src/task.rs#syscall_execve`、`syscall_fork`、`syscall_wait4`、`syscall_exit`、`syscall_getpid` 等。
- 文件：`modules/axsyscall/src/fs/syscall.rs#syscall_getdents64`、`syscall_readlinkat`、`syscall_chdir`、`syscall_getcwd`、`syscall_utimensat`。
- 网络：`modules/axsyscall/src/net/syscall.rs#syscall_recvmsg`、`syscall_sendmsg`、`syscall_recvfrom`、`syscall_getsockopt`。
- 信号：`modules/axsyscall/src/signal.rs#syscall_rt_sigprocmask`、`syscall_rt_sigreturn`。
- 多路复用：`modules/axsyscall/src/select.rs#syscall_pselect6`。
- 杂项：`modules/axsyscall/src/misc.rs#syscall_uname`、`syscall_prlimit64`。

### TRAP/IRQ（中断异常）

证据集中在架构相关陷阱分发。

- RISC-V：`modules/axhal/src/arch/riscv/trap.rs#riscv_trap_handler` → `handle_irq`、`handle_syscall`、`handle_signal`、`handle_page_fault`、`check_preempt_pending`。
- LoongArch：`modules/axhal/src/arch/loongarch/trap.rs#loongarch_trap_handler`，调用链与 RISC-V 类似。
- 中断使能：`modules/axhal/src/arch/riscv/mod.rs#enable_irqs` 设置 `sie` / `sstatus`。
- 注册：`modules/axhal/src/irq.rs#register_handler_common`、平台 `register_handler`。

### NET（网络协议栈）

网络协议栈原创实现证据较弱，主要是对 **smoltcp** 的封装。

- TCP/UDP/DNS：`modules/axnet/src/smoltcp_impl/tcp.rs#connect/recv/send/shutdown`、`udp.rs#bind`、`dns.rs#query`、`listen_table.rs#incoming_tcp_packet`、`mod.rs#snoop_tcp_packet`。
- Socket 抽象：`modules/axfilelike/src/impls/net/socket.rs#wait_event`。
- 用户接口：`ulib/axstd/src/net/tcp.rs#bind/connect/accept`、`ulib/axstd/src/net/udp.rs#bind/send_to`、`ulib/axlibc/c/network.c#inet_pton`、`ulib/axlibc/c/socket.c#accept4`。
- 应用验证：`user/apps/echo_server/src/main.rs#main`、`user/apps/remote_server/src/main.rs#main`。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块/特性

- **从零实现的 TCP/IP 协议栈**：网络层证据集中在 `modules/axnet/src/smoltcp_impl/*`，底层为第三方 `smoltcp` 封装，原创协议实现证据不足。
- **块设备驱动具体实现**：`init_drivers` 仅枚举 `block/display/net`，未体现 virtio-blk、SDHCI、NVMe 等具体控制器驱动的实现细节。
- **显示/GPU 驱动内部**：`axdisplay::init_display` 被调用，但驱动寄存器操作、帧缓冲管理等底层证据未体现。
- **高级内存管理**：swap、按需分页的磁盘回写、共享内存、mmap（除 ELF 加载外）、写时复制（COW）细节未在片段中直接体现。
- **SMP/多核调度**：仅见 `INITED_CPUS.fetch_add`，未体现 per-CPU 运行队列、核间中断、负载均衡等。
- **安全与隔离机制**：capabilities、seccomp、PID/网络/用户命名空间（mount namespace 除外）、cgroups 等底层证据未体现。
- **动态链接与加载器**：未体现 ELF interpreter、PLT/GOT 动态重定位、用户态动态链接器。
- **总线与外设**：USB、PCI、I2C、SPI、音频、电源管理、ACPI/设备树解析等底层证据未体现。
- **调试/跟踪设施**：除 `trace_store.rs#record_failure` 外，未体现 KGDB、perf、ftrace 等。

### 需要单独说明的第三方/公共代码

- `modules/axnet/src/smoltcp_impl/*` 明显基于 **smoltcp** 第三方网络栈。
- `modules/axfs/src/fs/ext4fs.rs` 属于“文件系统/第三方库”类别，应视为 EXT4 第三方实现或深度集成的封装。
- `ulib/axlibc/c/*` 包含大量类 musl 的 C 运行时函数（如 `math.c`、`stdio.c`、`network.c`、`socket.c`、`pow.c`），属于公共依赖或移植代码，不应直接等同于内核原创主体。

### 判断边界

- 本报告仅基于 AST Call Map、函数名、模块路径与调用链描述技术事实。
- “可能的系统方向”仅为代码结构暗示，**不是**对作品最终身份、原创性或参赛资格的结论。
- 未对任何代码的抄袭/违规情况进行判断。
