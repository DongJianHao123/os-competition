## 第四部分：分模块架构说明

### MM（内存管理）
- **证据较强**。缺页处理 `kernel/trap/{riscv,loongarch}/trap.c#pagefault_handler` 使用 `kalloc`、`kfree`、`mappages` 并通过 `get_fops()->readat` 从文件系统按需加载页面；`kernel/proc/exec.c#execve` 在加载程序时重建地址空间（`PGROUNDDOWN/PGROUNDUP`、`clear_vmas_on_exec`、`copyout`、`i_op->read`）；系统调用路径使用 `copyin/copyout` 进行用户/内核地址拷贝；`kernel/mem/slab.c` 提供 slab 分配器，`kernel/mem/buddysystem.c#mark_parent` 暗示伙伴系统存在。
- **未体现**：swap、页替换算法、完整 mmap/文件映射、NUMA、内存 cgroup 等高级机制。

### PROC（进程管理/任务调度）
- **传统进程管理证据充分**。`kernel/proc/proc.c` 包含 `clone`、`fork`、`exit`、`wait`、`scheduler`、`sched`、`yield`、`sleep`、`wakeup` 等；`kernel/proc/exec.c#execve` 实现程序执行；`kernel/proc/signal.c#handle_signal` 实现信号处理；`kernel/proc/pipe.c`、`semaphore.c`、`socket.c` 提供 IPC 机制；系统调用层有 `sys_clone`、`sys_exit`、`sys_wait4`、`sys_set_tid_address`、`sys_set_robust_list`、`sys_membarrier` 等 Linux 风格接口。
- **任务调度**：基于 `scheduler` + `sched` + `swtch` 的传统时间片/事件驱动调度，timer tick 触发 `wakeup`。
- **异步运行时**：**底层证据未体现**（无 async/await、waker、ring_fifo、task queue 等异步运行时原语）。

### FS（文件系统）
- **证据极强**。原创 VFS 层位于 `kernel/fs/vfs/`（`file.c`、`ops.c` 等），通过 `get_fops`、`vfs_ext_*` 等适配到 ext4；`kernel/fs/ext4/lwext4/` 为集成的第三方 lwext4 库，提供块分配、inode 分配、extent、目录索引、journal 等完整 ext4 实现；块缓存层 `kernel/driver/bio.c` 衔接到底层 VirtIO 磁盘。
- **注意**：`kernel/fs/ext4/lwext4/` 属于第三方库实现，不应视为原创内核主体。

### DEV（设备驱动）
- **证据较强**。块设备：`kernel/driver/{loongarch,riscv}/virtio_disk.c` 实现 VirtIO-PCI 块设备初始化与中断处理；字符设备：`kernel/mem/uart.c` 实现串口输出；中断控制器：`kernel/sys/plic.c#plicinithart`、`apic_init`、`extioi_init` 分别对应 RISC-V PLIC 与 LoongArch APIC/EXTIOI。
- **未体现**：网络适配器、GPU、USB、声卡等非块/非串口设备驱动。

### SYSCALL（系统调用）
- **证据中等但覆盖面较全**。参数解析集中在 `kernel/sys/syscall.c`（`argint`、`argaddr`、`argstr`、`argraw` 等）；文件类系统调用在 `kernel/sys/sysfile.c`（`sys_openat`、`sys_readv`、`sys_writev`、`sys_unlinkat`）；进程类在 `kernel/sys/sysproc.c`（`sys_clone`、`sys_exit`、`sys_wait4`、`sys_sleep` 等）；信号类在 `kernel/sys/syssig.c`（`sys_kill_signal`、`sys_tkill`）；内存类在 `kernel/sys/sysmem.c`（`sys_madvise`）；其他如 `sys_syslog`、`sys_shutdown`。
- 系统调用表/分发函数未在提供的证据中完整展开，但 handler 实现可见。

### TRAP/IRQ（中断异常）
- **证据较强**。双架构分别实现：`kernel/trap/riscv/trap.c` 与 `kernel/trap/loongarch/trap.c` 提供 `trapinit`/`trapinithart`、`usertrap`、`kerneltrap`、`usertrapret`、`pagefault_handler`；`kernel/trap/riscv/trap.c#timertick` 处理定时器中断；设备中断通过 `devintr` 分发到 `virtio_disk_intr` 等。

### NET（网络协议栈）
- **底层证据未体现**。虽然 `kernel/proc/socket.c` 存在 socket API（`recvfrom`、`socketfree`、`bind`、`listen`、`find_udp_remote_socket` 等），但仅为 socket 抽象或本地 IPC，**没有 NIC 驱动、IP/TCP/UDP 协议栈实现、网络缓冲区管理等完整网络栈证据**。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块/机制
1. **完整网络协议栈**：仅有 socket API 外壳，无 NIC 驱动、IP/TCP/UDP 协议实现、网络包收发缓冲区。
2. **异步运行时**：无 async/await、waker、ring buffer、task queue、future 等运行时原语。
3. **高级内存管理**：无 swap、页替换策略、完整 mmap/文件映射、NUMA、内存 cgroup。
4. **安全与隔离机制**：无 capabilities、namespaces、seccomp、LSM、cgroup 等。
5. **非块/非串口设备驱动**：无 GPU、声卡、USB、网络适配器等驱动。
6. **多核调度细节**：虽有 `mycpu`、`cpuid`、`spinlock`，但缺少 per-CPU runqueue、负载均衡、CPU 亲和性等详细证据。
7. **除 ext4 外的其他文件系统**：证据中仅见 ext4 集成。

### 需要单独说明的内容
- `kernel/fs/ext4/lwext4/` 为第三方 lwext4 库代码（537 个函数），属于文件系统/存储栈的重要组成部分，但**不应被夸大为本项目的原创内核主体**。
- “轻量任务调度或异步运行时”仅为保守方向提示；实际证据显示的是**传统进程调度**，异步运行时未体现。
- 本作品支持 **RISC-V 与 LoongArch 双架构**，但具体硬件抽象层完整度需结合全部源码进一步确认。
