## 第四部分：分模块架构说明

### MM（内存管理）

底层证据较强。关键锚点：

- `os/src/mm/memory_set.rs`：地址空间、页表、ELF 加载、COW fork 的核心结构。
- `os/src/syscall/process.rs#sys_brk`：调用 `expand_heap` / `shrink_heap`，实现用户堆动态调整。
- `os/src/syscall/process.rs#sys_msync`：通过 `filesystem::page_cache::file_page_sync_range` 回写共享文件映射页。
- `os/src/mm/shm.rs#shmget` / `remove_id`：System V 共享内存。
- `os/src/task/process.rs#clone_process` / `#exec`：使用 `MemorySet::fork_cow` / `MemorySet::from_elf`。
- `os/src/trap/mod.rs#handle_user_trap`：用户态异常/缺页入口。

可说明的能力：虚拟地址空间、页表、COW fork、堆、共享内存、文件映射页缓存回写。物理页分配器（buddy/slab）的具体实现在提供的锚点中**底层证据未体现**。

### PROC（进程管理）

底层证据强，属于传统进程管理。关键锚点：

- `os/src/task/process.rs#new` / `#clone_process` / `#exec`：PCB 创建、fork/clone、ELF 执行。
- `os/src/task/mod.rs#exit_current_and_run_next` / `#add_signal_to_process`：进程退出、信号投递。
- `os/src/task/manager.rs#insert_into_pid2process` / `#pid2process` / `#all_processes`：全局 PID 映射表。
- `os/src/task/processor.rs#current_process` / `#schedule`：当前进程获取与调度切换。
- `os/src/syscall/process.rs#sys_wait4` / `#sys_exit` / `#sys_clone`：wait/exit/clone 系统调用。

可说明的能力：PCB、父子进程关系、fork/exec/wait/exit、信号、PID 管理。cgroups、namespaces、容器相关机制**底层证据未体现**。

### 任务调度/异步运行时

关键锚点：

- `os/src/syscall/thread.rs#sys_thread_create`：`TaskControlBlock::new` + `add_task`，线程创建并入队。
- `os/src/task/processor.rs#schedule`：`context_switch` 上下文切换。
- `sync/src/condvar.rs#wait_with_mutex` / `os/src/syscall/sync.rs#sys_condvar_wait`：阻塞等待与唤醒。
- `os/src/syscall/sync.rs#sys_mutex_create`：互斥锁创建。

可说明的能力：线程调度、阻塞/唤醒、互斥锁/条件变量。轻量级任务调度有证据，但 **async/await、Future、Waker、Executor 等完整异步运行时底层证据未体现**；`任务调度/异步运行时` 224 的计数中大量为文件系统/同步系统调用，存在标签偏差。

### FS（文件系统）

底层证据强。关键锚点：

- `filesystem/src/vfs_defs/inode.rs`：VFS Inode 定义（96 个函数）。
- `filesystem/src/vfs/manager.rs#resolve_path_impl` / `#umount`：路径解析、挂载/卸载管理。
- `filesystem/src/vfs/file.rs#open`：文件打开、设备文件处理。
- `filesystem/src/vfs/dentry_cache.rs#dcache_lookup` / `#dcache_prune_subtree`：dentry 缓存。
- `filesystem/src/page_cache.rs`：页缓存（`file_page_sync_range`）。
- `filesystem/src/ramfs/inode.rs#rename` / `filesystem/src/procfs/mod.rs#new`：ramfs、procfs。
- `filesystem/ext4_rs/src/ext4_impls/{inode,balloc,extents,file}.rs`：ext4 实现。
- `os/src/fs/fdtable.rs#new_with_stdio`：文件描述符表。

可说明的能力：VFS、dentry cache、mount、fdtable、页缓存、ramfs、procfs、ext4。第三方 `ext4_rs` 应单独视为文件系统后端实现。

### DEV（设备驱动）

关键锚点：

- `os/src/main.rs#main`：`drivers::block::init`、UART、console sink。
- `os/src/drivers/gpu/mod.rs#new`：VirtIO GPU 初始化。
- `os/src/drivers/net/mod.rs#receive`：网卡收包。
- `patches/polyhal/src/arch/x86_64/apic.rs#init`：x86_64 APIC 初始化。

可说明的能力：块设备、串口、VirtIO GPU、网络 NIC、中断控制器初始化。USB、声卡、PCI 总线枚举（除 GPU/网卡外）、SATA/NVMe 原生驱动**底层证据未体现**。

### SYSCALL（系统调用）

关键锚点：

- `os/src/syscall/mod.rs#syscall`：统一系统调用分发器（174 条调用链，规模最大）。
- `os/src/syscall/fs.rs`：`sys_openat`、`sys_read`、`sys_write`、`sys_fsync`、`sys_mount`、`sys_renameat2` 等。
- `os/src/syscall/process.rs`：`sys_clone`、`sys_execve`（通过 `exec`）、`sys_wait4`、`sys_exit`、`sys_brk`。
- `os/src/syscall/thread.rs#sys_thread_create`。
- `os/src/syscall/sync.rs` / `ipc.rs`：mutex、condvar、semaphore。
- `os/src/syscall/stub.rs`：部分未实现或占位系统调用。

可说明的能力：覆盖 POSIX 文件、进程、线程、同步、IPC 等大类系统调用。

### TRAP/IRQ（中断异常）

关键锚点：

- `os/src/trap/mod.rs#handle_user_trap`：用户态陷入，处理 IRQ、时钟、信号。
- `os/src/trap/mod.rs#handle_kernel_trap`：内核态异常，处理定时器、内核页 fault panic。
- `patches/polyhal-trap/src/trapframe/riscv64.rs#fmt`：RISC-V 上下文帧。
- `patches/polyhal/src/arch/x86_64/apic.rs#init`：x86_64 APIC/中断控制器初始化。

可说明的能力：用户/内核 trap、时钟中断、信号投递、上下文帧、中断控制器。部分被归类到“中断/异常处理”的函数实际属于 VFS，需注意统计偏差。

### NET（网络协议栈）

关键锚点：

- `patches/lose-net-stack/src/connection/tcp.rs#{send_data,close,connect,accept,peer_writable,is_closed}`：TCP 连接管理。

可说明的能力：TCP/UDP/ARP、socket 抽象、网卡收发。主要协议栈代码位于 `patches/lose-net-stack`，应与 `os/src/net` 的 socket 层区分。

## 第六部分：边界与未体现内容

1. **第三方/补丁代码边界**  
   - `filesystem/ext4_rs/` 为 ext4 文件系统第三方实现；  
   - `patches/polyhal*` 为硬件抽象层与启动代码；  
   - `patches/lose-net-stack/` 为网络协议栈补丁/第三方实现。  
   这些不应直接等同于 `os/` 与 `filesystem/src/` 原创内核主体。

2. **统计标签偏差**  
   - “任务调度/异步运行时”224 个函数中，大量来自 `os/src/syscall/fs.rs` 等文件系统调用，不能单独支撑“完整异步运行时”结论；  
   - “中断/异常处理”356 个函数中，部分为 VFS 路径启发式归类，需以 `os/src/trap/` 与 `polyhal-trap` 实际函数为准。

3. **没有底层证据支撑的模块/机制**  
   - 完整 async/await 异步运行时（Future、Waker、Executor）；  
   - 安全机制：MAC、Capabilities、SELinux 类框架；  
   - 加密子系统；  
   - 音频/USB 子系统；  
   - PCI 总线枚举与管理（除 VirtIO GPU/网卡已使用的外设）；  
   - SATA/NVMe 原生驱动；  
   - 电源管理、虚拟化/Hypervisor、内核调试器；  
   - 容器机制：cgroups、namespaces；  
   - 多 CPU 调度细节（APIC 初始化有证据，但 per-CPU runqueue、IPI 等调度细节未直接体现）。

4. **判断性质说明**  
   本文档仅基于 AST Call Map、函数名、路径锚点与调用链描述技术事实；“可能的系统方向”仅为方向推断，不是最终身份或原创性结论。本文档不做抄袭/违规判定。
