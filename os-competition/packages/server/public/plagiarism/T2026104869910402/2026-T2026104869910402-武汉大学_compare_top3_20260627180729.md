#### 4. 核心代码调用流对比剖析

*   **【进程管理 / 系统调用 `sys_clone`】**
    *   **底层技术析理**: 此调用是创建新进程/线程的核心入口。目标项目在系统调用入口处即进行复杂的标志位解析（`clone_flags_from_raw`）和退出信号处理（`clone_exit_signal`），随后调用面向对象的 `Process::clone_process` 方法，体现了 Rust 的所有权传递和错误处理模式（`Ok`）。参考项目则是一个薄封装，直接委托给 C 语言过程式的 `clone` 函数，参数处理和逻辑均在 `clone` 内部完成。两者在抽象层次、职责划分和调用流上完全不同。

    *   **底层技术析理**: 两者均实现进程/任务的阻塞与唤醒。目标项目实现了基于 `Mutex` 和 `Condvar` 的高级同步原语，调用流涉及 `mutex.lock`、`wait_queue.push_back`、`task_ops().schedule` 等，将锁机制与调度分离。参考项目则是传统内核的 `sleep` 实现，通过 `acquire`/`release` 自旋锁保护睡眠条件，并直接调用 `sched()` 进行上下文切换。两者的同步模型（用户态可用的 Condvar vs 内核态自旋锁）和调度抽象层截然不同。
    *   **代码溯源链接**:
