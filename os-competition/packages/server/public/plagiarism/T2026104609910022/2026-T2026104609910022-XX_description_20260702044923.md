## 第四部分：分模块架构说明

### MM（内存管理）
- **已体现**：虚拟地址翻译、内核页表初始化、用户缓冲区映射、ELF 加载时的内存布局。
- **证据锚点**：
  - `src-la/mm/memory_set.rs#translated_refmut` → `virt_to_phys`
  - `src-la/main.rs#loongarch_main` → `enable_paging`、`init_direct_map`、`set_tlb_refill_entry`
  - `src/task/elf.rs#load_elf` / `src-la/task/elf.rs#load_elf` → `MemorySet::new_bare`、`KERNEL_SPACE.lock().token`
- **未直接体现**：物理页框分配器（buddy/slab）、swap、COW 页错误处理细节等底层证据未体现。

### PROC（进程管理/任务调度）
- **已体现**：完整传统进程管理，包括 `fork/clone`、`execve`（含 shebang 与动态解释器加载）、`wait/waitid`、`exit_group`、zombie、父子关系、信号掩码、定时器睡眠。
- **证据锚点**：
  - `src-la/syscall/process.rs#sys_clone` → `TaskControlBlock::clone_thread`、`TaskControlBlock::fork`、`add_task`
  - `src-la/syscall/process.rs#sys_execve` → `parse_shebang`、`load_interpreter`、`elf_interpreter`
  - `src-la/task/mod.rs#exit_current_and_run_next` → `mgr.exit_process`、`clear_child_tid_and_wake`
  - `src-la/task/processor.rs#run_tasks` → `fetch_task`、`__switch`、`mgr.switch_to_process`
- **调度**：基于 timer 中断的抢占 + `block/suspend/sleep_current_until` 的协作式阻塞，**不是 async/await 运行时**。

### FS（文件系统）
- **已体现**：统一的 `FS_MANAGER` 文件抽象层，支持 EXT4 后端、内存节点（`/tmp`、pipe、eventfd、epoll、timerfd、signalfd、socket fd）、`/proc/self/fd` 等特殊路径、fd 表复制/重定向。
- **证据锚点**：
  - `src-la/fs/manager.rs#open` → `FileHandle::Ext4`、`FileHandle::Mem`、`proc_self_fd`
  - `src-la/fs/manager.rs#read` / `#write` → 多类型 fd 分发
  - `src-la/fs/manager.rs#fd_link_target` → `/dev/stdin`、`/dev/stdout`、`/dev/null` 等
  - `src-la/fs/ext4.rs#lookup` / `#read_file` / `#list_root_dir`
- **第三方/移植代码**：`fs/ext4.rs` 与 `drivers/block.rs#read_ext4_file_range` 属于 EXT4 解析强约束代码，可能为第三方或移植实现，应单独标注。

### DEV（设备驱动）
- **已体现**：VirtIO 块设备（MMIO 与 PCI Legacy 两种探测路径）、SBI 控制台输入输出。
- **证据锚点**：
  - `src-la/drivers/block.rs#init_block_device` → `VirtioPciBlock::probe_first` / `VirtioMmioBlock::probe_first`
  - `src-la/drivers/block.rs#transfer_sector` / `#read_sector`
  - `src/console.rs#write_str` / `src-la/console.rs#write_str` → `sbi::console_putchar`
- **未体现**：中断控制器（PLIC/LS7A IOAPIC）、时钟源之外的其他字符设备、网络设备、块缓存层细节。

### SYSCALL（系统调用）
- **已体现**：大量 Linux 兼容系统调用分发，覆盖文件、进程、信号、定时器、poll/epoll、socket fd 等。
- **证据锚点**：
  - `src-la/syscall/mod.rs#syscall`：120 条分支，调用 `fs::sys_*`、`process::sys_*`
  - `src-la/syscall/fs.rs#sys_openat` / `#sys_newfstatat` / `#sys_readlinkat`
  - `src-la/syscall/process.rs#sys_clone` / `#sys_execve` / `#sys_wait4`

### TRAP/IRQ（中断异常）
- **已体现**：用户态/内核态 trap 入口、timer 中断使能、定时器到期处理、内核态异常 panic。
- **证据锚点**：
  - `src/trap/mod.rs#trap_handler` / `src-la/trap/mod.rs#trap_handler`
  - `src/trap/mod.rs#init` / `src-la/trap/mod.rs#init`
  - `src-la/loongarch_csr.rs#enable_timer_interrupt`
- **未体现**：外部设备中断（除 timer 外）、异常分类处理（如 page fault 专用路径证据不足）。

### NET（网络协议栈）
- **底层证据未体现**：仅存在 socket fd 抽象的系统调用（`sys_sendto`、`sys_sendmsg`、`sys_accept4`、`sys_socketpair`），由 `FS_MANAGER` 管理，**没有 TCP/IP 协议栈、网卡驱动、网络缓冲区管理等底层证据**。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块/能力
1. **完整网络协议栈**：仅有 socket fd 系统调用封装，无 TCP/IP、网卡驱动、协议状态机证据。
2. **物理内存分配器细节**：`KernelStack::alloc`、`MemorySet` 存在，但 buddy/slab/页框分配器未直接体现。
3. **块缓存与页缓存**：文件系统读写直接操作 EXT4/内存节点，未体现通用块缓存或页缓存层。
4. **多核/SMP**：无 CPU 核间中断、核间调度、自旋锁等证据。
5. **安全/权限模型**：无 UID/GID、capability、ACL、mmap 权限检查等证据。
6. **除 timer 外的中断控制器与外部 IRQ**：PLIC/IOAPIC、设备中断注册未体现。
7. **高级文件系统特性**：日志（journal）、软链接写入、硬链接、文件锁等未直接体现。
8. **用户态 C 库**：`user/` 为用户测试/库封装，不是内核主体。

### 需要单独说明的代码
- `fs/ext4.rs`、`drivers/block.rs#read_ext4_file_range` 等属于 EXT4 文件系统格式强约束代码，**可能是第三方库或移植代码**，不应直接等同于原创内核主体。

### 判断性质
- 本文档所有结论均基于 AST Call Map、函数名、模块路径与调用链；`src` 与 `src-la` 的双架构判断为“可能方向”，不代表最终硬件平台或原创性结论。
- 不做抄袭、违规或身份归属判断。
