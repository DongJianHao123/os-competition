## 第四部分：分模块架构说明

### MM（内存管理）

底层证据仅体现 **RISC-V SBI 远程 TLB/ASID/VMID 失效封装**，没有页表、页分配器、虚拟内存映射、物理内存管理等代码。

- 证据锚点：
  - `libs/sbi/sbi_remote_fence.c#sbi_remote_sfence_vma_asid` → `sbi_ecall`
  - `libs/sbi/sbi_remote_fence.c#sbi_remote_hfence_gvma_vmid` → `sbi_ecall`
  - `libs/sbi/sbi_legacy.c#sbi_legacy_remote_sfence_vma_asid` → `(umb_t), sbi_ecall`

这些函数用于向其他 Hart 或当前 Hart 发送 TLB/ASID/VMID 失效请求，属于内存一致性/地址转换缓存维护接口，而非完整内存管理子系统。

### PROC（进程管理/任务调度）

**底层证据未体现**。未观察到 `fork`、`exec`、`wait`、`exit`、`PCB`、`task_struct`、`scheduler`、`zombie` 处理、父子进程关系等结构或调用。

### FS（文件系统）

**底层证据未体现**。未观察到 VFS、inode、`open`/`read`/`write`/`close` 文件路径解析、块缓存、文件描述符表、磁盘分区等代码。

### DEV（设备驱动）

仅存在 **SBI 级抽象接口**（控制台、定时器、IPI、远程 fence），没有 MMIO 寄存器操作、中断控制器驱动、总线枚举、设备模型等。

- 证据锚点：
  - `libs/sbi/sbi_dbcn.c#sbi_dbcn_console_write` → `(umb_t), sbi_ecall`
  - `libs/sbi/sbi_timer.c#sbi_set_timer` → `(umb_t), sbi_ecall`

### SYSCALL（系统调用）

统计标签为“系统调用”的函数，实际底层证据均为 **RISC-V SBI ecall 封装**，不是传统操作系统意义上的用户态-内核态系统调用分发层。

- 证据锚点：
  - `libs/sbi/sbi.c#sbi_ecall`：内部闭环逻辑，所有 SBI 调用的共同入口
  - `libs/sbi/sbi_base.c#sbi_probe_extension` → `(umb_t), sbi_ecall`
  - `libs/sbi/sbi_base.c#sbi_get_spec_version` → `sbi_ecall`
  - `libs/sbi/sbi_dbcn.c#sbi_dbcn_console_write` → `(umb_t), sbi_ecall`
  - `libs/sbi/sbi_legacy.c#sbi_legacy_shutdown` → `sbi_ecall`

### TRAP/IRQ（中断异常）

**底层证据未体现**。未观察到 trap 向量表、`scause`/`sepc` 处理、PLIC/CLINT 中断控制器驱动、异常分发等代码。仅通过 SBI 使用定时器与 IPI 服务。

### NET（网络协议栈）

**底层证据未体现**。未观察到 socket、TCP/IP、网卡驱动、缓冲区管理、协议分层等代码。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块

- **完整内存管理子系统**：无页表遍历/创建、物理页分配器（buddy/slab）、`kmalloc`/`kfree`、虚拟地址空间管理。
- **进程管理/任务调度**：无 `fork`、`exec`、`wait`、`exit`、PCB、调度器、优先级、时间片、父子关系。
- **文件系统**：无 VFS、inode、目录缓存、块设备、文件描述符、路径解析。
- **设备驱动框架**：无 MMIO 寄存器访问、中断控制器（PLIC/CLINT）驱动、DMA、总线扫描。
- **中断与异常处理**：无 trap 入口、异常向量表、`scause`/`sepc`/`stval` 处理。
- **网络协议栈**：无 socket、TCP/IP、网卡驱动、网络缓冲区。

### 需要单独说明的依赖

- `third_party/libs/libfdt/` 是成熟的开源设备树库，**不应视为该作品的原创内核主体**，其功能是为系统提供 FDT 解析与 overlay 能力。
- `libs/sbi/` 中的代码主要是 **RISC-V SBI 协议封装**，将上层请求转换为 `ecall` 指令，本质上是固件接口适配层，而非内核实现的系统调用或驱动。

### 判断边界

- “RISC-V 底层系统软件 / SBI 客户端 / 最小内核原型”仅为基于代码路径的**可能方向**，不是对该作品最终身份或完整功能的结论。
- 本文档仅依据 AST Call Map、函数名、模块路径与调用链描述技术事实，**不做抄袭、违规或原创性判定**。
