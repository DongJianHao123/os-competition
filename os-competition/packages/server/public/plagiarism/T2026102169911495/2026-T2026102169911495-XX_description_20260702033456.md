## 第四部分：分模块架构说明

### MM（内存管理）

已实现用户态地址空间管理、多种映射后端、ELF 加载与共享内存接口。

- **地址空间后端**：`kernel/src/mm/aspace/backend/cow.rs#alloc_new_at` 实现了写时复制（COW）文件映射，调用 `file.read_at`、`phys_to_virt` 等完成物理页填充。
- **mmap/mprotect**：`kernel/src/syscall/mm/mmap.rs#sys_mmap` 支持 `Backend::new_alloc/cow/file/linear/shared` 等多种后端；`kernel/src/syscall/mm/mmap.rs#sys_mprotect` 调用 `aspace.protect` 修改页保护。
- **ELF 加载**：`kernel/src/mm/loader.rs#load_user_app` 与 `load` 负责从文件系统加载 ELF 并建立用户地址空间。
- **共享内存**：`kernel/src/syscall/ipc/shm.rs#sys_shmat` / `sys_shmdt` 通过 `SharedPages` 与 `aspace.map/unmap` 实现 SysV SHM。

**未体现**：底层页表操作、物理页分配器、缺页异常处理、swap 机制等底层证据未体现，实际可能依赖 `axhal`/`axaddr-space` 等底层 crate。

### PROC（进程管理/任务调度）

已实现进程/线程表、clone/fork/execve/exit/wait、pidfd、信号、进程组与会话。

- **生命周期**：`kernel/src/syscall/task/clone.rs#do_clone` 创建任务并写入 `parent_tid`、操作 `FD_TABLE.scope_mut`；`kernel/src/syscall/task/execve.rs#do_execve` 替换地址空间并重置文件表；`kernel/src/task/ops.rs#do_exit` 清理 futex、信号、共享内存；`kernel/src/syscall/task/wait.rs#sys_waitpid` 等待子进程。
- **进程关系**：`kernel/src/task/ops.rs#add_task_to_table` 维护 `PROCESS_TABLE`、`PROCESS_GROUP_TABLE`、`SESSION_TABLE`；`kernel/src/syscall/task/thread.rs#sys_getpid/sys_getppid` 读取进程父子关系。
- **pidfd**：`kernel/src/syscall/fs/pidfd.rs#sys_pidfd_open` / `sys_pidfd_getfd` / `sys_pidfd_send_signal` 提供 pidfd 能力。
- **信号**：`kernel/src/task/signal.rs#send_signal_to_process` / `send_signal_to_process_group` 与 `kernel/src/syscall/signal.rs#sys_rt_sigprocmask` 实现信号投递与掩码。
- **futex/robust list**：`kernel/src/task/ops.rs#handle_futex_death` 与 `kernel/src/task/futex.rs#wait_if` 支持线程同步原语。

**调度说明**：实际线程调度大量调用 `axtask::yield_now`、`axtask::spawn_with_name`、`axtask::set_current_affinity`，底层调度器证据来自 `axtask` 封装而非本作品原创的调度核心。

### FS（文件系统）

文件系统是本作品代码量最大的模块，覆盖 VFS、ext4、tmpfs、procfs、devfs、mount、pipe、splice 等。

- **VFS 高层**：`patches/axfs-ng/src/highlevel/file.rs#open` 负责路径解析、symlink 处理、打开文件。
- **ext4**：`patches/axfs-ng/src/fs/ext4/inode.rs#create` / `read_dir` / `update_metadata` 实现 ext4 目录/文件创建、读目录、元数据更新；`patches/axfs-ng/src/fs/ext4/fs.rs#new` 挂载 ext4。
- **伪文件系统**：
  - procfs：`kernel/src/pseudofs/proc.rs#lookup_child` 动态生成 `/proc/<pid>/stat|status|fd|maps|...`
  - tmpfs：`kernel/src/pseudofs/tmp.rs#new`
  - devfs：`kernel/src/pseudofs/dev/mod.rs#builder` 创建 `/dev/tty`、`/dev/random`、`/dev/event` 等
- **系统调用**：`kernel/src/syscall/fs/ctl.rs#sys_readlinkat`、`kernel/src/syscall/fs/fd_ops.rs#sys_openat` / `sys_fcntl` / `sys_close_range`、`kernel/src/syscall/fs/io.rs#sys_splice` / `sys_fallocate` 等。

**第三方说明**：`patches/axfs-ng/` 属于补丁/第三方库改造，不应视为本作品原创内核主体。

### DEV（设备驱动）

主要为伪设备与字符设备抽象，未见真实硬件驱动。

- **TTY/PTY**：`kernel/src/pseudofs/dev/tty/mod.rs#ioctl`、`kernel/src/pseudofs/dev/tty/terminal/ldisc.rs#new` / `read` / `poll`、`kernel/src/pseudofs/dev/tty/pty.rs#create_pty_pair` 实现终端、线路规程与伪终端。
- **输入/事件设备**：`kernel/src/pseudofs/dev/event.rs#ioctl` 处理 `EVIOCG*` 等 ioctl。
- **loop 设备**：`kernel/src/pseudofs/dev/loop.rs#ioctl` 实现循环挂载控制。
- **随机数/日志**：`kernel/src/syscall/sys.rs#sys_getrandom` 通过 `/dev/random` 读取；`kernel/src/pseudofs/dev/log.rs#bind_dev_log` 创建 `/dev/log` Unix socket。

**未体现**：真实块设备驱动、网卡驱动、中断驱动的底层寄存器操作、DMA 等。

### SYSCALL（系统调用）

系统调用分发与实现非常完整。

- **分发入口**：`kernel/src/syscall/mod.rs#handle_syscall` 包含 220+ 条系统调用分发。
- **文件类**：`kernel/src/syscall/fs/fd_ops.rs#sys_openat` / `sys_close_range`、`kernel/src/syscall/fs/ctl.rs#sys_readlinkat`、`kernel/src/syscall/fs/stat.rs#sys_statx`、`kernel/src/syscall/fs/memfd.rs#sys_memfd_create`。
- **网络类**：`kernel/src/syscall/net/socket.rs#sys_socket` / `sys_accept4`、`kernel/src/syscall/net/io.rs#sys_sendmsg` / `recv_impl`。
- **进程/线程类**：`kernel/src/syscall/task/clone.rs#do_clone`、`kernel/src/syscall/task/execve.rs#sys_execve` / `sys_execveat`、`kernel/src/syscall/task/exit.rs#sys_exit` / `sys_exit_group`、`kernel/src/syscall/task/wait.rs#sys_waitpid`。
- **IPC/同步/资源**：`kernel/src/syscall/ipc/shm.rs#sys_shmat` / `sys_shmdt`、`kernel/src/syscall/ipc/msg.rs#sys_msgget`、`kernel/src/syscall/sync/futex.rs#sys_futex`、`kernel/src/syscall/resources.rs#sys_prlimit64` / `sys_getrusage`。

### TRAP/IRQ（中断异常）

**底层证据未体现**。AST 中仅捕获到 4 个相关函数：

- `kernel/src/file/signalfd.rs#from_signal_info`
- `patches/loongarch64/boot.rs#enable_fp_simd`
- `kernel/src/pseudofs/dev/tty/terminal/termios.rs#signo_for`
- `kernel/src/pseudofs/dev/tty/terminal/termios.rs#default`

这些均非中断/异常处理核心。未见到 trap frame、IRQ controller、时钟中断入口、上下文切换汇编等底层代码。

### NET（网络协议栈）

仅见套接字系统调用层与地址/控制消息处理，未见完整协议栈实现。

- **socket 创建**：`kernel/src/syscall/net/socket.rs#sys_socket` 支持 `Tcp/Udp/Unix/Vsock`；`sys_socketpair` 创建 Unix socket pair。
- **IO**：`kernel/src/syscall/net/io.rs#sys_sendmsg`、`recv_impl` 处理收发；`kernel/src/syscall/net/cmsg.rs#push` 处理 ancillary data。
- **地址解析**：`kernel/src/syscall/net/addr.rs#read_from_user` 转换 `sockaddr_in/in6`。

**未体现**：TCP/IP 协议状态机、ARP/IP/ICMP/路由表、网卡驱动、skb 缓冲区等底层网络协议栈证据；网络部分大概率依赖 `axnet`/`smoltcp` 等外部传输实现。
