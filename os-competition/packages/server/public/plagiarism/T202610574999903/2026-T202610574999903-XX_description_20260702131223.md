## 第四部分：分模块架构说明

### MM（内存管理）
- **已有证据**：`src/trap/context.rs#app_init` 初始化用户态 Trap 上下文；`src/utils/memory.rs#print_memory_map` 在启动时打印内存布局。
- **未体现内容**：页表、MMU、物理/虚拟内存映射、堆分配器、内存分页/分段机制等**底层证据未体现**。当前仅停留在“打印内存信息”与“初始化寄存器上下文”层面。

### PROC（进程管理/任务调度）
- **已有证据**：`src/syscall/process.rs#sys_exit`、`sys_getpid`、`sys_yield` 与 `src/batch.rs#current_pid`。`sys_exit` 直接调用 `run_next_app` 切换到下一个应用。
- **未体现内容**：进程控制块（PCB）、`fork`/`exec`/`wait`/`zombie`、父子进程关系、独立地址空间、进程调度队列等**传统进程管理底层证据未体现**。
- **调度说明**：`src/batch.rs#run_next_app` 是唯一的调度函数，属于顺序批处理切换，**不是异步运行时**，也无时间片轮转、优先级调度、async/waker 等证据。

### FS（文件系统）
- **已有证据**：`src/syscall/fs.rs#sys_write` 将用户缓冲区通过 `sbi::console_putchar` 输出到控制台；`src/syscall/fs.rs#sys_read` 为内部闭环逻辑，疑似空实现/占位。
- **未体现内容**：文件描述符表、VFS、inode、目录、块设备、缓存、磁盘/存储驱动等**真实文件系统底层证据未体现**。`sys_write` 实质是控制台输出，不具备文件系统语义。

### DEV（设备驱动）
- **已有证据**：`src/sbi.rs#console_putchar` 与 `src/sbi.rs#shutdown` 调用 SBI 固件接口；`src/console.rs#write_str` 提供内核格式化打印。
- **未体现内容**：块设备驱动、字符设备框架、中断控制器（PLIC）、定时器（CLINT）、DMA、总线驱动等**底层证据未体现**。控制台输出依赖 RISC-V SBI 固件，不属于原创内核设备驱动主体。

### SYSCALL（系统调用）
- **已有证据**：用户侧封装见 `user/src/syscall.rs#sys_write`、`sys_getpid`、`sys_exit`、`sys_yield`；内核侧入口为 `src/syscall/mod.rs#syscall`，分发至 `sys_write`、`sys_read`、`sys_exit`、`sys_getpid`、`sys_yield`。
- **能力范围**：当前仅支持 5 类系统调用，覆盖控制台输出、进程标识、主动让出与退出。

### TRAP/IRQ（中断异常）
- **已有证据**：`src/trap/mod.rs#init` 设置 Trap 入口；`src/trap/handler.rs#trap_handler` 处理用户态陷入，分发系统调用或切换应用；`src/lang_items.rs#panic` 处理内核不可恢复错误，打印寄存器与回溯。
- **未体现内容**：外部中断控制器初始化、定时器中断、PLIC、多核 IPI、中断嵌套等**底层证据未体现**。

### NET（网络协议栈）
- **底层证据未体现**：源码中未发现任何网络设备驱动、协议栈（TCP/IP）、socket 接口或网络缓冲区管理相关函数。
