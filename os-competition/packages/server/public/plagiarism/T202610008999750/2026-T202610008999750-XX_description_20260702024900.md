## 第四部分：分模块架构说明

### MM（内存管理）
- **已体现证据**：
  - 用户态内存映射：`api/src/imp/mm/mmap.rs#sys_mmap`、`api/src/imp/mm/mmap.rs#sys_mprotect`、`api/src/imp/mm/mmap.rs#sys_munmap`。
  - 堆管理：`api/src/imp/mm/brk.rs#sys_brk` 操作 `process_data` 的 heap top/bottom。
  - ELF 加载：`core/src/mm.rs#load_user_app` 调用 `ElfFile::new` 并处理解释器路径 `CStr::from_bytes_with_nul`。
  - 用户指针校验：`api/src/ptr.rs#check_null_terminated`、`get_as_mut_slice`、`get_as_slice` 使用 `aspace.check_region_access`。
- **未体现/底层证据不足**：物理页分配器、页表自管理、缺页异常处理、swap、TLB 刷新策略等底层机制未在证据中直接体现。

### PROC（进程管理/任务调度）
- **已体现证据**：
  - 进程/线程表：`core/src/task.rs#add_thread_to_table` 维护 `PROCESS_TABLE`、`THREAD_TABLE`、`PROCESS_GROUP_TABLE`、`SESSION_TABLE`。
  - 创建与复制：`api/src/imp/task/clone.rs#sys_clone` 复制/共享 `CURRENT_DIR` 与 namespace；`api/src/imp/task/execve.rs#sys_execve` 替换地址空间。
  - 等待与退出：`api/src/imp/task/wait.rs#sys_waitpid` 按 `WaitPid::Pid/Pgid` 过滤子进程并处理僵尸；`api/src/imp/task/exit.rs#do_exit` 清理 fd 表、futex、发送信号。
  - 关系链：`sys_getpid`、`sys_getppid` 通过 `task_ext().thread.process().pid/parent` 访问。
- **调度器说明**：`axtask::current`、`axtask::sleep`、`axtask::yield_now`、`axtask::spawn`、`axtask::exit` 等调用表明项目使用 `axtask` 提供的调度能力，而非自实现完整调度器。

### FS（文件系统）
- **已体现证据**：
  - 文件描述符层：`api/src/imp/fs/fd_ops.rs#sys_openat`、`sys_dup2`、fd table、`Directory`/`File` 包装。
  - 文件读写与向量 IO：`api/src/imp/fs/io.rs#sys_readv`、`sys_writev`、`sys_sendfile`。
  - 目录与元数据：`api/src/imp/fs/ctl.rs#sys_getdents64`、`write_entry`；`api/src/imp/fs/stat.rs#sys_fstatat`、`sys_statx`。
  - 挂载：`api/src/imp/fs/mount.rs#sys_mount` 调用 `mount_fat_fs`；`sys_umount2` 调用 `umount_fat_fs`。
  - 路径与别名：`api/src/path.rs#new` 做 canonicalize；`api/src/imp/fs/stat.rs#stat_busybox_alias` 处理 busybox applet 别名。
  - 管道/标准 IO/网络文件抽象：`api/src/file/pipe.rs`、`api/src/file/stdio.rs`、`api/src/file/net.rs`。
- **未体现**：块设备驱动、磁盘缓存、具体 FAT 实现（仅见 `mount_fat_fs` 调用，实现位于 `axfs` 或更底层）。

### DEV（设备驱动）
- **底层证据未体现**。统计仅 3 个函数，且均集中在 `api/src/socket.rs` 的 socket 地址解析，不能说明存在字符设备、块设备、中断控制器、DMA 等驱动框架。

### SYSCALL（系统调用）
- **已体现证据**：
  - 统一入口：`src/syscall.rs#handle_syscall` 规模 11763，调用数 118，覆盖大量 `sys_*`。
  - 分类实现：`api/src/imp/fs/*`、`api/src/imp/task/*`、`api/src/imp/mm/*`、`api/src/imp/signal.rs`、`api/src/imp/futex.rs`、`api/src/imp/time.rs`、`api/src/imp/sys.rs` 等。
  - 使用 `Sysno` 进行系统调用号解析。

### TRAP/IRQ（中断异常）
- **底层证据未体现**。统计仅 3 个函数：`api/src/file/pipe.rs#read/write` 中检查 `has_pending_signal`，`api/src/signal.rs#post_trap_callback` 调用 `check_signals`。没有中断向量表、IRQ 控制器、异常分发、上下文保存/恢复等直接证据。`sys_arch_prctl` 涉及 TLS MSR 读写，但属于系统调用层而非异常处理层。

### NET（网络协议栈）
- **底层证据未体现**。统计仅 6 个函数，全部在 `api/src/socket.rs`，仅完成 `sockaddr_in/in6` 的 `read_from_user` / `write_to_user` 与 `family` 判断。`handle_syscall` 中虽出现 `sys_accept`、`sys_bind` 等符号，但无 TCP/IP 协议状态机、套接字缓冲区、ARP/IP/ICMP/TCP 实现证据。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块
1. **中断/异常处理框架**：无中断向量表、IRQ 控制器、异常分发、上下文保存/恢复代码；仅有 `post_trap_callback` 与信号检查。
2. **设备驱动框架**：无字符/块设备注册、无中断处理注册、无 DMA、无具体硬件驱动；`socket.rs` 中的函数属于 socket 地址解析辅助。
3. **网络协议栈**：仅有 `sockaddr_in/in6` 的序列化/反序列化，无 TCP/IP/UDP/ARP/ICMP 协议状态机、无 socket 缓冲区、无网卡驱动交互。
4. **物理内存与页表底层**：`mmap/mprotect` 仅到 syscall 层，无自实现页分配器、页表遍历、缺页异常处理、swap 等证据。
5. **文件系统块层/磁盘驱动**：`mount_fat_fs` 被调用，但具体 FAT 实现与块设备读写位于 `axfs` 或更底层，未在本项目证据中体现。
6. **SMP/多核调度**：无 CPU 核管理、自旋锁、IPI、per-CPU 变量等多核证据。
7. **安全/权限模型**：无 UID/GID、capability、ACL、文件权限检查等证据。

### 需要单独说明的第三方/基础组件
- 代码中大量调用 `axtask::*`、`axfs::*`、`axprocess::*`、`axhal::*`，说明项目构建在这些基础 crate 之上。本描述仅基于调用关系说明“使用了这些能力”，不判断其原创性归属，也不把这些基础组件夸大为本项目的原创内核主体。

### 判断性质说明
- 本方向判断为“保守系统方向提示”与 AST 调用证据的综合推断，**不是最终身份、原创性或抄袭结论**。
- 所有“实现了什么”均基于函数名、调用链、模块路径与文件分布；凡证据不足处已明确标注“底层证据未体现”。
