## 第四部分：分模块架构说明

### MM（内存管理）

- 有明确的 **VMA 查找** 与 **页缓存回写** 路径：`kernel/abi/linux/sys_mm.c#sys_msync` 调用 `mm_find_vma`、`mm_sync_shared_dirty_for_vnode`、`page_cache_writeback_vnode`。
- 用户态与内核态数据拷贝原语普遍存在：`copy_from_user`、`copy_to_user` 出现在大量 syscall 中。
- OOM 机制有选主入口：`kernel/mm/oom.c#oom_pick_victim_pid`。
- 用户态测试 `user/cmds/mm_stress.c#shared_file_read_after_mmap_write` 验证了 `mmap`/`munmap` 语义。
- **未直接体现**：物理页框分配器（buddy/slab）实现细节、页表遍历/TLB 刷新、缺页异常处理的具体代码。

### PROC（进程管理/任务调度）

- 存在完整的进程/线程生命周期管理：
  - 创建：`kernel/abi/linux/sys_proc.c#sys_clone3` -> `proc_clone`
  - 退出：`kernel/proc/exit.c#proc_exit` -> `fdtable_close_all`、`futex_wake_user`、`proc_reparent_children`
  - 等待：`kernel/proc/wait.c#proc_wait4` -> `proc_destroy_task`、`sched`
- 调度器核心：`kernel/proc/sched.c#sched` -> `context_switch`、`proc_runq_pick_locked`、`sched_scan_timers`、`sched_reap_zombies`。
- 信号、fdtable、futex、timer、pidfd 等周边机制均有代码锚点。
- **未体现独立的 async/await 运行时**；所谓“异步运行时”证据主要来自事件等待、底部半部、LWIP 与用户态代码，不能等同于完整协程调度器。

### FS（文件系统）

- **VFS 层**：`kernel/fs/vfs/path_resolution.c#vnode_lookup_path_openat2` 通过 `cur->ops->lookup/readlink` 分发到具体文件系统。
- **ext4 实现**：`kernel/fs/ext4.c#ext4_mount`、`ext4_vn_mkdir`、`ext4_vn_create`、`ext4_dir_add` 等，使用 `bcache` 作为块缓存。
- **管道与 procfs**：`kernel/fs/pipe.c#pipe_wait_interruptible_locked`、`kernel/fs/procfs.c#procfs_fwrite`。
- **Linux ABI 文件接口**：`kernel/abi/linux/sys_fs.c#sys_readv/writev` 通过 `vfs_get_file_ref` 访问文件。
- **未体现**：除 ext4 外的其他文件系统（如 tmpfs、FAT、ISO9660）、日志/JBD2 细节、块设备层完整框架。

### DEV（设备驱动）

- **网络设备**：
  - `kernel/drivers/net/virtio_net.c#virtio_net_send`（VirtIO 描述符环、DMA 同步）
  - `kernel/drivers/net/starfive_gmac.c#starfive_gmac_init`（StarFive GMAC 初始化、PHY）
  - `kernel/drivers/net/ls2k_gmac.c#ls2k_gmac_phy_init`
- **块设备**：
  - `kernel/drivers/block/virtio_blk.c#virtio_blk_wait_req`
  - `kernel/drivers/block/dw_sdio.c#sdio_xfer_block`
- **未体现**：USB、PCI 枚举、图形/声卡/输入设备、中断控制器（PLIC/GIC）驱动细节。

### SYSCALL（系统调用）

- **双 ABI 设计**：
  - 原生 A20 ABI：`kernel/abi/native/sys_core.c#sys_a20_handle_read/write`、`kernel/abi/native/sys_phase2.c#sys_a20_thread_create`
  - Linux ABI：`kernel/abi/linux/sys_proc.c#sys_clone3`、`kernel/abi/linux/sys_fs.c#sys_readv/writev`、`kernel/abi/linux/sys_mm.c#sys_msync`
- A20 ABI 以 **handle table** 为中心：`a20_handle_lookup_internal`、`task_get_a20_ht`、`a20_ht_create`。
- **未体现**：系统调用表完整映射、seccomp、ptrace 完整实现。

### TRAP/IRQ（中断异常）

- 入口函数明确：`kernel/core/trap.c#trap_handler`、`kernel/core/trap.c#kernel_trap_handler`。
- 调用链包含 `arch_handle_irq`、`arch_advance_syscall_epc`、`arch_local_irq_disable/enable` 等架构相关原语。
- **未体现**：具体 CPU 的异常向量表、PLIC/CLINT/GIC 初始化、中断分发表。

### NET（网络协议栈）

- 使用 **LWIP** 作为核心协议栈，集成在 `kernel/external/lwip`：
  - `kernel/external/lwip/src/core/udp.c#udp_input`
  - `kernel/external/lwip/src/core/tcp_in.c#tcp_receive`
  - `kernel/external/lwip/src/core/ipv6/ip6.c#ip6_output_if_src`
- 内核包装层：`kernel/net/socket_inet.c#net_inet_bottom_half_process_all` -> `proc_make_ready`，实现 socket 事件到任务的唤醒。
- 用户态测试：`user/cmds/socket_stress.c#udp_ping_pong`。
- **未体现**：独立的原创 TCP/IP 协议栈（LWIP 为第三方）、网络过滤/防火墙、高级 QoS。
