## 第四部分：分模块架构说明

### MM（内存管理）
- **已体现证据**：用户地址空间构建（`AddrSpace::build_user`）、ELF 加载（`load_elf`）、栈/堆映射（`init_stack` / `map_stack` / `map_heap`）、用户指针安全访问（`UserReadPtr` / `UserWritePtr`）、VMA 结构（`kernel/src/vm/vm_area.rs`）、地址计算（`lib/mm/src/address.rs`）。
- **底层证据未体现 / 较弱**：物理页框分配器、页表遍历与页错误处理流程、swap/缺页置换策略、内存 cgroup/limits 的完整实现。

### PROC（进程管理 / 任务调度）
- **已体现证据**：
  - 进程/线程控制：`kernel/src/task/task.rs#new_fork_clone`、`taskf.rs#fork` / `execve` / `exit`
  - 进程关系：`kernel/src/task/process_manager.rs`（进程组管理）、`kernel/src/task/wait_queue.rs`（等待队列）
  - 信号：`kernel/src/task/signal/sig_exec.rs`、`kernel/src/syscall/signal.rs`
  - futex：`kernel/src/task/futex.rs`
  - pidfd：`kernel/src/task/signal/pidfd.rs`
  - 系统调用：`sys_clone3`、`sys_execve`、`sys_wait4` / `sys_waitid`、`sys_exit_group`、`sys_prlimit64`、`sys_capget` / `sys_capset`
- **调度层面**：存在 `TIMER_MANAGER`、`TASK_MANAGER`、`hart.rs#user_switch_in` 等上下文切换锚点，以及 `IntrBySignalFuture`、`TimeoutFuture`、`PSelectFuture`、`EpollFuture` 等异步 Future，说明内核采用 **异步运行时 + 事件等待** 模型；但具体调度算法（CFS/优先级调度/SMP 负载均衡）的底层证据不足。

### FS（文件系统）
- **已体现证据**：
  - VFS 抽象层：`lib/vfs/src/dentry.rs`、`lib/vfs/src/file.rs`、`lib/vfs/src/path.rs`
  - 具体文件系统：`lib/ext4/`、`lib/fat32/`
  - 伪文件系统：`lib/osfs/src/proc/`（procfs）、`lib/osfs/src/dev/`（devfs/loop）、`lib/osfs/src/special/fscontext/`、`lib/osfs/src/special/io_uring/`
  - 事件通知：`lib/vfs/src/fanotify/mod.rs#publish`、`kernel/src/syscall/fanotify.rs`、`kernel/src/syscall/fs.rs#sys_inotify_init1`
  - 挂载相关：`sys_mount`、`sys_fsmount`、`sys_fsopen`、`sys_move_mount`、`sys_open_tree`
- **说明**：`lib/ext4/` 使用了 `ext4_fopen`、`ExtDir` 等接口，属于对第三方 ext4 库（如 lwext4）的 Rust 封装，应单独标注，不宜直接视为完全原创的文件系统实现。

### DEV（设备驱动）
- **已体现证据**：
  - 设备树（FDT）探测：`kernel/src/osdriver/probe.rs#probe_tree`、`probe_sd_mmc_devices`、`probe_char_device`
  - 串口控制台：`kernel/src/osdriver/mod.rs#probe_serial_console`、`probe_char_device_by_serial`
  - 块设备：`lib/driver/src/block/dw_mshc/mmc.rs`（SD/MMC 主机控制器）
  - 网络设备：`lib/driver/src/net/virtnet.rs`（VirtIO-Net）
- **底层证据未体现**：PCI / USB / SATA / AHCI / 显卡 / 声卡 / DMA 控制器等驱动；完整的设备电源管理框架。

### SYSCALL（系统调用）
- **已体现证据**：
  - 统一入口：`kernel/src/syscall/mod.rs#syscall`，通过 `SyscallNo::from_repr` 分发
  - 文件类：`kernel/src/syscall/fs.rs`、`fsmount.rs`、`fanotify.rs`
  - 进程类：`kernel/src/syscall/process.rs`
  - 其他：`kernel/src/syscall/bpf.rs`（`BpfMap::new`、`BpfProgram::new`）、`kernel/src/syscall/key.rs`（keyring）、`kernel/src/syscall/sche.rs`（sched affinity）
- **说明**：系统调用数量统计仅 36 个函数，说明大量 syscall 为薄封装或按类别聚合在少量文件中；底层参数校验、errno 映射、兼容性细节需结合具体文件进一步确认。

### TRAP/IRQ（中断异常）
- **已体现证据**：
  - 用户态陷阱：`kernel/src/trap/trap_handler/user_trap_handler/{riscv64,loongarch64}.rs#trap_handler`
  - 内核态陷阱：`kernel/src/trap/trap_handler/kernel_trap_handler/{riscv64,loongarch64}.rs#kernel_trap_handler`
  - 定时器中断：`TIMER_MANAGER.check`、`get_time_duration`
- **底层证据未体现**：完整中断向量表、异常恢复路径、SMP 核间中断（IPI）、设备 IRQ 路由与中断亲和性管理。

### NET（网络协议栈）
- **已体现证据**：
  - 套接字管理层：`SOCKET_SET`、`lib/net/src/interface.rs#check_poll`
  - TCP：`lib/net/src/tcp/tcpsr.rs#send` / `recv`、`lib/net/src/tcp/tcpstate.rs#block_on` / `shutdown`、`lib/net/src/tcp/listentable.rs#accept`
  - UDP：`lib/net/src/udp.rs#send_to` / `block_on`
  - 数据包嗅探：`lib/net/src/tcp/mod.rs#snoop_tcp_packet`
- **底层证据未体现**：ARP / ICMP / IP 路由表、网络命名空间、netfilter、完整拥塞控制算法实现。
