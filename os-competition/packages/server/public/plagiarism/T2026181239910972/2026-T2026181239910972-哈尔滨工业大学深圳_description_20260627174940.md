## 第四部分：分模块架构说明

### MM（内存管理）
- **物理内存**：内核使用伙伴系统分配物理页，证据包括 `crates/buddy-system/src/zone.rs#alloc/dealloc` 与 `anemone-kernel/src/mm/frame/buddy.rs#dealloc`，涉及 `FreeBlock`、`ZoneNode`、按 2 的幂次分块。
- **虚拟内存 / 用户空间**：`anemone-kernel/src/mm/uspace/mod.rs` 为用户空间管理核心；`mm/uspace/api/msync.rs#sys_msync`、`mm/uspace/shm/api/shmctl.rs#sys_shmctl` 提供 msync / SysV 共享内存系统调用。
- **缺页与异常**：`arch/riscv64/exception/trap/utrap.rs#rust_utrap_entry` 与 `arch/loongarch64/exception/trap/utrap.rs#rust_utrap_entry` 处理用户态异常，包含 `PageFaultInfo` 构造。
- **未体现**：swap、页缓存完整策略、大页 / NUMA 等底层证据未体现。

### PROC（进程管理 / 任务调度）
- **传统进程管理证据充分**：
  - 创建：`task/api/clone/mod.rs#kernel_clone` 构造新任务上下文并复制 fd 表（`task/files.rs#fork`）。
  - 执行：`task/api/execve/kernel.rs#kernel_execve_from_pathref` 加载 ELF 并建立用户栈（`task/api/execve/binfmt/elf/init_stack.rs#push`）。
  - 退出：`task/api/exit/mod.rs#kernel_exit` 设置 `ThreadGroupLifeCycle::Exited`、发送 `SIGCHLD`、回收 CPU 时间。
  - 等待：`task/api/wait4.rs#try_from_syscall_arg` 支持 wait4 语义。
  - 信号：`task/sig/mod.rs#perform_signal_action` 准备信号栈帧。
  - 进程关系：`task/topology/process_group.rs#create_session`、`move_to_process_group_if` 实现进程组 / 会话。
  - procfs：`fs/proc/tgid/stat.rs#build_stat_line`、`fs/proc/tgid/status.rs#build_status_text` 暴露进程信息。
- **调度**：`sched/wait.rs`、`sched/processor.rs` 提供等待-唤醒、运行队列、本地 / 远程 IPI 唤醒（`remote_wake_enqueue`）；`arch/riscv64/bootstrap.rs#bsp_setup` 中出现 `SchedClassPrv::RoundRobin`。
- **“异步运行时”说明**：底层证据未体现独立的 async/await executor；708 个函数主要来自同步调度原语、I/O 多路复用等待与文件系统异步路径封装，应理解为“任务调度 + I/O 等待框架”，而非完整异步运行时。

### FS（文件系统）
- **VFS 核心**：`fs/mod.rs#mount`、`vfs_rename_at`、路径解析、namespace 测试；`fs/inode.rs`、`fs/file.rs`、`fs/superblock.rs` 提供 inode/file/superblock 抽象。
- **具体文件系统**：
  - ramfs：`fs/ramfs/mod.rs#ramfs_mount`、`fs/ramfs/inode.rs#ramfs_rename`。
  - ext4：`fs/ext4/mod.rs#ext4_mount`、`fs/ext4/inode.rs#ext4_rename`、`fs/ext4/superblock.rs#ext4_load_inode`、`fs/ext4/file.rs#ext4_read_dir/copy_in/load_page`。
  - procfs：`fs/proc/mod.rs#init`、`fs/proc/tgid/*` 提供 `/proc/[pid]/stat|status|cmdline|environ|fd|exe|cwd` 等。
- **系统调用层**：`fs/api/openat.rs#sys_openat`、`fs/api/readlinkat.rs#sys_readlinkat`、`fs/api/read_write/pwritev2.rs#sys_pwritev2`、`fs/api/fallocate.rs#sys_fallocate`、`fs/api/iomux/pselect6.rs#sys_pselect6` 等。
- **第三方 / 外部依赖提示**：ext4 代码存在，但需人工确认是原创实现还是对外部 `ext4` crate 的封装；当前 AST 证据无法判定原创性。

### DEV（设备驱动）
- **设备模型**：`device/bus/mod.rs#register_driver` / `register_device`、KObject、bus type 抽象。
- **PCIe**：`device/bus/pcie/ecam.rs` 提供 ECAM 访问；`driver/pcie_drivers/bus.rs#enum_pcie_bus` / `alloc_resources` 实现总线枚举与 BAR 资源分配；`driver/pcie_drivers/platform.rs#probe` 解析设备树并建立 root complex。
- **VirtIO**：`driver/virtio/pcie.rs#probe` 探测 VirtIO over PCIe 设备。
- **串口 / 控制台**：`driver/serial/ns16550a.rs#probe`、`device/console.rs#on_system_boot`。
- **中断控制器**：`driver/intc/loongson_platic.rs#init`。
- **设备树**：`crates/device-tree/src/parser.rs#parse`、`unflattened.rs` 解析 DTB；`device/discovery/open_firmware.rs#of_platform_discovery` 基于设备树创建设备对象。

### SYSCALL（系统调用）
- **调用表注册**：`syscall/mod.rs#register_syscall_handlers` 初始化 `SYSCALL_TABLE`。
- **用户内存访问**：`syscall/user_access.rs#try_new` / `with_readable_ptr` / `with_writable_ptr` / `to_read` / `to_write` 提供 `UserReadPtr<T>` / `UserWritePtr<T>`，进行地址对齐、长度溢出与范围校验。
- **宏支持**：`crates/kernel-macros/src/syscall.rs#syscall_impl` 为系统调用函数生成入口。
- **已覆盖类型**：文件、进程、内存、信号、能力、IPC 等均有对应 `sys_*` 实现。

### TRAP/IRQ（中断异常）
- **架构相关陷阱**：`arch/riscv64/exception/trap/utrap.rs#rust_utrap_entry`、`arch/loongarch64/exception/trap/utrap.rs#rust_utrap_entry` 处理用户态异常、中断与缺页。
- **IRQ 框架**：`exception/intr/irq.rs#register_root_irq_domain`、`handle_irq`、`handle_domain_irq` 实现 IRQ domain 注册、硬件 IRQ 认领与描述符分发。
- **SMP 启动**：`arch/riscv64/bootstrap.rs#wake_up_aps` 通过 SBI 启动其他 hart。

### NET（网络协议栈）
- **底层证据未体现**：未在 AST Call Map 与函数分布中发现 socket、TCP/IP、网络设备驱动、协议栈等直接证据。
