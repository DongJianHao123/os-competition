## 第四部分：分模块架构说明

### MM（内存管理）

- **已体现证据**：ELF 加载器 `kernel/src/loader/mod.rs#load_elf` 使用 `ElfFile::new`、`VmArea::new`、`aligned_copy` 将程序段映射到虚拟地址空间；任务创建时 `kernel/src/task/mod.rs#clone_current` 复制父任务 trap frame 与地址空间相关状态；用户态数据拷贝通过 `copy_in_bytes` / `copy_out_bytes` 完成；RISC-V trap handler 统计 `KERNEL_ACCESS_FAULTS`、`LAST_FAULT_PC` 等缺页信息。
- **未直接体现**：底层物理页分配器（buddy/slab）、页表遍历/页回收、TLB shootdown 等核心实现细节在提供的 AST 片段中**底层证据未体现**。

### PROC（进程管理）

- **已体现证据**：具备较完整的传统进程管理：
  - 创建：`kernel/src/task/mod.rs#clone_current` / `fork_current`
  - 执行：`kernel/src/syscall/mod.rs#sys_execve` / `sys_execveat` / `exec_resolved`
  - 等待/回收：`kernel/src/syscall/mod.rs#sys_wait4` / `wait_reap` / `kernel/src/task/mod.rs#reap`
  - 父子关系：`kernel/src/task/mod.rs#reparent_children_to_init`、`task_by_pid`、`ppid`、`children`
  - 信号：`kernel/src/signal.rs#reset_for_exec` / `fork_inherit`、`sys_rt_sigtimedwait`、`sys_pidfd_send_signal`
  - 同步：`kernel/src/sync/futex.rs#futex_wait`
- PCB 字段可见：`pid`、`ppid`、`children`、`state`、`fd_table`、`cwd`、`exe_path`、`cmdline`、`signals`、`vfork_child` 等。

### 任务调度/异步运行时

- **已体现证据**：`kernel/src/task/mod.rs#schedule_next_after_trap_inner` 维护全局任务表 `TABLE`，切换 `CURRENT_PID`，检查 `any_runnable_except` / `any_waiting`，并驱动 `REAP_SWEEP_TICK` 回收；阻塞系统调用（如 `sys_read`、`sys_clock_nanosleep`）通过 `sleep_until`、`mark_socket_waiter`、`forget_sleeper` 进入等待；`futex_wait` 维护 `FUTEXES` 等待队列。
- **边界说明**：提供的片段中**未看到独立 async/await Runtime**（如 `Future`、`Waker`、`Executor`），阻塞机制更偏向基于任务状态与等待队列的协作/抢占式调度，因此“完整异步运行时”这一说法**底层证据未体现**。

### FS（文件系统）

- **已体现证据**：这是项目最强模块。存在多文件系统实现：
  - `ext2`：`kernel/src/fs/ext2.rs#smoke_test` 进行块级读写测试
  - `ext4`：`kernel/src/fs/ext4.rs#create/list/unlink/lookup/parse/read_dir/write_at`
  - `tmpfs`：`kernel/src/fs/tmpfs.rs`
  - `procfs`：`kernel/src/fs/procfs.rs#lookup` 动态生成 `/proc/pid/*`、`/proc/self`、`/proc/mounts` 等
  - `fat32`：`kernel/src/fs/fat32.rs#parse_dir`
  - `devfs`：`devfs::DevNode` / `BlockDevNode`
  - 文件事件通知：`kernel/src/fs/notify.rs#read_events`
- VFS 通用路径查找、挂载、重命名、链接、删除在 `kernel/src/fs/mod.rs` 与 `kernel/src/syscall/mod.rs#sys_mount` / `sys_renameat2` / `sys_unlinkat` / `sys_linkat` 中体现。

### DEV（设备驱动）

- **已体现证据**：VirtIO 块设备 `kernel/src/drivers/virtio_blk.rs#probe_mmio` / `probe_pci` / `init`；VirtIO 网设备 `kernel/src/drivers/virtio_net.rs#init_mmio` / `init_pci` / `receive`；PCI 总线 `kernel/src/drivers/pci.rs#assign_bars`。
- **边界说明**：驱动代码主要是对第三方 `VirtIOBlk` / `VirtIONet`  crate 的 MMIO/PCI 探测与包装，原创驱动逻辑（如自定义 DMA、中断处理、设备初始化状态机）在片段中**底层证据未体现**。

### SYSCALL（系统调用）

- **已体现证据**：`kernel/src/syscall/mod.rs#dispatch` 规模 22060、调用数 218，是系统调用总入口；具体实现覆盖文件、进程、时间、IPC、Keys 等：
  - 文件：`sys_openat`、`sys_openat2`、`sys_read`、`sys_write`、`sys_pread`、`sys_pwrite`、`sys_mount`、`sys_name_to_handle_at`、`sys_open_by_handle_at`、`sys_newfstatat`、`sys_statx`、`sys_fcntl`、`sys_ioctl`
  - 进程：`sys_execve`、`sys_wait4`、`sys_exit`、`sys_rt_sigtimedwait`
  - IPC：`kernel/src/syscall/sysv_ipc.rs#sys_semctl`、`sys_semtimedop`、`sys_msgctl`、`sys_shmctl`
  - Keys：`kernel/src/syscall/keys.rs#sys_add_key`、`sys_keyctl`
  - 时间：`sys_adjtimex`、`sys_timer_settime`、`sys_clock_nanosleep`

### TRAP/IRQ（中断异常）

- **已体现证据**：RISC-V 侧 `kernel/src/arch/riscv64/trap.rs#init` 设置 `stvec` 与定时器 `arm_timer`；`kernel/src/arch/riscv64/trap.rs#rust_trap_handler` 区分 syscall/timer/page fault 并维护 fault 统计；LoongArch64 侧 `kernel/src/arch/loongarch64/trap.rs#rust_trap_handler` / `csrrd` / `arm_timer` 提供同构入口。
- **未直接体现**：中断控制器（PLIC/APLIC）初始化、多核 IPI、设备中断分发等**底层证据未体现**。

### NET（网络协议栈）

- **已体现证据**：`kernel/src/net/mod.rs#init` 初始化 smoltcp `Interface`、IPv4 配置；TCP/UDP socket 管理：`add_tcp_socket`、`tcp_listen`、`tcp_recv`、`add_udp_socket`、`udp_send`、`udp_recv`、`udp_local_endpoint`；本地回环 `kernel/src/net/loopback.rs#find_udp`。
- **边界说明**：网络栈核心依赖 smoltcp，项目代码主要是 socket 生命周期与缓冲区包装，原创协议实现**底层证据未体现**。
