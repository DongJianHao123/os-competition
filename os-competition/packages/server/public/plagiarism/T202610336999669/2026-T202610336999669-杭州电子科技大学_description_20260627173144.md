## 第四部分：分模块架构说明

### MM（内存管理）

- **核心文件**：`os/src/mm/memory_set.rs`、`os/src/mm/address.rs`
- **关键证据**：
  - `os/src/mm/memory_set.rs#from_elf`：解析 ELF program header，创建 `MapArea`、分配 `VirtPageNum`、拷贝段数据，构建用户地址空间。
  - `os/src/task/task.rs#exec` 与 `os/src/task/task.rs#kernel_clone`：分别调用 `MemorySet::from_elf` 与 `MemorySet::from_existed_user` 完成进程替换与地址空间复制。
  - 系统调用中广泛使用 `current_user_token` + `translated_byte_buffer` / `translated_ref` / `translated_refmut` 做用户态缓冲区翻译。
- **可说明的架构**：基于页表的用户/内核地址空间分离、ELF 加载器、用户缓冲区按需翻译。
- **未体现**：交换（swap）、请求调页（demand paging）、COW 细节、page cache、NUMA 等底层证据未体现。

### PROC（进程管理）

- **核心文件**：`os/src/task/task.rs`、`os/src/task/mod.rs`、`os/src/task/processor.rs`、`os/src/syscall/process.rs`、`os/src/futex/mod.rs`、`os/src/signal/`
- **关键证据**：
  - `os/src/task/task.rs#fork` / `#kernel_clone` / `#exec` / `#new`：存在完整的 fork/clone/exec/新建进程路径。
  - `os/src/task/mod.rs#exit_current_and_run_next`：退出时将子进程 reparent 到 `INITPROC`，体现父子关系与僵尸进程处理。
  - `os/src/syscall/process.rs#sys_kill` / `#sys_rt_sigaction` / `#sys_rt_sigprocmask` / `#sys_rt_sigreturn`：信号发送与处理。
  - `os/src/futex/mod.rs#futex_wait` / `#do_futex`：futex 等待队列、requeue、robust list。
  - `os/src/syscall/process.rs#sys_setpgid` / `#sys_setsid` / `#sys_getpgid`：进程组与会话。
- **可说明的架构**：传统 UNIX 式进程模型，支持 fork/exec/exit/wait、信号、futex、进程组/会话、定时器。
- **未体现**：完整 POSIX 线程库（NPTL）全部语义、cgroup/namespace、capabilities、审计等底层证据未体现。

### 任务调度/异步运行时

- **重要限定**：**未见 async/await、Future、Waker、Executor 等完整异步运行时证据**，因此不宜描述为“完整异步运行时”。
