## 第四部分：分模块架构说明

### MM（内存管理）
底层证据较强。包含：
- **内核堆分配器**：`src/mm/heap_allocator.rs#alloc`、`src/mm/heap_allocator.rs#dealloc`，使用 `AllocHeader`、`FreeBlock`、`kernel_heap_start/end` 等结构，支持按 `Layout` 分配与释放。  
- **物理页框分配**：`src/mm/frame_allocator.rs#init_frame_allocator` 初始化基于 `PhysAddr` 的页框分配器。  
- **地址空间与页表**：`src/mm/memory_set.rs#map_kernel_sections` 建立内核段映射；`src/mm/page_table.rs` 提供 `MapPermission` 等页表权限抽象。  
- **用户内存扩展**：`src/syscall/fd.rs#sys_mmap` 通过当前任务的 memory set 增加映射。  
- **ELF 加载内存布局**：`src/loader/elf.rs#load_elf_with_argv` 创建 `MemorySet`、`MapArea`，布置用户栈与参数块。

### PROC（进程管理）
底层证据充分，不是轻量级任务包装。包含：
- **PCB 与父子关系**：`src/task/task.rs` 中任务结构包含 `pid`、`parent`、`children`、`status`、`files` 等字段。  
- **创建与克隆**：`src/task/mod.rs#fork_current_user_process_with_child_stack`、`src/task/task.rs#fork_child_trap_context`。  
- **执行映像**：`src/task/task.rs#exec_user_app`、`src/task/task.rs#exec_elf_bytes`、`src/task/task.rs#install_user_image`。  
- **等待与僵尸**：`src/syscall/task.rs#sys_waitpid`、`src/task/mod.rs#wait_zombie_child_for_current`、`src/task/mod.rs#exit_current`、`src/task/mod.rs#push_zombie_record`、`src/task/mod.rs#wake_child_exit_waiters`。  
- **信号/唤醒**：`src/task/mod.rs#send_signal_to_pid`、`src/task/wait_queue.rs#wake_pid`。

### 任务调度（含异步运行时标签说明）
代码体现的是**线程式任务调度**，而非完整 async runtime：
- **调度器**：`src/task/scheduler.rs#schedule_next_or_idle`、`src/task/scheduler.rs#handle_block`、`src/task/scheduler.rs#enter_with_event`。  
- **上下文切换**：`src/task/mod.rs#kernel_yield`、`src/task/mod.rs#kernel_sleep_ticks` 调用 `crate::arch::switch_task_context`。  
- **队列管理**：`task::manager::fetch`、`task::push_blocked_to_test_queue`、`task::push_sleep_task`。  
- **定时器抢占**：`src/trap/handler.rs#rust_trap_handler` 在定时器中断时触发调度。  
> 未观察到 `Future`、`Poll`、`waker`、`executor` 等异步运行时原语，因此“异步运行时”仅作为统计标签，不能视为完整 async runtime 实现。

### FS（文件系统）
呈现**双层结构**：
- **内存虚拟文件层**：`src/syscall/fd.rs` 维护 `CREATED_FILES`、`CREATED_DIRS`、`MOUNT_POINTS`、`PIPE_BUFFERS`，实现 `sys_openat`、`sys_mkdirat`、`sys_mount`、`sys_unlinkat`、`sys_pipe2` 等。  
- **块设备文件系统层**：`src/fs/ext4.rs` 提供 `lookup_path`、`read_file_range`、`create_file`、`write_file`、`read_file_from_inode`、`read_dir_entries_from_inode`，并通过 `crate::drivers::block_read_bytes/block_write_bytes` 访问底层块设备。  
> `src/fs/ext4.rs` 函数数量较多，可能包含或改编自第三方 ext4 解析实现，需单独审查其原创性，不宜直接等同于内核主体原创模块。

### DEV（设备驱动）
证据较强，主要围绕块设备与串口：
- **VirtIO-blk MMIO**：`src/drivers/virtio_mmio.rs#init`、`probe`、`read_block`、`write_block`。  
- **VirtIO-blk PCI**：`src/drivers/virtio_pci.rs#init`、`read_block`、`write_block`，涉及 `MmioBar`、`VirtQueueMem`、PCI 配置空间读写。  
- **块设备统一接口**：`src/drivers/mod.rs#block_read_bytes / block_write_bytes` 根据平台分发到 MMIO 或 PCI。  
- **串口输出**：`crate::drivers::serial::Serial::putchar` 被 `sys_write` 等调用用于早期打印。

### SYSCALL（系统调用）
证据充分：
- **统一分发器**：`src/syscall/mod.rs#syscall` 根据系统调用号路由到 `fd`、`task`、`time` 子模块。  
- **文件/IO 类**：`src/syscall/fd.rs#sys_read`、`sys_write`、`sys_openat`、`sys_pipe2`、`sys_mmap`、`sys_mount`、`sys_fstatat`、`sys_getdents64` 等。  
- **进程控制类**：`src/syscall/task.rs#sys_fork`、`sys_execve`、`sys_waitpid`、`sys_exit`、`sys_getpid`。  
- **时间类**：`src/syscall/time.rs#sys_nanosleep`。

### TRAP/IRQ（中断异常）
证据存在但较窄：
- **初始化**：`src/trap/mod.rs#init` 调用 `crate::arch::set_trap_entry` 设置陷阱入口；`src/arch/riscv64/mod.rs#set_trap_entry` 与 `src/arch/loongarch64/mod.rs#set_trap_entry` 为架构相关实现。  
- **统一处理**：`src/trap/handler.rs#rust_trap_handler` 处理定时器中断（`is_timer_interrupt`、`set_next_timer`）、系统调用（`crate::syscall::syscall`）以及内存访问异常（引用 `area.perm`、`VirtAddr`）。  
- **定时器配置**：`src/arch/loongarch64/mod.rs#enable_timer_interrupt`、`src/arch/loongarch64/mod.rs#timer_freq`。

### NET（网络协议栈）
**底层证据未体现**。源码统计与 Call Map 中均未出现网络设备驱动、TCP/IP 协议栈、socket 系统调用等相关函数或文件。
