## 第四部分：分模块架构说明

### MM（内存管理）
底层证据较强，核心集中在缺页处理与 ELF 加载路径：
- `os/src/mm/page_table.rs#handle_read_fault` / `handle_write_fault` 通过 `TrapType::LoadPageFault` / `StorePageFault` 进入，调用 `memory_set.handle_cow_addr` 与 `handle_lazy_addr`，说明实现了写时复制（COW）与懒分配。
- `os/src/task/task.rs#exec` 与 `os/src/mm/memory_set.rs#load_interp` 使用 `MemorySet::from_elf`、`elf.find_section_by_name`、`DL_INTERP_OFFSET` 等，说明支持 ELF 可执行文件加载与动态链接器解析。
- `arch/src/addr.rs` 提供通用虚拟/物理地址抽象。
- **未直接体现**：底层物理页分配器（buddy / slab）的具体实现、页表自映射、swap 等机制在提供的锚点中底层证据未体现。

### PROC（进程管理 / 任务调度）
底层证据强，具备传统进程管理要素：
- `os/src/syscall/process.rs#sys_clone` 使用 `CloneFlags`、`add_task`、`block_current_and_run_next`，说明支持进程/线程创建。
- `os/src/syscall/process.rs#sys_waitpid` / `sys_waitid` 处理 `is_zombie`、`exit_signal`，说明具备 wait / zombie 机制。
- `os/src/syscall/process.rs#sys_exec_resolved` 与 `os/src/task/task.rs#exec` 构成 exec 路径。
- `os/src/task/processor.rs#run_tasks` 通过 `fetch_task`、`context_switch_pt`、`handle_signals` 实现调度循环。
- `os/src/task/mod.rs#exit_current_and_run_next` 负责退出与清理。
- 同时存在 futex（`sys_futex_wait`、`sys_futex_wake`、`sys_futex_requeue`）、POSIX 消息队列（`sys_mq_open`、`sys_msgsnd`）、信号（`sys_sigprocmask`、`call_kernel_signal_handler`）等。
- **未直接体现**：完整 cgroup / namespace 实现、SMP 负载均衡、实时调度器（仅有 `sched_getattr/setattr` 等属性接口）。

### FS（文件系统）
底层证据强，呈现多层结构：
- VFS 层：`vfs` crate 提供通用 `Dentry` / `Inode` 抽象。
- 伪文件系统：`vfs/src/procfs/mod.rs#init_procfs`、`vfs/src/devfs/mod.rs#init_devfs` 实现 `/proc`、`/dev`。
- 磁盘文件系统：`ext4_rs-1.3.1` / `ext4/src/dentry.rs#concrete_lookup/create/unlink/rename`、`ext4/src/fs.rs#mount` 提供 ext4 支持；`easy-fs` 提供简易文件系统。
- 系统调用：`os/src/syscall/fs.rs` 密集实现 `openat`、`read`、`write`、`preadv`、`pwritev`、`mount`、`epoll`、`fcntl`、`poll` 等。
- **注意**：`ext4_rs-1.3.1`、`easy-fs` 属于第三方或适配库，不应视为本项目完全原创的内核主体。

### DEV（设备驱动）
底层证据较强，但大量代码来自外部/适配驱动：
- 块设备：`os/src/drivers/block/virtio_blk.rs#new`、`virtio-drivers/src/device/blk.rs#write` 实现 virtio-blk。
- 网络设备：`isomorphic_drivers/src/net/ethernet/intel/ixgbe.rs#new` / `try_handle_interrupt` 实现 Intel 10GbE。
- SD 主机：`visionfive2-sd/src/lib.rs#init_sdcard` 针对 VisionFive2。
- virtio-socket：`virtio-drivers/src/device/socket/connectionmanager.rs#send_recv` / `poll`。
- **注意**：`virtio-drivers`、`isomorphic_drivers`、`visionfive2-sd` 为第三方或硬件协议强约束代码，不应夸大成原创内核主体。

### SYSCALL（系统调用）
底层证据强：
- 入口：`os/src/syscall/mod.rs#syscall` 是统一分发器，调用链中出现 `current_task`、`gettid` 及大量 `sys_*`。
- 文件类：`os/src/syscall/fs.rs`（115 函数）覆盖打开、读写、目录、epoll、fcntl、mount 等。
- 进程类：`os/src/syscall/process.rs`（111 函数）覆盖 clone、exec、wait、sched、mq、prlimit、cap、uname 等。
- 网络类：`os/src/syscall/socket.rs` 覆盖 bind、sendmsg、recvmsg、setdomainname 等。
- 信号/内存/时间：`os/src/syscall/signal.rs`、`os/src/syscall/memory.rs`、`os/src/syscall/timesyscall.rs`。

### TRAP/IRQ（中断异常）
底层证据中等：
- `arch/src/loongarch64/trap.rs#loongarch64_trap_handler` 处理 `LoadPageFault`、`StorePageFault`、`IllegalInstruction`、`FloatingPointUnavailable` 等。
- `arch/src/loongarch64/unaligned.rs#emulate_load_store_insn` 实现未对齐访问模拟，调用 `read_fpr` / `write_fpr`。
- `os/src/boards/qemu.rs#irq_handler` 处理 PLIC / UART / BLOCK 中断。
- `arch/src/x86_64/apic.rs#init` 初始化 Local APIC / IO APIC。
- **未直接体现**：完整中断控制器抽象层、中断线程化、tickless 定时器等。

### NET（网络协议栈）
底层证据中等：
- 协议栈主体在 `lose-net-stack`：`tcp.rs#connect/sendto/send_data/interrupt`、`udp.rs#sendto`、`connection/mod.rs#analysis_arp`。
- 套接字抽象：`os/src/socket.rs#read_at/write_at/poll`。
- 系统调用：`os/src/syscall/socket.rs#sys_bind/sendmsg/recvmsg/setdomainname`。
- **注意**：`lose-net-stack` 属于第三方/适配协议栈；且内核 NET 模块函数仅 63 个，说明网络并非最核心原创部分。
