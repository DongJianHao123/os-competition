#### 4. 核心代码调用流对比剖析
* **【进程管理/调度器】**
* **底层技术析理**: 双方均实现 xv6 风格的进程调度器，核心循环为遍历进程表并调用 `swtch` 切换上下文。目标项目在调度器入口显式调用 `intr_off` 关闭中断，并在空闲时执行 `wfi` 指令等待中断，表明其针对 RISC-V 架构进行了深度电源管理适配。参考项目仅调用 `intr_on` 开启中断，无 `wfi` 指令，遵循更标准的 xv6 实现。两者在中断控制策略和空闲态处理上存在根本性架构差异。
* **代码溯源链接**:
  - 目标项目: `kernel/proc.c#scheduler`
  - 参考项目: `kernel/proc/proc.c#scheduler`

* **【内存管理/磁盘驱动】**
* **底层技术析理**: 在 VirtIO 磁盘读写路径中，目标项目使用 `R_DEV` 宏直接操作设备寄存器，采用物理地址直传模式。参考项目则使用 `PA2VA` 和 `VIRT2PHY` 宏进行物理地址到虚拟地址的双向转换，并通过 `kwalkaddr` 进行页表遍历。这揭示了两者在 IO 内存映射模型上的根本分歧：目标项目可能运行在直接物理寻址环境，而参考项目构建了完整的虚拟内存抽象层。
* **代码溯源链接**:
  - 目标项目: `kernel/virtio_disk.c#virtio_disk_rw`
