## 第四部分：分模块架构说明

### MM（内存管理）

底层证据显示存在页表管理、用户态地址空间构建、内核/用户拷贝、mmap/munmap 与 exec 加载。

- 关键证据：
  - `kernel/vm.c`：页表与虚拟内存核心（20 个函数）
  - `kernel/sysfile.c#sys_mmap` / `sys_munmap`：用户态内存映射与解除映射
  - `kernel/exec.c#kexec`：可执行文件加载、地址空间构造
  - `kernel/trap.c#usertrap` / `kernel/trap-la.c#usertrapret`：页表切换与返回用户态

- 未体现：交换（swap）、NUMA、大页、内存压缩、COW（除 fork 外无显式 COW 证据）等高级特性。

### PROC（进程管理/任务调度）

存在传统 Unix 式进程控制块、fork/clone、exec、exit/wait、kill、调度器、sleep/wakeup 机制。

- 关键证据：
  - `kernel/proc.c#kexit` / `scheduler` / `sched` / `sleep` / `yield` / `wakeup`
  - `kernel/proc.c#forkret` / `kfork` / `kclone`
  - `kernel/sysproc.c#sys_fork` / `sys_clone` / `sys_wait4` / `sys_exit` / `sys_kill`

- 说明：证据集中在“传统进程管理”，**“异步运行时（async/await、waker、task queue）”底层证据未体现**。`任务调度/异步运行时` 模块的 34 个函数实际多为传统调度与同步原语。

### FS（文件系统）

文件系统栈由 VFS/路径解析层、日志层、第三方 ext4 实现组成，支持目录、链接、管道、stat、mmap 文件映射等。

- 关键证据：
  - `kernel/fs.c#namex` / `ilock` / `iput`：路径解析与 inode 缓存
  - `kernel/sysfile.c#openat_impl` / `sys_linkat` / `sys_unlinkat` / `renameat_impl` / `sys_getdents64` / `sys_statx` / `sys_newfstatat`
  - `kernel/ext4.c#ext4_dirlink` / `ext4_readi` / `ext4_writei` / `ext4_lookup`
  - `kernel/log.c#install_trans`：日志事务
  - `mkfs/mkfs.c`：镜像制作工具

- 说明：`kernel/ext4.c` 属于文件系统/第三方库模块，应视为集成的 ext4 实现，而非完全原创内核主体。

### DEV（设备驱动）

设备层覆盖控制台 UART、VirtIO-Block 块设备、中断控制器。

- 关键证据：
  - `kernel/uart.c` / `kernel/uart-la.c`：串口控制台（RISC-V / LoongArch）
  - `kernel/virtio_disk-la.c#virtio_disk_init` / `virtio_disk_intr` / `virtio_notify`：LoongArch VirtIO-Block
  - `kernel/bio.c#bwrite` → `virtio_disk_rw`：块 I/O 下刷
  - `kernel/plic.c`：RISC-V PLIC
  - `kernel/trap-la.c#devintr` 中 `apic_complete` / `extioi_claim`：LoongArch 中断控制器

### SYSCALL（系统调用）

存在系统调用分发器与大量 Linux 兼容系统调用实现。

- 关键证据：
  - `kernel/syscall.c#syscall` / `argraw`
  - `kernel/sysfile.c`（59 个函数）：文件/IO/mmap/exec 类调用
  - `kernel/sysproc.c`（32 个函数）：进程/时间/信号类调用
  - `codex-basic-syscall-bundle/references/lib/syscall.c`：用户态 syscall 封装
  - 测试程序覆盖 `open/read/write/close`、`fork/wait/exit`、`mmap/munmap`、`getdents`、`pipe`、`clone`、`sched_yield` 等

### TRAP/IRQ（中断异常）

双架构陷阱/中断处理证据明显。

- 关键证据：
  - RISC-V：`kernel/trap.c#usertrap` / `kerneltrap` / `clockintr`，使用 `r_scause` / `r_sepc` / `sbi_set_timer`
  - LoongArch：`kernel/trap-la.c#usertrap` / `kerneltrap` / `trapinit` / `usertrapret`，使用 `r_csr_estat` / `r_csr_era` / `w_csr_eentry` / `w_csr_tcfg`
  - 设备中断：`kernel/trap-la.c#devintr` → `uartintr` / `clockintr` / `virtio_disk_intr`

**底层证据未体现**。源码中未出现 socket、tcp/ip、netdev、ethernet、skb 等网络相关函数或目录。
