    *   **代码溯源链接**:

        *   **测试与探测框架**: `codex_busy_box/` 和 `codex-basic-syscall-bundle/` 包含大量用户态系统调用探针和测试用例，体现了独立的测试体系。
    *   **参考项目独有**:
        *   **FAT32 文件系统**: `kernel/fat32.c` 及相关文件。
        *   **DMA 驱动**: `kernel/dmac.c`。
    *   **结论**: 这些单方存在的模块是独立开发的最有力证明，直接否定了“重度抄袭”的指控。

*   **降权项说明**:
    *   **公共基线代码 (xv6)**: `kernel/proc.c`, `kernel/sysproc.c`, `kernel/spinlock.c`, `user/sh.c` 等文件中的同名函数（如 `sleep`, `sched`, `fork1`, `parseexec`）是 xv6 教学内核的标准组成部分。其相似性源于共同祖先，不能作为项目间抄袭的证据。
    *   **领域强约束 trivial 函数**: `sys_getpid`, `cpuid`, `mycpu` 等函数实现空间极小，任何独立开发者都会写出几乎相同的代码，属于必然趋同。
    *   **跨板块错配证据**: 控方将用户态 Shell (`user/sh.c`) 的函数归入【进程管理】/【任务调度】，将控制台驱动 (`kernel/console.c`) 归入【任务调度】，此类证据因分类错误而无效。

### 6. 保守结论
本次审计的核心结论是**独立开发，底层架构显著差异**。12.27% 的客观 AST 相似度远低于抄袭阈值，且重合指纹几乎全部可归因于 xv6 公共教育基线和操作系统领域的强约束实现。目标项目在 CPU 架构（LoongArch vs RISC-V）、文件系统（ext4 vs FAT32）、测试体系等方面存在大量独立设计与实现，其唯一 AST 指纹数量远超共同指纹。控方提出的“重度复用”指控，其证据链建立在混淆公共基线、误读工程命名惯例（如 `k` 前缀）和跨板块硬凑证据之上，缺乏技术说服力。因此，综合技术分析认定，双方代码库的相似性不构成抄袭或违规复用，目标项目具备原创合理性。
