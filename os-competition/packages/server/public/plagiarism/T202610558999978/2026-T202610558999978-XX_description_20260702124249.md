## 第四部分：分模块架构说明

### MM（内存管理）

- **证据充分**。核心页分配器位于 `kernel/mm/kalloc.c`（`kalloc`/`kfree`）。
- 用户态页表与地址映射在 `kernel/mm/vm.c` 中体现（`mappages`、`walk`、`uvmcopy` 等）。
- 支持**写时复制（COW）**：`kernel/mm/kalloc.c#cowalloc` 通过引用计数与页复制实现。
- 支持**共享内存**：`kernel/mm/sharemem.c#shmgetat` / `deallocshm` 体现 System V 风格共享内存。
- 支持**mmap 缺页处理**：`kernel/interrupt/trap.c#mmap_handler` 在缺页异常路径中完成匿名/文件映射。

### PROC（进程管理/任务调度）

- **证据充分**。传统 PCB、fork/exec/exit/wait 在 `kernel/proc/proc.c` 与 `kernel/proc/exec.c` 中完整存在。
- 调度器为传统循环调度：`kernel/proc/proc.c#scheduler` / `scheduler1` / `scheduler2` 通过 `swtch` 切换上下文。
- 同步原语：`sleep` / `wakeup` 机制；另有信号量 `sys_sem_p` / `sys_sem_v`。
- IPC：`kernel/proc/messagequeue.c#sys_msgrcv` / `sys_mqget` 实现消息队列。
- 扩展机制：`sys_clone` / `join` 支持类线程创建；`sys_sigalarm` / `sigreturn` 支持定时信号；`setPriority` 提供优先级接口。
- **注意**：`scheduler1` / `scheduler2` 的具体策略差异在调用链中未充分展开，不能断言实现了复杂调度策略。

### FS（文件系统）

- **证据充分**。采用类 xv6 的 inode 日志文件系统。
- 路径解析：`kernel/filesystem/fs.c#namex`。
- 文件打开/创建/截断：`kernel/sysfile.c#sys_open`。
- 链接与目录操作：`sys_link`、`sys_unlink`、`sys_chdir`、`sys_chmod`。
- 文件抽象层：`kernel/filesystem/file.c#fileread` / `filewrite` 统一处理普通文件、管道、设备、socket。
- 日志事务：`begin_op` / `end_op` 保证元数据一致性。

### DEV（设备驱动）

- **证据中等**。包含：
  - UART 控制台：`kernel/driver/uart.c`、`kernel/driver/console.c`
  - 块设备：`kernel/driver/virtio_disk.c#virtio_disk_intr`
  - 网卡：`e1000_transmit` / `e1000_intr`（通过网络栈调用链体现）
- 未见 USB、NVMe、PCIe 枚举、GPU、音频等驱动。

### SYSCALL（系统调用）

- **证据较薄（从 syscall.c 本身看）**。`kernel/syscall.c` 仅体现参数解析 `argint`、`argstr`、`argaddr` 与分发 `syscall`。
- 大量具体系统调用处理函数分布在 `kernel/sysfile.c`、`kernel/sysproc.c`、`kernel/sysnet.c` 中，说明系统调用实现是“薄分发层 + 厚处理模块”结构。

### TRAP/IRQ（中断异常）

- **证据中等偏强**。RISC-V 异常入口完整：
  - `kernel/interrupt/trap.c#usertrap` / `kerneltrap`
  - `kernel/interrupt/trap.c#usertrapret`
- 中断分发：`kernel/interrupt/trap.c#devintr` 处理时钟、PLIC、virtio disk、e1000。
- 定时器初始化：`kernel/start.c#timerinit` 配置 CLINT。
- PLIC 中断控制器：`kernel/interrupt/plic.c#plicinithart` / `plic_claim` / `plic_complete`。

### NET（网络协议栈）

- **证据中等**。基于 mbuf 的极简网络栈：
  - `kernel/network/net.c#net_rx` / `net_tx_eth`
  - ARP：`net_rx_arp` / `net_tx_arp`
  - IP：`net_rx_ip` / `net_tx_ip`
  - UDP：`net_rx_udp` / `net_tx_udp`
  - socket 抽象：`kernel/sysnet.c#sockalloc` / `sockrecvudp`
- **未见 TCP 协议栈证据**。
