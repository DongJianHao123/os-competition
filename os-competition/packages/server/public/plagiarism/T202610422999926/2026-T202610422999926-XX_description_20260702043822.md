## 第四部分：分模块架构说明

### MM（内存管理）

底层证据显示内存管理按架构拆分：
- `os/components/wateros-mm/mm-impl/impl-loongarch64/src/kernel_elf.rs#from_elf_path` 创建 `LoongArch64AddressSpace` 并加载内核 ELF。
- `os/components/wateros-mm/mm-impl/impl-sv39/src/kernel_executable.rs#load_program_from_path_rec` 处理 RISC-V SV39 可执行文件与解释器脚本。
- 用户态/内核态数据拷贝通过 `copy_from_user`、`copy_to_user`、`copy_from_user_struct`、`copy_to_user_struct` 在多个 `sys_*` 中体现（如 `sys_readv`、`sys_writev`、`sys_execve.rs#read_string_array`）。
- 文件页缓存由 `os/components/wateros-vfs/vfs-impl/impl-page-cache/src/lib.rs` 中的 `GlobalFilePageCache` 相关测试函数体现（`acquire_open_ref / release_open_ref / flush / write`）。
- 块设备缓存见 `os/components/wateros-driver/driver-block/block-impl/impl-block-cache/src/lib.rs#write_allocate_then_read_hits_cache_line`。

**未直接体现**：物理页分配器（buddy/slab）的完整实现细节在提供的证据中不够清晰。

### PROC（进程管理/任务调度）

**进程管理**证据较强：
- `os/components/wateros-task/task-impl/impl-core/src/process.rs#reap_process_with_tasks`、`mark_task_exited`、`reparent_orphans`、`abort_forked_process` 体现 PCB、进程状态、孤儿收养、退出回收。
- `os/components/wateros-syscall/syscall-impl/impl-kernel/src/sys/clone.rs#do_clone_request / do_clone_thread` 支持 `clone` 创建线程/进程，处理 `cgroup_dir`、`child_tid`、`parent_tid`。
- `os/components/wateros-syscall/syscall-impl/impl-kernel/src/sys/task.rs#sys_exit / sys_exit_group / sys_waitpid / sys_waitid` 覆盖退出、等待、信号通知。
- `os/components/wateros-ipc/ipc-signal/src/lib.rs#send_process / fork_process` 覆盖信号分发与 fork 时信号状态复制。
- FD/当前工作目录/挂载命名空间在 fork/clone 时的继承与共享：
  - `os/components/wateros-vfs/vfs-impl/impl-fd-session/src/registry.rs#copy_fd_table_from_parent`
  - `os/components/wateros-vfs/vfs-impl/impl-fd-session/src/cwd.rs#copy_cwd_from_parent / share_cwd_from_parent`
  - `os/components/wateros-vfs/vfs-impl/impl-fs-bridge/src/mount_ns.rs#copy_mount_ns_from_parent / share_mount_ns_from_parent`

**任务调度**证据：
- `os/components/wateros-task/task-scheduler/scheduler-impl/impl-multi-class/src/scheduler.rs#schedule / schedule_wait` 实现 `Other / RR / Fifo` 策略。
- `os/components/wateros-task/task-scheduler/scheduler-impl/impl-round-robin/src/scheduler.rs#schedule / schedule_wait` 实现 Round-Robin。
- `os/components/wateros-task/task-scheduler/scheduler-api/api-v0/src/wait_queues.rs#enqueue_task / requeue_wait_queue / wake_all_waiters_for_task_exit / promote_wait_timeouts` 维护阻塞、就绪、睡眠、退出状态机。
- `os/components/wateros-task/task-scheduler/scheduler-impl/impl-multi-class/src/lib.rs#exit_current / wait_current_timeout_while` 提供上下文切换入口。

### FS（文件系统）

- **VFS 桥接层**：`os/components/wateros-vfs/vfs-impl/impl-fs-bridge/src/lib.rs#metadata`、`mount_table.rs#move_mount_at / recursive_bind / remount_aux_readonly` 处理挂载点、挂载命名空间、remount。
- **FD 会话层**：`os/components/wateros-vfs/vfs-impl/impl-fd-session/src/registry.rs#dup3_fd_for_task / copy_fd_table_from_parent / begin_shared_io_for_task` 管理 per-task 文件描述符表、共享 IO、文件锁继承。
- **页缓存**：`os/components/wateros-vfs/vfs-impl/impl-page-cache/src/lib.rs`。
- **procfs**：`os/components/wateros-fs/fs-procfs/procfs-impl/impl-kernel/src/lib.rs#read / format_stat / read_dir / metadata` 提供 `/proc` 视图。
- **pipe**：`os/components/wateros-ipc/ipc-pipe/pipe-impl/impl-ringbuf/src/lib.rs#test` 实现环形缓冲 pipe。
- **ext4**：`os/components/wateros-fs/fs-impl/impl-ext4/src/rw.rs#write_regular_file / read_range / rmdir` 与 `os/components/wateros-fs/fs-impl/impl-ext4-rs/src/lib.rs#rename / unlink` 为 **第三方 ext4 库的封装/适配**，不是原创文件系统实现。

### DEV（设备驱动）

- **块设备缓存**：`os/components/wateros-driver/driver-block/block-impl/impl-block-cache/src/lib.rs#write_allocate_then_read_hits_cache_line / write_through_updates_existing_cache_line` 实现 `CachingBlockDevice`。

### TRAP/IRQ（中断异常）

- **架构相关初始化**：

### NET（网络协议栈）

- `os/components/wateros-driver/driver-network/src/lib.rs#socket_*` 提供 socket 抽象。
- 系统调用层有 `sys_getpeername / sys_getsockname`、unix domain socket、`poll_engine` 等，但均依赖底层 smoltcp 或 unix socket 实现。
