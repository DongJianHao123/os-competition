## 第四部分：分模块架构说明

### MM（内存管理）

底层证据充分，是代码量最大的模块。

- **虚拟内存**：`os/src/mm/memory_set.rs`（148 函数）维护进程地址空间；`os/src/trap.rs#trap_handler` 在发生页错误时调用 `memory_set.handle_page_fault(VirtAddr, FaultAccess::{Instruction,Load,Store})`，实现按需分页。
- **物理页分配**：`os/src/mm/frame_allocator.rs#dealloc_block` 使用 `block_size`、`free_pages`、`mark_free_block` 等接口，呈现按 order 管理的伙伴式分配器特征。
- **内核堆**：`os/src/mm/layout_allocator.rs#dealloc` 按 `class_for_layout` 分 size-class（16/128/1024 等缓存）回收；`os/src/mm/slab.rs#dealloc_raw_slot` 提供 slab 分配；`os/src/mm/heap_allocator.rs#handle_alloc_error` 记录堆分配失败统计。
- **DMA**：`os/src/hal/dma.rs#dma_dealloc` 按 PPN 维护 DMA 物理页集合。
- **页缓存**：`os/src/fs/page_cache.rs#allocate_and_fill_page_for_write` 通过 `PAGE_CACHE_MANAGER` 管理缓存页，支撑文件读写。
- **mmap/msync**：`os/src/syscall/process.rs#sys_mmap` 使用 `MapArea::new_lazy` 建立懒映射；`sys_msync` 调用 `memory_set.msync_user_range`。

### PROC（进程管理/任务调度）

传统进程管理证据充分，不是轻量级任务包装。

- **PCB 与线程组**：`os/src/task/task.rs#TaskControlBlock::new/fork/exec/clone_thread` 创建任务；`thread_group` 维护进程组、会话、资源限制、凭证（credentials）、信号状态。
- **生命周期系统调用**：`sys_clone`、`sys_clone_thread`、`sys_execve`/`sys_execveat`、`sys_wait4`、`exit_current_group_and_run_next`、`exit_current_and_run_next` 完整覆盖 fork/exec/wait/exit/zombie 路径。
- **调度**：`os/src/task/processor.rs#run_tasks` 负责任务切换（`__switch`）、取就绪任务、idle 等待中断；`crate::timer::sleep_task_until_token` 与 futex 等待队列提供阻塞/睡眠机制。
- **信号**：`os/src/trap.rs#setup_signal_frame`、`os/src/syscall/signal.rs#sys_rt_sigtimedwait`、`sys_rt_sigprocmask` 等实现信号投递与屏蔽。
- **futex**：`os/src/futex.rs#sys_futex_waitv` 维护 `FUTEX_QUEUES`，用于线程同步。

### FS（文件系统）

文件系统是第二大模块，呈现多层结构。

- **VFS 与路径解析**：`os/src/fs/inode.rs#resolve_parsed_from_with_options`、`os/src/syscall/fs.rs#sys_openat` 处理路径查找、符号链接跟随、fd 表。
- **后端适配**：
  - **ext4**：`os/src/fs/adapter/ext4.rs`（125 函数）封装 ext4 文件操作（`read_at`、`write_at`、`create_file`、`unlink_child`、`rename_child`），底层调用 `Ext4File`/`path_ops` 等，属于对第三方 ext4 库的适配层。
  - **easy-fs**：`easy-fs/` 目录 136 个 `.rs` 文件，提供自研/集成的简易文件系统。
  - **procfs**：`os/src/fs/proc/` 暴露 `/proc/[pid]/`、`/proc/self`、`/proc/mounts` 等。
  - **ramfs/tmpfs/vfat**：用户测试 `ramfs_tmpfs_smoke.rs`、`vfat_stage6a_smoke.rs` 表明这些文件系统至少被挂载或测试过。
- **页缓存**：`os/src/fs/page_cache.rs` 在 VFS 与块设备之间提供缓冲。
- **镜像工具**：`fs-packer/src/main.rs#pack_easy`、`create_ext4_image` 支持 easy/ext4 两种后端，后者调用 host 的 `mkfs.ext4`。

### DEV（设备驱动）

设备驱动证据较完整，集中在 VirtIO 与平台设备。

- **设备管理**：`os/src/devices/scan.rs#register_console_device`、`register_mmio_fallback_devices`；`os/src/devices/manager.rs#register_block_device` 维护设备表。
- **块设备**：
  - `os/src/drivers/block/virtio_pci_blk.rs#try_new / request` 实现 VirtIO PCI 块设备读写，使用 `dma::virt_to_phys` 与 virtqueue。
  - `os/src/drivers/block/loop_device.rs#binding_from_fd` 提供 loop 设备。
  - `os/src/devices/manager.rs` 使用 `CachedBlockDevice` 做块缓存。
- **字符设备/控制台**：UART console、TTY。
- **网络设备**：`SmoltcpDevice` 作为 smoltcp 的底层设备。
- **平台发现**：`os/src/hal/platform/qemu_virt_riscv/fdt.rs` 通过 FDT 解析 PLIC 区域；`os/src/hal/platform/qemu_virt_loongarch/devices.rs#read_wall_time_us` 读取 RTC。
- **中断控制器**：`os/src/hal/arch/riscv64/irq.rs#claim_external_irq / enable_external_irq` 操作 PLIC。

### SYSCALL（系统调用）

系统调用层证据强，入口集中。

- **分发器**：`os/src/syscall.rs#syscall` 规模 14919，调用数 157，集中分发到各子模块。
- **文件类**：`os/src/syscall/fs.rs`（154 函数）覆盖 `openat`、`read`、`write`、`newfstatat`、`mknodat`、`fallocate`、`msync`、`readlinkat`、poll/select 等。
- **进程类**：`os/src/syscall/process.rs`（128 函数）覆盖 `clone`、`execve`、`wait4`、`nanosleep`、`clock_nanosleep`、`setitimer`、`prlimit64`、`prctl`、`setresuid/gid` 等。
- **网络类**：`os/src/syscall/net.rs` 覆盖 socket、connect、accept、sockaddr 读写。
- **信号/futex/IPC**：`os/src/syscall/signal.rs`、`os/src/futex.rs`、`os/src/syscall/ipc.rs#sys_shmctl`。
- **用户态封装**：`user/src/syscall.rs`（126 函数）提供用户库封装。

### TRAP/IRQ（中断异常处理）

数量较少但关键路径清晰。

- **异常分发**：`os/src/trap.rs#trap_handler` 区分指令/加载/存储页错误，调用 `memory_set.handle_page_fault`；保存 FPU。
- **内核态陷阱**：`os/src/trap.rs#trap_from_kernel_handler` 处理外部 IRQ、软件中断，调用 `arch::irq::claim_external_irq`、`dispatch_irq`。
- **返回用户态**：`os/src/trap.rs#trap_return` 恢复 FPU、准备返回用户态、处理可投递信号。
- **信号帧**：`os/src/trap.rs#setup_signal_frame` 在用户栈上构造信号处理帧。
- **RISC-V PLIC**：`os/src/hal/arch/riscv64/irq.rs#claim_external_irq / enable_external_irq`。

### NET（网络协议栈）

证据存在，但底层协议栈主要依赖第三方。

- **Socket 层**：`os/src/net/socket.rs`（98 函数）实现 TCP/UDP socket、`connect_tcp`、`tcp_listener_accept_ready`、多播组管理、poll 等待。
- **协议栈适配**：`os/src/net.rs` 维护 smoltcp `Interface`、`SocketSet`、loopback 队列、物理设备收发（`receive`、`consume`、`try_receive_physical`）。
- **驱动接口**：`SmoltcpDevice` 封装 VirtIO net 或类似物理设备。
- **边界**：没有证据表明实现了独立的 TCP/IP 协议栈；网络功能通过 smoltcp 库实现。

## 第六部分：边界与未体现内容

### 有证据但需说明边界的部分

1. **第三方/公共依赖代码不应视为原创内核主体**：
   - `easy-fs/` 可能是自研或集成的简易文件系统，但 `fs-packer` 的 ext4 后端直接调用 host 的 `mkfs.ext4`。
   - `os/src/fs/adapter/ext4.rs` 是对 ext4 库的适配层，而非自底向上实现的 ext4。
   - 网络协议栈底层依赖 `smoltcp`，`os/src/net.rs` 与 `os/src/net/socket.rs` 主要做接口封装。
   - ELF 加载可能依赖外部 ELF 解析库（`ElfFile::new` 等）。

2. **“任务调度/异步运行时”的命名需谨慎**：
   - 现有证据（`run_tasks`、`__switch`、睡眠队列、futex 队列、`kernel_worker`）更接近传统阻塞式进程/线程调度。
   - 独立的 `async/await` 运行时（如 waker、executor、future 组合）底层证据未充分区分。

### 底层证据未体现的模块/特性

- **存储高级特性**：磁盘 swap、页面换出、LVM、RAID、NVMe 驱动。
- **安全与隔离**：cgroup、namespace、seccomp、MAC（SELinux/AppArmor）、审计子系统。
- **总线与外设**：USB、SATA、AHCI、PCIe 枚举（除 VirtIO PCI 片段外）、GPU、声卡。
- **SMP/NUMA**：多核调度、核间中断（IPI）细节、NUMA 内存策略在提供的片段中未完整展开。
- **容器与虚拟化**：容器运行时支持、KVM/ hypervisor、热插拔、kexec。
- **独立网络协议栈**：完整的自研 TCP/IP 协议栈证据不足，网络层依赖 smoltcp。
- **高级文件系统**：btrfs、xfs、zfs、NFS、CIFS 等无证据。
- **实时调度**：`sys_sched_setscheduler` 存在，但完整实时调度器（EDF、优先级继承、调度域）证据有限。
