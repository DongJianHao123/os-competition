## 第四部分：分模块架构说明

### MM（内存管理）

底层证据充分，是本项目最重的模块。

- **物理内存管理**：`kernel/pmm.c` 实现页级分配器，`pmm_init` 通过 `free_range` 初始化空闲页，提供 `page_alloc`、`page_free`、`page_ref_inc`、`page_ref_dec`、`pa2page`、`page2pa` 等接口，并带 `pmm_selftest` 自测。
- **虚拟内存管理**：`kernel/vmm.c` 实现 SV39 风格页表操作，`walk` 负责页表遍历与页表页按需分配，`vmm_map`/`vmm_unmap` 完成映射/解除映射，`vmm_translate` 提供地址转换，`vmm_create_kernel_mapped_pagetable` 为进程创建内核映射页表。
- **进程相关内存**：`process_load` → `load_elf` → `load_segment` 加载 ELF；`map_user_stack` 映射用户栈；`process_exofork` 复制地址空间；`process_handle_cow_fault` 处理写时复制缺页；`sys_mem_map`/`sys_ipc_try_send` 支持进程间共享页映射。

### PROC（进程管理/任务调度）

有基本进程抽象，但传统进程管理语义不完整。

- 已实现：`process_init_first`、`process_load`、`process_run_first`、`process_schedule`、`process_exofork`、`process_destroy`、`process_set_status`、`process_current`、`process_find`。
- `process_schedule` 通过 `csr_write_satp` 与 `sfence_vma` 切换地址空间，说明存在基于 SATP 的上下文切换。
- **未体现**：`fork/exec/wait/exit/zombie` 等完整语义、父子进程关系/PCB 树、进程间信号、多核调度、优先级调度器、时间片轮转等底层证据未体现。

### 任务调度/异步运行时

- 仅有 `kernel/process.c#process_schedule` 与 `kernel/init.c#main` 中提到的 `runtime_selftest`，未在函数池中看到 `task`、`channel`、`waker`、`ring_fifo`、`await` 等异步运行时原语。
- **结论**：完整异步运行时底层证据未体现。

### FS（文件系统）

- 模块画像中“文件系统”仅统计到 `kernel/console.c#panic_impl`，该函数属于错误打印，与文件系统无关。
- 未看到 VFS、inode、目录项、文件描述符、缓冲区缓存、磁盘/块设备驱动等。
- **结论**：文件系统底层证据未体现。

### DEV（设备驱动）

- **SBI 控制台**：`kernel/sbi.c` 提供 `sbi_ecall`、`sbi_set_timer`、`sbi_console_getchar` 等 RISC-V SBI 调用；`kernel/console.c` 基于 SBI 实现 `printk`/`vprintk`/`console_putchar`。
- **定时器**：`kernel/timer.c` 通过 `sbi_set_timer` 设置下一次时钟中断，`timer_handle_interrupt` 处理定时器中断。
- **MMIO 设备访问**：`kernel/syscall.c#sys_read_dev` / `sys_write_dev` 允许用户态按对齐、按 2 的幂大小读写设备内存区域。
- **未体现**：块设备驱动、磁盘控制器、DMA、字符设备框架、设备树解析等。

### SYSCALL（系统调用）

- `kernel/syscall.c#syscall_dispatch` 是统一入口，分发至：
  - 内存类：`sys_mem_alloc`、`sys_mem_map`、`sys_mem_unmap`
  - IPC 类：`sys_ipc_try_send`、`sys_ipc_recv`
  - 控制台类：`sys_cgetc`、`sys_putchar`、`sys_print_cons`
  - 设备类：`sys_read_dev`、`sys_write_dev`
  - 进程控制类：`process_exofork`、`process_destroy`、`process_set_status`
- 辅助校验：`user_va_range_valid`、`user_perm_valid`、`user_perm_to_pte`、`device_range_valid`、`is_power_of_two`。

### TRAP/IRQ（中断异常）

- `kernel/trap.c#trap_init` 设置 `stvec`；`kernel/trap.c#trap_dispatch` 统一处理定时器中断、系统调用、COW 页故障与进程调度。
- `kernel/timer.c` 提供定时器初始化与中断处理。
- **未体现**：外部中断（PLIC）、异常分类处理（如非法指令、断点、访问越界等细分处理）、中断嵌套。

### NET（网络协议栈）

- 未找到任何网络相关函数或数据结构。
- **结论**：网络协议栈底层证据未体现。
