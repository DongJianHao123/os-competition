## 第四部分：分模块架构说明

### MM（内存管理）
底层证据显示该作品实现了用户态地址空间的高层管理接口：
- `kernel/src/syscall/mm/mmap.rs#sys_mmap` 支持匿名、COW、文件、线性、共享等多种 `Backend` 后端映射。
- `kernel/src/syscall/mm/mmap.rs#sys_mprotect` 调用 `aspace.protect` 修改映射权限。
- `kernel/src/syscall/ipc/shm.rs#sys_shmat/sys_shmdt` 通过 `Backend::new_shared` 与 `aspace.unmap` 实现 System V 共享内存。
- `kernel/src/mm/aspace/backend/cow.rs#alloc_new_at` 从文件读取数据填充物理页，出现 `file.read_at`、`phys_to_virt`。
- `kernel-elf-parser-local/src/user_stack.rs#app_stack_region` 负责 ELF 加载后的用户栈与 AuxVec 构造。

但**物理页分配器、页表操作、页错误处理、swap 机制**等底层证据未体现。

### PROC（进程管理/任务调度）
证据显示该作品实现了 Linux 风格的进程/线程生命周期与信号机制：
- `kernel/src/syscall/task/clone.rs#do_clone` 处理 `CLONE_*` 标志、写回 `parent_tid`/`pidfd`、继承 FD 表。
- `kernel/src/syscall/task/execve.rs#sys_execve` 解析可执行文件路径并替换进程映像。
- `kernel/src/task/ops.rs#do_exit` 清理 futex、共享内存、触发子进程通知。
- `kernel/src/syscall/task/wait.rs#sys_waitpid` 支持 `WNOHANG` 等选项的阻塞等待。
- `kernel/src/syscall/signal.rs#sys_rt_sigprocmask/sys_rt_sigaction` 维护信号掩码与动作表。
- `kernel/src/syscall/fs/pidfd.rs#sys_pidfd_open` 提供 pidfd 抽象。
- `kernel/src/pseudofs/proc.rs#lookup_child/child_names` 暴露 `/proc/[pid]/stat|status|maps|fd|...`。

但**调度器核心算法（如 CFS/RR）、上下文切换汇编、内核线程/idle 任务、运行队列**等底层证据未体现；项目大量调用 `axtask::*`，说明任务调度核心依赖外部运行时。

### FS（文件系统）
文件系统是该作品证据最强的模块：
- VFS 高层：`axfs-ng-local/src/highlevel/file.rs#open`、`axfs-ng-local/src/highlevel/fs.rs` 提供路径解析、符号链接处理、打开/创建语义。
- ext4 后端：`axfs-ng-local/src/fs/ext4/inode.rs#create/read_dir/append/link`、`axfs-ng-local/src/fs/ext4/fs.rs#new` 基于外部/本地 ext4 库实现磁盘文件系统。
- 伪文件系统：`kernel/src/pseudofs/tmp.rs`（tmpfs）、`kernel/src/pseudofs/proc.rs`（procfs）、`kernel/src/pseudofs/dev/*`（devfs）等。
- 系统调用：`sys_mount`、`sys_openat`、`sys_readlinkat`、`sys_statx`、`sys_fallocate`、`sys_splice`、`sys_memfd_create`、`sys_close_range` 等。
- 挂载框架：`kernel/src/syscall/fs/mount.rs#sys_mount` 调用 `axfs::mount_extra_block_device`。

### DEV（设备驱动）
该作品在 `kernel/src/pseudofs/dev/` 下实现了字符设备文件抽象，而非传统设备驱动模型：
- TTY：`kernel/src/pseudofs/dev/tty/mod.rs#ioctl`、`kernel/src/pseudofs/dev/tty/terminal/ldisc.rs#new/read/poll` 实现行规程、阻塞读、`PollSet` 事件等待。
- 输入事件：`kernel/src/pseudofs/dev/event.rs#ioctl`。
- 回环块设备：`kernel/src/pseudofs/dev/loop.rs#ioctl`。
- 随机数/显示/log：`kernel/src/pseudofs/dev/mod.rs#builder`、`kernel/src/pseudofs/dev/log.rs#bind_dev_log`。

### SYSCALL（系统调用）
- `kernel/src/syscall/mod.rs#handle_syscall` 是统一的系统调用分发入口，规模极大（217 条调用关系），覆盖文件、进程、内存、网络、IPC、信号、时间、资源限制等类别。
- 子模块包括 `kernel/src/syscall/fs/*`、`task/*`、`mm/*`、`net/*`、`ipc/*`、`signal.rs`、`sync/futex.rs`、`time.rs`、`resources.rs` 等。

### TRAP/IRQ（中断异常）
该模块**底层证据极弱**。提供的证据中仅出现：
- `kernel/src/file/signalfd.rs#from_signal_info`
- `kernel/src/pseudofs/dev/tty/terminal/termios.rs#signo_for/default`

这些属于信号/termios 辅助函数，**不是中断向量、异常分发、IRQ 控制器或时钟中断处理**。因此，架构级中断/异常处理机制底层证据未体现。

### NET（网络协议栈）
证据显示该作品实现了套接字系统调用层：
- `kernel/src/syscall/net/socket.rs#sys_socket` 支持 `AF_INET/AF_INET6/AF_UNIX/AF_VSOCK`。
- `kernel/src/syscall/net/socket.rs#sys_accept4` 封装 `socket.accept`。
- `kernel/src/syscall/net/io.rs#sys_sendmsg/recv_impl` 处理 `msghdr`、`cmsghdr`、`IoVectorBuf`、地址转换。

但**完整自研 TCP/IP 协议栈、网卡驱动、ARP/IP/路由表**等底层证据未体现；底层可能依赖 `axnet` 等外部 crate。
