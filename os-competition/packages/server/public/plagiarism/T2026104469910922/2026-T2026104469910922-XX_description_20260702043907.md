## 第四部分：分模块架构说明

### MM（内存管理）
底层证据强。核心文件 `arch/riscv/mm/vm.c` 提供页表遍历 `walk`、映射 `mappages`、用户空间分配 `uvmalloc`、地址空间复制 `uvmcopy`、COW 处理 `handle_cow_fault`、通用缺页 `handle_page_fault`、VMA 缺页 `handle_vma_fault`、文件映射缺页 `handle_file_vma_fault`。物理页分配由 `kernel/mm/kalloc.c#kalloc/kfree` 实现。启动阶段 `kvminit`/`kvmmap` 建立内核页表，并支持高地址映射切换。`copyin/copyout` 通过页表 walk 实现用户/内核数据拷贝。

### PROC（进程管理/任务调度）
底层证据强，属于**传统进程管理**，而非轻量级异步运行时。存在 PCB 分配 `kernel/core/proc.c#alloc_process`、进程创建 `fork`、程序执行 `execve_kernel`、退出 `exit`、等待 `wait4`、调度 `scheduler/sched/yield`、上下文切换 `swtch`。`sleep/wakeup` 机制用于阻塞式 I/O 与调度。`first_ret` 中挂载根文件系统并进入第一个用户进程。

### FS（文件系统）
底层证据强，采用**VFS + 具体文件系统**两层结构：
- VFS 层：`kernel/fs/vfs.c#vfs_init/vfs_mount_fs/vfs_create_at/vfs_unlink_at/vfs_mkdir_at/vfs_lookup_at`
- 自研 EasyFS：`kernel/fs/easyfs/*`，包括 inode 管理、目录查找、块分配 `balloc`、块映射 `bmap`、inode 持久化 `easyfs_write_inode`
- 第三方 ext4 集成：`kernel/fs/ext4fs/probe.c#ext4_probe`、`ext4_mount_root`、`ext4_read_inode`、`ext4_lookup_path`、`ext4_read_file`，属于对 ext4 的探测/读取层
- `mkfs/mkfs.c` 为宿主机工具，用于制作文件系统镜像

### DEV（设备驱动）
底层证据中等，覆盖：
- 块设备：`kernel/driver/virtio_blk.c#virtio_disk_init/rw/intr`，使用描述符环、中断与 sleep/wakeup
- 控制台：`arch/riscv/driver/uart.c#uartintr / hal_console_putc / pre_uart_init`
- 时钟：`arch/riscv/driver/timer.c#timerinit / pre_timerinit`，通过 CLINT/SBI 设置定时器
- 中断控制器：`arch/riscv/driver/PLIC.c`
- SBI 接口：`arch/riscv/driver/sbi.c#sbi_shutdown / sbi_set_timer`

### SYSCALL（系统调用）
底层证据强。`kernel/core/syscall.c#syscall` 根据系统调用号分发到 `syscalls[num]`；参数提取由 `argint/argaddr/argstr/argraw` 完成。文件类调用集中在 `kernel/core/sysfile.c`（read/write/close/dup/exec/pipe/chdir/lseek/newfstatat），进程类调用集中在 `kernel/core/sysproc.c`（fork/exit/wait4/brk/getpid/getppid/set_tid_address/shutdown）。用户态封装见 `user/ulib.c`。

### TRAP/IRQ（中断异常）
底层证据中等。M 态时钟陷阱 `arch/riscv/trap/mtrap.c#m_trap`；S 态入口 `arch/riscv/trap/trap.c#s_trap_handler` 处理外部中断（PLIC claim/complete）与异常；用户态陷阱 `usertrap` 处理缺页与系统调用；`usertrapret` 负责返回用户态；`trapinit` 设置 stvec。

### NET（网络协议栈）
**底层证据未体现**。源码与调用链中未出现 socket、TCP/IP、NIC 驱动、网络缓冲区等相关函数。
