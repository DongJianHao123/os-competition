## 第四部分：分模块架构说明

### MM（内存管理）

- **已体现证据**：
  - `kernel/src/syscall/mm/mmap.rs#sys_mmap` 支持 `Backend::new_alloc / new_cow / new_file / new_linear / new_shared` 等多种映射后端。
  - `kernel/src/syscall/mm/mmap.rs#sys_mremap_with_new_addr` 处理地址重新映射。
  - `kernel/src/mm/aspace/backend/cow.rs#alloc_new_at` 实现 COW 后端，调用 `file.read_at` 与 `phys_to_virt` 填充页面。
  - `kernel/src/mm/loader.rs#load_user_app` / `#load` 负责 ELF 加载与解析。
  - 用户栈构造依赖 `third_party/kernel-elf-parser/src/user_stack.rs#app_stack_region`。
- **未体现 / 依赖第三方**：物理页分配器初始化在 `third_party/arceos/axruntime/src/lib.rs#init_allocator` 中调用 `axalloc::global_init / global_add_memory`；页错误处理路径、TLB 刷新等底层证据未在 kernel 目录中直接体现。

### PROC（进程管理 / 任务调度）

- **已体现证据**：
  - 进程控制结构：`PROCESS_TABLE`、`PROCESS_GROUP_TABLE`、`SESSION_TABLE`、`TASK_TABLE`。
  - 生命周期：`kernel/src/syscall/task/clone.rs#do_clone`、`kernel/src/syscall/task/execve.rs#sys_execve`、`kernel/src/task/ops.rs#do_exit`、`kernel/src/syscall/task/wait.rs#sys_waitpid`。
  - 进程间关系：`sys_getpid`、`sys_getppid`、`sys_pidfd_open`、`sys_pidfd_getfd`、`sys_pidfd_send_signal`。
  - 信号：`kernel/src/syscall/signal.rs#sys_rt_sigprocmask`、`sys_rt_sigaction`、`sys_kill`。
  - procfs：`kernel/src/pseudofs/proc.rs#lookup_child`、`#child_names`、`#new_procfs`。
  - robust futex：`kernel/src/task/ops.rs#exit_robust_list`。
- **调度器说明**：核心调度实体由 `axtask` 提供（`third_party/axtask/src/task.rs`），内核层通过 `AxTaskExt` 扩展挂载 `proc_data`，实现 Linux 语义与轻量任务之间的映射。因此“完整调度器”的底层证据在第三方库中。

### FS（文件系统）

- **已体现证据**：
  - 系统调用层：`sys_openat`、`sys_mount`、`sys_umount2`、`sys_fcntl`、`sys_readlinkat`、`sys_splice`、`sys_fallocate`、`sys_memfd_create` 等。
  - 文件描述符表：`FD_TABLE`（含 scope 隔离）。
  - 挂载上下文：`FS_CONTEXT`。
  - 伪文件系统：`kernel/src/pseudofs/proc.rs`（procfs）、`kernel/src/pseudofs/dev/`（devfs，含 tty、pty、event、loop、display 等）。
  - 第三方实现：
    - ext4：`third_party/lwext4_rust/c/lwext4/src/ext4*.c`（block alloc、inode alloc、extent、journal、dir idx）。
    - 高层页缓存 / VFS：`third_party/arceos/axfs/src/highlevel/file.rs`。
- **未体现**：真实块设备驱动（loop 设备通过文件模拟）、NVMe/SATA/SD 等存储控制器底层证据未体现。

### DEV（设备驱动）

- **已体现证据**：
  - 字符设备与伪设备：`/dev/console`、`/dev/tty`、`/dev/pty`、`/dev/event`、`/dev/loop`、`/dev/random` 等。
  - TTY 行规程：`kernel/src/pseudofs/dev/tty/terminal/ldisc.rs`。
- **未体现**：真实硬件驱动（PCIe、USB、NIC 控制器、GPU、声卡等）底层证据未体现；显示相关仅见 `axdisplay::has_display` 等条件调用。

### SYSCALL（系统调用）

- **已体现证据**：
  - 集中分发：`kernel/src/syscall/mod.rs#handle_syscall`（规模 16973，调用数 235）。
  - 覆盖范围：进程、文件、内存、网络、IPC（msg/shm）、信号、时间、epoll/select、bpf（stub）、acct、prctl、resources 等。
  - 参数传递：使用 `UserConstPtr`、`UserPtr`、`vm_read` / `vm_write` 等用户态指针访问原语。
- **未体现**：seccomp、LSM、audit、ptrace 等高级系统调用机制底层证据未体现。

### TRAP/IRQ（中断异常）

- **已体现证据**：仅有 `kernel/src/file/signalfd.rs#from_signal_info`、`kernel/src/pseudofs/dev/tty/terminal/termios.rs#signo_for` / `#default` 等 3 个函数，均与信号 / termios 相关，**不是架构级 trap/irq 入口**。
- **未体现**：CPU 异常向量表、中断控制器（GIC/IOAPIC/PLIC）、时钟中断入口、上下文保存与恢复等底层证据未体现，推测由 `axhal` 等第三方库封装。

### NET（网络协议栈）

- **已体现证据**：
  - socket 系统调用：`kernel/src/syscall/net/socket.rs#sys_socket`、`sys_bind`、`sys_socketpair`；`kernel/src/syscall/net/io.rs#sys_sendmsg`、`recv_impl`。
  - 地址解析：`kernel/src/syscall/net/addr.rs#read_from_user`。
  - 第三方实现：`third_party/axnet-ng/src/tcp.rs`、`udp.rs`、`device/ethernet.rs`、`router.rs`、`vsock/` 等，支持 IPv4/IPv6、TCP/UDP、ARP、Unix Domain Socket、VSOCK。
- **未体现**：真实网卡驱动、DMA、checksum offloading、高级路由 / netfilter 等底层证据未体现。
