## 第四部分：分模块架构说明

### MM（内存管理）

底层证据较强。内核实现了 RISC-V Sv39 三级页表管理，关键函数包括：

- `kernel/vm.c#walk`：页表遍历，必要时分配页表页
- `kernel/vm.c#uvmcopy`：`fork()` 时复制用户地址空间
- `kernel/vm.c#uvmalloc` / `kernel/vm.c#uvmdealloc`：堆扩展与收缩
- `kernel/vm.c#vmfault`：缺页异常处理，调用 `kalloc()`、`mappages()` 进行按需映射
- `kernel/vm.c#copyin` / `kernel/vm.c#copyout`：用户态与内核态数据拷贝
- `kernel/trap.c#usertrap` 中处理页面错误并可能进入 `vmfault`

此外，`user/ulib.c#sbrklazy` 调用 `sys_sbrk`，结合 `vmfault` 可推断实现了**惰性堆分配（lazy sbrk）**：用户空间先扩展虚拟地址，实际物理页在首次访问缺页时分配。

### PROC（进程管理/任务调度）

底层证据完整，属于**传统 Unix 式进程模型**，而非现代异步运行时：

- `kernel/proc.c#allocproc` / `kernel/proc.c#freeproc`：PCB 分配与回收
- `kernel/proc.c#kfork`：创建子进程，调用 `uvmcopy`
- `kernel/proc.c#kexit` / `kernel/proc.c#kwait`：进程终止与等待
- `kernel/proc.c#scheduler` / `kernel/proc.c#sched` / `kernel/proc.c#yield`：轮转式上下文切换
- `kernel/proc.c#sleep` / `kernel/proc.c#wakeup`：条件等待与唤醒
- `kernel/proc.c#reparent`：处理孤儿进程

用户态测试 `user/forphan.c`、`user/dorphan.c`、`user/zombie.c` 进一步验证了 fork/exit/wait/僵尸进程/孤儿进程收养等机制。

### FS（文件系统）

底层证据强，采用类 xv6 的日志文件系统：

- `kernel/fs.c#writei` / `kernel/fs.c#readi`：inode 读写
- `kernel/fs.c#bmap`：直接/间接块映射
- `kernel/fs.c#namex` / `kernel/fs.c#dirlookup`：路径解析与目录查找
- `kernel/fs.c#ialloc` / `kernel/fs.c#balloc`：inode 与数据块分配
- `kernel/fs.c#ireclaim`：inode 回收
- `kernel/log.c#install_trans`：日志事务提交
- `kernel/sysfile.c#sys_open` / `sys_link` / `sys_unlink` / `sys_chdir`：文件系统系统调用
- `mkfs/mkfs.c#main`：宿主编译期构建磁盘镜像

### DEV（设备驱动）

证据中等，覆盖基本字符与块设备：

- **UART 串口**：`kernel/uart.c#uartinit`、`kernel/uart.c#uartwrite`、`kernel/uart.c#uartintr`
- **控制台**：`kernel/console.c#consoleinit`、`kernel/console.c#consoleread`、`kernel/console.c#consolewrite`
- **virtio 块设备**：`kernel/virtio_disk.c#virtio_disk_init`、`kernel/virtio_disk.c#virtio_disk_rw`、`kernel/virtio_disk.c#virtio_disk_intr`
- **PLIC 中断控制器**：`kernel/plic.c#plicinit`、`kernel/plic.c#plicinithart`、`kernel/plic.c#plic_claim`、`kernel/plic.c#plic_complete`

### SYSCALL（系统调用）

证据完整但函数数量统计偏少（仅 5 个），因为大量具体实现分散在 `sysproc.c` 与 `sysfile.c`：

- `kernel/syscall.c#syscall`：系统调用分发
- `kernel/syscall.c#argint` / `argstr` / `argaddr` / `argraw`：参数提取
- `kernel/sysproc.c`：进程相关调用（`sys_fork`、`sys_exit`、`sys_wait`、`sys_kill`、`sys_pause`、`sys_sched_yield`、`sys_getpid`、`sys_getppid`、`sys_times`、`sys_uptime`、`sys_uname`、`sys_getcwd` 等）
- `kernel/sysfile.c`：文件相关调用（`sys_open`、`sys_exec`、`sys_pipe`、`sys_link`、`sys_unlink`、`sys_chdir` 等）

扩展系统调用（如 `getcwd`、`getppid`、`times`、`uname`、`sched_yield`、`pause`）表明项目在基础 xv6 之上做了功能增强。

### TRAP/IRQ（中断异常）

证据中等：

- `kernel/trap.c#usertrap`：用户态陷阱入口，处理系统调用、缺页、外部中断
- `kernel/trap.c#kerneltrap`：内核态陷阱入口
- `kernel/trap.c#devintr`：中断分发，路由到 `clockintr`、`uartintr`、`virtio_disk_intr`、PLIC
- `kernel/trap.c#clockintr`：时钟中断，更新计时并触发调度
- `kernel/start.c#timerinit`：S 模式定时器初始化
- `kernel/plic.c`：PLIC 初始化与中断 claim/complete

### NET（网络协议栈）

**底层证据未体现**。当前 AST Call Map、函数列表与目录结构中均未出现网络接口、协议栈、套接字、TCP/IP、以太网驱动等相关函数或文件。

## 第六部分：边界与未体现内容

### 1. 没有底层证据支撑的模块

- **网络协议栈（NET）**：未出现任何网络设备驱动、套接字接口、TCP/IP/UDP 协议处理函数。
- **完整异步运行时**：虽然统计模块名为“任务调度/异步运行时”，但实际证据仅为 `sleep/wakeup`、`scheduler/sched/yield` 等传统阻塞式调度原语，未体现 `async/await`、`task`、`channel`、`waker`、`ring_fifo` 等异步运行时结构。
- **实时调度器**：未出现优先级调度、EDF、Rate-Monotonic 等实时调度证据。
- **安全/隔离增强模块**：未出现 capabilities、seccomp、ASLR、KASLR、SMAP/SMAP 等高级安全机制。
- **虚拟化支持**：未出现 Hypervisor 扩展、VMM、虚拟机退出处理等代码。

### 2. 需要单独说明的第三方/协议强约束代码

- `mkfs/mkfs.c` 是**宿主编译期工具**，用于生成 xv6 风格的磁盘镜像，不属于内核运行时主体。
- `kernel/virtio_disk.c`、`kernel/plic.c`、`kernel/uart.c` 中大量寄存器访问遵循 RISC-V 与 virtio/PLIC/16550 硬件协议规范，属于“受硬件协议强约束”的代码，不应被过度解读为原创内核算法。
- `user/grind.c`、`user/logstress.c`、`user/forphan.c`、`user/dorphan.c`、`user/myCall.c` 等为用户态测试程序，用于验证内核行为，但不是内核模块本身。
