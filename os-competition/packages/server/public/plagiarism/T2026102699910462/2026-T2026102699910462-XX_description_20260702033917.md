## 第四部分：分模块架构说明

### MM（内存管理）
底层证据充分。包含：
- 物理页分配：`mm/src/buddy/allocator.rs` 实现 Buddy 分配器，`kernel/src/memory.rs#initialize_page_allocator` 完成安装。
- 内核堆：`mm/src/heap/allocator.rs` 按 `SizeClass` 分桶，`mm/src/heap/large.rs` 处理大对象，`mm/src/slab/cache.rs` 提供 Slab 缓存。
- 用户地址空间：`kernel/src/user_mm.rs#protect_range` 处理 `mprotect` 语义；`kernel/src/runtime_page_table.rs` 提供运行时页表操作。
- TLB 一致性：`kernel/src/tlb.rs#verify_request_model` 涉及 ASID/CPU 作用域刷新。

### PROC（进程管理/任务调度）
具备传统进程管理证据。`kernel/src/process.rs` 中出现 PCB 类结构：进程 ID、线程组、文件表、信号动作、父子进程与僵尸子进程链表、`fork_child` / `create_initial_thread` / `exec_replace_context` / `wait_zombie_child`。`kernel/src/task/mod.rs` 提供调度器原语：线程创建、上下文切换、等待队列、工作队列、`SCHEDULER.lock()` 调度入队。`kernel/src/user.rs#sys_clone` / `sys_execve` / `sys_wait4` 进一步证实 fork/exec/wait/zombie 流程。

### FS（文件系统）
证据较强。`vfs/src/lib.rs` 提供跨内核的 VFS 抽象（`emit_dirent64` 等）；`kernel/src/fs/mod.rs` 实现路径查找、读写、重命名、procfs 挂载；`kernel/src/ext4.rs` 实现 EXT2/EXT4 只读解析（superblock、inode、目录项、extent 拷贝）；`kernel/src/initramfs.rs` 解析/生成 cpio newc 格式 initramfs。

### DEV（设备驱动）
中等证据。`kernel/src/device.rs` 维护驱动注册表；`kernel/src/block.rs` 提供块设备抽象；`kernel/src/virtio.rs` 通过 `DMA_ALLOCATIONS` 与 `page_alloc::free` 管理 virtio DMA；`firmware/fdt/src/tree.rs` 解析 FDT 以发现 virtio-mmio 区域与 PCI host bridge；`arch/loongarch64/src/smp.rs#start_secondary` 涉及多核启动。

### SYSCALL（系统调用）
`kernel/src/user.rs#handle_syscall` 是总入口（调用数 140+），向下分发文件、进程、内存、时间、信号、调度等系统调用。具体可见 `sys_openat`、`sys_read`、`sys_write`、`sys_pread64`、`sys_pwrite64`、`sys_newfstatat`、`sys_clone`、`sys_execve`、`sys_wait4`、`sys_brk`、`sys_munmap`、`sys_nanosleep`、`sys_clock_nanosleep`、`sys_sched_getaffinity` 等。

### TRAP/IRQ（中断异常）
`kernel/src/trap.rs#handle_breakpoint` 处理断点异常；`kernel/src/user.rs#handle_exception` 处理用户态缺页/异常并统计 `FAULT_COUNT`；`kernel/src/timer.rs#handle_interrupt` 驱动 tickless 定时器队列；`kernel/src/irq_lock.rs` 提供中断安全自旋锁。

### NET（网络协议栈）
**底层证据未体现完整网络协议栈**。仅 `kernel/src/net/socket.rs` 出现 socket 文件描述符表、`SocketType::Tcp/Udp`、`SockAddrIn` 拷贝与端口转换，没有 IP、TCP/UDP 状态机、ARP、NIC 驱动或数据包收发证据。
