## 第四部分：分模块架构说明

### MM（内存管理）

**证据较强**。核心路径包括用户态页故障处理、物理帧分配、用户地址空间校验、ELF 加载映射、memlock 等。

- `arceos/modules/axmm/src/backend/alloc.rs#handle_page_fault_alloc`：在页故障时分配连续/单帧，并处理帧释放与标志。
- `arceos/modules/axmm/src/aspace.rs#handle_page_fault_write`：处理写时复制/写权限页故障，支持 `grows_down` 后端。
- `pulse_core/src/mm/loader.rs#load_user_app`：ELF 解析与加载，设置 AuxEntry。
- `pulse_core/src/task/process.rs#validate_user_range` / `#align_user_range`：用户地址范围合法性校验。
- `pulse_core/src/task/process.rs#memlock_try_lock_range` / `#memlock_insert_range` / `#memlock_remove_range`：mlock/munlock 语义实现。

### PROC（进程管理/任务调度）

**证据强**。具备传统 Unix 进程管理所需的核心要素：PCB、父子关系、fork/exec、wait/zombie、线程、信号、futex、fd 表、credentials、fs_context、itimer 等。

- `pulse_core/src/task/process.rs#new_child_process`：构造子进程 PCB，初始化 credentials、fd_table、futex_table、memlock、resources 等。
- `pulse_core/src/task/process.rs#spawn_fork_from_trap_frame` / `#spawn_from_trap_frame`：从 trap frame 创建新进程/线程。
- `pulse_core/src/task/exec.rs#exec` / `#load_elf`：execve 路径，替换地址空间并加载 ELF。
- `pulse_core/src/task/process.rs#begin_group_exit` / `#finish_thread_exit` / `#reap_zombie_child`：进程组退出与僵尸回收。
- `pulse_syscalls/src/impls/task/wait.rs#sys_waitid` / `#sys_wait4`：waitpid/waitid 语义。
- `pulse_core/src/task/signal.rs#queue_signal_to_process` / `#pick_thread_for_process_signal`：进程级信号投递。
- `pulse_core/src/task/process.rs#futex_wait` / `#futex_wake_impl`：futex 等待与唤醒。
- `pulse_core/src/fd_table.rs#wait_ready`：fd 就绪等待。

**任务调度**：`arceos/modules/axtask/src/wait_queue.rs` 与 `run_queue.rs` 提供阻塞等待、超时、yield、退出等调度原语。但“异步运行时”底层证据不足，代码中未见 async/await 调度器，主要是同步阻塞式任务控制。

### FS（文件系统）

**证据强**。包含 VFS 层、mount 传播、procfs/tmpfs/devfs、ext4 第三方实现及大量 POSIX 文件系统调用。

- `pulse_syscalls/src/impls/fs/path.rs#sys_mount` / `#sys_umount2` / `#sys_mount_propagation`：mount、umount、传播类型（shared/private/slave/unbindable）。
- `crates/axfs-ng-vfs/src/mount.rs#propagate_new_mount`：mount 传播的核心 VFS 逻辑。
- `pulse_syscalls/src/impls/fs/path.rs#sys_openat` / `#sys_unlinkat`：文件打开与删除。
- `pulse_syscalls/src/impls/fs/io.rs#sys_read` / `#sys_write` / `#sys_pread64` / `#sys_pwrite64` / `#sys_sync`：读写与同步。
- `pulse_syscalls/src/impls/fs/control.rs#sys_ioctl`：终端/RTC 等 ioctl。
- `pulse_syscalls/src/impls/fs/meta.rs#sys_statx` / `#sys_utimensat`：元数据查询与修改。
- `arceos/modules/axfs/src/fs/procfs.rs#bootstrap` / `#get_inode` / `#render_proc_file`：procfs 实现。
- `arceos/modules/axfs/src/fs/tmpfs.rs#rename` / `#new_inode`：tmpfs。
- `arceos/modules/axfs/src/fs/devfs.rs#read_at` / `#rename`：devfs。
- `arceos/modules/axfs/src/fs/ext4/inode.rs#rename`：ext4 VFS 桥接。
- `crates/ext4plus/src/file_blocks/extent_tree.rs#write_at` / `#write_into_newly_allocated_extent`：ext4 extent 写。
- `crates/ext4plus/src/resolve.rs#resolve_path`：ext4 路径解析。

### DEV（设备驱动）

**证据较弱**。内核中可见设备枚举初始化，但具体驱动实现深度不足。

- `arceos/modules/axdriver/src/lib.rs#init_drivers`：探测并枚举 block/net/display 设备。
- `arceos/tools/bwbench_client/src/device.rs#new` / `#bind_interface` / `#send` / `#recv`：主机侧网络测试工具，非内核驱动。
- `arceos/tools/raspi4/chainloader/src/bsp/device_driver/bcm/bcm2xxx_pl011_uart.rs#init`：树莓派链式加载器 PL011 UART 初始化。

底层块设备控制器、网卡 MAC 驱动、显示驱动、DMA、中断路由等**底层证据未体现**。

### SYSCALL（系统调用）

**证据强**。具备完整的 Linux 兼容系统调用分发层。

- `pulse_syscalls/src/handler.rs#syscall_handler`：syscall 入口，记录时间、处理信号中断、调用分发器。
- `pulse_syscalls/src/handler.rs#syscall_dispatcher`：根据 `Sysno` 分发到 180+ 个具体实现。
- `pulse_syscalls/src/handler.rs#set_syscall_ret`：设置系统调用返回值。
- 覆盖 fs、net、task、ipc、mm、time、misc 等子系统的具体实现，如 `sys_openat`、`sys_bind`、`sys_waitid`、`sys_nanosleep`、`sys_rt_sigprocmask` 等。

### TRAP/IRQ（中断异常）

**证据弱**。仅见架构相关异常入口，缺少中断控制器、IRQ 分发、时钟中断处理等深度实现。

- `crates/axcpu/src/riscv/trap.rs#riscv_trap_handler`：RISC-V 异常入口，区分 syscall/page fault/breakpoint。
- `crates/axcpu/src/loongarch64/trap.rs#loongarch64_trap_handler`：LoongArch 异常入口。
- 页故障会进入 `handle_page_fault`，再调用 `axmm` 的页故障处理函数。

中断控制器（PLIC/APLIC/IOAPIC）、IRQ 注册与分发、时钟 tick 中断等**底层证据未体现**。

### NET（网络协议栈）

**证据中等，且大量为第三方代码**。原创部分主要是本地 socket 与 syscall 胶水，TCP/IP 协议主体来自 `smoltcp`。

- `pulse_core/src/net/mod.rs#new_pair` / `#read` / `#write`：本地 socket（类 Unix domain socket）的 ring buffer 实现。
- `pulse_syscalls/src/impls/net/socket.rs#sys_socket` / `#sys_bind` / `#sys_connect` / `#sys_accept4` / `#sys_shutdown`：socket 系统调用。
- `arceos/api/arceos_posix_api/src/imp/net.rs#recvfrom` / `#recv` / `#shutdown`：POSIX API 桥接。
- `crates/smoltcp/src/socket/tcp.rs`（259 函数）与 `crates/smoltcp/src/wire/{ipv4,ipv6,rpl}.rs`：第三方 TCP/IP/IPv6/RPL 协议栈主体。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块/能力

1. **完整异步运行时**：虽然模块画像中有“任务调度/异步运行时”，但代码证据主要是 `axtask` 的阻塞 wait_queue、sleep/yield、run_queue，未见 async/await 调度器、waker、poll 循环等异步运行时核心结构。
2. **中断控制器与 IRQ 分发**：仅有 RISC-V/LoongArch 异常入口，未见 PLIC/APLIC/IOAPIC 初始化、IRQ 注册、中断线程化、软中断等。
3. **具体硬件设备驱动**：`axdriver` 只有设备枚举，未见网卡 MAC、块控制器、显示控制器、DMA、PCI 等驱动的寄存器操作主体。
4. **SMP/多核调度**：未见 CPU 启动、核间中断、per-CPU 运行队列、锁的 NUMA 感知等证据。
5. **虚拟内存子系统深度**：有页故障处理与地址空间校验，但未见页表遍历/TLB 刷新细节、swap、COW 完整链路、mmap 文件映射等。
6. **安全/权限机制深度**：有 `Credentials` 与基本 capability 检查，但未见 SELinux/LSM、seccomp、namespace 隔离（除 mount 传播外）等。
7. **电源管理、时钟子系统、内核调试/跟踪**：底层证据未体现。
8. **原创网络协议栈**：TCP/IP 主体来自 `smoltcp`，原创网络代码主要是本地 socket ring buffer 与 syscall 适配。

### 需要单独说明的第三方/外部依赖

- `crates/smoltcp/`（2635 函数）是第三方 TCP/IP 协议栈，不应视为该作品原创网络协议栈主体。
- `crates/ext4plus/`（863 函数）是第三方 ext4 实现，作品通过 `arceos/modules/axfs/src/fs/ext4/` 进行桥接。
- `crates/axcpu/`、`crates/axfs-ng-vfs/` 等位于 `crates/` 下，应视为外部依赖或 ArceOS 生态组件，而非作品原创内核主体。
- `arceos/` 目录整体为 ArceOS 框架/模块，作品在其上通过 `pulse_core` 与 `pulse_syscalls` 构建类 Unix 兼容层。
