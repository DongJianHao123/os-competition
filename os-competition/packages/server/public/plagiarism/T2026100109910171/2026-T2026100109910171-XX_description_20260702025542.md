## 第四部分：分模块架构说明

### MM（内存管理）
**有底层证据**。关键路径包括：
- `kernel/src/syscall/mm/mmap.rs#sys_mmap` 支持 `Backend::new_alloc / new_cow / new_file / new_linear / new_shared` 多种映射后端；
- `kernel/src/mm/aspace/backend/cow.rs#alloc_new_at` 实现写时复制（COW）后端的页分配与数据填充，调用 `file.read_at`、`phys_to_virt` 等；
- `kernel/src/mm/loader.rs#load` / `load_user_app` / `build_app_stack` 完成 ELF 加载、动态链接器解析、用户栈构造；
- `kernel/src/syscall/ipc/shm.rs#sys_shmat` / `sys_shmdt` 实现 System V 共享内存的 attach/detach。

未直接体现：硬件页故障（page fault）处理程序、TLB 刷新、物理页分配器底层实现。

### PROC（进程管理）
**有较强底层证据**。关键路径包括：
- `deps/starry-process/src/process.rs#new` / `fork` / `exit` / `create_group` / `create_session` 构建 PCB，包含 `Session`、`ProcessGroup`、`ThreadGroup`、子进程表；
- `kernel/src/task/ops.rs#do_exit` 处理退出时信号、futex 唤醒、共享内存清理、robust futex list；
- `kernel/src/syscall/task/wait.rs#sys_waitpid` 实现 `waitpid` 语义，检查 `child.is_zombie` 与 `child.exit_code`；
- `kernel/src/syscall/task/execve.rs#sys_execve` 实现映像替换；
- `kernel/src/syscall/task/clone.rs#do_clone` / `sys_fork` 实现线程/进程克隆；
- `kernel/src/syscall/signal.rs#sys_rt_sigprocmask` / `sys_rt_sigtimedwait` 实现信号掩码与定时等待；
- `kernel/src/syscall/fs/pidfd.rs#sys_pidfd_open` / `sys_pidfd_getfd` / `sys_pidfd_send_signal` 实现 pidfd 机制。

说明：`deps/starry-process` 是独立依赖 crate，提供进程抽象；`kernel/src/task` 主要做包装与系统调用层处理。

### TASK/SCHED（任务调度/异步运行时）
**证据为“系统调用层 + 异步 IO 封装”，调度器本体底层证据不足**。关键路径包括：
- `kernel/src/syscall/sync/futex.rs#sys_futex` 调用 `axtask::yield_now` 与 `block_on`；
- `kernel/src/file/epoll.rs#check_and_register_waker` 使用 `Waker::from`、`file.poll`、`file.register`；
- `kernel/src/pseudofs/dev/tty/terminal/ldisc.rs#read` / `#new` 使用 `block_on`、`WaitPollable`、`PollSet`；
- `kernel/src/syscall/task/schedule.rs#sys_sched_setaffinity` 调用 `axhal::cpu_num`、`axtask::set_current_affinity`。

未直接体现：上下文切换、运行队列、时间片分配、时钟中断驱动的调度器核心（当前证据主要依赖 `axtask` / `axhal` 外部 crate）。

### FS（文件系统）
**有强底层证据**。关键路径包括：
- VFS/伪文件系统：`kernel/src/pseudofs/proc.rs`（procfs，含 `/proc/[pid]/stat/status/fd/...`）、`kernel/src/pseudofs/tmp.rs`（tmpfs）、`kernel/src/pseudofs/dev/`（devfs）；
- fd 表与文件操作：`kernel/src/syscall/fs/fd_ops.rs#sys_openat` / `sys_close_range` / `sys_fcntl`、`kernel/src/syscall/fs/io.rs#sys_fallocate` / `sys_splice`；
- 挂载：`kernel/src/syscall/fs/mount.rs#sys_mount`；
- 状态与链接：`kernel/src/syscall/fs/stat.rs#sys_statx`、`kernel/src/syscall/fs/ctl.rs#sys_readlinkat`；
- 内存文件：`kernel/src/syscall/fs/memfd.rs#sys_memfd_create`；
- IO 多路复用：`kernel/src/syscall/io_mpx/select.rs#do_select`、`kernel/src/syscall/io_mpx/poll.rs#do_poll`、`kernel/src/syscall/io_mpx/epoll.rs#do_epoll_wait`、`kernel/src/file/epoll.rs`。

### DEV（设备驱动）
**仅有伪设备/字符设备证据，底层硬件驱动证据未体现**。关键路径包括：
- TTY：`kernel/src/pseudofs/dev/tty/mod.rs#ioctl`（termios、winsize）、`kernel/src/pseudofs/dev/tty/terminal/ldisc.rs`（行规程）；
- 输入事件设备：`kernel/src/pseudofs/dev/event.rs#ioctl`；
- 回环块设备：`kernel/src/pseudofs/dev/loop.rs#ioctl`（`loop_info`）。

未直接体现：真实块设备控制器、网络网卡驱动、DMA、中断控制器驱动。

### SYSCALL（系统调用）
**有强底层证据**。中央入口为 `kernel/src/syscall/mod.rs#handle_syscall`，规模与调用数最大，负责 `Sysno` 解析、`LinuxError` 错误映射，并分发至：
- `kernel/src/syscall/fs/*`
- `kernel/src/syscall/net/*`
- `kernel/src/syscall/mm/*`
- `kernel/src/syscall/ipc/*`
- `kernel/src/syscall/task/*`
- `kernel/src/syscall/signal.rs`
- `kernel/src/syscall/time.rs`

### TRAP/IRQ（中断异常）
**底层证据未体现**。模块画像仅 3 个函数：
- `kernel/src/file/signalfd.rs#from_signal_info`
- `kernel/src/pseudofs/dev/tty/terminal/termios.rs#signo_for`
- `kernel/src/pseudofs/dev/tty/terminal/termios.rs#default`

三者均与信号字符/termios 相关，**不是硬件中断/异常向量、页故障、时钟中断处理程序**。

### NET（网络协议栈）
**底层证据未体现**。当前证据仅包含 socket 系统调用包装：
- `kernel/src/syscall/net/socket.rs#sys_socket` / `sys_accept4`

这些函数创建 `SocketInner::Tcp/Udp/Unix/Vsock` 并调用 `TcpSocket::new` 等，但 TCP/IP 协议栈完整实现、网卡驱动、数据包处理在提供的 AST 中未体现。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块/能力
1. **硬件中断/异常处理（TRAP/IRQ）**：模块画像仅 3 个函数，且均与 signalfd/termios 相关；无中断向量、时钟中断、页故障处理程序证据。
2. **完整网络协议栈（NET）**：仅 socket 系统调用包装与地址解析，TCP/IP 协议实现、网卡驱动、数据包收发底层证据未体现。
3. **底层块设备/磁盘/DMA/总线驱动**：除 `loop` 伪设备、`event` 输入设备、`tty` 字符设备外，真实硬件驱动证据未体现。
4. **任务调度器核心**：上下文切换、运行队列、时间片、抢占等底层实现未直接体现，当前证据主要是对 `axtask` / `axhal` 的调用与异步 IO 封装。
5. **物理内存管理底层**：物理页分配器、伙伴系统、SLAB、TLB 管理、页表遍历等未直接体现。
6. **安全/权限模型完整实现**：capabilities、LSM、ACL 等底层证据未体现。
7. **多核同步原语底层实现**：自旋锁、原子操作等依赖外部 crate，内核主体中未直接体现。

### 第三方/公共依赖说明
- `deps/starry-process/`（35 个 `.rs`）提供进程/进程组/会话/线程组抽象，是独立的进程管理依赖 crate；
- `axtask`、`axhal` 等前缀调用表明任务调度与硬件抽象依赖外部公共运行时/硬件抽象层；
- 上述依赖不应被夸大为本项目 `kernel/src/` 原创内核主体。

### 判断边界
- 本报告中“可能的系统方向”仅基于路径、函数名、调用链的保守推断，**不是最终身份或原创性结论**。
- 本报告**不做抄袭/违规判断**，也不对代码正确性、完整性、性能作评价。
