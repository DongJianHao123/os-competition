## 第四部分：分模块架构说明

### MM（内存管理）
底层证据较强。实现了物理页框分配器（`FRAME_ALLOCATOR`），并在初始化时区分 `MEMORY_LOW`/`MEMORY_HIGH`/`MEMORY_END` 等区域。地址空间通过 `MemorySet` 管理，支持 ELF 加载（`MemorySet::from_elf`）、裸地址空间创建、页表翻译（`PageTable::from_token`）、懒分配/共享映射/解除映射（`insert_lazy_framed_area`、`insert_shared_area`、`remove_framed_area`）。每个线程的 Trap 上下文页也纳入内存集管理（`dealloc_trap_cx`）。`sys_brk`、`sys_futex` 等调用中可见用户态地址翻译。

### PROC（进程管理/任务调度）
底层证据较强。存在 PCB（`TaskControlBlock` / `ProcessControlBlock`）、全局进程表 `PID2PCB`、父子关系、`fork`/`clone`/`spawn`/`exec`/`waitpid`/`exit` 完整生命周期。支持线程（`clone_thread`）、进程凭证与 capabilities（`task/cred.rs`）、信号（`syscall/sig.rs`）、futex、epoll、timerfd、资源限制（rlimit）、调度参数/亲和性/nice、`/proc` 伪文件系统等。调度器为传统阻塞式： `run_tasks` 取任务、`__switch` 切换、condvar 阻塞当前任务并转调度。**异步运行时（async/await、waker、executor）的底层证据未体现**。

### FS（文件系统）
底层证据较强。内核层实现了路径解析（`syscall/path.rs#analyse_path`）、VFS 式文件描述符操作、权限检查（`check_dir_search`、`cred.may_access_file`）、`procfs`（`/proc/[pid]/...`）。实际文件系统语义大量委托给第三方 C 库 **lwext4**（`lwext4_rust/c/lwext4`），包括 ext4 目录、extent、日志、块/ inode 分配等。`sys_mount` 明确使用 `"ext4_fs"` 字符串并检查 `CAP_SYS_ADMIN`。

### DEV（设备驱动）
底层证据较强。主要依赖 `virtio-drivers-la` 与 `virtio-drivers` 两个 crate，实现了 virtio-blk（块设备读写/flush）、virtio-socket（连接管理）、virtio-console、virtio-input，以及 PCI/MMIO 传输层。这些代码属于第三方/硬件协议实现，不应视为原创内核主体。

### SYSCALL（系统调用）
底层证据强。`syscall/mod.rs#syscall` 是统一入口，分发表中包含 200 余个系统调用（`sys_accept`、`sys_acct`、`sys_add_key`、`sys_exec`、`sys_clone`、`sys_open` 等），覆盖进程、文件、网络、信号、定时器、消息队列等，呈现 Linux 兼容接口。

### TRAP/IRQ（中断异常）
底层证据中等。核心入口为 `trap/mod.rs#trap_handler`，处理异常/中断时会查询页表、维护 FPU 属主、检查信号与定时器；`enable_timer_interrupt` 操作 `ecfg` 与 `LineBasedInterrupt`，偏向 LoongArch 风格。同时存在 `riscv/src/interrupt.rs`、`mcause.rs`、`scause.rs` 等 RISC-V 中断/异常解析辅助代码，但主内核路径以 `ecfg` 为主。

### NET（网络协议栈）
**完整内核网络协议栈（TCP/IP）底层证据未体现**。可见：
- 系统调用层有 `r-core/os/src/syscall/socket.rs#sys_recvmsg` 等 socket 接口；
- 设备层有 virtio-socket 连接管理；
- 示例代码 `virtio-drivers-la/examples/{x86_64,riscv}/src/tcp.rs` 使用外部 `smoltcp` 库实现 echo server。
因此网络部分目前仅体现 socket syscall 入口与 virtio-socket 设备，未体现独立内核 TCP/IP 协议栈。

  - 完整内核态 TCP/IP 网络协议栈（仅有 socket syscall 入口与 virtio-socket 设备）；
  - 异步运行时（async/await、waker、executor）；
  - GPU/图形、音频、ACPI/电源管理；
  - 虚拟化 Hypervisor（`hypervisorx64/hcounteren.rs` 仅为寄存器描述文件，无 VM 管理证据）；
