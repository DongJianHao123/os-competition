- **第三方库/公共依赖**: `lwext4` 库的所有函数均为公共依赖。控方所有指控均集中于此，属于将“共同依赖开源库”错误归因为“抄袭”。根据审计指令，跨板块同名或同 hash 只能作为弱信号，而本案中控方未能提供任何跨板块证据。
- **公共接口与泛型**: 目标项目大量使用 Rust 的 `Arc`、`Mutex`、`Vec` 等标准库泛型，以及 `error!`、`info!` 等宏，这些是 Rust 生态的公共基础，不能作为同源证据。

#### 6. 保守结论
本案整体 AST 相似度仅 10.48%，且所有重合点均被锁定在“文件系统/第三方库”板块，经溯源为双方共同依赖的开源项目 **lwext4**。目标项目在操作系统核心（进程、内存、网络、驱动等）的 3566 个唯一 AST 指纹与参考项目零重合，且采用 Rust 语言构建，与参考项目在架构、语言和运行时上存在根本性差异。控方“抄袭”指控混淆了第三方公共库依赖与原创代码复用，其证据链未能排除“共同依赖上游”这一最合理解释。综合判定，目标项目不存在对参考项目的系统性抄袭，整体风险极低，应认定为独立开发。
