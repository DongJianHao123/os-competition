## 第四部分：分模块架构说明

### MM（内存管理）
底层证据较强。核心包括用户虚拟地址空间管理（`UserVMSet`）、缺页异常处理、页表映射、KSM（Kernel Same-page Merging）写保护处理、页缓存（`PAGE_CACHE`）、`brk` 扩展堆、以及 virtio-blk 的 DMA 页框管理。

证据锚点：
- `os/src/mm/vm_set.rs#handle_unalloc_page_fault`：构造 `PTE::new`、设置 `MappingFlags`、刷新 TLB。
- `os/src/mm/ksm.rs#handle_ksm_write_fault`：处理 KSM 共享页写时复制。
- `os/src/syscall/process.rs#sys_brk`：堆顶调整，调用 `brk_request_is_valid`。
- `os/src/drivers/block/virtio_blk.rs#dma_dealloc`：DMA 页框回收。

### PROC（进程管理/任务调度）
传统进程管理底层证据充分，不是轻量级 task/channel。存在 `clone`/`execve`/`waitid`/`wait4`/`exit`、PCB、父子进程关系、`fd_table`、信号处理、线程组（`tasks`）、zombie 清理等。

证据锚点：
- `os/src/task/process.rs#_clone_inner`：PCB 复制、fd 表、socket、tid 写入用户态。
- `os/src/task/process.rs#execve`：ELF 加载、地址空间替换、TrapFrame 初始化。
- `os/src/syscall/process.rs#sys_waitid` / `sys_wait4`：阻塞等待子进程。
- `os/src/task/mod.rs#exit_current_and_run_next`：进程退出、zombie 处理。
- `os/src/task/processor.rs#run_tasks` / `schedule`：调度循环与上下文切换。

“异步运行时”特征在提供的 Call Map 中**未明确体现**（未见 waker/ring_fifo/await 状态机），现有证据更偏向传统抢占/协作式任务调度。

### FS（文件系统）
文件系统是证据最强的模块。实现了 VFS 层（全局 `GLOBAL_DCACHE`、`FS_MANAGER`、挂载表），并支持多种后端：FAT32、tmpfs、devfs、procfs、etcfs，以及通过 `lwext4_rust` 封装的 ext4。还有 pipe、fanotify/inotify 等机制。

证据锚点：
- `os/src/fs/mod.rs#init`：根文件系统、devfs 挂载。
- `os/src/fs/vfs/path.rs#resolve_path_inner`：路径解析与 dentry 缓存。
- `os/src/syscall/fs.rs#do_mount` / `sys_umount2`：挂载/卸载管理。
- `os/src/fs/fat32/dentry.rs#rename`、`os/src/fs/tmpfs/dentry.rs#rename`：具体 FS 的 rename 实现。
- `os/src/fs/lwext4/dentry.rs#find` / `create` / `rename`：ext4 后端。
- `os/src/fs/procfs/mod.rs#init_procfs`、`os/src/fs/devfs/mod.rs#init_devfs`：伪文件系统。

### DEV（设备驱动）
设备驱动证据较弱。仅见 virtio-blk 块设备驱动、网络 loopback 设备、x86_64 APIC 初始化、DTB 内存解析等。没有 USB、PCI 枚举、显卡、声卡、NVMe、SATA 等驱动的底层证据。

证据锚点：
- `os/src/drivers/block/virtio_blk.rs#read_block` / `write_block` / `dma_dealloc`。
- `os/src/net/mod.rs#init`：loopback 网络设备初始化。
- `polyhal/polyhal/src/arch/x86_64/apic.rs#init`。
- `polyhal/polyhal/src/mem.rs#init_dtb_once`。

### SYSCALL（系统调用）
系统调用接口证据强。`os/src/syscall/mod.rs#syscall` 是总入口，分派到文件、进程、网络、信号、时间、misc、futex 等子模块，且对 RISC-V/LoongArch 有架构相关信号处理代码。

证据锚点：
- `os/src/syscall/mod.rs#syscall`：199 个调用目标的总分发。
- `os/src/syscall/fs.rs#sys_openat` / `sys_read` / `sys_write` / `sys_pread64` / `sys_pwrite64`。
- `os/src/syscall/process.rs#sys_clone` / `sys_execve` / `sys_waitid`。
- `os/src/syscall/net.rs#sys_sendto`。
- `os/src/syscall/misc.rs#sys_epoll_pwait`。

### TRAP/IRQ（中断异常）
证据主要依赖 `polyhal` 架构层。LoongArch 有完整的 trap handler（非法指令、缺页、非对齐访问模拟），x86_64 有 IDT 初始化，内核主循环有 timer tick 计数。

证据锚点：
- `polyhal/polyhal-trap/src/trap/loongarch64.rs#loongarch64_trap_handler`。
- `polyhal/polyhal-trap/src/trap/loongarch64/unaligned.rs#emulate_load_store_insn`。
- `polyhal/polyhal/src/arch/x86_64/idt.rs#new`。
- `os/src/main.rs#kernel_interrupt`。

### NET（网络协议栈）
网络协议栈证据较强。实现了 socket 层、UDP/TCP、IPv4、ARP、Ethernet、邻居表、loopback 设备，以及 `epoll` 相关就绪通知。

证据锚点：
- `os/src/socket/mod.rs#close`。
- `os/src/socket/udp.rs#send_udp_user_buffer` / `send_udp_packet`。
- `os/src/net/ip.rs#ip_rcv` / `ip_queue_xmit`。
- `os/src/net/udp.rs#udp_rcv`。
- `os/src/net/tcp.rs#dispatch_kernel_tcp_service`。
- `os/src/net/arp.rs#arp_request` / `arp_reply`。
- `os/src/net/ethernet.rs#ethernet_rcv`。
- `os/src/net/neighbor.rs#neighbour_output`。

## 第六部分：边界与未体现内容

### 第三方/强约束代码说明
- **`lwext4_rust/`**：是第三方 ext4 文件系统 C 实现（`lwext4`）的 Rust 封装，**不应视为原创内核文件系统主体**。
- **`iperf/`、`netperf-2.7.0/`**：是用户态网络性能测试程序，**不是内核网络协议栈本身**。
- **`polyhal/`**：硬件抽象层，包含 boot、trap、内存、APIC、IDT、percpu 宏等，其中部分代码属于架构强约束/公共基础设施，需与内核业务逻辑区分。

### 没有底层证据支撑的模块/特性
- **完整设备驱动生态**：除 virtio-blk、loopback、APIC/DTB 外，未见 USB、PCI 枚举、NVMe、SATA、显卡、声卡等驱动。
- **完整中断控制器驱动**：仅见 x86_64 APIC 初始化与 LoongArch trap handler，未见 GIC/PLIC 等完整中断分发证据。
- **安全/加密子系统**：未见 LSM/SELinux、内核加密 API、文件系统加密等证据。
- **IPv6、高级 TCP 拥塞控制、网络设备驱动**：网络栈证据集中在 IPv4/UDP/TCP/ARP/Ethernet/loopback，未见真实 NIC 驱动。
- **swap/页面置换、NUMA 物理内存管理**：虽有 `get_mempolicy`/`validate_nodemask` 等接口痕迹，但无完整 swap 或 NUMA 分配证据。
- **实时调度算法细节、完整 CFS 实现**：调度器存在，但具体调度算法证据不足。
- **异步运行时核心机制**：未见 waker、ring_fifo、async/await 状态机等明确证据。

### 判断边界
- 本报告中“可能的系统方向”“疑似”“可能”等表述仅基于 AST Call Map、函数名、路径锚点与调用链的技术推断，**不是对作品最终身份、原创性或合规性的结论**。
- 本报告**不做抄袭/违规判断**。
