#### 4. 核心代码调用流对比剖析

* **【进程管理/调度器初始化】**
* **底层技术析理**: 双方 `processor.rs::new` 均调用 `KContext::blank` 构造空白 CPU 上下文。`KContext` 为 polyhal 硬件抽象层提供的跨平台 CPU 状态结构体，`blank` 是其关联函数，用于生成全零/默认值上下文。任何基于 polyhal 构建的操作系统在初始化 per-CPU 处理器结构时，都必须调用此构造函数为 idle 任务准备初始上下文。该调用链是 polyhal 框架的强制依赖，而非双方互相复制的结果。
* **代码溯源链接**:
  - 目标项目: `os/src/task/processor.rs#new`
  - 参考项目: `os/src/task/processor.rs#new`

* **【进程管理/当前任务获取】**
* **底层技术析理**: 目标项目 `current_task` 通过 `PROCESSORS[id].as_mut().unwrap().lock().current` 获取当前任务，其中 `PROCESSORS` 为多核 per-CPU 数组，`id` 由 `get_tp` 获取当前核编号，`lock()` 使用标准库 Mutex。参考项目通过 `PROCESSOR.exclusive_access().current` 获取，其中 `PROCESSOR` 为单核全局变量，`exclusive_access` 为自定义 UPIntrFreeCell 独占访问。目标项目面向 SMP 多核架构，参考项目面向单核架构，两者在并发模型、同步原语、处理器寻址上存在根本性差异。
* **代码溯源链接**:
  - 目标项目: `os/src/task/processor.rs#current_task`
  - 参考项目: `os/src/task/processor.rs#current_task`

* **【进程管理/进程克隆】**
* **底层技术析理**: 目标项目 `sys_clone` 调用 `process._clone` 方法，错误处理使用 `SysError::try_from(errno).unwrap_or`，随后调用 `block_current_and_run_next` 主动让出 CPU。参考项目 `sys_clone` 调用 `current_task.clone_task` 方法，错误处理使用 `CloneFlags::from_bits(flags as u32).ok_or_else`，随后调用 `add_task` 将新任务加入调度队列。目标项目以 Process 为克隆主体，参考项目以 Task 为克隆主体，两者在对象模型、错误处理策略、调度集成方式上均不同。
* **代码溯源链接**:
  - 目标项目: `os/src/syscall/process.rs#sys_clone`
  - 参考项目: `os/src/syscall/process.rs#sys_clone`

* **【进程管理/进程退出】**
* **底层技术析理**: 目标项目 `sys_exit` 通过 `current_task().and_then(|t| t.process.upgrade()).map(|p| p.getpid())` 获取进程 ID，体现了 Task 内嵌 Process 的层级关系，日志使用 `info!` 宏。参考项目 `sys_exit` 直接调用 `exit_current_and_run_next`，日志使用 `trace!` 宏。目标项目在退出路径上需要向上追溯 Process 对象，参考项目直接以 Task 为退出单元，架构抽象层级不同。
* **代码溯源链接**:
  - 目标项目: `os/src/syscall/process.rs#sys_exit`
  - 参考项目: `os/src/syscall/process.rs#sys_exit`

* **【内存管理/ELF 加载】**
* **底层技术析理**: 目标项目 `from_elf` 处理 `interp_base` 动态链接器基址，计算 `interp_base + ph.virtual_addr()` 作为段映射地址，并调用 `GLOBAL_DCACHE.get` 与 `KERNEL_VMSET.lock` 进行 VFS 集成。参考项目 `from_elf` 直接使用 `ph.virtual_addr()` 映射，调用 `Aux::new` 构造辅助向量。目标项目支持动态链接与 VFS 缓存集成，参考项目仅支持静态 ELF 加载，内存管理复杂度与集成度显著不同。
* **代码溯源链接**:
  - 目标项目: `os/src/mm/vm_set.rs#from_elf`
