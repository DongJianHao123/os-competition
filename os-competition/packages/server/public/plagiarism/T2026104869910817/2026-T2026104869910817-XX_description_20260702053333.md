## 第四部分：分模块架构说明

### MM（内存管理）
底层证据较强。存在按对象大小分级的 slab 分配器（`os/src/mm/allocator/slab_allocator.rs#dealloc`、`dealloc_by_layout`），包含 `cache16/cache128/cache1024` 等缓存；用户地址空间管理集中在 `os/src/mm/vm/uvm.rs`（如 `load_dl_interp_if_needed` 加载动态链接器 `/lib/libc.so`）和 `os/src/mm/user.rs`；跨进程内存访问通过 `os/src/syscall/mm.rs#sys_process_vm_readv/writev` 实现；用户指针校验通过 `UserPtrRaw::ensure_read` / `UserSliceRaw::ensure_write` 与当前任务 `vm_space` 交互。

### PROC（进程管理/任务调度）
底层证据较强。传统进程管理可见：`os/src/syscall/process.rs#sys_execve`、`sys_waitpid`、`sys_clone`、`sys_fork`、`sys_exit`、`sys_exit_group`；`os/src/task/task.rs#fork`、`do_exit`、`handle_zombie`、`do_group_exit` 维护父子、进程组、会话关系；`PROCESS_GROUP_MANAGER` 与 `TASK_MANAGER` 被频繁调用。调度侧采用异步运行时：`os/src/executor/mod.rs#spawn` 调用 `async_task::spawn` 与 `TASK_QUEUE.push`，`run_until_idle` 在多处理器队列上调度；`os/src/processor/schedule.rs#load_balance` / `migrate_tasks` 实现任务迁移；`os/src/processor/processor.rs#switch_out_current_task` 负责任务切换与地址空间切换。

### FS（文件系统）
底层证据强。VFS 层有 `os/src/fs/vfs/file.rs#open_file`、`os/src/fs/vfs/dentry.rs#find`；具体文件系统包括 FAT32（`os/src/fs/fat32/inode.rs#lookup`）、ext4（`os/src/fs/ext4/fstype.rs#mount`、`inode.rs#create`、`cache_read_at`、`cache_write_at`）、procfs（`os/src/fs/procfs/mod.rs#init_procfs`、`meminfo.rs#serialize`、`mounts.rs#serialize`）、devfs（`os/src/fs/devfs/mod.rs#init_devfs`）。系统调用层覆盖 `sys_openat`、`sys_read`、`sys_write`、`sys_readv`、`sys_writev`、`sys_pread`、`sys_pwrite`、`sys_getdents64`、`sys_mount`、`sys_unlinkat`。注意 ext4 子目录大量调用 `Ext4File`、`dir_mk`、`file.file_close` 等，呈现对第三方 ext4 实现的封装特征。

### DEV（设备驱动）
底层证据较强。字符设备：`os/src/devices/serial.rs#get_serial`、`scan_char_device`；块设备：`os/src/drivers/block/pci_blk.rs#new`（VirtIO-Blk over PCI）、`os/src/drivers/block/mmc/mod.rs#card_init`；网络设备：`os/src/devices/manager.rs#init_net` 初始化 `VirtIoNetDevImpl` 与 `LoopbackDevice`，`os/src/drivers/net/virtio_net.rs#receive`、`recycle_rx_buffer` 处理接收缓冲区；`os/src/devices/mod.rs#init` 统一完成设备树解析、MMIO 映射、中断使能。

### SYSCALL（系统调用）
底层证据强。入口为 `os/src/syscall/mod.rs#syscall`，规模近 1.8 万行、调用 186 处，使用 `SyscallId::from_repr` 分发。覆盖进程（`process.rs`）、文件（`fs.rs`）、内存（`mm.rs`）、IO 多路复用（`io.rs` 中 `epoll/pselect/ppoll`）、网络（`net.rs`）、信号（`signal.rs`）、futex（`futex.rs`）、IPC 信号量（`ipc/sem.rs`）、调度（`sche.rs`）、杂项（`misc.rs`）。用户指针访问统一走 `UserPtrRaw` / `UserSliceRaw` 校验。

### TRAP/IRQ（中断异常）
底层证据中等。HAL 层：`hal/src/component/irq/loongarch64/eiointc.rs#init`、`claim_irq`，`hal/src/component/irq/loongarch64/mod.rs#enable_irq/disable_irq/from_dt`，`hal/src/component/trap/loongarch64/mod.rs#save/restore`；RISC-V64 侧有 `hal/src/component/irq/riscv64.rs#enable_irq`。内核主体：`os/src/trap/mod.rs#user_trap_handler`、`kernel_trap_handler` 处理用户态/内核态陷阱，调用 `VirtAddr::from`、`TIMER_MANAGER.check`、`DEVICE_MANAGER.lock`、`IRQ_COUNTER.add_timer_irq_cnt`。纯 x86_64/ARM64 架构的 trap 代码底层证据未体现。

### NET（网络协议栈）
底层证据较强。基于 smoltcp：`os/src/net/mod.rs#init_network`、`modify_packet`，`os/src/net/tcp.rs#recv/send/shutdown`，`os/src/net/udp.rs#send_to`，`os/src/net/raw.rs#send`，`os/src/net/socketpair.rs#poll`。`SOCKET_SET.with_socket_mut::<tcp::Socket>`、`LISTEN_TABLE.handle_coming_packet` 等调用表明使用 smoltcp 的 socket set。另有 `os/src/net/crypto.rs#encode/check_alg` 实现类似 `AF_ALG` 的加密套接字。网络系统调用包括 `os/src/syscall/net.rs#sys_sendmsg`、`sys_accept`、`set`。

## 第六部分：边界与未体现内容

- **未体现或证据不足的模块**：
  - 安全/隔离机制：无 SELinux/LSM、cgroup、namespace、seccomp、eBPF 等底层证据
  - 虚拟化/Hypervisor：无 KVM、VMX、SVM、VM exit 处理等证据
  - 图形/GPU、音频、USB、NVMe、SATA 等驱动底层证据未体现
  - 实时调度形式化保证（EDF、优先级继承完整实现）证据不足
  - 内核调试器（kdb/kgdb）、崩溃转储、性能剖析器底层证据未体现
  - NUMA、ACPI、SMP 启动早期汇编/固件交互细节在提供的证据中未完整覆盖
  - 文件系统日志恢复、块设备缓存写回策略等底层细节证据不足

- **方向性说明**：
  - “通用宏内核/单体内核”“兼容 Linux 系统调用”“基于 smoltcp 的网络栈”等判断仅来自目录结构、函数名与调用链，属于**可能方向**，不是最终作品定位或原创性结论。
  - `os/src/fs/ext4/`、`os/src/net/` 中大量出现 `Ext4File`、`SOCKET_SET.with_socket_mut::<smoltcp::...>` 等，说明这些模块依赖或封装了第三方库/协议栈，**不应视为完全自实现的内核主体**。
  - 本文档仅用于技术理解，不做抄袭、违规或身份归属判断。
