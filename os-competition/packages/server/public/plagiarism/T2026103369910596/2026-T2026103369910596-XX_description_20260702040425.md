## 第四部分：分模块架构说明

### MM（内存管理）

底层证据充分。包含：

- **堆分配**：`os/src/mm/heap_allocator.rs#dealloc` 使用按 2 的幂对齐的块管理
- **物理帧分配**：`os/src/mm/frame_allocator.rs#dealloc_order` 基于 order 的伙伴式分配/回收
- **进程地址空间**：`os/src/mm/memory_set.rs` 提供 `from_elf`、`from_existed_user`、`mmap_anonymous`、`mmap_file`、`munmap_deferred`
- **缺页处理**：`handle_private_cow_fault`（私有写时复制）、`handle_shared_write_fault`（共享写回）
- **DMA 管理**：`os/src/drivers/virtio/mod.rs#dma_dealloc` 通过 `frame_dealloc_range` 回收 virtio 队列帧
- **延迟回收**：`DeferredUserReclaim` 在进程退出/卸载时批量释放用户资源

### PROC（进程管理/任务调度）

传统进程管理证据强，调度器证据中等。

- **PCB 与生命周期**：`os/src/task/process.rs#clone_process`、`#new`、`#spawn` 初始化 fd_table、memory_set、signal、itimer、credentials
- **退出与回收**：`os/src/task/mod.rs#exit_current_and_run_next_inner`、`reap_zombie_child_from_parent`
- **等待与同步**：`os/src/syscall/thread.rs#sys_waittid`、`os/src/sync/futex.rs#futex_wait_addr`、`os/src/task/wait_queue.rs` 系列函数
- **线程同步原语**：`sys_condvar_create/wait`、`sys_mutex_create/lock/unlock`、`sys_semaphore_create/down`
- **调度**：`os/src/sched/processor.rs#run_tasks`/`#schedule`、`os/src/sched/runqueue.rs#boost_process_cfs_tasks`，出现 `min_vruntime_ns`、`cfs_tasks` 等 CFS 线索
- **异步运行时**：仅有基于等待队列和定时器的阻塞/唤醒机制，**完整 Future/executor/waker 异步运行时底层证据未体现**

### FS（文件系统）

文件系统是本作品代码量最大的模块。

- **VFS 层**：`os/src/fs/inode.rs#lookup_inode_follow_with_path`、`#mount_is_readonly`、`#rename_at`
- **虚拟文件系统**：`procfs.rs`（`/proc`）、`devfs.rs`、`tmpfs.rs`、`pipe.rs`、`tty.rs`
- **块设备 I/O**：`os/src/drivers/block/virtio_blk.rs#write_blocks_many`、`#pump_completions`
- **第三方文件系统实现**：`fs/src/ext4_rs`（extents、balloc、inode、file）、`fs-fuse/src/pack/fat32.rs` / `ext4.rs` 为镜像打包工具
- **挂载表**：`MOUNT_TABLE` 在 `inode.rs` 中出现

### DEV（设备驱动）

设备驱动证据集中，种类有限。

- **块设备**：`os/src/drivers/block/virtio_blk.rs`（批量读写、完成队列轮询、等待队列）
- **网络设备**：`os/src/drivers/net/virtio_net.rs#try_recv`/`#try_send`/`#wait_tx_token`

### TRAP/IRQ（中断异常）

中断异常处理证据中等。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块/特性

- **完整异步运行时**：仅有等待队列 + 定时器阻塞，未发现 `Future`/`Waker`/`Executor` 等 async runtime 核心结构
- **GPU / 显示 / 声卡 / USB / SATA / NVMe 驱动**：仅见 virtio-blk、virtio-net、随机数设备
- **虚拟化/ hypervisor**：无 KVM/VMX/SVM、无 VM 切换相关证据
- **安全模块**：无 SELinux、AppArmor、Capabilities 完整实现、无 eBPF 证据
- **高级存储**：无 RAID、LVM、DM、swap、内存压缩、磁盘缓存层证据
- **NUMA / 高级 SMP**：仅有 `hartid`、CPU affinity、`loaded_user_harts`，无 NUMA 节点管理
- **内核模块/动态加载**：无 `insmod`/`kmod`/`module` 相关 ELF 加载证据
- **高级调试/追踪**：无 kdb、kgdb、ftrace 等证据
- **实时调度**：无 SCHED_FIFO/SCHED_RR 等实时调度器证据
- **启动加载器细节**：`bootloader` 目录仅 7 个文件，启动链底层证据有限

### 需要单独说明的第三方/工具代码

- `fs/src/ext4_rs/` 中的 ext4 实现、`fs-fuse/src/pack/fat32.rs` / `ext4.rs` 镜像打包工具属于**文件系统第三方库/辅助工具**，不应直接等同于内核原创文件系统主体
- `user/` 目录为用户态测试程序与 libc 封装，不是内核主体
