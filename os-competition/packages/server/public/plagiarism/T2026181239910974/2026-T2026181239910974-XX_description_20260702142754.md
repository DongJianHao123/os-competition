## 第四部分：分模块架构说明

### MM（内存管理）

- **证据强**。包含内核堆分配器（`os/src/mm/heap_allocator.rs#handle_alloc_error` 涉及 bucket 统计与任务清单打印）、用户地址空间 `MemorySet`（`os/src/mm/memory_set.rs#alloc_bootstrap_tls`）、页错误处理（`kernel_trap_handler` 调用 `KERNEL_SPACE.lock().handle_page_fault`）、用户态拷贝原语（`copy_to_user`、`copy_from_user` 遍布 syscall）、iovec 拷贝等。
- 支持 ELF 加载时的 TLS bootstrap，且页错误路径与任务调度、信号检查交织。

### PROC（进程管理/任务调度）

- **证据强**。具备完整传统进程管理：
  - `kernel_clone` 实现 fork/clone/vfork；
  - `sys_execve` / `sys_execveat` 实现 ELF 执行，含 interpreter 加载、busybox 兼容、exec ELF 缓存；
  - `sys_wait4` / `sys_waitid` 支持 zombie/stopped/continued 子进程等待；
  - `sys_exit` / `sys_exit_group` 与 `kernel_exit_inner` 处理退出、资源回收、孤儿进程托孤给 `INITPROC`；
  - 信号：`sys_rt_sigprocmask`、`sys_rt_sigtimedwait`、`sys_pidfd_send_signal`；
  - futex：`wait_on_futex`、`requeue_waiters_for_futex`、`wake_waiters_for_futex_bitset`；
  - 命名空间/资源：`time_ns_for_children`、`userns_id_map`、`setgroups`、`cgroup_procs`。
- 调度器表现为就绪/阻塞双队列模型，`switch_to` 完成上下文切换，`blocking_and_run_next` 处理阻塞。是否为“完整异步运行时”**底层证据未体现**，更偏向传统任务调度器。

### FS（文件系统）

- **证据极强**。是本项目最厚重的模块：
  - **VFS**：`dentry`、`inode`、`nameidata`、`mount`、`fdtable`、`file` 抽象；
  - **ext4**：`os/src/ext4/*` 实现超级块、块组、inode、extent、目录项、读写、sync；
  - **procfs**：`/proc/pid/stat`、`maps`、`fdinfo`、`namespace`、`autogroup`、`setgroups` 等；
  - **devfs/stdio**：`/dev/null`、`/dev/tty`、`/dev/rtc`、`/dev/random`、`/dev/loop-control` 等伪设备；
  - **cgroupfs**：`cgroup_procs_content`、`cgroup_add_proc`、`cgroup_kill`；
  - **fanotify**：`fanotify_notify_path_with_handle`、`fanotify_read_events`；
  - **pipe/FIFO**：`Pipe::new_fifo`；
  - **mount 系统调用**：`do_mount`、`do_ext4_mount`、`remove_mount_at`。
- `os/src/ext4` 虽位于内核源码树内，但无法仅凭路径判断其为完全原创实现还是移植/适配的第三方 ext4 实现，需在评审时单独核查。

### DEV（设备驱动）

- **证据弱且范围窄**。仅见：
  - VirtIO-Block MMIO 驱动（`os/src/drivers/block/virtio_blk.rs`）；
  - 块缓存管理（`os/src/drivers/block/block_cache.rs`）；
  - 块设备主设备号注册表（`os/src/drivers/block/registry.rs`）；
  - SBI UART 控制台（`os/src/sbi.rs`）。
- 无 PCI、USB、GPU、DMA、NIC、字符设备框架等其他设备驱动证据。

### SYSCALL（系统调用）

- **证据强**。`os/src/syscall/mod.rs#syscall` 为统一入口，调用数达 220，覆盖：
  - `task`：进程/线程/信号/futex/命名空间/keyctl；
  - `fs`：文件/目录/mount/sync/fallocate/fcntl；
  - `socket`：socket/bind/connect/sendto/sendmsg/getsockopt；
  - `miscfd`：fanotify、pidfd、misc fd；
  - `module`：内核模块加载 `finit_module`；
  - `bpf`：`run_attached_prog` 提供 eBPF-like 程序执行表层。

### TRAP/IRQ（中断异常）

- **证据存在但函数总量少（32）**。关键路径：
  - `os/src/trap/mod.rs#trap_handler`：用户态陷入入口，处理定时器、信号、用户内存访问激活；
  - `os/src/trap/mod.rs#kernel_trap_handler`：内核态陷入，处理页错误；
  - `os/src/arch/riscv64.rs#init_trap_entry` 与 `os/src/arch/loongarch64.rs#clear_timer_interrupt` 表明同时存在 RISC-V64 与 LoongArch64 架构支持。
- 缺少中断控制器（PLIC/APLIC/IOAPIC 等）大量代码证据。

### NET（网络协议栈）

- **底层证据未体现完整网络协议栈**。仅有：
  - `os/src/syscall/socket.rs` 中的 socket 系统调用表层（bind/connect/sendto/sendmsg/getsockopt/getpeername）；
  - `os/src/net/options.rs` 中的 socket option 默认值与 noop 匹配函数（共 5 个函数）。
- 无独立 TCP/IP、UDP、ARP、ICMP、路由表、skb、netdevice 等协议栈核心实现证据。socket 调用可能通过回环/UNIX 域或简单地址绑定存根完成。

7. **除 ext4/procfs/devfs/stdio/cgroup 外的其他文件系统**：如 FAT、NTFS、tmpfs 独立实现证据未体现。

### 需要单独说明的“可能方向”

- `os/src/ext4/*` 虽然位于内核源码树，但**无法仅凭路径和函数名判定其为完全原创实现还是移植/适配的第三方 ext4 库**，评审时应结合代码注释、提交历史、LICENSE 进一步核查。
- “任务调度/异步运行时”模块函数数量大，但底层证据更偏向**传统阻塞式任务调度 + futex/child 等待队列**，是否为现代 async/await 运行时**无法确认**。
- 架构支持线索同时出现 `riscv64.rs` 与 `loongarch64.rs`，但代码主体规模、启动流程、板级支持包（BSP）证据不足，不能断言完整支持两种架构。
