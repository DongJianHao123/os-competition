## 第四部分：分模块架构说明

### MM（内存管理）

- **证据锚点**：
  - `kernel/src/task/taskf.rs#execve`：调用 `AddrSpace::build_user`、`load_elf`、`map_heap`、`map_stack`。
  - `kernel/src/vm/elf.rs#init_stack`：构造用户栈，写入 `AuxHeader`、参数、环境变量。
  - `kernel/src/vm/vm_area.rs`（40 函数）、`lib/mm/src/address.rs`（42 函数）。
- **可说明的内容**：实现了用户虚拟地址空间构建、ELF 加载、栈/堆映射、用户态指针读写封装（`UserReadPtr` / `UserWritePtr`）。
- **未直接体现**：页错误处理细节、COW（写时复制）、swap、demand-paging 策略等底层证据未体现。

### PROC（进程管理/任务调度）

- **证据锚点**：
  - `kernel/src/syscall/process.rs#sys_clone3` / `__sys_clone`：`CloneFlags`、`PF_TABLE.new_pidfd`、线程/进程创建。
  - `kernel/src/syscall/process.rs#sys_execve` → `kernel/src/task/taskf.rs#execve`：ELF 替换地址空间。
  - `kernel/src/syscall/process.rs#sys_wait4` / `sys_waitid`：`PROCESS_GROUP_MANAGER`、`wait_queue`。
  - `kernel/src/task/taskf.rs#exit`：`TASK_MANAGER.remove_task`、信号/futex 清理、唤醒父进程。
  - `kernel/src/task/futex.rs#add_waiter` / `requeue_waiters`：futex 同步。
  - `kernel/src/task/signal/pidfd.rs#new_pidfd`：pidfd 机制。
  - `lib/osfs/src/proc/mod.rs#init_procfs`：`/proc` 伪文件系统。
- **可说明的内容**：传统进程/线程生命周期（fork/clone、exec、wait、exit）、进程组、线程组、信号、futex、pidfd、`/proc` 信息导出。
- **调度说明**：存在 `TIMER_MANAGER`、异步 Future（`IntrBySignalFuture`、`TimeoutFuture`、`EpollFuture`、`PSelectFuture`），但**显式 CPU 调度算法（如 CFS/RR）底层证据未体现**；“任务调度/异步运行时”统计中大量函数实际属于异步 I/O 与等待机制。

### FS（文件系统）

- **证据锚点**：
  - `lib/vfs/src/dentry.rs#lookup` / `rename`：VFS 目录项查找与重命名。
  - `lib/vfs/src/path.rs#walk_recursive`：路径遍历与符号链接解析。
  - `lib/vfs/src/file.rs#read_dir`：通用目录读取。
  - `lib/osfs/src/lib.rs#init`：`FS_MANAGER`、`BLOCK_DEVICE`、根文件系统挂载。
  - `kernel/src/syscall/fs.rs#sys_mount` / `fsmount.rs#sys_fsmount` / `sys_open_tree`：mount 系列系统调用。
  - `lib/ext4/src/dentry.rs#base_lookup` / `base_rename` / `fs.rs#base_mount`：ext4 实现。
  - `lib/fat32/src/dentry.rs#base_lookup` / `base_rename`：FAT32 实现。
  - `lib/vfs/src/fanotify/mod.rs#publish`、`kernel/src/syscall/fanotify.rs#sys_fanotify_mark`：文件事件通知。
- **可说明的内容**：分层 VFS、多文件系统（ext4、FAT32、proc、dev、loop）、mount 管理、文件句柄、fanotify/inotify、fd 表。
- **需注意**：`lib/ext4`、`lib/fat32` 等可能是移植或第三方实现，不应直接等同于原创内核主体。

### DEV（设备驱动）

- **证据锚点**：
  - `kernel/src/osdriver/probe.rs#probe_tree`：FDT 设备树探测，初始化块/字符/网络设备。
  - `kernel/src/osdriver/probe.rs#probe_sd_mmc_devices`：SD/MMC 设备匹配。
  - `kernel/src/osdriver/mod.rs#probe_serial_console`：串口控制台探测与 MMIO 映射。
  - `lib/driver/src/block/dw_mshc/mmc.rs#card_init` / `read`：SD/MMC 主机控制器命令序列与块读写。
  - `lib/driver/src/net/virtnet.rs#receive`：virtio-net 接收。
- **可说明的内容**：基于设备树的驱动探测、MMIO 映射、块设备（SD/MMC）、字符设备（串口）、网络设备（virtio-net）。
- **未直接体现**：USB、PCI、GPU、声卡、NVMe、AHCI 等驱动底层证据未体现。

### SYSCALL（系统调用）

- **证据锚点**：
  - `kernel/src/syscall/mod.rs#syscall`：205 条调用分发，覆盖进程、文件、信号、网络、bpf、keyring 等。
  - `kernel/src/syscall/process.rs#sys_execve` / `sys_clone3` / `sys_wait4`。
  - `kernel/src/syscall/fs.rs#sys_openat` / `sys_mount` / `sys_pread64` / `sys_writev`。
  - `kernel/src/syscall/signal.rs#sys_rt_sigtimedwait` / `sys_pidfd_open`。
  - `kernel/src/syscall/poll.rs#sys_epoll_pwait`。
  - `kernel/src/syscall/bpf.rs#sys_bpf`。
  - `kernel/src/syscall/key.rs#init_key` / `add_key`。
- **可说明的内容**：提供较完整的 Linux 兼容系统调用接口，支持进程、文件、mount、信号、epoll、BPF、keyring 等。
- **说明**：独立“系统调用”模块统计仅 36 个函数，是因为大量 syscall 实现被归类到进程/文件/信号等功能模块中。

### TRAP/IRQ（中断异常）

- **证据锚点**：
  - `kernel/src/trap/trap_handler/user_trap_handler/riscv64.rs#trap_handler`：`scause::read`、Exception/Interrupt 分发、`TIMER_MANAGER.check`。
  - `kernel/src/trap/trap_handler/user_trap_handler/loongarch64.rs#trap_handler`：`estat.cause`、`era::read`、`badv::read`、`TIMER_MANAGER.check`。
  - `kernel/src/trap/trap_handler/kernel_trap_handler/riscv64.rs#interrupt_handler`：`device_manager().handle_irq`。
  - `kernel/src/trap/trap_handler/kernel_trap_handler/loongarch64.rs#kernel_interrupt_handler`：定时器中断处理。
- **可说明的内容**：RISC-V64 与 LoongArch64 两套陷入/异常/中断入口，定时器中断检查，设备 IRQ 分发。
- **未直接体现**：具体中断控制器（PLIC/IOINTC/APLIC）配置、异常恢复细节、中断嵌套处理等底层证据未充分体现。

### NET（网络协议栈）

- **证据锚点**：
  - `lib/net/src/lib.rs#init_network`：IP、网关、回环、ETH0 初始化。
  - `lib/net/src/tcp/tcpsr.rs#send` / `recv`：`SOCKET_SET.with_socket_mut`、`SOCKET_SET.poll_interfaces`。
  - `lib/net/src/tcp/listentable.rs#accept` / `check_after_poll`：监听表与 SYN 队列。
  - `lib/net/src/udp.rs#send_to`：UDP 发送。
  - `lib/net/src/interface.rs#check_poll`：接口轮询与定时器管理。
  - `lib/driver/src/net/virtnet.rs#receive`：virtio-net 收包。
- **可说明的内容**：提供 TCP/UDP socket API、监听表、异步轮询、loopback/eth0 配置、virtio-net 驱动对接。
- **需注意**：代码中出现 `SOCKET_SET`、`tcp::Socket` 等命名，**可能基于或封装了第三方 TCP/IP 栈（如 smoltcp）**，不应直接视为完全原创的网络协议栈实现。
