## 第四部分：分模块架构说明

### MM（内存管理）

底层证据显示存在全局页/堆分配器、地址空间管理、按需缺页分配、共享内存及 DMA 管理。

- `arceos/modules/axruntime/src/lib.rs#init_allocator`：启动时调用 `axalloc::global_init` 与 `axalloc::global_add_memory` 初始化物理内存区域。
- `arceos/modules/axalloc/src/lib.rs#alloc`：buddy/SLAB 风格分配，支持 `balloc.alloc`、`add_memory`、`total_bytes`。
- `arceos/modules/axmm/src/backend/alloc.rs#handle_page_fault_alloc`：缺页时 `alloc_frame` 并调用 `pt.map` 建立映射。
- `arceos/modules/axmm/src/aspace.rs#try_clone`：地址空间克隆，涉及 `MemoryArea::new`、`PageIterWrapper`，用于 `fork/clone` 的内存复制/写时复制路径。
- `api/src/imp/mm/mmap.rs#sys_mmap` / `sys_mremap`：用户态 mmap/mremap 系统调用，操作 `VirtAddrRange`、查找空闲区域、设置页属性。
- `api/src/imp/mm/shm.rs#sys_shmat` / `sys_shmctl`：System V 共享内存管理，通过 `SHM_MANAGER` 维护。
- `arceos/modules/axdma/src/dma.rs#dealloc_coherent`：DMA 一致性内存释放。

### PROC（进程管理）

存在较完整的传统进程管理证据，包括 PCB/进程表、进程组、会话、fork/clone、execve、exit、wait、zombie、pidfd、信号等。

- `api/src/imp/task/clone.rs#sys_clone` / `api/src/imp/task/clone.rs#sys_fork`：进程/线程创建，复制 fd 表、当前目录、命名空间等。
- `api/src/imp/task/execve.rs#sys_execve`：ELF 程序加载，解析参数、环境变量，调用 `load_user_app_inner`。
- `api/src/imp/task/exit.rs#do_exit`：进程退出，清理 fd 表、发送信号、处理 robust futex、调用 `axtask::exit`。
- `api/src/imp/task/wait.rs#sys_waitpid` / `sys_waitid`：等待子进程，处理 `child.is_zombie`、`child.exit_code`、`child.free`。
- `api/src/imp/task/pidfd.rs#sys_pidfd_open` / `sys_pidfd_getfd` / `sys_pidfd_send_signal`：pidfd 机制。
- `core/src/task.rs#add_thread_to_table`：维护 `PROCESS_TABLE`、`THREAD_TABLE`、`PROCESS_GROUP_TABLE`、`SESSION_TABLE`。
- `crates/axprocess/src/lib.rs`：进程抽象核心，密度 38 个函数。

### 任务调度/异步运行时

以 `axtask` 为核心，支持多队列、上下文切换、等待队列、定时器唤醒、futex 同步。

- `arceos/modules/axtask/src/run_queue.rs#init`：初始化 idle/main 任务、`RUN_QUEUES[cpu_id]`、`CurrentTask::init_current`。
- `arceos/modules/axtask/src/run_queue.rs#switch_to`：上下文切换，保存/恢复寄存器上下文。
- `arceos/modules/axtask/src/run_queue.rs#exit_current`：当前任务退出，加入 `EXITED_TASKS`，通知 `WAIT_FOR_EXIT`。
- `arceos/modules/axtask/src/wait_queue.rs#wait_timeout` / `wait_timeout_until` / `notify_one`：基于等待队列的阻塞与超时唤醒。
- `api/src/imp/futex.rs#sys_futex`：用户态 futex 同步原语，维护 per-process `futex_table`。
- `api/src/imp/fs/poll.rs#sys_ppoll` / `sys_pselect6`、`api/src/imp/fs/eventpoll.rs#epoll_wait_inner`：I/O 多路复用，内部调用 `axtask::yield_now` 轮询/等待。

### FS（文件系统）

文件系统是证据最强的模块，具备 VFS 层、fd 表、多种后端（ramfs、fatfs、lwext4、procfs）及 mount 机制。

- `arceos/modules/axfs/src/root.rs#init_rootfs`：初始化根文件系统、`CURRENT_DIR`、`ROOT_DIR`。
- `arceos/modules/axfs/src/fops.rs#_open_at`：VFS 打开路径，调用 `root::lookup`、`root::create_file`。
- `api/src/imp/fs/fd_ops.rs#sys_openat`：系统调用层打开文件/目录，构造 `Directory`/`File` 并加入 fd 表。
- `api/src/imp/fs/io.rs#sys_readv` / `sys_writev` / `sys_preadv` / `sys_pwritev`：向量读写。
- `api/src/imp/fs/ctl.rs#sys_getdents64` / `write_entry`：目录项读取。
- `api/src/imp/fs/mount.rs#sys_mount` / `sys_umount2`：挂载/卸载，支持 `mount_fat_fs`。
- `api/src/imp/fs/eventpoll.rs#epoll_wait_inner`：epoll 事件等待。
- `arceos/modules/axfs/src/fs/lwext4_rust.rs`：lwext4 第三方库的 Rust 封装，提供 `read_dir`、`get_attr`、`create`、`read_at`、`write_at`、`truncate` 等。

### DEV（设备驱动）

驱动层证据集中在初始化、VirtIO-MMIO 探测、平台串口/控制台。

- `arceos/modules/axdriver/src/lib.rs#init_drivers`：统一驱动探测，遍历 block/net/display 设备。
- `arceos/modules/axdriver/src/virtio.rs#probe_mmio`：VirtIO-MMIO 设备探测。
- `arceos/modules/axhal/src/platform/x86_pc/uart16550.rs#init`：x86 串口初始化。
- `arceos/modules/axhal/src/platform/loongarch64_qemu_virt/console.rs#init_early`：龙架构早期控制台初始化。
- `arceos/modules/axhal/src/platform/x86_pc/mod.rs#rust_entry`：x86 启动入口，初始化 CPU、BSS、UART、时间。

### SYSCALL（系统调用）

存在统一的系统调用分发器与大量按类别封装的实现。

- `src/syscall.rs#handle_syscall`：中央分发器，规模 19502，调用 184 个目标，覆盖 fs/net/task/mm/sys/time/signal 等。
- `api/src/imp/fs/*`：文件类系统调用（openat、readv、writev、mount、getdents64、epoll 等）。
- `api/src/imp/net.rs#sys_socket` / `sys_accept4` / `sys_recvmsg` / `sys_sendmsg`：网络类系统调用。
- `api/src/imp/task/thread.rs#sys_arch_prctl`：x86 架构专用 prctl。
- `api/src/imp/sys.rs#sys_getrandom` / `sys_get_mempolicy`：杂项系统调用。

### TRAP/IRQ（中断异常）

多架构异常入口、系统调用、缺页、断点、IRQ 注册与分发均有证据。

- `arceos/modules/axhal/src/arch/x86_64/trap.rs#x86_trap_handler`：x86 异常入口，处理 `handle_page_fault`、`handle_syscall`。
- `arceos/modules/axhal/src/arch/riscv/trap.rs#riscv_trap_handler`：RISC-V 异常入口。
- `arceos/modules/axhal/src/arch/aarch64/trap.rs#handle_sync_exception`：ARM64 同步异常，解析 `ESR_EL1`。
- `arceos/modules/axhal/src/arch/loongarch64/trap.rs#loongarch64_trap_handler`：龙架构异常入口。
- `arceos/modules/axhal/src/irq.rs#register_handler_common` / `dispatch_irq_common`：IRQ 处理注册与分发。
- `arceos/modules/axhal/src/arch/x86_64/idt.rs#new`：IDT 初始化。

### NET（网络协议栈）

网络部分以 socket 系统调用适配和第三方 smoltcp 驱动为主，原创协议实现证据极少。

- `api/src/imp/net.rs#sys_socket` / `sys_accept4` / `sys_recvmsg` / `sys_sendmsg`：socket 相关系统调用。
- `api/src/socket.rs#read_from_user` / `write_to_user`：sockaddr 在用户态与内核态之间拷贝。
- `arceos/modules/axnet/src/smoltcp_impl/mod.rs#init`：初始化 `ETH0`、`InterfaceWrapper`、IP/网关。
- `arceos/modules/axnet/src/smoltcp_impl/tcp.rs#connect` / `recv` / `send`：TCP 操作封装。
- `arceos/modules/axnet/src/smoltcp_impl/udp.rs#bind`：UDP 绑定封装。
- `arceos/modules/axnet/src/smoltcp_impl/dns.rs#query`：DNS 查询封装。
