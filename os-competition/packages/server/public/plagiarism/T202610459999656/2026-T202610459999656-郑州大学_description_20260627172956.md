## 第四部分：分模块架构说明

### MM（内存管理）
底层证据较强。包含：
- 物理页框分配器：`os/src/mm/frame/allocator.rs#init_global_allocator` 初始化全局分配器并注册可用内存。
- 内核堆分配器：`os/src/mm/heap/allocator.rs`、`global_alloc.rs`、`slab.rs`、`slab_cache.rs`、`slot.rs`，按 16/32/64/128/256/512/1024/2048 字节分级 slab 分配，并支持大块 `vmalloc` 路径。
- 虚拟内存对象/区域：`os/src/mm/vmo.rs`（VMO，支持预读工作线程）、`os/src/mm/vmar.rs`（VMAR，处理页错误）。
- 页错误处理：`os/src/thread/exception.rs#kernel_trap_handler`、`os/src/mm/vmar.rs#handle_page_fault_with_injection`。

### PROC（进程管理/任务调度）
传统进程管理证据充分：
- PCB 语义：`os/src/process.rs` 中 `Process` 维护 pid、pgid、sid、parent/children、threads、exit_status、zombie 状态等。
- 进程创建：`os/src/syscall/proc.rs#sys_clone / do_clone3` → `os/src/syscall.rs#handle_clone` → `os/src/process.rs#publish_clone_child_thread`。
- 等待与回收：`os/src/syscall/proc.rs#do_wait4`、`os/src/process.rs#try_wait_status_under_lock` 处理 zombie/stopped/continued。
- 信号模型：`do_rt_sigtimedwait`、`do_kill_targets`、`do_thread_directed_signal`、`deliver_pending_signal`、`restore_signal_frame` 等覆盖实时信号、线程定向信号、信号帧、sigaltstack。
- 定时器与凭证：`service_process_timers`、`setresuid`、`setresgid`、`capget`/`capset`。
- 调度上下文切换：`os/src/thread/scheduler.rs#exit_current`、`os/src/thread/processor.rs#switch_to_thread` 直接操作 `CURRENT_THREAD_PTR` 并调用 `context_switch`。
- **异步运行时**：仅见 `Waiter`/`park_current` 等阻塞/等待原语，完整的 async runtime 调度器底层证据未直接体现。

### FS（文件系统）
文件系统是代码量最大的原创模块：
- VFS 层：`os/src/fs/inode_handle.rs`、`os/src/fs/utils/inode.rs` 提供统一 inode 读写接口。
- 具体文件系统：
  - `tmpfs`：`os/src/fs/tmpfs.rs#rename / #exchange`。
  - `ext4`：`os/src/fs/ext4.rs` 封装 `os/third_party/rsext4`，`page_cache.rs#commit_pages` 实现页缓存回写。
  - `procfs`：`os/src/fs/procfs.rs` 提供进程/线程状态、maps、smaps、fd、stat、status 等。
  - `sysfs`、`devpts`、`miscdevfs`：由 `os/src/fs/rootfs.rs#prepare_oscomp_testsuits_root_layout` 挂载。
- 管道：`os/src/fs/pipe.rs` 支持字节/包模式、数据移动、读写唤醒。
- 锁与复制：`os/src/fs/flock.rs`、`os/src/syscall/fs.rs` 中 `sendfile`/`copy_engine` 相关测试。

### DEV（设备驱动）
- **VirtIO-blk**：`os/src/drivers/virtio/blk/core.rs` 管理 bio 提交、flush、inflight token 完成；`disk.rs` 处理 VirtQueue、feature 协商、`submit_read`/`submit_write`。
- 平台探测：`os/src/hal/imp/platform/fdt.rs#probe` 解析 FDT 获取 CPU、syscon 等信息。
- 其它设备（如 PCI、字符设备、GPU、音频）底层证据未体现。

### SYSCALL（系统调用）
- 分类入口：`os/src/syscall/fs.rs`（文件 820 函数）、`net.rs`（网络 623 函数）、`proc.rs`（进程/信号 503 函数）、`mm.rs`（内存）、`futex.rs`（futex）、`ipc.rs`（IPC）。
- 分发与信号返回：`os/src/syscall.rs#handle_clone`、`os/src/syscall.rs#deliver_pending_signal`。

### TRAP/IRQ（中断异常）
- 架构相关入口：`os/src/hal/imp/arch/riscv64/trap.rs#trap_handler`、`os/src/hal/imp/arch/loongarch64/trap.rs#trap_handler`。
- 外部中断：`os/src/hal/irq/external.rs#handle_external_interrupt` 通过 HANDLERS 表分发。
- 定时器：`os/src/hal/imp/arch/riscv64/time.rs#on_timer_interrupt`、`os/src/hal/imp/arch/loongarch64/time.rs#on_timer_interrupt`。
- IPI：`crate::smp::on_ipi`、`ack_ipi_on_current_hart`。

### NET（网络协议栈）
- 薄封装层：`os/src/net/stack.rs` 提供 TCP/UDP IPv4 的 connect/send/recv/bind/listen 接口。
- 第三方依赖：通过 `smoltcp_now`、`smoltcp_duration_to_capped_us` 适配 `smoltcp`。
- 协议栈深度原创证据较弱，IPv6、路由、防火墙、网卡驱动等底层证据未体现。
