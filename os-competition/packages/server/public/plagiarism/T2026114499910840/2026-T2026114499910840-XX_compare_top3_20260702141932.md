## 6. 保守结论

双方客观 AST 相似度仅 6.5%，双向覆盖率均处于低位（目标 20.26%，参考 8.73%）。网络协议栈模块的高覆盖率（目标 57.53%）源于双方均集成了 lwIP 开源协议栈及 PolarSSL 加密库，属于共同第三方上游代码的必然结果，且 API 差异（`ip6_addr_eq` vs `ip6_addr_cmp`）反映独立版本选择，不构成对参考项目的抄袭。剔除 lwIP/PolarSSL 等第三方组件后，双方在原创内核代码（进程管理、内存管理、系统调用、文件系统、中断处理）上的实际重合度远低于表面数值。核心子系统在抽象层次（内核态 vs 用户态）、内存模型（哈希表 vs 内核对象锁）、同步策略（`arch_disable_interrupt` vs `lock/unlock`）上存在根本性架构差异。控方将大量网络/加密函数跨板块归入进程管理，违反同源分析基本规范。综合判定：目标项目为独立开发，不存在对参考项目的系统性代码搬运。
