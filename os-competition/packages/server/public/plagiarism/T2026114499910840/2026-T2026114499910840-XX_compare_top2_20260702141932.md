#### 6. 保守结论

目标项目与参考项目的客观 AST 相似度仅为 7.29%，双向覆盖率均处于低位（目标 19.97%，参考 10.29%）。所有被标记为“同板块底层逻辑一致”或“同板块同名同结构”的证据，经溯源分析后均指向 lwIP 与 PolarSSL 等第三方开源组件——两个项目分别独立集成了这些上游库，由此产生的 AST 调用图谱一致是上游同源的正常现象，而非目标项目对参考项目的代码搬运。在内核核心子系统（进程管理、内存管理、中断处理、系统调用、文件系统）上，双方使用完全不同的 API 命名空间、硬件抽象层和同步原语：目标项目采用自研 `procfs_*`/`hashmap_*`/`uacpi_*`/`ptrace_*` 体系，参考项目基于 RT-Thread 的 `rt_*`/`lwp_*`/`dfs_*` 生态。模块覆盖矩阵显示“网络协议栈”仅目标项目存在、“文件系统/第三方库”仅参考项目存在，进一步印证两者是独立演化的代码基。综合判定：无证据支持系统性抄袭指控，双方为独立开发。
