## 第四部分：分模块架构说明

### MM（内存管理）

底层证据显示存在两类内存管理：

1. **内核堆分配器**：`kernel/src/mm/alloc.c#sys_alloc` 调用 `CALL_MMAP`、`CALL_MORECORE`、`add_segment`、`check_malloced_chunk` 等，属于典型的 `dlmalloc` 风格堆实现。
2. **页帧/物理内存操作**：`kernel/src/mm/mm_syscall.c#msync_writeback_file_range` 使用 `alloc_frames_bytes`、`free_frames_bytes`、`mm_pgdir`、`phys_to_virt`；`kernel/src/task/task_syscall.c#task_do_execve` 与 `sys_clone_internal` 也调用 `alloc_frames`/`alloc_frames_bytes`。

**未体现**：完整的虚拟内存子系统细节（如缺页异常处理、按需分页、swap、page cache 写回策略）在现有证据中未充分展开。

### PROC（进程管理/任务调度）

证据显示存在**传统进程管理**，而非仅轻量级任务：

- 进程创建：`kernel/src/task/task_syscall.c#sys_clone_internal`（`arch_context_copy`、`add_sched_entity`）。
- 程序执行：`kernel/src/task/task_syscall.c#task_do_execve`（`arch_check_elf`、`vfs_openat`）。
- 进程退出：`kernel/src/task/task.c#task_exit_inner`（`remove_sched_entity`、`futex_on_exit_task`、`pidfd_on_task_exit`）。
- 等待/回收：`kernel/src/task/task_syscall.c#sys_waitid`、`sys_wait4`。
- 同步：`kernel/src/task/futex.c#sys_futex_wait`、`futex_on_exit_task`。
- 信号：`kernel/src/task/signal.c#sys_sigprocmask`。
- ptrace/pidfd：`kernel/src/task/task_syscall.c` 与 `kernel/src/fs/syscall/pidfd.c` 中相关调用。

调度方面：`schedule`、`task_block`、`remove_sched_entity`、`add_sched_entity` 等表明存在基于调度实体的抢占/协作调度。

**关于“异步运行时”**：模块画像中“任务调度/异步运行时”函数数量最多，但底层证据主要体现为内核线程阻塞/唤醒、lwIP `tcpip_thread` 消息循环、ACPI GPE 异步处理；独立的 `async/await` 运行时（如 `Future`、`Waker`、`Executor`）底层证据未体现。

### FS（文件系统）

文件系统是证据最强的模块之一：

- **VFS 层**：`kernel/src/fs/vfs/vfs_mount.c#vfs_mount_alloc`、`vfs_reconfigure_mount`；`kernel/src/fs/vfs/vfs_ops.c#vfs_unlinkat`；`kernel/src/fs/vfs/vfs_lookup.c#__vfs_filename_lookup`。
- **挂载命名空间**：`kernel/src/fs/syscall/fsfd.c#sys_move_mount`、`sys_fsmount`；`kernel/src/fs/syscall/generic.c#sys_open_tree`、`sys_mount`。
- **通用文件 syscall**：`kernel/src/fs/syscall/generic.c#sys_open_by_handle_at`、`sys_read`、`sys_write`、`sys_pread64`、`sys_pwrite64`、`sys_readlinkat`。
- **具体文件系统**：`modules/fs/ext/ext.c#ext_rename`、`ext_inode_clear_block_extent_locked` 表明实现了 EXT 系列文件系统。
- **伪文件系统**：`kernel/src/fs/proc/proc.c#procfs_lookup`、`procfs_iterate_shared`；`kernel/src/fs/proc/proc_puserns.c`、`proc_pmountinfo.c` 等提供 procfs；另有 `kernel/src/fs/pipe.c`、`kernel/src/drivers/pty.c` 等管道/PTY 支持。

### DEV（设备驱动）

驱动覆盖面较广：

- **图形/DRM**：`kernel/src/drivers/drm/drm_ioctl.c#drm_ioctl`、`drm_syncobj_fd_to_handle`；`kernel/src/drivers/drm/plainfb.c#plainfb_cursor_draw`；`modules/drivers/virtio/gpu.c#virtio_gpu_driver_ioctl`。
- **网络设备**：`modules/drivers/virtio/net.c#virtio_net_receive`。
- **块设备**：`modules/drivers/virtio/blk.c#virtio_blk_wait_slot`；`modules/drivers/nvme/nvme.c#nvme_process_queue_completions`。
- **USB 主机控制器**：`modules/drivers/usb/xhci-hcd.c#xhci_process_event_ring`。
- **输入设备**：`kernel/src/arch/x86_64/drivers/chars/ps2.c#ps2_keyboard_init`、`ps2_mouse_init`；`modules/drivers/hid/hid.c#hid_process_field`。
- **声音**：`modules/drivers/sound/sound.c#sound_hw_refine_fill_defaults`。
- **固件/平台**：`kernel/src/drivers/smbios.c#smbios_init`。

### SYSCALL（系统调用）

系统调用分散在多个子系统目录中，未看到统一 syscall 分发器证据，但各具体 syscall 实现清晰：

- 文件类：`kernel/src/fs/syscall/generic.c`、`fsfd.c`、`pidfd.c`、`poll.c`、`epoll.c`、`timerfd.c`。
- 进程类：`kernel/src/task/task_syscall.c`。
- 内存类：`kernel/src/mm/mm_syscall.c`。
- 网络类：`kernel/src/net/net_syscall.c`、`modules/net/netserver/lwip_socket.c#lwip_socket_ioctl`。

### TRAP/IRQ（中断异常）

证据规模较小但存在关键锚点：

- x86_64 早期中断初始化：`kernel/src/arch/x86_64/x86_64.c#arch_early_init` 配置 APIC、HPET、通用中断表。
- 设备 IRQ 注册：`kernel/src/arch/x86_64/drivers/chars/ps2.c#ps2_keyboard_init` 调用 `irq_regist_irq`。
- 软中断/定时器：`kernel/src/fs/syscall/timerfd.c#timerfd_softirq`。
- ACPI 事件异步处理：`kernel/src/acpi/event.c#async_run_gpe_handler`。

**未体现**：详细的中断入口/异常向量表、页故障处理程序、系统调用入口汇编等底层证据未在提供的 Call Map 中展开。

### NET（网络协议栈）

网络协议栈主体为 **lwIP**：

- lwIP 核心：`modules/net/netserver/lwip/core/init.c#lwip_init`、`tcpip.c#tcpip_thread_handle_msg`、`ipv4/ip4.c`、`ipv6/ip6.c`、`ipv6/nd6.c`、`core/tcp_in.c#tcp_receive`、`core/udp.c#udp_input`。
- Socket API：`modules/net/netserver/lwip/api/sockets.c#lwip_setsockopt_impl`、`lwip_getsockopt_impl`、`lwip_bind`、`lwip_connect`。
- 与内核任务集成：`modules/net/netserver/sys_arch.c#sys_thread_new` 调用 `task_create`；`sys_mbox_new`、`sys_arch_sem_wait` 使用内核同步原语。
- 内核原生网络：`kernel/src/net/netlink.c`、`kernel/src/net/socket.c` 提供 netlink/unix socket 支持。

**注意**：lwIP 属于第三方网络协议栈实现，不应直接等同于原创网络协议栈。

### 第三方库 / 公共依赖

- **lwIP**：`modules/net/netserver/lwip/*`，网络协议栈主体。
- **tinycrypt**：`kernel/src/libs/tinycrypt/ecc_dsa.c#uECC_verify`、`uECC_sign_with_k`，提供 ECC/DSA。
- **fdt（libfdt）**：`kernel/src/libs/fdt/*`，设备树解析。
- **flanterm**：`kernel/src/libs/flanterm/flanterm.c`，终端模拟/帧缓冲输出。
- **uACPI**：`kernel/src/acpi/*` 中大量 `uacpi_*` 函数，属于 ACPI 固件抽象层，可能为第三方或重度参考实现。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块/特性

- **独立异步运行时**：未看到 `Future`、`Waker`、`Executor`、`async/await` 等独立运行时结构，现有“异步”证据多指 ACPI GPE 异步处理、lwIP tcpip 线程及内核线程阻塞/唤醒。
- **完整虚拟内存子系统**：除 `execve` 页表准备与 `msync` 页帧操作外，缺页异常处理、按需分页、swap、page cache 写回策略等底层证据未体现。
- **完整异常/中断入口**：仅看到 `arch_early_init` 与 `irq_regist_irq` 等初始化锚点，未看到 IDT 向量表、页故障/系统调用入口汇编等。
- **除 x86_64 外的架构支持**：所有 arch 路径均指向 `kernel/src/arch/x86_64`，无 ARM/RISC-V 等证据。
- **原创网络协议栈**：网络层主体为 lwIP，属于第三方实现。
- **完整安全/LSM/能力机制**：未看到 capabilities、LSM hook、seccomp 等证据。
- **完整音频、视频渲染管线**：仅有 DRM ioctl 分发与声音硬件参数填充，未看到完整 ALSA/DRM KMS 渲染管线。
- **除 EXT/proc/dev/pipe/pty 外的文件系统**：如 FAT、NTFS、Btrfs、ZFS 等无证据。
- **完整 USB 设备类驱动**：仅看到 xHCI HCD，未看到 mass storage、HID、CDC 等独立类驱动证据。

### 判断边界说明

- 本报告仅基于 AST Call Map、函数名、模块名、路径锚点与调用链描述技术事实，不做原创性、抄袭或违规判定。
- “可能的系统方向”仅为保守推断，不代表项目最终定位或身份。
- 第三方库（lwIP、tinycrypt、fdt、flanterm 及可能的 uACPI）在代码量中占显著比例，评估原创性时应将其与内核主体代码区分。
