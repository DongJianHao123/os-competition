## 第四部分：分模块架构说明

### MM（内存管理）

**已实现**：
- RISC-V Sv39 页表管理：`walk`、`mappages`、`uvmunmap`、`uvmcopy`、`uvmalloc`、`uvmdealloc`（`kernel/vm.c`）。
- 内核地址空间构建：`kvmmake`、`kvmmap`、`proc_mapstacks`（`kernel/vm.c`）。
- 进程页表创建与销毁：`proc_pagetable`、`proc_freepagetable`（`kernel/proc.c`）。
- 用户态缺页处理：`vmfault`（`kernel/vm.c`）由 `usertrap`（`kernel/trap.c`）调用。
- 跨态拷贝：`copyin`、`copyout` 使用 `walkaddr` 与 `vmfault`。
- 物理页分配：`kalloc`、`kinit`（`kernel/kalloc.c`），证据显示为简单空闲链表。

**未体现**：SLAB/buddy 分配器、swap、页替换算法、大页/THP、NUMA、内存 cgroup 等底层证据未体现。

### PROC（进程管理/任务调度）

**已实现**：
- 传统 Unix PCB 生命周期：`allocproc`、`freeproc`、`kfork`、`kexit`、`kwait`、`reparent`（`kernel/proc.c`）。
- 调度器：`scheduler` 循环 + `swtch` + `sched`/`yield`/`sleep`/`wakeup`。
- 进程状态同步：自旋锁 `ptable.lock`、关中断临界区。
- 系统调用：`sys_fork`、`sys_exit`、`sys_wait`、`sys_kill`、`sys_pause`（`kernel/sysproc.c`）。

**未体现**：独立异步运行时、协程、task/channel、waker、ring buffer、优先级调度、CFS、多核 SMP 调度细节等底层证据未体现。`任务调度/异步运行时` 的 24 个函数实际为传统调度原语，不应视为完整 async runtime。

### FS（文件系统）

**已实现**：
- xv6 日志式文件系统：`begin_op` / `end_op` / `install_trans`（`kernel/log.c`）。
- inode 与目录：`ialloc`、`iget`、`iput`、`ilock`、`iunlock`、`ireclaim`、`dirlookup`、`dirlink`、`namex`（`kernel/fs.c`）。
- 块映射与缓存：`bmap`、`balloc`、`bread`、`bwrite`、`brelse`（`kernel/fs.c`、`kernel/bio.c`）。
- 文件读写：`readi`、`writei`（`kernel/fs.c`），`filewrite`、`fileread`（`kernel/file.c`）。
- 管道：`pipealloc`、`piperead`、`pipewrite`（`kernel/pipe.c`）。
- 系统调用：`sys_open`、`sys_link`、`sys_unlink`、`sys_chdir`、`sys_exec`、`sys_pipe` 等（`kernel/sysfile.c`）。
- 主机镜像工具：`mkfs/mkfs.c` 用于生成初始文件系统镜像。

**第三方/附加层**：
- `kernel/ext4.c` / `kernel/ext4_disk.c` 提供 ext4 只读访问能力，函数如 `ext4_read_block`、`ext4_disk_init`、`ext4_read_file`、`dir_lookup`。该部分应单独标注为第三方库或移植代码，不宜直接等同于原创内核文件系统主体。

### DEV（设备驱动）

**已实现**：
- 块设备：VirtIO disk 驱动（`kernel/virtio_disk.c#virtio_disk_init`、`virtio_disk_rw`、`virtio_disk_intr`）。
- 字符设备：UART 串口（`kernel/uart.c`）与控制台（`kernel/console.c`）。
- 中断控制器：PLIC（`kernel/plic.c`）用于外部中断路由。

**未体现**：网络适配器、GPU、USB、NVMe、SATA、键盘、鼠标、显示设备等底层证据未体现。

### SYSCALL（系统调用）

**已实现**：
- 原生系统调用分发：`kernel/syscall.c#syscall` → `syscalls[num]`。
- Linux ABI 兼容层：`kernel/linux_syscalls.c#linux_dispatch` 及 `l_exit`、`l_exit_group`、`l_getpid`、`l_getppid`、`l_chdir`、`l_unlinkat`、`l_mmap`、`l_nanosleep`、`l_uname`、`l_sched_yield` 等包装函数。
- 进程/文件/内存类系统调用分别落地于 `kernel/sysproc.c` 和 `kernel/sysfile.c`。

### TRAP/IRQ（中断异常）

**已实现**：
- RISC-V 陷阱初始化：`trapinit`、`trapinithart`（`kernel/trap.c`）。
- 用户态/内核态陷阱处理：`usertrap`、`kerneltrap`。
- 设备中断分发：`devintr` → `clockintr` / `uartintr` / `virtio_disk_intr` / `plic_claim` / `plic_complete`。
- 定时器：`clockintr` 设置 `stimecmp` 并触发调度。

### NET（网络协议栈）

**底层证据未体现**。代码路径、目录、函数名中均未发现 TCP/IP、socket、NIC 驱动、网络缓冲区等相关证据。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块/特性

1. **网络协议栈（NET）**：未发现 socket、TCP/IP、NIC 驱动、skb/网络缓冲区等任何证据。
2. **独立异步运行时**：未发现 task、channel、waker、ring_fifo、async/await 运行时；现有 `任务调度/异步运行时` 统计应理解为传统调度原语。
3. **高级内存管理**：无 SLAB/buddy、swap、页替换、大页、NUMA、cgroup 内存子系统证据。
4. **多核 SMP 高级特性**：虽有 `cpuid`、`mycpu`、`PLIC`，但不足以支撑完整 SMP 调度、CPU 亲和、RCU、per-CPU 变量等结论。
5. **除 UART/VirtIO/PLIC 外的设备驱动**：无 GPU、USB、NVMe、SATA、键盘、鼠标、显示、声卡等驱动证据。
6. **安全/隔离增强机制**：无 namespace、cgroup、seccomp、 capabilities、ASLR、KASLR、栈保护等证据。
7. **文件系统写支持的 ext4**：现有 ext4 代码仅体现 `read_block`、`read_file` 等只读路径，未体现 ext4 写入、日志、扩容等。

### 需要单独说明的边界

- **`mkfs/` 目录**：`mkfs/mkfs.c` 是主机端工具，用于生成文件系统镜像，**不属于内核运行时**。
- **`user/` 目录**：`sh.c`、`init.c`、`grind.c`、`stressfs.c` 等为用户态测试程序与 shell，**不属于内核主体**。
- **`kernel/ext4.c` / `kernel/ext4_disk.c`**：函数命名、调用链与典型 lwext4 等公共实现高度相似，应视为第三方库或移植层，**不宜直接等同于原创内核文件系统**。
- **Linux 系统调用兼容层**：`linux_syscalls.c` 提供 ABI 包装，但底层仍通过原生 `sys_*` 实现，不能等同于完整 Linux 兼容内核。

### 判断性质声明

- 本报告中的“基于 xv6-riscv 骨架”“Linux ABI 兼容层”“ext4 第三方库”等表述均为**基于代码结构、函数名与调用链的保守技术推断**，不是对项目身份、原创性或违规行为的最终结论。
- 所有“已实现”描述均以上述 AST Call Map 与函数证据为限；“未体现”表示当前提供的底层证据不足以支撑该模块存在。
