## 第四部分：分模块架构说明

### MM（内存管理）
**底层证据强**。核心实现位于 `os/src/mm/`：
- `os/src/mm/memory_set.rs#from_elf`：按 ELF program header 建立用户地址空间，使用 `MapArea::new`、`VirtAddr`、`VirtPageNum`。
- `os/src/mm/page_table.rs` 与 `os/src/mm/address.rs`：页表、虚拟/物理地址转换。
- `os/src/task/id.rs#dealloc_user_res`：进程/线程退出时回收用户资源，调用 `memory_set.remove_area_with_start_vpn` 并 `flush_current_user_tlb`。
- `ProcessControlBlock::fork` 使用 `MemorySet::from_existed_user` 复制地址空间；`exec` 使用 `MemorySet::from_elf` 重新加载。

未明确体现：写时复制（COW）、swap、共享内存、大页、NUMA。

### PROC（进程管理/任务调度）
**底层证据强**，但属于**传统进程/线程模型**，而非 async runtime：
- `os/src/task/process.rs#fork` / `#exec` / `#new`：维护 `ProcessControlBlock`、子进程表、任务表、文件描述符表、信号掩码。
- `os/src/syscall/process.rs#sys_fork` / `#sys_clone` / `#sys_exec` / `#sys_wait4` / `#wait4_core`：完整的 fork/exec/wait/zombie 路径。
- `os/src/syscall/thread.rs#sys_thread_create` / `#sys_waittid`：线程创建与等待。
- `os/src/task/processor.rs#run_tasks` / `#run_tasks_until_pid_exit_timeout`：任务调度循环，使用 `__switch` 进行上下文切换。
- 同步原语：`os/src/syscall/sync.rs#sys_futex`、`#sys_mutex_create`、`#sys_condvar_wait`。
- 信号机制：`os/src/syscall/process.rs#sys_rt_sigtimedwait`、`#sys_rt_sigaction`、`#dequeue_pending_signal`、`os/src/syscall/signal.rs#sys_rt_sigprocmask`。

未明确体现：CFS、SMP、CPU 亲和性、cgroups。

### FS（文件系统）
**底层证据强**，呈现多层结构：
- 系统调用层：`os/src/syscall/fs.rs` 实现 `sys_openat`、`sys_read`、`sys_write`、`sys_pread64`、`sys_pwrite64`、`sys_dup3`、`sys_mount`、`sys_umount2`、`sys_newfstatat`、`sys_ftruncate`、`sys_readlinkat`、`sys_sendfile` 等。
- 描述符管理：`fd_table`、`fd_data_table`、`fd_path_table`、`fd_append_table`、`fd_dir_table`。
- Overlay 语义：`overlay_dirs`、`overlay_deleted`、`overlay_child_entries`。
- 管道：`os/src/fs/pipe.rs#make_pipe`。
- `easy-fs`：`easy-fs/src/vfs.rs#create`、`easy-fs/src/layout.rs#clear_size`、`easy-fs-fuse/src/main.rs#easy_fs_pack`。
- 内核态 `ext4` 解析器：`os/src/fs/ext4/inode.rs#load_with_path`、`#read_dir_entries`、`#parse_dir_entries`、`os/src/fs/ext4/superblock.rs#read`、`os/src/fs/ext4/group_desc.rs#read`、`os/src/fs/ext4/mod.rs#find_inode_by_path`。

未明确体现：日志（journaling）、写回缓存策略、硬链接完整实现、ACL。

### DEV（设备驱动）
**底层证据中强**：
- Virtio 块设备：`os/src/la_main.rs#virtio_blk_init`、`#virtio_read_sector`、`#pci_init_virtio_blk`；`os/src/drivers/block/virtio_blk.rs#new`。
- GPU / Input：`os/src/drivers/gpu/mod.rs#try_new`、`os/src/drivers/input/mod.rs#try_new`。
- PLIC / UART / 板级初始化：`os/src/boards/qemu.rs#device_init`、`#irq_handler`。
- 网络设备：`NET_DEVICE` 由网络栈使用。

注意：`la_main.rs` 中的 PCI 枚举与 virtio 寄存器操作属于硬件协议强约束代码，可能来自示例或 crate 封装。

### SYSCALL（系统调用）
**底层证据强**：
- 分发器：`os/src/syscall/mod.rs#syscall` 调用 110+ 个 `compat::sys_*`。
- 兼容层：`os/src/syscall/compat.rs` 提供 `write_user_pod`、`write_cstr_field`、`sys_set_tid_address` 等。
- 用户态封装：`user/src/syscall.rs`。
- 分类清晰：`fs.rs`、`process.rs`、`sync.rs`、`signal.rs`、`time.rs`、`thread.rs`。

### TRAP/IRQ（中断异常）
**底层证据中强**：
- `os/src/trap/mod.rs#trap_handler`：统一入口，处理系统调用追踪、定时器、信号、外部中断。
- `os/src/boards/qemu.rs#irq_handler`：PLIC claim/complete，分发 block/keyboard/mouse/UART 中断。
- `os/src/boards/qemu.rs#device_init`：配置 PLIC 优先级、阈值、外部中断使能。

未明确体现：NMI、M-mode 异常、中断嵌套详细策略。

### NET（网络协议栈）
**底层证据中等，为轻量实现**：
- `os/src/net/mod.rs#net_interrupt_handler`：接收帧、ARP 响应、调用 `LOSE_NET_STACK.analysis`。
- `os/src/net/tcp.rs#write`、`os/src/net/udp.rs#write`：构造并发送数据包。
- `os/src/net/socket.rs`：socket 表、端口分配、`add_socket` / `get_socket` / `push_data` / `pop_data`。
- 用户态示例：`user/src/bin/tcp_simplehttp.rs`、`user/src/bin/udp.rs`。

未明确体现：完整 TCP 状态机、拥塞控制、IP 路由、ICMP、IPv6、DNS。
