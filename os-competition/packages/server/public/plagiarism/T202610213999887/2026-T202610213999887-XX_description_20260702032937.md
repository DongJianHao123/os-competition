## 第四部分：分模块架构说明

### MM（内存管理）
- **已体现**：用户虚拟内存管理、页表（`PageTable::from_token`）、`VirtAddr`/`VPNRange`、ELF 加载（`memory_set.rs#from_elf` / `from_elf_lazily`）、延迟分配（`handle_lazy_allocation_area`）、写时复制（`pre_handle_cow_and_lazy_alloc`）、`sys_msync`/`sys_brk` 等系统调用、页缺异常处理。
- **证据锚点**：`os/src/mm/memory_set.rs#from_elf_lazily`、`os/src/mm/memory_set.rs#pre_handle_cow_and_lazy_alloc`、`os/src/arch/riscv64/trap/mod.rs#trap_handler`。
- **未体现**：swap、页替换算法、NUMA、huge page、内核堆分配器细节。

### PROC（进程管理）
- **已体现**：完整传统进程管理线索：PCB（`task.rs#kernel_clone`、`initproc` 含 Arc、原子字段、父子关系、线程组）、`clone`/`execve`/`exit`/`exit_group`/`wait`、信号处理（`signal/mod.rs#handle_signal`、`sys_rt_sigprocmask`、`sys_rt_sigtimedwait`）、futex（`futex/futex.rs#futex_wait`、`futex/robust_list.rs#exit_robust_list`）、procfs（`/proc/pid`、`/proc/meminfo`、`/proc/cpuinfo` 等）。
- **证据锚点**：`os/src/task/task.rs#kernel_clone`、`os/src/syscall/task.rs#sys_execve`、`os/src/task/processor.rs#run_tasks`、`os/src/task/wait.rs#fetch`。
- **未体现**：cgroup/namespace、ptrace、实时调度类、SMP 多核进程迁移。

### 任务调度/异步运行时
- **已体现**：基于 `run_tasks` 的调度循环、`fetch_task` 取任务、`__switch` 上下文切换、`switch_to` 时间统计、`TIME_MANAGER` 定时器睡眠唤醒（`task/manager.rs#set_wait_alarm`）、socket/block_on 中的阻塞等待队列。
- **证据锚点**：`os/src/task/processor.rs#run_tasks`、`os/src/task/manager.rs#set_wait_alarm`。
- **未体现**：**完整异步运行时底层证据未体现**（无 async/await executor、waker、work-stealing 等），现有证据属于传统阻塞/定时器唤醒式调度。

### FS（文件系统）
- **已体现**：VFS 层（`fs/file.rs`、`fs/pipe.rs`、`fs/namei.rs`）、路径查找与创建（`filename_lookup`、`create_file_from_dentry`）、挂载树（`fs/mount.rs#read_proc_mounts`）、devfs（`fs/dev/mod.rs#init_devfs`）、procfs 大量伪文件、`old/inode_trait.rs` 兼容层。
- **第三方/适配代码**：`ext4/`（`ext4/inode.rs#write_direct`、`ext4/mod.rs#mknod`、`ext4/block_group.rs#dealloc_inode`）与 `fat32/`（`fat32/file.rs#write`）代码量大，应视为移植或强参考实现，不宜直接等同于原创内核主体。
- **证据锚点**：`os/src/fs/namei.rs#create_file_from_dentry`、`os/src/fs/dev/mod.rs#init_devfs`、`os/src/fs/proc/meminfo.rs#serialize`。
- **未体现**：日志恢复、RAID/LVM、块层 elevator、独立页缓存策略。

### DEV（设备驱动）
- **已体现**：PCI 枚举（`arch/la64/drivers/pci.rs#init`）、设备树解析（`Fdt::from_ptr`）、VirtIO net（`drivers/net/mod.rs#recv`、`net/mod.rs#receive`）、VirtIO block（`arch/la64/virtio_blk.rs#unshare`、`arch/riscv64/virtio_blk.rs#unshare`）、定时器中断配置。
- **证据锚点**：`os/src/arch/la64/drivers/pci.rs#init`、`os/src/drivers/net/mod.rs#recv`、`os/src/arch/la64/virtio_blk.rs#unshare`。
- **未体现**：USB、SATA/NVMe 原生驱动、GPU、声卡、中断控制器（PLIC/IOAPIC）完整驱动。

### SYSCALL（系统调用）
- **已体现**：大分发器 `syscall/mod.rs#syscall`（152 条调用链），覆盖 fs、task、net、signal、sched、futex 等；`syscall/fs.rs` 中 read/write/pread/pwrite/openat/renameat2/unlinkat/fallocate/msync 等实现完整。
- **证据锚点**：`os/src/syscall/mod.rs#syscall`、`os/src/syscall/fs.rs#sys_read`、`os/src/syscall/task.rs#sys_execve`。
- **未体现**：seccomp、audit、io_uring、BPF。

### TRAP/IRQ（中断异常）
- **已体现**：双架构异常入口（`arch/la64/trap/mod.rs#trap_handler`、`arch/riscv64/trap/mod.rs#trap_handler` / `kernel_trap_handler`）、初始化（`trap/mod.rs#init`）、LoongArch 定时器（`trap/timer.rs#set_next_trigger`、`enable_timer_interrupt`）、`estat` 异常原因解析。
- **证据锚点**：`os/src/arch/la64/trap/mod.rs#trap_handler`、`os/src/arch/riscv64/trap/mod.rs#trap_handler`、`os/src/arch/la64/trap/timer.rs#set_next_trigger`。
- **未体现**：中断控制器完整驱动、IPI、NMI、多核中断亲和性。

### NET（网络协议栈）
- **已体现**：socket 抽象层（`net/socket.rs`）、TCP/UDP（`net/tcp.rs#connect`、`net/udp.rs#bind`）、Unix domain socket（`net/unix.rs`）、监听表（`net/listentable.rs#accept`）、loopback 抓包（`net/loopback.rs#snoop_tcp_from_ip`）、网络接收入口（`net/mod.rs#receive`）。
- **证据锚点**：`os/src/net/socket.rs#read`、`os/src/net/tcp.rs#connect`、`os/src/net/udp.rs#bind`、`os/src/net/mod.rs#receive`。
- **未体现**：**独立 TCP/IP 协议栈底层证据未体现**（代码中直接使用 `smoltcp` 的 `SOCKET_SET`、`tcp::Socket`、`IpEndpoint` 等类型），IPv6 仅配置解析、路由/防火墙/netfilter 未体现。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块/特性
1. **完整异步运行时**：无 async/await executor、waker、任务队列等证据，现有调度为传统阻塞/定时器唤醒模型。
2. **SMP 多核支持**：无 IPI、多核调度器、CPU 亲和性完整实现（仅 `sys_sched_getaffinity` 接口）。
3. **独立网络协议栈**：TCP/IP 核心依赖 smoltcp（`SOCKET_SET`、`tcp::Socket` 等），非自研协议栈。
4. **块设备/存储高级特性**：无 RAID、LVM、日志恢复、块层 elevator、swap、页替换算法。
5. **安全与隔离机制**：无 seccomp、LSM、BPF、namespace/cgroup、ptrace。
6. **外设驱动**：无 USB、SATA/NVMe 原生驱动、GPU、声卡。
7. **电源管理与虚拟化**：无相关代码证据。
8. **EXT4/FAT32 的原创性**：代码量大但属于典型的第三方文件系统实现形态，应作为移植/适配模块单独评估，不宜直接视为原创内核主体。
