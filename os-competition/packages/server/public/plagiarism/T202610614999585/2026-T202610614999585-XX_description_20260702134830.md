## 第四部分：分模块架构说明

### MM（内存管理）
- **已体现**：页故障处理（四架构 trap handler 均调用 `handle_page_fault`）、`mmap`/`munmap`/`mprotect`/`madvise`/`msync`/`fallocate`/`brk` 类系统调用、ELF 加载时的地址空间映射、全局分配器初始化、共享文件映射的同步与清理、TLB flush。
- **证据锚点**：
  - `api/src/imp/mm/mmap.rs#sys_mmap`
  - `api/src/imp/mm/mmap.rs#sys_madvise`
  - `arceos/modules/axhal/src/arch/riscv/trap.rs#riscv_trap_handler`
  - `arceos/modules/axruntime/src/lib.rs#init_allocator`
- **未体现/弱证据**：swap、页替换算法、huge page、NUMA、完整页表管理原创实现（多依赖 `axhal` 封装）。

### PROC（进程管理/任务调度）
- **已体现**：进程/线程模型（`ProcessData`、`ThreadData`、`PROCESS_TABLE`、`PROCESS_GROUP_TABLE`）、`clone`/`execve`/`exit`/`waitpid`/`getpid`/`getppid`、文件描述符与 FD_CLOEXEC 清理、信号（`sigprocmask`、`rt_sigtimedwait`、`sigchld`）、futex、SysV 共享内存、CPU 亲和性、`sched_setaffinity`/`sched_getaffinity`、等待队列、任务退出通知。
- **证据锚点**：
  - `api/src/imp/task/clone.rs#sys_clone`
  - `api/src/imp/task/exit.rs#do_exit`
  - `api/src/imp/task/wait.rs#sys_waitpid`
  - `axtask/src/run_queue.rs#exit_current`
  - `api/src/imp/task/schedule.rs#sys_sched_setaffinity`
- **未体现/弱证据**：完整 CFS 调度器、cgroups、ptrace、完整 namespace 实现（仅有 clone flags 与 mount 传播）、内核抢占细节。

### FS（文件系统）
- **已体现**：VFS 抽象（`SimpleDir`/`SimpleFile`/`NodeOpsMux`）、procfs（`/proc/[pid]/stat/status/maps/mounts/coredump_filter/oom_score_adj/task` 等）、tmpfs、ext4 包装（`axfs-ng`）、mount/umount、挂载传播（shared/private/slave/peer group）、fd 管理、`openat`/`read`/`write`/`splice`/`copy_file_range`/`ioctl`/`linkat`/`unlinkat`/`readlinkat`、pipe。
- **证据锚点**：
  - `api/src/imp/fs/mount.rs#sys_mount`
  - `core/src/vfs/mod.rs#latest_mount_for_proc`
  - `api/src/imp/fs/fd_ops.rs#sys_openat`
  - `api/src/imp/fs/io.rs#sys_splice`
  - `core/src/vfs/proc.rs#lookup_child`
  - `arceos/modules/axfs-ng/src/fs/ext4/inode.rs#create`
- **未体现/弱证据**：ext4 日志恢复、原创块设备驱动、其他原创文件系统（如 FAT/Btrfs/ZFS）、完整缓存/IO 调度。

### DEV（设备驱动）
- **已体现**：`axdriver` 驱动框架初始化与设备枚举（block/display/net）、x86_64 IDT、RISC-V stvec 设置、工具链中的 PL011 UART 与 raw socket 网卡示例。
- **证据锚点**：
  - `arceos/modules/axdriver/src/lib.rs#init_drivers`
  - `arceos/modules/axhal/src/arch/x86_64/idt.rs#new`
  - `arceos/modules/axhal/src/arch/riscv/mod.rs#set_trap_vector_base`

### SYSCALL（系统调用）

  - `src/syscall.rs#handle_syscall`

### 1. 没有底层证据支撑的模块/能力
- **完整原创异步运行时**：现有证据集中在 `axtask` 线程/任务调度、等待队列、yield/block，未见独立的 waker/futures/executor 原创实现。
- **swap / 页替换 / huge page / NUMA**：内存管理证据集中在 mmap、页故障、分配器初始化，未见 swap 分区、LRU/Clock 页替换、大页、NUMA 相关代码。
- **原创块设备/存储控制器驱动**：未见 NVMe、AHCI、SD/MMC、SATA、IDE 等驱动的原创实现；`axdriver` 主要是框架与枚举。
- **USB / GPU / 声卡 / 高级外设驱动**：底层证据未体现。
- **完整中断控制器驱动**：仅有 trap handler 与 IDT/stvec 设置，GIC/PLIC/IOAPIC/LIOINTC 等具体控制器驱动多为框架封装。
- **完整网络安全/过滤**：netfilter、iptables、eBPF、seccomp、LSM、audit 等底层证据未体现。
- **完整 cgroup / 资源限制**：除 `rlim` 与 CPU 亲和性外，cgroup v1/v2、CPU/memory/io 限制底层证据未体现。
- **完整 ptrace / 调试 / 内核探针**：底层证据未体现。
- **文件系统日志恢复与高级特性**：ext4 包装存在，但日志恢复、extent、inline data 等高级特性底层证据未体现。

### 2. 第三方/公共依赖说明
- **ArceOS 框架**：大量代码位于 `arceos/` 目录，包含 `axhal`、`axtask`、`axruntime`、`axdriver`、`axnet`、`axfs-ng`、`axlibc` 等模块，属于项目基座或公共依赖。
- **smoltcp**：`arceos/modules/axnet/src/smoltcp_impl/` 显示网络协议栈基于 smoltcp 构建，不应视为原创 TCP/IP 协议实现。
- **ext4 crate**：`arceos/modules/axfs-ng/src/fs/ext4/` 为 ext4 文件系统的包装/适配层，底层 ext4 逻辑来自第三方 crate。
- **musl / busybox**：用户态 C 库与 init 程序为外部二进制，仅作为兼容性测试线索。

### 3. 判断性质说明
- 本文档所有结论均基于 AST Call Map、函数名、模块路径与调用链，属于**技术事实描述**。
- “基于 ArceOS 的 Linux ABI 兼容内核”仅为代码线索指向的**可能方向**，不构成最终作品身份或原创性判定。
- 未做抄袭/违规判断；未体现模块不代表作品不存在，仅表示在当前静态证据中未观察到可引用的底层实现。
