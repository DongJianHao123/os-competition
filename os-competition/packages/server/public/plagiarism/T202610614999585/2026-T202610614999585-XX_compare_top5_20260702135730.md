## 4. 核心代码调用流对比剖析

## 6. 保守结论

双方 AST 相似度 36.97%，共同指纹 1390 个，但经路径溯源剥离，其中大量位于 `arceos/`、`axlibc/`、`axhal/`、`axfs-ng/`、`axnet/` 等共同上游开源框架与第三方库路径。进程管理板块覆盖率仅 26.17%/23.14%，且项目自身代码在抽象层设计（`StarryTaskExt` vs `current_thread_data`）、同步策略（`RwLock` 读锁 vs `Mutex` 互斥锁）、文件系统后端（`SimpleFs` vs `DynamicFs`）、套接字类型支持（Unix 域套接字扩展）等关键架构决策上存在实质性差异。控方所列“底层逻辑一致”函数经路径审查均属 ArceOS 框架标准实现或第三方公共代码，跨板块同名配对（如 `do_exit` 内核 vs shell）不构成同板块主证据。综合判定：**双方为基于共同开源框架（ArceOS）的独立开发项目，不构成代码抄袭或系统性复用**。建议后续比对先行剥离 `arceos/`、`axlibc/`、第三方库等公共上游模块，仅针对项目自身业务代码进行差异化分析。
