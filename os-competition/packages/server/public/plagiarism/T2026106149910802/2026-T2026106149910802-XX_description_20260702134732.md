## 第四部分：分模块架构说明

### MM（内存管理）

底层证据较充分，覆盖物理页框分配、多架构缺页异常、地址空间复制与 ELF 解释器加载。

- `arch/src/loongarch64/trap.rs#loongarch64_trap_handler`：读取 `badv`、`estat.cause`，处理 `LoadPageFault` / `StorePageFault`。
- `arch/src/aarch64/trap.rs#handle_exception`：读取 `ESR_EL1`、`FAR_EL1`，处理 `InstructionPageFault`。
- `os/src/mm/frame_allocator.rs#init_frame_allocator`：初始化 `FRAME_ALLOCATOR`，对物理内存区域按 `u128` 填充管理。
- `os/src/task/task.rs#fork`：通过 `MemorySet::from_existed_user` 复制用户地址空间。
- `os/src/mm/memory_set.rs#load_interp`：解析 ELF section、处理 `DL_INTERP_OFFSET`，支持动态链接器加载。

**未体现**：swap、COW 细节、页缓存写回策略、NUMA 等底层证据未体现。

### PROC（进程管理/任务调度）

传统进程管理有底层证据，但“异步运行时”未直接体现。

- `os/src/syscall/process.rs#sys_clone3`：解析 `CloneFlags`，调用 `current_task.fork` 后 `add_task`。
- `os/src/syscall/process.rs#sys_exec`：读取 inode 全部数据、构造参数向量、替换当前进程映像。
- `os/src/task/task.rs#fork`：新建 `KernelStack`、`FdTable`、`MemorySet`，具备 PCB 复制能力。
- `os/src/task/processor.rs#run_tasks`：调度循环，调用 `fetch_task` 与 `context_switch_pt`。
- `os/src/syscall/process.rs#sys_exit` / `sys_exit_group`：调用 `exit_current_and_run_next`。
- `user/src/lib.rs#waitpid`：用户侧封装 `sys_waitpid` 与 `yield_`。
- `os/src/syscall/process.rs#sys_sigaction` / `sys_sigreturn`：信号注册与返回。
- `os/src/task/futex.rs#futex_wait` / `futex_wait_bitset`：futex 等待队列与阻塞切换。
- `os/src/fs/epoll.rs#wait`：epoll 等待与 poll 事件处理。

**未体现**：显式 `Future` / `Waker` / `Executor` 异步运行时、实时调度器、CPU 亲和性完整实现、cgroup/namespace。

### FS（文件系统）

文件系统是证据最强的模块，具备 VFS 抽象与多种具体文件系统。

- `vfs/src/lib.rs#init`：从 `FILE_SYSTEMS` 查找并挂载 `devfs`、`procfs`、`ext4` 等。
- `os/src/syscall/fs.rs#sys_mount`：使用 `BLOCK_DEVICE` 与 `FILE_SYSTEMS.find_fs("Ext4")` 完成挂载。
- `ext4/src/fs.rs#mount` / `ext4/src/dentry.rs#concrete_lookup` / `concrete_unlink` / `concrete_rename`：ext4 挂载、查找、删除、重命名。
- `ext4_rs-1.3.1/src/ext4_impls/balloc.rs#balloc_alloc_block` / `ext4_rs-1.3.1/src/ext4_impls/inode.rs#allocate_new_block`：ext4 块分配与 inode 块分配。
- `easy-fs-fuse/src/main.rs#easy_fs_pack` / `efs_test`：easy-fs 打包与测试工具。
- `vfs/src/devfs/mod.rs#init_devfs` / `vfs/src/procfs/mod.rs#init_procfs`：设备文件系统与 proc 文件系统初始化。
- `os/src/fs/stdio.rs`：标准输入输出文件实现。
- `os/src/syscall/fs.rs#sys_getdents` / `sys_readlinkat` / `sys_utimensat` / `sys_fallocate`：目录遍历、符号链接、时间戳、预分配。

**注意**：`ext4_rs-1.3.1`、`easy-fs` 属于第三方/移植库，不应全部视为原创内核主体。

### DEV（设备驱动）

设备驱动证据集中在 virtio 与平台中断控制器。

- `virtio-drivers/src/device/blk.rs#read` / `write`：virtio-blk 读写，使用 `State::wait_until_queue_notified`。
- `virtio-drivers/src/queue.rs#fake_read_write_queue`：virtqueue 描述符链处理。
- `virtio-drivers/src/device/socket/connectionmanager.rs#send_recv` / `poll` / `incoming_connection`：virtio-socket 连接管理。
- `os/src/drivers/chardevice/uart.rs#new`：NS16550a UART 字符设备初始化。
- `os/src/drivers/plic.rs#enable` / `disable` / `set_priority` / `set_threshold`：RISC-V PLIC 中断控制。
- `arch/src/x86_64/apic.rs#init`：x86_64 Local APIC / IO-APIC 初始化。

**注意**：`virtio-drivers/` 为第三方 crate，仅做集成使用。

### SYSCALL（系统调用）

存在统一系统调用分发入口，覆盖文件、进程、网络、epoll。

- `os/src/syscall/mod.rs#syscall`：总入口，根据 syscall 号分发（调用链规模 123）。
- `os/src/syscall/fs.rs`：`sys_openat`、`sys_read`/`sys_write`、`sys_pread64`/`sys_pwrite64`、`sys_readv`/`sys_writev`、`sys_getdents`、`sys_mount`、`sys_utimensat`、`sys_fallocate`、`sys_readlinkat` 等。
- `os/src/syscall/process.rs`：`sys_clone3`、`sys_exec`、`sys_exit`、`sys_sigaction`、`sys_futex`、`sys_prlimit64`、`sys_setitimer`、`sys_nanosleep`、`sys_getpid`/`sys_getppid`/`sys_gettid` 等。
- `os/src/syscall/socket.rs`：`sys_sendmsg`、`sys_recvfrom`。
- `os/src/fs/epoll.rs`：`sys_epoll_pwait`、`sys_epoll_create1`、`sys_epoll_ctl`。

### TRAP/IRQ（中断异常）

各架构有 trap handler，但中断子系统深度证据有限。

- `arch/src/loongarch64/trap.rs#loongarch64_trap_handler`：LoongArch 异常/中断处理。
- `arch/src/loongarch64/trap.rs#tlb_init`：TLB refill 与页表宽度配置。
- `arch/src/aarch64/trap.rs#handle_exception`：AArch64 异常处理。
- `arch/src/x86_64/idt.rs#new`：IDT 初始化。
- `os/src/drivers/plic.rs`：PLIC 中断使能/屏蔽/优先级。

**未体现**：完整中断路由框架（如统一 IRQ descriptor、中断线程化）、时钟中断源细节、多核 IPI。

### NET（网络协议栈）

网络功能主要依赖 `lose-net-stack`，内核侧做 socket 封装。

- `lose-net-stack/src/connection/tcp.rs#send_data` / `connect` / `accept` / `close` / `readable`：TCP 连接生命周期。
- `lose-net-stack/src/connection/udp.rs#sendto`：UDP 发送。
- `lose-net-stack/src/connection/mod.rs#analysis_ip` / `analysis_arp`：IP/ARP 包解析。
- `lose-net-stack/src/packets/arp.rs#build_data`：ARP 报文构造。
- `os/src/socket.rs#poll` / `drop`：内核 socket 文件状态与关闭。
- `os/src/syscall/socket.rs#sys_sendmsg` / `sys_recvfrom`：系统调用入口。

**注意**：`lose-net-stack/` 为第三方/移植网络栈，内核侧证据以封装为主。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块/特性

- **显式异步运行时**：未观察到 `Future` / `Waker` / `Executor` / `async fn` 构成的运行时，现有“任务调度/异步运行时”标签实际对应传统阻塞式任务切换。
- **安全与隔离机制**：capabilities、ACL、seccomp、SELinux、namespace、cgroup 等底层证据未体现。
- **SMP 与多核调度**：仅见 `sys_getcpu` 等少量接口，未体现 CPU 间 IPI、自旋锁/睡眠锁的完整多核框架。
- **电源管理、虚拟化/ hypervisor、图形/声卡、实时调度、内核调试器**：底层证据未体现。
- **文件系统高级特性**：配额（quota）、日志（journaling）内部机制在 `ext4_rs` 中可能有实现，但作为第三方库集成，非本内核原创主体。
- **完整网络栈原创性**：`lose-net-stack/` 为第三方/移植实现，内核侧仅提供 socket 文件封装与系统调用转发。

### 需要单独说明的第三方/公共依赖

- `virtio-drivers/`、`ext4_rs-1.3.1/`、`easy-fs/`、`lose-net-stack/` 属于外部 crate 或 vendored 代码，提供 virtio 设备、ext4/easy-fs 文件系统、TCP/UDP 协议栈能力，不应被夸大为本内核的原创主体。
- `arch/` 目录包含多架构支持代码，但部分文件（如 `virtio-drivers/examples/`）属于驱动示例，而非内核运行时代码。

### 判断性质说明

- 本报告中“可能的系统方向”“保守系统方向提示”仅基于目录结构、函数分布与 AST 调用链的统计推断，**不是对作品身份、原创性或最终成绩的结论**。
- 本报告不做抄袭、违规或学术不端判断。
