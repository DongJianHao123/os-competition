## 第四部分：分模块架构说明

### MM（内存管理）
**证据**：`kernel/src/syscall/mm/mmap.rs#sys_mmap` 实现了多种 mmap 后端（alloc、cow、file、linear、shared）；`kernel/src/mm/aspace/backend/cow.rs#alloc_new_at` 实现 COW 页复制；`kernel/src/syscall/mm/process_vm.rs` 实现跨进程内存读写；第三方 `memory_set` 提供地址空间区域管理。

**可说明**：内核具备用户态地址空间映射、文件映射、共享内存、COW、跨进程 VM 访问等能力。物理页分配、页错误处理、地址空间底层管理依赖第三方组件。

### PROC（进程管理/任务调度）
**证据**：`kernel/src/task/process.rs#new` 构造含凭证、能力、资源限制、futex 表、执行控制状态、命名空间的 PCB；`kernel/src/syscall/task/clone.rs#do_clone` 创建线程/进程；`kernel/src/syscall/task/execve.rs#do_execve` 执行程序替换；`kernel/src/task/ops.rs#do_exit` 处理退出与收割；`kernel/src/syscall/task/wait.rs` 实现 waitpid/waitid；`kernel/src/task/futex.rs` 实现 futex 等待/唤醒/重排队；`kernel/src/task/signal.rs` 实现信号投递；`kernel/src/pseudofs/proc.rs` 与 `cgroup.rs` 暴露进程与 cgroup 信息。

**可说明**：实现了传统进程管理（clone/exec/exit/wait/zombie/PCB/父子关系/线程组/会话/信号/futex/cgroup/命名空间）。调度器核心未在内核层体现，可能由第三方 `axtask` 提供。

### FS（文件系统）
**证据**：`kernel/src/syscall/fs/mount.rs` 实现 mount/umount/fsmount/move_mount/open_tree；`kernel/src/syscall/fs/fd_ops.rs` 实现 openat2/fcntl/add_to_fd；`kernel/src/syscall/fs/io.rs` 实现 splice/copy_file_range/pwritev/fallocate；`kernel/src/syscall/fs/ctl.rs` 实现 readlinkat 等；`kernel/src/file/*` 实现文件描述符、lease、signalfd、pidfd、netlink；`kernel/src/pseudofs/*` 实现 procfs、cgroupfs、devfs、tty、loop。

**可说明**：具备较完整的 Linux 兼容 VFS 层、挂载命名空间、多种伪文件系统。底层 ext4 与 VFS 基础设施来自 `lwext4_rust` 与 `axfs-ng`。

### DEV（设备驱动）
**证据**：模块统计仅 1 个函数；内核层仅有 `kernel/src/pseudofs/dev/tty/mod.rs#ioctl`、`kernel/src/pseudofs/dev/loop.rs#ioctl` 等伪设备节点处理，无硬件寄存器操作、中断控制器、时钟、块设备驱动代码。

**可说明**：**原创设备驱动底层证据未体现**。所有硬件抽象、驱动初始化依赖 `third_party/rust-patches/axdriver`、`axhal` 等第三方层。

### SYSCALL（系统调用）
**证据**：`kernel/src/syscall/mod.rs#handle_syscall` 为集中式分发器；子目录覆盖 `fs`、`net`、`task`、`mm`、`sync`、`ipc`、`signal`、`bpf` 等；关键函数包括 `sys_keyctl`、`sys_prctl`、`sys_sched_setaffinity`、`sys_finit_module`、`sys_futex`、`sys_mq_open` 等。

**可说明**：实现了较广的 Linux syscall 表面，覆盖文件、网络、进程、内存、同步、IPC、信号、BPF 等类别。

### TRAP/IRQ（中断异常）
**证据**：内核自身仅 5 个函数；架构相关 trap 处理位于 `third_party/rust-patches/axcpu/src/{aarch64,loongarch64,riscv}/trap.rs`，包含 `handle_page_fault`、`handle_breakpoint`、ESR/scause 解析。

**可说明**：**内核自身中断/异常处理底层证据极弱**，主要依赖第三方架构抽象层。

### NET（网络协议栈）
**证据**：`kernel/src/syscall/net/*` 提供 socket 系统调用；`crates/axnet-ng/src/lib.rs#init_network` 初始化 smoltcp 接口；`third_party/rust-patches/starry-smoltcp/src/socket/tcp.rs#process/dispatch` 为 TCP 状态机主体。

**可说明**：网络协议栈主体为第三方 `starry-smoltcp`，内核层主要负责 socket API 与协议栈集成。
