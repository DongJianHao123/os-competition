## 第四部分：分模块架构说明

### MM（内存管理）

- **地址空间与 VMA**：`kernel/src/mm/address_space.rs` 与 `kernel/src/syscall/memory.rs#sys_mmap` 是核心证据，`sys_mmap` 中直接构造 `VirtAddr`、`Vma`、`VmaPermission`，说明实现了基于 VMA 的地址空间管理。
- **按需分页**：`kernel/src/arch/riscv64/trap.rs#trap_handler_rust` 调用 `addr_space.install_demand_page`；`kernel/src/arch/loongarch64/trap.rs#la64_trap_handler` 调用 `addr_space.handle_page_fault`，说明支持缺页异常处理。
- **页帧与 TLB**：`kernel/src/mm/percpu_cache.rs`、`kernel/src/mm/tlb.rs` 出现 `get_cache`、`get_manager` 等调用，说明有 per-CPU 页帧缓存与 TLB 管理。
- **内存 cgroup / 调试**：`kernel/src/mm/memcg.rs`、`kernel/src/mm/frame_ownership_dump_dbg`、`kernel/src/mm/uaf_dbg_enabled` 说明有内存控制组与 use-after-free 调试辅助。
- **userfaultfd**：`kernel/src/kernel/userfaultfd.rs#ioctl_api` 出现 `UFFD_API` 处理，说明实现了 userfaultfd 接口。

### PROC（进程管理/任务调度）

- **进程控制块**：`kernel/src/sched/process.rs` 中 `PROCESS_TABLE` 是全局 PCB 表，字段涵盖 `pid/tgid/pgid`、`children`、`fd_table`、`mm`、`cgroup_path`、`nsproxy`、`comm`、`trap_context`、CPU 时间片、rlimit 等。
- **生命周期**：
  - 创建：`kernel/src/sched/process_fork.rs#fork_with_ctx`、`kernel/src/syscall/process_clone.rs#do_fork_clone`
  - 执行：`kernel/src/syscall/process_exec.rs#sys_execve` → `kernel/src/exec/mod.rs#execve_from_kernel_args`
  - 等待：`kernel/src/syscall/process_wait.rs#sys_waitpid` → `kernel/src/sched/process_exit.rs#wait_nonblock` / `waitid_reap`
  - 退出：`kernel/src/sched/process_exit.rs#exit` → `kernel/src/sched/process.rs#mark_zombie` / `reap_or_defer`
- **调度器**：
  - `kernel/src/sched/process.rs#select_next` 选择下一个可运行任务，支持 `next_runnable_after`、`next_startup_leader`、cgroup 冻结/CPU 限制检查。
  - `kernel/src/sched/process_sched.rs#timer_preemption_due` 基于 `rdtime` 与优先级判断是否需要抢占。
  - `kernel/src/sched/process_sched.rs#switch_thread_startup_to_leader` 处理线程启动后切到 leader。
- **I/O 等待**：`kernel/src/fs/write_io.rs#sys_writev` 调用 `suspend_current_for_io` / `enter_wait_channel`；`kernel/src/fs/read_io.rs#sys_readv` 调用 `suspend_current_and_run_next`，说明调度器支持因 I/O 阻塞而主动让出 CPU。
- **信号**：`kernel/src/kernel/signal_syscall.rs#do_kill`、`sys_rt_sigtimedwait`、`sys_rt_sigprocmask`。
- **命名空间/cgroup**：`kernel/src/fs/proc/pid.rs` 读取 `nsproxy` 的 cgroup/ipc/mnt/net/pid/time/user/uts ns；`kernel/src/kernel/cgroup.rs` 有 `cgroup_move_process`、`cgroup_io_can_proceed`。

### FS（文件系统）

- **VFS 层**：`kernel/src/fs/mod.rs`、`kernel/src/fs/ramfs.rs`、`kernel/src/fs/fd.rs`、`kernel/src/fs/file_lock.rs` 构成核心 VFS，支持 inode、目录遍历、FD 表、文件锁。
- **伪文件系统**：
  - `kernel/src/fs/procfs.rs` 及 `kernel/src/fs/proc/*`：/proc/pid、/proc/sys、/proc/net、/proc/mounts 等。
  - `kernel/src/fs/sysfs.rs`：sysfs。
  - `kernel/src/fs/devfs.rs`：devfs。
  - `kernel/src/fs/pipe.rs`：管道。
  - `kernel/src/fs/epoll.rs`：epoll。
- **挂载与命名空间**：`kernel/src/syscall/mount.rs` 实现 `sys_mount`、`sys_fsmount`、`sys_move_mount`、`sys_open_tree`、`sys_mount_setattr`、`sys_umount2`，并操作 `MOUNT_INFO_TABLE`、`PENDING_MOUNTS`，支持 bind/overlay/move mount。
- **文件通知/锁**：`kernel/src/fs/notify/fanotify.rs`、`kernel/src/fs/inotify.rs`、`kernel/src/fs/file_lock.rs`。
- **文件句柄**：`kernel/src/fs/file_handle.rs#sys_name_to_handle_at` / `sys_open_by_handle_at` 说明支持 `fhandle`（`CONFIG_FHANDLE`）。
- **第三方 ext4**：`kernel/src/fs/ext4.rs` 是对第三方 ext4 库的包装/适配层，不是从零实现的 ext4。

### DEV（设备驱动）

- **VirtIO 块设备**：`kernel/src/driver/virtio/blk.rs#read_sync` / `write_sync` 通过 `BlkRequest` 构造读写请求并轮询完成。
- **VirtIO 网卡**：`kernel/src/driver/virtio/net.rs#new` / `recv_packet` 初始化 VirtIO net 并收包。
- **驱动框架自测**：`kernel/src/driver/mod.rs#self_test` 出现 `MmioManager`、`InterruptForwarder`、`DeviceCapManager`，但仅为自测桩，未看到大量真实设备驱动。
- **未体现**：UART、RTC、SATA/NVMe、USB、GPU、音频、Framebuffer 等设备驱动底层证据未体现。

### SYSCALL（系统调用）

- **统一分发**：`kernel/src/syscall/mod.rs#syscall_inner` 是核心分发器，内部调用 323 个目标，覆盖 AIO、io_uring、mount、open、clone、exec、wait、mmap、socket、keyring、seccomp、userfaultfd、time 等。
- **文件相关**：`kernel/src/syscall/open.rs`、`read_io.rs`、`write_io.rs`、`fcntl.rs`、`mount.rs`、`dir.rs`。
- **进程相关**：`kernel/src/syscall/process_clone.rs`、`process_exec.rs`、`process_wait.rs`、`process_group.rs`。
- **内存相关**：`kernel/src/syscall/memory.rs`。
- **网络相关**：`kernel/src/syscall/socket.rs`。
- **安全/其他**：`kernel/src/security/keyring.rs`、`kernel/src/seccomp/mod.rs`、`kernel/src/kernel/userfaultfd.rs`、`kernel/src/io_uring/mod.rs`、`kernel/src/ipc/sysv_*.rs`。

### TRAP/IRQ（中断异常）

- **架构 trap 入口**：
  - RISC-V：`kernel/src/arch/riscv64/trap.rs#trap_handler`、`trap_handler_rust`
  - LoongArch：`kernel/src/arch/loongarch64/trap.rs#init`、`la64_trap_handler`
- **功能**：异常原因解析、page fault 分发、timer tick（调用 `net_stack::on_timer_tick`）、调度抢占、中断记录。
- **未体现**：完整的中断控制器驱动（PLIC/APLIC/IOINTC 仅在 `arch/loongarch64/trap.rs#init` 中提及 `eiointc::init`）、SMP IPI、MSI/MSI-X、IRQ 子系统底层证据未充分体现。

### NET（网络协议栈）

- **Socket 层**：`kernel/src/net/socket.rs` 实现 `setsockopt_fd`、`getsockopt_fd`、`recv_timeout_deadline_fd`，支持 UNIX、AF_ALG、AF_NETLINK 等。
- **后端依赖 smoltcp**：`kernel/src/net/backend.rs#new` 调用 `net_stack::add_tcp_socket`、`add_udp_socket`、`add_raw_socket`、`add_icmp_socket`，说明 TCP/UDP/ICMP/RAW IP 由 smoltcp 提供。
- **协议栈事件**：`kernel/src/net/net_stack.rs#on_timer_tick` / `init_dns`、`kernel/src/net/tcp_retransmit.rs#on_sack`、`kernel/src/net/dns.rs`。
- **Netlink**：`kernel/src/net/af_netlink.rs#build_getlink_dump` / `build_getroute_dump` 生成 netlink 报文。
- **/proc/net**：`kernel/src/net/procnet.rs` 生成 TCP/UDP 等连接信息。
- **第三方说明**：TCP/IP 协议核心不是从零实现，而是对 smoltcp 的包装与调度。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块/能力

- **完整中断子系统**：仅有 `arch/riscv64/trap.rs`、`arch/loongarch64/trap.rs` 等异常入口；PLIC/APLIC/IOINTC 等中断控制器的完整驱动、SMP IPI、MSI/MSI-X 管理底层证据未体现。
- **除 VirtIO blk/net 外的设备驱动**：UART、RTC、SATA/NVMe、USB、GPU、音频、Framebuffer、I2C/SPI 等设备驱动底层证据未体现。
- **从零实现的文件系统**：`ext4` 出现在 `kernel/src/fs/ext4.rs`，但调用链显示其为第三方库的包装/适配层，不是从零实现的 ext4。
- **从零实现的 TCP/IP 协议栈**：`net/backend.rs` 直接调用 `smoltcp` 的 `add_tcp_socket`/`add_udp_socket` 等，TCP/IP 核心由 smoltcp 提供。
- **架构覆盖**：仅有 RISC-V64 与 LoongArch64 的 trap 相关证据；x86_64、aarch64 等其他架构底层证据未体现。
- **C 语言子系统**：`.c` 文件仅 15 个，未形成 C 编写的驱动或子系统，大概率仅为启动/汇编辅助。
- **用户态 libc/运行时**：`user/` 目录为用户测试程序与 Shell，不是内核。
- **形式化验证、安全证明、unsafe 规范文档**：底层证据未体现。
- **实时调度完整实现**：仅看到 `sched_policy` 与优先级检查，完整 RT 调度器底层证据未充分。
- **虚拟化/容器完整栈**：cgroup、namespace 有证据，但 hypervisor/KVM、容器运行时等未体现。

### 判断边界说明

- 本文档对“系统方向”的判断仅基于目录名、函数名、调用链与模块统计，是**保守的技术方向推测**，不是对作品最终身份、原创性或完整性的结论。
- 对第三方库（smoltcp、ext4 库）的说明，仅表示相关模块存在明显的第三方依赖包装，不夸大其为原创内核主体。
- 本文档不做抄袭、违规或评分判断。
