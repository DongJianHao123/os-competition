## 第四部分：分模块架构说明

### MM（内存管理）
- **证据**：`kernel/main.c#main` 调用 `kinit`、`kvminit`；`kernel/proc.c#kfork` 使用 `PA2PTE`、`PTE2PA`、`PTE_FLAGS`、`PX` 等页表宏；`kernel/arch/riscv/start.c#start` 配置 M/S 模式 CSR。
- **说明**：具备 RISC-V/LoongArch 页表与内核地址空间初始化能力，进程 fork 时进行页表级地址空间复制。但物理内存分配器（如 `kalloc`/`kfree`）内部实现、slab/页缓存策略等**底层证据未充分体现**。

### PROC（进程管理/任务调度）
- **证据**：`kernel/proc.c#scheduler`、`sched`、`yield`、`sleep`、`wakeup`、`myproc`、`cpuid`；`kernel/sysproc.c#sys_fork`、`sys_exit`、`sys_wait`、`sys_clone`、`sys_exit_group`、`sys_kill`、`sys_tgkill`、`sys_futex`、`sys_setpgid`、`sys_setsid`、`sys_gettid`；`kernel/proc.c#reparent`、`kill_thread_group`；`kernel/futex.c`；`kernel/signal.c`。
- **说明**：存在以 PCB 为核心的传统进程管理，支持 fork/exit/wait/clone、线程组、信号、futex。`exec` 路径在提供的 Call Map 中**底层证据未直接体现**（仅 `user/init.c` 调用库函数 `exec`）。

### 任务调度/异步运行时
- **证据**：内核调度器为 `kernel/proc.c#scheduler` + `swtch`；`704` 个函数中大量为 lwIP 的 timer、netconn、socket 回调（如 `tcpip_thread_handle_msg`、`lwip_netconn_do_writemore`）。
- **说明**：**不应将 lwIP 内部机制等同于内核 async runtime**。内核调度是传统的基于 proc 表的轮转/协作式调度，非独立轻量级 task/channel/waker runtime。

### FS（文件系统）
- **证据**：VFS 层在 `kernel/sysfile.c#sys_open`、`sys_openat`、`sys_readlinkat`、`sys_getcwd`，使用 `i_op`（`lookup`/`create`/`readlink`）与 `f_op`（`open`）；后端为 `kernel/fs-ext4/lwext4`，关键函数包括 `ext4_balloc_alloc_block`、`ext4_balloc_free_block`、`ext4_ialloc_alloc_inode`、`ext4_ialloc_free_inode`、`ext4_extent_remove_space`、`ext4_dir_dx_init`、`jbd_journal_prepare`；用户态 `mkfs` 工具存在。
- **说明**：具备类 Unix 文件描述符层与 ext4 后端，支持块分配、inode 分配、extent、目录索引与日志。lwext4 为第三方库。

### DEV（设备驱动）
- **证据**：`kernel/virtio_disk.c#virtio_disk_intr`、`kernel/arch/riscv/virtio_disk.c#virtio_disk_intr`、`kernel/arch/loongarch/virtio_disk.c#virtio_disk_intr`；`kernel/arch/loongarch/virtio_pci.c#find_capability`/`find_virtio_blk_pci_dev`；`kernel/uart.c#uartinit`/`uartputc_sync`；`kernel/console.c#consoleinit`/`consolewrite`；`kernel/arch/riscv/plic.c#plicinithart`；`kernel/bio.c#bwrite -> virtio_disk_rw`。
- **说明**：支持 virtio 块设备（RISC-V MMIO 与 LoongArch PCI 两种形态）、UART 串口控制台、PLIC 中断控制器。设备驱动函数总数少，未见 GPU、USB、声卡、网卡原生驱动等。

### SYSCALL（系统调用）
- **证据**：`kernel/sysfile.c`（open/openat/readlinkat/getcwd 等）、`kernel/sysproc.c`（fork/exit/clone/wait/kill/futex/signal/sched 等）。
- **说明**：提供 POSIX-like 系统调用接口，文件与进程类调用证据充分。

### TRAP/IRQ（中断异常）
- **证据**：`kernel/trap.c#kerneltrap` -> `devintr`、`myproc`、`mycpu`；`kernel/virtio_disk.c#virtio_disk_intr` -> `wakeup`；`kernel/spinlock.c#pop_off`/`push_off`；`kernel/arch/riscv/plic.c#plicinithart`。
- **说明**：具备内核态陷阱分发、设备中断处理、中断使能管理。异常向量表与完整用户态陷阱路径细节**底层证据未充分展开**。

### NET（网络协议栈）
- **证据**：`kernel/lwip/src/core/init.c#lwip_init`、`kernel/lwip/src/api/tcpip.c#tcpip_thread_handle_msg`、socket API、TCP/UDP/IP/ICMP/ND/DHCP/PPP 以及 SNMP、MQTT、mDNS、SNTP、HTTP、SMTP、TFTP 等应用层。
- **说明**：网络功能完整，但**主体为第三方 lwIP**，本项目更多是集成/移植，而非原创网络协议栈。
