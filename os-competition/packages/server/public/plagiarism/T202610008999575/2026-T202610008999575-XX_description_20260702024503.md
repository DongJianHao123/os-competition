## 第四部分：分模块架构说明

### MM（内存管理）

**证据较强**。存在多级页帧分配器（`frame_allocator.rs` 中 `alloc_mega`/`dealloc` 维护 `recycled_giga/recycled_mega/recycled_std`），支持 1G/2M/标准页拆分合并；用户地址空间通过 `memory_set.rs#from_elf_with_interp_loader` 构建，含 `MapArea`、`VirtAddr`、`OFFSET_FOR_USER_APP`；页异常在 `trap_handler` 中调用 `crate::mm::mmap::tick_sync`；`sys_brk` 调用 `mmap::do_brk`。  
**未体现**：NUMA、swap、page cache、COW 细节仅部分可见（`share_from_parent`），大页热插拔等无证据。

### PROC（进程管理/任务调度）

**证据强**。具备完整 PCB（`process/pcb.rs`）、fork/exec/wait/exit/zombie 路径、`clone`/`clone3`/`pthread_create` 线程创建、`copy_fd_table_between_processes`、信号机制（`sys_kill`、`call_signal_handler`、`push_signal_frame`、`restore_signal_context`）、futex 等待队列、`epoll`/`eventfd`、`sched_setaffinity`/`sched_setscheduler`/`sched_setparam`、资源限制 `setrlimit64`、`sys_sysinfo`、`sys_getrusage` 等。调度器 `schedule.rs` 实现 deadline/realtime/fair/idle 四级队列，`processor.rs#run_tasks` 完成上下文切换。  
**未体现**：完整的 CFS 红黑树、SMP 负载均衡、cgroups v2 控制器仅见 procfs 占位。

### FS（文件系统）

**证据强**。`tmpfs.rs` 提供内存文件系统并构造 musl/LTP 测试环境；`procfs.rs` 实现 `/proc` 伪文件系统（`mount_procfs`、`ProcStatusInode`、`ProcStatInode`、`MemInfoInode`、`MountsInode` 等）；`devfs.rs` 提供设备节点；`inode.rs#open_file` 与 `file_tree.rs` 实现 dentry 查找与创建；`pipe.rs` 实现管道；`ext4fs` 目录实现 extent 树、目录项、`truncate`、`raw_write_at`。  
**需说明**：`ext4fs` 代码规模大、结构接近成熟实现，可能为第三方代码或重度移植，不宜直接视为原创内核主体。

### DEV（设备驱动）

**证据中等**。支持 `virtio-blk` 与 `virtio-net`（RISC-V 与 LoongArch 目录下均有），`block_cache.rs` 提供块缓存；`pci/mod.rs#decode` 解析 PCI BAR。  
**未体现**：非 VirtIO 的 SATA/NVMe/USB/字符设备驱动、中断控制器（PLIC/LS7A）细节、DMA 一致性管理细节证据不足。

### SYSCALL（系统调用）

**证据强**。`syscall/mod.rs#syscall` 作为统一分发器，调用数达 157，覆盖 process/fs/net/bpf 等；`syscall/process.rs` 107 个函数，实现 `ioctl`、`pselect6`、`ppoll`、`utimensat`、`kill`、`rt_sigtimedwait`、`exec`、`clone` 等；`syscall/fs.rs` 实现 `openat`、`read`、`write`、`pread64`、`pwrite64`、`writev`、`readv`、`splice`、`unlinkat`、`fallocate`；`syscall/net.rs` 实现 `recvfrom`、`connect`、`getpeername`、`getsockname`、`setsockopt`。  
**未体现**：完整 seccomp、audit、namespace（`unshare` 有入口但细节未展开）、capabilities 证据不足。

### TRAP/IRQ（中断异常）

**证据弱**。仅 13 个函数，核心入口为 `os/src/arch/{riscv,la}/trap/mod.rs#trap_handler` 与 `trap_return`，完成用户态/内核态切换、定时器中断、页异常同步；`enable_timer_interrupt`、`set_kernel_trap_entry` 等完成基本初始化。  
**未体现**：外部中断分发框架、IRQ 亲和性、中断线程化、设备中断共享机制等底层证据未体现。

### NET（网络协议栈）

**证据中等**。`net/socket.rs` 基于 smoltcp 实现 socket 抽象，维护 `SOCKET_SET` 与 `SOCKET_WAIT_QUEUES`；`net/mod.rs` 通过 `NET_DEVICE` 调用 VirtIO-Net 驱动收发；`net/netlink.rs` 实现 netlink 写与路由属性解析，可与 `poll`/`select`/`epoll` 集成。  
**需说明**：核心 TCP/IP 协议逻辑依赖 smoltcp 第三方库，本项目主要提供 socket 适配层与驱动对接层。

## 第六部分：边界与未体现内容

- **没有底层证据支撑的模块/特性**：

  - 高级中断子系统：仅见定时器中断使能与基本 trap 入口，未见 PLIC/LS7A 中断分发、IRQ 亲和性、中断线程化。
  - 非 VirtIO 设备驱动：未见 SATA/NVMe/USB/字符设备/显示/音频驱动。
  - 安全与隔离机制：未见完整 seccomp、LSM、capabilities、namespace 实现细节。
  - 高级内存特性：未见 swap、NUMA、page cache、内存压缩、热插拔。
  - 其他文件系统：仅见 tmpfs/procfs/devfs/ext4，未见 FAT/NFS/日志文件系统等。
  - 虚拟化/ hypervisor、电源管理、内核调试/性能分析子系统均无证据。

  - VirtIO 驱动可能基于 `virtio-drivers` 等公共 crate 封装。

- **判断性质说明**：
  - 本方向判断仅基于 AST Call Map、函数名、目录路径与调用链，属于“可能方向”而非最终原创性或身份结论。
