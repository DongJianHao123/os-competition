## 第四部分：分模块架构说明

### MM（内存管理）

- **证据锚点**：
  - `kernel/src/syscall/mm.rs#sys_mmap`
  - `kernel/src/mm/memory_set/fault.rs#handle_write_page_fault_with_mapping`
  - `kernel/src/syscall/process/fault.rs#import_thread_stack_for_write_fault`
  - `kernel/src/syscall/shm.rs#sys_read_writes_into_fork_cow_user_buffer`
- **说明**：实现了用户态地址空间、页表、物理页分配、`mmap`/`mprotect`/`msync`、写时复制（COW）、懒分配页故障、线程栈镜像、共享内存以及 ELF 加载时的内存映射。`UserBuffer` 通过 `page_table()` 进行用户态读写校验。
- **未体现**：交换（swap）、大页、NUMA、ASLR 等底层证据未体现。

### PROC（进程管理/任务调度）

- **证据锚点**：
  - `kernel/src/syscall/process/sys/clone_exec.rs#sys_clone`
  - `kernel/src/syscall/process/sys/clone_exec.rs#sys_execve`
  - `kernel/src/syscall/process/sys/wait.rs#sys_wait4`
  - `kernel/src/syscall/process/thread_group.rs#reap_unreachable_zombies`
  - `kernel/src/syscall/process/signal/delivery.rs#deliver_current_sync_signal_handler`
  - `kernel/src/syscall/process/futex_table.rs#queue_mut`
- **说明**：支持 POSIX 风格的进程/线程组模型，包括 `clone`、`execve`、`wait4`、僵尸进程回收、PID 表、信号投递与处理、`setitimer`、futex 等待队列、进程凭证（`setuid/setgid/setresuid` 等）以及 `sched_getaffinity/setscheduler`。`USER_PROCESSES` 作为全局进程表被频繁加锁访问。
- **调度层面**：存在 `UserScheduler`、任务睡眠/唤醒、`clock_nanosleep` 等轻量调度机制，但 **未见 CFS、SMP、多核负载均衡等完整调度器底层证据**。

### FS（文件系统）

- **证据锚点**：
  - `kernel/src/syscall/fs/path.rs#sys_openat`
  - `kernel/src/fs/vfs/dentry.rs#resolve_child_uncached`
  - `kernel/src/fs/vfs/dentry.rs#validate_ramfs_child_target`
  - `kernel/src/fs/ramfs/io.rs#getdents`
  - `kernel/src/fs/ext4/mod.rs#mount_root`
  - `kernel/src/fs/ext4/mod.rs#read_inode`
  - `kernel/src/fs/procfs/dir.rs#lookup_child`
- **说明**：实现了 VFS 层（dentry、inode、fd_table、lookup、mount）、RAMFS 内存文件系统、EXT4 只读/解析路径（超级块、inode、目录项读取）、PROCFS 动态节点（`/proc/pid/stat`、`/proc/self/fd` 等）以及 socket 文件抽象。`fd_table` 支持线程组内共享与同步。
- **未体现**：EXT4 写入、日志、元数据更新等底层证据未体现。

### DEV（设备驱动）

- **证据锚点**：
  - `kernel/src/drivers/virtio/blk.rs#init_first`
  - `kernel/src/drivers/virtio/blk.rs#read_sector`
  - `kernel/src/drivers/virtio/blk.rs#write_sector`
  - `kernel/src/drivers/bus/mod.rs#virtio_block_devices`
  - `kernel/src/drivers/block/mod.rs#primary`
- **说明**：支持基于 MMIO/PCI 的 virtio-blk 块设备初始化与读写，块层提供 `primary` 设备选择。这是目前驱动模块的主要证据。
- **未体现**：字符设备、TTY、串口、RTC、物理网卡、PCI 非 virtio 设备等底层证据未体现。

### SYSCALL（系统调用）

- **证据锚点**：
  - `kernel/src/syscall/dispatch.rs#syscall_dispatch_with_context`
  - `kernel/src/syscall/dispatch.rs#syscall_enter`
- **说明**：存在一个大型系统调用分发表（覆盖 169 条分支），将 `fs`、`process`、`mm`、`select` 等子系统组织在 `kernel/src/syscall/` 下。参数与返回值通过 `TrapFrameAccess` 与 trap frame 交互。
- **未体现**：seccomp、ptrace、namespace、cgroup 等系统调用语义底层证据未体现。

### TRAP/IRQ（中断异常）

- **证据锚点**：
  - `kernel/src/arch/riscv64/trap.rs#riscv64_trap_handler`
  - `kernel/src/arch/loongarch64/trap.rs#loongarch64_trap_handler`
  - `kernel/src/arch/riscv64/trap.rs#init_trap_vector`
  - `kernel/src/arch/loongarch64/trap.rs#init_trap_vector`
  - `kernel/src/arch/loongarch64/timer.rs#enable_timer_interrupt`
- **说明**：按架构实现了 trap vector 初始化、时钟中断使能、用户态异常入口，统一处理系统调用、缺页异常、时钟中断与信号返回。
- **未体现**：多核 IPI、中断亲和性、设备中断路由等底层证据未体现。

### NET（网络协议栈）

- **证据锚点**：
  - `kernel/src/net/mod.rs#new`
  - `kernel/src/net/mod.rs#udp_send`
  - `kernel/src/net/mod.rs#udp_recv`
  - `kernel/src/net/mod.rs#tcp_listen`
  - `kernel/src/net/loopback.rs#consume`
- **说明**：仅体现一个轻量的 socket 封装层，基于 loopback 设备提供 UDP/TCP socket 的创建、绑定、发送、接收与监听接口。函数数量仅 18 个，厚度较薄。
- **未体现**：物理网卡驱动、ARP、IP 路由、完整 TCP 状态机、拥塞控制等底层证据未体现；该层大概率封装了第三方网络栈 crate。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块/特性

1. **完整网络协议栈**：仅见 `kernel/src/net/mod.rs` 对 UDP/TCP socket 的薄封装与 loopback 设备，未见物理网卡驱动、ARP、IP 路由、TCP 完整状态机、拥塞控制等底层实现。
2. **高级调度器**：未见 CFS、SMP、多核调度、CPU 负载均衡、调度组等证据。
3. **EXT4 写路径**：`ext4` 模块证据集中在挂载、inode 与目录项读取，未见写入、日志、元数据更新底层实现。
4. **字符设备/TTY/串口/RTC 驱动**：测试用例中提及 `/dev/rtc`，但内核驱动层未见相关底层代码。
5. **安全与隔离机制**：未见 MAC、capabilities 完整实现（仅有少量 credential 快照）、cgroups、namespaces、seccomp 等证据。
6. **虚拟化、电源管理、ACPI、多核启动（SMP）** 等底层证据未体现。
7. **第三方/公共依赖**：`kernel/src/fs/ext4/` 与 `kernel/src/net/` 可能封装了第三方 crate；`tools/user-smoke*.rs` 为测试程序，不属于内核主体。
