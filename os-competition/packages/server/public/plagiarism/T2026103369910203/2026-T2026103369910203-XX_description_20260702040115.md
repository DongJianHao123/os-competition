## 第四部分：分模块架构说明

### MM（内存管理）

底层证据较强。存在独立的物理页框分配器、虚拟地址抽象、用户地址空间构建与 VMA 管理。

- `lib/mm/src/frame.rs#init_frame_allocator`：基于 `FRAME_ALLOCATOR` 与 `PhysAddr::page_number`，利用 `RAM_END` 与 `kernel_end_phys()` 初始化可用物理页框。
- `lib/mm/src/address.rs`：提供 `PhysAddr`、`VirtAddr` 及 `page_number()` 等地址运算抽象。
- `kernel/src/vm/vm_area.rs`：VMA（虚拟内存区域）管理。
- `kernel/src/vm/elf.rs#init_stack`：为 ELF 程序构造用户栈，写入 `AuxHeader`。
- `kernel/src/task/taskf.rs#execve`：调用 `AddrSpace::build_user` → `load_elf` → `map_stack` / `map_heap` 构建新进程地址空间。
- `kernel/src/vm/user_ptr.rs#read_c_string`：用户态指针安全读取。

### PROC（进程管理/任务调度）

传统进程管理底层证据充分，不是单纯的轻量任务/channel。

- `kernel/src/syscall/process.rs#sys_execve`：ELF 可执行文件加载入口，调用 `Path::resolve_symlink_through` 与地址空间构建。
- `kernel/src/syscall/process.rs#sys_clone3` / `__sys_clone`：支持 `CloneFlags`、pidfd、子进程创建。
- `kernel/src/task/taskf.rs#fork`：复制 PCB、父子关系、fdtable、地址空间等。
- `kernel/src/task/taskf.rs#exit`：进程退出，清理 futex、共享内存、通知父进程。
- `kernel/src/syscall/process.rs#sys_wait4` / `sys_waitid`：依赖 `PROCESS_GROUP_MANAGER` 与 `wait_queue` 实现等待子进程/进程组。
- `kernel/src/task/process_manager.rs#add_process` / `remove`：进程组管理。
- `kernel/src/task/signal/sig_exec.rs#kill/stop/cont`：信号投递与线程组处理。

### FS（文件系统）

文件系统是本项目代码量最大的模块，证据最强。

- VFS 层：`lib/vfs/src/dentry.rs#lookup/rename`、`lib/vfs/src/path.rs#walk_recursive`、`lib/vfs/src/file.rs#read_dir`。
- 系统调用层：`kernel/src/syscall/fs.rs#sys_openat`、`sys_mount`、`sys_readv/writev/splice/readlinkat` 等。
- 挂载管理：`kernel/src/syscall/fsmount.rs#sys_fsmount/sys_move_mount/sys_fsopen/sys_open_tree`，通过 `BLOCK_DEVICE.get` 与 `FS_MANAGER` 选择文件系统类型。
- 具体文件系统：
  - `lib/osfs/src/lib.rs#init`：初始化根文件系统、procfs、devfs。
  - `lib/ext4/src/fs.rs#base_mount`、`lib/ext4/src/dentry.rs#base_lookup/base_rename`：ext4 文件系统封装（调用 `ext4_fopen`、`ExtDir` 等，疑似对 C 库/ext4 的 Rust 包装）。
  - `lib/fat32/src/dentry.rs#base_lookup/base_rename`：FAT32 支持。
- 特殊文件：`lib/osfs/src/proc/`（procfs）、`lib/osfs/src/dev/loopx/`（loop 设备）、`lib/osfs/src/special/fscontext/`（文件系统上下文）、`lib/osfs/src/special/io_uring/`。

### DEV（设备驱动）

设备驱动证据较强，覆盖块设备、串口、网络设备。

- `kernel/src/osdriver/probe.rs#probe_tree`：设备树遍历，初始化 `BLOCK_DEVICE`、`CHAR_DEVICE`，识别 virtio、dw_mshc、serial。
- `kernel/src/osdriver/probe.rs#probe_sd_mmc_devices`：SD/MMC 设备探测。
- `kernel/src/osdriver/mod.rs#probe_serial_console` / `probe_char_device_by_serial`：串口控制台初始化。
- `lib/driver/src/block/dw_mshc/mmc.rs#card_init` / `read`：SD/MMC 主机控制器命令序列与块读写。
- `lib/driver/src/block/dw_mshc/registers.rs`：寄存器定义（43 个函数，硬件协议强约束代码）。
- `lib/driver/src/net/virtnet.rs#receive`：VirtIO 网卡接收路径。

### SYSCALL（系统调用）

系统调用入口集中，业务实现分散。

- `kernel/src/syscall/mod.rs#syscall`：统一分发器，调用数达 196，覆盖文件、进程、信号、调度、BPF、fanotify/inotify、mount 等。
- `kernel/src/syscall/fs.rs`：文件相关 syscall 最密集（66 个函数）。
- `kernel/src/syscall/process.rs`：进程/资源/能力相关 syscall。
- `kernel/src/syscall/signal.rs`：信号与 pidfd。
- `kernel/src/syscall/fsmount.rs`：新挂载 API。
- `kernel/src/syscall/bpf.rs#sys_bpf`：BPF 子系统入口。

### TRAP/IRQ（中断异常）

该模块函数数量最少（43），但关键路径存在。

- `kernel/src/trap/trap_handler/user_trap_handler/riscv64.rs#trap_handler`：用户态陷入，解析 `scause`，区分 `Exception`/`Interrupt`，调用 `TIMER_MANAGER.check`。
- `kernel/src/trap/trap_handler/user_trap_handler/loongarch64.rs#trap_handler`：LoongArch64 用户态陷入，读取 `estat`、`era`、`badv`，调用 `TIMER_MANAGER.check`。
- `kernel/src/trap/trap_handler/kernel_trap_handler/riscv64.rs#interrupt_handler`：内核态中断，调用 `device_manager().handle_irq` 与 `TIMER_MANAGER.check`。
- `kernel/src/trap/trap_handler/kernel_trap_handler/loongarch64.rs#kernel_interrupt_handler`：LoongArch64 内核态中断处理。

### NET（网络协议栈）

网络模块代码量不大但路径完整。

- `lib/net/src/lib.rs#init_network`：初始化 `ETH0`、IP、网关、回环地址。
- `lib/net/src/tcp/tcpsr.rs#send` / `recv`：TCP 发送/接收，调用 `SOCKET_SET.with_socket_mut` 与 `block_on`。
- `lib/net/src/tcp/listentable.rs#accept` / `check_after_poll`：监听表、SYN 队列、连接建立。
- `lib/net/src/tcp/mod.rs#snoop_tcp_packet`：TCP 包嗅探。
- `lib/net/src/udp.rs#send_to` / `block_on`：UDP 发送。
- `lib/net/src/interface.rs#check_poll`：轮询网络接口并管理定时器。

**第三方/外部依赖说明**：`lib/net` 中 `SOCKET_SET`、`tcp::Socket`、`udp::Socket`、`poll_interfaces` 等命名与使用方式高度符合 `smoltcp` 等第三方 TCP/IP 栈的封装模式，应视为对第三方网络栈的集成而非完全自研协议实现。
