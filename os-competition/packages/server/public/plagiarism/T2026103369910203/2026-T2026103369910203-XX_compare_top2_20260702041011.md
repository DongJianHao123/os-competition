### 降权项说明
以下证据类型在本案中**降权处理**，不作为强同源证据：

1. **VFS trait 标准方法名**（`get_attr`、`base_open`、`base_create`、`base_lookup`、`base_mount`）：这些函数名由 VFS trait 接口强制统一，任何 Rust VFS 实现必须使用相同名称。跨不同 procfs 节点（如 `exe/inode.rs` vs `meminfo.rs`）的同名匹配属于硬凑。

2. **POSIX/Linux 系统调用名**（`sys_execve`、`sys_clone`、`sys_getppid`、`sys_exit_group`、`sys_set_tid_address`、`sys_rt_sigtimedwait`）：这些名称由 Linux ABI 规范强制规定，任何 Linux 兼容内核必须使用。

3. **Linux procfs 固定格式字符串**（`/proc/meminfo` 的 `MemTotal`、`MemFree` 等）：字段名与顺序由 Linux 内核文档固定，任何兼容实现必然一致。

4. **Rust 语言惯用法**（`Arc::new`、`Mutex::lock()`、`iter().map().reduce().unwrap()`、`vec!`/`Vec::new`、`Ok`/`Err`/`Some`）：这些是 Rust 标准库的基础 API 调用，不具备独创性。

5. **硬件 datasheet 强制寄存器操作**（UART 8250 偏移、DW-MSHC MMC 命令字）：寄存器地址与操作序列由芯片手册固定，独立驱动必然趋同。

6. **空实现函数**（多处 `base_create`/`base_lookup` 仅返回 `Err`）：不支持创建/查找的常规空实现，无比对价值。

7. **跨板块同名匹配**：控方将 `get_attr`（`exe/fd/fdinfo` vs `meminfo/mounts/self_`）、`base_open`（`interrupts/meminfo/fdinfo` vs `meminfo/mounts/self_`）等不同 procfs 节点的同名 trait 方法强行配对，属于跨板块硬凑，不构成同板块主证据。
