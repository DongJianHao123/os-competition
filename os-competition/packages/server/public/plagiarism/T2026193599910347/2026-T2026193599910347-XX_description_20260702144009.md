## 第四部分：分模块架构说明

### MM（内存管理）

底层证据显示项目实现了基于 RISC-V SV39 的内存管理：
- 物理页框分配：`modules/axruntime/src/mm.rs#init_frame_allocator` 使用 `BumpFrameAllocator` 初始化 `FRAME_BASE`、`FRAME_COUNT`。
- 页表操作：`clone_user_root` 实现进程 fork/clone 时的页表复制并标记 COW；`walk_page`、`unmap_user_page` 用于页表遍历与解除映射。
- 缺页处理：`modules/axruntime/src/trap.rs#trap_handler` 调用 `mm::handle_cow_fault`。
- 用户地址安全访问：`UserPtr<T>` / `UserSlice` 在 `current_root_pa` 下做用户态地址翻译与读写校验。

未体现：swap、按需分页（除 COW 外）、slab/伙伴分配器等高级内存策略。

### PROC（进程管理/任务调度）

进程管理有明确证据，不是单纯的轻量任务：
- 进程表：`PROC_PPID`、`PROC_CLEARTID`、`PROC_ROOT`、`PROC_CWD` 等 per-PID 数组。
- 生命周期：`sys_clone`、`sys_execve`、`sys_exit`、`sys_wait4` → `process::waitpid`。
- Futex：`modules/axruntime/src/futex.rs#wait` / `wake` 支持线程同步原语。
- FD 表：每个进程独立的文件描述符表，支持 `close_cloexec_fds`、`dup`、`fcntl` 等。

任务调度/异步运行时：
- 抢占式 tick 调度：`runtime.rs#on_tick` 维护 `RUN_QUEUE`、`SLEEP_QUEUE`，设置 `NEED_RESCHED`。
- 上下文切换：`schedule_once`、`preempt_current`、`yield_now` 调用 `crate::scheduler::switch` 与 `mm::switch_root`。
- 异步执行器：`modules/axruntime/src/async_exec.rs#spawn`、`poll`、`waker_wake` 提供基于 `RawWaker` 的 async runtime。

### FS（文件系统）

文件系统是项目最重的模块：
- VFS 层：`modules/axfs/src/mount.rs` 的 `MountTable` / `MountPoint` 管理多挂载点。
- 后端：`memfs`、`devfs`、`procfs` 在 `axfs` 内实现；`ext4.rs`、`fat32.rs` 属于“文件系统/第三方库”分类，可能是第三方库适配或测试封装。
- 系统调用层：`sys_openat`、`sys_pread64`、`sys_pwrite64`、`sys_newfstatat`、`sys_readv`、`sys_writev` 通过 `mounts.fs_for`、`resolve_fd`、`read_vfs_at` 访问 VFS。

### DEV（设备驱动）

仅发现 VirtIO-MMIO 两类驱动：
- 网卡：`modules/axruntime/src/virtio_net.rs#try_init_device`、`recv`、`send`、`handle_irq`、`setup_queue`。
- 块设备：`modules/axruntime/src/virtio_blk.rs#try_init_device`、`submit_request`、`handle_irq`。

未体现：PCIe、USB、NVMe、SATA、显示、音频、键盘等驱动。

### SYSCALL（系统调用）

`modules/axruntime/src/syscall.rs` 是系统调用中枢：
- `dispatch` 函数规模约 9491，调用 119 个具体 `sys_*` 函数。
- 覆盖文件、进程、网络、内存、时间、信号掩码、`sched_getaffinity`/`setaffinity`、`ioctl`、`sysinfo` 等。
- 所有涉及用户态指针的调用均通过 `UserPtr` / `UserSlice` 做地址校验。

### TRAP/IRQ（中断异常）

RISC-V S-mode 陷阱处理：
- `modules/axruntime/src/trap.rs#trap_handler` 处理 timer、external（PLIC）、exception、ecall。
- `enable_timer_interrupt`、`enable_external_interrupts`、`enable_interrupts`、`init` 配置 `stvec`、`sscratch`、`sie`、`sstatus`。
- `runtime.rs#resume_user_from_trap` 负责异常返回用户态。

### NET（网络协议栈）

网络部分主要是第三方库 `smoltcp` 的封装：
- `modules/axnet/src/smoltcp_impl.rs#init`、`poll`、`socket_accept`、`socket_connect`、`socket_send`、`socket_recv`、`socket_poll`、`send_arp_probe`、`run_tcp_loopback`。
- `runtime.rs#idle_loop` 与 `on_tick` 中调用 `axnet::poll` 驱动网络。
- 用户态测试程序 `tcp_echo`、`udp_echo`、`net_bench` 通过 socket 系统调用验证。

未体现：原创 TCP/IP 协议栈；传输层/网络层/链路层核心算法均来自 `smoltcp`。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块/能力

- **除 VirtIO blk/net 外的设备驱动**：PCIe、USB、NVMe、SATA、显示、音频、键盘、串口（除 SBI 输出外）等未体现。
- **高级内存管理**：swap、按需分页（除 COW 外）、slab/伙伴分配器、大页、NUMA 等未体现。
- **SMP/多核调度**：`sys_sched_getaffinity` 返回单 CPU 掩码，多核调度、核间中断未体现。
- **安全与隔离机制**： capabilities、seccomp、ACL、审计、ASLR 等未体现。
- **完整信号机制**：仅 `sys_rt_sigprocmask` 有存根式证据，信号投递、信号处理栈未体现。
- **调试/追踪**：ptrace、perf、kprobe、堆栈回溯等未体现。
- **内核模块/动态加载**：未体现。
- **电源管理、热插拔、虚拟化**：未体现。

### 需要单独说明的第三方/约束代码

- `modules/axnet/src/smoltcp_impl.rs` 及 `网络协议栈/第三方库` 分类表明网络层大量依赖 `smoltcp`。
- `modules/axfs/src/ext4.rs`、`modules/axfs/src/fat32.rs` 及 `文件系统/第三方库` 分类表明 ext4/FAT32 可能是第三方库集成或测试适配层，不应直接视为原创内核文件系统主体。
