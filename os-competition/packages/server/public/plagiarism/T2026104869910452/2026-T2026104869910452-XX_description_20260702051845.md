## 第四部分：分模块架构说明

### MM（内存管理）
- **已实现**：物理页框分配器、内核/用户地址空间、ELF 加载栈初始化、用户指针校验。
- **证据**：
  - `lib/mm/src/frame.rs#init_frame_allocator`：初始化 `FRAME_ALLOCATOR`，按页号管理物理内存。
  - `kernel/src/vm/vm_area.rs`：VMA 相关管理。
  - `kernel/src/vm/elf.rs#init_stack`：构造 ELF 启动栈、AuxVector。
  - `kernel/src/vm/user_ptr.rs#read_c_string`：用户态字符串安全读取。
- **未明确体现**：mmap/munmap 完整语义、swap、page cache、COW（写时复制）细节、大页/NUMA 支持。

### PROC（进程管理/任务调度）
- **已实现**：传统 Unix 进程模型（fork/clone、execve、exit、wait、zombie、进程组、会话、信号、pidfd、futex）。
- **证据**：
  - `kernel/src/syscall/process.rs#sys_clone3` / `__sys_clone`：clone 标志解析、pidfd 创建。
  - `kernel/src/syscall/process.rs#sys_execve` / `kernel/src/task/taskf.rs#execve`：ELF 加载与地址空间重建。
  - `kernel/src/task/taskf.rs#exit`：资源回收、信号通知、进程移除。
  - `kernel/src/syscall/process.rs#sys_wait4` / `sys_waitid`：进程组等待。
  - `kernel/src/task/process_manager.rs`：进程组管理。
  - `kernel/src/task/futex.rs`：futex 等待队列。
- **调度方面**：`sys_sched_getaffinity` / `sys_sched_setaffinity` 与 `TIMER_MANAGER.check` 表明有定时器与亲和性支持，但**抢占式调度算法细节底层证据未充分体现**。

### FS（文件系统）
- **已实现**：VFS 层、路径解析、符号链接、重命名、挂载、procfs、devtmpfs/loop、fanotify/inotify、ext4、FAT32。
- **证据**：
  - `lib/vfs/src/dentry.rs#lookup` / `rename`：VFS 核心路径与重命名。
  - `lib/vfs/src/file.rs#read_dir`：目录读取。
  - `lib/osfs/src/proc/mod.rs#init_procfs`：procfs 初始化（cmdline、exe、maps、status、fd、fdinfo 等）。
  - `kernel/src/syscall/fs.rs#sys_mount` / `kernel/src/syscall/fsmount.rs#sys_fsmount`：mount/fsopen/fsmount/move_mount。
  - `lib/ext4/src/dentry.rs#base_lookup` / `base_rename`：ext4 后端。
  - `lib/fat32/src/dentry.rs#base_lookup` / `base_rename`：FAT32 后端。
- **第三方约束**：`lib/ext4` 直接调用 `ext4_fopen` 等 C 风格 API，属于对第三方 ext4 库的封装，不是自研完整 ext4。

### DEV（设备驱动）
- **已实现**：设备树（DTB）探测、串口控制台、DW MSHC MMC/SD、VirtIO 网络、loopback。
- **证据**：
  - `kernel/src/osdriver/probe.rs#probe_tree`：统一设备树探测入口。
  - `kernel/src/osdriver/probe.rs#probe_sd_mmc_devices`：MMC 设备匹配。
  - `lib/driver/src/block/dw_mshc/mmc.rs#card_init` / `read`：SD/MMC 卡初始化与块读写。
  - `kernel/src/osdriver/mod.rs#probe_serial_console`：串口控制台初始化。
  - `lib/driver/src/net/virtnet.rs#receive`：VirtIO 网卡接收。
- **未明确体现**：PCI 枚举、USB、GPU、音频、NVMe、SATA、DMA 统一框架。

### SYSCALL（系统调用）
- **已实现**：中央系统调用分发器，覆盖文件、进程、信号、调度、BPF、fanotify、mount、keyring 等。
- **证据**：
  - `kernel/src/syscall/mod.rs#syscall`：199 条调用链分发。
  - `kernel/src/syscall/fs.rs#sys_openat` / `sys_readv` / `sys_writev` / `sys_splice`。
  - `kernel/src/syscall/process.rs#sys_execve` / `sys_clone3` / `sys_wait4`。
  - `kernel/src/syscall/bpf.rs#sys_bpf`：BPF 程序/映射创建。
  - `kernel/src/syscall/fanotify.rs#sys_fanotify_init` / `sys_fanotify_mark`。
- **未明确体现**：seccomp、ptrace、io_uring 完整实现（虽有 `io_uring/inode.rs` 但仅见 NOP 处理）。

### TRAP/IRQ（中断异常）
- **已实现**：RISC-V64 与 LoongArch64 双架构用户态/内核态陷阱处理、定时器中断、异常分发。
- **证据**：
  - `kernel/src/trap/trap_handler/user_trap_handler/riscv64.rs#trap_handler`。
  - `kernel/src/trap/trap_handler/user_trap_handler/loongarch64.rs#trap_handler`。
  - `kernel/src/trap/trap_handler/kernel_trap_handler/riscv64.rs#kernel_trap_handler` / `interrupt_handler`。
  - `kernel/src/trap/trap_handler/kernel_trap_handler/loongarch64.rs#kernel_interrupt_handler`。
- **未明确体现**：中断线程化、IRQ 亲和性、APIC/PLIC/HTIF 等具体中断控制器完整驱动。

### NET（网络协议栈）
- **已实现**：TCP/UDP socket、监听表、接口轮询、定时器驱动的异步阻塞。
- **证据**：
  - `lib/net/src/tcp/tcpsr.rs#send` / `recv`。
  - `lib/net/src/udp.rs#send_to` / `block_on`。
  - `lib/net/src/tcp/listentable.rs#accept` / `check_after_poll`。
  - `lib/net/src/interface.rs#check_poll`。
- **说明**：`SOCKET_SET`、`tcp::Socket`、`poll_interfaces` 等 API 风格与 `smoltcp` 高度一致，**应视为对第三方网络栈的封装/适配**，而非自研完整 TCP/IP 协议栈。

- **io_uring 完整实现**：仅见 `IORING_OP_NOP` 处理，完整 SQ/CQ 异步 IO 证据不足。

- `lib/ext4` 直接调用 `ext4_fopen` 等 C 风格 API，属于对第三方 ext4 实现的封装，**不应视为自研完整 ext4 文件系统**。

- `lib/fat32` 可能是自有实现或基于现有 Rust FAT 库，需结合源码进一步确认原创边界。

### 3. 判断性质说明
- 本文档所有模块划分、实现范围与“可能方向”均基于 AST Call Map、函数名、路径锚点与调用链，**不是对作品最终身份、原创性或违规情况的判定**。
- “传统进程管理已实现”等结论基于 `fork/exec/wait/exit/PCB/进程组` 等底层证据；“异步运行时”基于 Future/Timer/epoll 相关证据，但**具体调度算法、抢占策略、SMP 负载均衡细节证据有限**，需源码进一步确认。
