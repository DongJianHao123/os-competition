### 4. 核心代码调用流对比剖析
* **【进程管理/系统调用路径】**
* **底层技术析理**: 以 `sys_getppid` 为例，两者虽为同名 POSIX 系统调用，但底层实现模型截然不同。目标项目基于 RT-Thread 的 **LWP（轻量级进程）** 抽象，通过 `lwp_self` 获取当前 LWP 结构体，再调用 `lwp_to_pid` 进行 ID 映射，全程无显式内核锁。参考项目则直接操作其自定义的 **proc 结构体**，通过 `acquire` 自旋锁保护临界区，调用 `curr_proc` 获取当前 CPU 的进程指针后直接读取父进程 ID，最后 `release` 解锁。两者在对象模型、同步策略上存在根本性架构差异。
