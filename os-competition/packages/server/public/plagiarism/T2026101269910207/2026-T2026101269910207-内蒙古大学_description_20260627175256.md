## 第四部分：分模块架构说明

### MM（内存管理）
底层证据主要体现为 **LWP 用户态内存映射与系统调用层内存拷贝**，而非独立的物理页分配器实现。
- `rt-thread/components/lwp/lwp_syscall.c#sys_open` / `sys_thread_create` / `sys_sched_getaffinity` 均调用 `kmem_get` / `kmem_put`，说明内核为 syscall 参数/结果分配临时内核缓冲区。
- `sys_thread_create` 调用 `lwp_map_user`，表明存在用户态虚拟地址空间映射机制。
- `rt-thread/components/dfs/dfs_v2/filesystems/procfs/proc_pid.c#stat_single_show` 调用 `rt_aspace_count_vsz`，说明存在地址空间（aspace）抽象。
- `rt-thread/libcpu/risc-v/common64/trap.c#handle_trap` 调用 `handle_user`、`illegal_inst_recoverable`，可能涉及用户态访存异常处理。
- **未直接体现**：物理页框分配器、页表遍历、TLB 刷新、swap 等底层细节在本次 Call Map 中证据不足。

### PROC（进程管理/任务调度）
证据较强，分为 RT-Thread 原生线程调度和 LWP 进程两层。
- 原生调度：`rt-thread/src/components.c#rtthread_startup` 初始化并启动调度器；`rt-thread/src/thread.c#_thread_init` 完成线程栈与调度上下文初始化；`rt-thread/src/scheduler_mp.c#rt_exit_critical` 触发调度；`rt-thread/src/signal.c#rt_thread_handle_sig` 处理线程信号。
- LWP 进程：`rt-thread/components/lwp/lwp_pid.c` 提供 `lwp_waitpid`、`_wait_sibling_exit`、`_thread_exit`、`lwp_terminate`；`lwp_signal.c` 提供 `lwp_thread_signal_timedwait`；`lwp_futex.c` 提供 `_futex_wait`；`lwp_session.c` / `lwp_pgrp.c` / `lwp_jobctrl.c` 提供 session、process group、job control；`terminal/freebsd/tty.c` 提供终端控制；`procfs` 暴露 `/proc` 接口。
- 动态加载：`rt-thread/components/libc/posix/libdl/dlmodule.c#dlmodule_exec` / `dlmodule_exec_custom` 加载并启动动态模块。
- **注意**：没有证据表明存在完整的 POSIX fork/exec/wait/zombie 语义；LWP 是 RT-Thread 的轻量级进程扩展。

### FS（文件系统）
证据强，但大量为第三方/移植代码。
- `rt-thread/components/dfs/` 同时存在 `dfs_v1` 与 `dfs_v2` 两套实现。
- 后端包括 NFS（`nfs_xdr.c`）、FAT/elmfat（`ff.c` v1/v2）、procfs（`proc_pid.c`、`procfs.c`）。
- `machines/qemu-virt-riscv64/applications/judge_ext4_scan.c` 实现了一个 **raw ext4 扫描工具**，可读取 superblock、inode、extent、directory block，但不是可挂载的 ext4 文件系统驱动。
- `rt-thread/components/lwp/lwp_syscall.c#sys_mount` 调用 `dfs_mount` / `dfs_remount`，说明 syscall 层支持挂载操作。

### DEV（设备驱动）
证据最强，覆盖范围极广。
- **USB**：`rt-thread/components/drivers/usb/cherryusb/port/` 下包含 AIC、MUSB、HPMicro、Bouffalolab、FSDev、ChipIdea 等多个 USB Device 控制器移植；`USBD_IRQHandler` 系列为中断入口。
- **SDIO/SDHCI**：`dev_sdhci.c` 提供 SDHCI host 初始化与 I/O 设置。
- **WLAN**：`dev_wlan.c` / `dev_wlan_mgnt.c` 提供无线设备控制与连接管理。
- **传感器/图形/PCI/电源域/OFW**：`sensor_cmd.c`、`_graphic_control`、PCI procfs、`pm-domain-scmi.c`、`rt_ofw_console_setup` 等。
- 大量驱动代码属于 **CherryUSB 等第三方栈的移植与板级适配**，不应视为完全原创的内核主体。

### SYSCALL（系统调用）
证据集中。
- `rt-thread/components/lwp/lwp_syscall.c` 一个文件即含 219 个函数，是系统调用实现主体。
- 典型锚点：`sys_open`、`sys_mount`、`sys_sched_getaffinity`、`sys_thread_create`。
- 调用链显示 syscall 内部通过 `kmem_get/put` 进行内核缓冲管理，通过 `lwp_user_accessable` / `lwp_get_from_user` 进行用户态数据校验/拷贝。

### TRAP/IRQ（中断异常）
证据强，且跨多架构。
- RISC-V：`rt-thread/libcpu/risc-v/common64/trap.c#handle_trap`、`rt-thread/libcpu/risc-v/rv64/trap.c#handle_trap`。
- AArch64：`rt-thread/libcpu/aarch64/common/trap.c#rt_hw_trap_exception`。
- TI DSP：`rt-thread/libcpu/ti-dsp/c6x/trap.c#rt_hw_process_exception`。
- 设备中断：`USBD_IRQHandler` 系列（USB）、`sdio_dw_probe` / `rt_sdhci_setup_host`（SDIO）。

### NET（网络协议栈）
**底层证据未体现原创网络协议栈**。
- 统计中“网络协议栈”仅 4 个函数，全部来自测试桩 `iperf_compat.c` / `loongarch_basic_stub.c` 的 `iperf_tcp_body` / `iperf_udp_body`，仅做 `rt_kprintf` / `putstr` 输出。
- 实际网络功能由 **第三方 lwIP** 提供，代码中同时出现 lwIP 1.4.1、2.0.3、2.1.2 三个版本，位于 `rt-thread/components/net/lwip/`。lwIP 的函数被分析器分散归类到“任务调度/异步运行时”、“内存管理”、“系统调用”等模块，但它不是本项目原创内核模块。
