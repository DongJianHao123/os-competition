### 4. 核心代码调用流对比剖析
* **【进程管理/系统调用】**
* **底层技术析理**: 以`sys_exit`为例，两者虽为同名POSIX系统调用，但底层实现揭示了完全不同的进程模型。目标项目（RT-Thread）的`sys_exit`调用`lwp_thread_exit`，操作对象是`rt_thread`，属于其LWP（轻量级进程）模型下的线程退出。参考项目（xv6）的`sys_exit`调用`exit`，操作对象是`proc`结构体，涉及传统Unix进程的完整资源回收。两者的内存抽象、调度实体和同步策略截然不同。
* **代码溯源链接**:
  - 目标项目: `rt-thread/components/lwp/lwp_syscall.c#sys_exit`
  - 参考项目: `kernel/sysproc.c#sys_exit`

* **【第三方库/公共依赖】**
* **底层技术析理**: `nd6_process_autoconfig_prefix`是LwIP协议栈中处理IPv6无状态地址自动配置（SLAAC）前缀的标准函数。其调用流（`IP6_ADDR` -> `ip6_addr_assign_zone` -> `ip6_addr_isinvalid` -> ...）是LwIP源码的固定逻辑。目标项目使用`ip6_addr_cmp`，参考项目使用`ip6_addr_eq`，这是LwIP不同版本（2.0.x vs 2.1.x或更高）间的API演进，并非抄袭后的重命名掩饰。任何集成对应版本LwIP的项目都会呈现此差异。
* **代码溯源链接**:
  - 目标项目: `rt-thread/components/net/lwip/lwip-2.1.2/src/core/ipv6/nd6.c#nd6_process_autoconfig_prefix`
  - 参考项目: `kernel/lwip/core/ipv6/nd6.c#nd6_process_autoconfig_prefix`

### 5. 单方模块与降权项
- **单方模块**: 目标项目独有的`文件系统/第三方库`（如ext4扫描应用）和`第三方库`（如CherryUSB组件）模块，是目标项目独立开发的直接证据，参考项目无任何对应实现。
- **降权项**: 控方将大量LwIP网络函数（`tcp_process`, `sha1_process`等）错误归类为“进程管理”板块，制造了虚假的模块重合。这些函数应归入“第三方库/公共依赖”。此外，`atoi`, `strchr`等泛型C标准库函数，因功能单一、实现模式固定，不能作为同源抄袭的证据。

### 6. 保守结论
本次审计的客观AST相似度仅为8.44%，远低于构成抄袭嫌疑的阈值。所有强证据均指向双方项目共同集成的第三方开源库（LwIP/PolarSSL），这是嵌入式/操作系统开发中的正常实践。目标项目拥有超过一万个独立AST指纹，并包含大量参考项目不具备的原创模块（如完整的RT-Thread IPC、LWP、procfs、设备驱动框架），其独立开发的基础坚实。控方将第三方库代码错误归类为内核原创模块进行指控，在技术上不成立。综合判定，目标项目不存在对参考项目内核原创代码的系统性抄袭或违规复用行为。
