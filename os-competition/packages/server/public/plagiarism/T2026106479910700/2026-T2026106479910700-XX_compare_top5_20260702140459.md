#### 4. 核心代码调用流对比剖析
* **【进程管理/调度入口】**
* **底层技术析理**: 目标项目的 `sched` 函数调用流为 `holding -> intr_get -> mycpu -> myproc -> copytrapframe -> hsai_swtch -> panic`，而参考项目为 `holding -> intr_get -> mycpu -> mykthread -> swtch -> panic`。核心差异在于：1) 执行体抽象不同，目标项目直接操作 `myproc`（进程），参考项目操作 `mykthread`（内核线程）；2) 上下文切换机制不同，目标项目通过 HAL 层的 `hsai_swtch` 并伴有 `copytrapframe` 操作，显示其支持多架构和更复杂的上下文保存，而参考项目直接调用架构相关的 `swtch`。这体现了进程模型与线程模型、以及硬件抽象层设计的根本性架构差异。
* **代码溯源链接**:
  - 目标项目: `kernel/process.c#sched`
