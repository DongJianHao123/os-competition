## 第四部分：分模块架构说明

### MM（内存管理）

底层证据较强。项目在两个架构目录中均实现了地址空间管理：

- `os/src/mm/memory_set.rs#from_elf_at_base`：按 ELF program header 创建 `MapArea`，建立虚拟页映射并拷贝数据。
- `os-la/src/mm/memory_set.rs#munmap`：按 `VirtAddr` 区间解除映射并刷新 TLB。
- `os-la/src/mm/memory_set.rs#from_existed_user`：克隆 `data_frames`，用于 fork 后的写时复制/内存共享。
- `os-la/src/mm/memory_set.rs#split_area_by_range`：对已有映射区域按范围拆分，支持 `mprotect`/`munmap` 类操作。
- 页异常处理：`os/src/trap/mod.rs#trap_handler` 与 `os-la-patch/polyhal-trap/src/trap/loongarch64.rs#loongarch64_trap_handler` 均处理 `LoadPageFault`/`StorePageFault`/`InstructionPageFault`，后者还包含非对齐访存模拟 `emulate_load_store_insn`。

### PROC（进程管理/任务调度）

底层证据中等偏强，体现的是传统进程模型而非异步运行时：

- 进程/线程控制块：`TaskControlBlock` / `current_task` / `PID2PCB` / `PID2TCB`。
- 生命周期：`sys_thread_create`、`sys_clone`、`sys_exec`、`sys_waitpid`/`sys_wait4`、`sys_exit`/`sys_exit_group`。
- 调度循环：`os/src/task/processor.rs#run_tasks` → `fetch_task` → `__switch`。
- 同步原语：`futex_wait`、`condvar.wait`、robust list、`set_tid_address`。
- 资源限制：`sys_prlimit64`、`sys_setrlimit`、`sys_setitimer`。

**异步运行时底层证据未体现**：未发现 `waker`、`executor`、`Future` 调度或 async/await 运行时结构。

### FS（文件系统）

底层证据强，呈现“Rust 文件层 + C ext4 库”两层结构：

- Rust 层：`os-la/src/fs/inode.rs#open`、`os/src/fs/inode.rs#getdents`、`os-la/src/fs/pipe.rs#splice_to_pipe`、`os-la/src/fs/mod.rs#create_proc_dir`。
- 系统调用层：`sys_read`/`sys_write`/`sys_pread64`/`sys_pwrite64`/`sys_openat`/`sys_mount` 等。
- C 库层：`lwext4_rust/c/lwext4/src/ext4.c`、`ext4_extent.c`、`ext4_journal.c`、`ext4_balloc.c`、`ext4_dir_idx.c` 负责 ext4 块分配、extent、日志、目录索引。
- 挂载与根文件系统：`ROOT_SB`、`MNT_SB`、`do_mount`。

### DEV（设备驱动）

底层证据强，但大量代码位于补丁/第三方驱动框架中：

- 块设备：`os-la/src/drivers/virtio_blk.rs#new` 通过 PCI/MMIO 初始化 `VirtIOBlk`。
- 声音设备：`os-la-patch/virtio-drivers/src/device/sound.rs#play` / `pcm_xfer`。
- 控制台设备：`os-la-patch/virtio-drivers/src/device/console.rs#send`。
- Socket 设备：`os-la-patch/virtio-drivers/src/device/socket/connectionmanager.rs#poll` / `send_recv`。
- 架构抽象：`polyhal` 提供 x86_64 IDT/APIC、`loongarch64` trap 入口、RISC-V 寄存器读写等。

### SYSCALL（系统调用）

底层证据强：

- 两个大型分发器：`os-la/src/syscall/mod.rs#syscall`（11800 规模）与 `os/src/syscall/mod.rs#syscall`（12622 规模）。
- 文件类：`sys_read`/`sys_write`/`sys_pread64`/`sys_pwrite64`/`sys_readv`/`sys_writev`/`sys_openat`/`sys_mount`。
