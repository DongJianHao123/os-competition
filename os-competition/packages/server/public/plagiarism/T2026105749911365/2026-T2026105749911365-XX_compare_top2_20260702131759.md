* **代码溯源链接**:
  - 目标项目: `kernel/sysfile.c#sys_mmap`

  - **公共接口/泛型函数名**: `printf`, `memset`, `memmove`, `acquire`, `release` 等属于标准 C 库或操作系统内核中最通用的接口，其调用图谱的相似性不具备鉴别力，不能作为同源证据。

### 6. 保守结论
综合全模块证据，目标项目与参考项目的整体 AST 相似度仅为 10.5%，远低于可判定为“抄袭”或“高度复用”的阈值。双方共有的 50 个 AST 指纹几乎全部落在 xv6 教学内核所定义的公共原语、RISC-V 架构强制规范及标准 C 库接口范畴内，属于独立开发过程中的必然重合。在进程管理、内存管理、文件系统等核心板块，双方展现出显著的架构级差异，特别是目标项目独立集成的 ext4 文件系统和一系列 Linux 兼容系统调用，以及参考项目独有的信号和线程机制，共同排除了“整体搬运”或“底层逻辑完全复用”的可能性。现有证据充分支持“双方基于共同公开祖先（xv6）进行独立演化”的结论，整体同源风险极低。
