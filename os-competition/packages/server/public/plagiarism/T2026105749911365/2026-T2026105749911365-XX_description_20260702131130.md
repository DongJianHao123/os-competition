## 第四部分：分模块架构说明

### MM（内存管理）

底层证据较强。核心页表机制位于 `kernel/vm.c`：

- `walk()` 实现三级页表遍历与中间页分配，调用 `PA2PTE`、`PTE2PA`、`PX`、`kalloc`、`memset`。
- `mappages()` / `uvmunmap()` / `uvmcopy()` / `uvmalloc()` / `freewalk()` 覆盖映射、解除、复制、分配、释放。
- 物理内存分配器在 `kernel/kalloc.c`：`kfree()` 调用 `acquire`、`memset`、`panic`、`release`。
- 程序加载与地址空间重建在 `kernel/exec.c#kexec`，调用 `PGROUNDUP`、`kalloc`、`kfree`、`loadseg`、`copyout`、`flags2perm`。
- 用户态内存请求：`sys_brk`（`kernel/sysproc.c`）、`sys_mmap`（`kernel/sysfile.c`）调用 `growproc`、`kalloc`、`mappages`。
- 缺页/拷贝：`vmfault()` 被 `copyin`/`copyout` 调用。

### PROC（进程管理/任务调度）

底层证据较强，属于**传统进程管理**，而非独立异步运行时：

- PCB 与进程表：`kernel/proc.c#allocproc`、`freeproc`、`procinit`。
- 生命周期：`kfork_ex`（fork）、`kexec`（exec）、`kwait`（wait）、`kexit`（exit）、`reparent`（托孤）、`killed`。
- 调度：`scheduler()` 使用 `swtch` 进行上下文切换；`yield`、`sleep`、`wakeup` 实现阻塞与唤醒。
- 系统调用入口：`sys_fork`、`sys_exit`、`sys_wait`、`sys_wait4`、`sys_kill`、`sys_brk`、`sys_pause` 等位于 `kernel/sysproc.c`。

**独立的协程/future/waker/异步运行时底层证据未体现**；现有“任务调度/异步运行时”分类下的函数多为传统调度、锁、中断上下文。

### FS（文件系统）

底层证据强，呈现“日志型文件系统 + ext4 兼容层”结构：

- 核心 inode/路径/目录操作：`kernel/fs.c#namex`、`dirlink`、`dirlookup`、`ialloc`、`iput`、`ireclaim`、`readsb`、`writei`。
- 块映射：`kernel/fs.c#bmap` 调用 `ext4_bmap` / `ext4_bmap_alloc` / `balloc` / `log_write`。
- 日志与事务：`kernel/log.c#initlog`、`write_log`、`install_trans`、`write_head`。
- 系统调用：`kernel/sysfile.c#sys_open`、`sys_openat`、`sys_write`、`sys_exec`、`sys_link`、`sys_unlink`、`sys_unlinkat`、`sys_chdir`、`sys_pipe`、`sys_mmap`。
- 镜像构建工具：`mkfs/mkfs.c#main`、`ialloc`、`balloc`、`iappend` 用于在宿主机生成磁盘镜像。

### DEV（设备驱动）

底层证据中等，覆盖基本字符与块设备：

- **UART**：`kernel/uart.c#uartinit`、`uartwrite`、`uartputc_sync`、`uartintr`。
- **Console**：`kernel/console.c#consoleinit`、`consoleread`、`consolewrite`、`consoleintr`。
- **Virtio 块设备**：`kernel/virtio_disk.c#virtio_disk_init`、`virtio_disk_rw`、`virtio_disk_intr`。
- **PLIC**：`kernel/plic.c#plicinit`、`plicinithart`、`plic_claim`、`plic_complete`。
- 块缓冲层：`kernel/bio.c#bread`、`bwrite`、`bget`、`brelse` 最终落到 `virtio_disk_rw`。

### SYSCALL（系统调用）

底层证据在 `kernel/syscall.c` 中较薄：

- 仅 `argint`、`argaddr`、`argstr` 三个辅助函数被明确归入该模块。
- 系统调用分发由 `kernel/syscall.c#syscall` 完成（在统计中被划入“任务调度/异步运行时”），通过 `syscalls[num]` 表跳转。
- 实际处理函数大量分布在 `kernel/sysproc.c` 与 `kernel/sysfile.c`。

### TRAP/IRQ（中断异常）

底层证据中等，RISC-V 特征明显：

- 初始化：`kernel/trap.c#trapinit`、`trapinithart`。
- 异常入口：`kernel/trap.c#usertrap`、`kerneltrap`。
- 中断分发：`kernel/trap.c#devintr` 调用 `clockintr`、`uartintr`、`virtio_disk_intr`、`plic_claim`/`plic_complete`。
- 锁与中断屏蔽：`kernel/spinlock.c#push_off`、`pop_off` 调用 `intr_get`/`intr_off`/`intr_on`。

### NET（网络协议栈）

**底层证据未体现**。未出现 socket、TCP/IP、NIC 驱动、网络缓冲区等相关函数或目录。

## 第六部分：边界与未体现内容

1. **网络协议栈（NET）**：没有任何 socket、TCP/IP、网络设备驱动相关函数或目录，**底层证据未体现**。
2. **独立异步运行时**：统计中“任务调度/异步运行时”模块的函数实为传统进程调度、锁、中断上下文；**协程、future、waker、ring_fifo 等异步运行时机制底层证据未体现**。
3. **ext4 兼容代码**：`kernel/ext4_write.c` 及 `kernel/fs.c` 中的 `ext4_*` 函数大量涉及 `le16/le32` 字节序转换，明显属于 ext 文件系统协议/第三方移植代码，应视为**文件系统兼容层或外部依赖**，不宜直接等同于原创内核主体。
4. **mkfs 工具**：`mkfs/mkfs.c` 是宿主机磁盘镜像生成程序，不属于内核运行时。
5. **系统调用模块统计偏薄**：`kernel/syscall.c` 仅体现 `argint`/`argaddr`/`argstr` 三个参数解析函数，完整分发与处理函数分散在 `syscall()`、`sysproc.c`、`sysfile.c` 中。
6. **高级内存子系统**：未体现 swap、按需分页（除 `vmfault` 局部路径外）、NUMA、COW（写时复制）等明确证据。
7. **安全/加密/图形/音频**：均无底层证据。
8. **方向判断仅为可能**：本报告基于 AST Call Map、函数名、路径锚点与调用链进行技术推断，**不是对作品身份、原创性或合规性的最终结论**。
