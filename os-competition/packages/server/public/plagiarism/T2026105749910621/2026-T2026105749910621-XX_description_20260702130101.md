## 第四部分：分模块架构说明

### MM（内存管理）

- **证据锚点**：`kernel-riscv/kernel/vm.c`、`kernel-riscv/kernel/trap.c#usertrap`、`kernel-riscv/kernel/linux_syscall.c#linux_sys_mmap`、`kernel-riscv/kernel/exec.c#kexec`
- 实现了 RISC-V Sv39 页表管理、内核/用户地址空间切换（`MAKE_SATP`）、`copyin/copyout` 用户态拷贝
- 支持 Linux 兼容 `mmap`（`linux_mmap_alloc_vma`、`linux_mmap_map_pages`、`linux_mmap_pte_perm`）
- 物理页分配通过 `kalloc/kfree` 体现
- **未体现**：swap、按需分页（demand paging）、COW（copy-on-write）无明确底层证据

### PROC（进程管理/任务调度）

- **证据锚点**：`kernel-riscv/kernel/proc.c#kexit`、`kernel-riscv/kernel/proc.c#scheduler`、`kernel-riscv/kernel/proc.c#kfork`、`kernel-riscv/kernel/sysproc.c#sys_fork/exit/wait`、`kernel-riscv/kernel/linux_syscall.c#linux_sys_clone/wait4`
- 传统 Unix 进程模型：`fork`/`exit`/`wait`/`kill`、PCB（`struct proc`）、父子关系（`reparent`）、僵尸进程处理
- 调度为基于 `swtch` 的协作/抢占式循环，含 `sleep/wakeup/yield`
- **注意**：模块画像中的“异步运行时”仅来自统计标签，底层未体现 `async/await`、`task`、`waker` 等异步运行时结构

### FS（文件系统）

- **证据锚点**：`kernel-riscv/kernel/fs.c#namex`、`kernel-riscv/kernel/sysfile.c#sys_open/link/unlink/chdir`、`kernel-riscv/kernel/ext4_lite.c#ext4_lite_probe/lookup_path/read_inode_data`、`kernel-riscv/kernel/linux_syscall.c#linux_sys_openat/getdents64/newfstatat`
- 双层文件系统：
  - 底层：`xv6` 日志文件系统（`fs.c`、`sysfile.c`、`log.c`、`bio.c`）
  - 扩展层：`ext4_lite` 只读兼容 ext4，以及 `/proc`、`/tmp`、`/dev` 等虚拟文件（`linux_proc_materialize_file`、`linux_dev_virtual_file`、`linux_tmp_*`）
- `mkfs/mkfs.c` 用于构建初始磁盘镜像
- **第三方/移植依赖**：`ext4_lite.c` 属于 ext4 兼容子集，应单独说明，不宜夸大为主体原创文件系统

### DEV（设备驱动）

- **证据锚点**：`kernel-riscv/kernel/uart.c#uartinit/uartintr/uartwrite`、`kernel-riscv/kernel/virtio_disk.c#virtio_disk_init/intr`、`kernel-riscv/kernel/plic.c#plicinithart/claim/complete`、`kernel-riscv/kernel/console.c#consoleinit/consoleread/consolewrite`
- 控制台通过 UART 实现输入输出
- 块设备基于 VirtIO MMIO 磁盘
- 中断路由通过 RISC-V PLIC

### SYSCALL（系统调用）

- **证据锚点**：`kernel-riscv/kernel/syscall.c#syscall`、`kernel-riscv/kernel/linux_syscall.c#linux_syscall`、`kernel-riscv/kernel/linux_syscall.c#linux_sys_openat/read/write/execve/mmap/clone/wait4`
- 双表分发：
  - 原生 xv6 系统调用表 `syscalls[]`
  - Linux ABI 兼容表 `linux_syscalls[]`，覆盖文件、进程、内存、mount、time 等
- 参数解析：`argint/argstr/argraw` 与 `linux_argraw`

### TRAP/IRQ（中断异常）

- **证据锚点**：`kernel-riscv/kernel/trap.c#usertrap`、`kernel-riscv/kernel/trap.c#kerneltrap`、`kernel-riscv/kernel/trap.c#clockintr`、`kernel-riscv/kernel/spinlock.c#push_off/pop_off`
- RISC-V S-mode 异常/中断处理：ecall、page fault、timer interrupt、external interrupt
- 时钟中断驱动调度与 `wakeup`
- 自旋锁通过 `push_off/pop_off` 实现中断屏蔽嵌套

### NET（网络协议栈）

- **底层证据未体现**：无网络设备驱动、无 socket、TCP/IP、协议栈相关文件或函数证据
