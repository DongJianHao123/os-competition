## 第四部分：分模块架构说明

### MM（内存管理）

底层证据显示实现了：
- 物理页框分配器：`init_frame_allocator`、`dealloc_contiguous`（`kernel/src/mm/frame_allocator.rs`）
- 内核堆与 `kernel_phys_to_virt` 映射
- 地址空间 `MemorySet`、按 ELF 加载用户程序：`MemorySet::from_elf`（`kernel/src/mm/memory_set.rs`）
- 进程级 mmap/munmap：`mmap_current`（`kernel/src/task/manager_wrapper.rs`）
- fork 时复制地址空间：`MemorySet::from_existed_user`（`kernel/src/task/manager_wrapper.rs#fork_current`）
- 上下文切换时切换页表：`activate_page_table`（`kernel/src/task/manager_wrapper.rs#run_tasks`）

**未体现**：交换（swap）、写时复制（COW）、按需分页（demand paging）、共享内存、内存去重等高级机制。

### PROC（进程管理/任务调度）

底层证据显示实现了传统进程/线程模型：
- PCB/TCB、PID/TID 管理、父子进程关系：`try_waitpid`、`exit_current_and_run_next`、`reap_process`/`reap_thread`
- fork/exec/exit/wait/sleep/join：`fork_current`、`exec_current`、`waitpid_current`、`thread_join_current`
- 线程状态与就绪队列：`pick_ready_thread_in_process`、`count_ready_threads_in_process`
- Stride 调度：tickets、`stride_from_tickets`、`update_process_stride_by_alpha`、`pick_ready_process_by_stride`、`set_process_tickets_current`
- 定时器唤醒：`wake_sleeping_threads`
- 管道阻塞：`block_current_on_pipe_read` / `block_current_on_pipe_write`

**未体现**：完整的异步运行时（async/await、waker、future）、信号（signal）、POSIX 线程同步原语（semaphore/mutex 条件变量等）。

### FS（文件系统）

底层证据显示：
- 集成第三方 `fatfs` crate（`third_party/fatfs/src/*`），非原创实现
- 内核包装层：`kernel/src/fs/fatfs.rs#open`、`remove_recursive`、`kernel/src/fs/path.rs#normalize_path`
- EXT4 包装/ramdisk 入口：`fs::ext4fs::init`、`rootfs_ramdisk`、`test_ext4_magic`
- 块缓存：`kernel/src/block/cache.rs#get_block_cache`
- 块设备抽象：`kernel/src/block/blockio.rs#read` / `write`
- 文件描述符表：`new_fd_table`、`dup2`、`close`

**未体现**：原创 FAT/EXT4 文件系统核心（第三方库实现）、VFS 层完整抽象、硬链接/软链接、文件权限 ACL、日志型文件系统恢复机制。

### DEV（设备驱动）

底层证据显示：
- VirtIO Block MMIO 驱动：`kernel/src/block/virtio_blk.rs#read_sector_sync` / `write_sector_sync`
- VirtIO Block PCI 驱动：`kernel/src/block/virtio_pci_blk.rs#read_sector_sync` / `write_sector_sync`
- 设备发现：`discover_disks`（MMIO + PCI）
- VirtIO 队列与 DMA：`VirtioQueue`、`VirtioBlkDma`、`virtio_mmio.rs`
- PCI 基础扫描：`parse_virtio_pci_caps`、`enable_pci_device`、`ensure_mem_bar`

**未体现**：网络设备、显示/帧缓冲、字符设备（串口仅用于 console log）、USB、NVMe、AHCI 等。

### SYSCALL（系统调用）

底层证据显示：
- 内核分发器：`kernel/src/syscall/mod.rs#syscall`
- 进程类：`sys_fork`、`sys_exec`、`sys_waitpid`、`sys_exit`、`sys_mmap`、`sys_munmap`、`sys_yield`、`sys_sleep`、`sys_set_process_tickets` 等
- 文件类：`sys_open`、`sys_chdir`、`sys_close`、`sys_dup2`、`sys_getcwd`、`sys_getdents` 等
- 用户态封装库：`user/rust/ulib/src/io.rs`、`process.rs`、`syscall.rs`

**未体现**：完整的 POSIX 兼容层、socket 系列调用、信号相关调用、ptrace 等调试接口。

### TRAP/IRQ（中断异常）

底层证据显示：
- 双架构 trap 入口：`riscv_trap_handler`（`kernel/src/trap/riscv64/mod.rs`）、`loongarch_trap_handler`（`kernel/src/trap/loongarch64/mod.rs`）
- 系统调用转发：`handle_syscall`
- 定时器中断与抢占：`handle_interrupt` -> `timer::tick` -> `preempt_current_and_run_next`
- 中断开关：`disable_interrupt` / `enable_interrupt`

**未体现**：缺页异常（page fault）的完整处理链、非法指令/访问越界等异常的分发与信号化、中断下半部/软中断机制。

### NET（网络协议栈）

**底层证据未体现**。未在 AST Call Map 或目录结构中发现 TCP/IP、socket、网卡驱动、协议栈相关函数。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块/机制
- **网络协议栈（NET）**：无 socket、TCP/IP、网卡驱动相关代码
- **异步运行时**：无 `Future`、`Waker`、`async/await` 调度器证据，任务调度为传统 Stride 调度
- **高级内存管理**：无 swap、COW、demand paging、共享内存、大页
- **完整原创文件系统**：FAT 核心来自 `third_party/fatfs`；EXT4 仅为包装/ramdisk 入口
- **字符设备与多媒体**：无串口字符驱动以外的设备、无显示/音频/USB/NVMe
- **进程间通信**：仅发现管道阻塞原语，无消息队列、共享内存、信号量、信号
- **安全与权限**：无 ACL、capabilities、ASLR、栈保护等证据
- **SMP 调度细节**：多核启动（`HART_LOCALS`、`MASTER_READY`）有证据，但跨核调度、核亲和性、IPI 等细节未体现

### 需要单独说明的第三方/约束代码
- `third_party/fatfs/src/*` 为外部 FAT 文件系统库，不应视为该作品的原创内核主体
