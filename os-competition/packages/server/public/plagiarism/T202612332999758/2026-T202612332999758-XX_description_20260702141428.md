## 第四部分：分模块架构说明

### MM（内存管理）
底层证据集中在**用户虚拟地址空间**与**ELF 加载**。
- `os/src/mm/vm/uvm.rs#map_elf` 将 ELF 段映射到用户空间，使用 `PhysAddr`、`VmRegion`。
- `os/src/mm/vm/uvm.rs#load_dl_interp_if_needed` 负责动态链接器 `/lib/libc.so` 加载。
- `os/src/mm/user.rs` 提供 `UserMutPtr` / `UserMutSlice`，用于用户指针合法性校验。
- `os/src/syscall/mm.rs#sys_process_vm_readv/writev` 支持跨进程虚拟内存读写。
- `os/src/syscall/process.rs#sys_brk` 调整堆顶。

**物理页框分配器与页表实现**在提供的片段中未直接展开，仅通过 `meminfo` 等 procfs 接口间接反映。

### PROC（进程管理）
具备较完整的传统进程生命周期证据。
- `os/src/task/task.rs#fork` 创建新 PCB；`handle_zombie` 处理僵尸进程回收。
- `os/src/task/task.rs#exit_robust_list` 实现 futex robust list 清理。
- `os/src/syscall/process.rs#sys_clone/sys_fork` 提供 clone/fork 系统调用。
- `os/src/syscall/process.rs#sys_execve` 加载并执行新程序。
- `os/src/syscall/process.rs#wait_common/find_waitable_child` 实现 wait/waitpid。
- 信号、setuid/setgid、pidfd、进程组、会话等均有对应 syscall 证据。
- `os/src/fs/procfs/` 暴露 `/proc/[pid]/status/stat/comm/exe/fd` 等接口。

### 任务调度/异步运行时
不是简单的协程包装，而是与 SMP 结合的异步运行时。
- `os/src/executor/mod.rs#spawn` 使用 `async_task::spawn` 创建任务，并维护 `TASK_QUEUE`、per-CPU `PROCESSORS` 队列、RR/RT 入队逻辑。
- `os/src/executor/mod.rs#run_until_idle` 实现 work-stealing 与空闲检测。
- `os/src/processor/processor.rs#check_preempt` 在 IO/系统调用路径中触发 `yield_now`。
- `os/src/processor/processor.rs#pop_front` 处理本地队列、RT 队列、可窃取队列的优先级。
- `os/src/syscall/sche.rs#sys_sched_getaffinity/setaffinity` 提供 CPU 亲和性。

### FS（文件系统）
VFS + 多文件系统实例。
- `os/src/fs/vfs/dentry.rs#walk_with_depth` 实现带符号链接解析的路径遍历。
- `os/src/fs/fs.rs#open_or_create_file` 与 `DCACHE` 维护全局 dentry 缓存。
- `os/src/syscall/fs.rs#sys_mount/umount2` 处理挂载点替换与回滚。
- `os/src/syscall/fs.rs#sys_openat/read/write/pread/pwrite/getdents64/splice/fnctl` 覆盖常规文件 IO。
- `os/src/fs/devfs/mod.rs#init_devfs` 初始化 `/dev/null`、`/dev/rtc`、`/dev/loop*` 等。
- `os/src/fs/procfs/mod.rs#init_procfs` 初始化 `/proc` 伪文件系统。
- `os/src/fs/ext4_native/` 提供原生 ext4 读写、日志、inode/组描述符管理。

### DEV（设备驱动）
覆盖字符设备、块设备、网络设备。
- `os/src/devices/serial.rs#get_serial/scan_char_device`：基于 DTB 的 UART 串口初始化。
- `os/src/devices/manager.rs#init_net`：VirtIO-Net 设备初始化。
- `os/src/drivers/net/virtio_net.rs#recycle_rx_buffer`：VirtIO-Net 接收缓冲区回收。
- `os/src/drivers/block/mmio_blk.rs#read_block_async/write_block_async/read_block_wfi/write_block_wfi`：MMIO 块设备同步/异步读写。
- `os/src/drivers/block/mmc/mod.rs#card_init/send_cmd`：MMC/SD 卡控制器驱动。

### SYSCALL（系统调用）
统一入口 + 分类实现。
- `os/src/syscall/mod.rs#syscall` 为总入口，调用数达 245，负责参数解析与分发。
- `os/src/syscall/fs.rs`：文件与挂载相关调用（101 个函数）。
- `os/src/syscall/process.rs`：进程控制。
- `os/src/syscall/mm.rs`：内存相关。
- `os/src/syscall/net.rs`：网络与 socket 配置。
- `os/src/syscall/futex.rs`：futex 同步原语。
- `os/src/syscall/sche.rs`：调度接口。
- `os/src/syscall/signal.rs`：信号接口。

### TRAP/IRQ（中断异常）
HAL 抽象 + OS 层 handler。
- `hal/src/component/trap/loongarch64/mod.rs#save/restore`：LoongArch64 上下文保存/恢复。
- `hal/src/component/irq/loongarch64/mod.rs#from_dt/enable_irq/disable_irq`：LoongArch64 EIOINTC + PLATIC 中断控制器。
- `hal/src/component/irq/riscv64.rs#enable_irq`：RISC-V64 PLIC 中断使能。
- `os/src/trap/mod.rs#user_trap_handler/kernel_trap_handler`：OS 层统一 trap 处理，涉及定时器中断、设备中断、IPI、TLB shootdown。

### NET（网络协议栈）
自研/轻量级网络栈，覆盖链路层到传输层。
- `os/src/net/wire_util.rs#new`：IPv4/IPv6/UDP/TCP 包构造辅助。
- `os/src/net/arp/protocol.rs#ingress_arp_learn`：ARP 学习与解析。
- `os/src/net/udp/protocol.rs#parse_udp_ipv4`：UDP over IPv4 入包处理。
- `os/src/net/tcp/state_machine.rs#process_data_impl/process_ack`：TCP 状态机。
- `os/src/net/backend/tcp_ops.rs#tcp_connect/tcp_close/poll`：TCP 后端连接管理。
- `os/src/net/core_net.rs#process_pending_egress/process_udp_packet`：核心网络事件与出包。
- `os/src/net/socketpair.rs#poll`：UNIX domain socketpair。
- `os/src/net/crypto.rs#encode/check_alg`：AF_ALG 风格 netlink 加密接口解析。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块/能力
- **USB 子系统**：未出现 USB host/device 控制器、URB、HID、mass storage 等代码。
- **GPU / 显示 / 音频**：无 framebuffer、DRM、声卡驱动证据。

- **安全模块 / LSM**：无 SELinux、AppArmor、capabilities 完整实现（仅有 setuid/gid 等传统 POSIX 凭证）。
- **高级电源 / ACPI / 休眠**：仅定时器中断，无 CPU 频率、睡眠、ACPI 表解析证据。
