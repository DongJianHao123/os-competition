## 第四部分：分模块架构说明

### MM（内存管理）
- **已体现证据**：`memory_set` 在 `clone/exec/wait/affinity` 中频繁出现；用户态内存访问通过 `copy_to_user` / `copy_from_user` 完成；`exec_current_from_path_with_argv_env` 重建地址空间并应用重定位；`trap_handler` 在异常返回时调用 `drain_recycled_processes_if_pressure` 进行僵尸进程内存回收；`kernel_heap_stats` 暴露堆压力。
- **底层证据未体现**：提供的锚点未直接展示页帧分配器、页表遍历/MMU 映射、swap/缺页异常处理等最底层代码。

### PROC（进程管理/任务调度）
- **已体现证据**：存在全局 `PROCESS_MANAGER` 与 PCB；支持 `tgid`、父进程、进程组、会话、group leader；`do_clone` 实现线程组共享（`ThreadGroupShared`、`SharedMemorySet`、`SharedFsContext`）；`sys_execve` 支持解释器、shebang、TLS、动态链接；`cleanup_exiting_threads` 处理 fd 回收、pipe/socket 等待者清理、wait4 唤醒；`sys_wait4`/`sys_waitid` 支持子进程收割与 `rusage` 累计；futex、robust list、`prctl`、`setpgid`、capabilities、`seccomp` 等均有实现。
- **调度机制**：`switch_to_process` 使用 `__restore` 返回用户态，`wfi_wait_for_ready_process` 结合定时器与信号超时轮询；就绪队列为 FIFO/简单优先级，未见完整 CFS 或 async runtime 证据。

### FS（文件系统）
- **已体现证据**：VFS 核心 `vfs_core` 拥有完整的 dentry、inode、mount、path、rename、handle、权限检查抽象；支持 `open_tree`、`move_mount`、`name_to_handle_at`、`open_by_handle_at` 等 Linux 兼容挂载/句柄接口；实现 `ramfs`、`procfs`、以及一个功能较完整的 `ext4`（目录项、extent、inode、重命名、块缓存）；块层有 `BLOCK_CACHE_MANAGER`；fd 层支持 `epoll`、`eventfd`、`timerfd`、`pipe`、`fcntl` 记录锁、`dnotify`。
- **第三方/移植说明**：`os/src/fs/ext4/ext4.rs` 及 210 个“文件系统/第三方库”函数应视为移植或高度参考实现，不宜直接等同于原创 VFS 主体。

### DEV（设备驱动）
- **已体现证据**：网络侧有 VirtIO-MMIO 网卡初始化与收发（`try_new_virtio_net_device`、`recv`、`recycle_send_buffers`）；块设备侧有 LoongArch PCI VirtIO-Blk 探测（`try_new_block_device`）；网络抽象层有 `ETH0_DEVICE`、`LOOPBACK_DEVICE`。
- **底层证据未体现**：无中断控制器（PLIC/APLIC/IOAPIC）、DMA、时钟源（除 SBI 定时器外）、串口/键盘/鼠标/GPU/声卡/USB/NVMe 等驱动。

### SYSCALL（系统调用）
- **已体现证据**：`os/src/syscall/mod.rs#syscall` 为中央分发器，调用链覆盖 242 个目标；下分 `fs`（path/mount/io/poll/fd）、`process`（clone/exit/wait/exec/sched/futex/prctl/signal）、`net`、`time`（timer/timerfd/clock）等子模块；与 `PROCESS_MANAGER`、`vfs_core`、`net` 层直接交互。

### TRAP/IRQ（中断异常）
- **已体现证据**：RISC-V 与 LoongArch 分别有 `trap_handler` / `loongarch_trap_handler`、`kernel_trap_handler`；支持定时器中断 `arm_next_timer_interrupt`、SBI 控制台输出、异常返回 `__restore`。
- **底层证据未体现**：仅 24 个函数，未见完整中断控制器驱动、中断注册/分发框架、设备 IRQ 处理。

### NET（网络协议栈）
- **已体现证据**：`smol.rs` 初始化 smoltcp-like 接口（`Interface::new`、`IpCidr`、`LoopbackDevice`）；TCP 有监听表与包嗅探 `tcp_snoop_ip_packet`；UDP、ICMP、packet socket、Unix domain socket、netlink、alg socket 均有实现；`socket_ioctl` 可查询 `ETH0_DEVICE`/`LOOPBACK_DEVICE` 的地址/标志。
- **第三方/移植说明**：`smol.rs` 及 TCP/IP 核心明显基于 smoltcp 或类似第三方栈，需与项目原创部分区分。
