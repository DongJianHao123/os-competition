#### 4. 核心代码调用流对比剖析
* **【进程管理/调度器】**
* **底层技术析理**: 目标项目 `scheduler` 在标准 xv6 调度循环基础上，增加了 `freeproc` 调用以在调度时主动回收僵尸进程资源，并显式使用 `intr_off/intr_on` 进行临界区保护。参考项目 `scheduler` 为标准 xv6 实现，仅包含 `acquire, intr_on, mycpu, release, swtch`。两者在进程生命周期管理与同步粒度上存在架构差异。
* **代码溯源链接**:
  - 目标项目: `kernel/proc.c#scheduler`
  - 参考项目: `kernel/proc/proc.c#scheduler`

* **【系统调用/参数解析】**
* **底层技术析理**: 双方 `argint` 函数均通过 `argraw` 从 Trap 帧中获取整型参数，`argstr` 通过 `argaddr` 获取地址后再调用 `fetchstr` 从用户空间拷贝字符串。这是 xv6 系统调用 ABI 的标准实现模式，任何遵循此 ABI 的内核都必然呈现相同的调用拓扑。
* **代码溯源链接**:
  - 目标项目: `kernel/syscall.c#argint`
