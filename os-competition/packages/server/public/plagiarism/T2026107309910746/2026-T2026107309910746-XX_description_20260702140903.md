## 第四部分：分模块架构说明

### MM（内存管理）

底层证据较强。包含物理页分配器、页表映射、缺页处理、进程地址空间复制/共享、exec 加载等。

- `kernel/kalloc.c#kinit`：初始化物理内存空闲链表，出现 `SUPERPGROUNDDOWN`、`pa2super`、`superfree`，暗示支持“大页/superpage”分配。
- `kernel/vm.c#vmfault`：缺页异常处理，调用 `kalloc`、`kfree`、`ismapped`、`PTE2PA` 等。
- `kernel/vm.c#uvmshare`：地址空间共享/写时复制相关，使用 `kref_get/kref_inc`。
- `kernel/exec.c#kexec`：ELF 加载与解释器处理，调用 `build_auxv`、`exec_open_vfs`、`copyout`。

### PROC（进程管理/任务调度）

底层证据强。实现了传统 PCB、fork/exit/wait、clone、线程组、信号、futex、robust list 等 Linux 兼容语义。

- `kernel/proc.c#kexit`：进程退出核心函数，涉及 `linux_group_dec_live`、`linux_handle_robust_list`、`linux_clear_child_tid`。
- `kernel/proc.c#scheduler` / `kernel/proc.c#sched`：协作式/抢占式上下文切换，调用 `swtch`。
- `kernel/proc.c#forkat` / `kernel/proc.c#kclone`：实现 `clone` 与 `fork`。
- `kernel/sysproc.c#sys_linux_futex` / `sys_linux_clone` / `sys_linux_wait4`：Linux 兼容 futex、线程创建、wait4。

**注意**：未见独立“async runtime / 协程调度器”证据，现有“任务调度/异步运行时”统计实际对应传统调度 + futex/sleep/yield 同步原语。

### FS（文件系统）

底层证据最强。采用 VFS 抽象，兼容 legacy xv6 文件系统，并集成第三方 lwext4 实现 ext4。

- `kernel/vfs.c#vfs_resolve_proc_path`：进程路径解析，支持 lib 路径透传、绝对路径处理。
- `kernel/vfs.c#vfs_mount`：挂载抽象，调用 `vfs_ops_for_type`、`vfs_private_for_mount`。
- `kernel/vfs.c#xv6_vfs_rename/unlink/link`：legacy xv6 FS 的 VFS 适配。
- `kernel/fs.c#namex` / `ireclaim`：传统 xv6 路径遍历与 inode 回收。
- `kernel/lwext4/src/ext4*.c`：第三方 ext4 实现，覆盖块分配、inode 分配、extent、journal、目录索引等。

### DEV（设备驱动）

底层证据较弱，仅覆盖基础设备。

- `kernel/uart.c#uartinit` / `uartwrite` / `uartputc_sync`：串口控制台。
- `kernel/virtio_disk.c#virtio_disk_intr`：virtio 块设备中断与请求完成。
- `kernel/virtio_net.c#virtio_net_init` / `reclaim_tx`：virtio 网卡初始化与发送回收。
- `kernel/plic.c#plicinithart` / `plic_claim` / `plic_complete`：RISC-V PLIC 中断控制器。
- `kernel/bio.c#bwrite`：buffer cache 写回，调用 `virtio_disk_rw`。

### SYSCALL（系统调用）

底层证据中等。内核同时存在传统 xv6 系统调用与大量 `sys_linux_*` 兼容调用。

- `kernel/syscall.c#syscall_lookup`：系统调用分发表。
- `kernel/syscall.c#argstr` / `argaddr` / `argint`：参数解析辅助。
- `kernel/sysfile.c#sys_open` / `sys_linux_openat`：打开文件，走 VFS。
- `kernel/sysfile.c#sys_linux_execve` / `sys_linux_accept` / `sys_linux_bind` / `sys_linux_connect` / `sys_linux_recvfrom`：Linux ABI 兼容。
- `user/ulib.c`：用户态系统调用封装，如 `open`、`recv`、`mmap`。

### TRAP/IRQ（中断异常）

底层证据较弱，但关键入口存在。

- `kernel/trap.c#usertrap`：用户态陷入入口，处理系统调用、异常、信号投递、进程终止。
- `kernel/trap.c#kerneltrap`：内核态陷入/中断入口。
- `kernel/trap.c#clockintr`：时钟中断，维护 `futex_tick` 与 `wakeup`。
- `kernel/trap.c#trapinit` / `trapinithart`：trap 初始化。
- `kernel/spinlock.c#pop_off` / `push_off`：中断屏蔽层级管理。

### NET（网络协议栈）

底层证据弱，仅 `kernel/net.c` 中少量函数，功能极简。

- `kernel/net.c#net_rx`：网卡接收入口，分发 ARP/IP。
- `kernel/net.c#ip_rx`：IP 包处理，查找本地绑定。
- `kernel/net.c#sys_send` / `sys_bind` / `sys_unbind`：用户态 socket 风格接口。
- `kernel/net.c#in_cksum`：IP/UDP 校验和。

## 第六部分：边界与未体现内容

### 1. 没有底层证据支撑的模块/能力

- **完整 TCP/IP 协议栈**：仅见极简 IP/ARP/UDP 风格函数，无 TCP 状态机、滑动窗口、拥塞控制等证据。
- **高级设备驱动**：无 GPU、USB、NVMe、SATA、AHCI、Framebuffer、Sound 等驱动证据。
- **独立 async runtime / 协程调度器**：统计中的“任务调度/异步运行时”实际为传统调度 + futex/sleep，无 task/channel/waker/ring_fifo 等 async runtime 结构。
- **安全/隔离机制**：无 capabilities、seccomp、LSM、ASLR、KASLR、SMAP/SMEP 等证据。
- **虚拟化支持**：无 hypervisor、KVM、VM exit 处理证据。
- **电源管理与 ACPI/DT**：无相关代码。
- **多核同步细节**：仅见 `spinlock`、`push_off/pop_off`，无 RCU、per-CPU 变量、NUMA 等证据。
- **原始网络设备驱动**：仅 virtio-net，无真实 NIC（e1000、RTL8139 等）驱动。

### 2. 第三方/公共依赖说明

- `kernel/lwext4/` 为第三方 ext4 文件系统实现，不应视为该作品原创内核主体；作品原创性主要体现在 VFS 适配层、Linux ABI 兼容层、进程/线程模型扩展等方面。
