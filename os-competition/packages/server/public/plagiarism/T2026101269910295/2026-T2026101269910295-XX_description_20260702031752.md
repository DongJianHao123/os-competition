## 第四部分：分模块架构说明

### MM（内存管理）

**有底层证据**。关键证据：

- `kernel/exec_elf.c#exec_elf_ext4` / `kernel/exec_elf.c#exec_elf_ext4_proc`：ELF 加载、静态 PIE 重定位、页表构建（`MAKE_SATP`、`PGROUNDUP`、`PX`）
- `kernel/la_basic.c#load_elf_to_proc`：将 ELF 段读入进程地址空间
- `kernel/sysproc.c#sys_sbrk` -> `growproc`：堆扩展
- `kernel/trap.c#usertrap` -> `growproc`：用户态缺页/异常时扩展堆
- `kernel/file.c#fileread` / `filewrite`：用户态缓冲区拷贝（`copyin`/`copyout`/`copyin2`/`copyout2`）

可说明实现了用户虚拟地址空间、ELF 可执行文件加载、brk 堆增长、用户/内核数据拷贝。但**页替换、swap、COW、页缓存等底层证据未体现**。

### PROC（进程管理/任务调度）

**有底层证据**。关键证据：

- `kernel/proc.c#sched` / `yield` / `sleep` / `wakeup`：传统协作/抢占式调度原语
- `kernel/proc.c#wait_kernel` / `kernel/sysproc.c#sys_waitpid` / `kernel/la_basic.c#sys_wait_impl`：wait 语义
- `kernel/la_basic.c#run_waitable_child` / `run_child_until_exit`：子进程运行与上下文切换
- `kernel/sysproc.c#sys_clone3` / `sys_clone` / `sys_fork`：进程/线程创建
- `kernel/sysproc.c#sys_exit` / `sys_exit_group` / `proc_exit`：退出
- `kernel/syscall.c#sys_rt_sigprocmask` / `kernel/proc.c#signal_send` / `proc_signal_deliver`：信号机制

可说明实现了传统 Unix 进程生命周期、父子关系、信号、sleep/wakeup 调度。同时存在 `find_runnable_child`、`dispatch_syscall`、`sys_pselect6` 等**轻量级多路复用/异步运行时风格**的代码，但**不是完整的 async/await 运行时**。

### FS（文件系统）

**有强底层证据**。关键证据：

- `kernel/sysfile.c#sys_openat` / `sys_open` / `sys_getdents64` / `sys_readlinkat` / `sys_unlinkat` / `sys_linkat` / `sys_mkdirat`：VFS/系统调用层
- `kernel/sysfile.c#fill_kstat_for_path` / `chown_path`：stat/chmod/chown 封装
- `kernel/file.c#fileread` / `filewrite`：文件读写抽象
- `kernel/lwext4/src/ext4.c` / `ext4_balloc.c` / `ext4_ialloc.c` / `ext4_extent.c` / `ext4_journal.c` / `ext4_dir_idx.c`：第三方 lwext4 实现的 ext2/3/4
- `kernel/la_basic.c#create_memfile` / `find_memfile` / `memfile_*`：内存文件/临时文件层
- `kernel/fat32.c#fat32_init`：存在 FAT32 初始化代码，但调用链证据有限

**必须单独说明**：`kernel/lwext4/` 是第三方 ext4 实现，不是本项目原创内核主体；原创工作主要体现在系统调用封装、路径解析、内存文件层与块设备对接。

### DEV（设备驱动）

**有底层证据，但范围有限**。关键证据：

- `kernel/la_virtio_pci.c#la_virtio_blk_init` / `find_virtio_blk` / `pci_assign_bars` / `legacy_setup_queue`：virtio-blk 初始化
- `kernel/la_virtio_pci.c#la_virtio_blk_read_sector`：virtio-blk 扇区读写
- `kernel/uart.c#uartputc` / `uartputc_sync` / `uartstart`：串口控制台
- `kernel/plic.c#plicinithart` / `plic_claim` / `plic_complete`：PLIC 中断仲裁
- `kernel/virtio_disk.c#virtio_disk_intr`：virtio 磁盘中断完成

可说明实现了 virtio-blk、UART、PLIC。但**GPU、声卡、网卡、USB、SATA 等驱动底层证据未体现**。

### SYSCALL（系统调用）

**有强底层证据**。关键证据：

- `kernel/la_basic.c#dispatch_syscall`：大规模系统调用分发（55+ 调用）
- `kernel/syscall.c#sys_pselect6` / `sys_rt_sigprocmask` / `sys_clock_adjtime_stub` / `sys_capget_stub` / `sys_shmctl` / `sys_uname` / `sys_writev` / `sys_readv` / `sys_sync`
- `kernel/sysfile.c#sys_bind` / `do_accept` / `sys_linkat` / `sys_mkdirat` / `sys_remove`
- `kernel/sysproc.c#sys_clock_nanosleep` / `sys_nanosleep` / `sys_clone3` / `sys_waitpid`

可说明实现了 Linux 兼容的系统调用接口，覆盖文件、进程、信号、时间、socket 封装等。

### TRAP/IRQ（中断异常）

**有底层证据**。关键证据：

- `kernel/trap.c#trapinithart`：设置 `stvec`、`sie`、`sstatus`、定时器
- `kernel/trap.c#usertrap` / `kerneltrap`：用户态/内核态异常入口，读取 `scause`/`sepc`/`sstatus`，调用 `devintr`
- `kernel/trap.c#usertrapret`：返回用户态
- `kernel/intr.c#push_off` / `pop_off`：中断屏蔽嵌套
- `kernel/plic.c`：PLIC 初始化与 claim/complete

可说明实现了 RISC-V 风格异常/中断处理、定时器中断、PLIC 外设中断。但**高级中断亲和性、MSI、虚拟中断等底层证据未体现**。

### NET（网络协议栈）

**底层证据未体现**。仅有 socket 相关系统调用封装：

- `kernel/sysfile.c#sys_bind` / `do_accept`
- `sock_find_bound` / `copyout_sockaddr`

没有 TCP/IP 协议栈、NIC 驱动、路由、skb 缓冲区等底层证据。因此**只能判定为“socket 系统调用封装”，不能判定为完整网络协议栈**。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块/特性

- **完整网络协议栈**：仅有 `sys_bind`/`do_accept` 等 socket 系统调用封装，无 TCP/IP、NIC 驱动、路由、skb 等实现
- **高级虚拟内存**：无 swap、页替换算法、COW、共享内存页缓存等明确证据
- **SMP/NUMA 高级支持**：仅有自旋锁、PLIC、hartid 相关代码，无核间 IPI、缓存一致性、NUMA 调度等证据
- **安全/权限子系统**：仅有 `mode`/`owner` 设置与 `capget` stub，无完整 capability、ACL、审计等
- **GPU/声卡/USB/SATA 等外设驱动**：未体现
- **正式 async/await 运行时**：`pselect6`、`find_runnable_child` 等体现轻量多路复用，但无 `Future`、`Waker`、`Executor` 等标准异步运行时结构

### 需要单独说明的第三方/公共代码

- `kernel/lwext4/` 是第三方 ext2/3/4 文件系统库（501 个函数），属于本项目集成的外部依赖，**不应视为原创内核主体**
- `xv6-user/` 中的 `sh.c`、`grind.c`、`xargs.c` 等用户态程序风格与 xv6 接近，属于测试/用户态工具链

### 判断性质说明

- 本文档所有方向判断（如“面向 RISC-V/LoongArch”、“类 Unix 兼容内核”）均基于路径前缀（`la_`）、函数名（`r_scause`、`w_stvec`、`PLIC` 等）与调用链，属于**可能方向**，不是最终身份或原创性结论
- 本文档不做抄袭、违规或原创性判定
