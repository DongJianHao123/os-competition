## 第四部分：分模块架构说明

### MM（内存管理）
底层证据较强。内核侧有全局分配器 `axalloc`（`global_init`、`global_add_memory`、`alloc`），地址空间管理 `axmm`（`AddrSpace`、`MemoryArea`、`Backend`、页表遍历 `PageIter4K`），并实现了写时复制（`handle_cow_fault`）、共享页故障（`handle_page_fault_shared`）以及映射克隆（`clone_user_mappings_from`）。用户态侧通过 `examples/shell/src/uspace/memory_map.rs` 提供 `sys_mmap`、`sys_msync` 等接口。  
**未体现**：交换（swap）、NUMA、大页（huge page）完整策略、内存 cgroup。

### PROC（进程管理/任务调度）
底层证据强，且不是轻量级任务包装。存在完整的 `fork`、`sys_clone`、`sys_execve`、`wait4/waitid`、`sys_exit_group` 等 Linux 风格进程原语；`Process` 结构包含 `pid`、`pgid`、`sid`、`live_threads`、`children`、`exit_code`、`term_signal` 等 PCB 字段；支持进程组/会话（`setpgid`、`setsid`、`getpgid`、`getsid`）和信号掩码（`sys_rt_sigprocmask`）。任务调度核心在 `kernel/task/axtask/`（`TaskInner`、`spawn_task`、`wait_queue`、`run_queue`、`reap_exited_tasks`），用户态上下文切换通过 `UspaceContext` / `enter_uspace` 完成。  
**未体现**：具体调度算法（如 CFS）、SMP/多核亲和性、实时调度策略。

### FS（文件系统）
底层证据强。`kernel/fs/axfs/` 实现了 VFS 层、根目录、挂载表以及 `/proc`、`/dev`、`/sys` 等合成文件系统；`kernel/fs/axfs/src/fs/ext4fs.rs` 集成第三方 ext4  crate（`Ext4::load`、`Ext4Disk`）。用户态 `examples/shell/src/uspace/fd_table.rs` 实现了 fd 表、`open/read/write/seek/stat/getdents64` 等 POSIX 文件操作，`mount_abi.rs` 实现了 `mount/umount2`。  
**注意**：ext4 实现属于第三方库集成，不应视为原创内核主体。

### DEV（设备驱动）
底层证据中等。`kernel/drivers/axdriver/src/lib.rs#init_drivers` 展示了块/网/显示设备的探测与枚举框架；`kernel/drivers/axdma/src/dma.rs` 提供 DMA 一致性内存分配/释放。`tools/raspi4/chainloader` 包含 Raspberry Pi 4 PL011 UART 初始化代码，但属于启动加载器工具，不是运行时内核驱动。`tools/bwbench_client` 使用主机 raw socket 进行带宽测试，亦非内核驱动。  
**未体现**：中断控制器、PCI/USB/GPU/音频驱动、块设备底层驱动细节。

### SYSCALL（系统调用）
底层证据强。`examples/shell/src/uspace/syscall_dispatch.rs#user_syscall` 是集中式系统调用分发入口；`api/arceos_posix_api/src/imp/` 提供 POSIX 封装（`net.rs`、`pthread/mutex.rs`、`io_mpx/epoll.rs`、`task.rs`）；`ulib/axlibc/src/` 提供 C 标准库包装（`dup3`、`sendto`、`recvfrom`）。系统调用返回值普遍采用 `neg_errno` 的 Linux 兼容风格。

### TRAP/IRQ（中断异常）
**底层证据未体现**。模块画像中该领域仅 2 个函数，且均非中断/异常入口（`ulib/axlibc/c/pow.c#exp_inline` 为数学函数，`kernel/sync/axsync/src/mutex.rs#may_interrupt` 为同步原语辅助函数）。未见到异常向量表、IRQ 控制器初始化、上下文保存/恢复、时钟中断注入等直接证据。

### NET（网络协议栈）
底层证据强。`kernel/net/axnet/src/smoltcp_impl/` 基于第三方 smoltcp 实现了 TCP（connect/recv/send/shutdown、listen table、loopback）、UDP（bind/recv_from）、DNS 查询以及数据包嗅探（`snoop_tcp_packet`）。用户态通过 `api/arceos_posix_api/src/imp/net.rs` 和 `examples/shell/src/uspace/fd_socket.rs` 暴露 socket API，并在 shell 中实现了本地 socket pair（`LocalSocketPairState`）。  
**注意**：TCP/IP 核心由 smoltcp 提供，应单独说明为第三方库集成。
