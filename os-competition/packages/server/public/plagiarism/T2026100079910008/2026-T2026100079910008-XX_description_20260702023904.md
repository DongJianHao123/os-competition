## 第四部分：分模块架构说明

### MM（内存管理）
底层证据充分。内核堆使用 `os/src/basic/global_allocator.rs` 中的全局分配器，并维护 `allocated` 哈希表做地址追踪；物理页框由 `os/src/memory/frame_allocator.rs` 的伙伴系统分配器管理；虚拟地址空间在 `os/src/memory/address_space.rs` 中实现，包含页表操作与用户态缺页处理 `handle_page_fault`；另有 `PAGE_METADATAS`、`ZERO_PAGE` 等页元数据机制。`fork` 时会复制地址空间。

### PROC（进程管理/任务调度）
不是单纯的轻量级任务调度，而是具备完整进程语义的实现。证据包括：
- `os/src/multitask/task.rs#fork` / `execute_init` / `load_elf`
- `os/src/syscall/process.rs#clone` / `execve` / `wait4` / `exit_group`
- `os/src/multitask/process_data.rs` 维护进程级信号、资源限制、任务列表
- `os/src/multitask/futex.rs`、`os/src/multitask/wait.rs` 提供线程同步
- `os/src/multitask/signal/*` 实现信号投递与处理
调度器通过 `TASK_SCHEDULER.lock` 与 `Task::switch_to` 完成上下文切换。

### FS（文件系统）
内核最重的模块。原创 VFS 层位于 `os/src/filesystem/vfs/*`，包含 `DEntry`、`INode`、`FileSystem`、`Mount`、`walk_path` 等；具体文件系统包括：
- `procfs`：`os/src/filesystem/procfs/*`
- `pipe`：`os/src/filesystem/other/pipe.rs`
- `ext4` 适配层：`os/src/filesystem/ext4/*`
- 第三方改编库：`edited_lib/ext4_rs/*`（块分配、inode 分配、extent 管理）与 `edited_lib/rust-elf/*`（ELF 解析，用于 `execve`/`load_elf`）

### DEV（设备驱动）
证据较弱，仅 17 个函数。实现了 VirtIO-MMIO/PCI 块设备扫描、块设备抽象与 HAL 共享内存接口。关键路径：
- `os/src/devices/virtio.rs#scan_virtio_devices`
- `os/src/devices/mod.rs#add_block_device`
- `os/src/devices/block_device.rs#read`
- `os/src/devices/hal.rs#share`

### SYSCALL（系统调用）
`os/src/syscall/*` 共 43 个函数，覆盖文件、进程、信号、futex、epoll、时钟、杂项等。入口由 trap handler 分发，参数通过架构相关 `syscall_args` 读取。

### TRAP/IRQ（中断异常）
`os/src/trap/trap.rs#new` 负责将硬件异常/中断码解析为 `Exception`/`Interrupt` 枚举；`os/src/trap/trap_handler.rs#trap_handler` 是统一入口，处理系统调用、缺页、调度切换等。架构相关代码分布在 `os/src/arch/riscv64/*` 与 `os/src/arch/loongarch64/*`。

### NET（网络协议栈）
**底层证据未体现**。AST Call Map、模块统计、目录结构中均未发现 socket、TCP/IP、网络设备驱动或协议栈相关函数。
