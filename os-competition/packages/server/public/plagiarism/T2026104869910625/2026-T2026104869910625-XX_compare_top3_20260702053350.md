#### 4. 核心代码调用流对比剖析
* **【进程管理/调度切换路径】**
* **底层技术析理**: 目标项目的 `sched` 函数调用 `hsai_swtch`，这是其 HSAI (Hardware Software Abstract Interface) 层提供的架构无关上下文切换接口，并在切换前调用 `copytrapframe` 保存陷阱帧，服务于其多架构支持。参考项目的 `sched` 直接调用底层汇编实现的 `swtch`，无 HSAI 抽象层。两者在上下文切换的抽象层次和对象管理模型上存在根本性架构差异。
* **代码溯源链接**:
  - 目标项目: `kernel/process.c#sched`
  - 参考项目: `kernel/proc/proc.c#sched`

* **【内存管理/用户态Trap处理】**
