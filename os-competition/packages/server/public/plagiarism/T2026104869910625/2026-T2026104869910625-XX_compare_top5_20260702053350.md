#### 4. 核心代码调用流对比剖析
* **【进程管理/进程退出路径】**
* **底层技术析理**: 两者均遵循 POSIX `exit` 语义，执行“关闭文件 -> 回收子进程 -> 调度离开”的流程。然而，目标项目采用**面向对象的 VFS 抽象层**，通过 `get_file_ops()->close` 实现多文件系统分发；参考项目则直接调用 `fileclose`。目标项目的调度实体为 `myproc`，采用**单层进程模型**；参考项目则使用 `mythread`/`mykthread`，采用**双层线程模型**。这些差异表明两者在文件系统和进程模型的核心架构上存在独立设计。
* **代码溯源链接**:
  - 目标项目: `kernel/process.c#exit`
  - 参考项目: `kernel/proc.c#exit`

* **【进程管理/调度器核心】**
* **底层技术析理**: 两者均包含 xv6 调度器的不变式检查序列（`holding -> intr_get -> mycpu -> panic -> 上下文切换`）。但目标项目使用 `hsai_swtch` 进行上下文切换，这是其**HSAI 硬件抽象层**的一部分，用于适配 LoongArch/RISC-V 多架构；参考项目则直接使用平台相关的 `swtch`。此外，目标调度器在切换前调用 `copytrapframe` 保存陷阱帧，体现了不同的上下文管理策略。
* **代码溯源链接**:
  - 目标项目: `kernel/process.c#sched`
