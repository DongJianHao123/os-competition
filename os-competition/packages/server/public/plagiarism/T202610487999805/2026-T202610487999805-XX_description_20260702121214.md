## 第四部分：分模块架构说明

### MM（内存管理）

底层证据较强。物理层使用按尺寸分级的 slab 分配器（`os/src/mm/allocator/slab_allocator.rs#dealloc`、`dealloc_by_layout`），涉及 `SlabBlock<S>::cap/floor`、`cache16/cache128/cache1024` 等结构。虚拟内存层通过 `os/src/mm/user.rs`、`os/src/mm/vm/uvm.rs` 维护用户地址空间，`uvm.rs#load_dl_interp_if_needed` 完成动态链接器加载。内核地址空间由 `KVMSPACE` 保护。用户缓冲区校验使用 `UserSliceRaw` / `UserPtrRaw`，并在 `sys_process_vm_readv/writev` 中实现跨进程 VM 空间访问。

### PROC（进程管理/任务调度）

传统进程管理证据充分。`os/src/task/task.rs#fork`、`handle_zombie`、`do_exit`、`do_group_exit` 维护 PCB、父子关系、线程组、僵尸回收；`os/src/syscall/process.rs#sys_execve`、`sys_waitpid`、`sys_clone`、`sys_fork` 提供 POSIX 风格进程控制；`os/src/task/task.rs#exit_robust_list` 与 `os/src/syscall/futex.rs#sys_futex` 实现 futex/robust list；`os/src/task/signal.rs#check_and_handle` 与 `os/src/syscall/signal.rs#sys_rt_sigprocmask` 处理信号。调度侧，`os/src/executor/mod.rs#run_until_idle`、`spawn`、`kernel_spawn` 与 `os/src/processor/schedule.rs#load_balance`、`migrate_tasks`、`update_load_avg` 共同构成异步运行时 + 多核 CFS 风格负载均衡。

### FS（文件系统）

VFS 与多种具体文件系统并存。`os/src/fs/vfs/dentry.rs#find`、`os/src/fs/vfs/file.rs#open_file` 提供路径解析与打开抽象；`os/src/fs/fat32/inode.rs#lookup` 实现 FAT32 目录查找；`os/src/fs/ext4/inode.rs#create`、`cache_read_at`、`cache_write_at` 实现 ext4 文件创建与页缓存读写；`os/src/fs/procfs/mod.rs#init_procfs`、`os/src/fs/devfs/mod.rs#init_devfs` 分别构建 procfs 与 devfs。`os/src/fs/fs.rs#init` 负责根文件系统与块设备匹配。

### DEV（设备驱动）

设备框架与具体驱动均有体现。`os/src/devices/mod.rs#init` 解析设备树并初始化设备管理器；`os/src/devices/serial.rs#get_serial`、`scan_char_device` 从设备树 `stdout-path` 定位串口；`os/src/devices/manager.rs#init_net` 初始化 virtio-net；`os/src/drivers/net/virtio_net.rs#receive`、`recycle_rx_buffer` 处理网络包收发；`os/src/drivers/block/mmc/mod.rs#card_init` 完成 MMC/SD 卡初始化；`os/src/drivers/block/pci_blk.rs#new` 提供 PCI 块设备支持。

### SYSCALL（系统调用）

`os/src/syscall/mod.rs#syscall` 是统一分发入口，规模与调用数均最大。其下按功能分散到：
- `os/src/syscall/fs.rs`：openat/read/write/pread/readv/writev/mount/unlinkat/getdents64 等
- `os/src/syscall/process.rs`：execve/waitpid/clone/fork/exit/exit_group/setuid/setgid 等
- `os/src/syscall/mm.rs`：process_vm_readv/writev
- `os/src/syscall/io.rs`：epoll/pselect/ppoll
- `os/src/syscall/net.rs`：sendmsg/accept
- `os/src/syscall/futex.rs`：futex
- `os/src/syscall/sche.rs`：sched_getaffinity/setaffinity

### TRAP/IRQ（中断异常）

HAL 层提供架构相关实现：`hal/src/component/irq/loongarch64/eiointc.rs#init/claim_irq`、`hal/src/component/irq/riscv64.rs#enable_irq`、`hal/src/component/trap/loongarch64/mod.rs#save/restore`。OS 层 `os/src/trap/mod.rs#user_trap_handler` 与 `kernel_trap_handler` 处理定时器中断、设备中断、系统调用与异常，并调用 `TIMER_MANAGER.check`、`DEVICE_MANAGER.lock`、`IRQ_COUNTER.add_timer_irq_cnt`。

### NET（网络协议栈）

网络栈明显基于 smoltcp：`SOCKET_SET.with_socket_mut::<tcp::Socket>`、`SOCKET_SET.poll_interfaces` 反复出现。`os/src/net/tcp.rs#recv/send/shutdown`、`os/src/net/udp.rs#send_to`、`os/src/net/raw.rs#send`、`os/src/net/socketpair.rs#poll` 覆盖主要 socket 类型。`os/src/net/mod.rs#init_network` 初始化接口，`modify_packet` 处理入站包与监听表。`os/src/net/crypto.rs#encode/encode_raw/check_alg` 提供类似 AF_ALG 的加密套接字支持。
