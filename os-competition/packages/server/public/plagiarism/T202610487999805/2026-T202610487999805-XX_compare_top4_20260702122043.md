## 4. 核心代码调用流对比剖析

### 【进程管理 / sys_clone 系统调用】

- **底层技术析理**:  
  双方均实现了 Linux `clone` 系统调用，但采用了截然不同的内存模型与上下文构造策略。  
  - **目标项目**: 通过 `UserPtrRaw::new(child_tid.0 as *mut u32).ensure_write(&mut task.get_vm_space().lock()).ok_or` 进行用户态指针安全校验，将 `parent_tid`/`child_tid` 的写入操作委托给 `UserPtrRaw` 封装层，体现了**安全内存访问抽象**。  
  - **参考项目**: 直接使用 `core::ptr::write` 写入 `child_trap_cx`，并通过 `child_trap_cx.set_sp`/`set_tp` 操作陷阱上下文寄存器，体现了**直接硬件上下文操控**。

- **底层技术析理**:
