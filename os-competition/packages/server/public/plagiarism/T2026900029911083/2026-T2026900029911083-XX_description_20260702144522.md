## 第四部分：分模块架构说明

### MM（内存管理）

- **证据**：`kernel/proc/exec.c#execve` 使用 `PGROUNDUP/PGROUNDDOWN` 进行 ELF 加载；`kernel/syscall/sysproc.c#sys_brk` / `sys_sbrk` 调用 `growproc` 调整进程堆；`kernel/trap/rv/trap.c#usertrapret` 使用 `MAKE_SATP` / `VIRT2PHY` 切换页表；`kernel/syscall/sysproc.c#sys_ppoll`、`sys_writev` 中出现 `kalloc/kfree`。
- **能力**：物理页分配/释放、进程地址空间增长、用户态参数拷贝（`copyin/copyout`）、execve 加载、陷阱返回时的页表切换。
- **未体现**：交换（swap）、按需分页（demand paging）的独立缺页处理路径、COW（写时复制）在证据中未明确出现。

### PROC（进程管理/任务调度）

- **证据**：`kernel/proc/proc.c#scheduler` / `sched` / `sleep` / `yield` 使用 `swtch` 进行上下文切换；`kernel/proc/proc.c#forkret` 进入用户态；`kernel/syscall/sysproc.c` 提供 `sys_fork`、`sys_clone`、`sys_exec`（转发 `sys_execve`）、`sys_exit`、`sys_wait` / `sys_waitpid` / `sys_wait4`、`sys_kill`、`sys_futex`、`sys_setreuid` / `sys_setregid`；`kernel/util/futex.c#futex_wake` 与 `kernel/trap/signal.c#sig_deliver` 支持 futex 与信号投递。
- **能力**：传统 Unix 进程模型、父子关系、调度循环、睡眠唤醒、线程级克隆、信号、futex。
- **说明**：该分类在输入中标记为“任务调度/异步运行时”，但证据显示的是经典调度器，**未发现 async/await、waker、ring_fifo 等异步运行时原语**。

### FS（文件系统）

- **证据**：
  - 原创/适配层：`kernel/fs/VFS_ext.c#vfs_ext_statx`、`vfs_ext_fstat`、`vfs_ext_futimens`、`vfs_ext_utimens`；`kernel/fs/vfs/file.c#fileclose`；`kernel/syscall/sysfile.c#sys_openat`、`sys_readv`、`sys_writev`、`sys_linkat`、`sys_renameat2`、`sys_mkdirat`、`sys_mknod`、`sys_symlinkat`、`sys_readlinkat`。
  - 第三方实现：`kernel/fs/lwext4/ext4_balloc.c#ext4_balloc_alloc_block` / `ext4_balloc_free_block`；`kernel/fs/lwext4/ext4_ialloc.c#ext4_ialloc_alloc_inode` / `ext4_ialloc_free_inode`；`kernel/fs/lwext4/ext4_extent.c#ext4_extent_remove_space`；`kernel/fs/lwext4/ext4_journal.c#jbd_journal_prepare`。
- **能力**：VFS 适配层将系统调用映射到 `lwext4`；支持 EXT4 的块/ inode 分配、extent、目录索引、journal、符号链接、硬链接、目录操作、时间戳。
- **说明**：`lwext4` 属于第三方库，不应视为该作品原创文件系统核心。

### DEV（设备驱动）

- **证据**：
  - 串口：`kernel/drive/uart.c#uartintr`、`uartstart`、`uartputc`。
  - RISC-V 块设备与中断：`kernel/drive/rv/virtio_disk.c#virtio_disk_intr`；`kernel/drive/rv/plic.c#plicinithart`。
  - LoongArch 块设备与 PCI：`kernel/drive/la/virtio_disk.c#virtio_disk_init`；`kernel/drive/la/virtio_pci.c#virtio_pci_read_caps` / `virtio_pci_set_queue_addr2`；`kernel/drive/la/pci.c#pci_scan_buses`。
- **能力**：UART 控制台、virtio-blk over PCI、PLIC（RV）与 APIC/extioi（LA）中断控制器初始化。

### SYSCALL（系统调用）

- **证据**：`kernel/syscall/syscall.c#argint`、`argaddr`、`argstr`；`kernel/syscall/sysfile.c#sys_openat` / `sys_readv` / `sys_writev`；`kernel/syscall/sysproc.c#sys_writev` / `sys_copy_file_range` / `sys_ppoll` / `sys_clone`；`kernel/syscall/syssig.c#sys_tgkill` / `sys_tkill` / `sys_kills`。
- **能力**：参数提取、文件/进程/信号系统调用分发，覆盖 open/read/write/close、exec/fork/clone/wait、rename/link/mkdir、poll、signal、ftruncate、copy_file_range 等。

### TRAP/IRQ（中断异常）

- **证据**：
  - LoongArch：`kernel/trap/la/trap.c#trapinit` 配置 `ecfg/eentry/tlbrentry/merrentry/tcfg`；`kernel/trap/la/trap.c#kerneltrap` 读取 `crmd/prmd/badi/badv` 并调用 `devintr`。
  - RISC-V：`kernel/trap/rv/trap.c#kerneltrap` / `usertrap` / `usertrapret` 操作 `scause/sepc/sstatus/satie/satp`；`kernel/trap/rv/trap.c#clockintr` 调用 `SBI_SET_TIMER`。
- **能力**：双架构陷阱初始化、内核态/用户态异常入口、设备中断分发、时钟中断。

### NET（网络协议栈）

- **底层证据未体现**：未出现任何网络协议栈相关源文件、函数或调用链（无 socket、TCP/IP、NIC 驱动、netif 等）。
