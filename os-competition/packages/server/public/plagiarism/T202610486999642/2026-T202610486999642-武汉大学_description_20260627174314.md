## 第四部分：分模块架构说明

### MM（内存管理）
- 有明确的用户地址空间复制与 ELF 加载证据：`os/src/mm/user_space.rs#from_existed_user`、`os/src/task/exec.rs#exec`。
- 页缓存机制可见：`os/src/fs/vfs/file.rs#read_regular_cached_at` 使用 `PAGE_CACHE` 并支持预读。
- `os/src/perf.rs` 中大量 brk/mmap 相关计数器（`BRK_GROW_CALLS`、`BRK_EAGER_MAPPED_PAGES` 等）说明内存分配/映射路径被跟踪。
- 未见到独立的 slab/buddy 分配器实现细节，底层物理页管理证据未充分展开。

### PROC（进程管理/任务调度）
- **传统进程管理证据充分**：`os/src/syscall/process/clone.rs#sys_clone_process_inner`、`os/src/task.rs#exit_current`、`os/src/syscall/wait.rs#scan_wait4_children` 覆盖 fork/clone、退出、僵尸、wait。
- **exec 路径完整**：`os/src/syscall/process/exec.rs#exec_loaded_program`、`os/src/task/exec.rs#exec`、`write_user_stack`。
- **信号与调试**：`os/src/syscall/signal.rs#sys_rt_sigtimedwait`、`os/src/syscall/process/ptrace.rs#sys_ptrace`。
- **命名空间与 capability**：`os/src/syscall/process/namespace.rs#sys_setns` / `sys_unshare`、`os/src/syscall/process/identity.rs#sys_prctl_ctx` / `sys_capset_ctx`。
- **调度循环**：`os/src/task/processor.rs#run_tasks` 使用 `__switch` 进行上下文切换。
- 该模块不是“轻量任务/channel”，而是具备 PCB、父子关系、进程组、会话、pidfd 等完整语义的进程管理。

### FS（文件系统）
- **VFS 层**：`os/src/fs/vfs/file.rs#open_vfs_file_impl`、`create_tmpfile_inode`、`read_regular_cached_at`。
- **挂载系统**：`os/src/fs/mount.rs#mount_bind_at`、`move_mount_at`、`mount_new_fs_on_target`、`mount_new_fs_at`、`ensure_mount_open`、`sync_all_mounts`。
- **特殊文件系统**：procfs（`os/src/fs/procfs.rs`）、devfs（`os/src/fs/devfs.rs`）。
- **高级文件机制**：file handle（`os/src/syscall/fs/file_handle.rs`）、epoll（`os/src/syscall/fs/epoll.rs`）、fanotify（`os/src/syscall/fs/fanotify.rs`）、splice/copy_file_range/sendfile（`os/src/syscall/fs/io.rs`）。
- **第三方 ext4**：`os/src/fs/ext4.rs` 封装 `lwext4_rust`。
- 文件系统是本作品代码量最大、证据最强的模块。

### DEV（设备驱动）
- **块设备层**：`os/src/drivers/block.rs`、`os/src/drivers/block_cache.rs` 提供 4K 读缓存、写回、失效、统计。
- **字符/输入设备**：`os/src/fs/devfs.rs#write_uinput`、`input_open_client`。
- **中断控制器**：RISC-V PLIC（`os/src/arch/riscv64/board.rs#device_init`）、LoongArch EIOINTC/PCH-PIC（`os/src/arch/loongarch64/board.rs#device_init`）。
- **PCI 解析**：`os/src/arch/loongarch64/board.rs#pci_mem_range_from_dtb`。
- 驱动模块函数数量相对较少，主要覆盖块设备与板级基础外设。

### SYSCALL（系统调用）
- **统一入口**：`os/src/syscall.rs#syscall_with_context` 包含 254 条调用链，分发到 accept/brk/clone/openat 等。
- **分类实现**：`os/src/syscall/fs/*`、`os/src/syscall/process/*`、`os/src/syscall/signal.rs`、`os/src/syscall/sem.rs`、`os/src/syscall/msg.rs`、`os/src/syscall/keyring.rs`、`os/src/syscall/aio.rs`、`os/src/syscall/time.rs` 等。
- 系统调用表规模与 Linux 兼容面较广。

### TRAP/IRQ（中断异常）
- **架构无关入口**：`os/src/arch/riscv64/trap.rs#trap_handler`、`os/src/arch/loongarch64/trap.rs#trap_handler`。
- **板级中断处理**：`os/src/arch/riscv64/board.rs#irq_handler`、`os/src/arch/loongarch64/board.rs#irq_handler`。
- **定时器**：`os/src/arch/loongarch64/timer.rs#to_unix_nanos`、`os/src/arch/loongarch64/sbi.rs#set_timer`。
- 中断/异常路径与调度、信号、计时器紧耦合。

### NET（网络协议栈）
- **底层证据未体现**：没有 IP、TCP、UDP、ICMP、路由、网卡驱动等实现。
- 仅存在**本地 socket / Unix domain** 相关证据：`os/src/fs/socket.rs#send_datagram`、`poll_with_wait`，以及 `os/src/perf.rs` 中对 local socket 的读写统计。
- 因此“网络协议栈”模块应理解为本地进程间通信，而非完整网络协议栈。

## 第六部分：边界与未体现内容

1. **完整网络协议栈底层证据未体现**  
   没有 IP、TCP、UDP、ICMP、路由表、网卡驱动等实现；现有 `fs/socket.rs` 与 `perf.rs` 中的 local socket 统计仅支持本地/Unix domain 通信。

2. **高级存储控制器驱动证据未体现**  
   仅见通用块设备层与 DTB 发现的块设备，没有 NVMe、AHCI/SATA、SD/eMMC 等具体控制器驱动的充分证据。

3. **多核 SMP 调度证据不充分**  
   存在 `BOOT_HART_ID` 与多 hart 板级初始化线索，但跨 hart 任务窃取、核间中断、per-CPU 调度队列等细节未在提供的证据中体现。

4. **图形/音频/GPU 驱动底层证据未体现**  
   未见到 framebuffer、GPU、声卡等相关驱动。

5. **ext4 实现依赖第三方库**  
   `os/src/fs/ext4.rs` 是对 `lwext4_rust` 的封装，不应视为本项目独立实现的文件系统内核主体。

6. **“任务调度/异步运行时”统计存在分类噪声**  
   该模块大量函数实际位于 `syscall/fs/*` 的阻塞/等待路径（如 `sys_epoll_wait_until`、`sys_mount`），反映的是文件系统阻塞语义，而非独立的 async runtime/waker 机制。

7. **物理内存分配器细节未充分展开**  
   虽然存在页缓存、地址空间、ELF 加载证据，但 buddy/slab/页框分配器等底层实现未在关键证据中直接呈现。

8. **本报告仅基于 AST Call Map、函数名、路径锚点与调用链**  
   所有“可能的系统方向”均为技术推断，不构成对项目最终身份、原创性或合规性的判定。
