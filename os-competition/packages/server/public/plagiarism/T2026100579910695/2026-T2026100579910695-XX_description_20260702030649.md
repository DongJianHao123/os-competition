## 第四部分：分模块架构说明

### MM（内存管理）

底层证据较强。内核实现了分级 slab 分配器、用户虚拟地址空间管理、ELF 加载时的内存区域构建，并支持跨进程地址空间读写。

- `os/src/mm/allocator/slab_allocator.rs#dealloc` / `dealloc_by_layout`：按 16/128/1024 等缓存分级回收内存。
- `os/src/mm/vm/uvm.rs#from_elf`：从 ELF 构建用户虚拟内存区域，使用 `UserVmArea::new`、`AuxHeader::new`。
- `os/src/mm/vm/uvm.rs#load_dl_interp_if_needed`：处理动态解释器 `/lib/libc.so` 的加载。
- `os/src/syscall/mm.rs#sys_process_vm_readv` / `sys_process_vm_writev`：通过 `UserSliceRaw` 在不同进程的 `VmSpace` 之间拷贝数据。

### PROC（进程管理）

传统进程管理底层证据充分，不是单纯的轻量任务。存在 PCB、fork/clone/exec/wait、进程组、会话、信号、robust futex、uid/gid 等结构。

- `os/src/syscall/process.rs#sys_clone` / `sys_fork`：解析 `CloneFlags`，创建新任务。
- `os/src/task/task.rs#fork`：复制 PCB，初始化信号管理器、进程组关系。
- `os/src/syscall/process.rs#sys_execve`：加载可执行文件，路径中出现 `/loongarch/musl/busybox`、`/riscv/musl/busybox`。
- `os/src/syscall/process.rs#sys_waitpid`：回收子进程，配合 `os/src/task/task.rs#handle_zombie` 处理 zombie。
- `os/src/task/task.rs#exit_robust_list` / `do_group_exit`：futex robust list 与线程组退出。

### 任务调度/异步运行时

证据显示存在异步运行时与多核调度器，包含任务队列、负载均衡、CPU 亲和性、迁移机制。

- `os/src/executor/mod.rs#spawn` / `run_until_idle` / `kernel_spawn`：基于 `async_task` 的任务创建与执行循环。
- `os/src/processor/schedule.rs#load_balance` / `migrate_tasks` / `update_load_avg`：多核负载均衡与任务迁移。
- `os/src/syscall/sche.rs#sys_sched_getaffinity` / `sys_sched_setaffinity`：CPU 亲和性设置。

### FS（文件系统）

文件系统栈是项目重点之一，覆盖 VFS、挂载管理、dentry 缓存、procfs、devfs、tmpfs、fat32，以及基于第三方库的 ext4。

- `os/src/fs/procfs/mod.rs#init_procfs`：构建 `/proc` 目录树（cpuinfo、meminfo、mounts、self 等）。
- `os/src/fs/devfs/mod.rs#init_devfs`：初始化 `/dev` 设备节点。
- `os/src/fs/vfs/file.rs#open_file` / `os/src/fs/fs.rs#open_or_create_file`：通用文件打开与创建。
- `os/src/fs/ext4/fstype.rs#mount` / `inode.rs#create` / `cache_read_at`：ext4 挂载与页缓存读写。
- `os/src/fs/fat32/dentry.rs#load_child_dentry` / `inode.rs#lookup`：FAT32 目录加载。
- `os/src/syscall/fs.rs#sys_mount` / `sys_openat` / `sys_getdents64`：系统调用层文件操作。

### DEV（设备驱动）

设备驱动证据覆盖 DT 解析、MMIO/PCI 扫描、串口、块设备、网络设备。

- `os/src/devices/manager.rs#map_devices` / `init_net`：扫描 MMIO/PCIe，初始化 virtio 块/网设备。
- `os/src/devices/serial.rs#get_serial` / `scan_char_device`：串口初始化与 stdout-path 解析。
- `os/src/drivers/block/pci_blk.rs#new`：VirtIO-PCI 块设备。
- `os/src/drivers/block/mmc/mod.rs#card_init`：MMC/SD 卡初始化。
- `os/src/drivers/net/virtio_net.rs#receive` / `recycle_rx_buffer`：VirtIO-Net 接收与缓冲区回收。

### SYSCALL（系统调用）

存在统一的系统调用分发器，并按功能拆分到多个子模块。

- `os/src/syscall/mod.rs#syscall`：总入口，使用 `SyscallId::from_repr` 分派。
- `os/src/syscall/fs.rs`：文件相关 syscall（openat/read/write/mount/pread/pwrite/readv/writev 等）。
- `os/src/syscall/process.rs`：进程控制（clone/fork/execve/waitpid/exit/setuid 等）。
- `os/src/syscall/mm.rs`：内存相关 syscall（process_vm_readv/writev）。
- `os/src/syscall/io.rs`：epoll/pselect/ppoll/pipe/mq_open 等。
- `os/src/syscall/net.rs`：socket 相关 syscall。
- `os/src/syscall/futex.rs`：futex 同步原语。

### TRAP/IRQ（中断异常）

HAL 层对 LoongArch64 与 RISC-V64 均有支持，OS 层实现了用户态/内核态陷阱处理。

- `hal/src/component/irq/loongarch64/mod.rs#from_dt` / `enable_irq` / `disable_irq`：LoongArch 中断控制器初始化与开关。
- `hal/src/component/irq/loongarch64/eiointc.rs#claim_irq` / `init`：中断认领与寄存器配置。
- `hal/src/component/irq/riscv64.rs#enable_irq`：RISC-V PLIC 中断使能。
- `hal/src/component/trap/loongarch64/mod.rs#save` / `restore`：异常上下文保存/恢复。
- `os/src/trap/mod.rs#kernel_trap_handler` / `user_trap_handler`：OS 层陷阱处理，涉及 `IRQ_COUNTER`、`DEVICE_MANAGER`、`TIMER_MANAGER`。

### NET（网络协议栈）

网络协议栈证据较强，基于 `smoltcp` 实现了 TCP/UDP/RAW/socketpair，并包含包修改、监听表、AF_ALG 风格 crypto 接口。

- `os/src/net/mod.rs#init_network` / `modify_packet`：网络初始化与入包处理，调用 `LISTEN_TABLE.handle_coming_packet`。
- `os/src/net/tcp.rs#recv` / `send` / `shutdown`：TCP 读写与关闭。
- `os/src/net/udp.rs#send_to`：UDP 发送。
- `os/src/net/raw.rs#send`：RAW socket 发送。
- `os/src/net/socketpair.rs#poll`：本地 socketpair 轮询。
- `os/src/net/crypto.rs#check_alg` / `encode`：AF_ALG 风格算法接口。
