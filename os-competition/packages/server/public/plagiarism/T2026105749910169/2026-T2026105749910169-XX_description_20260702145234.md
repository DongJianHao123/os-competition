## 第四部分：分模块架构说明

### MM（内存管理）

证据较强。核心围绕用户态地址空间、ELF 加载、页表映射与权限控制展开。

- `src/main.rs#addrspace_build_image` / `addrspace_build_image_dynamic_la` 使用 `AddrSpace`、`MapArea`、`Prot` 以及 `RvSv39` / `LaPt` 分别为 RISC-V 与 LoongArch 构造用户页表。
- `src/main.rs#load_user_image_from_slice` 解析 ELF，提取 load segments、TLS segment 与指针重定位区间。
- `src/user.rs#prepare_initial_stack_args_inner` 与 `prepare_initial_stack_builds_argc_argv_for_start_main` 构造符合 Linux ABI 的初始栈（argc/argv/envp/auxv）。
- `src/user.rs#protect_enforces_translated_read_and_write_permissions` 与 `load_segment_applies_elf_execute_permissions` 体现 `PROT_READ/PROT_WRITE/PROT_EXEC` 权限检查。
- `src/main.rs#addrspace_handle_trap` / `handle_user_trap` 处理用户态陷入，包括预算定时器抢占与页相关异常。

**未体现**：内核物理页框分配器、内核堆分配器、swap、COW 等底层机制在现有证据中未充分展开。

### PROC（进程管理/任务调度）

证据强。`src/process.rs` 中的 `ProcessRuntime` 实现了传统进程管理模型。

- `src/process.rs#exit_current` 设置 `RuntimeProcessState::Zombie`，合并运行时文件，唤醒父进程。
- `src/process.rs#clone_current_with_budget` 创建子进程/线程，设置栈、TLS、返回值与运行预算。
- `src/process.rs#wait4_current` 支持 `wait4` 语义，区分线程与进程子代。
- `src/process.rs#pselect6_current` / `ppoll_current` 实现 IO 多路复用并在管道空/满时切换到对应进程。
- `src/process.rs#futex_wait_current_or_block` / `sendto_port_current` / `recv_current` / `connect_current` / `accept_current_or_block` 提供 futex、管道、socket 等阻塞/唤醒机制。
- `src/process.rs#find_runnable_sibling_or_parent` 与 `park_on_sleep_or_block` 体现基于进程槽的调度。

调度策略为带预算定时器的轮询/抢占式：证据见 `src/main.rs#handle_user_trap` 中的 `preempt_current_to_peer`、`wake_expired_sleepers` 与 `RUN_TICK_BUDGET.charge`。

### FS（文件系统）

证据较强，分为两层：

1. **ext4 只读解析层**（`src/fs/ext4.rs`）
   - `read_regular_file_bounded`：按 extent 读取常规文件。
   - `lookup_path`：路径查找。
   - `discover_scripts` / `visit_directory_with_warnings`：目录遍历与测试脚本发现。
   - `read_inode` / `read_group_descriptor` / `parse_directory_block`：超级块、inode、组描述符、目录项解析。

2. **运行时 VFS / procfs 层**（`src/syscall/mod.rs`）
   - `seed_runtime_procfs` 构造 `/proc`、`/proc/self/fd`、`/etc/passwd` 等运行时节点。
   - `RuntimeNode` 提供目录/文件节点、`find_node`、`read_data`、`write_file_data` 等操作。
   - `proc_fd_target` 实现 `/proc/self/fd/N` 解析。

**未体现**：ext4 写入、日志、缓存、VFS 挂载点栈等完整文件系统特性。

### DEV（设备驱动）

证据集中在 VirtIO 块设备，其他类型设备证据不足。

- `src/virtio/pci.rs#probe_loongarch_with_trace`：PCI 探测、MMIO BAR 分配、设备使能。
- `src/virtio/pci.rs#setup_queue`：VirtIO 队列配置。
- `src/virtio/pci.rs#submit_read` / `src/virtio/mmio.rs#submit_read`：提交读请求、构造 descriptor ring、处理 `RequestHeader`。
- `src/main.rs#run_on_block_device` 将 ext4 镜像与块设备关联，作为系统启动入口。

**未体现**：字符设备、网络设备、显示设备、中断控制器驱动等。

### SYSCALL（系统调用）

证据强，呈现 Linux 兼容 ABI。

- `src/main.rs#handle_user_syscall` / `addrspace_handle_syscall` 是系统调用总入口。
- `src/syscall/mod.rs#dispatch` 分发到具体实现，覆盖 `sys_brk`、`sys_chdir`、`sys_clock_gettime`、`sys_clock_nanosleep`、`sys_close`、`sys_openat`、`sys_write` 等。
- `src/syscall/file.rs#sys_openat` / `sys_readlinkat` 处理文件路径解析与 fd 分配。
- `src/syscall/io.rs#sys_write` 处理文件/控制台写入与输出预算。
- `src/syscall/time.rs#read_timespec_nanos` 处理 timespec 读取。
- `src/syscall/mod.rs#sys_fcntl_lock` / `sys_setxattr` / `sys_msgsnd` / `sys_msgrcv` 等体现对 fcntl、xattr、System V IPC 的支持。

### TRAP/IRQ（中断异常）

证据中等，主要为架构相关陷入向量。

- `src/arch/riscv64.rs#riscv_trap_handler` / `src/arch/loongarch64.rs#loongarch_trap_handler` 分发系统调用、异常、信号。
- `src/arch/riscv64.rs#enter_user` / `src/arch/loongarch64.rs#enter_user` 负责返回用户态。
- `src/main.rs#probe_budget_timer` / `handle_user_trap` 实现预算定时器中断与抢占。
- `src/main.rs#maybe_deliver_pending_signal` 在陷入返回前投递信号。

**未体现**：完整中断控制器初始化、多核 IPI、设备中断分发等。

### NET（网络协议栈）

**底层证据未体现**。仅有：

- `src/netperf.rs#render_netperf_script`
- `src/netperf.rs#write_u32`

二者只是生成测试脚本文本，未出现 TCP/IP、socket 协议栈、网络设备驱动等实现。`process.rs` 中的 `sendto/recv/connect/accept` 为本地进程间 socket 模拟，不构成本地网络协议栈。

### 需要说明的判断边界

- 本项目呈现的是“面向 LoongArch64 / RISC-V64 的轻量级操作系统内核或 Linux 用户态兼容运行时”的技术特征，但具体是裸机内核、半托管运行时还是用户态模拟器，仅凭 AST 调用证据无法最终定论。
- `src/process.rs` 中的大量长命名函数（如 `pselect_on_empty_pipe_switches_to_parent_until_writer_wakes_reader`）同时包含 `assert!`/`assert_eq!`，说明源码中实现与单元测试高度混合，统计的函数数包含测试函数。
- `src/fs/ext4.rs` 与 ELF 解析相关代码可能包含公共算法或第三方参考实现，当前证据不足以判断其原创性比例，仅能确认其功能边界。
- 本文档不做抄袭、违规或最终身份判定，仅基于 AST Call Map、函数名、路径锚点与调用链描述技术事实。
