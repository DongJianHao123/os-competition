## 第四部分：分模块架构说明

### MM（内存管理）
底层证据显示该内核实现了基于页表与 VMA 的内存管理：
- 缺页异常处理：`kernel/sched/trap.c#pgfault_handler` 调用 `find_vma`、`kzalloc`、`mappages`、`fp->fops->read`，支持文件映射的按需分页。
- 堆管理：`kernel/sysproc.c#sys_brk` / `sys_sbrk` 调用 `growproc`。
- mmap 系列：`kernel/mm/mmap.c#sys_munmap`、`sys_mprotect` 分别调用 `do_munmap`、`do_mprotect`。
- 内核堆：`kzalloc` / `kfree` 在多处被调用。
- `kernel/mm/vm.c` 是内存管理最密集文件之一（45 个函数）。

**未体现**：交换（swap）、NUMA、大页、内存压缩、COW 详细逻辑底层证据不足。

### PROC（进程管理/任务调度）
证据支持传统进程/线程模型，而非轻量任务或异步运行时：
- 创建：`kernel/sched/proc.c#do_clone` 被 `sys_fork`、`sys_clone` 调用，涉及 `alloc_thread`、`create_proc`、`filedup`、`copyout`、`append_child`。
- 退出：`kernel/sched/proc.c#proc_exit` 调用 `freeprocvm`、`fileclose`、`reparent`、`pcb_q_change_state`。
- 等待：`kernel/sysproc.c#sys_wait4` -> `waitpid` -> `wait_one`。
- 信号：`kernel/ipc/signal.c#signal_handle`、`do_sigaction`、`do_sigprocmask`、`do_sigtimedwait`。
- IPC：`kernel/ipc/futex.c`、`kernel/ipc/pipe.c`、`kernel/atomic/cond.c`。
- 调度：`kernel/sched/sched.c#thread_scheduler` 使用 `queue_pop_atomic`、`swtch`、`tcb_change2_running`。

**异步运行时**：底层证据未体现（无 `async/await`、`waker`、`ring_fifo`、`future` 等结构）。

### FS（文件系统）
文件系统是代码量最大的部分，分为三层：
- **VFS 层**：`kernel/fs/vfs/vfs.c#namex`、`iget`、`iput`；`kernel/fs/file.c#fileclose`；`kernel/fs/sysfile.c` 中的大量系统调用。
- **具体文件系统**：
  - **ext4**：`kernel/fs/ext4*.c` 系列（ext4.c、ext4_extent.c、ext4_journal.c、ext4_balloc.c、ext4_inode.c、ext4_fs.c 等）函数量占绝对优势，**极可能为第三方库（如 lwext4）移植**。
  - **xv6fs**：`kernel/fs/xv6fs.c#xv6fs_writei`。
  - **procfs**：`kernel/fs/procfs.c#procfs_render_file` 读取内存/时间信息。
- **块设备/缓存**：`binit`、`kernel/fs/virtio_disk.c`、`kernel/arch/loongarch/drv_ahci.c`。

### DEV（设备驱动）
- **LoongArch 平台**：`kernel/arch/loongarch/drv_ahci.c`（AHCI SATA）、`kernel/arch/loongarch/pci.c`（PCI 枚举）、`kernel/arch/loongarch/extioi.c`（扩展 IO 中断）、`kernel/arch/loongarch/smoldtb.c`（设备树解析）。
- **QEMU/RISC-V 平台**：`kernel/fs/virtio_disk.c`（virtio-blk）、`kernel/arch/qemu/plic.c`（PLIC 中断控制器）、`kernel/arch/qemu/uart.c`（串口）、`kernel/arch/qemu/console.c`。
- 初始化入口：`kernel/init/main.c#disk_init` -> `ahci_init` / `pci_device_dump`。

### SYSCALL（系统调用）
- 参数解析基础设施：`kernel/syscall.c#argstr`、`arglong`、`argulong`、`argint`、`argraw`、`fetchstr`。
- 文件类系统调用集中在 `kernel/fs/sysfile.c`：`sys_openat`、`sys_pread64`、`sys_lseek`、`sys_chdir`、`sys_renameat2`、`sys_faccessat`、`generic_utimensat`。
- 进程类系统调用集中在 `kernel/sysproc.c`：`sys_fork`、`sys_clone`、`sys_exit`、`sys_wait4`、`sys_sleep`、`sys_brk`、`sys_sbrk`、`sys_kill`、`sys_tgkill`。
- 内存类：`kernel/mm/mmap.c#sys_munmap`、`sys_mprotect`。
- 其他：`kernel/sysother.c#sys_poweroff`。

### TRAP/IRQ（中断异常）
- 核心入口：`kernel/sched/trap.c#usertrap`、`kerneltrap`、`usertrapret`、`pgfault_handler`。
- 中断控制器：`kernel/arch/loongarch/extioi.c#extioi_init`、`kernel/arch/qemu/plic.c#plicinithart` / `plic_claim` / `plic_complete`。
- 中断开关：`kernel/atomic/spinlock.c#push_off` / `pop_off`。
- 设备中断：`kernel/fs/virtio_disk.c#virtio_disk_intr` 唤醒等待线程。

### NET（网络协议栈）
**底层证据未体现**。未出现网络设备驱动、套接字系统调用、TCP/IP 协议栈相关函数或目录。

4. **安全/权限模块**：无 ACL、Capability、SELinux/AppArmor、沙箱等证据。
5. **虚拟化/容器**：无 hypervisor、KVM、namespace、cgroup 相关证据。
6. **实时调度**：无 RT-scheduler、EDF、优先级继承等证据。

### 需要单独说明的第三方/硬件约束代码

- AHCI、PCI、PLIC、Virtio 等驱动包含大量硬件寄存器协议约束代码。

### 判断边界
- 本作品“可能的系统方向”仅基于 AST Call Map、函数名、路径锚点与调用链推断，**不是对项目最终身份、原创性或参赛资格的结论**。
