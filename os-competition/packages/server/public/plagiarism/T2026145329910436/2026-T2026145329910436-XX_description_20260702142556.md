## 第四部分：分模块架构说明

### MM（内存管理）

有较强证据支持用户态内存管理：
- `kernel/src/syscall/mm/mmap.rs#sys_mmap` / `sys_mprotect` 实现了 `mmap`/`mprotect`，支持 `Backend::new_alloc`、`Backend::new_cow`、`Backend::new_file`、`Backend::new_linear`、`Backend::new_shared` 等多种后端。
- `kernel/src/mm/aspace/backend/cow.rs#alloc_new_at` 实现了写时复制（COW）后端，调用 `file.read_at`、`phys_to_virt` 等完成文件页填充。
- `kernel/src/mm/loader.rs#load_user_app` / `load` 负责 ELF 可执行文件加载与解释器解析。
- `kernel/src/syscall/ipc/shm.rs#sys_shmat` / `sys_shmdt` 实现了 SysV 共享内存，使用 `Backend::new_shared` 与地址空间 `aspace.unmap`。
- `kernel/src/mm/access.rs` 为用户态指针访问辅助函数密集区（`vm_read`/`vm_write`）。

**未体现**：页错误/缺页异常处理器的具体实现、物理页分配器（buddy/slab）本体、swap/换出机制。

### PROC（进程管理/任务调度）

进程管理证据较完整：
- `kernel/src/syscall/task/clone.rs#do_clone` 与 `kernel/src/syscall/task/clone3.rs#sys_clone3` 实现 `clone`/`fork`。
- `kernel/src/syscall/task/execve.rs#sys_execve` 实现程序映像替换。
- `kernel/src/task/ops.rs#do_exit` 实现线程/进程退出，清理 futex、robust list、共享内存、信号。
- `kernel/src/syscall/task/wait.rs#sys_waitpid` 实现 `waitpid`/`wait4`。
- `kernel/src/syscall/fs/pidfd.rs#sys_pidfd_open` / `sys_pidfd_getfd` / `sys_pidfd_send_signal` 实现 pidfd 机制。
- `kernel/src/syscall/signal.rs` 与 `kernel/src/task/signal.rs` 实现信号投递、进程组信号、致命信号处理。
- `kernel/src/pseudofs/proc.rs#lookup_child` / `child_names` / `new_procfs` 暴露 `/proc` 接口（`stat`、`status`、`maps`、`mounts`、`cmdline`、`comm`、`exe`、`fd`、`task` 等）。
- `kernel/src/task/ops.rs#get_process_data` / `get_process_group` / `processes` / `add_task_to_table` 维护 `PROCESS_TABLE`、`PROCESS_GROUP_TABLE`、`SESSION_TABLE`。

**调度核心**：代码中大量调用 `axtask::yield_now`、`axtask::spawn_with_name`、`axtask::set_current_affinity`，但 `axtask` 调度器本体不在当前统计的 `kernel/src` 中，**底层调度器实现证据未体现**。

### FS（文件系统）

文件系统是最厚重的模块：
- **VFS 层**：`patches/axfs-ng/src/highlevel/file.rs#open` 提供路径解析、符号链接跟随、打开逻辑。
- **ext4**：`patches/axfs-ng/src/fs/ext4/inode.rs` / `fs.rs` 实现 ext4 的 inode 创建、目录读取、元数据更新、硬链接、追加写等；属于第三方补丁库。
- **伪文件系统**：
  - `kernel/src/pseudofs/tmp.rs#new`：tmpfs
  - `kernel/src/pseudofs/proc.rs`：procfs
  - `kernel/src/pseudofs/dev/mod.rs#builder`：devfs（包含 random、tty、event、display 等）
- **系统调用层**：`kernel/src/syscall/fs/*` 覆盖 `openat`、`mkdirat`、`readlinkat`、`mount`、`splice`、`fallocate`、`truncate`、`statx`、`getcwd`、`fcntl`、`close_range`、`memfd_create` 等。
- **特殊文件**：`kernel/src/file/epoll.rs`、`kernel/src/file/signalfd.rs`、`kernel/src/file/pidfd.rs` 实现 epoll、signalfd、pidfd。

**未体现**：日志型文件系统恢复、磁盘配额、ACL、原生块设备驱动。

### DEV（设备驱动）

设备相关代码主要集中在伪文件系统层：
- TTY/PTY：`kernel/src/pseudofs/dev/tty/mod.rs#ioctl`、`kernel/src/pseudofs/dev/tty/terminal/ldisc.rs`、`kernel/src/pseudofs/dev/tty/pty.rs#create_pty_pair`、`kernel/src/pseudofs/dev/tty/terminal/job.rs#set_foreground`。
- 输入事件设备：`kernel/src/pseudofs/dev/event.rs#ioctl`。
- 回环块设备：`kernel/src/pseudofs/dev/loop.rs#ioctl`。
- 日志套接字：`kernel/src/pseudofs/dev/log.rs#bind_dev_log`。
- 随机数/显示：`kernel/src/pseudofs/dev/mod.rs#builder`。
- 控制台：`kernel/src/file/mod.rs#add_stdio` 打开 `/dev/console`；`tools/cargo-axplat/template/src/console.rs#handle_irq` 处理控制台 IRQ 事件。

**未体现**：原生块设备驱动、原生网络设备驱动、PCI/USB/ACPI 驱动、硬件定时器中断驱动。

### SYSCALL（系统调用）

- 统一入口：`kernel/src/syscall/mod.rs#handle_syscall`，使用 `Sysno` 与 `LinuxError` 进行分发。
- 覆盖范围：文件系统、内存、进程、信号、futex、IPC（msg/shm）、网络 socket、epoll/select/poll、time、resource limit、prctl、sched affinity 等。
- 大量 `vm_read`/`vm_write` 调用说明实现了用户态/内核态数据拷贝。

### TRAP/IRQ（中断异常）

**底层证据非常弱**：
- 仅见 `tools/cargo-axplat/template/src/irq.rs#register` / `unregister` / `send_ipi` 三个 `todo!` 占位函数。
- `tools/cargo-axplat/template/src/console.rs#handle_irq` 为平台模板层控制台 IRQ 处理。
- `kernel/src/file/signalfd.rs#from_signal_info` 与 `kernel/src/pseudofs/dev/tty/terminal/termios.rs#signo_for` 属于信号/termios，不是中断异常处理。

**结论**：内核内部的中断向量表、异常分发、页错误处理、系统调用陷入/返回等**底层证据未体现**。

### NET（网络协议栈）

- 系统调用层证据：`kernel/src/syscall/net/socket.rs#sys_socket` / `sys_accept4` / `sys_socketpair`，`kernel/src/syscall/net/io.rs#sys_sendmsg` / `recv_impl`，`kernel/src/syscall/net/cmsg.rs#push`，`kernel/src/syscall/net/addr.rs#read_from_user`。
- 抽象：`SocketInner::Tcp`、`SocketInner::Udp`、`SocketInner::Unix`、`SocketInner::Vsock`、`StreamTransport`、`DgramTransport`、`TcpSocket`、`UnixSocket`。
- Unix 域套接字：`kernel/src/pseudofs/dev/log.rs#bind_dev_log` 使用 `UnixSocket::new`。

**结论**：仅见 socket 系统调用封装与传输抽象；TCP/IP 协议栈、UDP 协议栈、网络设备驱动、路由/ARP 等**底层证据未体现**。

## 第六部分：边界与未体现内容

### 没有底层证据支撑的模块

1. **内核级中断/异常处理**：仅见 `tools/cargo-axplat/template/src/irq.rs` 的 `todo!` 占位与控制台 IRQ 模板；`kernel/src` 中未见中断向量表、异常分发、系统调用陷入/返回、页错误处理器的具体实现。
2. **任务调度器核心**：代码大量调用 `axtask::*`，但 `axtask` 调度器本体不在当前仓库统计范围内，**调度算法、上下文切换、时间片管理底层证据未体现**。
3. **原生块设备驱动**：ext4 通过 `patches/axfs-ng` 实现，但底层块设备（如 virtio-blk、AHCI、NVMe）驱动**未体现**。
4. **原生网络协议栈与网络设备驱动**：仅见 socket 系统调用层与 `StreamTransport`/`DgramTransport` 抽象，TCP/IP、UDP、ARP、路由、网卡驱动**未体现**。
5. **硬件定时器/时钟中断驱动**：`kernel/src/task/timer.rs` 使用 `ALARM_LIST` 与 `monotonic_time_nanos`，但底层时钟中断源与定时器硬件驱动**未体现**。
6. **SMP 多核 IPI**：`tools/cargo-axplat/template/src/irq.rs#send_ipi` 为 `todo!`，多核调度、核间中断**未体现**。
7. **电源管理、安全模块/LSM、虚拟化、声卡、GPU 驱动、文件系统日志恢复、磁盘配额、ACL** 等**未体现**。

### 需要单独说明的内容

- **`patches/axfs-ng`** 属于第三方文件系统库的补丁/修改版本，其 ext4 实现不应被夸大为本项目的原创内核主体。
- **`tools/axconfig-gen`** 与 **`tools/cargo-axplat`** 是构建配置与平台适配工具，不属于运行时内核模块。
- 模块统计中将大量文件系统/网络/IPC 系统调用路径归入“任务调度/异步运行时”，是因为这些路径中使用了 `block_on`、`Waker`、`PollSet`、`axtask::*` 等异步运行时原语；这反映的是**异步运行时集成程度**，而非这些模块本身是调度器。
- 本文档仅基于 AST Call Map、函数名、路径锚点与调用链描述技术事实；关于项目原创性、代码来源合规性、是否抄袭等不在本文判断范围内。
